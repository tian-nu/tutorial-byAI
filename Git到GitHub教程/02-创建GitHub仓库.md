# 第2章 · 亲手创建你的第一个GitHub仓库——给代码安个"云上的家"

> "到现在为止，你有了一台装好 Git 的电脑，有了一个 GitHub 账号，还配好了 SSH 钥匙。现在的问题变成了：代码要从你的电脑推到哪去？"
>
> "就像搬家——你有行李（代码），也有交通工具（Git），但目的地还没定。你需要一个'地址'——一个在 GitHub 上属于你的空间，专门存放你这个项目的代码。"
>
> "这个空间，就叫**仓库（Repository）**。这一章，我们就在 GitHub 网站上点几下鼠标，亲手把这个'云上的家'建好。"

---

> **📌 占位符说明（本章和后续所有章节通用）**
>
> 在接下来的教程中，你会反复看到两个占位符：
> - `<你的用户名>`：请替换成你自己的 GitHub 用户名。比如你的用户名是 `zhangsan`，那就把 `<你的用户名>` 看成 `zhangsan`。
> - `<你的仓库名>`：请替换成你自己起的仓库名。本章默认用 `my-first-project`。如果你起了别的名字，后续所有出现 `my-first-project` 的地方都要换成你自己的名字。
>
> 比如：`github.com/<你的用户名>/my-first-project` → 实际就是 `github.com/zhangsan/my-first-project`。

---

## 2.1 在 GitHub 网页创建仓库——点击、填写、创建

### 2.1.1 我在做什么？

在 GitHub 网站上创建一个完全空白的仓库（Repository，以下简称 repo）。此术语需进附录。这个仓库会成为你本地代码在互联网上的"家"——你以后写的代码就推送到这里。

仓库这个词很好懂：就像现实中的仓库一样，一个 GitHub 仓库就是一个"存放代码和相关文件的空间"。每个仓库有自己的地址，有自己的访问权限控制（谁能看、谁能改），有自己的历史记录。

---

### 2.1.2 我做得对不对？

创建完仓库后，你应该看到：
- [ ] 浏览器地址栏显示 `github.com/<你的用户名>/my-first-project`
- [ ] 页面没有文件列表（空白状态）
- [ ] 页面中央有 "Quick setup" 引导信息

---

### 2.1.3 开始创建——从 GitHub 首页出发

**第1步：打开 GitHub 首页。**

打开你的浏览器，在地址栏输入 `github.com`，按 **Enter 键**。确保你已经登录了 GitHub——页面右上角应该能看到你的头像（一个圆形图标）。如果没有，点击右上角的 **"Sign in"** 按钮登录。

**第2步：找到创建仓库的入口。**

登录后，看页面右上角。在你的头像左边，有一个 **`+` 号**图标。用鼠标左键点击它。

> 📺 点击后会弹出一个下拉菜单，菜单里有几个选项。第一个选项是 **"New repository"**（新建仓库）。

用鼠标左键点击 **"New repository"**。

> 页面跳转到了一个新的页面。地址栏显示的是 `github.com/new`。这就是创建仓库的表单页面。

---

### 2.1.4 填写创建仓库的表单

这个页面从上到下有多个区域需要填写或选择。我们来逐一过一遍。

**① Owner（所有者）**

页面最上方有一个下拉框，旁边写着 "Owner"。默认选中的是你自己（你的 GitHub 用户名）。**不用改它**——仓库就创建在你的账号下。

**② Repository name（仓库名）**

Owner 下拉框下面有一个白色的输入框，旁边写着 "Repository name"（仓库名）。输入框里可能有灰色的提示文字。

用鼠标点击这个输入框，光标会在里面闪烁。输入：

```
my-first-project
```

这个名字的意思就是一个英文短语："我的第一个项目"。

> 📺 输入后，输入框右边可能会出现一个绿色的对勾 ✅，表示这个仓库名没有被占用（可以用）。
>
> 如果出现红色的叉号 ❌ 和提示文字，说明这个名字已经被别人用了。换一个名字，比如在末尾加上数字：`my-first-project-2024`、`my-first-project-001` 等等。起好新名字后，本章所有后续提到 `my-first-project` 的地方，都用你实际起的名字代替。

> ⚠️ **仓库名只能用英文、数字和连字符（-），不能用中文、空格或特殊符号。** 这就像给文件起名——英文和数字是最安全的。

**③ Description（描述，可选）**

Repository name 下面有一个输入框："Description (optional)"。这一项是**选填的**，你可以填也可以不填。

如果你填了，这段文字会出现在仓库页面的右侧（别人点进你的仓库时会看到）。现在可以填一句简单的话，比如：

```
我的第一个 GitHub 项目——跟着教程学习 Git 和 GitHub
```

现在不填也完全没关系，以后随时可以补。

**④ Public 还是 Private（公开还是私有）**

Description 下面有两个单选按钮：
- **Public**（公开）：互联网上的**任何人都能看到**这个仓库里的代码。
- **Private**（私有）：只有**你自己**和**你邀请的人**能看到。

用鼠标点击 **Public** 前面的小圆圈。选它——新手推荐选 Public，因为：
- 公开仓库**完全免费，不限功能**
- 私有仓库虽然也能用，但后面学 GitHub Pages 和 Actions 时，公开仓库的免费额度更充足
- 你现在写的代码也不涉及什么机密，公开完全没问题

> 🤔 **想多一点**：如果你以后想改仓库的可见性，随时可以在仓库的 Settings 里切换。所以现在选 Public，不代表"永远公开"——只是"现在公开"。

**⑤ 三个初始化复选框——全部不勾选！**

Public/Private 下面有一排三个复选框。**这三个全部不勾选！**（如果默认已经勾选了其中某个，用鼠标点击它取消勾选。）

我们来解释一下这三个复选框各自是干什么的：

- **"Add a README file"**（添加 README 文件）：README 是项目的"自我介绍"文件，GitHub 会自动生成一个简单的 README。听起来不错，但**这会往仓库里添加第一个提交（commit）**——仓库不再是空的了。如果仓库不空，等一下第3章你就会遇到麻烦。我们会在第3章自己手动创建 README，所以**不勾选**。
  
- **"Add .gitignore"**（添加 .gitignore 文件）：`.gitignore` 是一个配置，告诉 Git "哪些文件不要管"。比如你写 Go 代码，编译出来的 `.exe` 文件就不需要上传到 GitHub。这个非常有用，但**我们第3章会自己手动创建**，所以也**不勾选**。

- **"Choose a license"**（选择许可证）：许可证是一份法律文件，告诉别人"你可以怎么用我的代码"。常见的许可证有 MIT（最宽松，别人几乎可以做任何事）、GPL（用了你的代码就必须也开源）、Apache 2.0（类似 MIT 但有专利保护）。**现在选还太早**——等你以后真的想发布一个开源项目时再选。所以**不勾选**。

> ⚠️ **重要：三个复选框确认全部不勾选。** 如果你勾了其中任何一个，创建出来的仓库就会有"初始内容"——这会让你在第3章多走很多弯路。

**⑥ 创建仓库。**

三个复选框确认全部不勾选之后，页面拉到最下方。你会看到一个**绿色的大按钮**，上面写着 **"Create repository"**（创建仓库）。

用鼠标左键点击它。

---

### 2.1.5 创建成功后——你看到的页面

点击 "Create repository" 后，页面会短暂加载（地址栏旁边可能有一个小圆圈在转），然后跳转到你的仓库页面。

> 📺 因为你三个复选框全没勾选，创建出来的是一个**完全空白的仓库**。你看到的页面应该和下面的描述一致：
>
> - 页面左上角显示 `<你的用户名> / my-first-project`（旁边可能有一个 "Public" 的小标签，表示这是公开仓库）
> - 页面中央偏上有一个大标题，可能是 **"Quick setup"**——意思是"快速设置"。标题下面有一行灰色小字："if you've done this kind of thing before"（如果你以前做过这种事的话）
> - 标题下方有一条横着的地址栏，里面显示着 `git@github.com:<你的用户名>/my-first-project.git`——这是你仓库的 SSH 地址。等一下第3章会用到它。
> - 地址栏右边有一个灰色的复制按钮（两个方块叠在一起的图标）
> - 再往下是一大片命令行指引，分为 `…or create a new repository on the command line` 和 `…or push an existing repository from the command line` 两种。这些指引告诉你"新手怎么把代码弄上来"——但我们有教程，不用看它。
> - 页面中间的文件列表区域是**空的**——没有文件，可能只显示一行字 "Your repository is empty" 或者一片空白。

**✅ 恭喜你！你的第一个 GitHub 仓库创建成功了！**

---

### 2.1.6 检查你创建出来的是什么状态

> **场景一：你看到的和 2.1.5 描述的一样——文件列表是空的**

这是**状态A（完全空白）**。太好了！这就是我们想要的。直接跳到 2.2 节继续。

> **场景二：你看到文件列表里有文件（比如 `README.md`）**

这说明你创建仓库时不小心勾选了某个复选框。你处于**状态B**或**状态C**。

你有两个选择：

**选择A（推荐）：删除这个仓库，重新创建一个。**

删除方法：
1. 在仓库页面，看顶部标签栏，最右边有一个 **"Settings"** 标签（齿轮图标 ⚙️）。点击它。
2. 页面加载后，看左边那一长列菜单。用鼠标滚轮一直往下滚，滚到最底部。
3. 最底部有一个红色的区域，标题是 **"Danger Zone"**（危险区域）。在这个区域里找到 **"Delete this repository"**（删除这个仓库）这一行。
4. 点击 **"Delete this repository"** 按钮。GitHub 会弹出一个确认框。
5. 确认框里要求你输入仓库的全名来确认删除。输入：`<你的用户名>/my-first-project`（把你自己的用户名和仓库名替换进去），然后点击确认按钮。
6. 仓库被删除了，页面跳转回 GitHub 首页。重新从 2.1.3 节开始，**这次一定注意三个复选框全不勾选**。

**选择B：接受这个带文件的仓库，继续往下走。**

如果你不想删，也可以。但到了第3章，你的操作流程会不同——你需要先把远程仓库下载（clone）到本地，再添加文件，再推送。操作会比教程描述的多几步。如果你愿意接受这个挑战，跳到 2.2 节了解你当前的状态，然后继续。

---

### 2.1.7 不对怎么办？

❌ **错误：仓库名称被占用（创建时输入框下面有红色提示）**

换一个名字。在原来的名字后面加数字：`my-first-project-2024`、`learning-git-001` 等等。起好新名字后，本章所有后续提到 `my-first-project` 的地方，都用你实际起的名字代替。

❌ **错误：页面显示 404 或报错**

可能是你还没有登录。检查页面右上角是否有你的头像。如果没有，点击 "Sign in" 登录后，在地址栏输入 `github.com/<你的用户名>/my-first-project` 访问你的仓库。

---

## 2.2 三种仓库初始状态——理解你在哪里

新仓库创建之后，根据你是否勾选了初始化复选框，会呈现三种完全不同的页面状态。**理解这三种状态，能让你在第3章不走弯路。**

### 2.2.1 状态A：完全空白——你现在在这里

**你是怎么到的：** 三个复选框全不勾选 → 点击 "Create repository"。

> 📺 **你看到的页面特征：**
>
> - 页面中央有 "Quick setup" 引导区域
> - 有 SSH 地址：`git@github.com:<你的用户名>/my-first-project.git`
> - 下面有 "…or push an existing repository from the command line" 的指引
> - **没有任何文件列表**——文件区域是空的，或者显示 "Your repository is empty"

**下一步该做什么（第3章会详细讲）：** 在你的电脑上本地初始化 Git 项目，然后关联这个远程仓库，把代码推送（push）上来。流程是：本地 `git init` → `git remote add` → `git push`。

👇 这就是你现在的状态。你的操作路径最直接、最简单。

---

### 2.2.2 状态B：带一个 README 文件

**你是怎么到的：** 只勾选了 "Add a README file" → 点击 "Create repository"。

> 📺 **你看到的页面特征：**
>
> - 页面文件列表里有一个 `README.md` 文件
> - 文件列表下方是 README 的渲染内容（一般是仓库名）
> - **没有** "Quick setup" 引导区域
> - GitHub 已经帮你做了第一次提交（initial commit）——远程仓库有内容了

**下一步该做什么：** 你必须先把远程仓库下载（clone）到本地，在下载下来的文件夹里添加你的代码，然后再推送上去。不能直接本地 `git init` 然后 `git push`——因为远程有东西而本地什么都没有，会冲突。

---

### 2.2.3 状态C：完整的初始模板

**你是怎么到的：** 三个复选框全勾选或勾选了其中任意两个。

> 📺 **你看到的页面特征：**
>
> - 文件列表里有 `README.md`、`.gitignore`、`LICENSE` 三个文件（或其中的两个）
> - README 内容在文件列表下方渲染展示
> - GitHub 已经做了第一次提交

**下一步该做什么：** 与状态B相同——必须先 clone 再 push。

---

### 2.2.4 三者对比表

| | 状态A（你现在在） | 状态B | 状态C |
|---|---|---|---|
| 远程有初始提交吗？ | ❌ 没有 | ✅ 有 | ✅ 有 |
| 文件列表里有什么？ | 空的 | `README.md` | `README.md` + `.gitignore` + `LICENSE` |
| 本地怎么开始？ | 本地 init → push | clone → 加代码 → push | clone → 加代码 → push |
| 适合什么情况？ | 你本地已经有了代码，想推到网上 | 你还没开始写代码，想先在 GitHub 建仓库 | 你想一次性配齐仓库的所有配置 |

> ⚠️ 本教程假设你处于**状态A**。如果你不小心创建了状态B或C，建议参考 2.1.6 节的选择A——删除重建，选状态A。这会让第3章的学习顺利很多。

---

## 2.3 仓库页面导览——认识你仓库里的每一个"房间"

现在你正坐在自己仓库的"客厅"里（状态A的那个空白页面）。别急着走，花五分钟认识一下这个页面的每一个角落。这会让你以后操作时不再"迷路"。

### 2.3.1 页面顶部导航栏

在页面最顶部，有一行黑色的导航栏。从左到右依次是：

- **GitHub 图标**（最左边，一个猫脸章鱼身的图案——Octocat）：点击它会回到 GitHub 首页。
- **搜索框**（中间偏左）：输入关键词可以搜索仓库、代码、用户。
- **各种功能链接**（中间偏右）：Pull requests、Issues、Marketplace、Explore 等。
- **你的头像**（最右边，圆形）：点击会弹出菜单，里面有 "Your repositories"（你的仓库列表）、"Settings"（账号设置）等。

> 现在不用太关注这些。以后用到的时候我们再回来找。

---

### 2.3.2 仓库信息栏

在导航栏下方，是仓库的信息栏。从左到右：

- **仓库路径**：`<你的用户名> / my-first-project`，旁边有一个 "Public" 标签（灰色小圆角矩形）。这个路径告诉你"这是谁的仓库 / 叫什么名字"。
- **Pin 按钮**（图钉图标）：把你的仓库"钉"在列表顶部。用不到。
- **Watch 按钮**（眼睛图标）：关注这个仓库，有人发 Issue 或 PR 时通知你。点一下可以切换关注程度。
- **Fork 按钮**（叉子图标）：把别人的仓库复制一份到你自己名下。我们第6章会学这个。
- **Star 按钮**（星星图标）：给仓库点赞——表示你喜欢这个项目。Star 数越多，项目越受欢迎。

---

### 2.3.3 标签栏

在仓库信息栏下面，有一排标签（Tab）：

| 标签 | 英文 | 用途 |
|------|------|------|
| 代码 | **Code** | 查看文件列表、复制仓库地址。**你现在就在这个标签下。** |
| 问题 | **Issues** | Bug 追踪和功能讨论。第5章会学。 |
| 合并请求 | **Pull requests** | 代码审查和合并。第4章会学。 |
| 自动化 | **Actions** | 自动化流水线（测试、部署）。第7章会学。 |
| 项目 | **Projects** | 看板式的项目管理工具。 |
| 百科 | **Wiki** | 项目文档。 |
| 安全 | **Security** | 安全漏洞提醒。 |
| 洞察 | **Insights** | 项目统计（流量、贡献者等）。 |
| 设置 | **Settings** | 仓库设置。第1章加 SSH 密钥时用过，第8章 Pages 也会用到。 |

> 你现在在 **Code** 标签下（它是高亮显示的，一般是加粗或者有下划线）。其他标签目前不用管，以后用到的时候我们再来。

---

### 2.3.4 最关键的按钮：绿色的 "Code" 按钮

在文件列表区域的上方，有一个**绿色的按钮**，上面写着 **"▾ Code"**（注意右边有一个小三角）。这是整个仓库页面最重要的按钮之一。

用鼠标左键点击它。

> 📺 点击后会弹出一个面板，面板上方有三个标签（Tab）：
> - **HTTPS**：用 HTTPS 地址连接仓库（每次 push 都要输密码，不推荐）
> - **SSH**：用 SSH 地址连接仓库（配置一次就永久免密，我们第1章已经配好了）
> - **GitHub CLI**：用 GitHub 命令行工具（需要另外安装）
>
> 用鼠标点击 **SSH** 标签。面板里会显示一个地址，类似：
>
> ```
> git@github.com:<你的用户名>/my-first-project.git
> ```
>
> 这个地址就是你仓库的 SSH 克隆地址——第3章会用到它。地址右边有一个小按钮（两个方块叠在一起），点击它可以复制这个地址。

---

### 2.3.5 文件列表区域

在 Code 标签下方，就是文件列表区域。因为你的仓库是空的（状态A），这个区域目前是空白的，可能显示 "Your repository is empty"。

在文件列表区域的右上角，有几个小按钮：
- **"Add file"** 按钮：可以在这里创建新文件或上传文件。但我们不从这里创建——我们会在第3章从本地电脑推送代码上来。
- **"Go to file"** 按钮：搜索文件。仓库文件多了以后用。

> [可暂停点 1/1] 如果你累了，可以在这里休息。**恢复时**：打开浏览器，访问 `github.com/<你的用户名>/my-first-project`，从第3章开始即可。

| 三个复选框 | **全不勾选！** 创建完全空白仓库，方便第3章直接推送 |
| 创建按钮 | 页面最下方绿色 "Create repository" 按钮——**这次是真的要点！** |
| 创建后检查 | 页面没有文件列表 = 成功（状态A）。有文件列表 = 勾了复选框，建议删除重建 |
| 状态A（你在的） | 完全空白："Quick setup" 页面，无文件列表 → 直接本地 init → push |

| 状态C | 完整模板：B + `.gitignore` + `LICENSE`，需先 clone 再 push |
