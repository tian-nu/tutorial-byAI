# 第5章 · 终端与命令行入门

> "为什么后端工程师天天对着一个黑窗口？服务器上又没有桌面，你怎么操作它？cd、ls、rm 这些都是什么？"

---

## 5.1 终端是什么

### GUI 和 CLI

你熟悉的操作电脑方式是**GUI**（Graphical User Interface，图形用户界面）：
- 双击图标打开程序
- 用鼠标拖放文件
- 右键菜单做各种操作

但还有另一种方式——**CLI**（Command Line Interface，命令行界面）：
- 在一个黑窗口里打字
- 输入命令、回车
- 电脑执行并返回结果

这就是**终端**（Terminal）——一个让你和操作系统"打字聊天"的窗口。

### 为什么后端工程师必须用命令行

**原因一：服务器没有图形界面。**

你租的云服务器就是一台远程的电脑，但99%的情况下它没有装Windows桌面，也不会给你连显示器。你只能通过命令行远程登录操作它。

你不可能用鼠标去服务器上双击启动程序吧？服务器可能在美国，你的鼠标线有那么长吗？

**原因二：命令行效率更高。**

```bash
# 命令行操作：1秒完成
mv *.log /archive/

# GUI操作：打开文件夹 → 选中所有.log文件 → Ctrl+X → 打开archive → Ctrl+V
# 至少10秒
```

**原因三：自动化必须用命令行。**

如果你想每天凌晨3点自动备份数据库，你写一个Shell脚本（就是一系列命令的集合），让操作系统定时执行。GUI没法自动化。

### 打开终端

- **Windows**：Win+R → 输入 `cmd` 或 `powershell`（推荐用PowerShell）
- **Mac**：打开"终端"（Terminal.app）
- **Linux**：Ctrl+Alt+T

打开之后你看到一个黑窗口，上面有一行提示符（prompt）：

```
PS C:\Users\zhangsan>        ← Windows PowerShell
zhangsan@server:~$           ← Linux
```

`$` 或 `>` 是**提示符**，表示系统在等你的命令。闪烁的光标在等你打字。

## 5.2 基础命令（每个都配示例）

以下命令以Linux/Mac为主，Windows PowerShell稍有不同但大体相似。

### pwd —— 我在哪？

**p**rint **w**orking **d**irectory

```bash
$ pwd
/home/zhangsan
```

显示当前所在的目录。进终端后第一件事往往就是 `pwd` 看看自己在哪。

### ls —— 这里有什么？

**l**i**s**t

```bash
$ ls
Desktop  Documents  Downloads  Music  Pictures

$ ls -l            # 详细信息（权限、大小、修改时间）
-rw-r--r-- 1 zhangsan staff 1024 May 17 10:30 hello.go
drwxr-xr-x 2 zhangsan staff   64 May 15 14:22 projects

$ ls -a            # 显示隐藏文件（以.开头的文件）
.bashrc  .gitignore  .vscode

$ ls -la           # 两者结合
```

`-l`、`-a` 这些叫**选项**（options/flags），用于修改命令的行为。

### cd —— 换个地方

**c**hange **d**irectory

```bash
$ cd /home/zhangsan/projects   # 去指定目录（绝对路径）
$ cd projects                  # 去子目录（相对路径）
$ cd ..                        # 上一级
$ cd ../..                     # 上两级
$ cd ~                         # 回家目录（~代表家目录）
$ cd -                         # 回到上一个目录
$ cd                           # 直接cd也是回家目录
```

### mkdir —— 创建文件夹

**m**a**k**e **dir**ectory

```bash
$ mkdir my_new_project      # 创建一个文件夹
$ mkdir -p a/b/c            # 递归创建（a里面建b，b里面建c）
```

### touch —— 创建空文件

```bash
$ touch README.md           # 创建空的README.md
$ touch main.go util.go     # 一次创建多个
```

### rm —— 删除（小心！）

**r**e**m**ove

```bash
$ rm file.txt               # 删除文件
$ rm -r my_folder           # 删除文件夹（递归）
$ rm -rf my_folder          # 强制删除，不提示确认
```

> ⚠️ **`rm -rf` 非常危险！** 永远不要运行 `rm -rf /`（这会尝试删除整个系统）。也不要在不确定的时候用 `-rf`。

### cp —— 复制

**c**o**p**y

```bash
$ cp source.txt dest.txt        # 复制文件
$ cp -r source_dir dest_dir     # 复制文件夹（递归）
```

### mv —— 移动（剪切/重命名）

**m**o**v**e

```bash
$ mv old_name.txt new_name.txt   # 重命名
$ mv file.txt ../folder/         # 移动文件
```

`mv` 既管重命名又管移动——原理一样：改变文件的位置（名字也是位置的一部分）。

### cat —— 看文件内容（小文件）

**con**c**at**enate

```bash
$ cat hello.go
package main
import "fmt"
func main() {
    fmt.Println("Hello")
}
```

### less —— 看大文件（可分页）

```bash
$ less huge_log_file.log
# 按空格翻页，按q退出，按/搜索
```

### head / tail —— 看头看尾

```bash
$ head -n 20 log.txt     # 看前20行
$ tail -n 20 log.txt     # 看后20行
$ tail -f log.txt        # 实时追踪（日志不断追加，你不断看到新内容）
                         # 看日志的神器！Ctrl+C退出
```

### grep —— 搜索文本

**g**lobal **r**egular **e**xpression **p**rint

```bash
$ grep "error" log.txt           # 在log.txt中找包含"error"的行
$ grep -i "error" log.txt        # 忽略大小写
$ grep -r "TODO" ./project/      # 递归搜索整个目录
$ grep -v "DEBUG" log.txt        # 排除包含DEBUG的行
$ grep -c "error" log.txt        # 统计匹配行数
```

### find —— 找文件

```bash
$ find . -name "*.go"            # 在当前目录找所有.go文件
$ find . -type d -name "test"   # 找名为test的目录
$ find . -mtime -7               # 找7天内修改过的文件
```

### 管道（|）—— 串联命令

管道把前一个命令的**输出**变成后一个命令的**输入**。

```bash
# 在日志里找包含error的行，然后统计有多少行
$ grep "error" log.txt | wc -l

# 列出所有.go文件，然后从中搜索包含"main"的文件
$ find . -name "*.go" | xargs grep "main"

# 看看哪些进程占内存最大（排序取前5个）
$ ps aux | sort -k4 -r | head -5
```

### 重定向 —— 把输出存到文件

```bash
$ echo "hello" > file.txt        # 输出到文件（覆盖）
$ echo "world" >> file.txt       # 追加到文件末尾

# 把grep的结果保存到文件
$ grep "error" log.txt > errors.txt
```

## 5.3 环境变量

### 环境变量是什么

环境变量是操作系统级别的**全局变量**。所有程序都能读取它们。

```bash
$ echo $HOME        # 你的家目录
/home/zhangsan

$ echo $USER        # 当前用户名
zhangsan

$ echo $SHELL       # 当前shell
/bin/bash
```

### PATH —— 最重要的环境变量

当你在终端输入 `go` 并回车，操作系统去哪找 `go` 这个程序？

答案是：它在 **PATH** 里列出的目录中逐个去找。

```bash
$ echo $PATH
/usr/local/bin:/usr/bin:/bin:/usr/local/go/bin:/home/zhangsan/go/bin
```

上面列了5个目录。当你输入 `go` 时，操作系统会：
1. 去 `/usr/local/bin` 找 → 没有
2. 去 `/usr/bin` 找 → 没有
3. 去 `/bin` 找 → 没有
4. 去 `/usr/local/go/bin` 找 → **找到了！** 执行它

如果所有目录都找不到，就会报错：`command not found`

### 理解这个有什么用

你安装Go后，终端说 `go: command not found`——这就是因为Go的安装目录不在PATH里。你需要把它加上。

当你用 `go install` 安装了一个工具后，它被放到 `$GOPATH/bin` 目录，你也要确保这个目录在PATH里。

### 和Go相关的环境变量

```bash
$ go env           # 查看所有Go相关的环境变量

GOROOT=/usr/local/go       # Go的安装位置
GOPATH=/home/zhangsan/go   # 你的工作目录
GOBIN=$GOPATH/bin          # 编译好的可执行文件放哪
GOOS=linux                 # 目标操作系统
GOARCH=amd64               # 目标CPU架构
```
