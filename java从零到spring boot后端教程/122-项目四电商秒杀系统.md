> 💡 本章你会做出一个**电商秒杀系统**，重点是理解高并发场景下的架构设计：如何防止超卖、如何用 Redis 预减库存、如何用消息队列削峰。完成后的效果：1000 个人同时抢 100 件商品，最终正好卖出 100 件，不多不少。本章不仅写代码，还附带一份可视化架构图。

## 122.1 秒杀系统的难点

```
用户端：                    服务端：
"1秒内下单"    VS    "校验库存 → 扣库存 → 创建订单 → 扣款"

问题：
❌ 超卖：100 库存，卖出 150 件
❌ 数据库压力：5000 请求同时打到 MySQL
❌ 响应慢：用户等待超过 5 秒
```

### 架构方案

```
用户请求
  ↓
Nginx（限流 + 静态页）
  ↓
Spring Boot（接口层）
  ↓
Redis（预减库存，Lua 原子操作）
  ↓
RabbitMQ（异步下单，削峰填谷）
  ↓
MySQL（最终一致性，持久化订单）
```

## 122.6 Redis 减库存 Lua 脚本

### RedisConfig

```java
package com.example.seckill.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.data.redis.core.script.DefaultRedisScript;
import org.springframework.data.redis.serializer.GenericJackson2JsonRedisSerializer;
import org.springframework.data.redis.serializer.StringRedisSerializer;

@Configuration
public class RedisConfig {

    @Bean
    public RedisTemplate<String, Object> redisTemplate(
            org.springframework.data.redis.connection.RedisConnectionFactory factory) {
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        template.setConnectionFactory(factory);
        template.setKeySerializer(new StringRedisSerializer());
        template.setValueSerializer(new GenericJackson2JsonRedisSerializer());
        return template;
    }

    @Bean
    public DefaultRedisScript<Long> deductStockScript() {
        DefaultRedisScript<Long> script = new DefaultRedisScript<>();
        script.setResultType(Long.class);
        script.setScriptText("""
                local key = KEYS[1]
                local stock = tonumber(redis.call('get', key))
                if stock == nil or stock <= 0 then
                    return -1
                end
                redis.call('decr', key)
                return stock - 1
                """);
        return script;
    }
}
```

Lua 脚本解析：

| 行 | 含义 |
|----|------|
| `local stock = redis.call('get', key)` | 读取当前库存 |
| `if stock <= 0 then return -1` | 库存不足，返回 -1 |
| `redis.call('decr', key)` | 原子减 1 |
| `return stock - 1` | 返回剩余库存 |

> 💡 Redis 执行 Lua 脚本是原子的——读取和扣减之间不会有其他命令插入，从根源上防止超卖。

## 122.7 SeckillService——核心秒杀逻辑

```java
package com.example.seckill.service;

import com.example.seckill.entity.Product;
import com.example.seckill.repository.ProductRepository;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.amqp.rabbit.core.RabbitTemplate;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.data.redis.core.script.DefaultRedisScript;
import org.springframework.stereotype.Service;

import java.util.Collections;
import java.util.concurrent.TimeUnit;

@Slf4j
@Service
@RequiredArgsConstructor
public class SeckillService {

    private final RedisTemplate<String, Object> redisTemplate;
    private final DefaultRedisScript<Long> deductStockScript;
    private final RabbitTemplate rabbitTemplate;
    private final ProductRepository productRepository;

    private static final String STOCK_PREFIX = "seckill:stock:";
    private static final String ORDERED_SET = "seckill:ordered:users:";

    public String seckill(Long productId, Long userId) {
        // 1. 防重复下单
        String orderedKey = ORDERED_SET + productId;
        Boolean alreadyOrdered = redisTemplate.opsForSet().isMember(orderedKey, userId.toString());
        if (Boolean.TRUE.equals(alreadyOrdered)) {
            return "您已经参与过该商品的秒杀";
        }

        // 2. Redis 原子减库存
        String stockKey = STOCK_PREFIX + productId;
        Long remaining = redisTemplate.execute(
                deductStockScript,
                Collections.singletonList(stockKey)
        );

        if (remaining == null || remaining < 0) {
            return "秒杀已结束，库存不足";
        }

        // 3. 标记用户已下单
        redisTemplate.opsForSet().add(orderedKey, userId.toString());
        redisTemplate.expire(orderedKey, 1, TimeUnit.HOURS);

        // 4. 发送消息到队列，异步创建订单
        String message = productId + ":" + userId;
        rabbitTemplate.convertAndSend("seckill.exchange", "seckill.order", message);

        log.info("秒杀成功：用户{} 商品{} 剩余库存{}", userId, productId, remaining);
        return "秒杀成功";
    }

    public void initStockToRedis(Long productId) {
        Product product = productRepository.findById(productId)
                .orElseThrow(() -> new RuntimeException("商品不存在"));
        redisTemplate.opsForValue().set(STOCK_PREFIX + productId, product.getStock());
        log.info("商品{} 库存{}已加载到Redis", productId, product.getStock());
    }
}
```

## 122.8 OrderConsumer——异步创建订单

```java
package com.example.seckill.service;

import com.example.seckill.entity.Product;
import com.example.seckill.entity.SeckillOrder;
import com.example.seckill.repository.ProductRepository;
import com.example.seckill.repository.SeckillOrderRepository;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.amqp.rabbit.annotation.RabbitListener;
import org.springframework.stereotype.Component;
import org.springframework.transaction.annotation.Transactional;

@Slf4j
@Component
@RequiredArgsConstructor
public class OrderConsumer {

    private final SeckillOrderRepository orderRepository;
    private final ProductRepository productRepository;

    @RabbitListener(queues = "seckill.order.queue")
    @Transactional
    public void handleOrder(String message) {
        String[] parts = message.split(":");
        Long productId = Long.parseLong(parts[0]);
        Long userId = Long.parseLong(parts[1]);

        // MySQL 扣库存（乐观锁）
        Product product = productRepository.findById(productId)
                .orElseThrow(() -> new RuntimeException("商品不存在"));

        if (product.getStock() <= 0) {
            log.warn("MySQL库存不足：商品{} 用户{}", productId, userId);
            return;
        }

        product.setStock(product.getStock() - 1);
        productRepository.save(product);

        // 创建订单
        SeckillOrder order = new SeckillOrder();
        order.setProductId(productId);
        order.setUserId(userId);
        order.setStatus("PAID");
        orderRepository.save(order);

        log.info("订单创建成功：商品{} 用户{} MySQL剩余{}", productId, userId, product.getStock());
    }
}
```

## 122.9 RabbitMQ 配置

```java
package com.example.seckill.config;

import org.springframework.amqp.core.*;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class RabbitMQConfig {

    @Bean
    public TopicExchange seckillExchange() {
        return new TopicExchange("seckill.exchange");
    }

    @Bean
    public Queue seckillOrderQueue() {
        return new Queue("seckill.order.queue", true);
    }

    @Bean
    public Binding binding(Queue seckillOrderQueue, TopicExchange seckillExchange) {
        return BindingBuilder.bind(seckillOrderQueue)
                .to(seckillExchange).with("seckill.order");
    }
}
```

## 122.10 SeckillController

```java
package com.example.seckill.controller;

import com.example.seckill.common.Result;
import com.example.seckill.service.SeckillService;
import lombok.RequiredArgsConstructor;
import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/api/seckill")
@RequiredArgsConstructor
public class SeckillController {

    private final SeckillService seckillService;

    @PostMapping("/{productId}")
    public Result<String> seckill(
            @PathVariable Long productId,
            @RequestHeader("X-User-Id") Long userId) {
        String result = seckillService.seckill(productId, userId);
        if ("秒杀成功".equals(result)) {
            return Result.success(result);
        }
        return Result.error(400, result);
    }

    @PostMapping("/init/{productId}")
    public Result<String> initStock(@PathVariable Long productId) {
        seckillService.initStockToRedis(productId);
        return Result.success("库存已初始化到Redis");
    }
}
```

## 122.12 完成效果

成功运行后：
1. **库存预加载**：POST `/api/seckill/init/1` → Redis 写入商品库存
2. **正常秒杀**：用户抢购 → Redis 原子减库存 → RabbitMQ 异步下单 → MySQL 持久化
3. **防重复下单**：同一用户重复请求 → 返回"您已参与过"
4. **库存售罄**：库存为 0 后 → 返回"秒杀已结束"
5. **最终一致性**：Redis 库存数 = MySQL 库存数 = 初始库存 - 订单数

## 自测题

1. 只用 MySQL `UPDATE ... SET stock = stock - 1 WHERE stock > 0` 能防超卖吗？为什么还需要 Redis？
2. RabbitMQ 在这个系统中起什么作用？如果不用它，直接用同步方式有什么问题？
3. Lua 脚本中为什么不用 `DECR` 直接减，而要先 `GET` 判断？

<details>
<summary>点击查看答案</summary>

1. 能防超卖（行级锁保证原子性），但并发量高时 MySQL 会成为瓶颈。Redis 单线程处理极快（10万QPS），在 Redis 层过滤掉大部分请求，只有成功扣库存的请求才进入 MySQL，大幅降低数据库压力。
2. RabbitMQ 起削峰填谷作用。如果同步处理，1000 个并发请求同时写 MySQL，连接池很快耗尽。通过消息队列，请求先进入队列，Consumer 按自己的处理能力逐条消费，MySQL 不会被瞬时流量击垮。
3. 单纯 `DECR` 在库存 0 时会得到负数。需要先判断库存是否大于 0。Lua 脚本保证"判断 + 扣减"这个过程是原子的，不会出现"判断通过但扣减时库存已被别人扣完"的竞态条件。
</details>
