## 我在做什么？

上一章你写了一行 `printf("Hello, World!\n");` 就跑出了第一个程序。但 `#include` 是什么？`int main(void)` 为什么必须写？花括号 `{}` 到底框住了什么？

这一章，我们把C程序当作一栋房子，从地基到屋顶，把"骨架"看清楚。你会理解：一个C程序由哪些部件组成，每个部件为什么在那里，以及它们之间怎么配合。

学完本章，你拿到任何一个C程序，都能一眼看出它的结构层次。

## 7.1 最小C程序骨架

一个能跑的C程序，最少需要这些部件：

```c
#include <stdio.h>

int main(void)
{
    /* 你的代码写在这里 */
    return 0;
}
```

把它拆开，就是本章要讲的四件事：

```
┌──────────────────────────────────────────────┐
│              C程序的骨架                       │
├──────────────────────────────────────────────┤
│  #include <stdio.h>   ← 模块1：预处理指令      │
│                                              │
│  int main(void)      ← 模块2：main函数入口    │
│  {                    ← 模块3：花括号（函数体）│
│                       ← 模块4：语句与注释      │
│      /* 你的代码 */                           │
│      printf("Hello!");                       │
│      return 0;                               │
│  }                                            │
└──────────────────────────────────────────────┘
```

下面逐个拆解。

## 7.2 模块1：`#include` 预处理指令

### 7.2.1 它是什么？

`#include` 是一条 `*预处理指令*（此术语见附录F）`。以 `#` 开头的行，是给 **预处理器** 看的，不是给编译器看的。

**比喻**：编译器是一个翻译官，只懂英文（C语言）。预处理器是翻译官的助理，在翻译开始前，先帮你把草稿整理好。`#include` 就是助理收到的一条命令："把那个文件的内容复制过来贴在这里。"

### 7.2.2 两种写法

```c
#include <stdio.h>   /* 尖括号：去系统路径找 */
#include "myheader.h" /* 双引号：先去当前目录找，找不到再去系统路径 */
```

两者的区别在于搜索路径：
- `<>`：告诉预处理器去"系统仓库"（标准库路径）找。用于系统头文件。
- `""`：告诉预处理器先去"你手边的文件夹"（当前目录）找。用于你自己写的头文件。

**比喻**：`<>` 是去图书馆借书，`""` 是你自己从书包里掏笔记。

### 7.2.3 预处理的实际效果

你可以用 `gcc -E` 命令看看预处理后的结果：

```bash
gcc -E hello.c -o hello.i
```

打开 `hello.i`，你会发现原来的 `#include <stdio.h>` 消失不见了，取而代之的是上千行代码——这就是 `stdio.h` 文件的全部内容被"粘贴"过来了。

### 7.2.4 常见的头文件

| 头文件 | 中文名 | 提供什么 |
|--------|--------|----------|
| `<stdio.h>` | 标准输入输出 | `printf`, `scanf`, `getchar` 等 |
| `<stdlib.h>` | 标准库 | `malloc`, `free`, `exit` 等 |
| `<string.h>` | 字符串处理 | `strlen`, `strcpy`, `strcmp` 等 |
| `<math.h>` | 数学函数 | `sin`, `cos`, `sqrt` 等 |
| `<stdbool.h>` | 布尔类型 | `bool`, `true`, `false`（C99起） |

记住一条规则：**用哪个函数，就引入对应的头文件**。不知道某个函数在哪个头文件里？查手册即可。

## 7.3 模块2：`main` 函数——程序的入口

### 7.3.1 为什么是 `main`？

你写十个函数、一百个函数，操作系统只认一个：`main`。程序启动时，操作系统跳过所有其他函数，直接跳到 `main` 的第一行开始执行。

**比喻**：`main` 是整栋房子的唯一大门。不管里面有多少房间（函数），所有人都得从这道门进来。没有门，房子就是封闭的盒子，谁也进不去。

### 7.3.2 `int main(void)` 的句子成分

```c
int   main   (void)
│     │       │
│     │       └── 参数列表：void 表示不接收任何参数
│     └── 函数名：main，必须叫这个名字
└── 返回类型：int，表示向操作系统返回一个整数
```

- `int`：C语言是强类型语言，每个函数必须声明自己"返回什么类型的值"。`main` 返回 `int` 是C标准规定的。
- `main`：函数名。C语言**大小写敏感**，`Main`、`MAIN`、`main` 是三个不同的名字。
- `(void)`：参数列表。`void` 是C语言中的"空"，表示"这里什么都不需要"。另一种写法 `int main()` 在C语言中不等于"无参数"（它表示"参数未指定"），初学者请统一使用 `int main(void)`。

### 7.3.3 `main` 函数的其他合法写法

根据C标准，以下写法都是合法的：

```c
int main(void) { ... }           /* 无参数，推荐写法 */
int main(int argc, char *argv[]) { ... }  /* 接收命令行参数，后面会学 */
```

以下写法虽然能编译通过，但**不推荐**：

```c
void main(void) { ... }    /* 不符合C标准，返回值类型不能是void */
main() { ... }             /* 省略返回类型，C89默认为int，但不推荐 */
```

## 7.4 模块3：花括号 `{}` 与代码块

花括号 `{}` 定义了一个 **代码块**（block）。`main` 函数的代码块又叫 **函数体**（function body）。

```c
int main(void)
{                          /* ← 左花括号：函数体开始 */
    printf("Hello!\n");
    return 0;
}                          /* ← 右花括号：函数体结束 */
```

**比喻**：花括号像一面围墙。墙内是一个独立的世界，里面的东西属于这个函数。墙外的人（其他函数）不能直接使用墙内的东西。

### 7.4.1 缩进

你可能注意到代码里的空格：

```c
int main(void)
{
    printf("Hello!\n");   /* 前面有4个空格 */
    return 0;             /* 前面有4个空格 */
}
```

这个缩进不是语法要求，编译器不关心。但**你必须写**，因为：
- 没有缩进的代码就像一本书没有分段，没法读。
- 团队协作中，不缩进的代码会被同事骂。

建议统一使用 **4个空格** 缩进（不要用Tab，因为Tab在不同编辑器里显示宽度不同）。

### 7.4.2 花括号的风格

C语言中有两种常见的花括号写法：

```c
/* 风格A：K&R风格（推荐） */
int main(void) {
    printf("Hello!\n");
    return 0;
}

/* 风格B：Allman风格 */
int main(void)
{
    printf("Hello!\n");
    return 0;
}
```

选一种，整个教程从头用到尾。本教程使用 **Allman风格**（左括号独占一行）。

## 7.5 模块4：语句与注释

### 7.5.1 语句与分号

C语言中，一条"完整的命令"叫 **语句**（statement）。每条语句必须以分号 `;` 结尾。

```c
printf("Hello!\n");   /* 这是一条语句，末尾有分号 */
return 0;             /* 这也是一条语句 */
int x = 5;            /* 这还是一条语句 */
```

**比喻**：分号就是C语言的句号。你说完一句话，必须画句号。编译器靠分号来判断"这一句说完了，开始下一句"。

注意：`#include`、函数头 `int main(void)`、花括号 `{` `}` 后面**不加分号**。它们不是语句，是"标题"和"边界"。

```c
/* ❌ 错误示例 */
#include <stdio.h>;    /* #include 后面不能加分号 */

/* ❌ 错误示例 */
int main(void);        /* 函数头后面不能加分号 */

/* ✅ 正确示例 */
#include <stdio.h>
int main(void)
{
    printf("Hi!\n");   /* 只有语句末尾才加分号 */
    return 0;
}
```

### 7.5.2 注释：写给人类看的文字

注释是给程序员（包括未来的你自己）看的说明文字，编译器会**完全忽略**它们。

C语言有两种注释：

```c
/* 这是多行注释（传统风格）
   可以跨越多行
   编译器完全不看这里 */

// 这是单行注释（C99引入）
// 从双斜杠到行末都是注释
int x = 10;  // 行尾注释也很常用
```

**注释原则**：
- 不要写"废话注释"：`int x = 5; /* 把5赋值给x */` 这种注释毫无意义。
- 写"为什么"而不是"是什么"：`int timeout = 30; /* 30秒后超时，因为服务器最多响应25秒 */` 才是好注释。
- 修改代码时，**同步修改注释**。过时的注释比没有注释更危险。

```

│                    program.c                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌── 预处理指令区 ──────────────────────────────┐   │
│  │ #include <stdio.h>     // 引入标准输入输出    │   │
│  │ #include <stdlib.h>    // 引入标准库          │   │

│  └──────────────────────────────────────────────┘   │
│                                                     │

│  │                                               │   │
│  └──────────────────────────────────────────────┘   │
│                                                     │
│  ┌── 函数定义区 ────────────────────────────────┐   │

│  │      /* 语句2 */                              │   │

│  └──────────────────────────────────────────────┘   │
│                                                     │
```

## 7.7 常见错误

### 错误1：`#include` 后加分号

```c
/* ❌ 错误示例 */
#include <stdio.h>;    /* 预处理器会报错 */

/* ✅ 正确示例 */
#include <stdio.h>
```

### 错误2：`main` 函数头后加分号

```c
/* ❌ 错误示例 */
int main(void);   /* 分号让编译器以为这是一个函数声明，不是定义 */
{
    return 0;
}

/* ✅ 正确示例 */
int main(void)
{
    return 0;
}
```

### 错误3：花括号不配对

```c
/* ❌ 错误示例 */
int main(void)
{
    printf("Hello!\n");
    return 0;
/* 少了一个 } */

/* ✅ 正确示例 */
int main(void)
{
    printf("Hello!\n");
    return 0;
}
```

编译器报错信息通常是 `error: expected '}' at end of input`。如果看到这个，数一数花括号是否配对。

### 错误4：忘记 `return 0;`

```c
/* ❌ 错误示例 */
int main(void)
{
    printf("Hello!\n");
}

/* ✅ 正确示例 */
int main(void)
{
    printf("Hello!\n");
    return 0;
}
```

### 错误5：注释嵌套（传统注释不能嵌套）

```c
/* ❌ 错误示例 */
/* 外层注释开始
   /* 内层注释开始 */   ← 这里已经结束了外层注释！
   这里不再是注释，编译器会尝试解析    ← 报错
*/

/* ✅ 正确示例：用单行注释嵌套 */
/* 外层注释
   // 内层注释
   继续外层注释
*/
```

## 7.8 我在做什么？做得对不对？不对怎么办？

### 我在做什么？

我在理解C程序的"骨架"：`#include` 预处理指令怎么引入头文件、`main` 函数为什么是入口、花括号怎么定义代码块、语句为什么要用分号结尾、注释怎么写。

### 我做得对不对？

- 你能徒手写出一个最小C程序的完整骨架（包含 `#include`、`main`、`return`）→ 结构过关。
- 你知道 `#include <stdio.h>` 和 `#include "myheader.h"` 的区别 → 预处理指令过关。
- 你能说出 `main` 的三个部分（`int`、`main`、`(void)`）分别代表什么 → main函数过关。
- 你能解释注释的两种写法各适用于什么场景 → 注释过关。

### 不对怎么办？

| 症状 | 第一步检查 |
|------|-----------|
| 编译报错 `expected '}'` | 花括号是否配对？用编辑器的高亮功能逐对检查 |
| 编译报错 `expected ';'` | 某条语句末尾是否少了分号？ |
| 编译报错 `unknown type name` | 是不是 `#include` 后面加了分号？ |
| 链接报错 `undefined reference to 'main'` | `main` 是否拼错？是否写了 `main` 函数？ |
| 程序跑起来没输出 | 是否忘了 `printf`？是否忘了重新编译？ |

## 本章小结

| 知识点 | 说明 |
|--------|------|
| `#include` | 预处理指令，在编译前引入头文件内容 |
| `<stdio.h>` | 尖括号形式，从系统路径搜索头文件 |
| `"myheader.h"` | 双引号形式，优先从当前目录搜索 |
| `main` 函数 | 程序唯一入口，操作系统由此开始执行 |
| `int main(void)` | 推荐写法：返回int，无参数 |
| 花括号 `{}` | 定义代码块（函数体）的边界 |
| 分号 `;` | 语句结束符，必须写 |
| `/* */` 注释 | 传统多行注释，不能嵌套 |
| `//` 注释 | 单行注释（C99起），可嵌套于多行注释中 |

## 练习题

1. **骨架默写**：不看书，徒手写出一个最小C程序的完整骨架。然后编译验证。

2. **头文件实验**：把 `#include <stdio.h>` 这行删掉，编译看看报什么错。用报错信息反推 `#include` 的作用。

3. **注释实验**：写一个程序，把 `printf` 那一行用 `/* */` 注释掉，编译运行。观察输出是否消失，体会注释的"屏蔽"效果。

4. **花括号失配**：故意删掉一个花括号，编译。阅读报错信息，学会从报错信息中定位花括号问题。

5. **双引号 include**：在你自己的程序目录下创建一个文件 `myheader.h`（内容任意，比如 `/* 这是自定义头文件 */`），然后在主程序中用 `#include "myheader.h"` 引入它。用 `gcc -E` 查看预处理结果，确认你的文件内容被"粘贴"进去了。
