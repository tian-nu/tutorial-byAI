# 第48章 · 排序算法

## 48.1 冒泡排序：相邻比较，大的后移

冒泡排序是最直观的排序——就像水里的气泡，大的慢慢浮到水面（数组末尾）。

**核心思想**：重复遍历数组，每次比较相邻的两个元素，如果前大后小就交换。每完成一轮遍历，最大的那个就被推到末尾。下一轮遍历就少看最后一个元素（因为它已经就位了）。

```
初始:    [5, 3, 7, 1, 9]

第1轮:   [3, 5, 1, 7, 9]   ← 9被推到末尾（就位）
          3↔5  5↔1  1↔7  7↔9
第2轮:   [3, 1, 5, 7, 9]   ← 7被推到倒数第二（就位）
          3↔1  1↔5  5↔7
第3轮:   [1, 3, 5, 7, 9]   ← 5就位
          1↔3  3↔5
第4轮:   [1, 3, 5, 7, 9]   ← 3就位（没有交换）
结果:    [1, 3, 5, 7, 9]
```

**代码实现**：

```go
func bubbleSort(arr []int) {
    n := len(arr)
    for i := 0; i < n-1; i++ {
        swapped := false
        for j := 0; j < n-1-i; j++ {
            if arr[j] > arr[j+1] {
                arr[j], arr[j+1] = arr[j+1], arr[j]
                swapped = true
            }
        }
        if !swapped {
            break
        }
    }
}
```

**时间复杂度**：
- 最坏：O(n²)——完全逆序
- 最好：O(n)——本来就有序（`swapped`标志让算法提前退出）
- 平均：O(n²)

**空间复杂度**：O(1)（原地排序，不需要额外数组）

**稳定性**：稳定。相等的元素不会交换位置。

冒泡排序虽然简单，但几乎是所有排序中最慢的。它的价值在于教学和学习——理解了冒泡，就理解了排序问题的基本框架。

## 48.2 选择排序：每次找最小的放前面

**核心思想**：把数组分成"已排序区"（前面）和"未排序区"（后面）。每次从未排序区中找到最小的元素，把它和未排序区的第一个元素交换位置。

```
初始:    [5, 3, 7, 1, 9]
         |  未排序区  |

第1轮:   [1, 3, 7, 5, 9]   ← 找到最小1，和5交换
          已│  未排序区
第2轮:   [1, 3, 7, 5, 9]   ← 找到最小3，已经在位置上
          已排序│未排序区
第3轮:   [1, 3, 5, 7, 9]   ← 找到最小5，和7交换
          已排序区  │未排序
第4轮:   [1, 3, 5, 7, 9]   ← 找到最小7，已经在位置上
          已排序区      │
结果:    [1, 3, 5, 7, 9]   ← 全部已排序
```

**代码实现**：

```go
func selectionSort(arr []int) {
    n := len(arr)
    for i := 0; i < n-1; i++ {
        minIdx := i
        for j := i + 1; j < n; j++ {
            if arr[j] < arr[minIdx] {
                minIdx = j
            }
        }
        arr[i], arr[minIdx] = arr[minIdx], arr[i]
    }
}
```

**时间复杂度**：无论什么情况都是O(n²)——因为每次都要遍历剩余所有元素找最小值。

**空间复杂度**：O(1)（原地排序）

**稳定性**：不稳定。交换可能破坏相等元素的相对顺序。

选择排序比冒泡排序好在**交换次数少**——冒泡每次比较都可能交换，选择每轮只交换一次。但比较次数一样多（都是O(n²)）。

## 48.3 插入排序：打牌时的摸牌动作

**核心思想**：把数组分成"已排序区"（前面）和"未排序区"（后面）。每次拿未排序区的第一个元素，在已排序区中找到合适的位置插入——就像打牌时摸到一张新牌，插入到手里已经排好的牌中。

```
初始:    [5, 3, 7, 1, 9]
         已 未排序区

第1张(3): [3, 5, 7, 1, 9]   ← 3比5小，插入到5前面
           已排序 未排序

第2张(7): [3, 5, 7, 1, 9]   ← 7比5大，不动
           已排序区 未排序

第3张(1): [1, 3, 5, 7, 9]   ← 1一直往前挪，插入到最前面
           已排序区    未

第4张(9): [1, 3, 5, 7, 9]   ← 9比7大，不动
           全部已排序
```

**代码实现**：

```go
func insertionSort(arr []int) {
    for i := 1; i < len(arr); i++ {
        key := arr[i]
        j := i - 1
        for j >= 0 && arr[j] > key {
            arr[j+1] = arr[j]
            j--
        }
        arr[j+1] = key
    }
}
```

**时间复杂度**：
- 最坏：O(n²)——完全逆序，每次都要挪到最前面
- 最好：O(n)——本来就有序，内层循环一次都不执行
- 平均：O(n²)

**空间复杂度**：O(1)

**稳定性**：稳定。相等元素不会交换位置（`arr[j] > key`而不是`>=`）。

**插入排序的独特优势**：在**小数据量（n<50）**和**基本有序**的场景下，插入排序比快排和归并排序更快——因为它的常数因子非常小，而且没有递归开销。这正是Go的pdqsort在小数据量时退化为插入排序的原因。

## 48.4 快速排序：分治思想的巅峰

快速排序是整个排序界的"明星算法"。它被称为二十世纪十大算法之一。

**核心思想**：选一个"基准"（pivot），把数组分成两部分——左边都比基准小，右边都比基准大。然后对左右两部分递归做同样的操作。

```
初始:  [5, 3, 7, 1, 9, 2, 8, 4, 6]
         选中间元素2做基准

分区后: [1, 2, 5, 3, 9, 7, 8, 4, 6]
         ←都比2小  2 都比2大→

对左半 [1] 递归 → 已经只有1个元素，不用排
对右半 [5,3,9,7,8,4,6] 递归 → 选基准7...

继续分下去，直到每个子数组只有1个元素 → 整个数组排好了
```

**代码实现（Lomuto分区方案）**：

```go
func quickSort(arr []int, low, high int) {
    if low < high {
        pivotIdx := partition(arr, low, high)
        quickSort(arr, low, pivotIdx-1)
        quickSort(arr, pivotIdx+1, high)
    }
}

func partition(arr []int, low, high int) int {
    pivot := arr[high]
    i := low - 1
    for j := low; j < high; j++ {
        if arr[j] <= pivot {
            i++
            arr[i], arr[j] = arr[j], arr[i]
        }
    }
    arr[i+1], arr[high] = arr[high], arr[i+1]
    return i + 1
}
```

`partition`函数干了什么？它维护两个区域：
- `[low...i]`：已确认 ≤ pivot的区域
- `[i+1...j-1]`：已确认 > pivot的区域
- `[j...high-1]`：还没看的区域

每次看到一个≤pivot的元素，就把它换到"小区域"的末尾（i+1位置），小区域扩大一格。

**时间复杂度**：
- 平均：O(n log n)——每次分区大致平均分成两半
- 最坏：O(n²)——每次基准都是最小或最大（如已有序或完全逆序的数组）
- 最好：O(n log n)

**空间复杂度**：O(log n)（递归调用栈深度）

**稳定性**：不稳定。分区时的交换会打乱相等元素的相对顺序。

### 快排为什么快？

1. **局部性原理**：快排是原地排序，数据访问集中在数组的某个区域——CPU缓存命中率高
2. **常数因子小**：虽然和归并排序都是O(n log n)，但快排的内循环极其简洁（就一个比较+可能交换），实际常数比归并小得多
3. **尾递归可优化**：编译器可以把某些递归调用优化为循环

### 快排的最坏情况与改进

如果每次选最后一个元素做基准，面对已经有序的数组：

```
[1, 2, 3, 4, 5, 6] 选6做基准
→ [1,2,3,4,5] + [6]  （极不均匀的分区！）
→ [1,2,3,4] + [5] + [6]
→ ... 递归深度n层，每层处理n个元素 → O(n²)
```

**改进方案——三数取中法**：选low、mid、high三个位置的中间值做基准。

```go
func medianOfThree(arr []int, low, high int) int {
    mid := low + (high-low)/2
    if arr[low] > arr[mid] {
        arr[low], arr[mid] = arr[mid], arr[low]
    }
    if arr[low] > arr[high] {
        arr[low], arr[high] = arr[high], arr[low]
    }
    if arr[mid] > arr[high] {
        arr[mid], arr[high] = arr[high], arr[mid]
    }
    arr[mid], arr[high-1] = arr[high-1], arr[mid]
    return arr[high-1]
}
```

这大大降低了遇到最坏情况的概率。

## 48.5 归并排序：先分后合

**核心思想**：把数组分成两半，分别排序，然后把两个有序子数组合并成一个。

```
                     [5,3,7,1,9,2,8,4]
                    /                \
              [5,3,7,1]            [9,2,8,4]
             /        \            /        \
          [5,3]      [7,1]      [9,2]     [8,4]
          /   \       /   \      /   \      /   \
        [5]   [3]   [7]   [1]  [9]   [2]  [8]   [4]
          \   /       \   /      \   /      \   /
          [3,5]      [1,7]      [2,9]      [4,8]
             \        /            \        /
              [1,3,5,7]            [2,4,8,9]
                    \                /
                     [1,2,3,4,5,7,8,9]
```

分到不能再分（单个元素），然后自底向上合并。每次合并两个有序序列——这就是"归并"。

**代码实现**：

```go
func mergeSort(arr []int) []int {
    if len(arr) <= 1 {
        return arr
    }
    mid := len(arr) / 2
    left := mergeSort(arr[:mid])
    right := mergeSort(arr[mid:])
    return merge(left, right)
}

func merge(left, right []int) []int {
    result := make([]int, 0, len(left)+len(right))
    i, j := 0, 0
    for i < len(left) && j < len(right) {
        if left[i] <= right[j] {
            result = append(result, left[i])
            i++
        } else {
            result = append(result, right[j])
            j++
        }
    }
    result = append(result, left[i:]...)
    result = append(result, right[j:]...)
    return result
}
```

**时间复杂度**：无论什么情况都是O(n log n)——**归并排序是稳定的O(n log n)，不受输入数据影响。**

**空间复杂度**：O(n)——每次合并都要创建新数组存放结果。

**稳定性**：稳定。`left[i] <= right[j]`保证相等时先放左边的元素。

### 快排 vs 归并

| | 快排 | 归并 |
|---|------|------|
| 时间复杂度（平均） | O(n log n) | O(n log n) |
| 时间复杂度（最坏） | O(n²) | O(n log n) |
| 空间复杂度 | O(log n) | O(n) |
| 稳定性 | 不稳定 | 稳定 |
| 缓存友好度 | 高（原地排序） | 较低（需要额外数组） |
| 适用场景 | 内存中的大量数据 | 外部排序、链表排序 |

**比喻**：快排像"整理乱书架"——你在原地挪书，这摞放左边那摞放右边。归并像"合并两个已经整理好的书架"——你需要一个空书架来临时放结果。

## 48.6 Go的sort.Slice：pdqsort

Go 1.19之前，Go的`sort.Slice`底层是基于快速排序的introsort（内省排序）——先在快排、堆排之间切换，小数据量用插入排序。

Go 1.19引入了pdqsort（Pattern-Defeating Quicksort，模式击败快速排序）——这是一个结合了三种排序算法的"终极排序"：

```
pdqsort的策略：

1. 小数据量（n < 24）→ 插入排序（在小数据上插入排序最快）
2. 发现快排分区效果差（连续多次分区极不平衡）
   → 切换到堆排序（堆排序保证O(n log n)且不需要额外空间）
3. 正常情况 → 快速排序（三数取中+双向分区）
4. 数据接近有序 → 优化分区策略
5. 大量重复元素 → 三路分区（分成<,=,>三部分）
```

pdqsort的名字"Pattern-Defeating"的意思是：它能识别并"击败"那些会让普通快排退化的数据模式（已有序、有很多重复项等）。

使用上，开发者不需要关心这些细节——一行代码搞定：

```go
sort.Slice(arr, func(i, j int) bool {
    return arr[i] < arr[j]
})
```

Go标准库的pdqsort在大多数benchmark上都比之前的内省排序快10%~30%，而且保证了即使在最坏情况下也是O(n log n)。

### 五种排序算法全对比

```go
arr := make([]int, 10000)
for i := range arr {
    arr[i] = rand.Intn(100000)
}

func BenchmarkBubble(b *testing.B) {
    for i := 0; i < b.N; i++ {
        temp := make([]int, len(arr))
        copy(temp, arr)
        bubbleSort(temp)
    }
}
```

大致benchmark结果（10000个随机整数，MacBook Pro M1）：

| 算法 | 耗时（大致） | 比较 |
|------|------------|------|
| Bubble | ~150ms | 基准 |
| Selection | ~120ms | 1.25x |
| Insertion | ~80ms | 1.9x |
| Quick | ~0.8ms | 187x |
| Merge | ~1.2ms | 125x |
| sort.Slice(pdqsort) | ~0.5ms | 300x |

O(n²)算法和O(n log n)算法在数据量达到10^4时就拉开了两个数量级的差距。

## 🤔 想多一点：排序算法的"稳定"到底重不重要？

排序算法的稳定性是指：如果两个元素相等，排序后它们的相对顺序不变。

```go
type Student struct {
    Name  string
    Score int
}

students := []Student{
    {"张三", 90},
    {"李四", 90},
    {"王五", 85},
}

sort.SliceStable(students, func(i, j int) bool {
    return students[i].Score > students[j].Score
})
```

用稳定排序：张三和李四都是90分，张三在原数组中在李四前面——排序后张三还在李四前面。

用不稳定排序：张三和李四可能交换位置。

**什么时候稳定很重要？**

- **多级排序**：先按成绩排，再按姓名排——如果成绩排序是稳定的，那么同成绩的人已经按原始顺序（或上一级排序结果）排好了。Redis的SORT命令支持多字段排序，底层就依赖稳定性。
- **数据库ORDER BY**：`SELECT * FROM users ORDER BY age, name`——它期望先按age排，age相同的保持name的原有顺序。

**所以，稳定不是"可有可无的特性"——它决定了你能否用排序实现"先按A排，再按B排"这种多级排序。**

Go的`sort.Slice`（pdqsort）是不稳定的，`sort.SliceStable`（归并排序）是稳定的。如果你需要稳定的排序，用后者。

## 本章小结

| 知识点 | 要点 |
|--------|------|
| 冒泡排序 | 相邻比较交换，O(n²)，稳定，最慢但最简单 |
| 选择排序 | 每次找最小放前面，O(n²)，不稳定，交换次数少 |
| 插入排序 | 摸牌式插入已排序区，最好O(n)，稳定，小数据最优 |
| 快速排序 | 选基准分区递归，平均O(n log n)最坏O(n²)，不稳定，缓存友好 |
| 三数取中 | 改进基准选择，降低最坏概率 |
| 归并排序 | 分半排再合，O(n log n)永远稳定，稳定，O(n)空间 |
| pdqsort | Go 1.19+: 插入(小数据)+快排(一般)+堆排(退化)，混合策略 |
| 稳定性 | 相等元素相对顺序不变，多级排序需要 |
| 选型指南 | 小数据用插入，大数据用pdqsort，需要稳定用归并 |
