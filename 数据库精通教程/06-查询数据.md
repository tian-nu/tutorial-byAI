## 我在做什么？

前两章你学会了建表（DDL）和增删改数据（DML）。现在，你有了数据，但还没有好好"看"过它们。

这一章，我们学 SQL 中使用频率最高、用法最丰富的一个操作：**SELECT（查询）**。

SELECT 属于 **DQL（Data Query Language，数据查询语言）** *此术语见附录E*。在整个 SQL 世界里，你 80% 的时间都在写 SELECT——查数据、看数据、分析数据。如果说 SQL 是一门语言，那 SELECT 就是这门语言里最常用的动词。

学完这一章，你能从表中筛选出你想要的列，给它们起别名，去掉重复数据，限制返回行数，甚至用表达式计算新值。

## 一、SELECT * vs SELECT 列名：为什么不要用 *

### 1.1 最基本的 SELECT

```sql
SELECT * FROM users;
```

`*` 表示"所有列"。这条 SQL 的意思是：从 `users` 表里取出所有行的所有列。

输出：

```
+----+----------+-------------------------+-----------------------------+---------------------+
| id | username | email                   | password_hash               | created_at          |
+----+----------+-------------------------+-----------------------------+---------------------+
|  1 | alice    | alice_wang@example.com  | hashed_password_alice_123   | 2025-06-05 10:00:00 |
|  2 | bob      | bob@gmail.com           | hashed_password_bob_456     | 2025-06-05 10:00:01 |
|  3 | charlie  | charlie@outlook.com     | hashed_password_charlie_789 | 2025-06-05 10:00:02 |
|  6 | frank    | frank@example.com       | hashed_password_frank       | 2025-06-05 10:00:03 |
+----+----------+-------------------------+-----------------------------+---------------------+
```

### 1.2 指定列名

```sql
SELECT username, email FROM users;
```

输出：

```
+----------+-------------------------+
| username | email                   |
+----------+-------------------------+
| alice    | alice_wang@example.com  |
| bob      | bob@gmail.com           |
| charlie  | charlie@outlook.com     |
| frank    | frank@example.com       |
+----------+-------------------------+
```

### 1.3 为什么不要用 `SELECT *`？

❌ 坏习惯：

```sql
SELECT * FROM users;  -- 即使在只需要 username 和 email 的时候
```

✅ 好习惯：

```sql
SELECT username, email FROM users;
```

原因有三：

**1. 性能**：`SELECT *` 会把所有列的数据都从磁盘读到内存，再通过网络传输给你。如果表有 20 列，但你需要 2 列，你就是多传输了 90% 的无用数据。在大表上，这个差别非常明显。

**2. 可读性**：别人看你的 SQL，`SELECT *` 不知道你关心哪些列。`SELECT username, email` 一目了然。

**3. 列顺序依赖**：如果你的代码（如 Python/Go）通过列的顺序来取值（如 `row[0]` 是 username），而某天有人用 `ALTER TABLE` 调整了列的顺序，你的代码就会静默出错——取到错误的值且不报错。

> **想多一点**：`SELECT *` 在开发时确实方便——你可以快速看一眼表里有什么数据。但**在生产代码中**（比如后端 API 的 SQL），永远指定列名。这不是"优化"，这是"正确性"问题。

## 二、列别名：给列起个中文名

有时列名不够直观，你可以用 `AS` 关键字给它起一个**别名（Alias）** *此术语见附录E*：

```sql
SELECT
    username AS 用户名,
    email    AS 邮箱,
    created_at AS 注册时间
FROM users;
```

输出：

```
+----------+-------------------------+---------------------+
| 用户名   | 邮箱                     | 注册时间             |
+----------+-------------------------+---------------------+
| alice    | alice_wang@example.com  | 2025-06-05 10:00:00 |
| bob      | bob@gmail.com           | 2025-06-05 10:00:01 |
| charlie  | charlie@outlook.com     | 2025-06-05 10:00:02 |
| frank    | frank@example.com       | 2025-06-05 10:00:03 |
+----------+-------------------------+---------------------+
```

`AS` 关键字可以省略：

```sql
SELECT username 用户名, email 邮箱 FROM users;  -- 同样有效，但可读性略差
```

建议保留 `AS`，让 SQL 更清晰。

## 三、DISTINCT：去重

`DISTINCT` *此术语见附录E* 用来去掉重复的行。

### 3.1 基本用法

假设订单表中有多条记录来自同一个用户：

```sql
SELECT user_id FROM orders;
```

输出：

```
+---------+
| user_id |
+---------+
|       1 |
|       1 |
|       2 |
|       3 |
|       3 |
+---------+
```

加 `DISTINCT`：

```sql
SELECT DISTINCT user_id FROM orders;
```

输出：

```
+---------+
| user_id |
+---------+
|       1 |
|       2 |
|       3 |
+---------+
```

只保留不重复的 `user_id` 值。

### 3.2 DISTINCT 的作用范围

`DISTINCT` 作用于**所有选择的列**，而不仅仅是紧跟其后的那一列：

```sql
SELECT DISTINCT user_id, product FROM orders;
```

这条 SQL 会去掉 `(user_id, product)` 组合重复的行——只有当 `user_id` 和 `product` **同时相同**时，才会被去重。

> 如果你只想对某一列去重，但还想看其他列……这是做不到的。因为去重后，其他列的值就没法确定了（同一个 user_id 可能有多个不同的 product）。这种需求通常需要用 GROUP BY 来解决，第 09 章会讲。

## 四、LIMIT：限制返回行数

`LIMIT` *此术语见附录E* 用来限制查询返回的行数。

### 4.1 基本用法

```sql
SELECT * FROM users LIMIT 2;
```

只返回前 2 行：

```
+----+----------+------------------------+---------------------------+---------------------+
| id | username | email                  | password_hash             | created_at          |
+----+----------+------------------------+---------------------------+---------------------+
|  1 | alice    | alice_wang@example.com | hashed_password_alice_123 | 2025-06-05 10:00:00 |
|  2 | bob      | bob@gmail.com          | hashed_password_bob_456   | 2025-06-05 10:00:01 |
+----+----------+------------------------+---------------------------+---------------------+
```

### 4.2 什么时候用？

- **开发调试**：只想知道表里有什么数据，不想看全部 100 万行。
- **TOP N 查询**：查询"最新注册的 10 个用户"（配合 ORDER BY，第 08 章讲）。
- **分页**：第 1 页返回 10 条，第 2 页返回第 11-20 条（第 08 章讲）。

## 五、表达式计算：SELECT 不只是查，还能算

SELECT 不仅能取出已有的数据，还能**计算**出新值。

### 5.1 基本计算

```sql
SELECT
    product,
    amount,
    amount * 0.13 AS tax,
    amount * 1.13 AS total_with_tax
FROM orders;
```

输出：

```
+--------------+----------+----------+----------------+
| product      | amount   | tax      | total_with_tax |
+--------------+----------+----------+----------------+
| iPhone 15    |  6999.00 |  909.870 |       7908.870 |
| iPad Air     |  3499.00 |  454.870 |       3953.870 |
| MacBook Pro  | 12999.00 | 1689.870 |      14688.870 |
| AirPods Pro  |  1899.00 |  246.870 |       2145.870 |
| Apple Watch  |  2999.00 |  389.870 |       3388.870 |
+--------------+----------+----------+----------------+
```

SELECT 支持常见的算术运算符：`+`、`-`、`*`、`/`。

### 5.2 字符串拼接

```sql
SELECT CONCAT(username, ' <', email, '>') AS 用户信息
FROM users;
```

`CONCAT()` 函数把多个字符串拼接成一个。输出：

```
+----------------------------------+
| 用户信息                          |
+----------------------------------+
| alice <alice_wang@example.com>   |
| bob <bob@gmail.com>              |
| charlie <charlie@outlook.com>    |
| frank <frank@example.com>        |
+----------------------------------+
```

## 六、常量和函数：SELECT 可以不依赖表

### 6.1 查询常量

```sql
SELECT 'Hello World' AS greeting, 1 + 1 AS result;
```

输出：

```
+--------------+--------+
| greeting     | result |
+--------------+--------+
| Hello World  |      2 |
+--------------+--------+
```

### 6.2 查询系统函数

```sql
SELECT NOW() AS 当前时间, VERSION() AS MySQL版本, DATABASE() AS 当前数据库;
```

输出：

```
+---------------------+------------+--------------+
| 当前时间             | MySQL版本  | 当前数据库    |
+---------------------+------------+--------------+
| 2025-06-05 14:30:00 | 8.0.37     | my_shop      |
+---------------------+------------+--------------+
```

- `NOW()`：当前日期时间
- `VERSION()`：MySQL 版本号
- `DATABASE()`：当前选中的数据库名

这些函数在调试时非常有用——比如你在写一个复杂的 SQL 脚本，想确认在哪个数据库里执行。

> **想多一点**：SELECT 可以不 FROM 任何表，这有点反直觉——"数据不是都从表里来吗？" 但 SELECT 的本质是"计算并返回结果"，数据来源可以是表、常量、表达式、函数。MySQL 会为没有 FROM 的 SELECT 创建一张虚拟的单行表，然后在这张虚拟表上计算。

## 我做得对不对？

### 验证方法

```sql
-- 1. 查询指定列，确认只返回了 username 和 email
SELECT username, email FROM users;
-- 应该看到 4 行，每行只有 username 和 email 两列

-- 2. 测试别名，确认列名变成了中文
SELECT username AS 用户名, email AS 邮箱 FROM users;
-- 列标题应该显示"用户名"和"邮箱"

-- 3. 测试 DISTINCT，确认去重后只有 3 个 user_id
SELECT DISTINCT user_id FROM orders;
-- 应该只有 1, 2, 3（没有重复）

-- 4. 测试 LIMIT，确认只返回了 2 行
SELECT * FROM users LIMIT 2;
-- 应该只有 2 行

-- 5. 测试表达式计算，确认 tax 和 total_with_tax 计算正确
SELECT product, amount, amount * 0.13 AS tax, amount * 1.13 AS total_with_tax
FROM orders;
-- 手动验算：iPhone 15 的 tax = 6999 * 0.13 = 909.87
```

## 不对怎么办？

### 常见错误 1：`SELECT *` 在大表上

❌ 错误做法：

```sql
-- 在 1000 万行的表上执行
SELECT * FROM huge_table;
```

MySQL 会把这 1000 万行的所有列都读出来，传输给你。可能导致：
- 网络传输超时
- 客户端内存耗尽
- 数据库服务器 CPU 飙升

✅ 正确做法：

```sql
-- 先看表结构
DESC huge_table;

-- 只查需要的列
SELECT id, username FROM huge_table LIMIT 10;
```

### 常见错误 2：DISTINCT 作用范围误解

❌ 错误认知：

```sql
-- 以为这条 SQL 只对 user_id 去重，product 会随机保留一行
SELECT DISTINCT user_id, product FROM orders;
```

✅ 正确认知：`DISTINCT user_id, product` 是对 `(user_id, product)` 的**组合**去重。只有当两个用户有完全相同的商品时，才会被去重。如果你只想对 user_id 去重，请用 GROUP BY（第 09 章）。

### 常见错误 3：列名写错

❌ 错误做法：

```sql
SELECT usrname FROM users;  -- 少写了 e
```

MySQL 报错：`ERROR 1054 (42S22): Unknown column 'usrname' in 'field list'`

✅ 正确做法：不确定列名时，先用 `DESC users;` 查看表结构，确保列名拼写正确。

### 常见错误 4：别名中用了特殊字符

❌ 错误做法：

```sql
SELECT username AS 用户 名 FROM users;  -- 别名中有空格，没加引号
```

MySQL 报错或产生意外行为。

✅ 正确做法：

```sql
SELECT username AS '用户 名' FROM users;  -- 有空格或特殊字符时用引号包裹
```

如果别名只包含中文、英文、数字和下划线，可以不引号。但建议统一用单引号，避免意外。

## 本章小结

| 本章学了什么 | 对应命令/概念 | 注意事项 |
|-------------|-------------|---------|
| 指定列查询 | `SELECT 列1, 列2 FROM 表名;` | 生产代码中不要用 `SELECT *`，性能差且可读性差 |
| 列别名 | `SELECT 列名 AS 别名 FROM 表名;` | `AS` 可省略但建议保留；别名有空格时用引号 |
| 去重 | `SELECT DISTINCT 列名 FROM 表名;` | DISTINCT 作用于所有选择的列，不只是紧跟的那一列 |
| 限制行数 | `SELECT ... LIMIT n;` | 开发调试和大表查询时必用 |
| 表达式计算 | `SELECT 列1 * 列2 AS 新列 FROM 表名;` | SELECT 不只是"查"，还能"算" |
| 字符串拼接 | `SELECT CONCAT(...)` | 比用 `+` 更安全（MySQL 中 `+` 只能做加法） |
| 常量和函数 | `SELECT NOW(), VERSION(), DATABASE();` | 不需要 FROM 表也能执行 SELECT |
