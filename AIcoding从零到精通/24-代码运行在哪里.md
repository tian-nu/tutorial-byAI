## 你在哪写代码，代码就在哪跑

代码不是凭空运行的——它需要一台电脑来执行。区别只在于：**这台电脑是你的，还是别人的。**

```
┌──────────────────────────────────────────────────────┐
│  本地运行（Local）📖                                   │
│  代码在你的电脑上跑                                     │
│  只有你能看到结果                                       │
│  电脑关机 → 程序停                                      │
│  不用花钱                                              │
│  ─────────────────────────────────────────────────   │
│  线上部署（Deploy）📖                                  │
│  代码在云服务器上跑                                     │
│  全世界都能访问                                         │
│  24小时不停机                                          │
│  需要花钱（租服务器）                                    │
└──────────────────────────────────────────────────────┘
```

---

## 本地运行：你电脑上的"私人厨房"

**本地运行**📖 = 代码在你的电脑上执行。Python文件存你硬盘上，双击运行，结果在你屏幕上显示。

**优点**：
- 免费——你自己的电脑，不用花钱租服务器
- 即时——写完立刻跑，立刻看到结果
- 私密——只有你能看到，不用担心数据泄露

**缺点**：
- 只有你能用——别人访问不了
- 关机就停——你关电脑，程序就没了
- 性能有限——你的电脑就那么大算力

**本书前三个项目（个人主页、Todo应用、数据仪表盘）都只要求本地运行。** 你不需要买服务器，不需要花钱，在自己的电脑上就能看到完整效果。

---

## 线上部署：把代码放到"云"上

**线上部署**📖 = 把代码放到**云服务器**📖（别人专门用来跑代码的电脑）上，让它24小时运行。

> 📖 **云服务器** = 一台放在数据中心、永远不关机的电脑，你按月租用。听起来高大上，其实就是"别人的电脑，你花钱远程用"。

部署之后的变化：

```
之前：你在浏览器输入 file:///C:/Users/你的名字/Desktop/我的网页/index.html
      只有你的电脑能打开

之后：你在浏览器输入 https://www.你的域名.com
      全世界任何一个人都能打开
```

**同一个网页，两种打开方式**：

| | 本地 | 线上 |
|------|------|------|
| 地址 | `file:///C:/Users/.../index.html` | `https://www.example.com` |
| 谁能访问 | 只有你 | 全世界 |
| 需要花钱吗 | 免费 | 服务器月租（几十到几百元） |
| 关机影响 | 程序停止 | 不受影响（服务器永远开着） |

---

## 部署需要什么

如果你想把一个项目部署到线上，通常需要：

| 需要的 | 说明 | 费用 |
|--------|------|------|
| **云服务器** | 一台远程电脑，常见的有阿里云、腾讯云、AWS | 几十元/月起 |
| **域名**（可选） | 就是 `www.xxx.com`，让别人好记 | 几十元/年 |
| **运维知识** | 装软件、配环境、看日志、修Bug | 学习成本 |

**对于新手来说，部署是一个不小的门槛。** 这也是为什么本书只要求在项目四（聊天机器人）的时候，如果你感兴趣，再尝试部署。

## 一个常见的误解

> "我在Cursor里写代码，AI帮我把代码跑起来了——这算本地运行还是线上？"

**算本地运行。** Cursor是在你电脑上跑的，AI帮你生成的代码也是在你的电脑上执行的。AI只是一个"顾问"，它不替你跑代码——你的电脑才是真正执行代码的地方。

## 练习

打开Cursor（或Trae），对着AI说：

> "帮我写一个最简单的Python Flask网页应用，只有一个页面，显示'Hello，我现在运行在本地！'。告诉我怎么在本地运行它，以及怎么在浏览器打开。"

AI生成代码后：
1. 按AI说的步骤运行起来
2. 在浏览器里打开 `http://127.0.0.1:5000`
3. 观察浏览器地址栏——这是本地地址，不是线上地址
4. 试着把 `127.0.0.1` 换成 `localhost`，看看效果是不是一样
