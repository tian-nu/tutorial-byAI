# 103-微服务通信

> 💡 微服务拆开了，但它们还是要互相"说话"。餐厅前台（API 网关）告诉后厨（订单服务）"一份宫保鸡丁"，后厨告诉采购（库存服务）"鸡肉不够了"。本章梳理微服务间通信的三种方式：同步 HTTP、异步消息、RPC，并给出选择指南。

---

## 本章目标
- 掌握微服务间的三种通信方式
- 理解 Feign 声明式调用的原理
- 掌握服务降级与熔断（Sentinel）
- 知道什么时候用同步、什么时候用异步

---

## 103.1 三种通信方式对比

| | 同步 HTTP (Feign) | 异步消息 (MQ) | RPC (gRPC/Dubbo) |
|------|------|------|------|
| 协议 | HTTP/1.1 | AMQP / Kafka 协议 | HTTP/2 (gRPC) / TCP (Dubbo) |
| 序列化 | JSON | 自定义 | Protobuf (二进制) |
| 实时性 | 立即返回 | 异步处理 | 立即返回 |
| 性能 | 低 | 中 | 高 |
| 耦合度 | 高（服务必须在线） | 低（通过中间件解耦） | 高 |
| 调试难度 | 低（curl 可测） | 中（需看消息内容） | 高（二进制不可读） |

## 103.2 同步调用 — OpenFeign

### 依赖

```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-openfeign</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-loadbalancer</artifactId>
</dependency>
```

### 启用 Feign

```java
@SpringBootApplication
@EnableFeignClients
public class OrderApplication {
    public static void main(String[] args) {
        SpringApplication.run(OrderApplication.class, args);
    }
}
```

### 声明接口

```java
@FeignClient(name = "user-service", path = "/api/users")
public interface UserClient {

    @GetMapping("/{id}")
    Result<UserDTO> getUserById(@PathVariable Long id);

    @GetMapping("/batch")
    Result<List<UserDTO>> getUsersByIds(@RequestParam List<Long> ids);
}

@FeignClient(name = "inventory-service", path = "/api/inventory")
public interface InventoryClient {

    @PostMapping("/deduct")
    Result<Void> deduct(@RequestBody DeductRequest request);
}
```

### 使用

```java
@Service
@RequiredArgsConstructor
public class OrderService {

    private final UserClient userClient;
    private final InventoryClient inventoryClient;

    @Transactional
    public OrderDTO createOrder(CreateOrderRequest request) {
        // 1. 查用户信息
        Result<UserDTO> userResult = userClient.getUserById(request.getUserId());
        UserDTO user = userResult.getData();

        // 2. 扣库存
        inventoryClient.deduct(new DeductRequest(request.getSkuId(), request.getQuantity()));

        // 3. 创建订单
        // ...
    }
}
```

> 🤔 想多一点：`@FeignClient(name = "user-service")` 不需要写 IP！Feign 会通过服务名去注册中心查询实例列表，并自动负载均衡。如果你改了 user-service 的 IP，订单服务零改动。

## 103.3 服务降级与熔断 — Sentinel

### 问题

用户服务挂了 → 订单服务调用用户服务的线程全部阻塞 → 订单服务也假死了 → 雪崩。

### 熔断（Circuit Breaker）：快速失败

```
状态机：
  CLOSED（正常） → 连续失败 N 次 → OPEN（熔断，直接返回 fallback）
  OPEN → 等待 M 秒 → HALF_OPEN（试探放一个请求）
  HALF_OPEN → 成功 → CLOSED（恢复）
  HALF_OPEN → 失败 → OPEN（继续熔断）
```

### Feign + Sentinel 集成

```java
// Fallback 工厂
@Component
public class UserClientFallbackFactory implements FallbackFactory<UserClient> {
    @Override
    public UserClient create(Throwable cause) {
        return new UserClient() {
            @Override
            public Result<UserDTO> getUserById(Long id) {
                log.error("调用 user-service 失败: {}", cause.getMessage());
                // 返回兜底数据
                UserDTO defaultUser = new UserDTO();
                defaultUser.setId(id);
                defaultUser.setUsername("用户数据暂不可用");
                return Result.success(defaultUser);
            }
        };
    }
}

@FeignClient(name = "user-service", path = "/api/users",
             fallbackFactory = UserClientFallbackFactory.class)
public interface UserClient { ... }
```

```yaml
# sentinel 配置
spring:
  cloud:
    sentinel:
      transport:
        dashboard: localhost:8080   # Sentinel 控制台
      datasource:
        ds1:
          nacos:
            server-addr: localhost:8848
```

## 103.4 异步通信 — 消息队列

当调用不需要立刻返回结果时，用消息队列更好：

```
订单服务                              短信服务
   │                                     │
   │ 发消息"订单已创建"到 MQ               │
   │────────────────────────────→         │
   │                                     │ 等消息到达
   │ 立刻返回"订单已创建"                  │
   │                                     │ 消费消息 → 发送短信
```

```java
// 订单服务发送事件（不关心谁来消费）
@Service
@RequiredArgsConstructor
public class OrderService {
    private final RabbitTemplate rabbitTemplate;

    public void createOrder(Order order) {
        // ... 创建订单
        rabbitTemplate.convertAndSend("order.exchange", "order.created", order);
    }
}
```

## 103.5 通信方式选择指南

```
需要立即返回结果？
├── 是 → 调用频率高 + 对性能敏感？
│        ├── 是 → gRPC（Protobuf + HTTP/2）
│        └── 否 → Feign（HTTP + JSON）
└── 否 → 下游有多个消费者？
         ├── 是 → 消息队列（发布/订阅）
         └── 否 → 消息队列 或 异步 HTTP
```

## 103.6 小结

| 知识点 | 核心内容 |
|--------|----------|
| Feign | `@FeignClient` + 服务名 → 自动发现 + 负载均衡 |
| 熔断 | 快速失败，防止雪崩（CLOSED → OPEN → HALF_OPEN） |
| 降级 | 返回兜底数据（FallbackFactory） |
| 异步通信 | 消息队列发布事件，解耦 + 削峰 |
| 选择 | 即时响应=Feign/gRPC；可异步=MQ；高性能=gRPC |

## 103.7 自测题

**1. `@FeignClient(name = "user-service")` 中为什么只写了服务名，没写 IP 地址？Feign 怎么知道 user-service 在哪？**

**2. 什么是服务雪崩？熔断器如何防止雪崩？**

**3. 订单服务创建订单后需要"扣库存"和"发短信"。哪个用同步调用，哪个用异步消息？为什么？**

---

**答案提示**：1→Feign 从注册中心（Nacos/Consul）根据服务名查询实例列表（IP+端口），然后通过 LoadBalancer 选择其中一个。2→一个服务故障 → 调用方的线程全部阻塞等待 → 调用方也故障 → 连锁反应。熔断器检测到连续失败后直接返回 fallback，不给故障服务发请求，保护调用方。3→扣库存用同步（Feign），因为创建订单必须确认库存扣除成功；发短信用异步（MQ），因为发短信不决定订单成败，且可以异步处理。下一章——分布式基础理论。
