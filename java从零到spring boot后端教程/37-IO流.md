# 第37章 IO 流

> 想象你用水管浇花——水从水龙头通过水管流到花盆。Java 的 IO 流就是这样的"数据水管"：一端连接数据源（文件、网络、内存），另一端连接你的程序。数据像水流一样，一个字节一个字节（或一个字符一个字符）地流过。

---

## 37.1 IO 流分类

Java IO 流可以从两个维度分类：

```
                    ┌── 字节流（InputStream/OutputStream）→ 处理二进制（图片、视频、任何文件）
IO 流 ──┤
                    └── 字符流（Reader/Writer）→ 处理文本（自动编解码）
```

```
                    ┌── 节点流（直接连接数据源：FileInputStream）
IO 流 ──┤
                    └── 包装流（套在节点流上增强功能：BufferedInputStream）
```

**核心原则**：字节流能处理一切，字符流专为文本设计。读文本用字符流，读二进制用字节流。

---

## 37.2 字节流

### 37.2.1 FileInputStream / FileOutputStream

```java
import java.io.*;

// 写文件
try (FileOutputStream fos = new FileOutputStream("test.txt")) {
    fos.write("Hello, 世界!".getBytes());
    // getBytes() 使用平台默认编码——可能在不同系统上表现不同
}

// 读文件
try (FileInputStream fis = new FileInputStream("test.txt")) {
    byte[] buffer = new byte[1024];
    int bytesRead;
    while ((bytesRead = fis.read(buffer)) != -1) {
        String content = new String(buffer, 0, bytesRead);
        System.out.println(content);
    }
}
```

> ⚠️ `read()` 返回 `-1` 表示读到文件末尾。不要用 `read()` 是否返回 0 来判断！

### 37.2.2 BufferedInputStream / BufferedOutputStream

无缓冲的 IO 每次 `read()`/`write()` 都会触发系统调用（磁盘访问），非常慢。缓冲流在内存中维护一个缓冲区，批量读写，大幅提升性能。

```java
// 无缓冲 vs 有缓冲性能对比
// 复制一个 10MB 文件

// ❌ 无缓冲：每次一个字节读写
try (FileInputStream fis = new FileInputStream("source.dat");
     FileOutputStream fos = new FileOutputStream("dest.dat")) {
    int b;
    while ((b = fis.read()) != -1) {
        fos.write(b);       // 每个字节一次系统调用，极慢
    }
}

// ✅ 有缓冲：8KB 缓冲区
try (BufferedInputStream bis = new BufferedInputStream(
         new FileInputStream("source.dat"));
     BufferedOutputStream bos = new BufferedOutputStream(
         new FileOutputStream("dest.dat"))) {
    byte[] buffer = new byte[8192];
    int bytesRead;
    while ((bytesRead = bis.read(buffer)) != -1) {
        bos.write(buffer, 0, bytesRead);
    }
}
```

性能差异可能是 **100 倍以上**。永远给文件流套上缓冲！

## 37.3 字符流

### 37.3.1 FileReader / FileWriter

```java
// 写文本文件
try (FileWriter fw = new FileWriter("poem.txt")) {
    fw.write("床前明月光\n");
    fw.write("疑是地上霜\n");
}

// 读文本文件
try (FileReader fr = new FileReader("poem.txt")) {
    char[] buffer = new char[1024];
    int charsRead;
    while ((charsRead = fr.read(buffer)) != -1) {
        System.out.print(new String(buffer, 0, charsRead));
    }
}
```

### 37.3.2 BufferedReader / BufferedWriter

```java
// 按行读取（最常用）
try (BufferedReader br = new BufferedReader(new FileReader("data.txt"))) {
    String line;
    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
}

// 写入（带缓冲）
try (BufferedWriter bw = new BufferedWriter(new FileWriter("output.txt"))) {
    bw.write("第一行");
    bw.newLine();  // 跨平台换行符
    bw.write("第二行");
}
```

### 37.3.3 InputStreamReader / OutputStreamWriter

这两个是**字节流和字符流之间的桥梁**，可以指定编码：

```java
// 以 UTF-8 编码读取文件（推荐）
try (BufferedReader br = new BufferedReader(
         new InputStreamReader(
             new FileInputStream("data.txt"), StandardCharsets.UTF_8))) {
    String line;
    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
}

// 以 UTF-8 编码写入文件
try (BufferedWriter bw = new BufferedWriter(
         new OutputStreamWriter(
             new FileOutputStream("output.txt"), StandardCharsets.UTF_8))) {
    bw.write("包含中文的内容");
}
```

> ⚠️ **始终指定编码**！`FileReader` 使用系统默认编码，在 Windows（通常是 GBK）和 Linux（通常是 UTF-8）上行为不同，这是无数 bug 的根源。

## 37.4 try-with-resources（自动关闭）

Java 7 引入的 `try-with-resources` 语法，确保流在使用后自动关闭——即使发生异常也不会泄漏资源。

```java
// 旧方式（Java 6 及以前）：手动 finally 关闭，又臭又长
FileInputStream fis = null;
try {
    fis = new FileInputStream("data.txt");
    // 使用 fis...
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (fis != null) {
        try {
            fis.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

// ✅ 新方式：try-with-resources，自动关闭
try (FileInputStream fis = new FileInputStream("data.txt");
     BufferedInputStream bis = new BufferedInputStream(fis)) {
    int b;
    while ((b = bis.read()) != -1) {
        System.out.print((char) b);
    }
} catch (IOException e) {
    e.printStackTrace();
}
// fis 和 bis 都会自动关闭，顺序是声明的反序
```

原理：`try-with-resources` 要求资源实现 `AutoCloseable` 接口（`InputStream`、`OutputStream`、`Reader`、`Writer` 都实现了）。JVM 在 try 块结束后自动调用 `close()`。

## 37.5 java.nio.file.Files 工具类（NIO.2）

Java 7 的 NIO.2（`java.nio.file`）提供了更现代的文件操作 API。**现代项目优先使用这个**。

```java
import java.nio.file.*;
import java.nio.charset.StandardCharsets;
import java.util.List;

// 读取全部行
List<String> lines = Files.readAllLines(Path.of("data.txt"), StandardCharsets.UTF_8);

// 读取全部字节
byte[] bytes = Files.readAllBytes(Path.of("image.png"));

// 读取为字符串（Java 11+）
String content = Files.readString(Path.of("data.txt"), StandardCharsets.UTF_8);

// 写入字符串
Files.writeString(Path.of("output.txt"), "Hello, World!", StandardCharsets.UTF_8);

// 逐行读取（适合大文件——不会一次加载到内存）
try (var reader = Files.newBufferedReader(Path.of("large.txt"), StandardCharsets.UTF_8)) {
    String line;
    while ((line = reader.readLine()) != null) {
        // 处理每一行
    }
}

// 流式读取（Java 8+）
try (Stream<String> lines = Files.lines(Path.of("data.txt"), StandardCharsets.UTF_8)) {
    lines.filter(line -> !line.isBlank())
         .forEach(System.out::println);
}

// 文件和目录操作
boolean exists = Files.exists(Path.of("data.txt"));
Files.createDirectories(Path.of("a/b/c"));  // 递归创建目录（mkdir -p）
Files.copy(Path.of("src.txt"), Path.of("dest.txt"), StandardCopyOption.REPLACE_EXISTING);
Files.move(Path.of("old.txt"), Path.of("new.txt"));
Files.delete(Path.of("temp.txt"));
// Files.deleteIfExists(Path.of("temp.txt"));  // 不存在也不报错
```

## 37.6 IO vs NIO 选择指南

| 场景 | 推荐方案 |
|------|---------|
| 小文本文件读全部内容 | `Files.readString()` |
| 小文本文件逐行处理 | `Files.readAllLines()` |
| 大文本文件逐行处理 | `Files.newBufferedReader()` |
| 二进制文件复制 | `Files.copy()` |
| 需要和旧代码兼容 | `FileInputStream`/`BufferedReader` 等传统 IO |
| 高性能网络 IO | NIO 的 `Channel` 和 `Buffer`（本教程不深入） |

## 37.7 ❌ 常见错误

### 错误1：不关闭流

```java
// ❌ 资源泄漏！文件一直被占用
FileInputStream fis = new FileInputStream("data.txt");
// ... 使用 ...
// 忘记关闭！

// ✅ 用 try-with-resources
try (FileInputStream fis = new FileInputStream("data.txt")) {
    // ... 使用 ...
} // 自动关闭
```

### 错误2：不指定编码

```java
// ❌ FileReader 使用系统默认编码
BufferedReader br = new BufferedReader(new FileReader("data.txt"));

// ✅ 始终指定 UTF-8
BufferedReader br = new BufferedReader(
    new InputStreamReader(new FileInputStream("data.txt"), StandardCharsets.UTF_8)
);

// ✅ 或用 Files 工具类（它也指定了编码）
BufferedReader br = Files.newBufferedReader(Path.of("data.txt"), StandardCharsets.UTF_8);
```

### 错误3：大文件一次读完

```java
// ❌ 大文件（比如 2GB）一次读入内存可能 OOM
String content = Files.readString(Path.of("huge.log"));  // OutOfMemoryError！

// ✅ 逐行处理
try (BufferedReader br = Files.newBufferedReader(Path.of("huge.log"))) {
    String line;
    while ((line = br.readLine()) != null) {
        processLine(line);
    }
}
```

### 错误4：read() 返回值判断错误

```java
// ❌ 不能用 read() == 0 判断结束
byte[] buffer = new byte[1024];
int bytesRead;
// while ((bytesRead = fis.read(buffer)) != 0) { ... } // 死循环或提前终止

// ✅ 用 read() == -1 判断结束
while ((bytesRead = fis.read(buffer)) != -1) {
    // 处理数据
}
```

## 37.8 小结

| 知识点 | 一句话总结 |
|--------|-----------|
| 字节流 vs 字符流 | 字节流处理一切，字符流处理文本 |
| 缓冲流 | 必须套 BufferedStream，否则极慢 |
| try-with-resources | 自动关闭流，无需手动 finally |
| 编码 | 始终显式指定 UTF-8，不用系统默认 |
| Files 工具类 | NIO.2 现代 API，读写文件首选 |
| 大文件 | 逐行/分块读取，不要一次加载 |

## 37.9 自测题

**1.** 以下代码有什么问题？

```java
FileWriter fw = new FileWriter("output.txt");
fw.write("Hello");
// 忘记关闭
```

**2.** 用最简洁的方式写出以下操作：
- 读取 `config.json` 的全部文本内容（UTF-8）
- 将 `list`（`List<String>`）的每一行写入 `output.txt`，每行以换行分隔

**3.** 为什么说"永远给 FileInputStream 套上 BufferedInputStream"？不套会有什么后果？

> 答案提示：1. 文件内容可能没真正写入磁盘（缓冲区未刷新），且文件句柄泄漏；2. `Files.readString(Path.of("config.json"))` 和 `Files.write(Path.of("output.txt"), list)`；3. 每次 read/write 都是系统调用，无缓冲时每个字节都发起系统调用，速度慢 100+ 倍。
