# 82-事务管理

> 💡 你去银行转账：从 A 账户扣 1000，往 B 账户加 1000。如果扣完 A 的钱后银行系统宕机了——A 的钱扣了但 B 没收到？那你就破产了。数据库事务就是银行的"原子操作保险"：要么两步都成功，要么两步都回滚——A 的钱一分不少回到账上。本章不只是教你 `@Transactional`，更要让你理解它什么时候生效、什么时候不生效。

---

## 本章目标

## 82.1 什么是事务——转账的故事

ACID 四特性：

| 特性 | 全称 | 转账场景解释 |
|------|------|-------------|
| A | Atomicity（原子性） | A 扣钱 + B 加钱，一个整体，不可分割 |
| C | Consistency（一致性） | 转账前后，A+B 总金额不变 |
| I | Isolation（隔离性） | A 转账不影响 C 同时转账 |
| D | Durability（持久性） | 转账完成后，钱不会"丢失" |

```java
@Service
public class TransferService {

    @Transactional(rollbackFor = Exception.class)
    public void transfer(Long fromId, Long toId, BigDecimal amount) {
        Account from = accountRepository.findById(fromId)
                .orElseThrow(() -> new BusinessException("转出账户不存在"));
        Account to = accountRepository.findById(toId)
                .orElseThrow(() -> new BusinessException("转入账户不存在"));

        if (from.getBalance().compareTo(amount) < 0) {
            throw new BusinessException("余额不足");
        }

        from.setBalance(from.getBalance().subtract(amount));
        to.setBalance(to.getBalance().add(amount));

        accountRepository.save(from);
        accountRepository.save(to);
        // 任何一步失败，from 的扣款自动回滚
    }
}
```

## 82.2 @Transactional 核心属性

```java
@Transactional(
    propagation = Propagation.REQUIRED,      // 传播行为
    isolation = Isolation.DEFAULT,           // 隔离级别
    timeout = 30,                            // 超时秒数
    readOnly = false,                        // 是否只读
    rollbackFor = Exception.class,           // 什么异常回滚
    noRollbackFor = IllegalArgumentException.class  // 什么异常不回滚
)
```

### rollbackFor——最重要的属性

Spring 事务默认只在 **RuntimeException 及其子类** 和 **Error** 抛出时回滚。**受检异常默认不回滚！**

```java
// 受检异常不回滚——危险！
@Transactional
public void transfer() throws IOException {
    accountRepository.save(from);  // 不会被回滚！
    throw new IOException("网络异常");
}

// 正确：声明 rollbackFor
@Transactional(rollbackFor = Exception.class)
public void transfer() throws IOException {
    // 所有异常都回滚
}
```

> **最佳实践：永远写 `@Transactional(rollbackFor = Exception.class)`。**

## 82.3 事务传播行为——方法套方法时怎么办

| 传播行为 | 说明 | 场景 |
|----------|------|------|
| `REQUIRED`（默认） | 有事务就加入，没有就新建 | 大多数业务方法 |
| `REQUIRES_NEW` | 总是新建事务，挂起当前事务 | 日志记录（失败不影响主业务） |
| `SUPPORTS` | 有事务就加入，没有就非事务执行 | 查询方法 |
| `NESTED` | 嵌套事务（savepoint） | 部分回滚 |

### 实战：日志记录不能影响主业务

```java
@Service
public class OrderService {
    @Transactional
    public void createOrder(Order order) {
        orderRepository.save(order);    // 订单必须成功
        logService.recordLog(order);    // 日志失败不能回滚订单！
    }
}

@Service
public class LogService {
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void recordLog(Order order) {
        logRepository.save(new Log(order.getId()));
        // 这里抛异常只回滚日志事务，订单不受影响
    }
}
```

## 82.4 同类方法调用事务失效——最经典的坑

```java
@Service
public class UserService {

    @Transactional
    public void methodA() {
        userRepository.save(new User());
        this.methodB();  // ← 同类内部直接调用！
    }

    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void methodB() {
        // 这个事务不生效！methodB 在 methodA 的事务中执行
        userRepository.save(new User());
    }
}
```

**原因**：`@Transactional` 通过 Spring AOP 代理实现。外部调用 `userService.methodA()` 走的是代理对象。但 `this.methodB()` 是 Java 普通方法调用——直接调用目标对象本身，绕过了代理，事务注解失效。

```
外部调用 userService.methodA()
  → 代理对象.methodA()  ← 有事务增强
    → 目标对象.methodA()
      → this.methodB()  ← 直接调用，绕过代理！事务失效！
```

### 解决方案一：注入自身

```java
@Service
public class UserService {
    @Autowired
    private UserService self;

    @Transactional
    public void methodA() {
        userRepository.save(new User());
        self.methodB();  // 通过代理调用，事务生效
    }

    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void methodB() {
        userRepository.save(new User());
    }
}
```

### 解决方案二：拆分到不同的 Service

```java
@Service
public class UserServiceA {
    @Autowired
    private UserServiceB userServiceB;

    @Transactional
    public void methodA() {
        userRepository.save(new User());
        userServiceB.methodB();  // 跨类调用，走代理
    }
}

@Service
public class UserServiceB {
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void methodB() {
        userRepository.save(new User());
    }
}
```

## 82.5 @Transactional 不生效的其他场景

| 场景 | 原因 | 解决方案 |
|------|------|----------|
| 同类内部调用 | 绕过代理 | 注入自身或拆分 Service |
| 方法不是 public | AOP 只代理 public 方法 | 改为 public |
| 异常被 catch 吞掉 | 事务管理器感知不到异常 | 手动 setRollbackOnly() |
| 数据库引擎不支持事务 | MyISAM 不支持 | 使用 InnoDB |
| 非 Spring Bean | 没有代理 | 确保 @Service/@Component |

## 82.6 小结

| 知识点 | 核心内容 |
|--------|----------|
| ACID | 原子性、一致性、隔离性、持久性 |
| @Transactional | 声明式事务核心注解 |
| rollbackFor | 必须显式设为 Exception.class |
| 传播行为 | REQUIRED（默认）、REQUIRES_NEW（新事务） |
| 同类内部调用 | 绕过代理，事务失效的经典陷阱 |
| 解决方案 | 注入自身 / 拆分 Service / AopContext.currentProxy() |

```java
public class OrderService {
    @Transactional
    public void methodA() {
        orderRepository.save(new Order());
        this.methodB();
    }
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void methodB() {
        logRepository.save(new Log());
    }
}
```
