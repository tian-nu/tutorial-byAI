# 123-项目五：IM即时通讯

> 💡 本章你会做出一个**基于 WebSocket 的实时聊天系统**。完成后的效果：A 在浏览器里发一条消息，B 的页面立刻弹出来，无需刷新。支持一对一私聊和公共群聊。这是从"请求-响应"模式跨越到"服务端主动推送"模式的关键一步。

---

## 本章目标
- 理解 WebSocket 协议与 HTTP 的区别
- 使用 Spring WebSocket + STOMP 协议
- 实现一对一点对点聊天
- 实现公共聊天室（广播）
- 前端使用 SockJS + STOMP.js

## 123.1 WebSocket vs HTTP

```
HTTP:                         WebSocket:
客户端 ↦ 请求                 客户端 ↤↦ 服务端
服务端 ↤ 响应                 双向全双工, 一次握手后持续连接
(一问一答, 服务端不能主动发)  (服务端可以随时推送)
```

## 123.5 WebSocketConfig

```java
package com.example.im.config;

import org.springframework.context.annotation.Configuration;
import org.springframework.messaging.simp.config.MessageBrokerRegistry;
import org.springframework.web.socket.config.annotation.*;

@Configuration
@EnableWebSocketMessageBroker
public class WebSocketConfig implements WebSocketMessageBrokerConfigurer {

    @Override
    public void configureMessageBroker(MessageBrokerRegistry registry) {
        registry.enableSimpleBroker("/topic", "/queue");
        registry.setApplicationDestinationPrefixes("/app");
        registry.setUserDestinationPrefix("/user");
    }

    @Override
    public void registerStompEndpoints(StompEndpointRegistry registry) {
        registry.addEndpoint("/ws")
                .setAllowedOriginPatterns("*")
                .withSockJS();
    }
}
```

### 配置解析

| 配置 | 含义 |
|------|------|
| `/app` | 客户端发消息的前缀，如 `/app/chat.send` |
| `/topic` | 广播目的地（群聊），订阅了就能收到 |
| `/queue` | 点对点目的地（私聊），只有指定用户能收到 |
| `/user` | 用户目的地前缀，配合 `/queue` 实现私聊 |
| `/ws` | WebSocket 握手端点 |

```java
package com.example.im.entity;

import jakarta.persistence.*;
import lombok.Data;

@Data
@Entity
@Table(name = "chat_message")
public class ChatMessage {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "sender_id", nullable = false)
    private Long senderId;

    @Column(name = "receiver_id")
    private Long receiverId;

    @Column(nullable = false, length = 2000)
    private String content;

    @Column(length = 50)
    private String room;

    @Column(name = "message_type", length = 20)
    private String messageType = "CHAT";

    @Column(name = "sent_at", updatable = false)
    private java.time.LocalDateTime sentAt;

    @PrePersist
    protected void onCreate() {
        sentAt = java.time.LocalDateTime.now();
    }
}
```

## 123.8 ChatController

```java
package com.example.im.controller;

import com.example.im.dto.MessagePayload;
import com.example.im.entity.ChatMessage;
import com.example.im.service.ChatService;
import lombok.RequiredArgsConstructor;
import org.springframework.messaging.handler.annotation.MessageMapping;
import org.springframework.messaging.handler.annotation.Payload;
import org.springframework.messaging.simp.SimpMessagingTemplate;
import org.springframework.stereotype.Controller;

@Controller
@RequiredArgsConstructor
public class ChatController {

    private final SimpMessagingTemplate messagingTemplate;
    private final ChatService chatService;

    @MessageMapping("/chat.send")
    public void sendMessage(@Payload MessagePayload payload) {
        ChatMessage saved = chatService.saveMessage(payload);

        if (payload.getReceiverId() != null) {
            messagingTemplate.convertAndSendToUser(
                    payload.getReceiverId().toString(),
                    "/queue/messages",
                    saved
            );
        }
    }

    @MessageMapping("/chat.room")
    public void sendToRoom(@Payload MessagePayload payload) {
        ChatMessage saved = chatService.saveMessage(payload);

        messagingTemplate.convertAndSend(
                "/topic/room/" + payload.getRoom(),
                saved
        );
    }
}
```

### 关键解释

| 方法 | 路径 | 效果 |
|------|------|------|
| `convertAndSendToUser` | `/user/{userId}/queue/messages` | 只发给指定用户（私聊） |
| `convertAndSend` | `/topic/room/{room}` | 广播给所有订阅该房间的用户（群聊） |

## 123.9 ChatService

```java
package com.example.im.service;

import com.example.im.dto.MessagePayload;
import com.example.im.entity.ChatMessage;
import com.example.im.repository.ChatMessageRepository;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service
@RequiredArgsConstructor
public class ChatService {

    private final ChatMessageRepository messageRepository;

    @Transactional
    public ChatMessage saveMessage(MessagePayload payload) {
        ChatMessage message = new ChatMessage();
        message.setSenderId(payload.getSenderId());
        message.setReceiverId(payload.getReceiverId());
        message.setContent(payload.getContent());
        message.setRoom(payload.getRoom());
        message.setMessageType(payload.getType() != null ? payload.getType() : "CHAT");
        return messageRepository.save(message);
    }
}
```

## 123.10 前端聊天页面（关键 JS）

```html
<!DOCTYPE html>
<html>
<head><meta charset="UTF-8"><title>IM Chat</title></head>
<body>
<div id="messages"></div>
<input id="content" placeholder="输入消息...">
<button onclick="sendToRoom()">发送到群聊</button>
<button onclick="sendPrivate()">私聊给用户2</button>

<script src="https://cdn.jsdelivr.net/npm/sockjs-client@1/dist/sockjs.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/@stomp/stompjs@7/bundles/stomp.umd.min.js"></script>
<script>
const stompClient = new StompJs.Client({
    brokerURL: 'ws://localhost:8080/ws'
});

stompClient.onConnect = function(frame) {
    console.log('Connected: ' + frame);

    // 订阅群聊
    stompClient.subscribe('/topic/room/general', function(message) {
        showMessage(JSON.parse(message.body));
    });

    // 订阅私聊
    stompClient.subscribe('/user/queue/messages', function(message) {
        showMessage(JSON.parse(message.body));
    });
};

stompClient.onStompError = function(frame) {
    console.error('Error: ' + frame.headers['message']);
};

stompClient.activate();

function sendToRoom() {
    stompClient.publish({
        destination: '/app/chat.room',
        body: JSON.stringify({
            senderId: 1, senderName: 'Alice',
            content: document.getElementById('content').value,
            room: 'general', type: 'CHAT'
        })
    });
}

function sendPrivate() {
    stompClient.publish({
        destination: '/app/chat.send',
        body: JSON.stringify({
            senderId: 1, senderName: 'Alice',
            receiverId: 2,
            content: document.getElementById('content').value,
            type: 'CHAT'
        })
    });
}

function showMessage(msg) {
    const div = document.getElementById('messages');
    div.innerHTML += '<p><b>' + msg.senderId + ':</b> ' + msg.content + '</p>';
}
</script>
</body>
</html>
```

## 123.12 完成效果

成功运行后：
1. **群聊**：任意用户发送消息到 `/topic/room/general`，所有订阅者同时收到
2. **私聊**：消息只推送给 `receiverId` 指定的用户
3. **消息持久化**：所有消息存入 MySQL
4. **双向实时**：无需轮询，服务端主动推送

## 自测题

1. WebSocket 连接建立后，客户端和服务端谁可以主动发消息？
2. `/topic` 和 `/queue` 在 STOMP 中的语义区别是什么？
3. 如果不用 STOMP，直接用原始 WebSocket 怎么写？

<details>
<summary>点击查看答案</summary>

1. 双方都可以随时主动发消息。WebSocket 是全双工协议，连接保持期间客户端和服务端地位对等。
2. `/topic` 是发布-订阅模式（一对多广播），消息到达后转发给所有订阅者；`/queue` 是点对点模式（一对一），消息只发给指定用户。
3. 实现 `WebSocketHandler` 接口，处理 `afterConnectionEstablished`、`handleTextMessage`、`afterConnectionClosed` 等事件。但需要自己管理会话（Session）列表、自己实现路由逻辑，代码量远大于 STOMP。
</details>
