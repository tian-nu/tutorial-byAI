## 我在做什么？

上一章你认识了整数和小数，但计算机不仅处理数字——它还要处理文字。`"Hello, World!"` 里面的每个字母，本质上是什么？`\n` 这种看不见的字符，怎么表达？

这一章聚焦两个特殊的数据类型：
1. **`char`**：它身兼两职——既是"最小的整数类型"，又是"字符类型"。你会理解 `*ASCII*（此术语见附录F）` 码表，看懂那个经典的 `'A' + 1 = 'B'` 公式。
2. **布尔**：在C语言中，"真"和"假"并不像其他语言那样有独立的 `bool` 类型。C89标准压根没有布尔类型，C99才引入 `_Bool`。你会看到事情有多"将就"。

## 10.1 `char` 的双重身份

`char` 是C语言中最"精神分裂"的类型。它既是整数，又是字符。

```c
char c = 65;       /* 存的是整数 65 */
char c = 'A';      /* 存的也是整数 65 */
```

上面两行代码在内存中完全等价——都往 `c` 里存了数字 `65`。区别只在于你**怎么解释**这个数字。

**比喻**：`char` 就像一个双面硬币。翻到这一面，它是整数（-128到127）；翻到另一面，它是字符（`'A'`到`'z'`等）。哪个面朝上，取决于你怎么用它。

## 10.2 ASCII 码表

`*ASCII*（此术语见附录F）`（American Standard Code for Information Interchange）是一张"字符到数字"的映射表。它定义了128个字符对应的数字编号。

### 10.2.1 核心区域

```
ASCII码表（部分，十进制）
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│ 编号 │ 字符 │ 编号 │ 字符 │ 编号 │ 字符 │ 编号 │ 字符 │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┤
│  0  │ NUL │  32 │ 空格│  64 │  @  │  96 │  `  │
│  ...│ ... │  33 │  !  │  65 │  A  │  97 │  a  │
│  10 │ \n  │  ...│ ... │  66 │  B  │  98 │  b  │
│  13 │ \r  │  48 │  0  │  ...│ ... │  ...│ ... │
│  ...│ ... │  49 │  1  │  90 │  Z  │ 122 │  z  │
│  26 │ EOF │  ...│ ... │  ...│ ... │  ...│ ... │
│  ...│ ... │  57 │  9  │  ...│ ... │  ...│ ... │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
```

关键要记住的：
- `'0'` = 48, `'1'` = 49, ..., `'9'` = 57（数字字符的ASCII值不等于数字本身！）
- `'A'` = 65, `'B'` = 66, ..., `'Z'` = 90
- `'a'` = 97, `'b'` = 98, ..., `'z'` = 122
- 空格 = 32
- 换行 `\n` = 10

### 10.2.2 字符算术

因为 `char` 本质是整数，所以可以参与算术运算：

```c
#include <stdio.h>

int main(void)
{
    char ch = 'A';

    printf("%c\n", ch);       /* 输出 A */
    printf("%d\n", ch);       /* 输出 65（ASCII值） */
    printf("%c\n", ch + 1);   /* 输出 B */
    printf("%c\n", ch + 32);  /* 输出 a（大写A+32=小写a） */

    return 0;
}
```

**为什么 `'A' + 32 = 'a'`？** 看ASCII表：大写字母从65开始，小写字母从97开始，差值是32。所以 `'A' + 32 = 65 + 32 = 97 = 'a'`。

**大小写转换公式**：
- 大写转小写：`ch + 32`（确保 `ch` 是大写字母）
- 小写转大写：`ch - 32`（确保 `ch` 是小写字母）

### 10.2.3 数字字符 vs 数字值

这是一个初学者极易混淆的点：

```c
char digit = '5';    /* 存的是ASCII值 53，不是数字 5 */
int value = 5;       /* 存的是真正的数字 5 */
```

`'5'` 和 `5` 在内存中是完全不同的值：

```
'5'  ──→  十进制 53  ──→  二进制 00110101
 5   ──→  十进制  5  ──→  二进制 00000101
```

**如何把数字字符转成真正的数字？**

```c
char digit = '7';
int num = digit - '0';   /* num = 7 */
```

因为 `'0'` 到 `'9'` 在ASCII表中是连续的，`'7' - '0' = 55 - 48 = 7`。

**如何把数字转成数字字符？**

```c
int num = 7;
char digit = num + '0';   /* digit = '7' */
```

## 10.3 转义字符

`*转义字符*（此术语见附录F）` 是以反斜杠 `\` 开头的特殊字符序列，用来表示那些"无法直接输入"或"有特殊含义"的字符。

### 10.3.1 常用转义字符

| 转义序列 | 含义 | ASCII值 |
|----------|------|---------|
| `\n` | 换行（newline） | 10 |
| `\t` | 制表符（tab） | 9 |
| `\\` | 反斜杠本身 | 92 |
| `\'` | 单引号 | 39 |
| `\"` | 双引号 | 34 |
| `\0` | `*空字符*（此术语见附录F）` | 0 |
| `\r` | 回车（carriage return） | 13 |
| `\a` | 响铃（alert） | 7 |

### 10.3.2 为什么需要转义？

```c
/* ❌ 错误示例 */
printf("他说："你好！"");   /* 编译器混乱：字符串在哪里结束？ */

/* ✅ 正确示例 */
printf("他说：\"你好！\"");  /* 反斜杠告诉编译器：后面的引号是字面字符 */
```

没有转义，编译器分不清哪个双引号表示"字符串结束"，哪个双引号表示"字符串内容里的引号"。

### 10.3.3 `\n` vs `\r\n`

- Unix/Linux/macOS：换行用 `\n`（LF）
- Windows：换行用 `\r\n`（CR+LF）
- 老Mac（OS 9以前）：换行用 `\r`（CR）

在C语言中，`printf` 的 `\n` 通常会被标准库自动转换为当前平台的换行风格。但如果你用二进制方式读写文件，可能会遇到 `\r\n` 的问题。

## 10.4 布尔值：C语言的"将就"

### 10.4.1 C89时代：没有布尔类型

在C89标准中，**没有布尔类型**。C语言用整数来表示真假：

```c
/* C89风格 */
int is_admin = 1;   /* 非0表示真 */
int is_guest = 0;   /* 0表示假 */

if (is_admin) {     /* 等价于 if (is_admin != 0) */
    printf("管理员登录\n");
}
```

**规则**：在需要布尔值的地方（如 `if`、`while` 条件）：
- 整数 `0` → 假（false）
- 整数**任何非0值** → 真（true）

```c
if (1)   { }   /* 真 */
if (100) { }   /* 真 */
if (-5)  { }   /* 真 */
if (0)   { }   /* 假 */
if ('\0'){ }   /* 假（空字符的ASCII值是0） */
if ('A') { }   /* 真（任何非零字符都是真） */
```

### 10.4.2 C99引入 `_Bool` 和 `<stdbool.h>`

C99标准引入了真正的布尔类型 `_Bool`，以及更方便的头文件 `<stdbool.h>`：

```c
#include <stdbool.h>   /* 引入 bool, true, false */

int main(void)
{
    bool is_valid = true;    /* 终于有了真正的布尔类型！ */
    bool has_error = false;

    if (is_valid) {
        printf("数据有效\n");
    }

    if (!has_error) {        /* ! 是逻辑非 */
        printf("没有错误\n");
    }

    return 0;
}
```

`<stdbool.h>` 其实做了三件事：
1. `#define bool _Bool`
2. `#define true 1`
3. `#define false 0`

所以 `bool` 本质上还是 `_Bool`，`true` 和 `false` 本质上是整数 `1` 和 `0`。C语言仍然没有"纯正"的布尔类型，但 `<stdbool.h>` 让它看起来像有了。

### 10.4.3 布尔值的大小

```c
#include <stdio.h>
#include <stdbool.h>

int main(void)
{
    printf("_Bool: %zu 字节\n", sizeof(_Bool));   /* 通常 1 */
    printf("bool: %zu 字节\n", sizeof(bool));     /* 通常 1 */

    return 0;
}
```

`_Bool` 只占1个字节，因为它只需要存 `0` 或 `1`。

## 10.5 字符I/O函数入门

既然学了字符类型，不妨提前认识两个字符输入输出函数（第14章会详细讲）。

### 10.5.1 `putchar`：输出一个字符

```c
#include <stdio.h>

int main(void)
{
    putchar('H');
    putchar('i');
    putchar('\n');

    return 0;
}
```

### 10.5.2 `getchar`：读取一个字符

```c
#include <stdio.h>

int main(void)
{
    int ch;   /* 注意：用int而不是char！原因第14章会讲 */

    printf("请输入一个字符：");
    ch = getchar();

    printf("你输入了：");
    putchar(ch);
    putchar('\n');
    printf("它的ASCII值是：%d\n", ch);

    return 0;
}
```

## 10.6 常见错误

### 错误1：混淆数字字符和数值

```c
/* ❌ 错误示例 */
char c = '5';
int x = c * 2;        /* x = 53 * 2 = 106，不是 10！ */

/* ✅ 正确示例 */
char c = '5';
int x = (c - '0') * 2;   /* x = 5 * 2 = 10 */
```

### 错误2：用 `int` 存 `getchar` 返回值

```c
/* ❌ 错误示例 */
char c = getchar();   /* 如果是 EOF，char 装不下 */
if (c == EOF) { ... } /* 可能永远不成立 */

/* ✅ 正确示例 */
int c = getchar();    /* 用 int 存，能装下 EOF（-1） */
if (c == EOF) { ... }
```

### 错误3：忘记 `<>` 括起 `stdbool.h`

```c
/* ❌ 错误示例 */
#include <bool.h>     /* 没有这个头文件！ */

/* ✅ 正确示例 */
#include <stdbool.h>  /* 正确的头文件名 */
```

### 错误4：`char` 溢出

```c
/* ❌ 错误示例 */
char c = 200;   /* 200 超出了 char 的范围（-128~127） */
/* 编译器可能警告，c 的实际值可能是 -56 */

/* ✅ 正确示例 */
unsigned char c = 200;   /* 无符号char范围 0~255，可以存200 */
```

## 10.7 我在做什么？做得对不对？不对怎么办？

### 我在做什么？

我在理解 `char` 的双重身份（整数和字符），学习ASCII码表，用转义字符表示特殊字符，以及C语言中布尔值的表示方式（C89用整数，C99用 `_Bool`/`<stdbool.h>`）。

### 我做得对不对？

- 你能说出 `'A'` 的ASCII值（65）→ ASCII过关。
- 你能解释 `'5'` 和 `5` 的区别 → 数字字符vs数值过关。
- 你能写出至少5个转义字符及其含义 → 转义字符过关。
- 你知道C语言中 `0` 是假，非 `0` 是真 → 布尔过关。

### 不对怎么办？

| 症状 | 第一步检查 |
|------|-----------|
| 字符运算结果不对 | 是否混淆了字符值和ASCII值？ |
| 输出字符串出现乱引号 | 是否忘了转义 `\"` 或 `\'`？ |
| `bool` 类型报错 | 是否 `#include <stdbool.h>`？ |
| `getchar` 返回值异常 | 接收变量是否用了 `int` 而不是 `char`？ |
| 字符比较不用 `==` 而用 `=` | 是否混淆了赋值和比较？ |

## 本章小结

| 知识点 | 说明 |
|--------|------|
| `char` 的双重身份 | 既是1字节整数（-128~127），又是字符 |
| ASCII码表 | 字符到数字的映射，共128个（0-127） |
| `'A'`=65, `'a'`=97 | 大写字母ASCII值，小写比大写大32 |
| 数字字符转换 | `'7' - '0' = 7`，`7 + '0' = '7'` |
| 转义字符 | `\n`(换行) `\t`(制表) `\"`(双引号) `\\`(反斜杠) `\0`(空字符) |
| 布尔值（C89） | 0=假，非0=真 |
| 布尔值（C99） | `#include <stdbool.h>` 后可用 `bool`/`true`/`false` |
| `_Bool` | C99内置布尔类型，占1字节 |

## 练习题

1. **ASCII打印**：写程序，用循环打印大写字母A到Z及其ASCII值。

2. **大小写转换**：声明 `char ch = 'g';`，用算术运算把它变成大写 `'G'`，输出结果。

3. **数字字符转换**：声明 `char c1 = '3', c2 = '8';`，把两个字符转换成真正的数字，相加，输出结果（应该是11）。

4. **转义字符测试**：用 `printf` 输出以下字符串，观察效果：
   - `"Hello\nWorld"`（换行）
   - `"Hello\tWorld"`（制表符）
   - `"他说：\"C语言很有趣！\""`（转义引号）
   - `"路径是 C:\\Users\\Admin"`（转义反斜杠）

5. **布尔值实验**：声明 `int flags[] = {0, 1, 100, -1, '\0', 'A'};`，用 `if` 判断每个值是真是假，输出结果。确认"0是假，非0是真"的规则。
