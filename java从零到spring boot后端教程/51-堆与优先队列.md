## 51.1 什么是堆？

**堆是一种特殊的完全二叉树**，满足堆属性：
- **最小堆**：每个节点的值 ≤ 其子节点的值（根最小）
- **最大堆**：每个节点的值 ≥ 其子节点的值（根最大）

```
最小堆示例：
          1          ← 堆顶，最小值
        /   \
       3     5
      / \   / \
     7   9 6   8

完全二叉树：除了最后一层，所有层都是满的，且最后一层从左到右排列。
```

**用数组存储堆**（因为是完全二叉树，可以用数组紧凑存储）：

```
数组索引:  0   1   2   3   4   5   6   7
值:       1   3   5   7   9   6   8  null

父子关系（索引从0开始）：
- 索引i的左子节点: 2*i + 1
- 索引i的右子节点: 2*i + 2
- 索引i的父节点: (i-1) / 2
```

## 51.2 最小堆的完整Java实现

核心操作：**上浮（siftUp）** 和 **下沉（siftDown）**。

```java
public class MinHeap {
    private int[] heap;
    private int size;

    public MinHeap(int capacity) {
        heap = new int[capacity];
        size = 0;
    }

    // 插入元素 O(log n)
    public void offer(int val) {
        if (size == heap.length) {
            throw new IllegalStateException("堆已满");
        }
        heap[size] = val;
        siftUp(size);     // 上浮到正确位置
        size++;
    }

    // 删除并返回堆顶（最小值）O(log n)
    public int poll() {
        if (size == 0) throw new NoSuchElementException("堆为空");
        int top = heap[0];
        heap[0] = heap[size - 1];  // 把最后一个元素移到堆顶
        size--;
        siftDown(0);               // 下沉到正确位置
        return top;
    }

    // 查看堆顶 O(1)
    public int peek() {
        if (size == 0) throw new NoSuchElementException("堆为空");
        return heap[0];
    }

    public int size() { return size; }
    public boolean isEmpty() { return size == 0; }

    // 上浮：新元素从底部向上找位置
    private void siftUp(int index) {
        while (index > 0) {
            int parent = (index - 1) / 2;
            if (heap[index] >= heap[parent]) break;  // 满足堆属性
            swap(index, parent);
            index = parent;
        }
    }

    // 下沉：堆顶元素向下找位置
    private void siftDown(int index) {
        while (true) {
            int left = 2 * index + 1;
            int right = 2 * index + 2;
            int smallest = index;

            if (left < size && heap[left] < heap[smallest]) {
                smallest = left;
            }
            if (right < size && heap[right] < heap[smallest]) {
                smallest = right;
            }

            if (smallest == index) break;  // 堆属性已满足
            swap(index, smallest);
            index = smallest;
        }
    }

    private void swap(int i, int j) {
        int temp = heap[i];
        heap[i] = heap[j];
        heap[j] = temp;
    }
}
```

**siftUp 过程演示（插入 2 到已有堆 [1,3,5,7,9,6,8]）：**

```
步骤1: 把2放在末尾
       1
     /   \
    3     5
   / \   / \
  7   9 6   8
 /
2

步骤2: 2 < 7(parent)，交换
       1
     /   \
    3     5
   / \   / \
  2   9 6   8
 /
7

步骤3: 2 < 3(parent)，再交换
       1
     /   \
    2     5
   / \   / \
  3   9 6   8
 /
7

步骤4: 2 > 1(parent)，停止 ✓
```

## 51.3 Java标准库：PriorityQueue

```java
// 默认是最小堆（小的优先出队）
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
minHeap.offer(5);
minHeap.offer(2);
minHeap.offer(8);
minHeap.offer(1);

System.out.println(minHeap.poll());  // 1
System.out.println(minHeap.poll());  // 2
System.out.println(minHeap.poll());  // 5
System.out.println(minHeap.poll());  // 8

// 最大堆：传入Comparator.reverseOrder()
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
maxHeap.offer(5);
maxHeap.offer(2);
maxHeap.offer(8);
System.out.println(maxHeap.poll());  // 8（最大优先）

// 自定义优先级的对象
PriorityQueue<Task> taskQueue = new PriorityQueue<>(
    Comparator.comparingInt(Task::getPriority)  // 数字越小优先级越高
);
taskQueue.offer(new Task("修Bug", 1));
taskQueue.offer(new Task("写文档", 3));
taskQueue.offer(new Task("开会", 2));
System.out.println(taskQueue.poll().name);  // "修Bug"
```

## 51.4 Top K 问题（高频面试题）

**问题**：从100万个数字中找出最大的K个。

```java
// 思路：维护一个大小为K的最小堆
// 遍历所有数字，堆不满时直接加入，堆满了则比较：
//   如果 当前数字 > 堆顶（堆中最小），则替换堆顶
// 遍历完，堆中剩下的就是最大的K个

public List<Integer> topK(int[] nums, int k) {
    PriorityQueue<Integer> minHeap = new PriorityQueue<>(k);

    for (int num : nums) {
        if (minHeap.size() < k) {
            minHeap.offer(num);
        } else if (num > minHeap.peek()) {
            minHeap.poll();          // 移除堆中最小的
            minHeap.offer(num);       // 放入当前更大的
        }
    }

    List<Integer> result = new ArrayList<>(minHeap);
    return result;
}
// 时间复杂度：O(n log k)，空间复杂度：O(k)
// 对比：全排序需要 O(n log n)，当n很大而k很小时，堆方法快得多
```

**为什么用最小堆而不是最大堆？** 因为我们要的是"最大的K个"。维护最小堆，堆顶是K个里的最小值。遍历时，如果新元素比堆顶大，替换堆顶（淘汰最小的），最终堆里就是最大的K个。

## 51.5 堆排序

```java
public void heapSort(int[] arr) {
    int n = arr.length;

    // 第一步：建堆（从最后一个非叶子节点开始下沉）
    // 最后一个非叶子节点索引 = n/2 - 1
    for (int i = n / 2 - 1; i >= 0; i--) {
        siftDown(arr, n, i);
    }

    // 第二步：逐个将堆顶（最大值）交换到末尾
    for (int i = n - 1; i > 0; i--) {
        swap(arr, 0, i);         // 堆顶（最大）放到已排序区
        siftDown(arr, i, 0);    // 对剩余元素重建堆
    }
}

// 最大堆下沉
private void siftDown(int[] arr, int heapSize, int index) {
    while (true) {
        int left = 2 * index + 1;
        int right = 2 * index + 2;
        int largest = index;

        if (left < heapSize && arr[left] > arr[largest]) {
            largest = left;
        }
        if (right < heapSize && arr[right] > arr[largest]) {
            largest = right;
        }

        if (largest == index) break;
        swap(arr, index, largest);
        index = largest;
    }
}

private void swap(int[] arr, int i, int j) {
    int temp = arr[i];
    arr[i] = arr[j];
    arr[j] = temp;
}
// 时间复杂度：O(n log n)，空间复杂度：O(1) — 原地排序
// 堆排序是不稳定排序（相同元素可能改变相对顺序）
```

**堆排序过程演示（[5,2,8,1,9]）：**

```
建堆后（最大堆）：[9, 5, 8, 1, 2]
    9
   / \
  5   8
 / \
1   2

第1轮：交换9和2 → [2,5,8,1,9] → 下沉2 → [8,5,2,1,9]
第2轮：交换8和1 → [1,5,2,8,9] → 下沉1 → [5,1,2,8,9]
第3轮：交换5和2 → [2,1,5,8,9] → 下沉2 → [2,1,5,8,9]
第4轮：交换2和1 → [1,2,5,8,9] → 完成
```

## 51.6 优先队列在后端的应用

### 任务调度系统

```java
// 定时任务调度：按执行时间排序
PriorityQueue<ScheduledTask> scheduler = new PriorityQueue<>(
    Comparator.comparingLong(ScheduledTask::getExecuteTime)
);
scheduler.offer(new ScheduledTask("发送日报", now + 3600000));  // 1小时后
scheduler.offer(new ScheduledTask("心跳检测", now + 30000));   // 30秒后
scheduler.offer(new ScheduledTask("数据备份", now + 7200000));  // 2小时后

// scheduler.peek() 始终是最近要执行的任务
while (!scheduler.isEmpty() && scheduler.peek().getExecuteTime() <= now) {
    ScheduledTask task = scheduler.poll();
    task.execute();
}
```

### 电商系统的订单排序

```java
// VIP优先 + 金额高的优先
PriorityQueue<Order> orderQueue = new PriorityQueue<>((a, b) -> {
    if (a.isVip() != b.isVip()) return a.isVip() ? -1 : 1;  // VIP优先
    return Long.compare(b.getAmount(), a.getAmount());          // 金额从大到小
});
```

### 日志级别过滤

```java
// 从海量日志中找出ERROR级别的Top 10
PriorityQueue<LogEntry> topErrors = new PriorityQueue<>(10,
    Comparator.comparingInt(LogEntry::getSeverity)
);
```

## 51.7 ❌ 常见错误

| 错误 | 后果 | 正确做法 |
|---|---|---|
| 默认PriorityQueue以为最大堆 | 输出顺序反了 | 默认最小堆，最大堆用`Comparator.reverseOrder()` |
| Top K用最大堆 | 堆里一直存着最小的K个 | 求最大K个用最小堆，求最小K个用最大堆 |
| 忘记PriorityQueue不保证遍历顺序 | for-each输出无序 | 必须poll()才能按优先级出队，内部数组只是部分有序 |
| siftDown时不判左右子节点是否超size | ArrayIndexOutOfBounds | 每次判断`left < size`和`right < size` |
| 堆排序建堆从根开始下沉 | 结果错误 | 必须从最后一个非叶子节点（n/2-1）开始往前 |

## 小结

| 概念 | 解释 |
|---|---|
| 堆 | 完全二叉树，维护父-子大小关系 |
| 最小堆 | 父≤子，堆顶是最小值 |
| 最大堆 | 父≥子，堆顶是最大值 |
| 上浮 siftUp | 新元素从底部上升到正确位置 O(log n) |
| 下沉 siftDown | 堆顶元素下沉到正确位置 O(log n) |
| PriorityQueue | Java的优先队列，默认最小堆 |
| Top K | 维护大小为K的堆，O(n log k) |
| 堆排序 | 建堆+依次取堆顶，O(n log n)，原地排序 |
