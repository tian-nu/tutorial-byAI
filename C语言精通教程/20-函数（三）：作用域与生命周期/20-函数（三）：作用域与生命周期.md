## 我在做什么？

你已经知道怎么定义函数、传递参数。但还有一个问题：变量在哪里能用？变量什么时候出生、什么时候死亡？

一个变量不是随时随地都能访问的。你在 `main` 里定义的变量，别的函数里不能直接用。这就是 *作用域* *此术语见附录F* 和 *生命周期* *此术语见附录F* 管的事。读完这一章，你能理解局部变量、全局变量、静态局部变量的区别，并且知道什么时候该用哪个。

## 一、作用域：变量在哪里可见？

### 1.1 比喻：房间里的东西

```text
你的房间                          室友的房间
┌──────────────┐                 ┌──────────────┐
│ 桌子         │                 │ 床           │
│ 椅子         │                 │ 台灯         │
│ 你的手机     │                 │ 室友的手机     │
│              │                 │              │
│ 你不能直接拿  │                 │ 你不能直接拿   │
│ 室友的手机 →  │                 │ 你的手机 →    │
└──────────────┘                 └──────────────┘
```

作用域就是"房间"——变量定义在哪个花括号 `{}` 里，就只能在这个花括号里用。

### 1.2 块作用域（Block Scope）

C语言中，每一对花括号 `{}` 创建一个**块**（block）。块内定义的变量，只能在块内使用。

```c
#include <stdio.h>

int main() {
    int a = 10;          // a 的作用域：从这一行到 main 的 }

    if (a > 5) {
        int b = 20;      // b 的作用域：if 的花括号内
        printf("a=%d, b=%d\n", a, b);  // 都能访问
    }
    // printf("%d\n", b);  // 错误！b 已经不存在了

    return 0;
}
```

```text
main 的块：
┌──────────────────────────────┐
│ int a = 10;                  │
│                              │
│ if (a > 5) {                 │
│   ┌──────────────────────┐   │
│   │ int b = 20;          │   │  ← b 只能在这里用
│   │ printf("%d", b);     │   │
│   └──────────────────────┘   │
│                              │
│   // 这里 b 不可见             │
│   return 0;                  │
└──────────────────────────────┘
```

### 1.3 函数作用域

函数内部定义的变量（包括形参），只在函数内部可见。不同函数可以有同名变量，它们互不干扰。

```c
#include <stdio.h>

void func_a() {
    int x = 100;         // func_a 的 x
    printf("func_a: x = %d\n", x);
}

void func_b() {
    int x = 200;         // func_b 的 x，和 func_a 的 x 完全独立
    printf("func_b: x = %d\n", x);
}

int main() {
    int x = 300;         // main 的 x，又是独立的
    func_a();            // 输出 func_a: x = 100
    func_b();            // 输出 func_b: x = 200
    printf("main: x = %d\n", x);  // 输出 main: x = 300
    return 0;
}
```

三个 `x` 互不干扰，就像三个不同房间里各有一张桌子。

## 二、全局变量：所有人都能用的变量

### 2.1 什么是全局变量

定义在所有函数**外面**的变量，叫全局变量。整个文件里的所有函数都能访问它。

```c
#include <stdio.h>

int counter = 0;          // 全局变量，定义在所有函数外面

void increment() {
    counter++;            // 可以访问
}

void print_counter() {
    printf("counter = %d\n", counter);  // 也可以访问
}

int main() {
    increment();
    increment();
    print_counter();      // 输出 counter = 2
    printf("main: counter = %d\n", counter);  // 也可以访问
    return 0;
}
```

```text
全局变量 counter：
┌──────────────────────────────────────┐
│ counter = 0                          │
│                                      │
│  ┌──────────┐  ┌──────────┐  ┌──────┐│
│  │increment │  │print_    │  │main  ││
│  │counter++ │  │counter   │  │      ││
│  └──────────┘  └──────────┘  └──────┘│
│       ↑              ↑           ↑    │
│       └──────────────┸───────────┘    │
│           三个函数都能访问 counter     │
└──────────────────────────────────────┘
```

### 2.2 全局变量的风险

全局变量好用，但很危险：

```c
int balance = 1000;       // 全局：账户余额

void withdraw() {
    balance -= 500;       // 取钱
}

void check() {
    balance -= 100;       // 扣手续费——等等，哪个函数改了余额？
    // 在一个大项目中，很难追踪谁改了全局变量
}
```

**经验法则**：能用局部变量解决的问题，绝不用全局变量。全局变量是最后的手段。

## 三、静态局部变量（static）

### 3.1 普通局部变量的"失忆症"

普通局部变量每次函数被调用时重新创建，函数返回时销毁，值不保留。

```c
#include <stdio.h>

void count_visits() {
    int visits = 0;       // 每次调用都重新初始化为 0
    visits++;
    printf("访问次数：%d\n", visits);
}

int main() {
    count_visits();       // 输出：访问次数：1
    count_visits();       // 输出：访问次数：1  ← 还是 1！
    count_visits();       // 输出：访问次数：1  ← 还是 1！
    return 0;
}
```

每次调用 `count_visits`，`visits` 都被重新创建并初始化为 0。

### 3.2 static 关键字：让变量"记住上次的值"

加上 `static` 关键字，变量**只初始化一次**，函数返回后值保留，下次调用继续用。

```c
#include <stdio.h>

void count_visits() {
    static int visits = 0;   // 静态局部变量，只初始化一次
    visits++;
    printf("访问次数：%d\n", visits);
}

int main() {
    count_visits();       // 输出：访问次数：1
    count_visits();       // 输出：访问次数：2
    count_visits();       // 输出：访问次数：3
    return 0;
}
```

```text
第一次调用 count_visits()：
  visits 创建，初始化为 0 → visits++ → 1 → 函数返回，visits 保留

第二次调用 count_visits()：
  visits 还在（值为 1），跳过初始化 → visits++ → 2 → 函数返回

第三次调用 count_visits()：
  visits 还在（值为 2），跳过初始化 → visits++ → 3
```

### 3.3 static 的真实位置

普通局部变量放在 *栈* *此术语见附录F* 上，函数返回就销毁。`static` 变量放在静态存储区，程序启动时创建，程序结束时销毁。

```text
内存布局（简化版）：

┌──────────────┐  ← 高地址
│     栈       │  ← 普通局部变量、函数调用信息
│     ↓       │     每次函数调用在这里分配，返回时回收
│              │
│              │
│     ↑       │
│     堆       │  ← 动态分配（第30章讲）
│              │
│  静态存储区   │  ← static 变量、全局变量
│              │     程序启动到结束一直存在
│   代码区     │  ← 程序指令
└──────────────┘  ← 低地址
```

## 四、作用域遮蔽（Shadowing）

### 4.1 内层变量覆盖外层变量

如果内层块定义了和外层同名的变量，内层会"遮蔽"外层——在内层里，你看到的是内层的变量。

```c
#include <stdio.h>

int x = 100;              // 全局变量 x

int main() {
    printf("1: x = %d\n", x);   // 输出 100（全局的 x）

    int x = 200;           // 局部变量 x，遮蔽了全局的 x
    printf("2: x = %d\n", x);   // 输出 200（局部的 x）

    {
        int x = 300;       // 更内层的 x，遮蔽了外层的 x
        printf("3: x = %d\n", x);  // 输出 300
    }

    printf("4: x = %d\n", x);   // 输出 200（回到外层的 x）
    return 0;
}
```

```text
全局 x = 100
  └── main 的局部 x = 200
        └── 内层块的 x = 300  ← 最内层，遮蔽了所有外层
```

### 4.2 避免遮蔽

遮蔽会让代码难以阅读。**尽量不要在不同作用域使用同名变量**，除非有非常好的理由。

## 五、生命周期对比

| 变量类型 | 创建时机 | 销毁时机 | 存储位置 | 可见范围 |
|----------|----------|----------|----------|----------|
| 局部变量 | 进入函数/块 | 离开函数/块 | 栈 | 本函数/块内 |
| 全局变量 | 程序启动 | 程序结束 | 静态存储区 | 整个文件 |
| static 局部变量 | 第一次执行到定义处 | 程序结束 | 静态存储区 | 本函数内 |
| 形参 | 函数被调用时 | 函数返回时 | 栈 | 本函数内 |

## 六、常见错误与排错

### 6.1 在函数外访问局部变量

❌ 错误示例：
```c
#include <stdio.h>

void func() {
    int secret = 42;
}

int main() {
    printf("%d\n", secret);  // 错误：secret 不在 main 的作用域内
    return 0;
}
```

编译错误：
```text
error: 'secret' undeclared (first use in this function)
```

✅ 正确示例：
```c
#include <stdio.h>

int secret = 42;     // 改为全局变量（但不推荐，视情况而定）

int main() {
    printf("%d\n", secret);
    return 0;
}
```

### 6.2 返回局部变量的地址

❌ 错误示例（这个问题在第26章会详细讲）：
```c
int* create_number() {
    int n = 100;
    return &n;     // 错误！返回局部变量的地址，函数返回后 n 已销毁
}
```

### 6.3 static 误解

❌ 错误理解：
```c
void func() {
    static int x = 0;
    x = 0;          // 每次调用都手动重置为 0，static 白写了
    x++;
    printf("%d\n", x);
}
```

✅ 正确用法：
```c
void func() {
    static int x = 0;   // 初始化只在第一次执行
    x++;                // 每次调用累加
    printf("%d\n", x);
}
```

## 我做得对不对？

用以下程序验证：

```c
#include <stdio.h>

int global_counter = 0;      // 全局变量

void update_static() {
    static int static_counter = 0;  // 静态局部变量
    static_counter++;
    printf("static_counter = %d (应为递增值)\n", static_counter);
}

void update_global() {
    global_counter++;
}

int main() {
    // 测试 static
    update_static();  // 应为 1
    update_static();  // 应为 2
    update_static();  // 应为 3

    // 测试全局
    update_global();
    update_global();
    printf("global_counter = %d (应为 2)\n", global_counter);

    // 测试作用域遮蔽
    int x = 10;
    {
        int x = 20;
        printf("内层 x = %d (应为 20)\n", x);
    }
    printf("外层 x = %d (应为 10)\n", x);

    return 0;
}
```

编译运行：
```bash
$ gcc -Wall -o test test.c
$ ./test
static_counter = 1 (应为递增值)
static_counter = 2 (应为递增值)
static_counter = 3 (应为递增值)
global_counter = 2 (应为 2)
内层 x = 20 (应为 20)
外层 x = 10 (应为 10)
```

## 不对怎么办？

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| `undeclared` 错误 | 访问了不在作用域内的变量 | 检查变量定义在哪里，是否在当前位置可见 |
| static 变量每次都被重置 | 在函数体内又写了 `static int x = 0;` 之后又手动赋了 `x = 0;` | static 初始化只做一次，不要手动重置 |
| 全局变量值被意外修改 | 多个函数修改同一个全局变量 | 减少全局变量使用，或加注释标注修改者 |
| 变量名冲突 | 不同作用域用了同名变量导致遮蔽 | 给不同作用域的变量起不同名字 |

## 本章小结

| 知识点 | 一句话总结 |
|--------|-----------|
| 作用域 | 变量在哪个花括号内定义，就在哪个花括号内可用 |
| 局部变量 | 函数/块内定义的变量，离开作用域就销毁 |
| 全局变量 | 所有函数外定义的变量，整个文件可用 |
| static 局部变量 | 只初始化一次，值在函数调用间保留 |
| 生命周期 | 变量从创建到销毁的时间段 |
| 栈 | 局部变量和函数调用信息存储的地方 |
| 作用域遮蔽 | 内层同名变量覆盖外层变量 |
