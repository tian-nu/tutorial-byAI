# 33-Cursor实战：从零写一个网页

> **本篇属于**：第三篇·Cursor从入门到精通 | **预计阅读**：35分钟 | **难度**：★★★☆☆
>
> **前置条件**：已完成第26-29章，会使用Chat、Composer、Tab补全。

前面几章都是"学招式"，这一章我们**打实战**——用Cursor从零开始，写一个完整的网页。

**需求**：做一个个人主页，包含名字、照片、简介、联系方式。

---

## 33.1 准备工作

### **1.** 新建项目文件夹

在你的电脑上找一个地方（比如桌面），新建一个文件夹，命名为 `my-homepage`。

### **2.** 在Cursor里打开这个文件夹

打开Cursor，点击"Open Folder"（打开文件夹），选择你刚创建的 `my-homepage` 文件夹。

### **3.** 新建HTML文件

在左侧文件列表空白处右键 → 新建文件 → 输入 `index.html` → 回车。

---

## 33.2 第一步：用Composer生成初始页面

### **1.** 按 `Ctrl + I` 打开Composer

### **2.** 在输入框里输入以下Prompt：

```
帮我写一个个人主页，包含以下内容：
1. 我的名字：张三
2. 一张照片（先用占位图，src写 "photo.jpg"，我等会替换）
3. 一段简介：大家好，我是张三，一名热爱编程的开发者。我喜欢用代码创造实用的工具。
4. 联系方式：邮箱 zhangsan@example.com，GitHub链接 github.com/zhangsan

要求：
- 页面居中显示，美观大方
- 深色背景，白色文字
- 图片用圆形裁切
- 只用一个 index.html 文件，CSS写在<style>标签里
```

> **📖 术语：HTML**——网页的骨架，定义页面里有什么内容（标题、段落、图片、按钮）。
>
> **📖 术语：CSS**——网页的皮肤，控制页面长什么样（颜色、大小、位置、动画）。
>
> **📖 术语：JavaScript**——网页的大脑，让页面有交互行为（点击按钮、弹窗、动画）。

### **3.** 按回车，等待AI生成

几秒钟后，AI会生成完整的 `index.html`，包含内嵌的CSS样式。

### **4.** 点"Apply"应用到文件

---

## 33.3 第二步：在浏览器打开看看效果

### **1.** 保存文件：`Ctrl + S`

### **2.** 在文件管理器中找到 `index.html`，双击打开

（或者右键 → 打开方式 → 选择你的浏览器）

### **3.** 你应该看到：

- 一个深色背景的个人主页
- 中间有你的名字"张三"
- 一个圆形照片区域（显示占位图）
- 一段简介文字
- 联系方式

---

## 33.4 第三步：不满意？改Prompt，让AI再改

假设你觉得页面太单调了，想加点东西。

### **1.** 回到Cursor，选中 `index.html` 的全部内容

### **2.** 按 `Ctrl + L` 打开Chat

### **3.** 输入：

```
这个页面太单调了，帮我做以下改进：
1. 在名字下面加一行副标题，写"全栈开发者 | Python爱好者"
2. 联系方式不要纯文字，改成三个按钮样式的图标（邮箱、GitHub、微信）
3. 在页面底部加一个"© 2026 张三"的版权信息
4. 点击联系方式按钮时，弹出一个提示，说"已复制到剪贴板"
```

### **4.** AI会给出修改后的代码，点"Apply"应用到文件

### **5.** 保存，刷新浏览器，查看效果

---

## 33.5 第四步：再加点JavaScript交互

### **1.** 继续在Chat里输入：

```
帮我在页面里加一个彩蛋功能：
- 当用户连续点击名字5次时，背景颜色随机变化
- 在页面底部显示点击计数
```

### **2.** AI会给出修改后的代码。注意这次多了 `<script>` 标签，这就是JavaScript代码。

### **3.** 点"Apply" → 保存 → 刷新浏览器 → 试试连点5次名字

---

## 33.6 如果出错了怎么办

### 🔴 问题1：网页打不开（双击没反应）

**原因**：文件扩展名可能不对。

**检查**：文件名必须是 `index.html`，不是 `index.html.txt` 或 `index`。在文件管理器里看文件类型，确保是"HTML文件"。

### 🔴 问题2：样式没生效

**可能原因**：CSS代码有语法错误。

**解决**：选中 `<style>` 标签里的全部内容，在Chat（Ctrl+L）里输入"检查这段CSS有没有语法错误"。

### 🔴 问题3：中文乱码

**可能原因**：HTML文件缺少编码声明。

**解决**：在Chat里输入："帮我在HTML里加一个meta标签，设置编码为UTF-8"。AI会告诉你加上 `<meta charset="UTF-8">`。

### 🔴 问题4：照片不显示

**可能原因**：照片路径不对，或者你还没有放照片。

**解决**：
1. 找一张照片（`.jpg` 或 `.png`），改名为 `photo.jpg`
2. 把照片放到 `my-homepage` 文件夹里（和 `index.html` 在同一层）
3. 刷新浏览器

## 小结

| 你学到了什么 | 一句话 |
|-------------|--------|
| 从零到一的完整流程 | 新建文件夹 → 打开Cursor → 写Prompt → AI生成 → 浏览器查看 → 不满意再改 |
| Cursor写网页的核心技巧 | 第一次用Composer生成骨架，后续用Chat逐步迭代 |
| 排错要点 | 文件扩展名要正确、编码要声明、照片路径要匹配 |
| 迭代思维 | 先跑通，再美化，不要第一次就追求完美 |

## 自测

1. 如果你想让AI帮你写一个网页，应该用Composer还是Chat？为什么？
2. 网页在浏览器里打开后，你发现样式不对（比如颜色不是你想要的），你应该怎么做？
3. 如果有照片不显示，你应该检查哪三件事？

> **答案**：
> 1. 第一次生成用Composer，因为它可以一次性生成完整的HTML。后续修改用Chat，因为改单个文件效率更高。
> 2. 回到Cursor，选中CSS代码，在Chat里描述你想要的样式，让AI修改。然后Apply，保存，刷新浏览器。
> 3. 检查三件事：①照片文件名是否和代码里写的匹配（比如代码写的是 `photo.jpg`，照片也必须叫 `photo.jpg`）；②照片是否放在和 `index.html` 同一个文件夹；③照片格式是否正确（`.jpg` 或 `.png`）。
