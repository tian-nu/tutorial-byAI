# 第2章 · 创建你的第一个GitHub仓库——给代码安个"云上的家"

> "到现在为止，你有了一台装好 Git 的电脑，有了一个 GitHub 账号，还配好了 SSH 钥匙。现在的问题变成了：代码要从你的电脑推到哪去？"
>
> "就像搬家——你有行李（代码），也有交通工具（Git），但目的地还没定。你需要一个'地址'——一个在 GitHub 上属于你的空间，专门存放你这个项目的代码。"
>
> "这个空间，就叫**仓库（Repository）**。这一章，我们就在 GitHub 网站上点几下鼠标，把这个'云上的家'建好。"

---

## 2.1 在 GitHub 网页创建仓库——点击、填写、创建

### 2.1.1 我在做什么？

在 GitHub 网站上创建一个完全空白的仓库（Repository，以下简称 repo）。此术语需进附录。这个仓库会成为你本地代码在互联网上的"家"——你以后写的代码就推送到这里。

仓库这个词很好懂：就像现实中的仓库一样，一个 GitHub 仓库就是一个"存放代码和相关文件的空间"。每个仓库有自己的地址，有自己的访问权限控制（谁能看、谁能改），有自己的历史记录。

### 2.1.2 打开 GitHub 并确认登录

**第1步：打开浏览器。**

如果你的浏览器已经打开，直接跳到第2步。如果没打开：

- 在桌面上找到浏览器图标（Chrome / Edge / Firefox 的图标），用鼠标**左键双击**它。

**第2步：访问 GitHub。**

用鼠标左键点击浏览器顶部的**地址栏**（白色长条），输入：

```
github.com
```

按回车键。

> 📺 GitHub 网站加载出来了。黑色导航栏在页面顶部。

**第3步：确认已登录。**

看页面右上角：

- 如果右上角有一个**圆形头像**（可能是灰色人形图案或你的照片），说明你已登录。✅
- 如果右上角没有头像，写着 "Sign in" 和 "Sign up"，说明你没有登录。点击 "Sign in"，输入你的邮箱/用户名和密码登录。

确认登录后，继续往下看。

### 2.1.3 找到"创建仓库"的入口

**第4步：点击右上角的 `+` 号。**

看页面的**右上角**。在圆形头像的**左边**，有一个小小的 **`+`** 号（加号）。它的图案就是一个加号 `＋`，颜色是白色的（在黑色导航栏上）。

用鼠标左键点击这个 **`+`** 号。

> 📺 点击后弹出一个**下拉菜单**。这个菜单很短，只有几个选项：
>
> - "New repository"（新建仓库）
> - "Import repository"（导入仓库）
> - "New gist"（新建代码片段）
> - "New organization"（新建组织）
> - "New project"（新建项目）

把鼠标移到第一个选项 **"New repository"** 上。不要点其他选项。用鼠标左键点击它。

> 📺 页面跳转。你现在进入了**创建仓库的表单页面**。页面上方有大标题 "Create a new repository"（创建一个新仓库）。标题下面是一个表单，有很多输入框和选项。我们一个一个来填。

### 2.1.4 填写仓库创建表单——逐项讲解

从页面上方往下看，逐项操作。

---

**① Owner（所有者）——这个仓库属于谁？**

表单最上方有一个标签叫 "Owner"（所有者）。旁边是一个下拉框（一个长条方框，右边有向下箭头 `▾`），里面显示的是你的 GitHub 用户名。

这是告诉你：这个仓库将会创建在你的名下，地址是 `github.com/你的用户名/仓库名`。

如果你只有一个 GitHub 账号（大概率如此），这里不用管，保持默认就好。如果你属于某个组织（Organization），下拉框里可能会列出组织的名字——选你自己就行。

---

**② Repository name（仓库名称）——给你的仓库取个名字。**

"Owner" 下方有一个输入框。输入框前面有一个 `☆`（星号，表示必填）。输入框里的灰色文字可能是 "Enter a repository name..."（输入仓库名称）。

用鼠标点击这个输入框，光标会在里面闪烁。输入：

```
my-first-project
```

这个名字的意思就是一个英文短语："我的第一个项目"。

**命名规则（要注意）：**

- ✅ 可以用英文小写字母：`a` 到 `z`
- ✅ 可以用数字：`0` 到 `9`
- ✅ 可以用连字符（减号）：`-`
- ❌ 不能用大写字母。如果你输入 `My-First-Project`，GitHub 会自动把它变成 `my-first-project`
- ❌ 不能用空格。空格是非法的
- ❌ 不能用中文、日文、韩文等非英文字符
- ❌ 不能用下划线开头（`_my-project` 不行，但 `my_project` 理论上可以——虽然不推荐）

**仓库名称会出现在 URL 中。** 你这个仓库的网址会是：

```
github.com/你的用户名/my-first-project
```

取一个好名字很重要——以后你每次访问这个仓库，都靠这个名字。所以取一个有意义的、不太随便的名字。

输入完成后，看输入框的右边：

- 如果名字合法（没有被别人占用），输入框右边会出现一个**绿色的对勾** ✅。
- 如果名字已经被别人占用了（比如很多人会取 `test`、`hello-world` 这样的名字），输入框下面会出现**红色提示文字**："`my-first-project` is already taken"（这个名字已经被占用了）。这时候你需要换一个名字——在后面加数字，比如 `my-first-project-2024` 或 `my-first-project-001`，直到出现绿色对勾。

如果出现绿色对勾 ✅，说明名字可以使用，继续往下。

---

**③ Description（描述）——一句话介绍你的仓库（可选）。**

仓库名称输入框的下方，还有一个输入框。标签是 "Description (optional)"（描述，可选）。

用鼠标点击这个输入框，可以输入一句简短的话描述这个仓库是干什么的。比如：

```
学习Git和GitHub的练习仓库
```

这句话会显示在你仓库的首页上（在"About"区域），让别人一眼知道这个仓库是做什么的。

这个字段是**可选的**——不填也没关系，以后随时可以修改。填完或跳过，继续往下。

---

**④ Public / Private（公开还是私有）——谁可以看到你的代码？**

"Description" 下方有两个并排的单选按钮（圆形的按钮，选了哪个哪个里面就会出现一个圆点）。这就是选择仓库的**可见性**。

两个选项分别是：

| 选项 | 图标 | 含义 |
|------|------|------|
| **Public**（公开） | 一本书打开的图案 | 全世界任何人都能看到你的代码。他们可以浏览、复制（Fork），但不能修改（除非你授权）。 |
| **Private**（私有） | 一把锁的图案 | 只有你自己和你明确邀请的人才能看到你的代码。 |

本教程建议你选 **Public**（公开）。用鼠标左键点击 "Public" 前面的圆形按钮，让它里面的圆点亮起来。

为什么选 Public？

- 你是来学习的，不怕别人看到代码。
- Public 仓库完全免费，没有任何限制。
- Private 仓库免费版有一些限制（协作者数量等），对学习来说没必要。
- 每个程序员都是从"写得烂"开始的——没有人会嘲笑你的学习项目。

> 如果你以后为公司做项目，或者代码里有密码、密钥之类敏感信息，应该选 Private。但现在不用担心这些。

---

**⑤ 三个初始化复选框——本次全部不勾选！**

这是本页最需要小心的地方。在 "Public / Private" 下方，有三个复选框（小方框，勾选了里面就有一个对勾 `✓`）。

| 复选框文字 | 中文意思 | 勾选后会怎样 |
|-----------|---------|------------|
| **Add a README file** | 添加一个 README 说明文件 | GitHub 自动在仓库里创建一个 `README.md` 文件 |
| **Add .gitignore** | 添加一个 .gitignore 忽略文件 | GitHub 自动创建一个 `.gitignore` 文件模板 |
| **Choose a license** | 选择一个许可证 | GitHub 自动创建一个 `LICENSE` 许可证文件 |

**你的操作：三个复选框全部保持不勾选状态。** 如果哪个前面有对勾，用鼠标左键点击那个小方框，把对勾取消掉。

> ⚠️ **为什么全不勾选？——这是本章最重要的知识点。**
>
> 如果你勾选了任何一个复选框，GitHub 会在你创建仓库的同时，往仓库里写入至少一个文件。这就意味着：**远程仓库里有内容了**。
>
> 但你的电脑上是空的啊！当你的本地代码和远程仓库的内容对不上时，就会产生冲突。解决冲突对新手来说非常痛苦。
>
> 全不勾选的意思是：**我要一个完全空白的仓库**。里面什么都没有。然后从第3章开始，我们把本地的代码"推"进去——干干净净，没有冲突。
>
> 后面第3章我们会在本地自己创建 `README.md` 和 `.gitignore`，所以这里不勾选不意味着你不用它们——只是不在这里让 GitHub 帮你自动创建。

**三个复选框的分别解释（帮助理解，不用操作）：**

- **Add a README file**：README（读我）是一个 Markdown 格式的说明文件。此术语需进附录。打开仓库首页时，这个文件的内容会自动显示在文件列表下方——就像仓库的"封面介绍"。你可以在 README 里写项目介绍、安装方法、使用说明等等。
- **Add .gitignore**：`.gitignore` 是一个配置文件，告诉 Git "这些文件不用管"。此术语需进附录。比如你的代码编译后产生的 `.exe` 文件、IDE 自动生成的配置文件夹、包含密码的 `.env` 文件——这些都不应该上传到公开仓库，`.gitignore` 就是排除它们的名单。
- **Choose a license**：许可证（License）是一份法律文件，声明别人可以用你的代码做什么。此术语需进附录。常见的有 MIT（最宽松："随便用，但要保留我的版权声明"）、GPL（传染性强："用了我的代码，你的代码也必须开源"）、Apache 2.0（像 MIT 但有专利保护）。没有许可证的情况下，默认"保留所有权利"——别人不能合法地使用你的代码。本教程暂时不涉及。

---

**⑥ 最后的确认和创建。**

三个复选框确认全部不勾选之后，页面拉到最下方。你会看到一个**绿色的大按钮**，上面写着 **"Create repository"**（创建仓库）。

用鼠标左键点击它。

> [可暂停点 2/3] 点击创建按钮之前，如果你时间不够，可以在此暂停。**恢复时**：打开 GitHub 网站，登录后从 2.1.3 节的第4步继续。

### 2.1.5 创建成功后——你看到的页面

点击 "Create repository" 后，页面会短暂加载（地址栏旁边可能有一个小圆圈在转），然后跳转到你的仓库页面。

> 📺 因为你三个复选框全没勾选，创建出来的是一个**完全空白的仓库**。你看到的页面应该和下面的描述一致：
>
> - 页面左上角显示 `你的用户名 / my-first-project`（旁边可能有一个 "Public" 的小标签，表示这是公开仓库）
> - 页面中央偏上有一个大标题，可能是 **"Quick setup"**——意思是"快速设置"。标题下面有一行灰色小字："if you've done this kind of thing before"（如果你以前做过这种事的话）
> - 标题下方有一条横着的地址栏，里面显示着 `git@github.com:你的用户名/my-first-project.git`——这是你仓库的 SSH 地址。等一下第3章会用到它。
> - 地址栏右边有一个灰色的按钮（可能是复制按钮，两个方块叠在一起的图标）
> - 再往下是一大片命令行指引，分为 `…or create a new repository on the command line` 和 `…or push an existing repository from the command line` 两种。这些指引告诉你"新手怎么把代码弄上来"——但我们有教程，不用看它。
> - 页面中间的文件列表区域是**空的**——没有文件，可能只显示一行字 "Your repository is empty" 或者一片空白。

**✅ 恭喜你！你的第一个 GitHub 仓库创建成功了！**

### 2.1.6 我做得对不对？

- [ ] 页面左上角显示了 `你的用户名 / my-first-project`
- [ ] 页面没有文件列表（或者文件列表是空的）
- [ ] 页面中央有 "Quick setup" 的引导信息
- [ ] 有一个地址显示着 `git@github.com:你的用户名/my-first-project.git`

### 2.1.7 不对怎么办？

❌ **错误1：页面有文件列表（比如有一个 `README.md`）**

这说明你勾选了表单里的某个复选框（很可能是 "Add a README file"）。

你有两个选择：
- **选择A（推荐）**：删掉这个仓库，重新创建一个不勾选复选框的。删除方法：仓库页面 → 顶部标签栏最右边 "Settings" → 左边栏拉到最底部 → "Danger Zone" 区域 → "Delete this repository" → 输入仓库全名确认。然后重新做 2.1.3~2.1.4。
- **选择B**：接受这个带 README 的仓库。但到了第3章，你的操作流程会不同——需要先 `git clone` 把仓库拉到本地，再添加文件，再推送。操作会复杂不少。

❌ **错误2：页面显示 404 或报错**

可能是你还没有登录。检查页面右上角是否有你的头像。如果没有，点击 "Sign in" 登录后，在地址栏输入 `github.com/你的用户名/my-first-project` 访问你的仓库。

❌ **错误3：仓库名称被占用（创建时输入框下面有红色提示）**

换一个名字。在原来的名字后面加数字：`my-first-project-2024`、`learning-git-001` 等等。起好新名字后，本章所有后续提到 `my-first-project` 的地方，都用你实际起的名字代替。

## 2.2 三种仓库初始状态——理解你在哪里

新仓库创建之后，根据你是否勾选了初始化复选框，会呈现三种完全不同的页面状态。**理解这三种状态，能让你在第3章不走弯路。**

### 2.2.1 状态A：完全空白——你现在在这里

**你是怎么到的：** 三个复选框全不勾选 → 点击 "Create repository"。

> 📺 **你看到的页面特征：**
>
> - 页面中央有 "Quick setup" 引导区域
> - 有 SSH 地址：`git@github.com:你的用户名/my-first-project.git`
> - 下面有 "…or push an existing repository from the command line" 的指引
> - **没有任何文件列表**——文件区域是空的，或者显示 "Your repository is empty"

**下一步该做什么（第3章会详细讲）：** 在你的电脑上本地初始化 Git 项目，然后关联这个远程仓库，把代码推送（push）上来。流程是：本地 `git init` → `git remote add` → `git push`。

👇 这就是你现在的状态。你的操作路径最直接、最简单。

---

### 2.2.2 状态B：带一个 README 文件

**你是怎么到的：** 只勾选了 "Add a README file" → 点击 "Create repository"。

> 📺 **你看到的页面特征：**
>
> - 页面文件列表里有一个 `README.md` 文件
> - 文件列表下方是 README 的渲染内容（一般是仓库名）
> - **没有** "Quick setup" 引导区域
> - GitHub 已经帮你做了第一次提交（initial commit）——远程仓库有内容了

**下一步该做什么：** 你必须先把远程仓库下载（clone）到本地，在下载下来的文件夹里添加你的代码，然后再推送上去。不能直接本地 `git init` 然后 `git push`——因为远程有东西而本地什么都没有，会冲突。

---

### 2.2.3 状态C：完整的初始模板

**你是怎么到的：** 三个复选框全勾选或勾选了其中任意两个。

> 📺 **你看到的页面特征：**
>
> - 文件列表里有 `README.md`、`.gitignore`、`LICENSE` 三个文件（或其中的两个）
> - README 内容在文件列表下方渲染展示
> - GitHub 已经做了第一次提交

**下一步该做什么：** 与状态B相同——必须先 clone 再 push。

---

### 2.2.4 三者对比表

| | 状态A（你现在在） | 状态B | 状态C |
|---|---|---|---|
| 远程有初始提交吗？ | ❌ 没有 | ✅ 有 | ✅ 有 |
| 文件列表里有什么？ | 空的 | `README.md` | `README.md` + `.gitignore` + `LICENSE` |
| 本地怎么开始？ | 本地 init → push | clone → 加代码 → push | clone → 加代码 → push |
| 适合什么情况？ | 你本地已经有了代码，想推到网上 | 你还没开始写代码，想先在 GitHub 建仓库 | 你想一次性配齐仓库的所有配置 |

> ⚠️ 本教程假设你处于**状态A**。如果你不小心创建了状态B或C，建议参考 2.1.7 节的选择A——删除重建，选状态A。这会让第3章的学习顺利很多。

---

## 2.3 GitHub 仓库页面导览——每个按钮是做什么的

仓库建好了，我们花几分钟认识一下这个页面。你不用记住每个按钮的功能——知道它们大概在哪就好。以后用到的时候再回来查。

> [可暂停点 3/3] 本节是导览性质，不需要操作。如果你累了，看完本节可以在下一章开始前休息。**恢复时**：浏览器打开你的仓库页面（`github.com/你的用户名/my-first-project`），从第3章开始即可。

### 2.3.1 页面顶部区域

从页面最上面往下看：

- **最顶部**：黑色的导航栏。左边有一个 GitHub 的章鱼猫图标，右边有 `+` 号、你的头像等。
- **导航栏下方**：你的仓库路径。比如 `zhangsan / my-first-project`。旁边有一个小标签写着 "Public" 或 "Private"——这是你仓库的可见性。
- **仓库路径下方**：一排**标签（Tab）**。从左到右依次是：

| 标签 | 中文意思 | 是干什么的 |
|------|---------|-----------|
| **▾ Code** | 代码 | 仓库的主页面。显示文件列表。你大部分时间都在这个页面。 |
| **Issues** | 问题追踪 | 报告 Bug、提出功能建议、讨论。像一个项目的"待办事项清单"。 |
| **Pull requests** | 合并请求 | 当你写完代码，想把它合并进主分支时，在这里发起请求。 |
| **Actions** | 自动化 | GitHub 自带的自动化工具——让代码 push 后自动测试、自动部署。 |
| **Projects** | 项目看板 | 像 Trello 一样的看板，管理任务进度。 |
| **Wiki** | 维基文档 | 给仓库写详细文档的地方。 |
| **Security** | 安全 | 安全漏洞扫描和依赖检查。 |
| **Insights** | 数据分析 | 仓库的统计数据：谁提交了多少代码、什么时候活跃。 |
| **Settings** | 设置 | 仓库的配置面板。改名、改可见性、删仓库、管理协作者都在这里。 |

你当前在 **Code** 标签页——这是默认显示的页面。

### 2.3.2 Code 标签页的布局

在标签下方，Code 页面分为几个区域：

**① 文件列表区域（左侧大面积）**

当前是空的（状态A）。以后你推送代码后，这里会列出所有文件和文件夹。你可以点击文件名查看内容，点击文件夹进入子目录。

**② 右侧 "About" 区域**

一个小卡片，写着 "About"（关于）。下面是你在创建时填的 Description（描述文字）。下面可能还有一些链接——"Website"（网站网址）、"Topics"（标签分类）等。这些都是可选的。

**③ 绿色 "▾ Code" 按钮（文件列表上方偏右）**

这是一个非常重要的按钮。用鼠标左键点击它，会弹出一个面板。

> 📺 弹出来的面板顶部有三个标签（Tab）：
>
> - **HTTPS**——地址以 `https://github.com/` 开头
> - **SSH**——地址以 `git@github.com:` 开头 ← **这是我们要用的！**
> - **GitHub CLI**——地址以 `gh repo clone` 开头（GitHub 命令行工具专用）
>
> 用鼠标点击 **SSH** 标签。面板里会显示一个地址，类似：
>
> ```
> git@github.com:你的用户名/my-first-project.git
> ```
>
> 这个地址就是你仓库的 SSH 克隆地址——第3章会用到它。地址右边有一个小按钮（两个方块叠在一起），点击它可以复制这个地址。

记住这个按钮的位置——以后每次需要获取仓库地址时，都要来这里。

### 2.3.3 Settings 标签页（特别重要）

点击顶部标签栏最右边的 **"Settings"** 标签。

> 📺 页面跳转，进入仓库设置页面。这个页面有一个长长的左边栏，有很多分类选项。我们挑重要的说：

- **General**（常规设置）：第一个选项，默认打开。在这里可以改仓库名称、改描述、改默认分支名。拉到最底部有一个 **"Danger Zone"**（危险区域）——里面的红色按钮可以做危险操作，比如 "Change repository visibility"（改可见性：公开变私有、私有变公开）和 "Delete this repository"（删除仓库）。

> ⚠️ **"Delete this repository"（删除仓库）**——点击这个按钮后，你需要输入仓库全名（`你的用户名/仓库名`）来确认。确认后仓库永久消失，**无法恢复**。除非你非常确定，否则不要点这个按钮。

- **Collaborators**（协作者）：在这里添加其他人，让他们也能往你的仓库推送代码。
- **Branches**（分支）：配置分支保护规则，防止误操作（比如禁止直接往 main 分支 push）。
- **Pages**（网页部署）：GitHub Pages 的配置入口——可以免费把你的仓库变成一个网站。第8章会专门讲。
- **Actions** 分组下面的选项用于配置自动化流水线。第7章会讲。

现在不用改任何设置。熟悉一下位置就好。用鼠标点击顶部标签栏的 **"Code"** 标签回到仓库主页。

## 🤔 想多一点：为什么 GitHub 要用 "Repository" 这个词？

"Repository" 这个词来自拉丁语 `repositorium`，意思是"存放东西的容器"。在英语里，repository 可以指仓库、储藏室、博物馆的藏品库。

GitHub 选择了这个词，因为它非常贴切：一个代码仓库里面存放的不仅仅是代码文件，还包括：

- 所有的历史版本（谁在什么时候改了什么）
- 分支（同一份代码的多个平行世界）
- Issue 讨论记录
- Pull Request 审查记录
- 自动化流水线配置

在程序员的口语里，"仓库"和 "repo" 是同一个意思。当你听到人说"我新建了一个 repo"或"克隆一下我的仓库"，他说的就是 GitHub（或 GitLab、Gitee 等）上的一个项目存储空间。

## 本章小结

| 知识点 | 要点 |
|--------|------|
| 创建仓库入口 | GitHub 页面右上角 `+` 号 → "New repository" |
| Owner（所有者） | 默认是你自己，不用改 |
| Repository name | 英文小写 + 数字 + 连字符。如 `my-first-project` |
| Description | 可选，一句话描述仓库用途 |
| Public vs Private | 新手选 Public（公开免费无限制） |

| 创建按钮 | 页面最下方绿色 "Create repository" 按钮 |
| 状态A（你在的） | 完全空白："Quick setup"页面，无文件列表 |
| 状态B | 带 README：有文件列表，需先 clone 再 push |
| 状态C | 完整模板：B + `.gitignore` + `LICENSE`，需先 clone 再 push |
| SSH 地址获取 | Code 标签页 → 绿色 "▾ Code" 按钮 → SSH Tab → 复制地址 |
| 仓库页面导航 | Code / Issues / Pull requests / Actions / Settings 等标签 |
| 删除仓库 | Settings → Danger Zone → Delete this repository → 输入仓库名确认（不可恢复） |
