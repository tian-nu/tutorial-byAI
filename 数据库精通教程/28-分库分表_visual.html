🗂️ 分库分表策略

空间关系图：垂直拆分 → 水平拆分 → 数据路由

步骤
垂直拆分 水平拆分 数据路由 当前高亮

📦 拆分前：单库单表

my_shop

users orders products logs
所有表都在一个库里，所有数据都在一台机器上
⬇ 拆分

📐 第一步：垂直拆分（按业务模块）

user_db

users

order_db

orders

product_db

products

log_db

logs
不同表放到不同库，每个库独立部署、独立扩容

📊 第二步：水平拆分（按数据行）

分片 0

orders (id%4=0)
250万行

分片 1

orders (id%4=1)
250万行

分片 2

orders (id%4=2)
250万行

分片 3

orders (id%4=3)
250万行
同一张表（orders）的数据按 order_id % 4 分散到 4 个分片，每个分片 250 万行

🔀 第三步：数据路由（写入时）

应用
INSERT INTO orders
(order_id=12347)
→
Proxy
12347 % 4 = 3
→
分片 3
orders_3
Proxy 根据分片键 order_id 计算路由：12347 % 4 = 3 → 写入分片 3

📖 第四步：数据路由（查询时）

精确查询（带分片键）：
SELECT * FROM orders
WHERE order_id = 12347
→
只查分片 3
✅ 高效
范围查询（不带分片键）：
SELECT * FROM orders
WHERE amount > 100
→
分片 0
+
分片 1
+
分片 2
+
分片 3
→
合并结果
⚠️ 全分片扫描
💡 带分片键的查询只查一个分片，性能最高；不带分片键的查询需要扫描所有分片，性能差
[初始状态] 一个单体数据库，所有表都在一个库里。随着数据量增长，单表超过 2000 万行，单机 TPS 达到上限，需要拆分。
