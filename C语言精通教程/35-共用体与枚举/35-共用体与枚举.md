## 我在做什么？

结构体让你把不同类型的数据"拼接"在一起。但有时候你需要的是：同一块内存，在不同时间存储不同类型的数据。比如一个变量，有时候存整数，有时候存浮点数。这就是共用体。而枚举则让你给整数起有意义的名字，让代码更易读。

读完本章你能：用共用体（union）让多个变量共享同一块内存，用枚举（enum）定义有名字的整数常量，理解共用体的应用场景和注意事项。

## 一、共用体（Union）

### 1.1 什么是共用体

*共用体* *此术语见附录F*（*Union* *此术语见附录F*）是一种特殊的数据类型，它的所有成员**共享同一块内存空间**。你可以把它想象成一个"多面手"——同一块内存，这次当整数用，下次当浮点数用。

```text
结构体 (struct) — 各自拥有点空间：
  +--------+--------+--------+
  | int a  | float b| char c |
  +--------+--------+--------+
  a 有 4 字节，b 有 4 字节，c 有 1 字节，总共 9 字节（+ 填充）

共用体 (union) — 共享同一块空间：
  +--------------------------------+
  | int a / float b / char c       |
  +--------------------------------+
  只有 4 字节（取最大成员的大小）
  任何时候只能存其中一种类型
```

### 1.2 定义与使用

```c
#include <stdio.h>

union Data {
    int   i;
    float f;
    char  str[20];
};

int main() {
    union Data d;

    d.i = 100;
    printf("d.i = %d\n", d.i);     // 100

    d.f = 3.14f;
    printf("d.f = %.2f\n", d.f);   // 3.14
    printf("d.i = %d\n", d.i);     // 垃圾值！f 覆盖了 i 的内存

    return 0;
}
```

**关键规则**：同一时间只有**最后一个被赋值的成员**是有效的。当你给 `d.f` 赋值后，`d.i` 的内容就被覆盖了，读出来是乱码。

### 1.3 sizeof 共用体

```c
union Data {
    int   i;       // 4 字节
    float f;       // 4 字节
    char  str[20]; // 20 字节
};

printf("%zu\n", sizeof(union Data));  // 输出 20
```

共用体的大小等于**最大成员的大小**。因为所有成员共享同一块内存，这块内存必须大得足够容纳最大的那个成员。

### 1.4 共用体的初始化

在 C99 及以后，共用体只能初始化第一个成员：

```c
union Data d = {42};    // 初始化为 int 42（第一个成员是 int）
union Data d2 = {.f = 3.14f};  // C99 指定成员初始化
```

## 二、共用体的应用场景

### 2.1 场景一：类型双关（Type Punning）

当你想用不同的方式解释同一段二进制数据时：

```c
#include <stdio.h>

union FloatInspector {
    float f;
    unsigned int bits;
};

int main() {
    union FloatInspector fi;
    fi.f = 3.14f;

    // 查看浮点数在内存中的二进制表示
    printf("浮点数: %f\n", fi.f);
    printf("二进制: 0x%08X\n", fi.bits);

    return 0;
}
```

这里 `f` 和 `bits` 占用同一块 4 字节内存。你存一个 float，然后用 unsigned int 的方式读出它的二进制位模式。

**注意**：严格来说，这种用法在 C 标准中属于未定义行为（访问的不是最近写入的成员），但大多数编译器都支持它。更规范的做法是使用 `memcpy` 或 `char*` 逐字节操作。

### 2.2 场景二：变体类型（Tagged Union）

在实际项目中，共用体通常和枚举配合使用，形成一个"标签联合"：

```c
#include <stdio.h>
#include <string.h>

// 枚举：定义值的类型
typedef enum {
    TYPE_INT,
    TYPE_FLOAT,
    TYPE_STRING
} ValueType;

// 带标签的变体类型
typedef struct {
    ValueType type;   // 标签：当前存的是什么类型
    union {
        int   i;
        float f;
        char  s[32];
    } value;          // 数据：根据标签决定读取哪个成员
} Variant;

void print_variant(const Variant *v) {
    switch (v->type) {
        case TYPE_INT:
            printf("整数: %d\n", v->value.i);
            break;
        case TYPE_FLOAT:
            printf("浮点数: %.2f\n", v->value.f);
            break;
        case TYPE_STRING:
            printf("字符串: %s\n", v->value.s);
            break;
    }
}

int main() {
    Variant v1 = {TYPE_INT, .value.i = 42};
    Variant v2 = {TYPE_FLOAT, .value.f = 3.14f};
    Variant v3 = {TYPE_STRING};
    strcpy(v3.value.s, "Hello");

    print_variant(&v1);
    print_variant(&v2);
    print_variant(&v3);

    return 0;
}
```

这就是 C 语言中"一个变量存多种类型"的标准做法。很多动态语言（如 Python、JavaScript）的变量实现底层就用了类似的技术。

## 三、枚举（enum）

### 3.1 什么是枚举

*枚举* *此术语见附录F* 是一种让你给整数常量起名字的机制。与其在代码里写晦涩的 `1`、`2`、`3`，不如写有意义的 `RED`、`GREEN`、`BLUE`。

```c
enum Color {
    RED,    // 默认从 0 开始
    GREEN,  // 1
    BLUE    // 2
};
```

### 3.2 基本使用

```c
#include <stdio.h>

enum Weekday {
    MONDAY = 1,    // 手动指定从 1 开始
    TUESDAY,       // 2
    WEDNESDAY,     // 3
    THURSDAY,      // 4
    FRIDAY,        // 5
    SATURDAY,      // 6
    SUNDAY         // 7
};

int main() {
    enum Weekday today = WEDNESDAY;

    printf("today = %d\n", today);  // 输出 3

    if (today == WEDNESDAY) {
        printf("今天是星期三\n");
    }

    return 0;
}
```

### 3.3 枚举值的规则

```c
enum Example {
    A,          // 0（默认从 0 开始）
    B,          // 1（自动递增）
    C = 10,     // 10（手动指定）
    D,          // 11（继续递增）
    E = 20,     // 20
    F           // 21
};
```

规则：
- 第一个枚举值默认为 0
- 每个后续枚举值自动 +1
- 可以手动指定任何值
- 不同的枚举常量可以有相同的值（虽然通常不这样做）

### 3.4 枚举 vs #define

```c
// 方式一：用 #define
#define STATUS_OK    0
#define STATUS_ERROR 1
#define STATUS_BUSY  2

// 方式二：用枚举
enum Status {
    STATUS_OK,    // 0
    STATUS_ERROR, // 1
    STATUS_BUSY   // 2
};
```

| 特性 | enum | #define |
|------|------|--------|
| 类型安全 | 有类型检查 | 纯文本替换，无类型 |
| 调试可见 | 调试器知道名字 | 预处理后只剩数字 |
| 作用域 | 有作用域 | 全局生效 |
| 可用在 switch | 编译器可能警告遗漏 | 不会警告 |
| 存储空间 | 通常占 int 大小 | 不占空间（纯文本替换） |

**推荐**：能用 enum 的地方尽量用 enum，它提供了更好的类型安全和可读性。

### 3.5 typedef + enum

```c
typedef enum {
    LOG_DEBUG,
    LOG_INFO,
    LOG_WARN,
    LOG_ERROR
} LogLevel;

void log_message(LogLevel level, const char *msg) {
    const char *prefix;
    switch (level) {
        case LOG_DEBUG: prefix = "[DEBUG]"; break;
        case LOG_INFO:  prefix = "[INFO]";  break;
        case LOG_WARN:  prefix = "[WARN]";  break;
        case LOG_ERROR: prefix = "[ERROR]"; break;
        default:        prefix = "[????]";  break;
    }
    printf("%s %s\n", prefix, msg);
}
```

## 四、共用体与结构体的嵌套

### 4.1 结构体中的共用体

```c
typedef enum {
    SHAPE_CIRCLE,
    SHAPE_RECTANGLE,
    SHAPE_TRIANGLE
} ShapeType;

typedef struct {
    ShapeType type;
    union {
        struct { double radius; } circle;
        struct { double width; double height; } rectangle;
        struct { double base; double height; } triangle;
    } data;
} Shape;

double area(const Shape *s) {
    switch (s->type) {
        case SHAPE_CIRCLE:
            return 3.14159 * s->data.circle.radius * s->data.circle.radius;
        case SHAPE_RECTANGLE:
            return s->data.rectangle.width * s->data.rectangle.height;
        case SHAPE_TRIANGLE:
            return 0.5 * s->data.triangle.base * s->data.triangle.height;
        default:
            return 0.0;
    }
}
```

这是"标签联合"（Tagged Union）的经典用法。一个 Shape 变量可以表示圆形、矩形或三角形，占用其中最大一种形状所需的内存。

## 五、常见错误

### 5.1 读取共用体最后写入成员以外的成员

```c
union Data d;
d.i = 42;
printf("%f\n", d.f);  // ❌ 未定义行为：读取的不是最后写入的成员
```

虽然很多编译器"能做你想做的事"，但 C 标准说这是未定义行为。安全的做法是读取最后写入的那个成员。

### 5.2 给枚举变量赋任意整数

```c
typedef enum { RED, GREEN, BLUE } Color;

// ❌ 编译可能通过，但逻辑上不对
Color c = 999;  // 999 不对应任何颜色

// ✅ 始终使用枚举常量
Color c = RED;
```

C 语言本身不强制检查枚举值的范围，所以你需要自己保持自律。

### 5.3 共用体包含指针成员时忘记释放

```c
union Data {
    int i;
    char *str;  // 指向动态分配的内存
};

union Data d;
d.str = malloc(100);
// ... 使用 d.str
d.i = 42;  // ❌ 覆盖了 str 指针！之前 malloc 的内存泄漏了！
free(d.str);  // 此时 d.str 已经变成了垃圾值
```

管理共用体中的指针成员需要格外小心，配合标签（tag）来记录当前活跃成员。

## 六、我做得对不对？

创建并运行以下程序：

```c
#include <stdio.h>
#include <string.h>

// 枚举：定义 IP 地址类型
typedef enum {
    IP_V4,
    IP_V6
} IPVersion;

// 带标签的共用体：表示 IP 地址
typedef struct {
    IPVersion version;
    union {
        unsigned char v4[4];       // IPv4: 4 字节
        unsigned short v6[8];      // IPv6: 16 字节
    } addr;
} IPAddress;

void print_ip(const IPAddress *ip) {
    if (ip->version == IP_V4) {
        printf("IPv4: %d.%d.%d.%d\n",
               ip->addr.v4[0], ip->addr.v4[1],
               ip->addr.v4[2], ip->addr.v4[3]);
    } else {
        printf("IPv6: ");
        for (int i = 0; i < 8; i++) {
            printf("%04x", ip->addr.v6[i]);
            if (i < 7) printf(":");
        }
        printf("\n");
    }
}

int main() {
    IPAddress ip1;
    ip1.version = IP_V4;
    ip1.addr.v4[0] = 192;
    ip1.addr.v4[1] = 168;
    ip1.addr.v4[2] = 1;
    ip1.addr.v4[3] = 100;

    IPAddress ip2;
    ip2.version = IP_V6;
    ip2.addr.v6[0] = 0x2001;
    ip2.addr.v6[1] = 0x0db8;
    ip2.addr.v6[2] = 0x0000;
    ip2.addr.v6[3] = 0x0000;
    ip2.addr.v6[4] = 0x0000;
    ip2.addr.v6[5] = 0x0000;
    ip2.addr.v6[6] = 0x0000;
    ip2.addr.v6[7] = 0x0001;

    print_ip(&ip1);
    print_ip(&ip2);

    printf("sizeof(IPAddress) = %zu\n", sizeof(IPAddress));

    return 0;
}
```

预期输出：
```
IPv4: 192.168.1.100
IPv6: 2001:0db8:0000:0000:0000:0000:0000:0001
sizeof(IPAddress) = 20
```

## 七、不对怎么办？

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| 读取共用体成员得到奇怪的值 | 读取的不是最后写入的成员 | 确保读取的成员就是最后写入的那个 |
| 枚举变量值不对 | 枚举定义时插入了手动赋值 | 检查是否有手动指定值打断了自动递增 |
| `sizeof(union)` 比预期大 | 忘了共用体大小 = 最大成员大小 | 检查所有成员，确认最大成员 |
| 共用体指针成员内存泄漏 | 切换成员前没有释放指针指向的内存 | 在给其他成员赋值前，先 free 指针成员 |
| 编译器报枚举 switch 警告 | switch 未覆盖所有枚举值 | 添加 `default` 分支或覆盖所有枚举常量 |

## 本章小结

| 知识点 | 要点 |
|--------|------|
| 共用体定义 | `union 名称 { 成员... };` 所有成员共享内存 |
| 共用体大小 | 等于最大成员的大小 |
| 有效成员 | 同一时间只有最后一个被赋值的成员有效 |
| 标签联合 | 枚举 + 共用体，实现安全的多类型变量 |
| 枚举定义 | `enum 名称 { 常量1, 常量2, ... };` |
| 枚举值 | 默认从 0 开始递增，可手动指定 |
| 枚举 vs #define | 枚举有类型检查和调试优势 |
| 类型双关 | 共用体可用来以不同方式解释同一段内存 |
