# 第17章 · 变量与常量

> "变量就是程序里的'盒子'——给数据贴个标签，以后用这个名字就能找到它。Go的盒子和其他语言有什么不一样？为什么声明方式有好几种？"

---

## 17.1 变量声明：Go的四种"贴标签"方式

变量就是一个有名字的存储位置。你可以把它想象成一个**带标签的盒子**——标签上写着名字，盒子里装着数据。以后你想用这个数据，叫名字就行。

Go语言的变量声明有四种方式，下面逐一来看。

### 方式一：标准声明（最啰嗦，最明确）

```go
var name string
name = "张三"
```

第一行：声明一个叫 `name` 的变量，类型是 `string`（字符串）。这一行只是"造了个盒子贴上标签"，盒子里面现在什么都没有（准确的说是零值，后面会讲）。

第二行：往盒子里放东西——把 `"张三"` 这个字符串放进去。

`var` 是关键字，表示"我要声明变量"。Go的变量声明格式是 `var 变量名 类型`，类型写在变量名**后面**。这和C/Java完全不同（它们是 `类型 变量名`）。Go是静态类型语言，不同于Python等动态类型语言（变量类型在运行时才确定，详见附录I）。同时，Go是强类型语言（不同类型不能混用，必须显式转换，详见附录I），不像JavaScript等弱类型语言（不同类型可自动转换，详见附录I）。

为什么Go要把类型放后面？设计者的解释是：这样更自然——"声明一个叫name的变量，类型是string"，而不是"声明一个string类型的叫name的变量"。而且类型放后面可以省略类型推断（后面会看到），代码更简洁。

### 方式二：声明并初始化（类型自动推断）

```go
var name = "张三"
```

省略类型，Go编译器从右边的值 `"张三"` 自动推断出 `name` 的类型是 `string`。

```go
var age = 25        // Go推断为 int
var height = 1.75   // Go推断为 float64
var isStudent = true // Go推断为 bool
```

### 方式三：短变量声明（最常用）

```go
name := "张三"
```

`:=` 是Go的"短变量声明"符号。它同时做了三件事：**声明 + 类型推断 + 赋值**。

这是Go代码里最常见的写法。注意 `:=` 是声明+赋值，而 `=` 是纯赋值（变量必须已经声明过）。

```go
name := "张三"   // ✅ 声明并赋值
name = "李四"     // ✅ 纯赋值（name已存在）
name := "王五"   // ❌ 编译错误！name已经被声明过了
```

一个例外：如果 `:=` 左边至少有一个**新变量**，那它就可以用：

```go
name := "张三"          // name是新变量
name, age := "李四", 25 // ✅ 可以！age是新变量，name只是被重新赋值
```

`:=` 不能在函数体外使用。每个Go源文件的最外层（包级别）只能用 `var`。

```go
package main

var globalVar = "可以在外面用var"  // ✅

// globalVar2 := "不能在外面用:="  // ❌ 编译错误！

func main() {
    localVar := "函数里面可以用:="  // ✅
}
```

### 方式四：声明后赋值（先占位，后填值）

```go
var name string
var age int
var isStudent bool

name = "张三"
age = 25
isStudent = true
```

当你不确定初始值时，先声明，后面再赋值。

### 批量声明

如果有很多变量要声明，可以用括号分组：

```go
var (
    name       string = "张三"
    age        int    = 25
    isStudent  bool   = true
)
```

短变量声明也可以批量：

```go
name, age, isStudent := "张三", 25, true
```

### 四种方式怎么选

| 场景 | 推荐写法 |
|------|----------|
| 函数内，知道初始值 | `name := "张三"`（短声明，最简洁） |
| 函数内，不知道初始值 | `var name string` 然后后面赋值 |
| 包级别（函数外） | `var name = "张三"`（只能用var） |
| 需要明确类型 | `var name string = "张三"` |

> 🤔 **想多一点**：为什么Go搞这么多声明方式？因为不同场景有不同的最佳选择。Python里只有 `x = 1`，简单是简单了，但你看不到类型信息。C++里 `int x = 1`，类型很清楚但写法死板。Go在这两者之间取了一个平衡——快速写代码时用 `:=`，需要显式类型时用 `var`，包级别用 `var`。

## 17.2 零值：没有赋值的变量不是"空的"

在其他很多语言里，声明一个变量但不赋值，里面装的可能是**随机的垃圾数据**（因为内存里原来是什么就是什么）。这经常导致诡异的Bug——有时候程序能跑，有时候不能。

Go不一样。**Go保证每个变量声明后都会被自动初始化为其类型的"零值"（Zero Value）。** 你不会遇到"未初始化的垃圾数据"。

每种类型的零值：

| 类型 | 零值 | 解释 |
|------|------|------|
| `int`, `int8`, `int64` 等所有整数 | `0` | 数字的零 |
| `float32`, `float64` | `0.0` | 小数的零 |
| `bool` | `false` | 布尔的"假" |
| `string` | `""` | 空字符串，不是nil |
| 指针（pointer） | `nil` | 空指针 |
| 切片（slice） | `nil` | 空切片 |
| 映射（map） | `nil` | 空映射 |
| 通道（channel） | `nil` | 空通道 |
| 接口（interface） | `nil` | 空接口 |
| 函数（func） | `nil` | 空函数 |

举个例子：

```go
var i int
var s string
var b bool
var p *int

fmt.Println(i)   // 0
fmt.Println(s)   // （空字符串，没有任何输出）
fmt.Println(b)   // false
fmt.Println(p)   // <nil>
```

零值机制让Go代码更安全——你永远不会读到未初始化的垃圾数据。这也是为什么Go里很多操作可以直接对零值做而不需要先判空。

比如一个 `int` 类型的计数器，声明就是0，不用特意赋初值：

```go
var counter int
counter++
fmt.Println(counter)  // 1
```

## 17.3 命名规范：驼峰命名，大写导出小写私有

Go对变量命名有一套明确的**社区规范**（虽然编译器不强制，但所有人都在遵守）。

### 驼峰命名法（camelCase）

Go使用驼峰命名：
- `userName`（小驼峰：首字母小写，后面的单词首字母大写）
- `GetAllUsers`（大驼峰：每个单词首字母都大写）

Go**不用**下划线分隔（不像Python的 `user_name`）。唯一的例外是测试文件：`xxx_test.go`。

```go
userName := "张三"       // ✅ 推荐
user_name := "张三"      // ❌ 不符合Go风格
maxRetryCount := 3      // ✅
max_retry_count := 3    // ❌
```

### 大写导出，小写私有（重点！）

这是Go最核心的可见性规则，而且**不需要任何关键字**（不像Java需要 `public`/`private`）：

- **首字母大写** = 公开（exported），其他包可以访问
- **首字母小写** = 私有（unexported），只有本包内部可以访问

```go
package user

var PublicVar = "任何人都能访问"    // 首字母大写 → 公开
var privateVar = "只有user包能访问"  // 首字母小写 → 私有

func GetName() string {  // 公开函数
    return "张三"
}

func validateEmail(email string) bool {  // 私有函数
    return true
}
```

这个规则同时适用于：变量、函数、结构体、结构体字段、接口、方法——一切名字。Go没有 `public`/`private`/`protected` 关键字，就靠首字母大小写。

这个设计非常巧妙——你一眼就能看出一个东西是公开的还是私有的，不需要找关键字。

### 变量名长度

Go社区的建议是：**作用域越小，名字可以越短；作用域越大，名字越要清楚地表达含义。**

```go
// 好：循环变量i范围很小，一个字母够了
for i := 0; i < 10; i++ {
    fmt.Println(i)
}

// 好：包级常量，范围很大，名字要完整
const DefaultMaxConnectionCount = 100
```

## 17.4 常量：不能改的变量

常量就是在**编译时就确定**且**运行期间不可修改**的值。用 `const` 关键字声明。

```go
const Pi = 3.14159
const MaxRetry = 3
const AppName = "Go后端教程"
```

常量和变量的区别：
- 常量用 `const`，变量用 `var` 或 `:=`
- 常量必须在声明时赋值
- 常量不能用 `:=`
- 常量的值在编译时就确定了，不能是运行时才能知道的值（比如函数调用的结果）

```go
const Pi = 3.14            // ✅
const Max = getMax()       // ❌ 编译错误！函数返回值是运行时确定的
const now = time.Now()     // ❌ 编译错误！

var now = time.Now()       // ✅ 变量可以
```

### 批量声明常量

```go
const (
    StatusOK    = 200
    StatusNotFound = 404
    StatusError = 500
)
```

### 常量的类型

常量可以是布尔、数字（整数、浮点数、复数）和字符串：

```go
const (
    IsEnabled = true
    MaxUsers  = 1000
    Rate      = 0.75
    Greeting  = "你好"
)
```

> 🤔 **想多一点**：你可能会想："常量不就是不能改的变量吗，有什么大不了的？"常量的意义不仅是"不能改"，更是"编译时确定"。这意味着编译器可以做很多优化——比如把常量直接嵌入到二进制文件中，运行时不用再计算。另外，有些东西必须用常量，比如数组的长度（后面会讲）。

## 17.5 iota：Go的计数器神器

`iota` 是Go语言里一个很特别的东西——它是**常量计数器**，只能在 `const` 声明块里使用。

### 基本用法

```go
const (
    Monday = iota     // 0
    Tuesday           // 1
    Wednesday         // 2
    Thursday          // 3
    Friday            // 4
    Saturday          // 5
    Sunday            // 6
)
```

每次在 `const` 块里出现 `iota`，它从0开始，每增加一行常量声明，`iota` 的值就加1。

注意：`Tuesday` 后面没有 `= iota`，但Go会自动沿用上一行的表达式，并且 `iota` 自动递增。

### iota的原理

每遇到一个 `const` 关键字，`iota` 重置为0。然后在同一个 `const` 块中，每新增一行常量声明，`iota` 加1。

```go
const a = iota  // 0
const b = iota  // 0（新的const块，iota重置了）

const (
    c = iota    // 0
    d = iota    // 1
    e = iota    // 2
)
```

### 高级用法：文件大小单位

这是一个经典的面试/笔试题目，也是 `iota` 最优雅的应用：

```go
const (
    _  = iota             // 0，用_忽略
    KB = 1 << (10 * iota) // 1 << 10 = 1024
    MB                    // 1 << 20 = 1048576
    GB                    // 1 << 30 = 1073741824
    TB                    // 1 << 40
)
```

逐步拆解：
- 第一行：`iota = 0`，赋值给 `_`（忽略）
- 第二行：`iota = 1`，`1 << (10 * 1)` = `1 << 10` = 1024
- 第三行：`iota = 2`，`1 << (10 * 2)` = `1 << 20` = 1048576
- 第四行：`iota = 3`，`1 << (10 * 3)` = `1 << 30` = 1073741824
- ...

`<<` 是左移位运算符—— `1 << n` 就是把1的二进制往左移n位，相当于 2ⁿ。所以 `1 << 10` = 2¹⁰ = 1024。（位运算符详见第19章）

### iota也可以参与运算

```go
const (
    Read    = 1 << iota  // 1 << 0 = 1
    Write                 // 1 << 1 = 2
    Execute               // 1 << 2 = 4
)
```

这种模式非常适合定义**位掩码（bitmask）**——后面学文件权限时你会经常见到：

```go
const (
    FlagA = 1 << iota  // 1 (二进制 001)
    FlagB               // 2 (二进制 010)
    FlagC               // 4 (二进制 100)
)

flags := FlagA | FlagC  // 1 | 4 = 5 (二进制 101)，同时拥有FlagA和FlagC（按位或运算符详见第19章）
```

> 🤔 **想多一点**：`iota` 是Go里少数几个"魔法"特性之一。它本质上就是一个行号计数器，但因为可以在编译时计算，所以很适合生成有规律的常量序列。不过不要滥用它——如果常量之间的关系不直观（比如 `Monday=1` 但 `Friday=5`），手动写出数字反而更清晰。

## 本章小结

| 知识点 | 核心要点 |
|--------|----------|
| 四种声明方式 | `var x type` / `var x =` / `x :=` / `var x type` 后赋值 |
| `:=` vs `=` | `:=` 声明+赋值，`=` 纯赋值；`:=` 不能用在函数外 |
| 零值 | Go保证变量有初始值（int→0, string→"", bool→false, 指针→nil） |
| 命名规范 | 驼峰命名；首字母大写=公开导出，首字母小写=包内私有 |
| 常量 | `const` 声明，编译时确定，不可修改 |
| iota | 常量计数器，每个const块从0开始递增；适合枚举和位掩码 |
