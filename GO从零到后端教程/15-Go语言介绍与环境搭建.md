## 15.1 Go的诞生故事：三个老炮受够了C++

2007年，Google有三个工程师——Ken Thompson、Rob Pike和Robert Griesemer——他们实在是受够了。

Ken Thompson是谁？Unix操作系统的发明者之一，B语言的作者，图灵奖得主。放到武侠小说里就是独孤求败级别的人物。Rob Pike，Unix核心开发者之一，UTF-8编码的共同设计师。Robert Griesemer，Google V8 JavaScript引擎的核心开发者（V8就是Chrome浏览器里跑JavaScript的那个引擎）。

这三个大佬每天在Google写C++。那个时候Google的后端服务基本都是C++写的，C++功能强大没错，但编译速度实在太慢了——改一行代码，可能要等十几分钟甚至半小时才能编译完。你能想象吗？改了一行，然后泡杯咖啡、上个厕所、聊会天，回来发现还没编译完。

而且C++太复杂了。关键字多、特性多、各种历史遗留问题。三个大佬想：我们能不能做一门新语言？要求很简单：

1. **编译必须快**——不能喝咖啡等编译
2. **语法必须简单**——不能让新人花三个月学语法
3. **天生支持并发**——现代服务器都是多核的，必须充分利用
4. **能编译成独立的可执行文件**——部署时不需要装一堆依赖

于是Go语言诞生了。2009年11月，Go正式对外开源发布。2012年3月，Go 1.0发布，承诺向后兼容（这个承诺一直保持到今天）。

Go语言的吉祥物是一只**地鼠（Gopher）**。为什么是地鼠？因为Gopher这个词本身就是"地鼠"，和"Go for"谐音。而且地鼠这种动物非常高效、目标明确——正好契合Go语言的设计哲学。

### Go的设计哲学

Go的核心理念可以用三个词概括：

**简洁（Simplicity）**：Go只有**25个关键字**。对比一下：Java有50多个，C++有90多个，Python有35个。Go的设计者刻意控制了语言的复杂度——"少即是多"。

**高效（Efficiency）**：Go是静态类型（变量类型在编译时确定，详见附录I）的编译型语言（写完代码需要先编译成机器码才能运行，详见附录I），编译快、运行快、内存占用少。编译出的程序直接是二进制文件，不需要虚拟机，不需要解释器（Python等解释型语言则需要解释器逐行翻译执行，详见附录I）。

**并发（Concurrency）**：Go从语言层面支持并发编程。goroutine（轻量级线程）和channel（管道通信）是Go最闪亮的两张名片。

> 🤔 **想多一点**：很多人问"为什么学Go而不是学Python做后端？"Python做后端当然可以（Django、Flask都很成熟），但Go有两个Python无法比拟的优势：一是并发——Python有全局解释器锁（GIL），多线程实际上是"假多线程"，而Go的goroutine是真并发；二是部署——Go编译出来就是一个文件，拷过去就能跑，Python需要装解释器和一堆依赖包。当然，做数据科学、AI、脚本工具，Python明显更强。选择语言要根据场景来，不存在"最好的语言"。

## 15.2 安装Go：让你电脑能听懂Go语言

### Windows用户

1. 打开浏览器，访问 `https://go.dev/dl/`
2. 下载Windows版的安装包（扩展名是 `.msi`）
3. 双击下载的 `.msi` 文件，一路点"Next"（默认会安装到 `C:\Program Files\Go\`）
4. 打开**命令提示符**或**PowerShell**，输入：

```powershell
go version
```

如果看到类似这样的输出（版本号以你下载的为准）：

```
go version go1.xx.x windows/amd64
```

恭喜！安装成功。

### Mac用户

1. 访问 `https://go.dev/dl/`
2. 下载macOS版的安装包（`.pkg` 文件）
3. 双击安装，一路点"继续"
4. 打开**终端（Terminal）**，输入：

```bash
go version
```

看到版本号就成功了。如果你用Homebrew，也可以一条命令搞定：

```bash
brew install go
```

### Linux用户（Ubuntu/Debian）

```bash
sudo apt update
sudo apt install golang-go
go version
```

或者从官网下载tar包手动安装（推荐，因为apt源里的版本可能比较旧）：

```bash
# 请到 https://go.dev/dl/ 查看最新版本的下载链接，替换下面URL中的版本号
wget https://go.dev/dl/go1.xx.x.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.xx.x.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc
go version
```

## 15.3 GOROOT和GOPATH：两个容易搞混的概念

安装好Go之后，有两个环境变量你需要了解：`GOROOT` 和 `GOPATH`。它们名字很像，但作用完全不同。

### GOROOT：Go语言的"老家"

`GOROOT` 是Go语言的**安装目录**。里面放着Go编译器、标准库源码、各种官方工具。

在Windows上通常是 `C:\Program Files\Go\`，在Mac/Linux上通常是 `/usr/local/go/`。

你不需要手动设置这个环境变量——安装程序会自动搞定。如果你输入 `go env GOROOT`，就能看到它的值：

```bash
$ go env GOROOT
C:\Program Files\Go    # Windows
/usr/local/go          # Mac / Linux
```

你可以把它想象成一栋房子的**地基**——Go语言本身就在这个目录里，你不会改动它。

### GOPATH：你的"工作台"

`GOPATH` 是Go语言的**工作目录**。你写的代码、下载的第三方库，都放在这里。

在Go 1.11之前（Go Modules出现之前），你**必须**把所有Go代码放在 `GOPATH/src/` 下面才能编译。如果你把代码放在别的地方，Go根本不认。

这导致了一个尴尬的问题：你写的所有Go项目，不管是不是相关的，都必须挤在同一个目录树下。就像你写日记、记账本、写小说，全部必须放在同一个文件夹里。

### Go Modules：新时代来啦！

从Go 1.11开始，Go引入了**Go Modules**机制。从Go 1.16开始，Go Modules成为默认行为。

有了Go Modules，你的项目可以放在硬盘上**任何位置**，不再受GOPATH的限制。每个项目有自己的 `go.mod` 文件，用来声明"我是谁"和"我依赖了哪些包"。

举个例子：

```bash
# 在你喜欢的任意位置创建项目
mkdir ~/my-project
cd ~/my-project
go mod init my-project    # 创建 go.mod，初始化模块
```

生成的 `go.mod` 文件长这样：

```
module my-project

go 1.21  （实际版本号取决于你安装的Go版本）
```

`module my-project` 就相当于给你的项目起了个名字。以后别人引用你的代码、你引用别人的代码，都靠这个名字。

> 🤔 **想多一点**：那GOPATH现在还有什么用？它现在主要用来存放**缓存**。当你运行 `go mod tidy` 下载依赖包时，这些包会被缓存到 `GOPATH/pkg/mod/` 下面。你不需要关心这个目录的内容，Go会自动管理。所以现在的结论很简单：**GOPATH让Go自动管，你只管在自己的项目目录里写代码就行。**

## 15.4 配置开发环境：VS Code（强烈推荐）

在写代码之前，你得有个趁手的编辑器。**记事本可以写代码，但不适合写代码**——就像你可以用石头敲钉子，但没人会这么做。

本节教你配置 VS Code 作为 Go 的开发环境。如果你已经有自己用惯的编辑器（GoLand、Vim、Emacs），可以跳过。

### 为什么选 VS Code

- 免费，跨平台（Windows/Mac/Linux 一样用）
- Go 支持极好（微软和 Go 团队共同维护 Go 扩展）
- 保存时自动格式化（gofmt）、自动补全、跳转定义、断点调试
- 全世界的 Go 开发者八成都在用

### 第一步：安装 VS Code

去 `https://code.visualstudio.com/` 下载安装。Windows 用户注意：**安装时勾选"添加到 PATH"**（这样你就能在终端里输入 `code .` 打开当前目录）。

装完后打开 VS Code，你会看到一个欢迎页。先不管它。

### 第二步：安装 Go 扩展

1. 点击左侧活动栏的**扩展图标**（四个方块拼成的图标，或者按 `Ctrl+Shift+X`）
2. 在搜索栏输入 `go`
3. 找到 **Go**（作者：Go Team at Google），点击**安装**
4. 安装完成后，点右下角弹出的提示，安装所有 Go 工具（gopls、dlv、staticcheck 等）。这些工具会帮你自动补全、跳转定义、调试代码。

### 第三步：验证配置

用 VS Code 打开你刚才创建的 `hello` 项目文件夹：

```bash
code ~/go-projects/hello
```

（如果在 Windows PowerShell 里 `code` 命令不生效，关掉 PowerShell 重开一次，或者直接从开始菜单启动 VS Code，点 `File → Open Folder`。）

打开 `main.go`，你会看到：
- 关键字（`package`、`func`、`import`）有颜色高亮
- 输入 `fmt.` 时会自动弹出 `fmt` 包的所有函数列表
- 保存文件（`Ctrl+S`）时，代码自动格式化——**缩进、空格、import 顺序全部自动调好**

### 第四步：在 VS Code 里直接运行代码

不需要切回终端。在 `main.go` 里按 `F5`（或者点左侧的「运行和调试」→「创建 launch.json」→ 选 Go），VS Code 会直接在下方终端窗口运行你的程序并显示结果。

### 必备快捷键（现在就记住）

| 快捷键 | 功能 | 为什么重要 |
|--------|------|-----------|
| `Ctrl+S` | 保存 + 自动格式化 | 你会在每次写完代码后本能地按 |
| `Ctrl+~` | 打开/关闭内置终端 | 不用切窗口 |
| `Ctrl+P` | 快速打开文件 | 项目文件多了以后救命 |
| `Ctrl+Shift+P` | 命令面板 | 搜任何功能（"格式化文档"、"Go: Install Tools"） |
| `F12` | 跳转到定义 | 点一个函数名按F12，直接跳到它被定义的地方 |
| `Ctrl+单击` | 同上（鼠标版） | 按住Ctrl点函数名，效果同F12 |
| `F5` | 开始调试 | 运行程序，可以设断点 |

### 如果配置失败

- **Go 扩展安装后在右下角弹出"无法安装工具"**：通常是网络问题。打开终端，手动设置 Go 代理：
  ```bash
  go env -w GOPROXY=https://goproxy.cn,direct
  ```
  然后按 `Ctrl+Shift+P` → 输入 `Go: Install/Update Tools` → 全选 → 确定。

- **代码没有颜色/没有自动补全**：检查 Go 扩展是否已启用。点击扩展图标，搜索 `@installed go`，确认 Go 扩展显示为"已启用"。

- **`code` 命令不存在**：Windows 用户重新运行 VS Code 安装程序，确保勾选 "Add to PATH"；Mac 用户在 VS Code 里按 `Cmd+Shift+P` → 输入 `Shell Command: Install 'code' command in PATH`。

> 💡 从现在开始，**所有代码都在 VS Code 里写**。它比记事本强一万倍，而且免费。花10分钟配置好，后面115章都受益。

## 15.5 第一个程序：Hello, World!

终于到了最激动人心的时刻——写你的第一个Go程序。

### 创建项目

在你喜欢的位置新建一个文件夹，比如 `~/go-projects/hello/`，在里面创建一个文件 `main.go`。

用任何文本编辑器（VS Code、Notepad++、记事本都行）打开 `main.go`，输入以下内容：

```go
package main

import "fmt"

func main() {
	fmt.Println("Hello, World!")
}
```

保存文件。就这5行。麻雀虽小，五脏俱全。

### 运行它

打开终端，进入这个目录，运行：

```bash
go run main.go
```

你会看到：

```
Hello, World!
```

你刚刚写出了人生第一个Go程序！这个程序在计算机上运行了，计算机理解了你写的代码，把它变成了屏幕上的"Hello, World!"。

### 逐行拆解

这5行代码，每一行都不是多余的。让我们像拆乐高一样拆开它：

**第1行：`package main`**

每个Go文件的第一行必须是 `package` 声明，告诉Go："我这个文件属于哪个包（package）？"

`main` 是一个特殊的包名——它告诉Go："我这个包是可以直接运行的程序，不是一个给别人用的库。"如果一个Go文件不属于 `main` 包，它不能独立运行，只能被别的程序引用。

**第2行：空行**

Go编译器不关心空行。但程序员关心——用空行分隔不同逻辑的部分，代码更可读。

**第3行：`import "fmt"`**

`import` 的意思是"我要用到标准库里的某个包"。`"fmt"` 是Go标准库里的格式化包（format的缩写），负责格式化输入输出——打印文字、读取输入、格式化字符串等。

等你后面学到更复杂的程序，import里面会写很多包名：

```go
import (
	"fmt"
	"os"
	"net/http"
)
```

**第4行：空行**

又是空行，分隔 import 和 main 函数。

**第5行：`func main() {`**

`func` 是关键字，表示"我要定义一个函数"。`main` 是函数名，这个名字是**有特殊含义的**——`main` 函数是程序的**入口**。程序启动时，Go运行时会自动找到 `main` 包里的 `main` 函数，从它的第一行开始执行。

注意：`main` 函数**没有参数，也没有返回值**。`()` 里面什么都不写。这和其他语言（比如C语言的 `int main(int argc, char *argv[])`）不一样。

`{` 是函数体的开始。Go的语法规定：**左花括号必须和 `func` 在同一行**，不能另起一行。如果你写成：

```go
func main()
{  // ❌ 编译错误！
	fmt.Println("Hello, World!")
}
```

Go会直接报错，编译不过。这是Go的设计者刻意为之的——强制统一的代码风格（后面讲到 `gofmt` 你就明白了）。

**第6行：`fmt.Println("Hello, World!")`**

`fmt.Println` 是调用 `fmt` 包里的 `Println` 函数。`Println` = Print + ln（line），意思是"打印一行并自动换行"。

- `fmt.` 表示"我要用fmt包里的东西"
- `Println` 是函数名
- `("Hello, World!")` 是传给函数的参数——要打印的字符串

`fmt` 包里还有几个兄弟函数：
- `fmt.Print("Hello")` —— 打印，不换行
- `fmt.Println("Hello")` —— 打印，自动换行
- `fmt.Printf("Hello, %s!", "World")` —— 格式化打印（类似C的printf）

**第7行：`}`**

函数体结束。程序执行到这里，`main` 函数就结束了。`main` 函数结束意味着整个程序结束。

> 🤔 **想多一点**：你可能在想——"为什么不是Python那样的 `print('Hello')` 而要写 `fmt.Println`？看起来好啰嗦啊。"这是Go的设计哲学：**显式优于隐式**。`fmt.Println` 明确告诉你这个函数来自 `fmt` 包，而不是某个不知道从哪冒出来的全局函数。当你的项目有几千行代码、引入了十几个包时，这种显式声明会让代码的可读性大幅提升——你一眼就能看出每个函数来自哪个包。

## 15.6 go run / go build / go install：三个命令的区别

你已经用过 `go run` 了，但Go还有 `go build` 和 `go install`。这三个命令都跟"编译运行"有关，但用途不同。

### go run：编译并运行（临时）

```bash
go run main.go
```

`go run` 做了两件事：
1. 先把 `main.go` 编译成一个**临时的可执行文件**（放在系统的临时目录里）
2. 然后**立即运行**这个临时文件

`go run` 适合**开发调试**时快速看看效果。它产生的临时文件在程序结束后可能就删掉了。

它的缺点是**慢**——每次 `go run` 都要重新编译。而且生成的临时文件你找不到也拷不走。

### go build：只编译（保留结果）

```bash
go build main.go
```

`go build` 只做**编译**，不运行。它会生成一个可执行文件放在当前目录下：
- Windows：`main.exe`
- Mac / Linux：`main`

然后你可以自己运行它：

```bash
# Windows
.\main.exe

# Mac / Linux
./main
```

`go build` 适合**最终交付**时使用——你可以把这个可执行文件直接拷贝到服务器上运行，服务器上甚至不需要装Go环境（因为Go编译出来的是二进制文件，自带运行所需的一切）。

如果你在项目根目录下直接运行 `go build`（不带文件名），它会自动找 `main` 包编译，生成的可执行文件名就是项目文件夹的名字。

### go install：编译并安装到系统

```bash
go install
```

`go install` 编译后，会把可执行文件放到 `GOPATH/bin/` 目录下。然后你可以在任何位置（任何终端、任何目录）直接运行这个程序——因为 `GOPATH/bin/` 一般已经加到了系统的PATH环境变量里。

`go install` 适合**安装工具类程序**——比如你写了一个代码生成器，用 `go install` 安装后，以后在任何项目里都能直接调用它。

### 三种方式总结

| 命令 | 做什么 | 生成的文件在哪 | 适合场景 |
|------|--------|---------------|----------|
| `go run` | 编译 + 运行 | 系统临时目录（用完可能删） | 开发调试 |
| `go build` | 只编译 | 当前目录 | 部署交付 |
| `go install` | 编译 + 安装 | `GOPATH/bin/` | 安装工具 |

### 体验一下差异

你可以在刚才的 `hello` 项目里试试：

```bash
go build          # 生成 hello.exe（Windows）或 hello（Mac/Linux）
ls -lh hello*     # 看看文件多大
./hello           # 运行它
```

你会发现编译出来的文件有1~2MB。为什么"Hello, World!"这么简单的程序要1MB？因为Go把整个运行时（runtime）都编译进去了——包括垃圾回收器（GC，自动清理不再使用的内存，详见附录I）、goroutine调度器等等。好处是你部署时不需要额外安装任何东西，坏处就是哪怕最简程序也有点体积。但这1~2MB在未来只会越来越无所谓——磁盘和带宽都很便宜。

## 15.7 跨平台编译：在Windows上编译出Linux程序

Go的编译器自带**交叉编译**能力——你可以在一台电脑上编译出给另一台电脑运行的程序。比如在Windows上编译出一个能在Linux服务器上跑的程序。

这就涉及两个环境变量：

- **GOOS**（Go Operating System）：目标操作系统
- **GOARCH**（Go Architecture）：目标CPU架构

### 常用组合

| GOOS | GOARCH | 说明 |
|------|--------|------|
| `windows` | `amd64` | Windows 64位 |
| `windows` | `386` | Windows 32位 |
| `linux` | `amd64` | Linux 64位（服务器最常见） |
| `linux` | `arm64` | Linux ARM 64位（树莓派等） |
| `darwin` | `amd64` | macOS Intel芯片 |
| `darwin` | `arm64` | macOS Apple Silicon（M1/M2/M3） |

### 动手试试

在Windows的PowerShell里：

```powershell
$env:GOOS = "linux"
$env:GOARCH = "amd64"
go build
```

你会得到一个叫 `hello` 的文件（没有 `.exe` 后缀，因为Linux不认这个）。这个文件无法在你的Windows上运行，但把它拷到Linux服务器上就能直接跑。

在Mac或Linux的终端里：

```bash
GOOS=linux GOARCH=amd64 go build
```

也是一样的效果。

如果你经常需要交叉编译，可以在环境变量里设置好，也可以每次在命令行里直接指定：

```bash
# 一条命令编译出三个平台的版本
GOOS=windows GOARCH=amd64 go build -o hello.exe
GOOS=linux GOARCH=amd64 go build -o hello-linux
GOOS=darwin GOARCH=amd64 go build -o hello-mac
```

这个特性非常实用：你开发用的可能是Windows，但服务器是Linux，用不着在服务器上装Go编译器——在本地编译好直接上传就行。

> 🤔 **想多一点**：不是所有语言都能轻松交叉编译。比如C/C++交叉编译需要配置复杂的工具链，Python更是"寄希望在目标机器上有Python解释器"。Go从设计之初就考虑了交叉编译的需求，这是Go在云原生和容器部署领域特别受欢迎的原因之一——你可以在CI/CD流水线里一条命令编译出多个平台的镜像。
