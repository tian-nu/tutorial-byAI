# 附录F · HTTP状态码大全

> "HTTP状态码是服务器和客户端之间的'暗号'——200表示一切正常，404表示找不到，500表示服务器崩了。本附录列出所有常用HTTP状态码，附中文说明和典型场景。"

---

## F.1 1xx：信息响应

请求已收到，继续处理中。

| 状态码 | 名称 | 说明 | 典型场景 |
|--------|------|------|---------|
| 100 | Continue | 继续发送请求体 | 大文件上传前先询问服务器 |
| 101 | Switching Protocols | 切换协议 | HTTP升级到WebSocket |
| 102 | Processing | 正在处理 | WebDAV长时间处理 |

后端工作中很少直接处理1xx——框架和中间件自动处理了。

---

## F.2 2xx：成功

| 状态码 | 名称 | 说明 | 典型场景 |
|--------|------|------|---------|
| 200 | OK | 请求成功 | GET/PUT/PATCH成功返回 |
| 201 | Created | 资源已创建 | POST创建新资源 |
| 202 | Accepted | 已接受，处理中 | 异步任务提交 |
| 204 | No Content | 成功但无响应体 | DELETE成功后 |
| 206 | Partial Content | 部分内容 | 断点续传、视频分片 |

### 使用指南

- **200 vs 201**：200是"搞定了，这是结果"；201是"创建好了，这是新资源"。创建操作建议返回201
- **204 No Content**：删除操作的最佳响应——"删完了，没什么可说的"

---

## F.3 3xx：重定向

| 状态码 | 名称 | 说明 | 典型场景 |
|--------|------|------|---------|
| 301 | Moved Permanently | 永久重定向 | 域名从 old.com 迁到 new.com |
| 302 | Found | 临时重定向 | 未登录跳转到登录页 |
| 304 | Not Modified | 未修改 | 浏览器缓存（返回空body） |
| 307 | Temporary Redirect | 临时重定向（保留方法） | 类似302但不改POST为GET |
| 308 | Permanent Redirect | 永久重定向（保留方法） | 类似301但不改POST为GET |

### 301 vs 302

- **301**：告诉搜索引擎"旧URL永久不用了，请用新URL收录"
- **302**：告诉搜索引擎"旧URL还会用的，这只是临时跳转"

错误地把302写成301会导致SEO问题——搜索引擎会停止索引你的旧URL。

---

## F.4 4xx：客户端错误

| 状态码 | 名称 | 说明 | 典型场景 |
|--------|------|------|---------|
| 400 | Bad Request | 请求参数错误 | JSON格式错误、缺少必填字段 |
| 401 | Unauthorized | 未认证 | 缺少/无效的JWT Token |
| 402 | Payment Required | 需要付费 | 几乎不用（预留），近年支付场景偶尔使用 |
| 403 | Forbidden | 禁止访问 | 已登录但权限不够 |
| 404 | Not Found | 资源不存在 | ID不存在、路由未注册 |
| 405 | Method Not Allowed | 方法不允许 | 用POST访问只支持GET的路由 |
| 406 | Not Acceptable | 无法接受 | 请求的格式不支持 |
| 408 | Request Timeout | 请求超时 | 客户端请求太慢 |
| 409 | Conflict | 冲突 | 邮箱已注册、用户名重复 |
| 410 | Gone | 资源永久不存在 | 已删除且不保留痕迹 |
| 411 | Length Required | 需要Content-Length | 服务器要求指定长度 |
| 413 | Payload Too Large | 请求体太大 | 上传文件超过限制 |
| 414 | URI Too Long | URL太长 | GET参数过多 |
| 415 | Unsupported Media Type | 不支持的媒体类型 | Content-Type不是application/json |
| 422 | Unprocessable Entity | 无法处理的实体 | 参数格式正确但语义错误 |
| 429 | Too Many Requests | 请求过于频繁 | 触发限流、被限频 |

### 401 vs 403（经常混淆）

- **401**："你是谁？我没见过你。"（缺少认证信息）
- **403**："我知道你是谁，但你不配。"（有认证但无权限）

就像进公司大楼：
- 401：没带工卡，门禁不认识你
- 403：有工卡，但试图进CEO办公室

### 404 vs 410

- **404**：这个资源可能暂时不存在，也可能从没存在过
- **410**：这个资源以前有，现在永久删除了

## F.5 5xx：服务端错误

| 状态码 | 名称 | 说明 | 典型场景 |
|--------|------|------|---------|
| 500 | Internal Server Error | 服务器内部错误 | panic、未捕获异常、空指针 |
| 501 | Not Implemented | 功能未实现 | 接口留空还没做 |
| 502 | Bad Gateway | 网关错误 | Nginx代理的后端服务挂了 |
| 503 | Service Unavailable | 服务不可用 | 正在维护或过载 |
| 504 | Gateway Timeout | 网关超时 | 后端服务响应太慢 |
| 505 | HTTP Version Not Supported | HTTP版本不支持 | 用了HTTP/0.9 |

### 500错误的排查思路

出现500时，第一反应不是改代码，而是：
1. 看日志——panic信息里通常有精确的文件和行号
2. 查数据库——是不是连接断了、SQL写错了
3. 查外部依赖——Redis是不是挂了、第三方API是不是超时了
4. 本地重现——用同样的参数本地调一下

## F.6 状态码选择速查表

"我要返回什么状态码？"

| 场景 | 推荐状态码 |
|------|-----------|
| GET成功，返回数据 | 200 |
| POST创建资源成功 | 201 |
| POST异步任务提交 | 202 |
| PUT/PATCH更新成功 | 200 |
| DELETE删除成功 | 204 |
| 请求参数校验失败 | 400 |
| 未登录访问需认证接口 | 401 |
| Token过期 | 401 |
| 已登录但无权限 | 403 |
| 用户被封禁 | 403 |
| 资源ID不存在 | 404 |
| 接口路由未定义 | 404 |
| 邮箱/用户名已存在 | 409 |
| 上传文件太大 | 413 |
| 触发限流 | 429 |
| 代码panic | 500 |
| 数据库连接失败 | 500 |
| 上游服务挂了 | 502 |
| 服务熔断/降级 | 503 |

---

后端的职责就是用正确的状态码告诉前端发生了什么——选对状态码，排查问题和监控告警都会方便很多。
