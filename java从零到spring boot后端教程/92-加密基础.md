# 92-加密基础

> 💡 你给朋友寄一份机密文件。你可以用带锁的箱子（加密），也可以只给文件折两折用胶水粘一下（编码）。如果有人半路截获——带锁的箱子他没钥匙打不开；胶水粘的他撕开就能看。加密不是让数据"看不见"，而是让数据"看不懂"。本章带你区分编码、哈希、对称加密、非对称加密这四个最容易被混淆的概念。

## 92.1 编码 vs 加密 vs 哈希——一图分清

| | 编码（Encoding） | 加密（Encryption） | 哈希（Hashing） |
|------|------|------|------|
| 能否还原 | ✅ 能 | ✅ 能（有密钥） | ❌ 不能 |
| 需要密钥 | ❌ 不需要 | ✅ 需要 | ❌ 不需要 |
| 目的 | 格式转换 | 保密 | 验证完整性 |
| 例子 | Base64、URL Encode | AES、RSA | SHA-256、BCrypt |
| 类比 | 把中文翻译成英文 | 把文件锁进保险箱 | 做一碗汤，不能还原成食材 |

### 用代码感受

```java
// 编码：可逆，无密钥
String encoded = Base64.getEncoder().encodeToString("hello".getBytes());
// → "aGVsbG8="
String decoded = new String(Base64.getDecoder().decode(encoded));
// → "hello"

// 哈希：不可逆，无密钥
String hashed = DigestUtils.sha256Hex("hello");
// → "2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824"
// 你无法从这一串还原出 "hello"

// 加密：可逆，需要密钥
Cipher cipher = Cipher.getInstance("AES");
cipher.init(Cipher.ENCRYPT_MODE, secretKey);
byte[] encrypted = cipher.doFinal("hello".getBytes());
// → 一堆乱码
cipher.init(Cipher.DECRYPT_MODE, secretKey);
byte[] decrypted = cipher.doFinal(encrypted);
// → "hello"
```

## 92.2 哈希算法——验证完整性，不保密

### 常见哈希算法

| 算法 | 输出长度 | 安全性 | 现状 |
|------|----------|--------|------|
| MD5 | 128 bit | ❌ 已破解 | 禁止用于安全场景 |
| SHA-1 | 160 bit | ❌ 已破解 | 禁止用于安全场景 |
| SHA-256 | 256 bit | ✅ 安全 | 当前主流 |
| SHA-512 | 512 bit | ✅ 安全 | 更保守的选择 |
| BCrypt | 可配置 | ✅ 安全 | 密码存储首选 |

### 哈希的用途

1. **文件完整性校验**：下载文件时网站提供的 SHA-256 值，你可以算一遍对比
2. **密码存储**：数据库不存明文密码，存哈希值。登录时对输入做哈希再对比
3. **数据去重**：给文件算哈希，相同哈希 = 相同文件

### 为什么密码一定要用 BCrypt 而不是 SHA-256？

```java
// ❌ 错误：用 SHA-256 存密码
String hashedPassword = DigestUtils.sha256Hex("123456");
// → "8d969eef6ecad3c29a3a629280e686cf0c3f5d5a86aff3ca12020c923adc6c92"
```

问题在于**彩虹表攻击**。攻击者提前算好了一万个常见密码的 SHA-256 值：`123456 → 8d969...`。拿到你数据库，一比对就知道明文密码。

**BCrypt 为什么安全？**

```java
BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();

String hash1 = encoder.encode("123456");
// → "$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy"

String hash2 = encoder.encode("123456");
// → "$2a$10$eGpQrFJcLJD8TZXyHCFdjOqOwKBvCkVqX3FAe7MGnpFG3tR9uFn4q"
```

同一个密码"123456"，两次编码结果不同！因为 BCrypt 自动加了随机盐（salt）。攻击者不能提前算彩虹表，必须对每个盐从头暴力破解。

## 92.3 对称加密——一把钥匙开一把锁

**对称**的含义：加密和解密用**同一把密钥**。

### AES 加密实战

```java
public class AesUtil {

    private static final String ALGORITHM = "AES/GCM/NoPadding";

    public static String encrypt(String plainText, SecretKey key) throws Exception {
        Cipher cipher = Cipher.getInstance(ALGORITHM);
        cipher.init(Cipher.ENCRYPT_MODE, key);

        byte[] iv = cipher.getIV();  // GCM 模式自动生成 IV
        byte[] cipherText = cipher.doFinal(plainText.getBytes(StandardCharsets.UTF_8));

        // IV + 密文拼接，IV 不需要保密
        byte[] combined = new byte[iv.length + cipherText.length];
        System.arraycopy(iv, 0, combined, 0, iv.length);
        System.arraycopy(cipherText, 0, combined, iv.length, cipherText.length);

        return Base64.getEncoder().encodeToString(combined);
    }

    public static String decrypt(String encryptedText, SecretKey key) throws Exception {
        byte[] combined = Base64.getDecoder().decode(encryptedText);

        byte[] iv = new byte[12];  // GCM 推荐 12 字节 IV
        byte[] cipherText = new byte[combined.length - 12];
        System.arraycopy(combined, 0, iv, 0, 12);
        System.arraycopy(combined, 12, cipherText, 0, cipherText.length);

        Cipher cipher = Cipher.getInstance(ALGORITHM);
        cipher.init(Cipher.DECRYPT_MODE, key, new GCMParameterSpec(128, iv));

        return new String(cipher.doFinal(cipherText), StandardCharsets.UTF_8);
    }

    public static SecretKey generateKey() throws Exception {
        KeyGenerator keyGen = KeyGenerator.getInstance("AES");
        keyGen.init(256);  // AES-256
        return keyGen.generateKey();
    }
}
```

### 对称加密的优缺点

| 优点 | 缺点 |
|------|------|
| 速度快，适合大数据量 | 密钥分发困难：怎么把密钥安全地给对方？ |
| 实现简单 | N 个人之间通信需要 N(N-1)/2 个密钥 |

> 🤔 想多一点：AES-GCM 是目前推荐的 AES 模式，因为它同时提供加密和完整性校验。老旧的 AES-ECB 模式不安全——相同的明文块会产生相同的密文块，能被看出模式。

## 92.4 非对称加密——两把钥匙

**非对称**的含义：一把公钥加密，一把私钥解密。公钥可以全网公开，私钥只有自己知道。

```
公钥加密 → 密文 → 私钥解密
```

类比：你有个信箱。信箱入口（公钥）所有人都可以往里投信。但只有你有信箱的钥匙（私钥）能取信。

### RSA 示例

```java
KeyPairGenerator keyGen = KeyPairGenerator.getInstance("RSA");
keyGen.initialize(2048);
KeyPair keyPair = keyGen.generateKeyPair();

PublicKey publicKey = keyPair.getPublic();
PrivateKey privateKey = keyPair.getPrivate();

// 公钥加密
Cipher cipher = Cipher.getInstance("RSA");
cipher.init(Cipher.ENCRYPT_MODE, publicKey);
byte[] encrypted = cipher.doFinal("secret message".getBytes());

// 私钥解密
cipher.init(Cipher.DECRYPT_MODE, privateKey);
byte[] decrypted = cipher.doFinal(encrypted);
```

### 非对称加密的优缺点

| 优点 | 缺点 |
|------|------|
| 公钥可公开分发，解决密钥分发问题 | 速度慢，比对称加密慢 100-1000 倍 |
| 支持数字签名（私钥签名 + 公钥验签） | 密钥很长（RSA-2048 = 256 字节） |

### 现实中的用法：混合加密

HTTPS 的做法：
1. 用 RSA 交换一个临时的 AES 密钥
2. 后续通信全部用 AES 加密（快）
3. 既有 RSA 的密钥分发优势，又有 AES 的速度优势

## 92.5 数字签名——"谁发的"比"发了什么"还重要

```
签名：用私钥对消息哈希加密 → 得到签名
验签：用公钥解密签名 → 得到哈希 → 与重新计算的哈希对比
```

```java
// 签名
Signature signature = Signature.getInstance("SHA256withRSA");
signature.initSign(privateKey);
signature.update("this is the message".getBytes());
byte[] digitalSignature = signature.sign();

// 验签
signature.initVerify(publicKey);
signature.update("this is the message".getBytes());
boolean verified = signature.verify(digitalSignature);  // true
```

## 92.6 Java KeyStore——密钥的安全存放

密钥不能写在代码里。Java KeyStore 是一种加密文件格式，用于安全存储密钥和证书：

```bash
keytool -genkeypair -alias mykey -keyalg RSA -keysize 2048 \
  -keystore keystore.jks -validity 365 -storepass your_keystore_password
```

Spring Boot 配置：

```yaml
server:
  ssl:
    key-store: classpath:keystore.jks
    key-store-password: your_keystore_password
    key-alias: mykey
```

## 92.7 小结

| 知识点 | 核心内容 |
|--------|----------|
| 编码 | 可逆无密钥（Base64、URL Encode） |
| 哈希 | 不可逆无密钥（SHA-256、BCrypt） |
| 对称加密 | 可逆同一密钥（AES），快 |
| 非对称加密 | 可逆公私钥对（RSA），慢但解决密钥分发 |
| 数字签名 | 私钥签名 + 公钥验签 |
| 密码存储 | 必须用 BCrypt/Argon2，加随机盐 |

**3. HTTPS 为什么同时用了 RSA 和 AES？只用 RSA 或只用 AES 有什么问题？**

---

**答案提示**：1→Base64 是编码（可逆、无密钥），任何人都能解码，不能保密。AES 是加密（可逆、需密钥），没有密钥无法解密，可以保密。2→只有 C 适用。MD5 和 SHA-256 没有盐，可以被彩虹表攻击，而且 SHA-256 计算太快，暴力破解成本低。AES 是加密算法（可逆），用于保密传输，不用于密码存储。3→RSA 解决了"安全交换密钥"的问题但速度慢；AES 快但无法安全分发密钥。HTTPS 用 RSA 传输一个临时 AES 密钥，之后用 AES 通信——两者结合取其长。下一章——HTTPS 与证书管理。
