## 我在做什么？

想象一场寻宝游戏：主办方在城市的各个角落藏了宝箱。第一个宝箱里有一张纸条写着"去市图书馆，找红色封面的书"，你到了图书馆，在红色书里找到第二张纸条："去中央公园，最大的橡树下"，如此反复，直到最后一张纸条写着"恭喜，宝藏在这里！"。

**单向链表**就是这样的寻宝游戏：
- 每个宝箱是一个**节点**（Node），里面装着**数据**（宝物线索）和**一张纸条**（指向下一个节点的指针）。
- 你只需要知道第一个宝箱在哪（**头指针** head），就能顺着纸条找到所有节点。
- 最后一个节点的纸条是空的（NULL），表示寻宝结束。

```
head             Node A           Node B           Node C
  │            ┌─────────┐     ┌─────────┐     ┌─────────┐
  └───────────>│ data:A  │     │ data:B  │     │ data:C  │
               │ next ───┼────>│ next ───┼────>│ next ───┼──> NULL
               └─────────┘     └─────────┘     └─────────┘
                  "纸条指向B"     "纸条指向C"     "纸条为空—终点!"
```

本章你会亲手写出一个完整的单向链表，包含创建节点、头插法、尾插法、遍历和释放内存。这是所有链表变体的基石。

## 43.1 节点的结构体——自引用的妙用

`*此术语见附录F*`

链表的核心是一个**自引用结构体**（Self-Referential Structure）：结构体内部包含一个指向同类型结构体的指针。

```c
/* 单向链表的节点定义 */
typedef struct Node {
    int            data;   /* 数据域：存储实际数据 */
    struct Node   *next;   /* 指针域：指向下一个节点 */
} Node;
```

```
         ┌──────────────────┐
         │  Node 结构体       │
         │  ┌──────┬──────┐  │
         │  │ data │ next │  │
         │  │ (int)│(指针)│  │
         │  └──────┴──────┘  │
         └──────────────────┘
```

关键点解释：

1. `struct Node *next` 中的 `struct Node` 不能写成 `Node`，因为在定义完成之前，`Node` 这个别名还不存在。
2. `next` 存储的是另一个 `Node` 的**地址**，不是另一个 `Node` 本身。正因如此，节点在内存中可以是分散的，靠指针串联。
3. 数据域 `data` 这里用 `int` 举例，实际中可以换成任何类型（`char`、`float`、结构体、甚至 `void*` 通用指针）。

为什么叫"自引用"？因为结构体 `Node` 的成员 `next` 指向的类型就是 `Node` 自身——它描述了自己如何连接下一个自己。

## 43.2 创建节点——malloc 登场

链表的节点是动态分配的，用 `malloc` 在堆上申请内存。每个节点独立存在，不会因为函数返回而消失。

```c
#include <stdlib.h>  /* malloc, free */

/*
 * create_node: 创建一个新节点
 * @value: 要存入节点的数据
 * @return: 指向新节点的指针，失败返回 NULL
 */
Node* create_node(int value) {
    Node *new_node = (Node*)malloc(sizeof(Node));
    if (new_node == NULL) {
        /* malloc 失败（内存耗尽），返回 NULL */
        return NULL;
    }
    new_node->data = value;  /* 填入数据 */
    new_node->next = NULL;   /* 新节点暂时不指向任何人 */
    return new_node;
}
```

调用 `create_node(42)` 后在内存中发生了什么：

```
堆内存:
    ┌─────────┐
    │ data: 42│
    │ next: ──┼──> NULL
    └─────────┘
       ^
       │
    new_node (指针变量，存着这个堆地址)
```

> **安全要点**：每次 `malloc` 后必须检查返回值是否为 `NULL`。如果系统内存不足，`malloc` 会返回 `NULL`，之后对 `NULL` 指针的任何操作都会导致段错误（Segmentation Fault）。

## 43.3 头插法——新节点永远插在最前面

**头插法**（Head Insertion）是最简单的插入方式：让新节点的 `next` 指向当前头节点，然后更新头指针。

```
步骤图解：

初始链表:  head --> [A] --> [B] --> NULL

步骤1: 创建新节点 [X]
         head --> [A] --> [B] --> NULL
         [X] --> NULL

步骤2: 让 [X] 的 next 指向当前头节点
         head --> [A] --> [B] --> NULL
                   ^
         [X].next ─┘

步骤3: 更新 head 指向 [X]
         head --> [X] --> [A] --> [B] --> NULL
```

```c
/*
 * insert_head: 在链表头部插入新节点
 * @head: 指向头指针的指针（为什么是二级指针？见下文）
 * @value: 要插入的数据
 * @return: 成功返回 0，失败返回 -1
 *
 * 注意：必须传 Node**，因为要修改 head 指针本身！
 */
int insert_head(Node **head, int value) {
    Node *new_node = create_node(value);
    if (new_node == NULL) {
        return -1;  /* 内存分配失败 */
    }
    new_node->next = *head;  /* 新节点的 next 指向原来的第一个节点 */
    *head = new_node;        /* 头指针指向新节点 */
    return 0;
}
```

**为什么传 `Node**`（二级指针）？**

这是链表操作中最容易困惑的地方。回忆一下C语言的参数传递：所有参数都是**按值传递**的。如果你传 `Node* head`，函数内修改 `head` 只修改了局部副本，不会影响外部的指针。

```
❌ 错误写法 —— 用一级指针：
void insert_head_wrong(Node *head, int value) {
    Node *new_node = create_node(value);
    new_node->next = head;
    head = new_node;   /* 只修改了局部副本！函数返回后无效 */
}

调用：insert_head_wrong(my_head, 42);
      —— my_head 的值没有任何变化！

✅ 正确写法 —— 用二级指针：
int insert_head(Node **head, int value) {
    Node *new_node = create_node(value);
    new_node->next = *head;   /* *head 是外部的头指针的值 */
    *head = new_node;         /* 通过解引用修改外部的头指针 */
    return 0;
}

调用：insert_head(&my_head, 42);
      —— my_head 被正确更新！
```

## 43.4 尾插法——新节点放在最末尾

**尾插法**（Tail Insertion）需要先找到链表的最后一个节点，然后把新节点挂在它后面。

```
初始链表:  head --> [A] --> [B] --> NULL

步骤1: 找到最后一个节点（遍历到 next == NULL 的节点）
         找到 [B]

步骤2: 让 [B] 的 next 指向新节点
         head --> [A] --> [B] --> [X] --> NULL
```

```c
/*
 * insert_tail: 在链表尾部插入新节点
 * @head: 指向头指针的指针
 * @value: 要插入的数据
 * @return: 成功返回 0，失败返回 -1
 */
int insert_tail(Node **head, int value) {
    Node *new_node = create_node(value);
    if (new_node == NULL) {
        return -1;
    }

    /* 情况1：链表为空，新节点就是头节点 */
    if (*head == NULL) {
        *head = new_node;
        return 0;
    }

    /* 情况2：链表不为空，遍历到最后一个节点 */
    Node *current = *head;
    while (current->next != NULL) {
        current = current->next;
    }
    /* 此时 current 指向最后一个节点 */
    current->next = new_node;
    return 0;
}
```

**头插法 vs 尾插法对比**：

| 特性 | 头插法 | 尾插法 |
|------|--------|--------|
| 时间复杂度 | O(1)，每次固定两步 | O(n)，每次遍历到末尾 |
| 数据顺序 | 逆序（最后插入的排最前） | 顺序（先插入的排最前） |
| 适用场景 | 不关心顺序时用，效率高 | 需要保持插入顺序时用 |

## 43.5 遍历链表——按图索骥

链表遍历的核心模式：用一个指针从头走到尾，每走一步打印或处理当前节点的数据。

```c
/*
 * print_list: 打印链表中的所有数据
 * @head: 链表头指针
 */
void print_list(Node *head) {
    if (head == NULL) {
        printf("链表为空。\n");
        return;
    }

    Node *current = head;
    int   index = 0;
    while (current != NULL) {
        printf("节点[%d]: %d\n", index, current->data);
        current = current->next;  /* 移动到下一个节点 */
        index++;
    }
    printf("共 %d 个节点。\n", index);
}
```

遍历过程的"慢动作"：

```
current 指向:
  第1次循环: [A] --> 打印 "节点[0]: A 的值" --> current = current->next
  第2次循环: [B] --> 打印 "节点[1]: B 的值" --> current = current->next
  第3次循环: [C] --> 打印 "节点[2]: C 的值" --> current = current->next
  第4次循环: current == NULL, 退出循环
```

**遍历模式三要素**：
1. 从 `head` 开始
2. 循环条件：`current != NULL`
3. 步进操作：`current = current->next`

## 43.6 释放链表——不释放就是内存泄漏

链表节点分散在堆上，不能只释放一个。必须**逐个遍历并释放每个节点**，并且要在释放前保存下一个节点的地址，否则就"断了线"。

```
❌ 错误做法——释放后找不到了：
free(current);
current = current->next;  /* 危险！current 已经指向被释放的内存 */

✅ 正确做法——先保存，再释放：
Node *next = current->next;  /* 先记下下一个节点的地址 */
free(current);               /* 安全释放当前节点 */
current = next;              /* 移动到下一个节点 */
```

```c
/*
 * free_list: 释放整个链表的所有节点
 * @head: 指向头指针的指针
 *
 * 释放后会将头指针设为 NULL，防止悬空指针。
 */
void free_list(Node **head) {
    Node *current = *head;
    while (current != NULL) {
        Node *next = current->next;  /* 保存下一个节点地址 */
        free(current);               /* 释放当前节点 */
        current = next;              /* 前进 */
    }
    *head = NULL;  /* 避免调用方持有悬空指针 */
}
```

## 43.7 完整代码清单

以下是一个完整的、可编译运行的单向链表程序。把它保存为 `singly_linked_list.c`，用 `gcc -o list singly_linked_list.c && ./list` 运行。

```c
/*
 * singly_linked_list.c —— 单向链表完整实现
 * 编译: gcc -std=c99 -Wall -o list singly_linked_list.c
 * 运行: ./list (Linux/Mac) 或 list.exe (Windows)
 */

#include <stdio.h>
#include <stdlib.h>

/* ==================== 节点结构体 ==================== */
typedef struct Node {
    int          data;
    struct Node *next;
} Node;

/* ==================== 函数声明 ==================== */
Node* create_node(int value);
int   insert_head(Node **head, int value);
int   insert_tail(Node **head, int value);
void  print_list(Node *head);
int   list_length(Node *head);
void  free_list(Node **head);

/* ==================== 创建节点 ==================== */
Node* create_node(int value) {
    Node *new_node = (Node*)malloc(sizeof(Node));
    if (new_node == NULL) {
        fprintf(stderr, "错误：内存分配失败！\n");
        return NULL;
    }
    new_node->data = value;
    new_node->next = NULL;
    return new_node;
}

/* ==================== 头插法 ==================== */
int insert_head(Node **head, int value) {
    Node *new_node = create_node(value);
    if (new_node == NULL) return -1;

    new_node->next = *head;
    *head = new_node;
    return 0;
}

/* ==================== 尾插法 ==================== */
int insert_tail(Node **head, int value) {
    Node *new_node = create_node(value);
    if (new_node == NULL) return -1;

    if (*head == NULL) {
        *head = new_node;
        return 0;
    }

    Node *current = *head;
    while (current->next != NULL) {
        current = current->next;
    }
    current->next = new_node;
    return 0;
}

/* ==================== 打印链表 ==================== */
void print_list(Node *head) {
    if (head == NULL) {
        printf("(空链表)\n");
        return;
    }

    Node *current = head;
    int   count   = 0;

    printf("链表内容: ");
    while (current != NULL) {
        printf("%d", current->data);
        if (current->next != NULL) {
            printf(" -> ");
        }
        current = current->next;
        count++;
    }
    printf(" -> NULL\n");
    printf("节点总数: %d\n", count);
}

/* ==================== 计算链表长度 ==================== */
int list_length(Node *head) {
    int count = 0;
    Node *current = head;
    while (current != NULL) {
        count++;
        current = current->next;
    }
    return count;
}

/* ==================== 释放链表 ==================== */
void free_list(Node **head) {
    Node *current = *head;
    while (current != NULL) {
        Node *next = current->next;
        free(current);
        current = next;
    }
    *head = NULL;
}

/* ==================== 演示主函数 ==================== */
int main(void) {
    Node *list = NULL;   /* 空链表 */

    printf("===== 演示1：头插法 =====\n");
    insert_head(&list, 10);
    insert_head(&list, 20);
    insert_head(&list, 30);
    print_list(list);   /* 预期: 30 -> 20 -> 10 -> NULL */
    printf("注意：头插法的结果是逆序的！\n\n");

    /* 清空链表，重新开始 */
    free_list(&list);
    printf("链表已清空。\n\n");

    printf("===== 演示2：尾插法 =====\n");
    insert_tail(&list, 10);
    insert_tail(&list, 20);
    insert_tail(&list, 30);
    print_list(list);   /* 预期: 10 -> 20 -> 30 -> NULL */
    printf("注意：尾插法保持了插入顺序！\n\n");

    printf("===== 演示3：混合使用 =====\n");
    insert_head(&list, 5);    /* 插在头部 */
    insert_tail(&list, 40);   /* 插在尾部 */
    print_list(list);         /* 预期: 5 -> 10 -> 20 -> 30 -> 40 -> NULL */

    printf("\n===== 演示4：空链表 =====\n");
    free_list(&list);
    print_list(list);  /* 预期: (空链表) */

    return 0;
}
```

**编译和运行**：

```bash
# 编译（开启所有警告）
gcc -std=c99 -Wall -Wextra -o list singly_linked_list.c

# 运行
./list
```

**预期输出**：

```

链表内容: 30 -> 20 -> 10 -> NULL
节点总数: 3
注意：头插法的结果是逆序的！

链表已清空。

===== 演示2：尾插法 =====
链表内容: 10 -> 20 -> 30 -> NULL
节点总数: 3
注意：尾插法保持了插入顺序！

===== 演示3：混合使用 =====
链表内容: 5 -> 10 -> 20 -> 30 -> 40 -> NULL
节点总数: 5

===== 演示4：空链表 =====
(空链表)
```

## 不对怎么办？

### 常见Bug 1：段错误（Segmentation Fault）

**现象**：程序运行到某一行直接崩溃。

**根因排查**：

```c
/* 最容易出错的模式：对 NULL 解引用 */
Node *current = *head;         /* 如果 *head 是 NULL */
while (current->next != NULL)  /* ❌ 此时 current 是 NULL，current->next 崩溃！ */
```

**修复方法**：
```c
/* 先判断 current 是否为 NULL */
Node *current = *head;
while (current != NULL && current->next != NULL) {  /* ✅ 先判空 */
    current = current->next;
}
```

### 常见Bug 2：忘记更新头指针

**现象**：插入第一个节点后，`head` 仍然是 `NULL`。

```c
/* ❌ insert_head 传了一级指针，修改无效 */
void insert_head_wrong(Node *head, int value) {  /* 不是 Node** */
    Node *new_node = create_node(value);
    new_node->next = head;
    head = new_node;  /* 只改了局部变量 */
}

/* 调用 */
Node *list = NULL;
insert_head_wrong(list, 10);
/* list 仍然是 NULL！ */
```

**修复方法**：传 `&list`，函数参数用 `Node **head`。

### 常见Bug 3：内存泄漏——只释放了头节点

**现象**：程序运行久了内存不断增长。

```c
/* ❌ 只释放了头节点，后面的节点都泄漏了 */
free(head);
head = NULL;
```

**修复方法**：必须遍历释放每个节点，如 `free_list` 所示。

### 常见Bug 4：在 `free_list` 中访问已释放内存

```c
/* ❌ 释放后用 current->next */
Node *current = head;
while (current != NULL) {
    free(current);
    current = current->next;  /* 危险！current 已被释放 */
}
```

**修复方法**：释放前保存 `next`。

### 常见Bug 5：尾插法中 `head` 为 NULL 时没有特殊处理

```c
/* ❌ 没有检查空链表 */
int insert_tail_wrong(Node **head, int value) {
    Node *new_node = create_node(value);
    Node *current = *head;
    while (current->next != NULL) {  /* 如果 *head 是 NULL，崩溃！ */
        current = current->next;
    }
    current->next = new_node;
    return 0;
}
```

**修复方法**：加入 `if (*head == NULL) { *head = new_node; return 0; }` 提前处理。

## 本章小结

| 要点 | 说明 |
|------|------|
| 节点结构体 | `data` 存放数据，`next` 指向下一个同类结构体（自引用） |
| 头插法 | O(1)，新节点插入头部，结果逆序 |
| 尾插法 | O(n)，新节点插入尾部，保持插入顺序 |
| 遍历模式 | `while (current != NULL) { ... current = current->next; }` |
| 二级指针 | 需要修改头指针时传 `Node**`，否则修改只作用于副本 |
| 内存管理 | `malloc` 后检查 NULL，`free` 前保存 next，释放后置 NULL |
| 空链表安全 | 所有操作必须先判断 `head == NULL` 的情况 |
