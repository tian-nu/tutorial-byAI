## 一、目标与完成效果

**一句话目标**：给登录、注册、发文章三个表单加上完整的校验、错误提示、防重复提交和反馈提示。

**完成后你将看到**：
- 登录表单：邮箱格式不对时输入框下方弹出红色提示，密码过短时显示强度条，提交期间按钮显示"提交中..."且不可再次点击，成功/失败弹出 toast 消息。
- 注册表单：用户名、邮箱、密码均有校验规则，空字段无法提交。
- 发文章表单：标题和内容必填，空着点提交会看到红色错误提示。

## 三、分步操作

### 步骤 1：表单校验 — 提交前检查必填与格式

**我在做什么？**
就像写字楼的前台，访客进来前要先检查：姓名填了没？手机号格式对不对？不合格就不让进。表单校验就是前台的职责。

**核心思路**：用一个 `errors` 响应式对象，key 是字段名，value 是错误信息。提交时遍历所有规则，有任何一个不通过就不发请求。

先创建一个通用的校验工具函数，放在 `src/utils/validate.js`：

```js
// src/utils/validate.js
export function validateForm(form, rules) {
  const errors = {}
  for (const [field, fieldRules] of Object.entries(rules)) {
    const value = form[field]
    for (const rule of fieldRules) {
      const error = rule(value, form)
      if (error) {
        errors[field] = error
        break  // 一个字段只显示第一条错误
      }
    }
  }
  return {
    valid: Object.keys(errors).length === 0,
    errors
  }
}

// 常用规则工厂
export const required = (msg = '此项为必填') => (val) => !val?.trim() ? msg : ''
export const isEmail = (msg = '请输入有效的邮箱地址') => (val) =>
  val && !/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(val) ? msg : ''
export const minLength = (n, msg) => (val) =>
  val && val.trim().length < n ? (msg || `至少需要 ${n} 个字符`) : ''
```

**用起来**——改造 `Login.vue` 的 `<script setup>`：

```html
<script setup>
import { ref, reactive } from 'vue'
import { validateForm, required, isEmail, minLength } from '@/utils/validate'

const form = reactive({ email: '', password: '' })
const errors = reactive({ email: '', password: '' })
const submitting = ref(false)

const rules = {
  email: [required('请输入邮箱'), isEmail('邮箱格式不正确')],
  password: [required('请输入密码'), minLength(6, '密码至少 6 位')]
}

async function handleSubmit() {
  // 第一步：校验
  const { valid, errors: newErrors } = validateForm(form, rules)
  Object.assign(errors, { email: '', password: '' })  // 先清空
  if (!valid) {
    Object.assign(errors, newErrors)  // 填入错误
    return  // 校验不通过，直接返回
  }

  // 第二步：提交（后面步骤完善）
}
</script>
```

🤔 **想多一点**：为什么先清空 `errors` 再填入？因为用户可能上次填错了邮箱，这次改对了邮箱但密码没填——如果不先清空，邮箱的错误提示会"残留"在那里，用户还以为自己邮箱仍然不对。

**我做得对不对？**
在浏览器中打开登录页，不填任何内容直接点"登录"按钮，应该看到控制台没有任何网络请求发出（因为 `return` 提前结束了）。

**不对怎么办？**
- 如果点击后仍然发了请求 → 检查 `if (!valid)` 的 `return` 是否写在了 `fetch` 之前。
- 如果 `validateForm` 报错 "xxx is not a function" → 检查 `rules` 对象的每个值是不是数组，数组里是不是函数。

### 步骤 2：错误提示 — 输入框下方显示红色消息

**我在做什么？**
校验拿到了 `errors` 对象，但用户还看不到。现在让每个输入框下面出现一条红色的错误文字，像超市收银台的警示灯：有问题就亮，没问题就灭。

**改造 `<template>`**：

```html
<template>
  <form @submit.prevent="handleSubmit">
    <div class="form-group">
      <label>邮箱</label>
      <input v-model="form.email" type="email" placeholder="请输入邮箱" />
      <span v-if="errors.email" class="error-tip">{{ errors.email }}</span>
    </div>

    <div class="form-group">
      <label>密码</label>
      <input v-model="form.password" type="password" placeholder="请输入密码" />
      <span v-if="errors.password" class="error-tip">{{ errors.password }}</span>
    </div>

    <button type="submit" :disabled="submitting">
      {{ submitting ? '提交中...' : '登录' }}
    </button>
  </form>
</template>
```

**加样式**：

```css
.error-tip {
  color: #e74c3c;
  font-size: 13px;
  margin-top: 4px;
  display: block;
}
```

❌ **错误示例**：用 `alert(errors.email)` 弹窗提示。
✅ **正确示例**：用 `v-if` 条件渲染在输入框正下方，不打断用户操作流。

🤔 **想多一点**：为什么错误提示要放在输入框下方而不是弹窗？因为用户在填表单时的视线流是从上到下的，错误信息出现在他刚填写完的位置正下方，不需要移动视线就能看到。弹窗需要用户手动关闭，打断了操作节奏。

**我做得对不对？**
- 不填邮箱，点登录 → 邮箱下方出现红色"请输入邮箱"
- 填一个格式错误的邮箱如 `abc`，点登录 → 出现"邮箱格式不正确"
- 填正确邮箱和密码 → 无错误提示

**不对怎么办？**
- 错误提示不出现 → 检查 `<span>` 的 `v-if="errors.email"`，以及 `errors` 是否用 `reactive` 声明。
- 错误提示出现了但不消失 → 检查是否在 `handleSubmit` 开头先清空了 `errors`。

### 步骤 3：提交中状态 — 按钮禁用防重复

**我在做什么？**
用户在等服务器响应时，可能因为心急连点好几下"登录"按钮，结果发了 3 次请求。这就像电梯关门按钮——按一次就好，按再多也不会让门关得更快，但电梯至少不会因为多按几下就出故障。我们的表单也需要这种"防抖"保护。

**实现**：用 `submitting` 这个 `ref` 变量，提交前设为 `true`，请求结束（无论成功失败）设为 `false`。

```js
async function handleSubmit() {
  const { valid, errors: newErrors } = validateForm(form, rules)
  Object.assign(errors, { email: '', password: '' })
  if (!valid) {
    Object.assign(errors, newErrors)
    return
  }

  submitting.value = true  // 🔒 锁上
  try {
    const res = await fetch('/api/login', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(form)
    })
    if (!res.ok) throw new Error('登录失败')
    // 成功处理...
  } catch (e) {
    // 失败处理...
  } finally {
    submitting.value = false  // 🔓 解锁
  }
}
```

注意 `finally`：不管成功还是失败，都要解锁按钮，否则按钮会永远卡在"提交中..."。

❌ **错误示例**：只在 `.then()` 里设 `submitting = false`，忘了 `.catch()` 里也设。
✅ **正确示例**：用 `try/catch/finally`，`finally` 中解锁。

**我做得对不对？**
打开浏览器开发者工具 Network 面板，将网络设为 "Slow 3G"，然后快速连点 3 次登录按钮。应只发出 1 次请求。

**不对怎么办？**
- 仍然能连点 → 检查 `<button>` 是否加了 `:disabled="submitting"`。
- 按钮一直灰色不恢复 → 检查 `finally` 块是否存在。

### 步骤 4：成功/失败反馈 — Toast 提示

**我在做什么？**
提交成功了，用户需要知道"成了！"；提交失败了，用户需要知道"为什么"。这就需要一个**Toast**（像烤面包机弹出面包片一样，从屏幕某处弹出一个小消息，几秒后自动消失）。

**创建 Toast 组件** `src/components/Toast.vue`：

```html
<template>
  <Teleport to="body">
    <div v-if="visible" :class="['toast', type]">
      {{ message }}
    </div>
  </Teleport>
</template>

<script setup>
import { ref, watch } from 'vue'

const props = defineProps({
  message: String,
  type: { type: String, default: 'success' },  // success | error
  duration: { type: Number, default: 3000 },
  show: Boolean
})

const emit = defineEmits(['close'])
const visible = ref(false)

watch(() => props.show, (val) => {
  if (val) {
    visible.value = true
    setTimeout(() => {
      visible.value = false
      emit('close')
    }, props.duration)
  }
})
</script>

<style scoped>
.toast {
  position: fixed;
  top: 20px;
  left: 50%;
  transform: translateX(-50%);
  padding: 12px 24px;
  border-radius: 8px;
  color: #fff;
  font-size: 14px;
  z-index: 9999;
  animation: slideDown 0.3s ease;
}
.toast.success { background: #27ae60; }
.toast.error { background: #e74c3c; }
@keyframes slideDown {
  from { opacity: 0; transform: translateX(-50%) translateY(-20px); }
  to   { opacity: 1; transform: translateX(-50%) translateY(0); }
}
</style>
```

**在 Login.vue 中使用**：

```html
<template>
  <!-- ... 表单 ... -->
  <Toast :show="toast.show" :message="toast.message" :type="toast.type" @close="toast.show = false" />
</template>

<script setup>
// ...
const toast = reactive({ show: false, message: '', type: 'success' })

async function handleSubmit() {
  // ... 校验 ...
  submitting.value = true
  try {
    const res = await fetch('/api/login', { /* ... */ })
    if (!res.ok) throw new Error('登录失败')
    toast.show = true
    toast.message = '登录成功！正在跳转...'
    toast.type = 'success'
  } catch (e) {
    toast.show = true
    toast.message = e.message || '网络错误，请稍后重试'
    toast.type = 'error'
  } finally {
    submitting.value = false
  }
}
</script>
```

🤔 **想多一点**：为什么要用 `<Teleport to="body">`？因为如果 Toast 放在组件内部，可能会被父元素的 `overflow: hidden` 或 `z-index` 影响，导致弹不出来或被遮挡。Teleport 把 DOM 节点"传送"到 `<body>` 下，确保它永远在最顶层。

**我做得对不对？**
- 输入正确账号密码 → 顶部滑入绿色提示"登录成功"
- 输入错误密码 → 顶部滑入红色提示
- 等 3 秒自动消失

**不对怎么办？**
- Toast 不出现 → 检查 `<Teleport to="body">` 的 `to` 值是不是 `"body"`（字符串），不是没加引号的 `body`。
- Toast 出现但不消失 → 检查 `setTimeout` 的回调是否正确设置了 `visible.value = false`。

### 步骤 5：密码强度提示（注册表单专属）

**我在做什么？**
用户设密码时，给他实时反馈密码够不够强。就像健身房的体测仪，输入一个密码立刻告诉你："太弱了（全是数字）→ 一般（加了字母）→ 强（大小写+数字+符号）"。

**创建密码强度计算函数**，追加到 `src/utils/validate.js`：

```js
export function passwordStrength(password) {
  if (!password) return { level: 0, label: '', color: '' }
  let score = 0
  if (password.length >= 6) score++
  if (password.length >= 10) score++
  if (/[a-z]/.test(password) && /[A-Z]/.test(password)) score++
  if (/\d/.test(password)) score++
  if (/[^a-zA-Z0-9]/.test(password)) score++

  if (score <= 1) return { level: 1, label: '弱', color: '#e74c3c' }
  if (score <= 3) return { level: 2, label: '中等', color: '#f39c12' }
  return { level: 3, label: '强', color: '#27ae60' }
}
```

**在 Register.vue 中使用**：

```html
<template>
  <div class="form-group">
    <label>密码</label>
    <input v-model="form.password" type="password" />
    <!-- 密码强度条 -->
    <div class="strength-bar" v-if="form.password">
      <div class="strength-fill" :style="{ width: strength.level * 33 + '%', background: strength.color }"></div>
      <span :style="{ color: strength.color }">密码强度：{{ strength.label }}</span>
    </div>
    <span v-if="errors.password" class="error-tip">{{ errors.password }}</span>
  </div>
</template>

<script setup>
import { computed } from 'vue'
import { passwordStrength } from '@/utils/validate'

const strength = computed(() => passwordStrength(form.password))
</script>

<style scoped>
.strength-bar {
  margin-top: 6px;
  height: 6px;
  background: #eee;
  border-radius: 3px;
  overflow: hidden;
}
.strength-fill {
  height: 100%;
  transition: width 0.3s ease;
}
</style>
```

❌ **错误示例**：只在提交时检查密码强度，用户填完才发现太弱要重填。
✅ **正确示例**：用 `computed` 实时响应 `form.password` 变化。

**我做得对不对？**
在注册页密码框依次输入：
- `123` → 显示"弱"，红色
- `abc123` → 显示"中等"，橙色
- `MyP@ssw0rd` → 显示"强"，绿色

## 六、小结

| 概念 | 比喻 | 关键代码 |
|------|------|----------|
| **表单校验 (form validation)** | 写字楼前台，不合格不让进 | `validateForm(form, rules)` |
| **防重复提交 (disabled)** | 电梯关门按钮，按一次就好 | `submitting = true → false`（在 `finally` 中） |
| **Toast 提示** | 烤面包机弹出的面包片 | `<Teleport to="body">` + 自动消失 |
| **密码强度** | 健身房的体测仪 | `computed(() => passwordStrength(...))` |

---

## 七、术语附录

| 术语 | 解释 |
|------|------|
| **表单校验 (Form Validation)** | 在数据提交到服务器之前，先在浏览器端检查数据是否符合规则。就像快递员收件前先检查包裹有没有封好。 |
| **disabled** | HTML 属性，为 `true` 时按钮变灰且不可点击。注意：`<button disabled>` 表示禁用，`<button :disabled="false">` 表示不禁用。 |
| **防重复提交** | 在请求进行中时锁定提交按钮，防止用户多次点击导致重复请求。通常结合 `disabled` 和一个布尔状态变量使用。 |
| **Toast** | 一种轻量级的消息提示，从屏幕边缘弹出，短暂停留后自动消失。不打断用户操作，比 `alert` 弹窗友好得多。 |
| **Teleport** | Vue 3 内置组件，将 DOM 节点"传送"到指定位置（如 `<body>`）。解决弹窗、Toast 被父元素 CSS 遮挡的问题。 |

## 八、已知坑点与禁止事项

- **禁止在 `.then()` 和 `.catch()` 里分别写 `submitting = false`**：容易遗漏，统一用 `finally`。
- **禁止在组件内直接操作 `document.body.appendChild`**：用 `<Teleport>` 代替，这是 Vue 的推荐方式。
- **校验函数必须纯函数**：不要在校验函数里访问 DOM 或修改外部状态。
- **Toast 的 `duration` 不要太短**：3000ms（3秒）是经验值，太短用户来不及看，太长阻塞操作。
