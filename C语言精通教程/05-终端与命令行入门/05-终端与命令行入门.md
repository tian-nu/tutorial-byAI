## 一、终端是什么？

### 1.1 回到上古时代

在鼠标和图形界面发明之前，人们怎么操作电脑？

答案是：**打字**。

你敲一行命令，电脑执行，把结果打印在屏幕上——然后你再敲下一行。这就是 *终端* *此术语见附录F* 的原始形态。

```text
1970 年代的终端：

+----------------------------------+
| $ ls                             |  <-- 你敲的命令
| hello.c  world.c  README.txt     |  <-- 电脑的回答
| $ gcc hello.c                    |  <-- 你敲的下一条命令
| $ ./a.out                        |
| Hello, World!                    |  <-- 程序的输出
| $                                |  <-- 等待你的下一条命令
+----------------------------------+
```

今天你打开终端（Terminal），看到的仍然是这个模式——一个等待输入命令的黑框框。

### 1.2 GUI vs CLI

操作电脑有两种方式：

| 方式 | 全称 | 比喻 |
|------|------|------|
| GUI | Graphical User Interface | 像逛商场——看到什么点什么 |
| CLI | Command Line Interface | 像打电话订餐——用精确的语言描述你要什么 |

*GUI* *此术语见附录F* 是图形界面——你平时用的 Windows 桌面、macOS 桌面，就是 GUI。用鼠标点、拖、双击。

*CLI* *此术语见附录F* 是命令行界面——终端就是 CLI 的入口。用键盘敲命令。

```text
同一个操作，两种方式：

GUI 方式：                          CLI 方式：
+--------------------------------+  +----------------------+
| 打开"文件管理器"                |  | $ ls                 |
| 双击 C 盘                      |  | $ cd /home/user      |
| 双击 Users 文件夹               |  | $ ls                 |
| 双击你的用户名                  |  | $ cd documents       |
| 双击 Documents 文件夹           |  | $ ls                 |
| 找到一个叫 hello.c 的文件       |  | hello.c              |
+--------------------------------+  +----------------------+
```

### 1.3 为什么学 C 语言必须用终端？

你可能会想："我能不能用 VS Code 写代码，点一下运行按钮就完事？"

可以，但迟早要回到终端。原因：

1. **C 语言标准工具链就是命令行**：`gcc`、`make`、`gdb`（调试器）都是命令行工具。
2. **理解编译过程**：IDE 的"运行"按钮背后做了什么？你不知道。用终端，你一步一步看到预处理、编译、链接。
3. **服务器没有 GUI**：如果你将来在服务器上写 C 程序，服务器只有命令行。
4. **精准控制**：编译器有几十个选项（警告级别、优化级别、目标平台），GUI 不可能全部暴露。

### 1.4 终端、Shell、命令行——分清三个概念

这三个词常被混用，但含义不同：

```
你                   终端窗口             Shell
|                      |                    |
| 敲键盘               | 显示文字           | 理解你的命令
| 输入 "ls"            | 传给 Shell         | 收到 "ls"
|                      |                    | 找到 ls 程序
|                      |                    | 执行 ls
|                      | 显示结果           | 返回结果
| 看到文件列表         |                    |
```

- **终端（Terminal）**：那个黑框框窗口，负责显示和接收输入。
- **Shell** *此术语见附录F*：运行在终端里的程序，负责理解你的命令（是"命令解释器"）。常见的 Shell 有 Bash、Zsh、PowerShell。
- **命令行（Command Line）**：你输入的那一行文字。

日常中，说"打开终端"就行，这三个概念在里面自然工作。

## 二、基础命令速查

下面这些命令是你学 C 语言期间每天都会用到的。建议打开终端，每读一条就敲一遍。

### 2.1 我在哪？—— pwd

```bash
$ pwd
/home/student
```

`pwd` = **P**rint **W**orking **D**irectory（打印当前工作目录）。

这是你进入终端后应该敲的第一个命令——确认自己在哪里。

```text
文件系统就像一栋大楼，你站在某个楼层某个房间。
pwd 告诉你："你现在在 3 楼 302 室。"
```

### 2.2 周围有什么？—— ls

```bash
$ ls
Desktop  Documents  Downloads  hello.c  world.c
```

`ls` = **l**i**s**t（列出当前目录下的文件和子目录）。

```bash
$ ls -l          # 详细列表（权限、大小、修改时间）
$ ls -a          # 显示隐藏文件（以 . 开头的文件）
$ ls -la         # 以上两者结合
```

### 2.3 换个地方—— cd

```bash
$ cd Documents        # 进入 Documents 目录
$ cd ..               # 返回上一级
$ cd ~                # 回到你的"家目录"（home directory）
$ cd /                # 去根目录
$ cd -                # 回到刚才待的目录
```

`cd` = **c**hange **d**irectory（切换目录）。

```text
cd 就像在大楼里走动：
  cd Documents   --> 走进 "Documents" 房间
  cd ..          --> 回到走廊（上一级）
  cd ~           --> 回你的办公室（家目录）
  cd /           --> 去大楼门口（根目录）
```

### 2.4 创建目录—— mkdir

```bash
$ mkdir projects        # 创建一个叫 projects 的目录
$ mkdir -p a/b/c        # 创建嵌套目录（a 里面建 b，b 里面建 c）
```

`mkdir` = **m**a**k**e **dir**ectory（创建目录）。

### 2.5 删除文件—— rm

```bash
$ rm old_file.txt       # 删除文件
$ rm -r old_folder/     # 删除目录及其内容（递归删除）
$ rm -f stubborn.txt    # 强制删除（不提示确认）
```

**警告**：CLI 没有回收站。`rm` 删除的文件是真的没了，不可恢复。删除前再三确认。

```text
GUI 删除文件：放到回收站，可以反悔
CLI 删除文件：直接粉碎，没有后悔药
```

### 2.6 查看文件内容—— cat

```bash
$ cat hello.c           # 在终端中显示 hello.c 的全部内容
$ cat file1.txt file2.txt  # 连接显示多个文件
```

`cat` = con**cat**enate（连接并显示）。名字虽然叫"连接"，但最常用的场景是"快速看一眼文件内容"。

### 2.7 其他常用命令

```bash
$ cp source.txt dest.txt    # 复制文件
$ mv old.txt new.txt        # 移动/重命名文件
$ touch newfile.txt         # 创建一个空文件
$ clear                     # 清屏（或按 Ctrl+L）
$ man ls                    # 查看 ls 命令的手册（manual）
$ ls --help                 # 查看 ls 命令的快速帮助
```

## 三、编译器是什么？

### 3.1 编译器：把"人话"翻译成"机器话"

在第 01 章你知道了编译器的概念。现在我们从实际操作的角度再理解一次。

*编译器* 就是一个程序，输入是 C 源代码，输出是可执行文件。

```text
  hello.c                   hello (或 hello.exe)
+-----------+             +-------------------+
| #include  |             | 机器码：           |
| int main()|  编译器     | 01010110...       |
| {         | ==========> | 11010010...       |
|   ...     |   gcc       | 00101001...       |
| }         |             | ...               |
+-----------+             +-------------------+
   源代码                      可执行文件
  (人类能读)                  (机器能读)
```

### 3.2 GCC：最流行的 C 编译器

**GCC**（GNU Compiler Collection）是世界上最流行的 C 编译器。它是免费的、开源的，在 Linux、macOS、Windows 上都能用。

```bash
# 检查 gcc 是否已安装
$ gcc --version
gcc (Ubuntu 11.4.0-1ubuntu1~22.04) 11.4.0
# 如果你看到类似输出，说明 gcc 已安装
```

如果没安装：

- **Linux (Ubuntu/Debian)**：`sudo apt install gcc`
- **macOS**：`xcode-select --install`（会安装 Command Line Tools，包含 gcc）
- **Windows**：推荐安装 MinGW-w64 或使用 WSL（Windows Subsystem for Linux）

## 四、第一个 C 程序：从源码到运行

### 4.1 创建源文件

用你喜欢的编辑器（VS Code、Vim、记事本等）创建一个文件 `hello.c`：

```c
#include <stdio.h>

int main() {
    printf("Hello, World!\n");
    return 0;
}
```

### 4.2 编译

```bash
$ gcc hello.c
```

如果一切顺利，这条命令不会输出任何东西——**没有消息就是好消息**。

编译后，当前目录会多出一个文件：

- Linux/macOS：`a.out`
- Windows：`a.exe`

`a.out` 是默认的输出文件名（a = assembler output，历史原因）。

### 4.3 运行

```bash
# Linux/macOS
$ ./a.out
Hello, World!

# Windows
$ a.exe
Hello, World!
```

为什么要加 `./`？因为操作系统只会在"系统路径"中找可执行文件，当前目录通常不在系统路径里。`./a.out` 的意思是"在当前目录下找 a.out 并运行"。

### 4.4 指定输出文件名

```bash
$ gcc hello.c -o hello        # 输出文件命名为 hello
$ ./hello
Hello, World!
```

`-o` 选项指定输出文件名。这是你以后每天都会用的选项。

### 4.5 开启警告

```bash
$ gcc hello.c -o hello -Wall -Wextra
```

- `-Wall`：开启常见的警告（W = warning, all = 全部）。
- `-Wextra`：开启额外的警告。

警告不会阻止编译，但会告诉你代码中可能有问题。**永远不要忽略警告**——C 语言中，今天的警告就是明天的 bug。

```text
编译选项速查：

gcc hello.c                 最简编译，输出 a.out
gcc hello.c -o hello        指定输出名为 hello
gcc hello.c -Wall           开启常见警告
gcc hello.c -g              包含调试信息（给 gdb 用）
gcc hello.c -O2             开启优化（O = optimization, 2 = 级别）
gcc hello.c -std=c11        指定 C 语言标准版本
```

### 4.6 完整编译流程回顾

还记得第 01 章讲的四步吗？你可以让 gcc 分步执行：

```bash
# 只做预处理
$ gcc -E hello.c -o hello.i

# 只做编译（C -> 汇编）
$ gcc -S hello.c -o hello.s

# 只做汇编（汇编 -> 目标文件）
$ gcc -c hello.c -o hello.o

# 链接（目标文件 -> 可执行文件）
$ gcc hello.o -o hello
```

## 五、终端操作技巧

### 5.1 快捷键

| 快捷键 | 作用 |
|--------|------|
| `Ctrl+C` | 终止当前正在运行的程序 |
| `Ctrl+D` | 发送"输入结束"信号（常用于退出交互式程序） |
| `Ctrl+L` | 清屏（和 `clear` 命令一样） |
| `Tab` | 自动补全文件名或命令 |
| `↑` / `↓` | 翻阅历史命令 |
| `Ctrl+R` | 搜索历史命令 |

### 5.2 路径提示

```bash
# 当前目录：.
$ ls .            # 列出当前目录

# 上级目录：..
$ cd ..           # 回到上一级

# 家目录：~
$ cd ~            # 回到你的家目录

# 通配符：*
$ ls *.c          # 列出所有 .c 结尾的文件
$ rm *.o          # 删除所有 .o 结尾的文件
```

### 5.3 遇到问题怎么办？

```bash
# 不确定某个命令怎么用？
$ man 命令名       # 查看完整手册
$ 命令名 --help    # 查看快速帮助

# 终端卡住了？
# 按 Ctrl+C 终止当前程序
# 按 Ctrl+D 退出当前 Shell 会话

# 不知道自己在哪？
$ pwd            # 永远先确认位置
```

## 我做得对不对？

验证方法——完成以下操作，确认全部成功：

1. 打开终端，执行 `pwd`，看到了当前目录路径。
2. 执行 `ls`，看到了当前目录下的文件列表。
3. 创建一个 `hello.c` 文件，内容为标准的 Hello World 程序。
4. 执行 `gcc hello.c -o hello`，没有报错。
5. 执行 `./hello`（或 `hello.exe`），看到了 `Hello, World!`。
6. 执行 `cat hello.c`，看到了你写的源代码。

如果以上 6 步全部成功，恭喜你——你已经具备了学习 C 语言所需的所有终端技能。

## 不对怎么办？

**常见困惑 1**："终端打开后，光标一直在闪，但我不确定该输入什么。"

解答：这是正常的。终端等你输入命令。你可以从 `pwd` 开始——这是最安全的命令，什么都不会改变，只是告诉你"你在哪"。然后再试试 `ls` 看看周围有什么。

**常见困惑 2**："我输入 `gcc hello.c` 后报错 'gcc: command not found'。"

解答：说明 gcc 还没安装。根据你的操作系统，按本章 3.2 节的安装指引操作。如果已经安装了但还是报这个错，可能是 gcc 不在系统路径中——搜索"gcc not found [你的操作系统]"获取解决方案。

**常见困惑 3**："我输入 `./a.out` 后报错 'Permission denied'。"

解答：给文件添加执行权限：
```bash
chmod +x a.out
./a.out
```
Windows 上通常不会遇到这个问题，因为 .exe 文件默认有执行权限。

**常见困惑 4**："我编译通过了，但运行时什么也没输出。"

解答：检查你的 `printf` 语句：
- 字符串后面有没有 `\n`？没有 `\n` 的话，输出可能被缓冲在内存中，程序退出前没来得及显示。加上 `\n` 试试。
- `return 0;` 前面有没有分号？缺分号会导致编译错误，但如果你看到的是空白输出，可能你编译的是旧版本的文件——重新保存并编译。

**常见困惑 5**："终端里怎么粘贴？Ctrl+V 不行。"

解答：终端里粘贴的快捷键通常是 `Ctrl+Shift+V`（Linux）或 `Cmd+V`（macOS 终端）。Windows 终端中，右键点击通常会粘贴。不同终端模拟器可能不同，但一般不会用 `Ctrl+V`（因为 `Ctrl+V` 在终端中有特殊含义）。
