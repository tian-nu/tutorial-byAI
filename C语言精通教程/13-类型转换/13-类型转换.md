## 我在做什么？

前几章你学了各种数据类型：`int`、`double`、`char`……它们各有各的大小和规则。但当你把 `int` 和 `double` 放在一起运算时，会发生什么？当你把一个 `long long` 塞进一个 `char` 里，又会发生什么？

答案是：**类型转换**。C语言在两种情况下会自动转换类型——你也可能手动要求转换。理解转换规则，是写出正确程序的关键。

这一章，你会学到：
1. `*隐式转换*（此术语见附录F）`：编译器自动帮你做的转换。
2. `*整型提升*（此术语见附录F）`：一个微妙的自动转换规则。
3. 显式转换（强制转换）：你主动告诉编译器"把这个类型变成那个类型"。
4. 转换中可能出现的各种陷阱。

## 13.1 隐式转换：编译器自动做的事

当你把两个不同类型的值放在一起运算时，编译器会自动把"小的"类型转换成"大的"类型，保证运算在同一个"尺寸"上进行。

### 13.1.1 转换阶梯

如果把C语言的数据类型按"大小"排序，就像一条楼梯：

```
转换阶梯（从低到高）
─────────────────────────
  long double      (最高)
  double
  float
  unsigned long long
  long long
  unsigned long
  long
  unsigned int
  int               (最低)
```

**规则**：当两个操作数类型不同时，编译器把"较低"的类型提升到"较高"的类型，然后在那个较高的类型上做运算。

**比喻**：你去五金店买材料，一个零件尺寸是厘米（int），另一个是毫米（double精度）。店员不会用两种尺子各量各的——他把厘米换算成毫米，统一用毫米尺子量。这就是隐式转换。

### 13.1.2 隐式转换示例

```c
#include <stdio.h>

int main(void)
{
    int a = 5;
    double b = 2.5;

    /* a 被隐式转换为 double，然后做 double 加法 */
    double result = a + b;   /* 5.0 + 2.5 = 7.5 */

    printf("%f\n", result);   /* 输出 7.500000 */

    /* 反过来：double 赋值给 int，小数部分被截断 */
    int truncated = b;        /* 2.5 被截断为 2 */
    printf("%d\n", truncated); /* 输出 2 */

    return 0;
}
```

### 13.1.3 赋值时的隐式转换

当把一种类型的值赋给另一种类型的变量时，也会发生隐式转换：

```c
double pi = 3.14159;
int x = pi;        /* double → int：小数部分丢弃，x = 3 */

char c = 300;      /* int → char：300超出了char范围(-128~127) */
                   /* 实际值取决于实现，通常为 300 - 256 = 44 */
                   /* 编译器可能给出警告 */
```

**特别注意**：把大类型赋给小类型，可能"溢出"或"截断"。编译器通常只会给你一个警告，不会报错。这是C语言"信任程序员"理念的体现——也是危险的来源。

## 13.2 整型提升

### 13.2.1 什么是整型提升？

C语言有一个微妙的规则：在表达式中，所有比 `int` 小的整数类型（`char`、`short`）参与运算时，会先被自动提升为 `int`（如果 `int` 能装下该类型的所有值）或 `unsigned int`。

这就是 `*整型提升*（此术语见附录F）`。

```c
char a = 'A';    /* char 类型，占 1 字节 */
char b = 'B';

/* a + b 运算时，a 和 b 都被提升为 int，然后做 int 加法 */
int result = a + b;   /* 65 + 66 = 131 */
```

**为什么有这条规则？** 因为CPU的寄存器宽度通常是32位（int大小），在寄存器里做32位运算比做8位运算更快，也更自然。

### 13.2.2 整型提升的陷阱

```c
#include <stdio.h>

int main(void)
{
    char a = 100;
    char b = 100;
    char c = a + b;   /* a 和 b 先提升为 int 相加得 200，
                         然后 200 被截断放进 char（-128~127） */
    printf("%d\n", c); /* 输出可能是 -56（200 超出了 char 范围） */

    return 0;
}
```

**比喻**：你在一个小本子（char）上写100，另一个小本子上也写100。你把两个本子上的数字加起来（整型提升，在大草稿纸（int）上算），得到200。然后你想把200抄回小本子——但小本子一页只能写-128到127。200抄不进去，你得撕掉一截，结果变成了-56。

## 13.3 显式转换（强制类型转换）

当隐式转换不够用，或者你希望明确表达意图时，可以手动进行**强制类型转换**（cast）。

### 13.3.1 语法

```c
(目标类型) 表达式
```

```c
int a = 5, b = 2;
double result;

result = a / b;              /* 整数除法：result = 2.0 */
result = (double)a / b;      /* a 先转为 double：result = 2.5 */
result = a / (double)b;      /* b 先转为 double：result = 2.5 */
result = (double)(a / b);    /* 先做整数除法，再转 double：result = 2.0 */
```

注意最后一行：`(double)(a / b)` 先计算 `a / b`（整数除法，得2），再转 `double` 得 `2.0`。括号的位置至关重要。

### 13.3.2 强制转换的常见用途

**用途1：避免整数除法截断**

```c
int total = 253;
int count = 10;
double avg = (double)total / count;   /* 25.3 而不是 25.0 */
```

**用途2：消除编译器警告**

```c
int x = 100;
long long y = (long long)x;   /* 虽然隐式转换也行，但显式更清晰 */
```

**用途3：截断浮点数**

```c
double pi = 3.14159;
int integer_part = (int)pi;   /* 3，丢弃小数部分（不是四舍五入） */
```

## 13.4 常见陷阱

### 陷阱1：整数除法后再转换（已经晚了）

```c
/* ❌ 错误示例 */
double result = (double)(3 / 2);   /* 先算 3/2=1，再转 double 得 1.0 */

/* ✅ 正确示例 */
double result = (double)3 / 2;     /* 3.0 / 2 = 1.5 */
```

### 陷阱2：有符号和无符号混用

这是第9章提过的陷阱，再强调一次：

```c
#include <stdio.h>

int main(void)
{
    unsigned int a = 10;
    int b = -20;

    printf("%d\n", a + b);   /* 你可能期望 -10，但实际输出一个巨大的正数 */
    /* 因为 b 被隐式转换为 unsigned int，-20 变成了一个接近 43 亿的数 */

    return 0;
}
```

**规则**：当 `unsigned int` 和 `int` 一起运算时，`int` 被转换为 `unsigned int`。这是一个非常危险的隐式转换。

### 陷阱3：大类型赋给小类型

```c
/* ❌ 危险 */
int big = 1000000;
short small = big;   /* 1000000 超出了 short 的范围，截断后值不可预测 */
/* 编译器通常只给警告，不报错 */

/* ✅ 安全：检查范围 */
if (big >= SHRT_MIN && big <= SHRT_MAX) {
    short small = (short)big;
} else {
    printf("值超出范围，无法转换\n");
}
```

### 陷阱4：浮点数转整数不是四舍五入

```c
double a = 3.7;
int b = (int)a;   /* b = 3，不是 4！ */

/* 如果需要四舍五入 */
int c = (int)(a + 0.5);   /* 3.7 + 0.5 = 4.2 → 截断得 4 */

/* 或者用 math.h 的 round 函数 */
int d = (int)round(a);    /* 4 */
```

### 陷阱5：`sizeof` 返回值的类型

```c
/* sizeof 返回 size_t 类型（无符号整数），与有符号数比较时注意 */
int len = 5;
if (len > sizeof(int)) {   /* 有符号 int 与无符号 size_t 比较，可能意外 */
    /* ... */
}
```

## 13.5 完整示例：类型转换大全

```c
#include <stdio.h>
#include <limits.h>
#include <math.h>

int main(void)
{
    printf("=== 隐式转换 ===\n");
    int a = 10;
    double b = 3.0;
    printf("int + double: %d + %f = %f\n", a, b, a + b);
    /* a 被隐式转换为 double */

    printf("\n=== 整型提升 ===\n");
    char c1 = 100;
    char c2 = 100;
    int sum = c1 + c2;   /* c1 和 c2 先提升为 int 再相加 */
    printf("char + char = int: %d + %d = %d\n", c1, c2, sum);
    printf("(sizeof char+char): %zu\n", sizeof(c1 + c2));
    /* sizeof(char+char) 是 sizeof(int)，不是 sizeof(char) */

    printf("\n=== 强制转换 ===\n");
    int total = 100;
    int count = 3;
    double avg1 = total / count;             /* 整数除法 */
    double avg2 = (double)total / count;     /* 浮点除法 */
    printf("整数除法: %d/%d = %f\n", total, count, avg1);
    printf("浮点除法: %d/%d = %f\n", total, count, avg2);

    printf("\n=== 截断（不是四舍五入） ===\n");
    double pi = 3.14159;
    printf("(int)%.5f = %d\n", pi, (int)pi);
    double d = 3.7;
    printf("(int)%.1f = %d (截断)\n", d, (int)d);
    printf("(int)(%.1f + 0.5) = %d (四舍五入)\n", d, (int)(d + 0.5));

    printf("\n=== 有符号与无符号混用 ===\n");
    unsigned int u = 10;
    int s = -5;
    printf("unsigned + signed: %u + %d = %u\n", u, s, u + s);
    /* 输出一个巨大的正数 */

    return 0;
}
```

## 13.6 我在做什么？做得对不对？不对怎么办？

### 我在做什么？

我在学习C语言中类型之间的转换规则：编译器自动做的隐式转换（包括整型提升），以及我手动做的强制类型转换。我还理解了转换中可能出现的截断、溢出、有符号/无符号混用等陷阱。

### 我做得对不对？

- 你能解释 `3 / 2` 和 `(double)3 / 2` 的区别 → 强制转换过关。
- 你知道 `char` 参与运算时会被提升为 `int` → 整型提升过关。
- 你能说出 `(int)3.7` 的结果是 `3` 而不是 `4` → 截断理解过关。
- 你知道 `unsigned int` 和 `int` 混用时的危险 → 安全意识过关。

### 不对怎么办？

| 症状 | 第一步检查 |
|------|-----------|
| 除法结果没有小数部分 | 是否在整数除法之后才转换？ |
| 浮点转整数结果少1 | 是否期待四舍五入？`(int)` 是截断 |
| 大数变成奇怪的负数 | 是否赋给了太小类型导致溢出？ |
| 有符号和无符号运算结果异常 | 是否发生了隐式转换？ |
| `char` 运算结果超出预期 | 是否忘了整型提升？ |

## 本章小结

| 知识点 | 说明 |
|--------|------|
| 隐式转换 | 不同类型运算时，编译器自动向"高"类型转换 |
| 整型提升 | `char`、`short` 参与运算时先提升为 `int` |
| 强制转换 `(类型)` | 程序员手动指定类型转换 |
| 转换阶梯 | long double > double > float > unsigned long long > ... > int |
| 截断 | 浮点转整数时丢弃小数部分（不是四舍五入） |
| 有符号/无符号混用 | 有符号会被转换为无符号，可能导致意外结果 |
| 大转小 | 可能溢出或截断，编译器通常只给警告 |

## 练习题

1. **隐式转换**：声明 `int a = 7; double b = 2.0;`，计算 `a / b` 和 `b / a`，输出结果。解释为什么不需要手动转换也能得到浮点结果。

2. **强制转换时机**：计算 `(double)(5 / 2)` 和 `(double)5 / 2`，比较结果。解释为什么括号位置不同导致结果不同。

3. **整型提升实验**：声明 `char a = 50, b = 60;`，用 `sizeof(a + b)` 输出 `a + b` 的类型大小。解释为什么是 `sizeof(int)` 而不是 `sizeof(char)`。

4. **有符号/无符号陷阱**：声明 `unsigned int u = 5; int s = -10;`，比较 `u > s` 的结果。输出 `u + s` 的值。解释结果。

5. **四舍五入实现**：写一个函数，输入一个 `double` 值，返回它四舍五入后的整数。不要用 `math.h` 的 `round` 函数，自己用类型转换实现。
