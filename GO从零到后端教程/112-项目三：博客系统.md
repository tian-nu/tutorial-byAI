# 第112章 · 项目三：博客系统

> "Todo是单表的CRUD，认证系统是单表加安全，现在是时候面对真正的多表关联了。博客系统有文章、分类、标签、评论——它们互相缠绕，就像一团毛线。别担心，我们会一根一根地理清楚。本章学完后，你就具备了搭建任何'内容型'应用的能力——CMS、论坛、文档系统，都是在博客系统的基础上衍生出来的。"

## 112.1 需求分析

### 这个项目要做什么

一个多用户博客系统，用户可以：
- 写文章（支持Markdown格式）
- 给文章分类（一篇文章属于一个分类，就像一本书只能放一个书架）
- 给文章打标签（一篇文章可以有多个标签，就像一篇文章可以有多个关键词）
- 评论他人的文章
- 按分类和标签浏览文章

|------|------|------|------|------|
| 1 | 用户 | 注册/登录 | — | 复用第111章 |

| 7 | 分类 | 创建分类 | POST | /api/categories |

### 不做什么
- 不做文章草稿/发布状态切换（只做已发布）
- 不做点赞/收藏功能
- 不做全文搜索（用MySQL LIKE）
- 不做Markdown实时预览（那是前端的事）

## 112.2 数据库设计：五张表的爱恨情仇

五张表之间的关系就像一个小社会：

```
   categories ──┐
                 ├──< articles（文章属于分类，作者属于用户）
    users ───────┘
       │
       ├──< comments（评论属于用户和文章）
       │
    articles ──< article_tags >── tags（多对多，需要中间表）
       │
       └──< comments
```

### 建表SQL

```sql
CREATE DATABASE IF NOT EXISTS blog_db
    DEFAULT CHARACTER SET utf8mb4
    DEFAULT COLLATE utf8mb4_unicode_ci;

USE blog_db;

CREATE TABLE users (
    id              BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    email           VARCHAR(255)    NOT NULL,
    password_hash   VARCHAR(255)    NOT NULL,
    nickname        VARCHAR(100)    NOT NULL DEFAULT '',
    avatar          VARCHAR(500)    NOT NULL DEFAULT '',
    is_active       TINYINT         NOT NULL DEFAULT 1,
    created_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    UNIQUE INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE categories (
    id          BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    name        VARCHAR(100)    NOT NULL,
    description VARCHAR(500)    NOT NULL DEFAULT '',
    created_at  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    UNIQUE INDEX idx_name (name)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE articles (
    id            BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    title         VARCHAR(255)    NOT NULL,
    slug          VARCHAR(300)    NOT NULL,
    content       LONGTEXT        NOT NULL,
    summary       VARCHAR(500)    NOT NULL DEFAULT '',
    cover_image   VARCHAR(500)    NOT NULL DEFAULT '',
    author_id     BIGINT UNSIGNED NOT NULL,
    category_id   BIGINT UNSIGNED NOT NULL DEFAULT 0,
    view_count    BIGINT UNSIGNED NOT NULL DEFAULT 0,
    is_published  TINYINT         NOT NULL DEFAULT 1,
    created_at    DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at    DATETIME        DEFAULT NULL,
    PRIMARY KEY (id),
    INDEX idx_slug (slug),
    INDEX idx_author_id (author_id),
    INDEX idx_category_id (category_id),
    INDEX idx_created_at (created_at),
    INDEX idx_deleted_at (deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE tags (
    id          BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    name        VARCHAR(100)    NOT NULL,
    created_at  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    UNIQUE INDEX idx_name (name)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE article_tags (
    article_id  BIGINT UNSIGNED NOT NULL,
    tag_id      BIGINT UNSIGNED NOT NULL,
    PRIMARY KEY (article_id, tag_id),
    INDEX idx_tag_id (tag_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE comments (
    id          BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    article_id  BIGINT UNSIGNED NOT NULL,
    user_id     BIGINT UNSIGNED NOT NULL,
    content     TEXT            NOT NULL,
    parent_id   BIGINT UNSIGNED NOT NULL DEFAULT 0,
    created_at  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at  DATETIME        DEFAULT NULL,
    PRIMARY KEY (id),
    INDEX idx_article_id (article_id),
    INDEX idx_user_id (user_id),
    INDEX idx_parent_id (parent_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 关系图谱详解

文章和分类是**一对多**：一个分类下有很多文章，一篇文章只属于一个分类。就像图书馆——"计算机"分类下有很多书，但每本书只能放在一个分类的书架上。

文章和标签是**多对多**：一篇文章可以有多个标签（"Go"、"后端"、"微服务"），一个标签下可以有多篇文章。多对多必须用中间表`article_tags`来关联。就像学生和课程——一个学生选多门课，一门课有很多学生，中间需要一张选课表。

评论支持**嵌套**：`parent_id=0`表示一级评论，`parent_id>0`表示回复某条评论。这样就能实现"楼中楼"的效果——A评论了文章，B回复了A的评论。

`slug`字段：文章的URL友好标识。比如标题是"如何学习Go语言"，slug就是`how-to-learn-go`。用slug而不是数字ID，既对SEO友好，又对人友好。

## 112.4 API设计

### 统一响应格式

沿用前面的 `{code, message, data}` 格式。

### 文章接口

#### 创建文章 — POST /api/articles（需登录）

请求体：
```json
{
    "title": "Go语言入门教程",
    "content": "# 第一章\n\nGo是一门...",
    "summary": "这是一篇Go入门教程",
    "category_id": 1,
    "tag_ids": [1, 3, 5]
}
```

`tag_ids` 是标签ID数组——创建文章时同时建立文章和标签的关联。

响应（201）：
```json
{
    "code": 201,
    "message": "创建成功",
    "data": {
        "id": 1,
        "title": "Go语言入门教程",
        "slug": "go-language-tutorial",
        "content": "# 第一章...",
        "summary": "这是一篇Go入门教程",
        "author": { "id": 1, "nickname": "张三" },
        "category": { "id": 1, "name": "后端开发" },
        "tags": [
            { "id": 1, "name": "Go" },
            { "id": 3, "name": "后端" }
        ],
        "created_at": "2025-01-01T10:00:00Z"
    }
}
```

#### 文章列表 — GET /api/articles?page=1&page_size=10&category_id=1&tag_id=3&keyword=Go&author_id=1

全部参数可选：
- `page`、`page_size`：分页
- `category_id`：按分类筛选
- `tag_id`：按标签筛选
- `keyword`：搜索标题和摘要（用LIKE）
- `author_id`：只看某人的文章

#### 文章详情 — GET /api/articles/:id

响应包含文章全文、作者信息、分类、所有标签。

#### 更新文章 — PUT /api/articles/:id（需登录，只能改自己的）

请求体同创建，所有字段可选。

#### 删除文章 — DELETE /api/articles/:id（需登录，只能删自己的）

### 评论接口

#### 发表评论 — POST /api/articles/:id/comments（需登录）

请求体：
```json
{
    "content": "写得真好！",
    "parent_id": 0
}
```

#### 文章评论列表 — GET /api/articles/:id/comments

不分页，一次返回所有评论，按时间正序排列。

## 112.5 核心代码实现

### model/article.go — 文章模型

```go
package model

import (
    "strings"
    "time"

    "gorm.io/gorm"
)

type Article struct {
    ID          uint64         `gorm:"primaryKey" json:"id"`
    Title       string         `gorm:"size:255;not null" json:"title"`
    Slug        string         `gorm:"size:300;not null" json:"slug"`
    Content     string         `gorm:"type:longtext;not null" json:"content"`
    Summary     string         `gorm:"size:500" json:"summary"`
    CoverImage  string         `gorm:"size:500" json:"cover_image"`
    AuthorID    uint64         `gorm:"not null" json:"author_id"`
    CategoryID  uint64         `gorm:"not null;default:0" json:"category_id"`
    ViewCount   uint64         `gorm:"default:0" json:"view_count"`
    IsPublished int8           `gorm:"default:1" json:"is_published"`
    CreatedAt   time.Time      `json:"created_at"`
    UpdatedAt   time.Time      `json:"updated_at"`
    DeletedAt   gorm.DeletedAt `gorm:"index" json:"-"`

    Author   User     `gorm:"foreignKey:AuthorID" json:"author"`
    Category Category `gorm:"foreignKey:CategoryID" json:"category"`
    Tags     []Tag    `gorm:"many2many:article_tags" json:"tags"`
}

func (Article) TableName() string {
    return "articles"
}

type CreateArticleReq struct {
    Title      string   `json:"title" binding:"required"`
    Content    string   `json:"content" binding:"required"`
    Summary    string   `json:"summary"`
    CoverImage string   `json:"cover_image"`
    CategoryID uint64   `json:"category_id"`
    TagIDs     []uint64 `json:"tag_ids"`
}

type UpdateArticleReq struct {
    Title      string   `json:"title"`
    Content    string   `json:"content"`
    Summary    string   `json:"summary"`
    CoverImage string   `json:"cover_image"`
    CategoryID *uint64  `json:"category_id"`
    TagIDs     []uint64 `json:"tag_ids"`
}

func generateSlug(title string) string {
    // 注意：本简化版仅处理英文标题。中文标题会被 ToLower 和 ReplaceAll 处理后仍为原样，
    // 如需正确生成中文 slug（如 "你好世界" → "ni-hao-shi-jie"），可引入 github.com/gosimple/slug 等库。
    slug := strings.ToLower(title)
    slug = strings.ReplaceAll(slug, " ", "-")
    return slug
}
```

`gorm:"foreignKey:AuthorID"` 告诉GORM：Article的Author关联是通过AuthorID字段去users表找的。同样，`many2many:article_tags` 告诉GORM多对多关联通过中间表`article_tags`。

### model/article.go — CRUD方法

```go
func CreateArticle(db *gorm.DB, authorID uint64, req *CreateArticleReq) (*Article, error) {
    article := &Article{
        Title:      req.Title,
        Slug:       generateSlug(req.Title),
        Content:    req.Content,
        Summary:    req.Summary,
        CoverImage: req.CoverImage,
        AuthorID:   authorID,
        CategoryID: req.CategoryID,
    }

    err := db.Transaction(func(tx *gorm.DB) error {
        if err := tx.Create(article).Error; err != nil {
            return err
        }

        if len(req.TagIDs) > 0 {
            tags := make([]Tag, len(req.TagIDs))
            for i, id := range req.TagIDs {
                tags[i] = Tag{ID: id}
            }
            if err := tx.Model(article).Association("Tags").Replace(tags); err != nil {
                return err
            }
        }

        return nil
    })

    if err != nil {
        return nil, err
    }

    return GetArticleByID(db, article.ID)
}

func GetArticleByID(db *gorm.DB, id uint64) (*Article, error) {
    var article Article
    err := db.Preload("Author").Preload("Category").Preload("Tags").
        First(&article, id).Error
    if err != nil {
        return nil, err
    }
    return &article, nil
}

func ListArticles(db *gorm.DB, page, pageSize int, categoryID, tagID, authorID *uint64, keyword string) (*ListResp, error) {
    var articles []Article
    var total int64

    query := db.Model(&Article{})

    if categoryID != nil && *categoryID > 0 {
        query = query.Where("category_id = ?", *categoryID)
    }
    if authorID != nil && *authorID > 0 {
        query = query.Where("author_id = ?", *authorID)
    }
    if keyword != "" {
        like := "%" + keyword + "%"
        query = query.Where("title LIKE ? OR summary LIKE ?", like, like)
    }
    if tagID != nil && *tagID > 0 {
        query = query.Joins("JOIN article_tags ON article_tags.article_id = articles.id").
            Where("article_tags.tag_id = ?", *tagID)
    }

    query.Count(&total)

    offset := (page - 1) * pageSize
    err := query.Preload("Author").Preload("Category").Preload("Tags").
        Order("created_at DESC").
        Offset(offset).Limit(pageSize).
        Find(&articles).Error
    if err != nil {
        return nil, err
    }

    return &ListResp{
        List:     articles,
        Total:    total,
        Page:     page,
        PageSize: pageSize,
    }, nil
}

func UpdateArticle(db *gorm.DB, id, authorID uint64, req *UpdateArticleReq) (*Article, error) {
    var article Article
    if err := db.Where("id = ? AND author_id = ?", id, authorID).
        First(&article).Error; err != nil {
        return nil, err
    }

    err := db.Transaction(func(tx *gorm.DB) error {
        updates := map[string]interface{}{}
        if req.Title != "" {
            updates["title"] = req.Title
            updates["slug"] = generateSlug(req.Title)
        }
        if req.Content != "" {
            updates["content"] = req.Content
        }
        if req.Summary != "" {
            updates["summary"] = req.Summary
        }
        if req.CategoryID != nil {
            updates["category_id"] = *req.CategoryID
        }
        if len(updates) > 0 {
            if err := tx.Model(&article).Updates(updates).Error; err != nil {
                return err
            }
        }

        if req.TagIDs != nil {
            tags := make([]Tag, len(req.TagIDs))
            for i, id := range req.TagIDs {
                tags[i] = Tag{ID: id}
            }
            if err := tx.Model(&article).Association("Tags").Replace(tags); err != nil {
                return err
            }
        }

        return nil
    })

    if err != nil {
        return nil, err
    }

    return GetArticleByID(db, article.ID)
}

func DeleteArticle(db *gorm.DB, id, authorID uint64) error {
    result := db.Where("id = ? AND author_id = ?", id, authorID).
        Delete(&Article{})
    if result.RowsAffected == 0 {
        return gorm.ErrRecordNotFound
    }
    return result.Error
}
```

关键点：`db.Transaction` 包裹文章创建和标签关联，保证原子性——要么文章和标签都创建成功，要么都回滚。不会出现"文章创建了但标签关联失败"的半吊子状态。

更新和删除都加了 `AND author_id = ?` 条件——**只有作者自己能改/删自己的文章**。这是业务逻辑中的权限控制。

### model/comment.go — 评论模型

```go
package model

import (
    "time"
    "gorm.io/gorm"
)

type Comment struct {
    ID        uint64         `gorm:"primaryKey" json:"id"`
    ArticleID uint64         `gorm:"not null" json:"article_id"`
    UserID    uint64         `gorm:"not null" json:"user_id"`
    Content   string         `gorm:"type:text;not null" json:"content"`
    ParentID  uint64         `gorm:"default:0" json:"parent_id"`
    CreatedAt time.Time      `json:"created_at"`
    DeletedAt gorm.DeletedAt `gorm:"index" json:"-"`

    User     User      `gorm:"foreignKey:UserID" json:"user"`
    Children []Comment `gorm:"foreignKey:ParentID" json:"children,omitempty"`
}

func (Comment) TableName() string {
    return "comments"
}

type CreateCommentReq struct {
    Content  string `json:"content" binding:"required"`
    ParentID uint64 `json:"parent_id"`
}

func CreateComment(db *gorm.DB, articleID, userID uint64, req *CreateCommentReq) (*Comment, error) {
    comment := &Comment{
        ArticleID: articleID,
        UserID:    userID,
        Content:   req.Content,
        ParentID:  req.ParentID,
    }
    if err := db.Create(comment).Error; err != nil {
        return nil, err
    }

    db.Preload("User").First(comment, comment.ID)
    return comment, nil
}

func ListComments(db *gorm.DB, articleID uint64) ([]Comment, error) {
    var comments []Comment
    err := db.Where("article_id = ?", articleID).
        Preload("User").
        Preload("Children.User").
        Order("created_at ASC").
        Find(&comments).Error
    return comments, err
}

func DeleteComment(db *gorm.DB, id, userID uint64) error {
    result := db.Where("id = ? AND user_id = ?", id, userID).
        Delete(&Comment{})
    if result.RowsAffected == 0 {
        return gorm.ErrRecordNotFound
    }
    return result.Error
}
```

`Children []Comment` 配合 `gorm:"foreignKey:ParentID"` 实现了嵌套评论的预加载。GORM会自动把parent_id匹配的评论填充到Children字段里——你不用手动递归查询。

### handler/article.go — 文章Handler

```go
package handler

import (
    "net/http"
    "strconv"

    "blog-api/model"

    "github.com/gin-gonic/gin"
    "gorm.io/gorm"
)

type ArticleHandler struct {
    DB *gorm.DB
}

func NewArticleHandler(db *gorm.DB) *ArticleHandler {
    return &ArticleHandler{DB: db}
}

func (h *ArticleHandler) Create(c *gin.Context) {
    userID := c.GetUint64("user_id")

    var req model.CreateArticleReq
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(http.StatusBadRequest, gin.H{
            "code": 400, "message": "参数错误: " + err.Error(), "data": nil,
        })
        return
    }

    article, err := model.CreateArticle(h.DB, userID, &req)
    if err != nil {
        c.JSON(http.StatusInternalServerError, gin.H{
            "code": 500, "message": "创建失败", "data": nil,
        })
        return
    }

    c.JSON(http.StatusCreated, gin.H{
        "code": 201, "message": "创建成功", "data": article,
    })
}

func (h *ArticleHandler) List(c *gin.Context) {
    page, _ := strconv.Atoi(c.DefaultQuery("page", "1"))
    pageSize, _ := strconv.Atoi(c.DefaultQuery("page_size", "10"))
    keyword := c.Query("keyword")

    parseUint64 := func(key string) *uint64 {
        if v := c.Query(key); v != "" {
            n, err := strconv.ParseUint(v, 10, 64)
            if err == nil {
                return &n
            }
        }
        return nil
    }

    resp, err := model.ListArticles(h.DB, page, pageSize,
        parseUint64("category_id"),
        parseUint64("tag_id"),
        parseUint64("author_id"),
        keyword,
    )
    if err != nil {
        c.JSON(http.StatusInternalServerError, gin.H{
            "code": 500, "message": "查询失败", "data": nil,
        })
        return
    }

    c.JSON(http.StatusOK, gin.H{
        "code": 200, "message": "success", "data": resp,
    })
}

func (h *ArticleHandler) GetByID(c *gin.Context) {
    id, _ := strconv.ParseUint(c.Param("id"), 10, 64)

    article, err := model.GetArticleByID(h.DB, id)
    if err != nil {
        c.JSON(http.StatusNotFound, gin.H{
            "code": 404, "message": "文章不存在", "data": nil,
        })
        return
    }

    c.JSON(http.StatusOK, gin.H{
        "code": 200, "message": "success", "data": article,
    })
}

func (h *ArticleHandler) Update(c *gin.Context) {
    userID := c.GetUint64("user_id")
    id, _ := strconv.ParseUint(c.Param("id"), 10, 64)

    var req model.UpdateArticleReq
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(http.StatusBadRequest, gin.H{
            "code": 400, "message": "参数错误", "data": nil,
        })
        return
    }

    article, err := model.UpdateArticle(h.DB, id, userID, &req)
    if err != nil {
        c.JSON(http.StatusInternalServerError, gin.H{
            "code": 500, "message": "更新失败", "data": nil,
        })
        return
    }

    c.JSON(http.StatusOK, gin.H{
        "code": 200, "message": "更新成功", "data": article,
    })
}

func (h *ArticleHandler) Delete(c *gin.Context) {
    userID := c.GetUint64("user_id")
    id, _ := strconv.ParseUint(c.Param("id"), 10, 64)

    if err := model.DeleteArticle(h.DB, id, userID); err != nil {
```

## 本章小结

本章搭建了一个完整的多表关联博客系统，核心收获：

1. **多表关系**：一对多（文章-分类）、多对多（文章-标签）、自引用（评论嵌套）
2. **GORM关联预加载**：`Preload` 一条语句搞定所有关联查询，不用写JOIN
3. **事务管理**：`db.Transaction` 保证创建文章+关联标签的原子性
4. **权限控制**：更新和删除必须验证 `author_id`，防止越权操作
5. **嵌套评论**：用 `ParentID` 自引用实现楼中楼，`Children` 预加载自动递归

> 🚀 博客系统是"内容型应用"的范本。你学到的多表关联、权限控制、评论系统这些模式，在论坛、知识库、文档管理系统中同样适用。下一章，我们要面对真正的高并发挑战——电商秒杀系统。
