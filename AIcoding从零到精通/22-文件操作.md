## 程序怎么读写文件：打开冰箱门

程序运行的时候，数据存在变量里。但程序一关，变量就没了——就像你脑子里记的东西，睡一觉可能就忘了。

要让数据"留下来"，就得写到文件里。读文件就是把存下来的数据再拿出来。

这个过程就像**开冰箱**：

```
打开冰箱门 → 拿东西/放东西 → 关冰箱门
  open()      read()/write()    close()
```

Python操作文件三步走：

```python
# 第一步：打开文件（就像打开冰箱门）
file = open("日记.txt", "w", encoding="utf-8")

# 第二步：写东西（就像往冰箱里放东西）
file.write("2026年6月5日，晴。今天开始学AI编程。\n")

# 第三步：关闭文件（就像关冰箱门）
file.close()
```

> 📖 `"w"` 是"write"（写模式），`"r"` 是"read"（读模式），`encoding="utf-8"` 是告诉Python"用UTF-8编码读写中文"——没有这一句，中文可能会乱码。

---

## 三种文件操作模式

| 模式 | 代码 | 说明 | 比喻 |
|------|------|------|------|
| 读 | `"r"` | 打开已有文件，读取内容 | 打开冰箱看里面有什么 |
| 写 | `"w"` | 创建新文件（已有则覆盖！） | 清空冰箱，放新东西 |
| 追加 | `"a"` | 在文件末尾加内容，不删旧的 | 在冰箱里加东西，不扔旧的 |

> ⚠️ **`"w"` 模式会覆盖原文件！** 如果你用 `"w"` 打开一个已有文件，原来的内容就没了。如果只是想加内容，用 `"a"`（追加模式）。

## 更安全的写法：with语句

每次都写 `close()` 很麻烦，而且容易忘。Python提供了一种更安全的写法——`with`：

```python
# with 语句会自动帮你关文件，就像自动关门
with open("日记.txt", "r", encoding="utf-8") as file:
    content = file.read()      # 读取全部内容
    print(content)             # 打印出来
# 出了with的缩进块，文件自动关闭
```

**推荐始终用 `with`**，不用手动 `close()`。

## AI生成代码示例：日记本程序

> 以下代码由AI生成：

```python
import os                               # 导入os模块，用来检查文件是否存在

diary_file = "我的日记.txt"              # 文件名

# 写日记
def write_diary():
    date = input("请输入日期（如2026-06-05）：")       # 让用户输入日期
    content = input("请输入日记内容：")                 # 让用户输入内容
    with open(diary_file, "a", encoding="utf-8") as f:  # 追加模式打开
        f.write(f"[{date}] {content}\n")                # 写入日期+内容
    print("日记已保存！")

# 读日记
def read_diary():
    if os.path.exists(diary_file):                     # 检查文件是否存在
        with open(diary_file, "r", encoding="utf-8") as f:
            all_content = f.read()                      # 读取全部内容
            print("===== 我的日记 =====")
            print(all_content)
    else:
        print("还没有日记，先写一篇吧！")

# 简单菜单
while True:
    choice = input("1-写日记  2-看日记  3-退出\n请选择：")
    if choice == "1":
        write_diary()
    elif choice == "2":
        read_diary()
    elif choice == "3":
        print("再见！")
        break
```

**逐行讲解**：
- 第1行：`import os` 导入操作系统模块，后面用它检查文件是否存在。
- 第3行：定义文件名变量，方便修改。
- 第6-11行：`write_diary()` 函数——让用户输入日期和内容，用 `"a"` 追加模式写入文件。
- 第14-21行：`read_diary()` 函数——先检查文件是否存在，存在就读取全部内容并打印。
- 第17行：`os.path.exists()` 检查文件是否存在，避免"文件不存在"报错。
- 第24-32行：一个简单的菜单循环，根据用户选择调用对应函数。

## 常见文件格式：JSON

JSON（📖 术语：JavaScript Object Notation，一种轻量级数据交换格式）是AI和程序之间交流最常用的格式。**它长得像Python字典，但本质是文本。**

```json
{
    "name": "张三",
    "age": 28,
    "skills": ["Python", "AI"]
}
```

你会发现在第23章（API）大量用到JSON。Python读写JSON很方便：

```python
import json

# 写JSON：字典 → JSON字符串 → 文件
data = {"name": "张三", "age": 28}
with open("data.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False)  # ensure_ascii=False 保证中文不变成乱码

# 读JSON：文件 → JSON字符串 → 字典
with open("data.json", "r", encoding="utf-8") as f:
    loaded = json.load(f)
    print(loaded["name"])  # 输出：张三
```

## 排错：找不到文件怎么办？

最常见的文件操作错误：

```
FileNotFoundError: [Errno 2] No such file or directory: 'xxx.txt'
```

翻译：**"找不到你说的那个文件"**。

**三个检查步骤**：
1. 文件名写对了吗？（注意大小写、后缀名）
2. 文件在程序同一个目录下吗？（最简单的办法：**把文件放在和.py程序同一个文件夹里**）
3. 路径写对了吗？

**推荐做法**：把文件放在程序同目录，用**相对路径**（只写文件名，不写完整的 `C:\Users\...` 路径）。

```python
# ✅ 推荐：相对路径——文件在程序旁边
with open("日记.txt", "r", encoding="utf-8") as f:
    ...

# ❌ 不推荐：绝对路径——换个电脑就跑不了
with open("C:/Users/张三/Desktop/日记.txt", "r", encoding="utf-8") as f:
    ...
```
