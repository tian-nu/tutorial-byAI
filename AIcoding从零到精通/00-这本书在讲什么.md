## 传统编程 vs AI编程：从"自己做饭"到"指挥厨师"

想象一下你想吃一盘番茄炒蛋。

**传统编程**就像你**自己做饭**：

1. 你得先去菜市场挑番茄、买鸡蛋、买葱姜蒜
2. 你得知道番茄怎么切、鸡蛋怎么打、火候怎么控制
3. 你得记住先放油还是先放蛋、放多少盐、炒多久
4. 如果炒糊了，你得自己想办法补救
5. 最后能不能吃，全看你的厨艺水平

**AI编程**就像你**指挥厨师做饭**：

1. 你告诉厨师："我要一盘番茄炒蛋，不要太咸，蛋要嫩一点"
2. 厨师帮你挑食材、切菜、炒菜
3. 你尝一口，觉得不够咸——"再加点盐"——厨师马上调整
4. 你不需要知道火候怎么控制、油温多少度
5. 但你至少得知道：这是番茄炒蛋，不是红烧肉，咸了不好吃

**AI编程不是"不用学编程了"，而是"不用死记硬背每一行代码怎么写"**。你从"写代码的人"变成了"指挥AI写代码的人"——你负责说清楚要做什么，AI负责把它做出来。

## 为什么2026年，你必须学AI编程

你可能听说过一句话："AI会取代程序员"。

**这话只说对了一半**。真相是：

> AI不会取代程序员，但**会用AI的程序员**会取代**不会用AI的程序员**。

就像汽车发明后，马车夫失业了，但司机这个职业诞生了。AI编程工具正在改变编程的方式，就像汽车改变了出行方式一样。

**现在学AI编程的三个理由**：

1. **门槛降低了**：以前你学编程，得先背语法、记函数、理解指针、搞懂面向对象……现在你只需要会说清楚你要什么，AI帮你写代码。你学编程的时间从"一年"缩短到了"一个月"。

2. **效率提升了**：一个会用AI的程序员，工作效率可能是不会用AI的3-5倍。那些重复的、无聊的代码（比如写100个差不多的API接口），AI几秒钟就能搞定。

3. **窗口期还在**：2026年，AI编程工具已经成熟，但真正"精通AI编程"的人还不多。现在学，你是先行者；等两年再学，你就是追赶者。

## 术语约定：看不懂的词？别慌

这本书里会出现很多你可能没听过的词：**Token、Prompt、LLM、API、Composer、Agent……**

别被这些词吓到。**我们约定一件事**：

> 本书中，每一个陌生术语第一次出现时，都会用大白话解释清楚。你不需要提前记住任何东西。

比如：
- "Token"——第一次出现时我们会说：Token就是把文字切成的小积木块，每个Token大概等于3-4个汉字
- "Prompt"——第一次出现时我们会说：Prompt就是你给AI的指令说明书

**而且所有术语都收录在附录E「技术术语速查」里**，你随时可以翻回去查，不用担心学了后面忘了前面。

## 这本书不教什么（也很重要）

| 不教什么 | 为什么 |
|---------|--------|
| 传统编程语言的完整语法 | 本仓库有《Go从零到后端教程》和《Java从零到Spring Boot后端教程》专门讲 |
| AI/ML模型训练、深度学习 | 这是另一个领域，和用AI写代码不是一回事 |
| 前端框架（React/Vue）深入 | 本书只用到最基础的HTML/CSS/JS |
| 后端框架（Spring/Gin）深入 | 同上，本书只用到Python Flask最基础的功能 |
| DevOps、云部署、数据库深入 | 同样是Go/Java系列教程的覆盖范围 |
| 所有AI编程工具 | 本书只深度讲**Cursor + Trae**两个，其他工具在第一章简介 |

**明确排除这些内容，是为了让你聚焦**。学完这本书，你不会变成"全栈工程师"，但你会变成一个**能用AI高效写代码的人**。如果你想深入学习后端开发，本仓库的Go/Java教程会接住你。

## 如何使用本书

### 零基础读者

1. **按顺序读**：从第零篇到第七篇，一章一章来
2. **跟着做，不要只读**：每章有练习，打开你的电脑，动手做一遍
3. **遇到不会的词，查附录E**：你不需要记住所有术语，但要知道去哪找
4. **每篇结束都是一个自然暂停点**：累了就停，下一篇再继续，不用赶
5. **遇到报错不要慌**：这是学习的一部分，附录D有常见错误排错指南

### 有编程基础读者

1. **跳过第零篇和第二篇**：你已经知道编程是什么
2. **重点读第一篇**（Prompt Engineering）和**第三篇**（Cursor）：这是最有价值的
3. **第四篇可选**（Trae）：如果你只用Cursor，跳过Trae也可以
4. **第六篇挑着做**：找感兴趣的项目做，不用全做

### 所有人

1. **附录C（Prompt模板库）建议打印出来**：放在手边随时参考
2. **附录D（常见错误排错）收藏起来**：出问题了第一时间翻

## 在开始之前，还有一件事

你可能听说过有人用AI写代码，**直接把API密钥、数据库密码都发给了AI**——然后密钥泄露了，损失惨重。

**这本书会在第54章专门讲安全与隐私**，但在那之前，请记住：

> **绝对不要把密钥、密码、Token发给AI。** 用占位符代替，比如把 `API_KEY=sk-abc123` 换成 `API_KEY=your_api_key_here`。
