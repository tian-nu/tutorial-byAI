# 第44章 · 栈与队列

> "切片和链表是'全能选手'——你想在哪插入就在哪插入，想在哪删除就在哪删除。但有时候，太自由反而是负担。比如你要做一个浏览器的'后退'按钮——你永远只关心'上一个页面是什么'，不会关心后退了5步之后的那个页面。这时候，一个只能从顶部操作的'栈'就比一个能到处访问的切片更合适：它限制了你，但也保护了你——你不会不小心把中间的页面删掉。"

## 44.1 栈（Stack）：LIFO，一叠盘子

**栈的核心规则只有一句话：后进先出（LIFO：Last In, First Out）。**

想象一叠待洗的盘子：
- 新盘子放在最上面（Push）
- 取盘子时从最上面拿（Pop）
- 只能操作最上面那个（Top/Peek）
- 不能从中间抽一个（不是切片！）

```
     ┌───┐
Push→│ 3 │←Top（栈顶）
     ├───┤
     │ 2 │
     ├───┤
     │ 1 │←Bottom（栈底）
     └───┘
```

### Go实现栈（基于切片）

Go标准库没有专门的Stack类型，但用切片实现栈极其简单：

```go
type Stack struct {
    data []int
}

func NewStack() *Stack {
    return &Stack{data: []int{}}
}

func (s *Stack) Push(val int) {
    s.data = append(s.data, val)
}

func (s *Stack) Pop() (int, bool) {
    if len(s.data) == 0 {
        return 0, false
    }
    val := s.data[len(s.data)-1]
    s.data = s.data[:len(s.data)-1]
    return val, true
}

func (s *Stack) Peek() (int, bool) {
    if len(s.data) == 0 {
        return 0, false
    }
    return s.data[len(s.data)-1], true
}

func (s *Stack) IsEmpty() bool {
    return len(s.data) == 0
}

func (s *Stack) Size() int {
    return len(s.data)
}
```

所有操作都是O(1)：
- `Push`：在切片尾部追加（均摊O(1)）
- `Pop`：从切片尾部删除（O(1)）
- `Peek`：读切片尾部元素（O(1)）

**为什么用切片的尾部做栈顶？** 因为切片尾部操作是O(1)且不会触发大量数据移动。如果从头部操作，每次都要把所有元素往前挪一位——那栈的效率就变成O(n)了。

### 栈的五大经典应用

#### 应用一：函数调用栈

这可能是你每天都在用、只是没意识到的栈。

当你写：
```go
func A() {
    B()
}

func B() {
    C()
}

func C() {
    fmt.Println("Hello")
}

func main() {
    A()
}
```

Go运行时会维护一个调用栈：

```
C被调用时：               B被调用时：              A被调用时：              main被调用时：
┌─────┐                  ┌─────┐                 ┌─────┐                 ┌─────┐
│  C  │←栈顶              │  B  │                 │  A  │                 │main │
├─────┤                  ├─────┤                 ├─────┤                 ├─────┤
│  B  │                  │  A  │                 │main │
├─────┤                  ├─────┤                 ├─────┤
│  A  │                  │main │
├─────┤                  ├─────┤
│main │
└─────┘
```

C执行完→弹出C，回到B；B执行完→弹出B，回到A；A执行完→弹出A，回到main。

**这就是栈！** 每次函数调用Push一个栈帧（存局部变量、返回地址），函数返回时Pop。所有编程语言的函数调用都是这样实现的。

#### 应用二：括号匹配

这是面试中的经典问题——判断一个字符串中的括号是否正确配对。

```go
func isValid(s string) bool {
    stack := []rune{}
    pairs := map[rune]rune{
        ')': '(',
        ']': '[',
        '}': '{',
    }
    for _, ch := range s {
        if ch == '(' || ch == '[' || ch == '{' {
            stack = append(stack, ch)
        } else if ch == ')' || ch == ']' || ch == '}' {
            if len(stack) == 0 {
                return false
            }
            top := stack[len(stack)-1]
            stack = stack[:len(stack)-1]
            if top != pairs[ch] {
                return false
            }
        }
    }
    return len(stack) == 0
}

fmt.Println(isValid("()[]{}"))   // true
fmt.Println(isValid("([)]"))     // false
fmt.Println(isValid("{[]}"))     // true
```

思路：
1. 遇到左括号→Push
2. 遇到右括号→Pop栈顶，检查是否匹配
3. 最后栈为空→全部配对成功

#### 应用三：浏览器的前进后退

浏览器的后退功能天生就是栈的典型应用：

```
浏览页面A → 浏览页面B → 浏览页面C → 浏览页面D

回退栈（后退）：
┌───┐
│ D │  ← 当前页面
├───┤
│ C │
├───┤
│ B │
├───┤
│ A │
└───┘

前进栈（前进）：
（空）

点击"后退"：
回退栈Pop出D → Push到前进栈
当前页面变成C

点击"前进"：
前进栈Pop出D → Push到回退栈
当前页面变成D
```

Go实现：

```go
type BrowserHistory struct {
    backStack    []string
    forwardStack []string
    current      string
}

func (b *BrowserHistory) Visit(url string) {
    b.backStack = append(b.backStack, b.current)
    b.current = url
    b.forwardStack = nil
}

func (b *BrowserHistory) Back() string {
    if len(b.backStack) == 0 {
        return b.current
    }
    b.forwardStack = append(b.forwardStack, b.current)
    b.current = b.backStack[len(b.backStack)-1]
    b.backStack = b.backStack[:len(b.backStack)-1]
    return b.current
}

func (b *BrowserHistory) Forward() string {
    if len(b.forwardStack) == 0 {
        return b.current
    }
    b.backStack = append(b.backStack, b.current)
    b.current = b.forwardStack[len(b.forwardStack)-1]
    b.forwardStack = b.forwardStack[:len(b.forwardStack)-1]
    return b.current
}
```

#### 应用四：深度优先搜索（DFS）

栈天然适合实现DFS——一条路走到黑，没路了就退回来换条路。这个在第49章会详细展开。这里先看个简化版——迷宫问题：

```go
func dfs(grid [][]int, startX, startY int) {
    type Point struct{ x, y int }
    stack := []Point{{startX, startY}}
    visited := make(map[Point]bool)

    for len(stack) > 0 {
        p := stack[len(stack)-1]
        stack = stack[:len(stack)-1]

        if visited[p] {
            continue
        }
        visited[p] = true
        fmt.Printf("访问 (%d,%d)\n", p.x, p.y)

        if p.x+1 < len(grid) && grid[p.x+1][p.y] == 0 {
            stack = append(stack, Point{p.x + 1, p.y})
        }
        if p.x-1 >= 0 && grid[p.x-1][p.y] == 0 {
            stack = append(stack, Point{p.x - 1, p.y})
        }
        if p.y+1 < len(grid[0]) && grid[p.x][p.y+1] == 0 {
            stack = append(stack, Point{p.x, p.y + 1})
        }
        if p.y-1 >= 0 && grid[p.x][p.y-1] == 0 {
            stack = append(stack, Point{p.x, p.y - 1})
        }
    }
}
```

#### 应用五：表达式求值

计算 `3 + 4 * 2` 时，不能从左到右——要先算 `4 * 2`，再加 `3`。这就是运算符优先级问题。栈可以优雅地解决它。

简化版（只有+和*）：

```go
func calculate(s string) int {
    numStack := []int{}
    num := 0
    prevOp := '+'
    for i, ch := range s {
        if ch >= '0' && ch <= '9' {
            num = num*10 + int(ch-'0')
        }
        if ch == '+' || ch == '*' || i == len(s)-1 {
            if prevOp == '+' {
                numStack = append(numStack, num)
            } else if prevOp == '*' {
                top := numStack[len(numStack)-1]
                numStack[len(numStack)-1] = top * num
            }
            prevOp = ch
            num = 0
        }
    }
    result := 0
    for _, v := range numStack {
        result += v
    }
    return result
}

fmt.Println(calculate("3+4*2"))   // 11
fmt.Println(calculate("3*4+2"))   // 14
```

思路：
- 遇到数字：累积（考虑多位数）
- 遇到运算符（或结尾）：看上一个运算符是什么
  - 上一个运算符是 `+`：当前数字Push到栈
  - 上一个运算符是 `*`：栈顶元素 × 当前数字，更新栈顶
- 最后栈里所有数字相加

## 44.2 队列（Queue）：FIFO，排队买奶茶

**队列的核心规则：先进先出（FIFO：First In, First Out）。**

你去买奶茶——你排在最前面，你是第一个点单、第一个拿到奶茶的人。后面来的人排在你后面，等你走了才轮到他。

```
Enqueue→ ┌───┬───┬───┐
         │ 1 │ 2 │ 3 │→ Dequeue
         └───┴───┴───┘
        队尾      队头
```

### Go实现队列（基于切片）

```go
type Queue struct {
    data []int
}

func NewQueue() *Queue {
    return &Queue{data: []int{}}
}

func (q *Queue) Enqueue(val int) {
    q.data = append(q.data, val)
}

func (q *Queue) Dequeue() (int, bool) {
    if len(q.data) == 0 {
        return 0, false
    }
    val := q.data[0]
    q.data = q.data[1:]
    return val, true
}

func (q *Queue) Peek() (int, bool) {
    if len(q.data) == 0 {
        return 0, false
    }
    return q.data[0], true
}

func (q *Queue) IsEmpty() bool {
    return len(q.data) == 0
}
```

**但是注意！** 这个实现的 `Dequeue()` 有性能问题：

```go
q.data = q.data[1:]
```

这一步看起来简单，但实际上会导致底层数组不断前移——所有元素都要往左挪一位。当队列中有n个元素时，Dequeue是O(n)而不是O(1)。

**栈为什么用切片尾部没问题？** 因为Pop取的是末尾元素，不用移动。而队列取的是头部——头部取走后所有后面的元素都要前移。

### 队列的三大经典应用

#### 应用一：BFS（广度优先搜索）

队列是实现BFS的天然工具——一层一层展开。第49章会详细讲。先看一个最短路径的例子：

```go
func shortestPath(grid [][]int, startX, startY, endX, endY int) int {
    type Point struct{ x, y, dist int }
    queue := []Point{{startX, startY, 0}}
    visited := make(map[[2]int]bool)

    for len(queue) > 0 {
        p := queue[0]
        queue = queue[1:] // 注意：切片头部出队是 O(n)，生产环境建议用 container/list 或 channel

        if p.x == endX && p.y == endY {
            return p.dist
        }

        key := [2]int{p.x, p.y}
        if visited[key] {
            continue
        }
        visited[key] = true

        dirs := [][2]int{{1, 0}, {-1, 0}, {0, 1}, {0, -1}}
        for _, d := range dirs {
            nx, ny := p.x+d[0], p.y+d[1]
            if nx >= 0 && nx < len(grid) && ny >= 0 && ny < len(grid[0]) && grid[nx][ny] == 0 {
                queue = append(queue, Point{nx, ny, p.dist + 1})
            }
        }
    }
    return -1
}
```

BFS用队列保证了"先访问近的，再访问远的"——这正是最短路径的保证。

#### 应用二：消息队列

在后端系统中，消息队列是最常见的队列应用。生产者往队列尾部发消息，消费者从队列头部拿消息。

Go的channel本质上就是一个并发安全的队列：

```go
ch := make(chan int, 10)
ch <- 1
ch <- 2
val := <-ch
fmt.Println(val) // 1（先进入的先出来）
```

带缓冲的channel就是队列——先进先出。

#### 应用三：任务调度

操作系统的进程调度、线程池的任务分配都可以用队列实现：

```go
type Task struct {
    ID       int
    Priority int
}

type TaskQueue struct {
    queue []Task
}

func (tq *TaskQueue) Submit(task Task) {
    tq.queue = append(tq.queue, task)
}

func (tq *TaskQueue) Execute() {
    for len(tq.queue) > 0 {
        task := tq.queue[0]
        tq.queue = tq.queue[1:]
        fmt.Printf("执行任务 %d\n", task.ID)
    }
}
```

## 44.3 循环队列：解决"假溢出"和Dequeue低效

### 普通队列的两大问题

**问题一：Dequeue是O(n)**。每次从头部取元素，后面所有元素都要往前挪。

**问题二：假溢出**。如果队列容量固定为5，Enqueue了5个元素（队列满），然后Dequeue了3个（队列有空位），再Enqueue时——队尾指针已经到末尾了，但前面有3个空位。这种现象叫"假溢出"。

```
初始:    [ ][ ][ ][ ][ ]
         ↑head     ←tail
满:      [1][2][3][4][5]
         ↑head        ←tail
出队3个: [_][_][_][4][5]
         ↑head    ←tail
想入队:  tail已经到尽头了！但前面还有3个空位——假溢出
```

### 循环队列的原理

把队列的数组想象成一个环形——tail到了数组末尾后，绕回头部。

```
物理数组: [0][1][2][3][4]
逻辑环:    0 → 1 → 2 → 3 → 4 → 0 → 1 → ...
```

### Go实现循环队列

```go
type CircularQueue struct {
    data     []int
    head     int
    tail     int
    size     int
    capacity int
}

func NewCircularQueue(capacity int) *CircularQueue {
    return &CircularQueue{
        data:     make([]int, capacity),
        head:     0,
        tail:     0,
        size:     0,
        capacity: capacity,
    }
}

func (cq *CircularQueue) Enqueue(val int) bool {
    if cq.size == cq.capacity {
        return false
    }
    cq.data[cq.tail] = val
    cq.tail = (cq.tail + 1) % cq.capacity
    cq.size++
    return true
}

func (cq *CircularQueue) Dequeue() (int, bool) {
    if cq.size == 0 {
        return 0, false
    }
    val := cq.data[cq.head]
    cq.head = (cq.head + 1) % cq.capacity
    cq.size--
    return val, true
}

func (cq *CircularQueue) Peek() (int, bool) {
    if cq.size == 0 {
        return 0, false
    }
    return cq.data[cq.head], true
}
```

关键技巧：`(head + 1) % capacity` 和 `(tail + 1) % capacity`——用取模运算实现"绕圈"。

```
capacity=5

初始:    head=0 tail=0 size=0
入队3:   [1][2][3][_][_]  head=0 tail=3 size=3
出队2:   [_][_][3][_][_]  head=2 tail=3 size=1
入队3:   [6][7][3][4][5]  head=2 tail=1 size=4
                           （tail绕了一圈：3→4→0→1）
入队1:   [6][7][3][4][5]  head=2 tail=2 size=5 满！
```

现在Enqueue和Dequeue都是真正的O(1)——没有元素移动，没有假溢出。

### 什么时候用循环队列

- **固定容量的缓冲区**：网络数据包缓冲、音频处理缓冲
- **高性能场景**：需要极致效率，不想反复分配内存（切片扩容）
- **Go的channel底层就是循环队列**——理解了循环队列，就理解了channel的内部结构

## 🤔 想多一点：栈和队列的"受限"反而是一种力量

你可能会想：为什么要用受限的结构？切片不就能干所有事吗？

**因为限制创造了秩序。** 就像交通规则——红灯停绿灯行是一种限制，但没有这个限制，大家都随便开，十字路口就会变成一个巨大的停车场。

- 栈保证了后进先出→天然适合回退（undo/redo）、递归、语法解析
- 队列保证了先进先出→天然适合顺序处理（任务调度、BFS、消息系统）

当你用切片做栈时，理论上你可以调用`data[2]`来访问中间的元素——但这意味着你在滥用栈。如果代码review时被人发现，他们会问："这里为什么需要一个栈？你是不是搞错了数据结构？"

**选对数据结构，不仅是选对性能，更是选对"语义"。** 栈告诉读代码的人："这里只关心最近的事情，历史不重要。" 队列告诉读代码的人："这里严格按照先来后到，谁也不许插队。"
