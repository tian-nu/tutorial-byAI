## 三、分步操作

### 步骤1：理解 Authorization 头——给请求贴上「通行证」

#### 我在做什么？

发布文章需要登录，但 HTTP 请求本身是「无状态」的——服务器不知道你是谁。所以需要把 token 贴在请求头上，就像快递员送货时出示工牌。

**Authorization 头的结构**：

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
               ↑      ↑
            类型     实际的 token 字符串
```

| 部分 | 含义 |
|------|------|
| `Bearer` | 认证类型，表示「持有者令牌」。读作「贝尔乐」，不是「熊」🐻 |
| `eyJhbG...` | JWT（JSON Web Token），一串被加密的 JSON 数据 |

**完整请求流**：

```
浏览器：
  POST /api/posts
  Headers: {
    Content-Type: application/json
    Authorization: Bearer eyJhbG...
  }
  Body: {
    title: "我的第一篇文章",
    content: "这是正文内容...",
    summary: "摘要"
  }
        ↓
后端：
  1. 从 Authorization 头取出 token
  2. 解密 token → 拿到用户 ID
  3. 用用户 ID 创建文章
  4. 返回 { id: 8, title: "我的第一篇文章", ... }
        ↓
浏览器：
  收到新文章数据 → 跳转首页
```

> 📌 **术语标记**：「Authorization 头」「Bearer Token」将收录于术语附录。

### 步骤2：未登录拦截——路由守卫

#### 我在做什么？

`/create` 页面需要登录才能访问。如果用户直接输入 URL 访问，应该被自动重定向到登录页。

**比喻**：电影院里的 VIP 厅——不是谁都能进，检票口会拦住你。

**修改 `src/router/index.js`**，在路由表后面添加导航守卫：

```js
// ... 前面的 import 和 routes 不变 ...

const router = createRouter({
  history: createWebHistory(),
  routes,
})

// 全局前置守卫：每次路由跳转前都会执行
router.beforeEach((to, from, next) => {
  // 需要登录才能访问的路由
  const authRequired = ['/create', '/edit']  // 后续加 /edit

  // 判断目标路由是否需要登录
  // to.path 是完整路径，如 '/create'；to.matched 是匹配到的路由记录
  const needAuth = authRequired.some(path => to.path.startsWith(path))

  if (needAuth) {
    const token = localStorage.getItem('token')
    if (token) {
      next()        // 有 token，放行
    } else {
      next('/login') // 没 token，去登录
    }
  } else {
    next()           // 不需要登录的路由，直接放行
  }
})

export default router
```

#### 代码逐行解释

| 代码 | 含义 |
|------|------|
| `router.beforeEach((to, from, next) => { ... })` | 注册全局前置守卫，每次路由跳转前执行 |
| `to` | 目标路由对象（你要去哪） |
| `from` | 来源路由对象（你从哪来） |
| `next()` | 放行，允许跳转 |
| `next('/login')` | 不放行，改去登录页 |
| `to.path.startsWith(path)` | 判断路径是否以 `/create` 或 `/edit` 开头 |

**守卫执行流程**：

```
用户点击「写文章」
    ↓
router.beforeEach 检查
    ↓
localStorage 有 token?
    ├── 是 → next() → 正常进入 /create
    └── 否 → next('/login') → 跳转到登录页
```

### 步骤3：实现发布文章页面

#### 我在做什么？

创建发布表单，包含标题、内容、摘要三个字段，提交时携带 token。

**修改 `src/views/CreatePost.vue`**：

```vue
<template>
  <div class="create-post">
    <h1>✏️ 发布文章</h1>

    <form @submit.prevent="handleCreate" class="form">
      <!-- 标题 -->
      <div class="form-group">
        <label>标题 <span class="required">*</span></label>
        <input
          v-model="title"
          type="text"
          placeholder="请输入文章标题"
          maxlength="100"
        />
        <span class="char-count">{{ title.length }}/100</span>
      </div>

      <!-- 摘要 -->
      <div class="form-group">
        <label>摘要</label>
        <textarea
          v-model="summary"
          rows="3"
          placeholder="文章摘要（可选，不填则自动截取正文前100字）"
          maxlength="200"
        ></textarea>
        <span class="char-count">{{ summary.length }}/200</span>
      </div>

      <!-- 正文 -->
      <div class="form-group">
        <label>正文 <span class="required">*</span></label>
        <textarea
          v-model="content"
          rows="12"
          placeholder="请输入文章正文（至少10个字符）"
        ></textarea>
        <span class="char-count">{{ content.length }} 字</span>
      </div>

      <!-- 错误提示 -->
      <p v-if="errorMsg" class="error">{{ errorMsg }}</p>

      <!-- 按钮 -->
      <div class="form-actions">
        <button type="submit" :disabled="loading">
          {{ loading ? '发布中...' : '📮 发布文章' }}
        </button>
        <router-link to="/" class="cancel-btn">取消</router-link>
      </div>
    </form>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { useRouter } from 'vue-router'
import { createPost } from '../api/index.js'

const router = useRouter()
const title = ref('')
const summary = ref('')
const content = ref('')
const errorMsg = ref('')
const loading = ref(false)

async function handleCreate() {
  errorMsg.value = ''

  // 1. 表单验证
  if (!title.value.trim()) {
    errorMsg.value = '标题不能为空'
    return
  }
  if (content.value.trim().length < 10) {
    errorMsg.value = '正文至少需要10个字符'
    return
  }

  // 2. 获取 token
  const token = localStorage.getItem('token')
  if (!token) {
    errorMsg.value = '请先登录'
    router.push('/login')
    return
  }

  // 3. 发送请求
  loading.value = true
  try {
    await createPost(
      {
        title: title.value.trim(),
        content: content.value.trim(),
        summary: summary.value.trim() || content.value.trim().slice(0, 100),
      },
      token
    )
    // 4. 成功 → 跳转首页
    router.push('/')
  } catch (err) {
    errorMsg.value = err.message || '发布失败，请稍后重试'
  } finally {
    loading.value = false
  }
}
</script>
```

#### 代码逐行解释

| 代码 | 含义 |
|------|------|
| `title.value.trim()` | 去掉首尾空格后再判断 |
| `content.value.trim().length < 10` | 正文至少10个有效字符 |
| `summary.value.trim() \|\| content.value.trim().slice(0, 100)` | 如果没填摘要，自动截取正文前100字 |
| `createPost({...}, token)` | 调用 API，传入文章数据和 token |
| `maxlength="100"` | HTML 原生属性，限制最大输入长度 |

#### 表单验证逻辑

```
用户点击「发布」
    ↓
标题为空？ ──是──→ 显示「标题不能为空」，停止
    ↓ 否
正文字数 < 10？ ──是──→ 显示「正文至少需要10个字符」，停止
    ↓ 否
有 token？ ──否──→ 显示「请先登录」，跳转登录页
    ↓ 是
发送 POST 请求
    ↓
成功 → 跳转首页
失败 → 显示错误信息
```

❌ **错误**：不验证直接发请求
```js
async function handleCreate() {
  await createPost({ title, content, summary }, token) // 标题为空也发过去了！
  router.push('/')
}
```

✅ **正确**：先验证，再发请求（见上面完整代码）

🤔 **想多一点**：为什么前端验证了，后端还要再验证一次？因为前端验证可以被绕过——任何人都能用浏览器开发者工具修改 JavaScript 代码，或者直接用 `curl` 发请求。前端验证是「礼貌的提醒」，后端验证是「真正的安检」。双重验证才是安全之道。

| 控制台报网络错误 | 后端没启动 | 确保后端/JSON Server 在运行 |

## 八、已知坑点与禁止事项

- ⚠️ **`Authorization` 头拼写容易错**：是 `Authorization`，不是 `Authorisation`（英式拼写）或 `Authorizaton`（少一个 i）。
- ⚠️ **Bearer 后面有空格**：`Bearer eyJhbG...`，不是 `BearereyJhbG...`。少了空格后端解析会失败。
- ⚠️ **`beforeEach` 中的 `next()` 必须调用**：如果不调用 `next()`，路由跳转会永远卡住，页面不动。
- ⚠️ **前端验证不能替代后端验证**：前端验证只是用户体验优化，安全必须靠后端。
