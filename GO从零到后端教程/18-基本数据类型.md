# 第18章 · 基本数据类型

> "程序就是数据的加工厂。你首先要搞清楚工厂里能处理什么样的原材料——数字、文字、真假判断。Go的数据类型不多，但每个都有它的脾气。"

---

## 18.1 bool：真与假，仅此而已

`bool` 是Go里最简单的类型——只有两个值：`true` 和 `false`。

```go
var isLoggedIn bool = true
var hasPermission bool = false

isAdmin := true
```

Go的bool有几个"硬"规则：

**1. bool只有true和false，没有中间状态。**

不像某些语言（C用0表示false，非0表示true；JavaScript有truthy/falsy的概念），Go里 `bool` 就是 `true` 和 `false`，一清二楚。

```go
if 1 {         // ❌ 编译错误！1不是bool
    fmt.Println("Go不允许这样")
}
```

你必须写：

```go
if 1 > 0 {     // ✅ 1>0 的结果是bool
    fmt.Println("Go要求显式的比较")
}
```

**2. bool不能和其他类型互相转换。**

```go
var b bool = true
var i int = int(b)   // ❌ 编译错误！不能把bool转成int
var b2 bool = bool(0) // ❌ 编译错误！不能把int转成bool
```

**3. bool的零值是false。**

```go
var flag bool
fmt.Println(flag)  // false
```

**4. 逻辑运算的结果是bool：**

```go
result := (5 > 3) && (2 < 4)  // true && true = true
```

`&&`（与）、`||`（或）、`!`（非）的运算结果都是 `bool` 类型。

## 18.2 整数全家福：一大家子

Go的整数类型可能是所有语言里最"齐全"的——有11种。

### 有符号整数（能表示负数）

| 类型 | 范围 | 占用字节 |
|------|------|----------|
| `int8` | -128 ~ 127 | 1字节 |
| `int16` | -32768 ~ 32767 | 2字节 |
| `int32` | -2,147,483,648 ~ 2,147,483,647 | 4字节 |
| `int64` | -9,223,372,036,854,775,808 ~ 9,223,372,036,854,775,807 | 8字节 |

### 无符号整数（只能≥0）

| 类型 | 范围 | 占用字节 |
|------|------|----------|
| `uint8` | 0 ~ 255 | 1字节 |
| `uint16` | 0 ~ 65535 | 2字节 |
| `uint32` | 0 ~ 4,294,967,295 | 4字节 |
| `uint64` | 0 ~ 18,446,744,073,709,551,615 | 8字节 |

### 架构相关的整数（最常用）

| 类型 | 说明 |
|------|------|
| `int` | 大小取决于CPU架构（64位系统就是64位，32位就是32位） |
| `uint` | 同上，无符号版本 |

**在64位系统上，`int` 等同于 `int64`，`uint` 等同于 `uint64`。**

### 什么时候用哪个

除非你有明确的理由（比如节省内存、与外部系统对接），否则直接用 `int`。它是Go最常用的整数类型。

```go
age := 25           // 自动推断为 int
count := 1000000    // int
var id int64 = 123456789012345  // 需要存很大数字时用int64
```

### byte和rune：整数的别名

Go里有两个类型别名很常见：

```go
type byte = uint8   // byte就是uint8，1个字节
type rune = int32   // rune就是int32，1个Unicode码点
```

- `byte`：用于处理原始字节数据——比如从文件里读出来的一堆字节，或者网络传输的二进制数据。
- `rune`：用于处理Unicode字符——比如一个中文字符（后面18.5节详细讲）。

### 整数运算的注意事项

**溢出**：超出范围会"绕回来"，不会报错。

```go
var x uint8 = 255
x = x + 1
fmt.Println(x)  // 0（溢出，绕回来了）
```

这不是Go的bug——这是所有语言都有的底层行为。一个字节只有8位，255（`11111111`）加1后变成 `100000000`（9位），最高位被丢掉，剩下 `00000000`，就是0。

**除法**：整数除法小数点后直接丢掉，不是四舍五入。

```go
fmt.Println(5 / 2)   // 2（不是2.5！整数除法截断）
fmt.Println(5.0 / 2) // 2.5（浮点数除法）
```

## 18.3 浮点数：不精确的小数

Go有两种浮点数：

| 类型 | 精度 | 占用字节 |
|------|------|----------|
| `float32` | 约6-7位有效数字 | 4字节 |
| `float64` | 约15-16位有效数字 | 8字节 |

默认的浮点类型是 `float64`（和 `int` 一样是最常用的）：

```go
height := 1.75      // 自动推断为 float64
var weight float32 = 65.5
```

### 精度问题：计算机不会算小数？

这是一个几乎所有编程语言都有的"问题"——浮点数是不精确的。

```go
fmt.Println(0.1 + 0.2)  // 0.30000000000000004
fmt.Println(0.1 + 0.2 == 0.3)  // false！
```

为什么0.1+0.2不等于0.3？这不是Go的bug，而是计算机硬件的根本限制。

### 浮点数为什么不精确

计算机内部用**二进制**存储数字。整数可以用二进制精确表示（比如5 = 101₂），但很多十进制小数在二进制里是**无限循环**的。

就好比在十进制里，1/3 = 0.333333...永远写不完。在二进制里，0.1（1/10）也是无限循环的。

计算机只能存有限位数（float32存32位，float64存64位），超出的部分被截断。所以0.1存在计算机里其实是"0.1的近似值"。

两个近似值加起来，结果当然也是近似的——0.30000000000000004 而不是 0.3。

### 怎么比较浮点数

既然不能直接用 `==`，怎么判断两个浮点数"相等"？

```go
import "math"

// 判断两个浮点数之差的绝对值是否小于一个很小的数
func almostEqual(a, b float64, epsilon float64) bool {
    return math.Abs(a-b) < epsilon  // 需 import "math"（标准库详见第33章）
}

fmt.Println(almostEqual(0.1+0.2, 0.3, 0.0000001))  // true
```

`epsilon`（希腊字母ε）是你能容忍的误差范围。比如0.0000001，只要差值小于这个数，就认为"相等"。

> 🤔 **想多一点**：那为什么银行系统不会算错钱？因为涉及钱的系统**绝对不用浮点数**。银行系统用整数（以"分"为单位）或者专门的定点小数类型。Go标准库里没有定点小数，但有很多第三方库。在你以后写后端处理金额时，记住：钱不要用 `float64` 存。用 `int64` 存"分"（100元存为10000分），或者用专门的decimal库。

## 18.4 字符串：一堆字符？

字符串是Go里最常用的类型之一，但它没有看上去那么简单。

### 双引号和反引号

```go
s1 := "这是普通字符串"
s2 := "包含\n换行\n的字符串"  // \n是换行符
s3 := `这是原始字符串
可以跨多行
里面的\n不会被转义`
```

双引号 `"..."` 支持转义字符（`\n`、`\t`、`\\` 等）。反引号 `` `...` `` 是原始字符串，里面的内容**原样保留**，`\n` 不会被解释为换行。

反引号特别适合写多行文本或正则表达式。

### 字符串是不可变的

```go
s := "hello"
s[0] = 'H'  // ❌ 编译错误！不能修改字符串中的某个字符
```

Go的字符串一旦创建就不能修改。如果你需要修改字符串，可以把它转成 `[]byte` 或 `[]rune`，修改后再转回来：

```go
s := "hello"
b := []byte(s)
b[0] = 'H'
s = string(b)
fmt.Println(s)  // "Hello"
```

### len() 返回的是字节数，不是字符数（重要！）

这是Go初学者最容易踩的坑：

```go
s := "你好"
fmt.Println(len(s))  // 6，不是2！
```

为什么是6不是2？因为 "你好" 这两个中文字符，在UTF-8编码下每个占3个字节。`len()` 数的是**字节数**。

```go
s := "hello你好"
fmt.Println(len(s))           // 11（5个英文字母 + 2个汉字×3字节）
fmt.Println(len([]rune(s)))   // 7（5个英文字母 + 2个汉字，字符数）
```

UTF-8编码规则：
- ASCII字符（英文、数字、符号）：1个字节
- 大部分中文：3个字节
- 少数特殊字符：2~4个字节

> 🤔 **想多一点**：为什么Go不让 `len("你好")` 返回2（字符数）？因为返回字节数更基础——所有字符串操作最终都是字节操作。如果你想知道字符数，用 `len([]rune(s))` 或标准库的 `utf8.RuneCountInString(s)`。Go选择了"让使用者自己决定需要什么"而不是"替使用者猜"。这是Go一贯的哲学。

## 18.5 byte vs rune：一个字节和一个字符

前面提到了 `byte` 和 `rune`，现在深入理解它们的区别。

### byte：一个字节

`byte` 本质上是 `uint8`，代表一个字节（0~255）。它只能表示ASCII字符（英文字母、数字、基本符号）。

```go
var b byte = 'A'
fmt.Println(b)  // 65（输出的是ASCII码，不是字符'A'）
fmt.Printf("%c\n", b)  // A（用%c格式化才能看到字符）
```

### rune：一个Unicode码点

`rune` 本质上是 `int32`，代表一个Unicode码点——"世界上任何一个字符"。

```go
var r rune = '你'
fmt.Println(r)  // 20320（输出的是Unicode码点值）
fmt.Printf("%c\n", r)  // 你
```

### 什么时候用byte，什么时候用rune

- **处理纯英文/二进制数据** → `byte`（HTTP请求体、文件内容、网络数据包）
- **处理中文/多语言文本** → `rune`（用户输入、文章内容、搜索结果）

### 用byte处理字符串 vs 用rune处理字符串

```go
s := "Hello你好"

for i := 0; i < len(s); i++ {
    fmt.Printf("%c ", s[i])
}
// 输出：H e l l o ä ½  å ¥ ½ （后6个字节是乱码！因为按字节遍历，拆散了汉字）

for _, r := range s {
    fmt.Printf("%c ", r)
}
// 输出：H e l l o 你 好 （正确！range自动按rune遍历）
```

**结论：遍历字符串用 `for range`，它会自动按字符（rune）而不是按字节来遍历。**

### byte和rune的底层关系

```go
var b byte = 65
var r rune = 20320

fmt.Println(byte(r))   // 把rune转成byte——可能截断
fmt.Println(rune(b))   // 把byte转成rune——安全
fmt.Println(string(b)) // "A"
fmt.Println(string(r)) // "你"
```

## 18.6 类型转换：Go不帮你猜

很多语言会自动做类型转换（比如JavaScript里 `"5" + 3 = "53"`），Go**绝对不会**。所有类型转换必须**显式**写出。

### 基本类型之间的转换

```go
var i int = 42
var f float64 = float64(i)  // int → float64
var u uint = uint(f)        // float64 → uint

fmt.Println(i, f, u)  // 42 42 42
```

格式很简单：`目标类型(值)`。

### string和其他类型的转换（容易踩坑！）

**int → string 不是你想的那样！**

```go
var i int = 65
s := string(i)
fmt.Println(s)  // "A"，不是"65"！
```

`string(65)` 的意思是"把65这个数字当作Unicode码点/ASCII码，转成对应的字符"。65在ASCII里是 `'A'`，所以结果是 `"A"`。

如果你想把数字转成字符串 `"65"`，必须用 `strconv` 包：

```go
import "strconv"

var i int = 65
s := strconv.Itoa(i)   // "65"
fmt.Println(s)         // "65"
```

`Itoa` = Integer to ASCII。同理，`Atoi` = ASCII to Integer：

```go
s := "42"
i, err := strconv.Atoi(s)
if err != nil {
    fmt.Println("转换失败")
}
fmt.Println(i)  // 42
```

### 类型转换总结表

| 转换目的 | 正确写法 | 错误写法 |
|----------|----------|----------|
| int → float64 | `float64(x)` | 不能隐式转换 |
| float64 → int | `int(x)`（截断小数） | 同上 |
| int → string | `strconv.Itoa(x)` | `string(x)` 变成字符！ |
| string → int | `strconv.Atoi(s)` | 没有直接转换 |
| float → string | `strconv.FormatFloat(x, ...)` | 不能 `string(x)` |
| byte → string | `string(b)` | — |
| rune → string | `string(r)` | — |
| string → []byte | `[]byte(s)` | — |
| string → []rune | `[]rune(s)` | — |

> 🤔 **想多一点**：Go强制显式类型转换，初学者可能觉得麻烦——"为什么不能像JS那样自动转？"答案藏在"隐式类型转换的代价"里。JS里 `[] + {}` 和 `{} + []` 的结果不一样；`"5" - 3` 是2但 `"5" + 3` 是 `"53"`。这些"聪明的自动转换"产生了无数Bug。Go的设计者选择了"让你多写一行，但永远不会猜错"。

## 本章小结

| 知识点 | 核心要点 |
|--------|----------|
| bool | 只有true/false，不能和数字互转，不能用作条件之外的隐式判断 |
| 整数 | 11种类型，最常用 `int`；有符号(int系列)和无符号(uint系列) |
| byte/rune | byte=uint8（字节），rune=int32（Unicode码点），处理中文用rune |
| 浮点数 | float32/float64；不精确！0.1+0.2≠0.3；钱不要用浮点数 |
| 字符串 | UTF-8编码；不可变；len()返回字节数不是字符数 |
| 类型转换 | 必须显式转换；string(65)→"A"不是"65"，正确用strconv.Itoa |
