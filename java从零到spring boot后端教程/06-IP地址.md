# 第06章 · IP地址

> "你要给朋友寄快递，得知道他家的地址。两台计算机要通信，也得知道对方的'地址'——这就是IP地址。"

## 06.1 IP地址是什么

IP地址（Internet Protocol Address）是计算机网络中每台设备的"门牌号"。格式通常是四组数字：

```
192.168.1.100
```

每一组取值范围 0~255（一个字节能表示的范围），所以总共约43亿个地址（IPv4）。

但现在设备太多了，43亿不够用——于是有了IPv6，比如：

```
2001:0db8:85a3:0000:0000:8a2e:0370:7334
```

IPv6有128位，地址数量大到"地球上每一粒沙子都能分配一个IP"。

## 06.2 公网IP vs 内网IP

| | 公网IP | 内网IP |
|---|-------|--------|
| 唯一性 | 全球唯一 | 局域网内唯一 |
| 谁分配 | ISP（电信/联通/移动） | 路由器 |
| 示例 | 12.34.56.78 | 192.168.1.100 |
| 能否从外网访问 | ✅ 可以 | ❌ 不可以（需要端口映射） |
| 费用 | 要钱 | 免费 |

你家里的电脑通常有一个内网IP（如 `192.168.1.100`），这是路由器分配的。当你访问外网（比如百度）时，路由器帮你做 **NAT（网络地址转换）**——把你的内网IP替换成家里的公网IP。

公司服务器通常有公网IP——不然用户怎么访问你们的网站呢？

## 06.3 特殊IP地址

| IP | 含义 | 场景 |
|----|------|------|
| `127.0.0.1` | 本机回环地址（localhost） | 自己访问自己 |
| `0.0.0.0` | 监听所有网络接口 | Spring Boot默认监听地址 |
| `192.168.x.x` | 内网C类地址 | 家庭/办公室局域网 |
| `10.x.x.x` | 内网A类地址 | 大型企业内网 |

## 06.4 Java InetAddress 演示

Java提供了 `java.net.InetAddress` 类来操作IP地址：

```java
import java.net.InetAddress;

public class IPDemo {
    public static void main(String[] args) throws Exception {
        // 获取本机IP
        InetAddress localHost = InetAddress.getLocalHost();
        System.out.println("本机主机名: " + localHost.getHostName());
        System.out.println("本机IP: " + localHost.getHostAddress());

        // 获取回环地址
        InetAddress loopback = InetAddress.getLoopbackAddress();
        System.out.println("回环地址: " + loopback.getHostAddress());  // 127.0.0.1

        // 判断是否回环地址
        System.out.println("127.0.0.1 是回环吗? " +
            InetAddress.getByName("127.0.0.1").isLoopbackAddress());  // true
    }
}
```

编译运行：

```bash
javac IPDemo.java
java IPDemo
```

输出示例：

```
本机主机名: MyComputer
本机IP: 192.168.1.100
回环地址: 127.0.0.1
127.0.0.1 是回环吗? true
```

> 🤔 想多一点：`127.0.0.1` 和 `localhost` 的关系——`localhost` 是一个主机名，默认解析到 `127.0.0.1`。你可以在 `C:\Windows\System32\drivers\etc\hosts`（Windows）或 `/etc/hosts`（Linux/macOS）里看到这个映射。也就是说，`localhost` 本质上是一个写死在本地文件里的DNS记录。第07章我们会详细讲DNS。

## 06.5 IP地址在Spring Boot中的体现

当你启动Spring Boot应用，控制台会打印：

```
Tomcat started on port(s): 8080 (http) with context path ''
```

这里的 "port 8080" 是绑定在哪个IP上的？默认情况下是 `0.0.0.0:8080`——意味着监听本机所有网络接口的8080端口。

你可以指定监听的IP和端口：

```yaml
# application.yml
server:
  address: 127.0.0.1   # 只监听本机，外部无法访问
  port: 8080
```

如果你的服务器有多张网卡（多个IP），`server.address` 可以用来限制服务只监听某个IP。
