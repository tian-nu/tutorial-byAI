## 46.1 树的基本概念

树是一种**层级关系**的数据结构，由节点（Node）和连接节点的边（Edge）组成。

```
         ┌───┐
         │ A │ ← 根节点（Root）
         └─┬─┘
      ┌────┴────┐
    ┌─┴─┐     ┌─┴─┐
    │ B │     │ C │ ← 内部节点
    └─┬─┘     └───┘
   ┌──┴──┐
 ┌─┴─┐ ┌─┴─┐
 │ D │ │ E │ ← 叶子节点（Leaf）
 └───┘ └───┘
```

### 核心术语

| 术语 | 含义 | 图中示例 |
|------|------|---------|
| 根节点（Root） | 最顶层的节点，唯一没有父节点的节点 | A |
| 父节点（Parent） | 直接上层节点 | B的父节点是A |
| 子节点（Child） | 直接下层节点 | A的子节点是B和C |
| 叶子节点（Leaf） | 没有子节点的节点 | D和E |
| 兄弟节点（Sibling） | 同一个父节点的节点 | B和C是兄弟 |
| 深度（Depth） | 从根到该节点的边数 | A深度为0，B深度为1，D深度为2 |
| 高度（Height） | 从该节点到最远叶子的边数 | D高度为0，B高度为1，A高度为2 |
| 度（Degree） | 一个节点的子节点数量 | A的度为2，B的度为2，C的度为0 |

### 二叉树

每个节点最多只有两个子节点的树叫二叉树。Go中这样定义：

```go
type TreeNode struct {
    Val   int
    Left  *TreeNode
    Right *TreeNode
}
```

```
        [5]
       /   \
     [3]   [8]
     / \   / \
   [1] [4][7] [9]
```

二叉树的每个节点最多有两个分支——左子节点和右子节点。这是最简形式也是最重要的树形结构。

### 满二叉树 vs 完全二叉树

**满二叉树**：每一层的节点数都达到最大值。第i层有2^i个节点。

```
        [1]
       /   \
     [2]   [3]
    /  \   /  \
   [4][5][6] [7]    ← 满二叉树：每层都满了
```

**完全二叉树**：除了最后一层外，每层都满；最后一层的节点都靠左排列。

```
        [1]
       /   \
     [2]   [3]
    /  \   /
   [4][5][6]      ← 完全二叉树：最后一层没满，但靠左
```

完全二叉树是堆数据结构的基础——第47章会用到。

## 46.2 二叉搜索树（BST）

二叉搜索树（Binary Search Tree）是二叉树加上一条规则：

**左子节点 < 父节点 < 右子节点**（所有左子树的值都小于根，所有右子树的值都大于根）

```
        [8]
       /   \
     [3]   [10]
     / \      \
   [1] [6]   [14]
       / \    /
      [4][7] [13]
```

验证一下：每条路径上，左<根<右：
- 3 < 8 < 10 ✓
- 1 < 3 < 6 ✓
- 4 < 6 < 7 ✓
- (nil) < 10 < 14 ✓
- 13 < 14 < (nil) ✓

### BST查找：O(log n)

从根节点开始，每次比较后只走一个分支：

```go
func search(root *TreeNode, target int) *TreeNode {
    if root == nil || root.Val == target {
        return root
    }
    if target < root.Val {
        return search(root.Left, target)
    }
    return search(root.Right, target)
}
```

```
查找 7：
[8] → 7 < 8，走左边 → [3] → 7 > 3，走右边 → [6] → 7 > 6，走右边 → [7] 找到了！
```

**比喻**：BST就像一本"猜数字"游戏书。你想的是7，书翻开中间一页写着"8"。你翻到"8"左边那半本继续找——这次看到"3"；又翻到"3"右边，看到"6"；再翻到"6"右边，找到"7"。每一步都排除一半的可能。

### BST插入：O(log n)

插入逻辑跟查找一样——先找到应该放的位置，然后挂上去。

```go
func insert(root *TreeNode, val int) *TreeNode {
    if root == nil {
        return &TreeNode{Val: val}
    }
    if val < root.Val {
        root.Left = insert(root.Left, val)
    } else {
        root.Right = insert(root.Right, val)
    }
    return root
}
```

递归到nil时创建新节点，然后回溯时将新节点挂到父节点上。

```
插入 5：
[8] → 5 < 8 → [3] → 5 > 3 → [6] → 5 < 6 → nil → 创建 [5]，挂在 [6] 左边
```

### BST删除：三种情况

删除节点是最复杂的操作，分三种情况：

**情况1：删除叶子节点**——直接删掉，父节点的对应指针置nil。

**情况2：删除只有一个子节点的节点**——让子节点直接替代被删节点。

**情况3：删除有两个子节点的节点**——找到"中序后继"（右子树的最小值），用它的值替换被删节点，然后删除那个后继节点。

```go
func deleteNode(root *TreeNode, key int) *TreeNode {
    if root == nil {
        return nil
    }
    if key < root.Val {
        root.Left = deleteNode(root.Left, key)
    } else if key > root.Val {
        root.Right = deleteNode(root.Right, key)
    } else {
        if root.Left == nil {
            return root.Right
        }
        if root.Right == nil {
            return root.Left
        }
        minNode := findMin(root.Right)
        root.Val = minNode.Val
        root.Right = deleteNode(root.Right, minNode.Val)
    }
    return root
}

func findMin(root *TreeNode) *TreeNode {
    for root.Left != nil {
        root = root.Left
    }
    return root
}
```

**"中序后继"原理**：删除有两个孩子的节点时，不能随便选一边让孩子上位——那样会破坏BST的性质。中序后继（右子树最小的节点）是唯一能保持BST性质不变的选择。它一定大于所有左子树的值，且一定小于右子树其他所有值。

### BST的退化问题

BST在理想情况下是O(log n)。但如果插入的顺序不好——比如按1, 2, 3, 4, 5的顺序插入：

```
[1]
  \
  [2]
    \
    [3]
      \
      [4]
        \
        [5]    ← 退化成了链表！
```

查找从O(log n)退化成了O(n)。这不是树的错——是插入顺序的错。

解决方案：**平衡树**。

## 46.3 二叉树的遍历

遍历就是"把树中所有节点访问一遍"。四种遍历方式各有各的用途。

### 前序遍历：根 → 左 → 右

先访问根，再递归访问左子树，最后递归访问右子树。

```
        [1]
       /   \
     [2]   [3]
     / \
   [4] [5]

前序遍历结果：1 → 2 → 4 → 5 → 3
```

**递归实现**：

```go
func preorder(root *TreeNode) {
    if root == nil {
        return
    }
    fmt.Print(root.Val, " ")
    preorder(root.Left)
    preorder(root.Right)
}
```

**非递归实现（用栈模拟）**：

```go
func preorderIterative(root *TreeNode) []int {
    if root == nil {
        return nil
    }
    result := []int{}
    stack := []*TreeNode{root}
    for len(stack) > 0 {
        node := stack[len(stack)-1]
        stack = stack[:len(stack)-1]
        result = append(result, node.Val)
        if node.Right != nil {
            stack = append(stack, node.Right)
        }
        if node.Left != nil {
            stack = append(stack, node.Left)
        }
    }
    return result
}
```

注意：先压右子节点再压左子节点——因为栈是后进先出，这样左子节点会先被弹出（符合根→左→右的顺序）。

### 中序遍历：左 → 根 → 右

先递归访问左子树，再访问根，最后递归访问右子树。

```
中序遍历结果：4 → 2 → 5 → 1 → 3
```

**递归实现**：

```go
func inorder(root *TreeNode) {
    if root == nil {
        return
    }
    inorder(root.Left)
    fmt.Print(root.Val, " ")
    inorder(root.Right)
}
```

**非递归实现**：

```go
func inorderIterative(root *TreeNode) []int {
    result := []int{}
    stack := []*TreeNode{}
    cur := root
    for cur != nil || len(stack) > 0 {
        for cur != nil {
            stack = append(stack, cur)
            cur = cur.Left
        }
        cur = stack[len(stack)-1]
        stack = stack[:len(stack)-1]
        result = append(result, cur.Val)
        cur = cur.Right
    }
    return result
}
```

非递归中序的核心思路：一路向左走到底，边走边压栈；到底后弹栈→访问节点→走右边；重复。

**BST中序遍历的重要性质：对BST做中序遍历，结果是有序的！**

```
        [8]
       /   \
     [3]   [10]
     / \      \
   [1] [6]   [14]

中序遍历：1 → 3 → 6 → 8 → 10 → 14 （升序！）
```

这就是BST的妙处——中序遍历天然给出了排序结果。如果你想得到一个有序序列，不需要再调用`sort.Ints()`，直接中序遍历BST就行。

### 后序遍历：左 → 右 → 根

先递归访问左子树，再递归访问右子树，最后访问根。

```
后序遍历结果：4 → 5 → 2 → 3 → 1
```

**递归实现**：

```go
func postorder(root *TreeNode) {
    if root == nil {
        return
    }
    postorder(root.Left)
    postorder(root.Right)
    fmt.Print(root.Val, " ")
}
```

**非递归实现（双栈法）**：

```go
func postorderIterative(root *TreeNode) []int {
    if root == nil {
        return nil
    }
    result := []int{}
    stack1 := []*TreeNode{root}

    for len(stack1) > 0 {
        node := stack1[len(stack1)-1]
        stack1 = stack1[:len(stack1)-1]
        result = append([]int{node.Val}, result...)
        if node.Left != nil {
            stack1 = append(stack1, node.Left)
        }
        if node.Right != nil {
            stack1 = append(stack1, node.Right)
        }
    }
    return result
}
```

双栈法的妙处：第一个栈做"根→左→右"（其实就是前序遍历变种），输出插入到结果的头部，就变成了"左→右→根"（后序遍历）。

**后序遍历的应用**：删除一棵树。你必须先删除子节点再删除父节点——否则父节点删了，子节点找不到了。

### 层序遍历（广度优先）

一层一层地遍历——先根节点，再第一层，再第二层……

```
层序遍历结果：1 → 2 → 3 → 4 → 5
```

**实现（用队列）**：

```go
func levelOrder(root *TreeNode) [][]int {
    if root == nil {
        return nil
    }
    result := [][]int{}
    queue := []*TreeNode{root}

    for len(queue) > 0 {
        levelSize := len(queue)
        level := []int{}
        for i := 0; i < levelSize; i++ {
            node := queue[0]
            queue = queue[1:]
            level = append(level, node.Val)
            if node.Left != nil {
                queue = append(queue, node.Left)
            }
            if node.Right != nil {
                queue = append(queue, node.Right)
            }
        }
        result = append(result, level)
    }
    return result
}
```

```
输入树：       输出：
        [1]     [[1],
       /   \     [2,3],
     [2]   [3]   [4,5,6,7]]
     / \   / \
   [4][5][6][7]
```

层序遍历用队列，而不是栈——这是BFS（广度优先搜索）在树上的应用。第49章会进一步展开。

### 四种遍历对比

| 遍历方式 | 顺序 | 典型应用 |
|---------|------|---------|
| 前序 | 根→左→右 | 复制树、序列化、前缀表达式 |
| 中序 | 左→根→右 | BST得到有序序列 |
| 后序 | 左→右→根 | 删除树、后缀表达式、计算目录大小 |
| 层序 | 逐层 | 打印树形结构、BFS基础 |

## 46.4 平衡树：防止退化

### BST退化的后果

当BST退化成链表时，查找从O(log n)变成O(n)。对于有100万个节点的树，这可能导致查询从20步变成100万步——差距巨大。

**平衡树的目标**：始终保持树的高度在O(log n)级别，不允许树长成"长棍"。

### AVL树

AVL树是最早的自平衡二叉搜索树（1962年）。它要求**任何节点的左右子树高度差不超过1**。

```
        [8]          高度:左=2 右=1 差=1 ✓
       /   \
     [3]   [10]      高度:左=1 右=0 差=1 ✓
     / \
   [1] [6]           高度:左=0 右=0 差=0 ✓
```

AVL树通过**旋转**来修正不平衡：

```
右旋（RR不平衡）：
      [3]              [2]
      /                / \
    [2]      →       [1] [3]
    /
  [1]

左旋（LL不平衡）：
    [1]                [2]
      \                / \
      [2]     →     [1] [3]
        \
        [3]
```

**四种旋转场景**：
- **LL**（左子树的左边插入）：一次右旋
- **RR**（右子树的右边插入）：一次左旋
- **LR**（左子树的右边插入）：先左旋再右旋
- **RL**（右子树的左边插入）：先右旋再左旋

AVL树的查找、插入、删除都是O(log n)——但它严格的高度平衡也意味着插入和删除时需要较多的旋转操作。

### 红黑树

红黑树是另一种自平衡二叉搜索树，比AVL更"松弛"——不要求严格的高度平衡，而是用**颜色规则**来保证大致平衡。

**红黑树的五条规则**：
1. 每个节点是红色或黑色
2. 根节点是黑色
3. 每个叶子（NIL）是黑色
4. 红色节点的两个子节点必须都是黑色（不能有连续的红色）
5. 从任一节点到其每个叶子的所有路径包含相同数量的黑色节点

```
          [8B]              B=黑色, R=红色
         /     \
       [3R]   [10B]
       /  \       \
     [1B] [6B]   [14R]
                   /
                [13B]
```

红黑树通过变色和旋转来维护上述规则。虽然它不像AVL那么严格平衡，但插入删除的开销更小——因此在实际工程中更常被使用。

**Go的`map`扩容决策参考了红黑树的思想**（虽然Go map本身是哈希表），Java的`TreeMap`、C++的`std::map`底层都是红黑树。

### 其他平衡树简介

| 树类型 | 特点 | 应用 |
|--------|------|------|
| B树 | 每个节点可以有多个子节点，高度极小 | 数据库索引、文件系统 |
| B+树 | B树变种，数据只存叶子节点 | MySQL InnoDB、MongoDB |
| 跳表 | 用多层链表实现快速查找 | Redis的有序集合 |
| Trie | 按字符分叉，前缀搜索极快 | 搜索引擎自动补全、IP路由 |

## 🤔 想多一点：树为什么无处不在？

从计算机科学的角度看，树之所以如此普遍，是因为**分治思想天然形成树形结构**。

- **目录结构是树**：`/home/user/projects/`——每个文件夹是节点，里面的文件/子文件夹是子节点
- **DOM是树**：`<html>`包含`<head>`和`<body>`，`<body>`包含`<div>`……
- **AST（抽象语法树）是树**：编译器把你的Go代码解析成树形结构，每个节点代表一个语法单元
- **数据库索引是树**：B+树让你在10亿条记录中查一条只需3~4次磁盘IO
- **路由表是树**：Trie树匹配URL前缀，决定请求发给哪个handler

树是一种**分治思维的数据结构体现**。当你需要"分而治之"时，树就是最自然的表达方式。理解了树，你就不只是理解了数据结构——你是理解了计算机科学中的一种核心思维模式。
