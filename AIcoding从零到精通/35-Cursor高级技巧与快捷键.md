## 35.1 常用快捷键速查表

> 以下快捷键以 **Windows** 为例。Mac用户把 `Ctrl` 换成 `Command` 即可。

| 快捷键 | 功能 | 一句话说明 |
|--------|------|-----------|
| `Ctrl + K` | 行内AI（In-line Chat） | 在代码里直接和AI对话，原地生成/修改代码 |
| `Ctrl + L` | 打开Chat面板 | 右侧和AI聊天对话 |
| `Ctrl + I` | 打开Composer面板 | 批量生成/修改多个文件 |
| `Ctrl + Shift + I` | 打开Composer（Agent模式） | 直接进入Agent自动驾驶模式 |
| `Tab` | 接受AI补全 | 灰色补全变真正的代码 |
| `Esc` | 拒绝AI补全 | 灰色补全消失 |
| `Ctrl + →` | 部分接受补全 | 一个字一个字接受AI的补全 |
| `Ctrl + Shift + L` | 打开Chat（带选中代码） | 先选中代码，再用这个快捷键，AI自动看到选中内容 |
| `Ctrl + Shift + K` | 行内编辑（带选中代码） | 选中代码后按，在行内直接问AI怎么改 |
| `Ctrl + Enter` | 在Chat里快速发送 | 不用鼠标点发送按钮 |
| `Ctrl + Backtick` | 打开终端 | 快速打开终端运行命令 |
| `Ctrl + Shift + F` | 全局搜索 | 在所有文件中搜索关键词 |
| `Ctrl + P` | 快速打开文件 | 输入文件名快速跳转 |

> **记忆技巧**：K（Kode）= 写代码，L（Language）= 聊天对话，I（Insert?）= 批量插入。

## 35.2 In-line Chat（Ctrl+K）：在代码里直接和AI对话

第28章我们学了Chat模式（Ctrl+L，在右侧面板），但还有一个更快的办法：**In-line Chat**。

### 怎么用：

**1.** 在编辑区，光标放到你想改代码的地方

**2.** 按 `Ctrl + K`，一个输入框在编辑区中间弹出

**3.** 输入你对AI的指令，比如："把这个for循环改成列表推导式"

**4.** 按回车，AI直接在原地生成修改后的代码

**In-line Chat vs 右侧Chat**：
- In-line Chat：快、就地修改、适合改一小段代码
- 右侧Chat：方便、能看上下文、适合复杂问题

## 35.3 多模型切换：GPT-4o vs Claude vs DeepSeek V3

Cursor支持多种模型，不同模型有不同特长：

| 模型 | 特长 | 适合场景 | 费用 |
|------|------|---------|------|
| **GPT-4o** | 速度快、通用型强、理解准 | ⭐ 日常主力，90%的场景 | 高级模型，有额度限制 |
| **Claude 3.5 Sonnet** | 超长上下文、代码风格好 | 大文件重构、看几千行代码 | 高级模型，有额度限制 |
| **DeepSeek V3** | 便宜、中文好、代码能力不错 | 省钱、中文场景、简单任务 | 便宜/免费 |

> **📖 术语：DeepSeek V3**——中国深度求索公司出品的AI模型。特点是**性价比极高**，中文理解能力出色，代码能力在开源模型中名列前茅。如果Cursor支持DeepSeek，日常简单任务用它可以省额度。

### 怎么切换模型：

在Chat或Composer的输入框上方，有个模型选择下拉菜单，点击切换。

**本书推荐策略**：
- 90%日常 → GPT-4o
- 大文件 → Claude 3.5
- 省钱/简单任务 → DeepSeek V3（如果Cursor支持）

## 35.4 Bug Finder：自动扫描代码问题

Cursor有一个自动诊断功能，能发现代码中的潜在问题。

### 怎么用：

**1.** 打开一个代码文件

**2.** 右侧Chat面板里输入：

```
帮我检查这个文件有没有Bug
```

**3.** AI会逐个扫描，列出：
- 潜在的空指针
- 未处理的异常
- 逻辑错误
- 代码风格问题

**4.** 对于每个问题，AI会给出修复建议

### 更高效的做法：

在Chat里输入：

```
@file calculator.py 帮我检查这个文件，找出所有Bug和潜在问题，按严重程度排序
```

## 35.5 终端AI：终端报错了直接问AI

在Cursor的终端里运行代码，报错了怎么办？

**两种方式**：

### 方式1：复制粘贴（通用）

1. 选中终端里的错误信息
2. 按 `Ctrl + Shift + L`（打开Chat，自动带上选中的错误信息）
3. 输入："这个报错是什么意思？怎么修？"

### 方式2：直接Ctrl+K（终端里也可以）

1. 光标点击终端
2. 按 `Ctrl + K`
3. 输入你的问题
4. AI直接在终端层面帮你分析

## 35.6 最佳实践总结：什么时候用什么？

这是Cursor篇最重要的一张表——**四大模式的决策指南**：

| 场景 | 用什么 | 快捷键 |
|------|--------|--------|
| 继续写一行代码 | **Tab补全** | 打字自动触发 |
| 改一小段代码（几行） | **In-line Chat** | `Ctrl + K` |
| 问一个问题、解释代码、改一个函数 | **Chat** | `Ctrl + L` |
| 从零写多个新文件 | **Composer（Normal）** | `Ctrl + I` |
| 给大型项目加功能、重构 | **Agent** | `Ctrl + Shift + I` |
| 想先看AI计划再决定 | **Composer（Plan）** | `Ctrl + I` 然后选Plan |
| 终端报错了不知道怎么修 | **终端AI** | 复制错误 → `Ctrl + L` |
| 想统一AI的编码风格 | **.cursorrules** | 手动创建文件 |

### 补充三条黄金经验：

1. **能Tab就Tab**：Tab补全最快，能用Tab就别开Chat——少打字，多产出。

2. **先跑通，再优化**：AI写的代码不一定完美，但大概率能跑。先让它跑起来，再让它优化。不要第一次就追求完美。

3. **出错是常态，迭代是方法**：AI写的代码第一次运行报错是正常的。把错误贴给AI，让它修复，反复几次直到成功——这就是AI编程的日常。

## 35.7 如果出错了怎么办

### 🔴 问题1：快捷键不生效

**可能原因**：输入法冲突、系统快捷键冲突、或者你记错了键。

**解决**：
1. 切换到英文输入法再试
2. 检查Cursor设置里的键盘快捷键（文件 → 首选项 → 键盘快捷方式）
3. 搜索"Cursor快捷键速查"对照确认

### 🔴 问题2：Bug Finder说"没找到问题"

**可能原因**：代码确实没问题，或者AI没有深入分析。

**解决**：更具体地描述你要检查什么。比如不要只说"检查Bug"，而是说"检查第30-50行的循环逻辑，有没有边界条件错误？"

### 🔴 问题3：DeepSeek V3不在模型列表里

**说明**：Cursor支持的模型列表会定期更新。如果DeepSeek V3不在列表里，就用GPT-4o-mini代替（免费版无限额度）。
