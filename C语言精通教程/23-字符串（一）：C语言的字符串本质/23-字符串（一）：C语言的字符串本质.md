## 我在做什么？

如果你从 Python 或 Java 转过来，你对字符串的印象可能是：`str = "hello"`，然后就能用 `.length()`、`.substring()`、`.split()` 等一堆方法。

但 C 语言没有"字符串类型"。

C 语言里的字符串，本质上就是一个**以 `\0` 结尾的 char 数组**。没有方法，没有对象，没有自动扩容——一切都靠你自己管理。读完这一章，你能理解 C 语言字符串的底层本质，区分 `char[]` 和 `char*` 的关键差异，以及正确使用 `strlen` 和 `sizeof`。

## 一、C语言没有字符串类型

### 1.1 一个思想实验

打开 Python 解释器，你可以这样：

```python
s = "hello"
print(len(s))   # 5
print(s[0])     # 'h'
```

Python 的 `str` 是一个完整的类型，自带长度信息、自带方法、自带内存管理。

C 语言不是这样。C 语言诞生于 1972 年，设计哲学是"简单到极致"。它没有内置字符串类型——因为字符串不过是一串字符，用字符数组就能表示，何必再发明一个新类型？

### 1.2 字符串就是 char 数组

```c
char str[6] = {'h', 'e', 'l', 'l', 'o', '\0'};
```

这就是 C 语言中的字符串——一个 `char` 数组，最后一个字符必须是 `\0`（空字符）。

```text
内存中的样子：
┌───┬───┬───┬───┬───┬───┐
│ h │ e │ l │ l │ o │ \0│
└───┴───┴───┴───┴───┴───┘
  [0] [1] [2] [3] [4] [5]
```

`\0` 是一个特殊字符，ASCII 码为 0。它不是数字 `0`，也不是字母 `O`。它是 C 语言识别字符串结束的唯一标志。

## 二、空字符 `\0`：字符串的"终点线"

### 2.1 为什么需要 `\0`

数组只有起始地址，没有长度信息。当 `printf` 打印一个字符串时，它怎么知道该停在哪里？

答案：**它从起始地址开始，逐个字符打印，直到遇到 `\0` 为止。**

```text
printf 看到 str：
┌───┬───┬───┬───┬───┬───┬───┬───┐
│ h │ e │ l │ l │ o │\0 │ ? │ ? │
└───┴───┴───┴───┴───┴───┴───┴───┘
  ↑                   ↑
 从这里开始          遇到 \0，停止
```

### 2.2 没有 `\0` 会怎样？

```c
#include <stdio.h>

int main() {
    char str[5] = {'h', 'e', 'l', 'l', 'o'};  // 没有 \0！
    printf("%s\n", str);  // 危险！
    return 0;
}
```

这个程序会打印出 `hello` 后面跟一串乱码，因为 `printf` 会一直往后读内存，直到碰巧遇到一个 `\0` 才停。如果它读到了操作系统保护的内存区域，就会触发 *段错误（Segmentation Fault）* *此术语见附录F*。

```text
没有 \0 时 printf 的行为：
┌───┬───┬───┬───┬───┬──────┬──────┬──────┐
│ h │ e │ l │ l │ o │ 垃圾  │ 垃圾  │ 垃圾  │
└───┴───┴───┴───┴───┴──────┴──────┴──────┘
  ↑                                       ↑
 从这里开始                         一直读，不知道停在哪
```

### 2.3 `\0` 的 ASCII 编码

`\0` 的 ASCII 码是 0（不是字符 `'0'`，后者的 ASCII 码是 48）。

```c
char c1 = '\0';   // ASCII 码 0，空字符
char c2 = '0';    // ASCII 码 48，数字字符 0
char c3 = 0;      // 等价于 '\0'，ASCII 码 0
```

## 三、字符串字面量

### 3.1 双引号自动加 `\0`

当你用双引号写字符串时，编译器自动在末尾加上 `\0`：

```c
char str[] = "hello";  // 编译器自动处理为 {'h','e','l','l','o','\0'}
```

`"hello"` 这种写法叫 *字符串字面量* *此术语见附录F*。编译器在编译时就把它变成了一个以 `\0` 结尾的字符数组。

```text
你写的：        "hello"
编译器看到的：   {'h', 'e', 'l', 'l', 'o', '\0'}
数组长度：      6（不是 5！）
```

### 3.2 验证：数组长度多了 1

```c
#include <stdio.h>

int main() {
    char str[] = "hello";
    printf("sizeof(str) = %zu\n", sizeof(str));  // 输出 6
    printf("str[5] = %d\n", str[5]);             // 输出 0（\0 的 ASCII 码）
    return 0;
}
```

输出：
```text
sizeof(str) = 6
str[5] = 0
```

## 四、`char[]` vs `char*`：关键区别

### 4.1 两种写法，本质不同

```c
char s1[] = "hello";   // 数组：在栈上分配，可修改
char *s2 = "hello";    // 指针：指向只读的字符串字面量
```

这两种写法看起来都能用，但**本质完全不同**。这是 C 语言新手最容易出错的地方之一。

### 4.2 比喻：自己的书 vs 图书馆的书

```text
char s1[] = "hello";
┌─────────────────────────────────┐
│ 栈内存（你的书架）               │
│ ┌───┬───┬───┬───┬───┬───┐       │
│ │ h │ e │ l │ l │ o │ \0│       │
│ └───┴───┴───┴───┴───┴───┘       │
│  你可以随便涂改这本书             │
└─────────────────────────────────┘

char *s2 = "hello";
┌─────────────────────────────────┐
│ 栈内存                            │
│ ┌──────────┐                    │
│ │ s2 (指针) │ ──────────┐        │
│ └──────────┘           │        │
└────────────────────────│────────┘
                         │
                         ▼
┌─────────────────────────────────────────┐
│ 只读数据区（图书馆的书，不能涂改）         │
│ ┌────┬────┬────┬────┬────┬────┐         │
│ │ h  │ e  │ l  │ l  │ o  │ \0 │         │
│ └────┴────┴────┴────┴────┴────┘         │
└─────────────────────────────────────────┘
```

`s1` 拥有自己的副本，可以修改。`s2` 只是一个指针，指向一块只读内存。

### 4.3 修改测试

```c
#include <stdio.h>

int main() {
    char s1[] = "hello";
    s1[0] = 'H';           // 正确：修改自己的副本
    printf("%s\n", s1);    // 输出 Hello

    char *s2 = "hello";
    // s2[0] = 'H';        // 危险！尝试修改只读内存，可能崩溃
    printf("%s\n", s2);

    return 0;
}
```

### 4.4 对比表

| | `char s[] = "hello"` | `char *s = "hello"` |
|---|---|---|
| 本质 | 字符数组 | 指向字符串字面量的指针 |
| 存储位置 | 栈（或静态存储区） | 指针在栈上，内容在只读数据区 |
| 可修改？ | 是 | 否（尝试修改可能导致段错误） |
| 可以重新赋值？ | 否（`s = "world"` 不允许） | 是（`s = "world"` 允许，只是改变指针指向） |
| sizeof | 返回整个数组大小（含 `\0`） | 返回指针大小（通常 8 字节） |
| 数组名 | 是数组首元素地址 | 本身就是指针变量 |

验证 `sizeof` 差异：

```c
#include <stdio.h>

int main() {
    char s1[] = "hello";
    char *s2 = "hello";

    printf("sizeof(s1) = %zu (整个数组)\n", sizeof(s1));  // 输出 6
    printf("sizeof(s2) = %zu (指针大小)\n", sizeof(s2));  // 输出 8（64位系统）
    return 0;
}
```

## 五、`strlen` vs `sizeof`

### 5.1 两个东西，一个容易搞混

- `strlen`：来自 `<string.h>`，返回字符串的**字符个数**（不含 `\0`）。
- `sizeof`：C 语言关键字，返回变量或类型在内存中占用的**字节数**。

```c
#include <stdio.h>
#include <string.h>

int main() {
    char str[] = "hello";

    printf("strlen(str)  = %zu\n", strlen(str));   // 输出 5（字符个数）
    printf("sizeof(str)  = %zu\n", sizeof(str));   // 输出 6（含 \0）

    return 0;
}
```

### 5.2 比喻：火车车厢

```text
"hello" 这列火车：

┌─────┬─────┬─────┬─────┬─────┬──────┐
│  h  │  e  │  l  │  l  │  o  │  \0  │
└─────┴─────┴─────┴─────┴─────┴──────┘
  ←─── strlen = 5 ───→
  ←────── sizeof = 6 ───────→
```

`strlen` 数的是乘客（字符），`sizeof` 数的是所有车厢（包括停在最后的空车厢 `\0`）。

### 5.3 `strlen` 的工作原理

`strlen` 没有魔法——它只是从起始地址开始，一个一个数，直到遇到 `\0`：

```c
// 自己实现 strlen（简化版）
size_t my_strlen(const char *s) {
    size_t count = 0;
    while (s[count] != '\0') {
        count++;
    }
    return count;
}
```

### 5.4 陷阱：`char*` 的 sizeof 不是字符串长度

```c
#include <stdio.h>
#include <string.h>

int main() {
    char *s = "hello world";

    printf("strlen(s)  = %zu\n", strlen(s));   // 输出 11
    printf("sizeof(s) = %zu\n", sizeof(s));    // 输出 8（指针大小！不是 11！）

    return 0;
}
```

这是最常见的错误之一：用 `sizeof` 去求一个 `char*` 指向的字符串长度。`sizeof` 永远返回指针本身的大小（8 字节），和字符串多长无关。

## 六、字符串的输入与输出

### 6.1 用 `printf` 输出

```c
#include <stdio.h>

int main() {
    char name[] = "Alice";
    printf("Hello, %s!\n", name);   // %s 打印字符串
    return 0;
}
```

`%s` 告诉 `printf`："这个参数是一个以 `\0` 结尾的 char 数组"。

### 6.2 用 `scanf` 输入（危险）

```c
#include <stdio.h>

int main() {
    char name[20];
    printf("请输入你的名字：");
    scanf("%s", name);      // 注意：没有 &，数组名本身就是地址
    printf("你好，%s！\n", name);
    return 0;
}
```

`scanf("%s")` 有两个严重问题：
1. 读到空格就停（输入 "Zhang San" 只能得到 "Zhang"）。
2. 没有边界检查——如果用户输入超过 19 个字符，就会越界。

### 6.3 用 `fgets` 安全输入

```c
#include <stdio.h>
#include <string.h>

int main() {
    char name[20];
    printf("请输入你的名字：");
    fgets(name, sizeof(name), stdin);  // 安全：最多读 19 个字符

    // fgets 会保留换行符，去掉它
    name[strcspn(name, "\n")] = '\0';

    printf("你好，%s！\n", name);
    return 0;
}
```

`fgets` 的三个参数：
- `name`：存放输入的目标数组。
- `sizeof(name)`：最多读多少字符（含 `\0`，所以实际最多读 19 个字符）。
- `stdin`：从哪里读（标准输入）。

## 七、常见错误与排错

### 7.1 忘记给 `\0` 留空间

❌ 错误示例：
```c
char str[5] = "hello";  // "hello" 需要 6 个字节（5 字符 + \0），数组只有 5 个
```

编译警告：
```text
error: initializer-string for array of chars is too long
```

✅ 正确示例：
```c
char str[6] = "hello";  // 或 char str[] = "hello"; 让编译器自己算
```

### 7.2 尝试修改字符串字面量

❌ 错误示例：
```c
char *s = "hello";
s[0] = 'H';  // 段错误！
```

✅ 正确示例：
```c
char s[] = "hello";
s[0] = 'H';  // 正确
```

### 7.3 用 sizeof 求 `char*` 的字符串长度

❌ 错误示例：
```c
char *s = "hello";
int len = sizeof(s);  // 总是 8，不是 5
```

✅ 正确示例：
```c
char *s = "hello";
int len = strlen(s);  // 正确：5
```

### 7.4 字符串比较用 `==`

❌ 错误示例：
```c
char s1[] = "hello";
char s2[] = "hello";
if (s1 == s2) {  // 比较的是地址，不是内容！
    printf("相等\n");
}
```

✅ 正确示例（第24章会详细讲）：
```c
if (strcmp(s1, s2) == 0) {
    printf("相等\n");
}
```

## 我做得对不对？

用以下程序验证：

```c
#include <stdio.h>
#include <string.h>

int main() {
    // 测试1：sizeof vs strlen
    char str1[] = "hello";
    printf("测试1：sizeof=%zu (应为 6), strlen=%zu (应为 5)\n",
           sizeof(str1), strlen(str1));

    // 测试2：char[] 可修改
    char str2[] = "hello";
    str2[0] = 'H';
    printf("测试2：%s (应为 Hello)\n", str2);

    // 测试3：char* 指向字面量
    char *str3 = "world";
    printf("测试3：%s (应为 world)\n", str3);

    // 测试4：char* 的 sizeof 是指针大小
    char *str4 = "a very long string";
    printf("测试4：sizeof(str4)=%zu (应为 8，64位系统)\n", sizeof(str4));
    printf("测试4：strlen(str4)=%zu (应为 18)\n", strlen(str4));

    return 0;
}
```

编译运行：
```bash
$ gcc -Wall -o test test.c
$ ./test
测试1：sizeof=6 (应为 6), strlen=5 (应为 5)
测试2：Hello (应为 Hello)
测试3：world (应为 world)
测试4：sizeof(str4)=8 (应为 8，64位系统)
测试4：strlen(str4)=18 (应为 18)
```

## 不对怎么办？

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| 打印字符串后面有乱码 | 缺少 `\0` 结尾 | 确保数组足够大，以容纳 `\0` |
| 修改字符串字面量崩溃 | 用 `char*` 指向了只读内存 | 改为 `char s[] = "..."` |
| `sizeof` 返回 8 而不是字符串长度 | 对 `char*` 使用了 `sizeof` | 用 `strlen` 求字符串长度 |
| `scanf` 输入带空格的字符串不完整 | `%s` 遇到空格就停 | 用 `fgets` 替代 |
| 字符串比较结果不对 | 用了 `==` 比较地址 | 用 `strcmp` 比较内容 |

## 本章小结

| 知识点 | 一句话总结 |
|--------|-----------|
| C 语言没有字符串类型 | 字符串是以 `\0` 结尾的 `char` 数组 |
| `\0`（空字符） | ASCII 码为 0，标记字符串结束 |
| 字符串字面量 | 双引号包裹的字符串，编译器自动加 `\0`，存储在只读区 |
| `char[]` vs `char*` | 数组拥有自己的可修改副本，指针指向只读字面量 |
| `strlen` | 返回字符串字符数（不含 `\0`），需要 `<string.h>` |
| `sizeof` | 返回变量占用的内存字节数，对 `char*` 返回指针大小 |
| `fgets` | 安全的字符串输入函数，有边界检查 |
