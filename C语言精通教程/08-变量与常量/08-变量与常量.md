## 我在做什么？

前两章你写的程序都是"死"的——每次运行，输出完全一样。`printf("Hello, World!\n");` 永远打印那一句话。

真正的程序需要"记东西"。比如一个计算器，你需要记住用户输入的数字；比如一个游戏，你需要记住玩家的分数。这种"记住东西"的能力，在C语言中靠 **变量** 来实现。

这一章，你会亲手创建内存中的"小盒子"（变量），往里放东西，取出来用，还会遇到一个吓人的概念——`*未定义行为（UB）*（此术语见附录F）`。别怕，这是全C语言最重要的安全概念，你迟早要面对它。

## 8.1 变量是什么？

### 8.1.1 内存盒子

想象计算机的内存是一面巨大的墙，墙上有几十亿个编号的小抽屉。每个抽屉有一个地址（编号），可以存放一个数字。

如果你直接跟计算机说："把数字42存到第12345678号抽屉里"，不出五分钟你就疯了。变量就是帮你解决这个问题的——**你给抽屉起一个人类能记住的名字，编译器帮你记住抽屉编号**。

```c
int score = 42;
```

这条语句做了什么？看下图：

```
内存墙（示意）
┌──────┬──────┬──────┬──────┬──────┬──────┐
│  ... │  ... │  ... │  ... │  ... │  ... │
├──────┼──────┼──────┼──────┼──────┼──────┤
│      │      │  42  │      │      │      │
│      │      │  ↑   │      │      │      │
│      │      │score │      │      │      │
└──────┴──────┴──────┴──────┴──────┴──────┘
                  ↑
         地址：0x7fff...（编译器自动管理）
         标签：score（你起的名字）
         内容：42（你存的值）
```

**比喻**：变量就像是你在酒店前台寄存行李。你把行李交给服务员（编译器），服务员给你一个号码牌（变量名）。以后你只要报"score"，服务员就知道去哪取你的行李。你不需要知道行李在哪个柜子里。

### 8.1.2 变量的三个要素

每个变量都有三个属性：

| 属性 | 说明 | 类比 |
|------|------|------|
| 名字 | 你起的标识符 | 号码牌上的编号 |
| 类型 | 决定了能存什么、占多大空间 | 寄存柜的大小（小柜子放不下大箱子） |
| 值 | 当前存放的内容 | 柜子里实际放的东西 |

## 8.2 声明与初始化

### 8.2.1 声明（declaration）

"声明"就是告诉编译器："我要一个变量，名字叫xxx，类型是xxx。"

```c
int score;   /* 声明一个int类型的变量，名为score */
```

此时发生的事情：
1. 编译器在内存中划出一块空间（`int` 通常占4个字节，即4个小抽屉连在一起）。
2. 把变量名 `score` 和这块空间的地址绑定。
3. **但空间里原来有什么，就还是什么**——这就是危险的来源。

### 8.2.2 `*初始化*（此术语见附录F）`（initialization）

"初始化"就是在声明的同时给变量一个初始值。

```c
int score = 0;   /* 声明并初始化为0 */
int age = 25;    /* 声明并初始化为25 */
int x, y, z;     /* 一次声明多个（不建议初学者这样做） */
```

**比喻**：声明是"租了一个柜子"，初始化是"租了柜子并立刻放东西进去"。如果你只声明不初始化，柜子里可能留着上一个租客的臭袜子——你不知道会取出什么。

### 8.2.3 赋值（assignment）

赋值是给**已经存在的变量**一个新的值。

```c
int score;      /* 声明 */
score = 100;    /* 赋值（不是初始化） */
score = 200;    /* 再次赋值，原来的100被覆盖 */
```

注意 `=` 在C语言中不是"等于"，而是"把右边的值放进左边的变量"。

```c
score = score + 10;  /* 把score当前的值加10，再放回score */
```

数学上，`score = score + 10` 是矛盾的（一个数不可能等于自己加10）。但在C语言中，`=` 是赋值动作，意思是"取出来、算一下、放回去"。

## 8.3 未初始化变量的危险（UB）

### 8.3.1 什么是UB？

`*未定义行为（UB）*（此术语见附录F）` 是C语言标准中一个核心概念：**对某些操作，C标准不规定结果是什么。编译器可以生成任何代码，程序可以输出任何东西，甚至崩溃。** 使用未初始化的变量就是典型的UB。

```c
#include <stdio.h>

int main(void)
{
    int x;               /* 声明但未初始化 */
    printf("%d\n", x);   /* 使用未初始化的变量 → UB！ */
    return 0;
}
```

这段代码编译可能不会报错（或者只会给一个警告），但运行结果完全不可预测。你可能看到0，可能看到32767，可能看到-858993460，可能每次运行都不一样。

**为什么？** 因为 `x` 对应的内存空间里，存放的是这块内存在被分配之前的"残留值"——可能是上一个程序留下的数据，可能是系统启动时的随机值。

### 8.3.2 UB的比喻

**比喻**：你去餐厅吃饭，服务员给你一个没有清洗的盘子。盘子里残留着上一位客人吃剩的菜渣。你没洗就盛饭，吃到了别人的剩菜。这就是UB——你不知道会吃到什么。

**正确做法**：永远初始化你的变量。

```c
/* ❌ 危险：未初始化就使用 */
int x;
printf("%d\n", x);   /* UB！ */

/* ✅ 安全：声明时初始化 */
int x = 0;
printf("%d\n", x);   /* 输出 0 */

/* ✅ 也可以：声明后立即赋值 */
int x;
x = 0;
printf("%d\n", x);   /* 输出 0 */
```

**金科玉律**：在C语言中，**声明变量时，立刻给它一个初始值**。养成这个习惯，能避免大量难以排查的bug。

## 8.4 变量命名规范

C语言中，变量名（标识符）有以下规则：

### 8.4.1 硬规则（违反会编译报错）

1. 只能包含字母（A-Z, a-z）、数字（0-9）和下划线（_）。
2. 不能以数字开头。
3. 不能是C语言的关键字（如 `int`、`if`、`return` 等）。
4. 大小写敏感：`Score`、`score`、`SCORE` 是三个不同的变量。

```c
/* ❌ 非法命名 */
int 1st_place;    /* 不能以数字开头 */
int my-name;      /* 连字符-不允许 */
int float;        /* float是关键字 */

/* ✅ 合法命名 */
int first_place;
int my_name;
int myNumber;
int _internal;    /* 下划线开头合法，但通常保留给系统内部使用 */
```

### 8.4.2 软规则（写给人类看的）

1. **名字要有意义**：`int x;` 不如 `int student_count;`。
2. **常见命名风格**：
   - 蛇形命名（snake_case）：`student_count`、`max_value`（推荐）
   - 驼峰命名（camelCase）：`studentCount`、`maxValue`
3. **选一种风格，坚持用到底**。混用会让代码难以阅读。

## 8.5 常量

有些值在程序运行过程中不应该改变。比如圆周率永远是3.14159，一天永远是24小时。C语言提供了两种方式定义常量。

### 8.5.1 `const` 常量

`const` 是一个关键字，它告诉编译器："这个变量一旦初始化，就不能再改。"

```c
const int HOURS_PER_DAY = 24;
HOURS_PER_DAY = 25;   /* 编译报错！const变量不能修改 */
```

**注意**：`const` 常量本质上还是一个变量（占用内存空间），只是在编译期被标记为只读。试图修改它会导致编译错误。

**命名约定**：`const` 常量通常用全大写字母命名，单词间用下划线分隔。这是给读者（人类）的信号："我是常量，别改我。"

### 8.5.2 `#define` `*宏常量*（此术语见附录F）`

`#define` 是预处理指令，它在编译之前进行**纯文本替换**。

```c
#define PI 3.14159
#define MAX_STUDENTS 100

int main(void)
{
    double area = PI * 10 * 10;   /* 预处理后变成 3.14159 * 10 * 10 */
    int scores[MAX_STUDENTS];     /* 预处理后变成 int scores[100]; */
    return 0;
}
```

在编译之前，预处理器会把代码中所有 `PI` 替换为 `3.14159`，所有 `MAX_STUDENTS` 替换为 `100`。**变量名本身不存在于最终的可执行文件中**。

### 8.5.3 `const` vs `#define`：怎么选？

| 比较维度 | `const` | `#define` |
|----------|---------|-----------|
| 本质 | 只读变量 | 文本替换 |
| 类型检查 | 有（编译器检查类型） | 无（纯文本） |
| 调试 | 可以（调试器能看到变量名） | 不可以（已经被替换成字面值） |
| 占用内存 | 是（每个编译单元一份） | 否（不占运行时内存） |
| 作用域 | 有（只在定义它的代码块内有效） | 无（从定义行到文件末尾都有效） |

**建议**：
- 优先使用 `const`（有类型检查，更安全）。
- 数组大小、条件编译等场景使用 `#define`（因为这些必须在编译期确定）。

```c
const double PI = 3.14159;           /* 数值常量，用 const */
#define ARRAY_SIZE 100               /* 数组大小，用 #define */
int data[ARRAY_SIZE];                /* 编译期必须知道大小 */
```

## 8.6 变量与常量的完整示例

```c
#include <stdio.h>

#define PI 3.14159

int main(void)
{
    /* 变量：可以改变 */
    int radius = 5;
    double area = 0.0;

    /* 常量：不能改变 */
    const int MAX_RADIUS = 100;

    /* 计算面积 */
    area = PI * radius * radius;

    printf("半径 = %d\n", radius);
    printf("面积 = %f\n", area);

    /* 修改变量 */
    radius = 10;
    area = PI * radius * radius;
    printf("新半径 = %d, 新面积 = %f\n", radius, area);

    /* MAX_RADIUS = 200;   ← 这行会编译报错！const不能修改 */

    return 0;
}
```

输出：

```
半径 = 5
面积 = 78.539750
新半径 = 10, 新面积 = 314.159000
```

## 8.7 常见错误

### 错误1：使用未初始化的变量

```c
/* ❌ 错误示例 */
int x;
int y = x + 5;   /* x未初始化，结果是UB！ */

/* ✅ 正确示例 */
int x = 0;
int y = x + 5;   /* y = 5 */
```

### 错误2：混淆 `=` 和 `==`

```c
/* ❌ 错误示例 */
int x = 5;
if (x = 10) {    /* 这是赋值，不是比较！x被改成了10 */
    printf("x等于10\n");
}

/* ✅ 正确示例 */
int x = 5;
if (x == 10) {   /* 这才是比较：x等于10吗？ */
    printf("x等于10\n");
}
```

这个错误极其常见，编译器通常不会报错（因为 `x = 10` 是合法的赋值表达式）。第12章会详细讲解。

### 错误3：`#define` 后加分号

```c
/* ❌ 错误示例 */
#define PI 3.14;    /* 分号成了替换内容的一部分！ */

double area = PI * r * r;
/* 预处理后变成：double area = 3.14; * r * r;   ← 语法错误！ */

/* ✅ 正确示例 */
#define PI 3.14     /* 不加分号 */
```

### 错误4：试图修改 `const` 变量

```c
/* ❌ 错误示例 */
const int x = 10;
x = 20;   /* 编译报错：assignment of read-only variable 'x' */
```

### 错误5：变量名以数字开头

```c
/* ❌ 错误示例 */
int 1st_value = 100;   /* 编译报错 */

/* ✅ 正确示例 */
int first_value = 100;
```

## 8.8 我在做什么？做得对不对？不对怎么办？

### 我在做什么？

我在学习如何创建变量（内存中命名的盒子）、如何初始化变量（给盒子放初始值）、理解为什么未初始化变量是危险的（UB），以及如何用 `const` 和 `#define` 定义常量。

### 我做得对不对？

- 你能声明一个变量并立刻初始化 → 声明与初始化过关。
- 你能解释"声明"和"初始化"的区别 → 概念过关。
- 你能说出为什么使用未初始化变量是UB → 安全意识过关。
- 你能区分 `const` 和 `#define` 的使用场景 → 常量过关。

### 不对怎么办？

| 症状 | 第一步检查 |
|------|-----------|
| 变量值每次运行都不同 | 是否忘了初始化？ |
| 程序崩溃或输出奇怪数字 | 是否使用了未初始化的变量？ |
| 编译报错 `expected identifier` | 变量名是否以数字开头？是否用了关键字？ |
| 编译报错 `assignment of read-only` | 是否试图修改 `const` 变量？ |
| `#define` 后出现奇怪的编译错误 | 是否在宏定义末尾加了分号？ |

## 本章小结

| 知识点 | 说明 |
|--------|------|
| 变量 | 有名字的内存空间，用来存储数据 |
| 声明 | 告诉编译器变量名和类型 |
| 初始化 | 声明时给定初始值（强烈推荐） |
| 赋值 | 给已存在的变量一个新值 |
| 未定义行为（UB） | C标准未规定结果的操作，可能产生任何结果 |
| `const` | 关键字，标记变量为只读 |
| `#define` | 预处理宏，进行纯文本替换 |
| 命名规范 | 字母/数字/下划线，不能以数字开头，不能是关键字 |
