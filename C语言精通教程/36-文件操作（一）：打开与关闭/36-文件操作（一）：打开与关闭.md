## 我在做什么？

到目前为止，你写的所有程序都是"一次性"的——运行时输入数据，运行结束数据就消失了。但真正的程序需要持久化：游戏存档、文本编辑器、数据库，都需要读写文件。这一章你将学会如何打开和关闭文件，这是所有文件操作的第一步。

读完本章你能：理解文件指针的概念，用 fopen 打开文件，选择正确的打开模式，用 fclose 关闭文件，以及理解 stdin/stdout/stderr 三个标准流。

## 一、文件操作的基本概念

### 1.1 程序眼中的文件

在 C 语言中，文件不只是一个 `.txt` 或 `.jpg`。C 把一切可以读写的东西都抽象为"流"——键盘输入是流，屏幕输出是流，磁盘文件也是流。

```text
+----------+    写入     +--------+
| 你的程序  | ----------> |  文件  |
|          |             |        |
|          | <---------- | (磁盘) |
+----------+    读取     +--------+
```

### 1.2 文件指针（FILE*）

C 通过 `FILE*` 类型来操作文件。`FILE` 是一个结构体（定义在 `<stdio.h>` 中），存储了文件的所有内部状态。你不需要知道 `FILE` 结构体内部长什么样，你只需要一个指向它的指针——*文件指针* *此术语见附录F*。

```c
FILE *fp;  // fp 是一个文件指针，指向一个 FILE 结构体
```

比喻：`FILE*` 就像你去图书馆借书时拿到的**索书号**。你不需要知道书在哪个书架、哪一层，你只需要拿着索书号就能找到书。`FILE*` 就是操作系统给你的"索书号"。

## 二、标准流：stdin、stdout、stderr

### 2.1 三个自动打开的文件

每个 C 程序启动时，操作系统自动为你打开了三个文件：

```text
                     +--------+
  stdin  (键盘) ---->|        |----> stdout (屏幕)
                     | 程序   |
  stderr (错误) ---->|        |----> 文件 (磁盘)
                     +--------+
```

| 名称 | 全称 | 含义 | 默认设备 | FILE* 指针 |
|------|------|------|---------|-----------|
| *stdin* *此术语见附录F* | standard input | 标准输入 | 键盘 | `stdin` |
| *stdout* *此术语见附录F* | standard output | 标准输出 | 屏幕 | `stdout` |
| *stderr* *此术语见附录F* | standard error | 标准错误 | 屏幕 | `stderr` |

你之前用的 `printf` 实际上就是往 `stdout` 写数据，`scanf` 就是从 `stdin` 读数据。

```c
printf("Hello\n");             // 等价于 fprintf(stdout, "Hello\n");
scanf("%d", &n);               // 等价于 fscanf(stdin, "%d", &n);
```

### 2.2 为什么需要 stderr？

当你用 `printf` 输出错误信息时，它们和正常输出混在一起：

```c
printf("Result: 42\n");        // 正常输出
printf("Error: file not found\n");  // 错误信息——混在正常输出里
```

如果用户把正常输出重定向到文件（`./program > result.txt`），错误信息也会被写入文件，用户看不到。而 `stderr` 专门用来输出错误信息，即使正常输出被重定向，错误信息仍然显示在屏幕上：

```c
fprintf(stdout, "Result: 42\n");            // 可能被重定向
fprintf(stderr, "Error: file not found\n"); // 始终显示在屏幕上
```

## 三、fopen：打开文件

### 3.1 基本语法

```c
FILE *fopen(const char *filename, const char *mode);
```

- `filename`：文件路径（字符串）
- `mode`：打开模式（字符串）
- 返回值：成功返回 `FILE*`，失败返回 `NULL`

### 3.2 打开模式

| 模式 | 含义 | 文件不存在时 | 文件存在时 |
|------|------|------------|-----------|
| `"r"` | 只读（read） | 返回 NULL，失败 | 从头开始读 |
| `"w"` | 只写（write） | 创建新文件 | **清空原内容**，从头开始写 |
| `"a"` | 追加（append） | 创建新文件 | **保留原内容**，在末尾追加 |
| `"r+"` | 读写 | 返回 NULL，失败 | 从头开始读写 |
| `"w+"` | 读写 | 创建新文件 | **清空原内容** |
| `"a+"` | 读写（追加） | 创建新文件 | 读取从头开始，写入在末尾追加 |

其中 `"w"` 是最危险的——它会**清空**文件原有内容。如果你只是想追加内容，请用 `"a"`。

### 3.3 文本模式 vs 二进制模式

在模式字符串末尾加 `b` 表示二进制模式：

| 模式 | 含义 |
|------|------|
| `"rb"` | 二进制只读 |
| `"wb"` | 二进制只写 |
| `"ab"` | 二进制追加 |
| `"r+b"` 或 `"rb+"` | 二进制读写 |

在 Linux 上，文本模式和二进制模式没有区别。在 Windows 上，文本模式会自动做换行符转换（`\n` ↔ `\r\n`），二进制模式不会。**如果你要读写的是非文本文件（图片、音频、可执行文件），务必使用二进制模式。**

### 3.4 打开文件的标准写法

```c
#include <stdio.h>

int main() {
    FILE *fp = fopen("data.txt", "r");
    if (fp == NULL) {
        fprintf(stderr, "无法打开文件 data.txt\n");
        return 1;  // 返回非 0 表示程序错误退出
    }

    // ... 使用文件 ...

    fclose(fp);  // 别忘了关闭！
    return 0;
}
```

**永远检查 fopen 的返回值！** 文件可能不存在、可能没有权限、可能被其他程序占用，任何时候 fopen 都可能失败。

## 四、fclose：关闭文件

### 4.1 为什么必须关闭

```c
int fclose(FILE *fp);
```

关闭文件的原因：
1. **释放资源**：操作系统能同时打开的文件数量有限（通常几百到几千个）。
2. **刷新缓冲区**：写入的数据可能还在内存缓冲区中，fclose 确保数据真正写入磁盘。
3. **防止数据丢失**：如果程序崩溃，未关闭的文件的缓冲区数据可能丢失。

### 4.2 关闭后不能再使用

```c
FILE *fp = fopen("data.txt", "r");
fclose(fp);
// fp 现在是"悬空指针"，不能再使用
// fgetc(fp);  // ❌ 未定义行为
```

### 4.3 检查 fclose 是否成功

```c
if (fclose(fp) != 0) {
    fprintf(stderr, "关闭文件时出错\n");
}
```

关闭文件也可能失败——比如磁盘已满无法写入缓冲区数据。

## 五、完整示例

### 5.1 创建一个文件并写入内容

```c
#include <stdio.h>

int main() {
    // 以写模式打开（会清空已有内容）
    FILE *fp = fopen("hello.txt", "w");
    if (fp == NULL) {
        fprintf(stderr, "错误：无法创建文件 hello.txt\n");
        return 1;
    }

    fprintf(fp, "Hello, File!\n");
    fprintf(fp, "这是第二行\n");

    if (fclose(fp) != 0) {
        fprintf(stderr, "错误：关闭文件时失败\n");
        return 1;
    }

    printf("文件写入成功\n");
    return 0;
}
```

编译运行后，当前目录下会出现 `hello.txt`，内容为：
```
Hello, File!
这是第二行
```

### 5.2 读取文件内容

```c
#include <stdio.h>

int main() {
    FILE *fp = fopen("hello.txt", "r");
    if (fp == NULL) {
        fprintf(stderr, "错误：无法打开文件 hello.txt\n");
        return 1;
    }

    char ch;
    while ((ch = fgetc(fp)) != EOF) {
        putchar(ch);  // 输出到屏幕
    }

    fclose(fp);
    return 0;
}
```

### 5.3 追加模式

```c
#include <stdio.h>

int main() {
    // "a" 模式：追加内容，不清空原有内容
    FILE *fp = fopen("hello.txt", "a");
    if (fp == NULL) {
        fprintf(stderr, "错误：无法打开文件\n");
        return 1;
    }

    fprintf(fp, "这是追加的第三行\n");

    fclose(fp);
    printf("追加成功\n");
    return 0;
}
```

运行后，`hello.txt` 变成：
```
Hello, File!
这是第二行
这是追加的第三行
```

## 六、常见错误

### 6.1 忘记检查 fopen 返回值

```c
// ❌ 危险：如果文件不存在，fp 是 NULL，后续操作崩溃
FILE *fp = fopen("nonexistent.txt", "r");
fprintf(fp, "data");  // 段错误！对 NULL 指针操作

// ✅ 正确：先检查
FILE *fp = fopen("nonexistent.txt", "r");
if (fp == NULL) {
    fprintf(stderr, "无法打开文件\n");
    return 1;
}
```

### 6.2 忘记关闭文件

```c
// ❌ 不良习惯
for (int i = 0; i < 1000; i++) {
    FILE *fp = fopen("data.txt", "r");
    // 使用 fp...
    // 忘记 fclose(fp) —— 每次循环泄露一个文件描述符
}
// 运行到几百次后，fopen 开始返回 NULL，因为系统资源耗尽

// ✅ 正确：用完就关
for (int i = 0; i < 1000; i++) {
    FILE *fp = fopen("data.txt", "r");
    if (fp == NULL) continue;
    // 使用 fp...
    fclose(fp);
}
```

### 6.3 用 "w" 模式打开想保留内容的文件

```c
// ❌ 如果 data.txt 已有重要内容，这行会清空它！
FILE *fp = fopen("data.txt", "w");

// ✅ 如果只是想追加内容：
FILE *fp = fopen("data.txt", "a");
```

### 6.4 在 Windows 上用文本模式读写二进制文件

```c
// ❌ 在 Windows 上，文本模式会转换 \n 为 \r\n
FILE *fp = fopen("image.png", "r");  // 读写二进制文件不要用文本模式

// ✅ 加 b 后缀
FILE *fp = fopen("image.png", "rb");
```

## 七、我做得对不对？

创建以下程序，验证文件打开、写入、关闭的完整流程：

```c
#include <stdio.h>

int main() {
    // 1. 创建并写入文件
    FILE *fp = fopen("test.txt", "w");
    if (fp == NULL) {
        fprintf(stderr, "创建文件失败\n");
        return 1;
    }
    fprintf(fp, "Line 1\n");
    fprintf(fp, "Line 2\n");
    fprintf(fp, "Line 3\n");
    fclose(fp);

    // 2. 读取文件
    fp = fopen("test.txt", "r");
    if (fp == NULL) {
        fprintf(stderr, "打开文件失败\n");
        return 1;
    }
    printf("文件内容:\n");
    char buffer[100];
    while (fgets(buffer, sizeof(buffer), fp) != NULL) {
        printf("  %s", buffer);
    }
    fclose(fp);

    // 3. 追加内容
    fp = fopen("test.txt", "a");
    if (fp == NULL) {
        fprintf(stderr, "追加模式打开失败\n");
        return 1;
    }
    fprintf(fp, "Line 4 (appended)\n");
    fclose(fp);

    // 4. 再次读取验证
    fp = fopen("test.txt", "r");
    if (fp == NULL) {
        fprintf(stderr, "打开文件失败\n");
        return 1;
    }
    printf("\n追加后:\n");
    while (fgets(buffer, sizeof(buffer), fp) != NULL) {
        printf("  %s", buffer);
    }
    fclose(fp);

    return 0;
}
```

预期输出：
```
文件内容:
  Line 1
  Line 2
  Line 3

追加后:
  Line 1
  Line 2
  Line 3
  Line 4 (appended)
```

## 八、不对怎么办？

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| fopen 返回 NULL，文件存在 | 路径错误、权限不足、文件被锁定 | 检查路径是否正确，`ls -l` 检查权限 |
| `Segmentation fault` 在 fopen 后 | 未检查 fopen 返回值，对 NULL 指针操作 | 总是检查 `if (fp == NULL)` |
| 文件内容被意外清空 | 用了 `"w"` 模式而非 `"a"` | 追加用 `"a"`，读写用 `"r+"` |
| 文件打开太多后 fopen 失败 | 忘记 fclose，文件描述符耗尽 | 每次 fopen 后确保有对应的 fclose |
| `fclose` 返回非 0 | 磁盘满、网络文件系统错误 | 检查磁盘空间，处理 fclose 的错误返回值 |

## 本章小结

| 知识点 | 要点 |
|--------|------|
| 文件指针 | `FILE*` 类型，指向文件的结构体 |
| 标准流 | `stdin`（键盘）、`stdout`（屏幕）、`stderr`（错误输出） |
| fopen | 打开文件，mode 指定读写方式 |
| 打开模式 | `"r"` 读、`"w"` 写（清空）、`"a"` 追加、`"r+"` 读写 |
| 二进制模式 | 加 `b` 后缀，如 `"rb"`、`"wb"` |
| fclose | 关闭文件，释放资源，刷新缓冲区 |
| 错误处理 | 每次 fopen 必须检查返回值是否为 NULL |
