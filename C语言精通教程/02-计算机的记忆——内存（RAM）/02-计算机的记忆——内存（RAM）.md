## 我在做什么？

上一章你知道了 CPU 是计算机的大脑。但大脑需要记忆——CPU 不能凭空算，它需要数据，也需要把计算结果存起来。

这一章就讲计算机的"记忆"——内存。读完你能理解：内存地址这个 C 语言中最重要的概念，以及为什么 C 语言的所有"麻烦"（手动管理内存）恰恰是它的"力量"。

## 一、内存是什么？

### 1.1 一条街上有无数个空房子

想象一条长得望不到头的街道。街道两旁，排列着无数个一模一样的小房子。每个房子只有拳头大小，刚好能放一个 0 到 255 之间的数字。

这条街就叫**内存**。

```
内存 = 一条极其长的街道

[  ] [  ] [  ] [  ] [  ] [  ] [  ] [  ] [  ] [  ] [  ] [  ] ...
  ↑    ↑    ↑    ↑    ↑    ↑    ↑    ↑    ↑    ↑    ↑    ↑
 每个方框都是一个"储物格"，能放一个 0-255 之间的数字
```

每个储物格的专业名称叫 *字节* *此术语见附录F*。一个字节 = 8 个比特 = 能表示 0 到 255 之间的一个整数。

### 1.2 门牌号：内存地址

每个储物格都有唯一的门牌号。这个门牌号叫 *内存地址* *此术语见附录F*。

```
门牌号：   0     1     2     3     4     5     6     7     8  ...
        +-----+-----+-----+-----+-----+-----+-----+-----+-----+
内容：   | 65  | 66  | 67  | 0   | 255 | 10  | 0   | 42  | ... |
        +-----+-----+-----+-----+-----+-----+-----+-----+-----+
```

这就是内存的本质：**一个巨大的一维数组，索引就是地址，每个格子存一个字节。**

用 C 语言的话说：**内存就是一个 `unsigned char` 数组**。

```c
// 如果把整个内存看成一个C语言数组，它长这样：
unsigned char 整个内存[8589934592];  // 8GB 内存 = 约 86 亿个格子

// 第 0 号地址存的是 65
// 第 1 号地址存的是 66
// 第 5 号地址存的是 10
// ...
```

### 1.3 CPU 怎么用内存？

CPU 做事的时候，需要从内存中"读"数据，做完后把结果"写"回内存。

```
CPU 做 "a = b + c" 的完整过程：

1. CPU 问内存："把第 1000 号地址的数据给我"（读 b）
   内存：[1000] = 5  -->  CPU 得到 5

2. CPU 问内存："把第 1004 号地址的数据给我"（读 c）
   内存：[1004] = 3  -->  CPU 得到 3

3. CPU 算：5 + 3 = 8

4. CPU 告诉内存："把 8 写进第 1008 号地址"（存 a）
   内存：[1008] = 8
```

```
用 ASCII 图描述这个过程：

        CPU                           内存
    +-----------+              +------------------+
    |           | --读1000-->  | [1000] = 5       |
    |  计算单元  | <---5-----  | [1001] = ...     |
    |  (ALU)    |              | [1002] = ...     |
    |           | --读1004-->  | [1003] = ...     |
    |  5+3=8    | <---3-----  | [1004] = 3       |
    |           |              | ...              |
    |           | --写1008-->  | [1008] = 8       |
    |           |   传8        | ...              |
    +-----------+              +------------------+
```

## 二、内存地址的"门牌号"比喻（深入版）

### 2.1 为什么地址很重要

在 C 语言中，你几乎无时无刻不在和地址打交道。这是 C 和 Python 等语言最大的区别。

```c
int x = 42;       // 在内存中找一个空位，存 42，给这个位置起个别名叫 x
int *p = &x;      // 把 x 的门牌号（地址）存进 p
```

`&x` 的意思是"取 x 的门牌号"。这就像你问邮递员："42 号房在哪？"邮递员告诉你："在幸福路第 100 号。"

```text
变量名       类型        门牌号(地址)   内容
+------+   +--------+   +-----------+  +----+
|  x   |   |  int   |   | 0x1000    |  | 42 |
+------+   +--------+   +-----------+  +----+

+------+   +--------+   +-----------+  +----------+
|  p   |   |  int*  |   | 0x2000    |  | 0x1000   |  <-- p 里存的是 x 的门牌号
+------+   +--------+   +-----------+  +----------+
```

### 2.2 "房子"有大有小

一个储物格（字节）只能存 0-255。但如果我要存一个 99999 的大数，一个格子装不下怎么办？

那就占用**连续多个格子**。

```text
int 类型占 4 个字节（4 个连续的格子）：

门牌号：  1000  1001  1002  1003
        +-----+-----+-----+-----+
        |  X  |  X  |  X  |  X  |  <-- 四个格子合起来表示一个 int
        +-----+-----+-----+-----+
         ^
         |
    这个 int 的地址是 1000（第一个格子的门牌号）
```

C 语言中不同数据类型占用的格子数不同：

| 类型 | 占用字节数 | 比喻 |
|------|-----------|------|
| `char` | 1 | 单身公寓（只占一格） |
| `short` | 2 | 一室一厅（占两格） |
| `int` | 4 | 两室一厅（占四格） |
| `long long` | 8 | 大平层（占八格） |
| `double` | 8 | 大平层（占八格） |

## 三、为什么 C 语言需要手动管理内存？

### 3.1 两种"管家"模式

在一栋公寓楼里，有两种管理方式：

**Python / Java 模式（带管家）**：
- 你要一个房间，告诉管家："给我一间房。"
- 你用完了，直接走人。
- 管家定期巡查，发现没人住的房间就收回来。
- 这个管家叫 *垃圾回收（GC）* *此术语见附录F*。

**C 语言模式（没有管家）**：
- 你要一个房间，自己去前台登记。
- 你用完了，**必须自己去前台退房**。
- 没人替你收。忘了退房，那间房就永远占着。
- 这就是 *内存泄漏* *此术语见附录F*。

```text
Python 方式：
   int[] scores = new int[100];  // 要 100 个格子
   scores = null;                // 不想要了
   // 管家（GC）过一会儿来收拾

C 语言方式：
   int *scores = malloc(100 * sizeof(int));  // 要 100 个格子
   free(scores);                              // 必须自己还！
   scores = NULL;                             // 养成好习惯：把指针置空
```

### 3.2 为什么 C 不给配管家？

因为管家要收"管理费"。

- 垃圾回收需要额外的 CPU 时间来扫描内存、标记垃圾、回收空间。
- 在游戏引擎中，每帧只有 16 毫秒，你不想让 GC 突然跑出来"扫垃圾"导致画面卡顿。
- 在嵌入式设备中，内存总共只有几 KB，GC 自己就要占一大块。

C 语言的选择是：**把控制权交给程序员**。你更聪明（理论上），你知道什么时候该释放内存，所以你自己来。

### 3.3 内存泄漏的比喻

内存泄漏就像你住酒店，退房时没办退房手续就回家了。

短期住一晚问题不大（小程序，运行完就结束）。但如果你长期住（服务器程序，24 小时运行），每次"忘了退房"就多占一间房，几个月后酒店所有房间都被你占满了——新客人进不来，酒店瘫痪。

这就是为什么服务器程序的内存泄漏是致命的：它会让程序越来越慢，直到耗尽所有内存，崩溃。

## 四、内存就是一个巨大的 unsigned char 数组

### 4.1 这个视角为什么重要

在 C 语言中，**一切数据类型最终都是内存中的字节**。

```c
int x = 16909060;  // 十进制

// 在内存中，这个 int 占用 4 个字节：
// 地址：  1000    1001    1002    1003
// 内容：  0x04    0x03    0x02    0x01   (小端序，概念见后文)
```

如果你用 `char*` 指针去读它，就能看到原始字节：

```c
int x = 16909060;
char *p = (char*)&x;  // 把 x 的地址当成 char 数组来读

// p[0] = 4, p[1] = 3, p[2] = 2, p[3] = 1
```

这就是 C 语言强大的地方：**你可以用任何视角去看同一块内存**。同样的字节，可以被解释为 `int`、`float`、`char`，甚至是一段代码。

### 4.2 内存布局全景

当你运行一个 C 程序时，操作系统会给它分配一块内存。这块内存被分成几个区域：

```
高地址（地址数字大）
+-------------------+
|     栈 (Stack)     |  <-- 局部变量放这里，自动管理
|        |          |      函数调用时增长，返回时收缩
|        v          |
|                   |
|        ^          |
|        |          |
|     堆 (Heap)      |  <-- 动态分配的内存放这里，手动管理
|                   |      malloc() 从这里拿，free() 还回去
+-------------------+
|   全局/静态数据区   |  <-- 全局变量、静态变量放这里
+-------------------+
|     代码区         |  <-- 你的程序指令（机器码）放这里
+-------------------+
低地址（地址数字小）
```

不用现在就记住每个区域的名字。后面学到指针、动态内存、函数时，这些概念会反复回来。

## 五、内存与性能

### 5.1 内存比硬盘快多少？

CPU 从不同地方拿数据的速度差异极大：

```
速度对比（数量级，非精确值）：

CPU 寄存器    :  █  (< 1 纳秒)       -- 就像伸手拿桌上的杯子
CPU 缓存      :  ██  (几纳秒)         -- 就像从抽屉里拿东西
内存 (RAM)    :  ████████  (约100纳秒) -- 就像走到隔壁房间拿东西
硬盘/SSD      :  ████████████████████████████████  (微秒到毫秒级)
                                        -- 就像坐飞机去另一个城市拿东西
```

内存比硬盘快约 10 万倍。这就是为什么程序要先把数据从硬盘加载到内存，然后在内存里操作——CPU 等不起硬盘的"慢"。

### 5.2 为什么 C 程序快？

C 语言快，很大程度是因为它对内存的控制精细：

- 知道每个变量占多少字节。
- 知道变量在内存的哪个位置。
- 不需要垃圾回收器的"扫描"开销。
- 数据布局紧凑，缓存友好。

## 本章小结

| 概念 | 一句话解释 |
|------|-----------|
| 字节 | 内存的最小可寻址单元，8 个比特，能存 0-255 |
| 内存地址 | 每个字节的唯一编号，就像街道上的门牌号 |
| 垃圾回收（GC） | 自动回收不再使用的内存，Python/Java 有，C 没有 |
| 内存泄漏 | 分配了内存但忘了释放，导致可用内存越来越少 |
| 栈 | 存放局部变量的内存区域，函数调用时自动增长、返回时自动收缩 |
| 堆 | 存放动态分配内存的区域，由程序员手动申请和释放 |

## 不对怎么办？

**常见困惑 1**："地址、指针、变量——这三个到底什么关系？"

解答：变量是"名字"（给人看的），地址是"位置"（给机器看的），指针是"存地址的变量"（存门牌号的特殊变量）。

```text
类比：你有一个朋友叫"张三"，他住在"北京市朝阳区幸福路 100 号"。
- 变量名 = "张三"（你叫他名字就行）
- 地址 = "北京市朝阳区幸福路 100 号"（寄快递时需要）
- 指针 = 你通讯录里记的"张三的地址是北京市朝阳区幸福路 100 号"
```

**常见困惑 2**："我写 Python 从来不知道内存地址，为什么学 C 就要知道？"

解答：因为 Python 的管家（GC）帮你处理了所有地址的事。你不需要知道张三住哪条街，管家会帮你找到他。C 语言没有管家，你必须自己记住每条街、每个门牌号。麻烦，但你可以精确控制。

**常见困惑 3**："内存泄漏到底多严重？我写个小程序也会泄漏吗？"

解答：小程序无所谓——程序退出时操作系统会回收所有内存。严重的是**长期运行的程序**：服务器、数据库、游戏——它们可能要连续运行几个月甚至几年。每天漏一点，累积起来就是灾难。
