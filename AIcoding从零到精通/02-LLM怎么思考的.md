## 先把结论给你：LLM是一个"超级复读机"

在深入细节之前，先把最核心的结论摆出来：

> **LLM（📖 术语：Large Language Model，大语言模型）本质上是一个"非常会猜下一个字的超级复读机"。**

它不会思考，不会推理，不会判断真假。它只做一件事：**根据你给它的上文，猜下一个最可能出现的字是什么**。

你可能会问："如果它只是猜下一个字，怎么能写出完整的代码、回答复杂的问题？"

答案就是：它猜下一个字 → 然后把猜出来的字加到上文里 → 再猜下一个字 → 再猜下一个字……就这样一个字一个字地"猜"出整段回答。

就像你手机上的输入法，你打"我今天心情很"，输入法会联想"好"、"糟糕"、"一般"。LLM就是这种联想的**超级加强版**——它读完了大半个互联网的内容，所以它"猜"得极其准确。

## Token是什么：把文字切成"积木块"

在理解LLM怎么"猜字"之前，你得先知道它"猜"的到底是什么。

AI不是按"汉字"来猜的，而是按**Token（📖 术语：令牌）**来猜的。

**Token** = 把文字切成的小积木块。每个Token大概等于3-4个汉字，或者一个英文单词。

举个例子，把一句话切成Token：

```
原文：我喜欢用Python写代码
Token化：["我", "喜欢", "用", "Python", "写", "代码"]
        → 6个Token

原文：Hello, AI Coding!
Token化：["Hello", ",", " AI", " Coding", "!"]
        → 5个Token
```

> **一条经验法则**：英文大约1个Token ≈ 0.75个单词，中文大约1个Token ≈ 1.5个汉字。

为什么用Token而不是直接用字？因为**Token是AI的"最小思考单位"**。AI的"大脑"只能理解Token，不能理解字。你把文字发给AI，它第一步就是把文字切成Token，然后在这些Token上做计算。

**对你来说重要的一件事**：AI的收费是按Token算的。你输入的文字（输入Token）+ AI输出的文字（输出Token），加起来就是你要付的钱（或者消耗的免费额度）。所以，**说废话 = 浪费Token = 浪费钱**。

## 概率预测：AI本质是"下一个字猜什么"的概率游戏

现在你知道AI看的是Token了。那它怎么猜下一个Token？

**AI的每一次输出，本质上都是一次概率计算**。

想象你给AI一句话："今天天气真____"

AI会计算所有可能的下一个字的概率：

```
"好"   → 概率 45%
"热"   → 概率 20%
"冷"   → 概率 15%
"差"   → 概率 10%
"不错" → 概率 8%
其他   → 概率 2%
```

AI选择概率最高的"好"作为输出，然后重复这个过程：

```
"今天天气真好，____"
→ "适合" 概率 35%
→ "可以" 概率 25%
→ "应该" 概率 20%
→ ...
```

**就这样一个字一个字地猜，最终猜出整段回答。**

### 这怎么解释AI能写代码？

你问AI："帮我写一个Python函数，计算1到100的和。"

AI看到这句话后的Token序列，它在训练时读过成千上万类似的代码，所以它"知道"：

```
"帮我写一个Python函数，计算1到100的和。"
→ 下一个最可能的Token是 "def" （Python函数定义关键字）
→ "def" 之后最可能是 "sum_" 或 "calculate" （函数名）
→ 之后最可能是 "("
→ 之后最可能是 "n" 或 ")"
→ 之后最可能是 ":"
→ 之后最可能是 "\n    " （换行+缩进）
→ 之后最可能是 "total" 或 "result" （变量名）
→ ...
```

```python
def sum_one_to_hundred():
    total = 0
    for i in range(1, 101):
        total += i
    return total
```

**AI不是"理解"了你的需求，而是"见过"太多类似的代码，所以它知道这种情况下最可能的下一个字是什么。**

> **你手机的输入法也是概率预测，但LLM的"语料库"是大半个互联网，所以它"猜"得极其精准——精准到看起来像在思考。**

## 上下文窗口：AI的"短期记忆"

现在你知道AI是猜下一个字了。但它猜的时候，参考了哪些信息？

答案是：**上下文窗口（📖 术语：Context Window）**——AI能"看到"的最近一段文字。

把它想象成AI的**短期记忆**：

- 你给AI说了一句话，这句话进入AI的短期记忆
- AI根据短期记忆里的所有内容，猜下一个字
- 你又说了一句话，第二句话也进入短期记忆
- AI继续猜，越来越懂你的意思

**但短期记忆是有容量限制的**。就像一个只能写固定页数的记事本——写满了，新的内容进来，旧的内容就要擦掉。

| 上下文窗口大小 | 大概能记住多少字 | 代表模型 |
|-------------|---------------|---------|
| 4K tokens | ~3000字 | 早期GPT-2 |
| 8K tokens | ~6000字 | 早期GPT-3.5 |
| 32K tokens | ~24000字 | GPT-4早期版本 |
| 128K tokens | ~96000字 | GPT-4o、GPT-4 Turbo |
| 200K tokens | ~150000字 | Claude 3.5 Sonnet |
| 1M tokens | ~750000字 | Claude 4（最新） |

**这对你意味着什么？**

- ✅ **窗口越大，AI能"记住"的对话越长**，可以一次处理更长的代码文件
- ❌ **窗口越大，越贵**，因为AI需要处理更多Token
- ❌ **窗口不是越大越好**：信息过载会让AI"分心"，就像考试带整本书不如带一页重点

**实战经验**：当你发现AI开始"答非所问"、"忘记了你前面说过的话"——八成是上下文窗口满了。这时候清了上下文重来，比继续追问更有效。

## 训练 vs 推理：AI"学习"和"用"是两回事

很多人搞混这两个概念，但它们的区别就像"上学"和"上班"。

### 训练（📖 术语：Training）

AI怎么变聪明的？**训练**。

```
训练 = 让AI"读"海量数据，学习语言规律
```

OpenAI、Anthropic这些公司，用成千上万台服务器，让AI"读"互联网上的文章、书籍、代码、对话……这个过程叫"训练"。

- **训练一次**：需要几个月，花费几千万美元
- **训练完成后**：AI获得了一个"大脑"——包含所有它学到的知识
- **这个"大脑"是固定的**：训练完成后，AI不会继续学习新东西

**训练就像你上学读了12年书，毕业之后你的知识储备就固定了。**

### 推理（📖 术语：Inference）

你用AI的时候，AI在做什么？**推理**。

```
推理 = AI根据你的问题，从它已经学到的知识里"猜"出答案
```

- **推理一次**：几毫秒到几秒
- **推理消耗**：按Token计费，输入+输出
- **推理不改变AI**：每次推理都是独立的，AI不会因为和你的对话而"变聪明"

**推理就像你上班，用你上学时学到的知识解决问题——你不会因为回答了一个问题就突然变聪明。**

### 为什么你要知道这个区别？

因为很多初学者会问："为什么AI不知道我昨天告诉它的东西？"

**答案：AI不会"记住"你们的对话来更新自己。** 每次对话的上下文窗口就是它的全部记忆，对话结束，记忆清零。

如果你想让AI长期记住某些信息，需要用外部工具（比如Cursor的Notepads功能）来存，每次对话时手动"喂"给AI。

---

## 一句话总结

> **LLM是一个"非常会猜下一个字的超级复读机"——它读过半个互联网的内容，所以当你给它一个开头，它能极其精准地猜出接下来最可能出现的字，一个字一个字地"猜"出整段代码、整篇文章。**

它不会思考，不会推理，不会判断真假。它的"聪明"来自于：**见过足够多，所以猜得足够准**。

## 自测

1. 用你自己的话解释Token是什么，为什么AI用Token而不是直接用汉字？
2. 如果你发现AI开始"答非所问"，最可能的原因是什么？你应该怎么做？
3. "训练"和"推理"的区别是什么？为什么AI不会记住你昨天告诉它的东西？

> **答案**：
> 1. Token是把文字切成的小积木块，大概1个Token ≈ 3-4个汉字。AI用Token是因为Token是它的"最小思考单位"——AI的底层计算只能处理Token，不能直接处理字。
> 2. 最可能的原因是上下文窗口满了，AI"忘记"了你前面说的话。这时候应该清上下文重来，而不是继续追问。
> 3. 训练是AI"上学"读数据学知识的过程（几个月），推理是AI"上班"用已有知识回答你的过程（几秒）。AI不会记住昨天的对话，因为它的知识在训练时就固定了，每次对话的上下文窗口就是它的全部记忆。
