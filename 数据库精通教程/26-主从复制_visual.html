🔗 主从复制架构

时序图：binlog 产生 → IO 线程拉取 → SQL 线程回放

步骤
主库 (Master) 从库 (Slave) 当前步骤高亮

🖥️ 主库 (Master)

📦 binlog（二进制日志）
mysql-bin.000001
记录所有写操作（INSERT/UPDATE/DELETE）
等待写入
② IO 线程拉取 binlog

🖥️ 从库 (Slave)

🔌 IO 线程 (I/O Thread)
连接主库 → 拉取 binlog → 写入 relay log
等待启动
📋 relay log（中继日志）
slave-relay-bin.000001
存储从主库拉取的 binlog 事件
等待数据
⚙️ SQL 线程 (SQL Thread)
读取 relay log → 执行 SQL → 从库数据更新
等待启动

角色位置职责
binlog主库磁盘记录所有写操作，复制的数据源
IO 线程从库连接主库，拉取 binlog，写入 relay log
SQL 线程从库读取 relay log，在从库上执行 SQL
relay log从库磁盘临时存储从主库拉取的 binlog 事件
