## 39.1 JSON序列化

JSON是后端开发中最常用的数据交换格式。Go的 `encoding/json` 包提供了完整的支持。

### Marshal：Go对象 → JSON字符串

```go
type User struct {
    Name  string `json:"name"`
    Age   int    `json:"age"`
    Email string `json:"email,omitempty"`
}

u := User{
    Name: "张三",
    Age:  25,
}

data, err := json.Marshal(u)
if err != nil {
    fmt.Println("序列化失败:", err)
    return
}
fmt.Println(string(data))
```

输出：
```json
{"name":"张三","age":25}
```

注意 `Email` 字段没有出现在JSON中——因为它是零值 `""` 且tag中有 `omitempty`。

### MarshalIndent：格式化JSON

```go
data, err := json.MarshalIndent(u, "", "  ")
```

输出：
```json
{
  "name": "张三",
  "age": 25
}
```

第二个参数是前缀（通常为空），第三个参数是缩进符（通常为两个空格）。

### Unmarshal：反序列化（JSON字符串还原为Go对象，详见附录I）

```go
jsonStr := `{"name":"李四","age":30,"email":"lisi@example.com"}`

var u User
err := json.Unmarshal([]byte(jsonStr), &u)
if err != nil {
    fmt.Println("反序列化失败:", err)
    return
}

fmt.Println(u.Name, u.Age, u.Email)
```

输出：`李四 30 lisi@example.com`

**注意**：`Unmarshal` 需要传指针 `&u`——因为它要**修改**u的值。

### Tag控制详解

```go
type Product struct {
    ID       int     `json:"id"`
    Name     string  `json:"name"`
    Price    float64 `json:"price,omitempty"`
    Internal string  `json:"-"`
}
```

| Tag | 效果 |
|-----|------|
| `json:"id"` | JSON中的键名是 `id` |
| `json:"price,omitempty"` | 零值时不出现在JSON中 |
| `json:"-"` | 该字段永远不参与序列化 |
| `json:",omitempty"` | 键名用字段名（首字母小写），零值省略 |
| `json:",string"` | 序列化时转成字符串类型 |

### omitempty的零值定义

| 类型 | 零值（omitempty会省略） |
|------|----------------------|
| int/float | `0` |
| string | `""` |
| bool | `false` |
| 指针/nil切片/nil map | `nil` |
| 空切片 `[]int{}` | **不会省略**（不是nil） |

### 处理未知JSON字段

有时候你不知道JSON里有哪些字段：

```go
jsonStr := `{"name":"张三","age":25,"extra":"多余信息"}`

var result map[string]interface{}
json.Unmarshal([]byte(jsonStr), &result)

fmt.Println(result["name"])
fmt.Println(result["age"])
fmt.Println(result["extra"])
```

`map[string]interface{}` 可以接收任意JSON对象。但值都是 `interface{}`，使用前需要类型断言。

### Encoder / Decoder：流式处理

如果你操作的是 `io.Reader` / `io.Writer`（比如HTTP请求/响应、文件），用 `Encoder` 和 `Decoder` 更方便：

```go
file, _ := os.Create("user.json")
defer file.Close()

encoder := json.NewEncoder(file)
encoder.Encode(User{Name: "张三", Age: 25})
```

```go
file, _ := os.Open("user.json")
defer file.Close()

decoder := json.NewDecoder(file)
var u User
decoder.Decode(&u)
```

`Encoder`/`Decoder` 在处理HTTP请求体时特别方便——直接对接 `http.Request.Body`（一个 `io.Reader`）。（net/http 详见第66章）

## 39.2 其他序列化格式

### XML简介

Go的 `encoding/xml` 包用法和JSON类似：

```go
type Person struct {
    XMLName xml.Name `xml:"person"`
    Name    string   `xml:"name"`
    Age     int      `xml:"age"`
}

p := Person{Name: "张三", Age: 25}
data, _ := xml.MarshalIndent(p, "", "  ")
fmt.Println(string(data))
```

输出：
```xml
<person>
  <name>张三</name>
  <age>25</age>
</person>
```

XML比JSON冗长得多，现代API很少用XML，但在某些领域（如SOAP接口、RSS订阅、配置文件）仍然存在。

### YAML简介

Go标准库不带YAML支持，需要用第三方库（如 `gopkg.in/yaml.v3`）：

```go
import "gopkg.in/yaml.v3"

type Config struct {
    Server   string `yaml:"server"`
    Port     int    `yaml:"port"`
    Database struct {
        Host string `yaml:"host"`
        Name string `yaml:"name"`
    } `yaml:"database"`
}

yamlData := `
server: localhost
port: 8080
database:
  host: 127.0.0.1
  name: myapp
`

var config Config
yaml.Unmarshal([]byte(yamlData), &config)
fmt.Println(config.Database.Host)
```

YAML最大的优势是**人类可读**——比JSON更简洁（不需要大括号和引号），特别适合写配置文件。Docker Compose、Kubernetes的配置文件都是YAML格式。

### Protobuf简介

Protocol Buffers（Protobuf）是Google开发的一种**二进制**序列化格式。

**特点**：
- 体积小（比JSON小3~10倍）
- 速度快（解析比JSON快20~100倍）
- 强类型（需要预先定义 `.proto` 文件）
- 跨语言（Go、Java、Python、C++……）

一个 `.proto` 文件示例：

```protobuf
syntax = "proto3";

message User {
    string name = 1;
    int32 age = 2;
    string email = 3;
}
```

编译后生成Go代码，然后序列化/反序列化：

```go
u := &pb.User{Name: "张三", Age: 25}
data, _ := proto.Marshal(u)

var u2 pb.User
proto.Unmarshal(data, &u2)
```

Protobuf常用于**微服务之间的通信**（gRPC默认用Protobuf）和高性能场景（如消息队列、日志传输）。

### 三种格式对比

| 格式 | 可读性 | 体积 | 速度 | 适用场景 |
|------|--------|------|------|---------|
| JSON | ★★★★★ | 中 | 中 | Web API、前后端通信 |
| XML | ★★★☆☆ | 大 | 慢 | 传统企业系统、SOAP |
| YAML | ★★★★★ | 小 | 中 | 配置文件 |
| Protobuf | ★☆☆☆☆ | 极小 | 极快 | 微服务间RPC、高性能传输 |

## 🤔 想多一点：序列化的坑

### 1. 未导出字段不被序列化

```go
type User struct {
    Name string `json:"name"`
    age  int    `json:"age"`
}

u := User{Name: "张三", age: 25}
data, _ := json.Marshal(u)
fmt.Println(string(data))
```

输出：`{"name":"张三"}`

`age` 字段首字母小写（私有的），`json.Marshal` 无法访问它。即使有 `json` tag也没用。

### 2. 循环引用

```go
type Node struct {
    Value int
    Next  *Node
}

a := &Node{Value: 1}
b := &Node{Value: 2}
a.Next = b
b.Next = a

json.Marshal(a)
```

`json.Marshal` 遇到循环引用会陷入无限递归——**直接报错**。设计数据结构时需要注意。

### 3. 数字精度问题

```go
var data = `{"price": 19.99}`
var result map[string]interface{}
json.Unmarshal([]byte(data), &result)
fmt.Printf("%T\n", result["price"])
```

输出：`float64`

JSON中的数字默认被解析为 `float64`。如果涉及金额，浮点数的精度问题可能导致误差。解决方案：在结构体里声明精确类型（如 `float64` 但你得接受精度），或先用 `json.Number` 处理后用 `decimal` 库。

## 本章小结

| 知识点 | 要点 |
|--------|------|
| Marshal | Go对象 → JSON，`json.Marshal()` / `MarshalIndent()` |
| Unmarshal | JSON → Go对象，传指针 `&obj` |
