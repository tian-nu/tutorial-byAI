# 第03章 · 计算机的仓库——硬盘

> "内存是'过目就忘'的短期记忆，硬盘是'写在纸上'的长期记忆。你存了一张照片，关机再开机它还在——因为它在硬盘上，而不是内存里。"

## 03.1 硬盘是什么

硬盘（Hard Disk / SSD）是计算机的"长期记忆"。数据写入硬盘后，即使断电也不会丢失。

| | 内存（RAM） | 硬盘（HDD/SSD） |
|---|-----------|---------------|
| 速度 | 纳秒级，极快 | 毫秒级，慢1000倍以上 |
| 持久性 | 断电即丢失 | 断电不丢失 |
| 容量 | 8-256GB | 256GB-数TB |
| 用途 | 运行中的程序数据 | 存储文件、数据库、日志 |
| 价格 | 较贵/GB | 便宜/GB |

传统机械硬盘（HDD）像唱片机——有个磁头在旋转的盘片上读写。SSD（固态硬盘）没有机械结构，全电子，速度快很多。现在服务器基本都用SSD了。

## 03.2 硬盘对Java程序员的重要性

你写的 `.java` 文件存在硬盘上。编译出的 `.class` 文件也存硬盘。Spring Boot打的 `.jar` 包存硬盘。MySQL的数据文件存硬盘。应用的日志文件存硬盘。

**磁盘IO（输入输出）是后端性能的最大瓶颈之一。** CPU处理一条指令的纳秒，磁盘读一个字节的时间，差了上千万倍。

```java
// 这段代码可能很慢——因为涉及磁盘IO
File file = new File("/var/log/myapp/access.log");  // 硬盘上的文件
BufferedReader reader = new BufferedReader(new FileReader(file));
String line;
while ((line = reader.readLine()) != null) {  // 每读一行都是一次磁盘IO
    System.out.println(line);
}
```

上面这段代码打开了硬盘上的一个大日志文件（比如500MB），一行一行读。每次 `readLine()` 都可能触发一次磁盘读取，而磁盘读取的速度比内存慢上千倍。

## 03.3 Java中的IO操作（预习）

Java提供了丰富的API来读写文件。这些API会在后续章节详细讲解，现在先看个轮廓：

```java
// 读文件——老派方式（Java 7之前）
BufferedReader reader = new BufferedReader(new FileReader("data.txt"));
String line = reader.readLine();

// 读文件——NIO方式（Java 7+）
Path path = Paths.get("data.txt");
List<String> lines = Files.readAllLines(path);

// 读文件——NIO.2 + Stream（Java 8+）
Files.lines(Paths.get("data.txt")).forEach(System.out::println);
```

三种方式都能读文件，但底层原理不同：
- `BufferedReader`：传统阻塞IO，读数据时线程停下来等
- `NIO`：非阻塞IO，可以一个线程管理多个文件读写
- `Files.lines()`：基于NIO的便捷封装，配合Stream API链式处理

> 🤔 想多一点：Spring Boot内置的Tomcat服务器，底层网络IO用的就是Java NIO。它能用少量线程处理成千上万个并发连接，靠的就是非阻塞IO——不让你为一个慢速连接白白浪费线程。这就是NIO在真实项目中的价值。

## 03.4 数据库与硬盘的关系

```sql
SELECT * FROM users WHERE id = 1;
```

这条SQL的背后发生了什么？MySQL服务进程收到查询请求，然后去硬盘上找到存 `users` 表的数据文件，把 `id=1` 那行的数据读到内存，再通过网络返回给你的Java应用。

**为什么加索引能加速查询？** 因为不加索引，MySQL必须把整个数据文件从硬盘读到内存，逐行扫描（全表扫描）。加了索引，MySQL先读索引文件（通常小得多），定位到目标数据在硬盘上的位置，只读那一点点——磁盘IO减少了几个数量级。

这就是为什么你在第18章学数据库时会反复听到"加索引"——它的本质是**减少磁盘IO**。
