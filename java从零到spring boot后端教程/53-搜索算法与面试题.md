## 53.1 线性搜索（暴力，但有时就是最优解）

```java
public int linearSearch(int[] arr, int target) {
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == target) {
            return i;
        }
    }
    return -1;  // 没找到
}
// 时间复杂度 O(n)，空间 O(1)
```

**什么时候线性搜索就够了？**
- 数据量很小（< 100）
- 数据无序且不能排序
- 链表结构（无法用二分）
- 只查一次（排序O(n log n) + 二分O(log n) 不如直接线性O(n)）

## 53.2 二分搜索（有序数组的加速器）

**前提：数组必须有序！**

```java
public int binarySearch(int[] arr, int target) {
    int left = 0, right = arr.length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;  // 防溢出写法
        if (arr[mid] == target) {
            return mid;
        } else if (arr[mid] < target) {
            left = mid + 1;    // target在右半区
        } else {
            right = mid - 1;   // target在左半区
        }
    }
    return -1;
}
// 时间复杂度 O(log n)，空间 O(1)
```

**二分查找变体（面试常考）**：

```java
// 1. 找第一个等于target的位置（有重复元素时）
public int findFirst(int[] arr, int target) {
    int left = 0, right = arr.length - 1, result = -1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) {
            result = mid;
            right = mid - 1;  // 继续往左找
        } else if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    return result;
}

// 2. 找第一个大于等于target的位置（lower_bound）
public int lowerBound(int[] arr, int target) {
    int left = 0, right = arr.length;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] >= target) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }
    return left;  // 返回的是插入位置
}
```

**二分查找的诗意**：你有一本1000页的书，每一次比较排除一半——10次比较就能从1000页中找到目标页。10亿条数据只需30次比较。

> 🤔 想多一点：`mid = left + (right - left) / 2` 而不是 `mid = (left + right) / 2`，因为 `left + right` 在int范围下可能溢出（当两者都接近Integer.MAX_VALUE时）。这在实际项目中真实发生过。

## 53.3 DFS（深度优先搜索）— 一条路走到黑

DFS用**栈**（递归隐式用调用栈，或显式用Stack），适合找**所有路径**、**连通性**。

### DFS遍历图

```java
public void dfs(Map<Integer, List<Integer>> graph, int start, boolean[] visited) {
    visited[start] = true;
    System.out.println("访问: " + start);

    for (int neighbor : graph.getOrDefault(start, Collections.emptyList())) {
        if (!visited[neighbor]) {
            dfs(graph, neighbor, visited);
        }
    }
}

// 使用：
Map<Integer, List<Integer>> graph = new HashMap<>();
graph.put(0, List.of(1, 2));
graph.put(1, List.of(0, 3, 4));
graph.put(2, List.of(0, 5));
graph.put(3, List.of(1));
graph.put(4, List.of(1));
graph.put(5, List.of(2));

boolean[] visited = new boolean[6];
dfs(graph, 0, visited);
// 输出：访问: 0 → 访问: 1 → 访问: 3 → 访问: 4 → 访问: 2 → 访问: 5
```

### DFS的后端应用

- **微服务调用链分析**：DFS遍历服务依赖图，检测循环依赖、找出调用路径
- **文件系统遍历**：`ls -R` 递归显示所有子目录
- **全排列/组合问题**：回溯算法本质就是DFS

## 53.4 BFS（广度优先搜索）— 一层一层剥开

BFS用**队列**，适合找**最短路径**。

```java
public void bfs(Map<Integer, List<Integer>> graph, int start) {
    boolean[] visited = new boolean[graph.size()];
    Queue<Integer> queue = new LinkedList<>();

    queue.offer(start);
    visited[start] = true;

    while (!queue.isEmpty()) {
        int node = queue.poll();
        System.out.println("访问: " + node);

        for (int neighbor : graph.getOrDefault(node, Collections.emptyList())) {
            if (!visited[neighbor]) {
                visited[neighbor] = true;
                queue.offer(neighbor);
            }
        }
    }
}
// 对同一个图，输出：访问: 0 → 访问: 1 → 访问: 2 → 访问: 3 → 访问: 4 → 访问: 5
```

### DFS vs BFS 选型

| 维度 | DFS | BFS |
|---|---|---|
| 数据结构 | 栈（递归/Stack） | 队列（Queue） |
| 空间 | O(深度) — 可能很深 | O(宽度) — 可能很宽 |
| 找最短路径 | ❌ 不能保证 | ✅ 天然找到最短 |
| 找所有路径 | ✅ | ❌ |
| 检测环 | ✅ | ✅ |
| 代码简洁度 | 递归版极简 | 需要显式队列 |

## 53.5 面试高频题（三道必刷题）

### 题1：两数之和（Two Sum）— LeetCode第1题

**问题**：给定数组和target，找出和为target的两个数的索引。

```java
// 方法1：暴力 O(n²)
public int[] twoSumBrute(int[] nums, int target) {
    for (int i = 0; i < nums.length; i++) {
        for (int j = i + 1; j < nums.length; j++) {
            if (nums[i] + nums[j] == target) {
                return new int[]{i, j};
            }
        }
    }
    return new int[]{};
}

// 方法2：HashMap O(n) — 以空间换时间
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>();  // value → index
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (map.containsKey(complement)) {
            return new int[]{map.get(complement), i};
        }
        map.put(nums[i], i);
    }
    return new int[]{};
}

// 输入: nums = [2, 7, 11, 15], target = 9
// i=0: 2, complement=7, map没有7, put(2,0)
// i=1: 7, complement=2, map有2, 返回[0,1]
```

**这题教会我们什么？** HashMap把O(n²)降到O(n)，这就是为什么Spring IoC用Map——以空间换时间是后端最常见的优化手段。

### 题2：有效括号（Valid Parentheses）— LeetCode第20题

```java
public boolean isValid(String s) {
    Deque<Character> stack = new ArrayDeque<>();
    for (char c : s.toCharArray()) {
        if (c == '(' || c == '{' || c == '[') {
            stack.push(c);                               // 左括号入栈
        } else {
            if (stack.isEmpty()) return false;            // 右括号多了
            char top = stack.pop();
            if (c == ')' && top != '(') return false;     // 不匹配
            if (c == '}' && top != '{') return false;
            if (c == ']' && top != '[') return false;
        }
    }
    return stack.isEmpty();  // 栈空了才算全匹配
}
// "()[]{}" → true
// "([)]"   → false（括号交叉了）
// "{[]}"   → true
```

**这题教会我们什么？** 栈是处理"嵌套结构"的天然工具。JVM调用栈、HTML标签匹配、JSON解析都用了栈。

### 题3：最长不重复子串 — LeetCode第3题

```java
public int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> map = new HashMap<>();  // char → 最后出现位置
    int maxLen = 0;
    int left = 0;  // 滑动窗口的左边界

    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (map.containsKey(c) && map.get(c) >= left) {
            left = map.get(c) + 1;  // 出现重复，左边界跳到重复字符的下一个位置
        }
        map.put(c, right);
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}

// 输入: "abcabcbb"
// right=0: 'a', left=0, map={a:0}, maxLen=1
// right=1: 'b', left=0, map={a:0,b:1}, maxLen=2
// right=2: 'c', left=0, map={a:0,b:1,c:2}, maxLen=3
// right=3: 'a', left=0→1(重复a在索引0), map={a:3,b:1,c:2}, maxLen=3
// right=4: 'b', left=1→2(重复b在索引1), map={a:3,b:4,c:2}, maxLen=3
// ...
// 返回: 3 ("abc")
```

**这题教会我们什么？** **滑动窗口**是处理子串/子数组问题的核心技巧。HashMap记录位置 + 左右指针维护窗口，O(n)解决。

## 53.6 ❌ 常见错误

| 错误 | 后果 | 正确做法 |
|---|---|---|
| 二分查找时`mid = (left + right) / 2` | left+right溢出 | 用`left + (right - left) / 2` |
| BFS/DFS遍历图不标记visited | 无限循环 | 访问过的节点必须标记 |
| Two Sum用暴力O(n²) | 数据量大时超时 | 用HashMap O(n) |
| 滑动窗口问题不更新left | 窗口不会收缩 | 先移动left再扩大right |
| 栈判断括号匹配忘记最后判空 | "("也会返回true | `return stack.isEmpty()` |

## 小结

| 概念 | 解释 | 时间复杂度 |
|---|---|---|
| 线性搜索 | 逐个遍历，适合小数据或无序 | O(n) |
| 二分搜索 | 有序数组，每次排除一半 | O(log n) |
| DFS | 深度优先，用栈，一条路走到黑 | O(V+E) |
| BFS | 广度优先，用队列，一层层展开 | O(V+E) |
| 滑动窗口 | 双指针维护窗口，处理子串问题 | O(n) |
| 空间换时间 | HashMap存中间结果，牺牲空间换速度 | — |
