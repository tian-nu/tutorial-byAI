# 第104章 · 云服务部署实战

> "你的Go程序写得再好，架构再优雅——如果只有你自己能在 `localhost:8080` 上访问它，那它就是个玩具。这一章手把手带你把它变成一个世界任何角落都能访问的真正线上服务。从购买服务器到绑定域名，从配置Nginx到安装SSL证书——每一步都有具体命令，照做就能上线。"

## 104.1 购买云服务器

### 选哪家

国内主流选择：
- **阿里云 ECS**：生态最完善，文档最全，国内市场份额第一
- **腾讯云 CVM**：价格稍便宜，适合个人和小团队，和微信生态集成好
- **AWS EC2**：全球市场份额第一，功能最强大，但中文文档少且价格偏高

对于后端工程师的个人项目和学习，**腾讯云或阿里云的轻量应用服务器**就足够了。

### 最低配置建议

| 用途 | CPU | 内存 | 带宽 | 参考价格（年） |
|------|-----|------|------|----------------|
| 个人项目/学习 | 2核 | 2GB | 3Mbps | ~100元 |
| 小型线上服务 | 2核 | 4GB | 5Mbps | ~300元 |
| 中等流量 | 4核 | 8GB | 10Mbps | ~1000元 |

### 操作系统选择

选 **Ubuntu 22.04 LTS**（Long Term Support，长期支持版）。原因：
1. 社区最大，遇到问题Google一搜就有答案
2. 软件包最新（不像CentOS那样仓库里的软件版本可能比较老）
3. LTS版本有5年安全更新保障

> 注意：CentOS 7已于2024年6月停止维护，CentOS 8于2021年底停止。**不要再安装CentOS了。** 如果因公司政策必须用RHEL系，选Rocky Linux或AlmaLinux。

### 购买后的检查清单

购买完成后，你会在控制台看到：
- **公网IP**：别人通过这个IP访问你的服务（记下来，马上要用）
- **root密码**：第一次登录用的（登录后立即改！）
- **SSH端口**：默认22

## 104.2 SSH登录服务器

### 基本登录

```bash
ssh root@你的服务器公网IP
```

第一次登录会提示：
```
The authenticity of host 'xxx.xxx.xxx.xxx' can't be established.
Are you sure you want to continue connecting (yes/no)?
```

输入 `yes`。这是正常的——你的电脑还没"认识"这台服务器，确认后就记住它的指纹了。

### 修改SSH端口（安全第一步）

黑客的扫描器会不停地尝试登录你服务器的22端口。改端口是最简单的防护：

```bash
sudo nano /etc/ssh/sshd_config
```

找到 `#Port 22`，改成：
```
Port 2222
```

然后重启SSH服务：
```bash
sudo systemctl restart sshd
```

之后登录就要指定端口了：
```bash
ssh -p 2222 root@你的IP
```

⚠️ 改端口之前，确认云服务商的安全组/防火墙允许新端口！否则你会把自己锁在外面！

### 配置免密登录

每次都输密码太麻烦，而且密码会被暴力破解。用SSH密钥是更安全也更方便的方式。

**Step 1：在你自己的电脑上生成密钥对**

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

一路回车，会在 `~/.ssh/` 下生成 `id_ed25519`（私钥）和 `id_ed25519.pub`（公钥）。

**Step 2：把公钥复制到服务器**

```bash
ssh-copy-id -p 2222 root@你的IP
```

输一次密码后，公钥就会被追加到服务器的 `~/.ssh/authorized_keys` 文件里。

**Step 3：测试免密登录**

```bash
ssh -p 2222 root@你的IP
```

不用输密码就直接上去了。

**Step 4（强烈建议）：禁用密码登录**

```bash
sudo nano /etc/ssh/sshd_config
```

修改：
```
PasswordAuthentication no
PubkeyAuthentication yes
```

```bash
sudo systemctl restart sshd
```

现在只有持有私钥的人才能登录你的服务器——密码方式完全关闭，暴力破解彻底失效。

## 104.3 服务器基础配置

### 创建普通用户（不要用root做日常操作）

```bash
adduser deploy
usermod -aG sudo deploy
```

### 安装必要软件

Ubuntu上：

```bash
sudo apt-get update
sudo apt-get upgrade -y
sudo apt-get install -y \
  docker.io \
  docker-compose-v2 \
  git \
  nginx \
  certbot \
  python3-certbot-nginx \
  ufw \
  htop \
  vim
```

**各软件说明：**

| 软件 | 用途 |
|------|------|
| docker.io | 容器运行时 |
| docker-compose-v2 | 多容器编排 |
| git | 拉代码 |
| nginx | 反向代理、静态文件、SSL终端 |
| certbot | Let's Encrypt自动获取/续期SSL证书 |
| ufw | 防火墙 |
| htop | 更好的top（进程监控） |
| vim | 文本编辑器 |

### 把deploy用户加入docker组

```bash
sudo usermod -aG docker deploy
```

重新登录后生效。这样 `deploy` 用户不用 `sudo` 就能执行 `docker` 命令。

## 104.4 域名注册和DNS解析

### 域名注册

在阿里云/腾讯云/GoDaddy等平台注册一个域名。`.com` 域名约60元/年，`.cn` 域名约30元/年（需要实名认证）。

### 国内服务器特别注意：备案

如果你用国内服务器（阿里云/腾讯云），域名必须**先备案**才能用。备案流程大约需要2-4周。

**临时方案：**
1. 直接通过IP访问 `http://你的IP:8080`
2. 买一台香港/新加坡的服务器（不需要备案）
3. 用 `nip.io` 这种免费服务：`http://myapp.你的IP.nip.io:8080` 会自动解析到你的IP

### DNS解析配置

注册域名后，在DNS管理后台添加记录：

| 记录类型 | 主机记录 | 记录值 | 说明 |
|----------|----------|--------|------|
| A | `@` | 你的服务器IP | 裸域名（example.com） |
| A | `www` | 你的服务器IP | www子域名 |
| A | `api` | 你的服务器IP | api子域名（api.example.com） |

DNS解析全球生效需要几分钟到几小时不等。用 `nslookup 你的域名` 检查是否生效。

## 104.5 Nginx安装配置

### Nginx的角色

在你的架构中，Nginx是"门童"：
- 80/443端口进来的请求先到Nginx
- Nginx根据域名/路径转发给对应的后端服务（你的Go应用）
- Nginx处理SSL加密/解密（你的Go应用只处理HTTP）
- Nginx直接提供静态文件（图片、CSS、JS等，不经过Go）

```
互联网 → 443端口 → Nginx（SSL解密）→ localhost:8080 → Go应用
                        ↓
                    /static/* 直接返回静态文件
```

### 基础站点配置

创建配置文件 `/etc/nginx/sites-available/myapp`：

```nginx
server {
    listen 80;
    server_name example.com www.example.com;

    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
    }

    location /static/ {
        alias /opt/myapp/static/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    client_max_body_size 10M;
}
```

**配置详解：**

- `server_name`：这个server块只处理 `example.com` 和 `www.example.com` 的请求
- `proxy_pass http://localhost:8080`：把请求转发给本机8080端口的Go应用
- `proxy_set_header X-Real-IP $remote_addr`：把真实客户端IP传给Go应用（否则Go看到的是 `127.0.0.1`）
- `proxy_set_header X-Forwarded-Proto $scheme`：告诉Go应用原始请求用的是HTTP还是HTTPS
- `location /static/`：`/static/` 开头的请求直接返回文件，高性能（Nginx的静态文件服务比Go快得多）
- `expires 30d`：浏览器缓存静态文件30天
- `client_max_body_size 10M`：允许上传最大10MB的文件

### 启用站点

```bash
sudo ln -s /etc/nginx/sites-available/myapp /etc/nginx/sites-enabled/
sudo nginx -t                                    # 检查配置语法
sudo systemctl reload nginx                      # 热重载（不中断服务）
```

## 104.6 SSL证书配置

### Let's Encrypt

Let's Encrypt提供**免费**的SSL证书，浏览器信任，3个月有效期，支持自动续期。

### 使用Certbot获取证书

```bash
sudo certbot --nginx -d example.com -d www.example.com
```

Certbot会自动：
1. 验证你对 `example.com` 的所有权
2. 生成证书
3. 修改Nginx配置（添加SSL相关的配置行）
4. 设置自动续期定时任务

完成后，访问 `https://example.com` 就能看到小绿锁了。

### 手动Nginx HTTPS配置

如果不用Certbot自动配置，手动写HTTPS配置：

```nginx
server {
    listen 80;
    server_name example.com www.example.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name example.com www.example.com;

    ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
    }
}
```

**关键点：**
- 第一个 `server` 块：所有HTTP请求都301重定向到HTTPS
- `ssl_certificate`：证书文件（fullchain包含证书+中间证书）
- `ssl_protocols TLSv1.2 TLSv1.3`：只允许TLS 1.2和1.3（TLS 1.0和1.1已不安全）

### 自动续期

Certbot安装时自动添加了定时任务。手动检查续期是否正常：

```bash
sudo certbot renew --dry-run    # 模拟续期，测试是否正常
```

## 104.7 部署Go应用

### 方式一：Docker部署（推荐）

在服务器上：

```bash
mkdir -p /opt/myapp
cd /opt/myapp

cat > docker-compose.yml << 'EOF'
version: "3.8"
services:
  api:
    image: yourusername/myapp:latest
    container_name: myapp_api
    restart: always
    ports:
      - "8080:8080"
    env_file:
      - .env
    volumes:
      - ./uploads:/app/uploads
    logging:
      driver: json-file
      options:
        max-size: "10m"
        max-file: "3"
EOF

cat > .env << 'EOF'
APP_ENV=production
DB_HOST=your_db_host
DB_PASSWORD=your_db_password
REDIS_ADDR=redis:6379
JWT_SECRET=your_jwt_secret
EOF

docker login
docker pull yourusername/myapp:latest
docker-compose up -d
```

### 方式二：手动部署 + systemd

如果不用Docker，直接部署编译好的二进制文件：

```bash
sudo mkdir -p /opt/myapp

scp myapp deploy@服务器IP:/opt/myapp/
scp config.yaml deploy@服务器IP:/opt/myapp/

sudo cp /opt/myapp/myapp.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable myapp
sudo systemctl start myapp
```

其中 `myapp.service` 文件内容参考第99章99.9节。

### 部署后验证

```bash
curl http://localhost:8080/health

curl http://你的域名/api/users
curl https://你的域名/api/users   # 确认HTTPS正常工作
```

## 104.8 防火墙配置

### UFW（Uncomplicated FireWall）

UFW是iptables的友好封装，让防火墙配置变得直观。

```bash
sudo ufw default deny incoming       # 默认拒绝所有入站
sudo ufw default allow outgoing      # 默认允许所有出站

sudo ufw allow 2222/tcp              # SSH（改过的端口）
sudo ufw allow 80/tcp                # HTTP
sudo ufw allow 443/tcp               # HTTPS

sudo ufw enable                      # 启用防火墙
sudo ufw status verbose              # 查看状态

sudo ufw allow from 10.0.0.0/8 to any port 3306   # 只允许内网访问MySQL
```

### 云服务商安全组

**重要！** 云服务器有两层防火墙：
1. **云服务商的安全组**（在控制台配置）——先于操作系统
2. **操作系统防火墙**（如UFW）

两层都必须放行，端口才能通。比如你 `ufw allow 8080` 了，但安全组没有放行8080——外面还是连不进来。

### 最安全配置：只开放必要端口

```
公开端口:
  - 2222 (SSH，改了端口)
  - 80 (HTTP)
  - 443 (HTTPS)

内网端口（安全组限制IP访问）:
  - 3306 (MySQL)
  - 6379 (Redis)
  - 8080 (Go应用——通过Nginx反向代理访问，不需要直接对外)
```

## 104.9 上线检查清单

部署完成后，逐项检查：

### 安全

- [ ] 服务器SSH端口已从22改为其他端口
- [ ] 已禁用root登录（`PermitRootLogin no`）或至少禁用密码登录
- [ ] SSH密钥登录已配置，密码登录已禁用
- [ ] 防火墙（UFW + 安全组）只开放必要端口
- [ ] 数据库未暴露到公网（MySQL端口不开放对外）
- [ ] `.env` 文件权限为600
- [ ] Docker容器以非root用户运行
- [ ] SSL证书已配置，HTTP自动跳转HTTPS
- [ ] `JWT_SECRET` 是强随机字符串（至少256位）
- [ ] Go应用未在debug模式下运行（`GIN_MODE=release`）

### 功能

- [ ] `https://域名/health` 返回200
- [ ] 核心API接口可正常调用
- [ ] 静态文件（CSS/JS/图片）正常加载
- [ ] 数据库连接正常
- [ ] Redis连接正常（如果使用）

### 运维

- [ ] systemd服务（或Docker容器）设置为开机自启
- [ ] 日志正常输出（`journalctl -u myapp -f` 或 `docker logs`）
- [ ] 日志轮转已配置（防止日志撑满磁盘）
- [ ] SSL证书自动续期已测试（`certbot renew --dry-run`）
- [ ] 监控已配置（至少要有基础的健康检查告警）

### 性能

- [ ] Nginx `gzip` 压缩已开启
- [ ] 静态资源有缓存头（`expires`）
- [ ] `client_max_body_size` 已合理设置
- [ ] Docker容器有内存/CPU限制

### 备份

- [ ] 数据库有定期备份任务
- [ ] 备份文件存储在和服务器不同的位置（远程/另一台机器）
