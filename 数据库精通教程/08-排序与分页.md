## 我在做什么？

前几章你学会了 SELECT 和 WHERE——可以查询出满足条件的数据了。但默认情况下，数据库返回数据的顺序是"它觉得方便的顺序"，不是你想要的顺序。而且如果数据有 10000 行，你不可能一次看完，需要一页一页看。

这一章解决两个问题：
1. **排序**：让数据按你想要的顺序排列（比如按价格从高到低，按注册时间从新到旧）。
2. **分页**：只取其中一页的数据，而不是全部。

这两个功能是网页开发中最常见的需求——几乎每个列表页都需要排序和分页。

## 一、ORDER BY：排序

`ORDER BY` *此术语见附录E* 用来指定排序规则。

### 1.1 升序和降序

```sql
-- 按订单金额从小到大排序（默认升序）
SELECT * FROM orders ORDER BY amount;

-- 按订单金额从大到小排序（指定 DESC 降序）
SELECT * FROM orders ORDER BY amount DESC;

-- 升序可以显式写 ASC（不写默认也是 ASC）
SELECT * FROM orders ORDER BY amount ASC;
```

| 关键字 | 含义 | 示例 |
|--------|------|------|
| `ASC` | 升序（从小到大，从早到晚） | 价格从低到高，时间从旧到新 |
| `DESC` | 降序（从大到小，从晚到早） | 价格从高到低，时间从新到旧 |

### 1.2 多列排序

你可以按多列排序——先按第一列排序，如果第一列相同，再按第二列排序。

```sql
-- 先按 user_id 升序，同一用户再按 amount 降序
SELECT * FROM orders
ORDER BY user_id ASC, amount DESC;
```

### 1.3 多列排序规则

多列排序的语法：每列后面可以单独指定 ASC/DESC：

```sql
ORDER BY col1 ASC, col2 DESC;  -- col1 升，col2 降
ORDER BY col1, col2 DESC;     -- col1 省略 → 默认 ASC，col2 DESC ✅
```

## 二、ORDER BY 与 NULL

在 MySQL 中，NULL 被认为比任何非 NULL 值都**小**。

```sql
-- 表中有一些行的 phone 是 NULL
SELECT * FROM users ORDER BY phone ASC;
```

结果：所有 phone 为 NULL 的行都会排在**最前面**。

如果想让 NULL 排在**最后面**：

```sql
-- 先按 phone 是否为 NULL 排序，NULL 在后，再按 phone 本身排序
SELECT * FROM users
ORDER BY
    phone IS NULL ASC,  -- 0（非 NULL）排在 1（NULL）前面
    phone ASC;
```

## 三、LIMIT + OFFSET：分页

`LIMIT` *此术语见附录E* + `OFFSET` *此术语见附录E* 是 MySQL 中最常用的分页方案。

### 3.1 基本语法

```sql
-- 取 10 条，跳过前 0 条 → 第 1 页
SELECT * FROM users
ORDER BY id
LIMIT 10 OFFSET 0;

-- 取 10 条，跳过前 10 条 → 第 2 页
SELECT * FROM users
ORDER BY id
LIMIT 10 OFFSET 10;

-- 取 10 条，跳过前 20 条 → 第 3 页
SELECT * FROM users
ORDER BY id
LIMIT 10 OFFSET 20;
```

### 3.2 简化写法

MySQL 支持简化写法：`LIMIT 偏移量, 条数`

```sql
-- 等价于 LIMIT 10 OFFSET 0 → 第 1 页
SELECT * FROM users LIMIT 0, 10;

-- 等价于 LIMIT 10 OFFSET 10 → 第 2 页
SELECT * FROM users LIMIT 10, 10;
```

> 注意：顺序是 `LIMIT 偏移量, 条数`，而 `LIMIT 条数 OFFSET 偏移量`。两种写法的偏移量和条数顺序相反！别记反了。

### 3.3 分页公式

对于第 `page` 页，每页 `page_size` 条：

```
OFFSET = (page - 1) * page_size
```

比如：
- 第 1 页 → OFFSET = 0 × 10 = 0 → `LIMIT 10 OFFSET 0`
- 第 2 页 → OFFSET = 1 × 10 = 10 → `LIMIT 10 OFFSET 10`
- 第 3 页 → OFFSET = 2 × 10 = 20 → `LIMIT 10 OFFSET 20`

## 四、深分页：为什么 `LIMIT 100000, 10` 这么慢？

如果你数据量很大，比如有 100 万行，你会发现：

```sql
-- 第 1 页 → 很快
SELECT * FROM users ORDER BY id LIMIT 10 OFFSET 0;

-- 第 10001 页 → 很慢！
SELECT * FROM users ORDER BY id LIMIT 10 OFFSET 100000;
```

为什么慢？因为 MySQL 做了这个过程：

1. 找到 id 从 1 到 100000 的行 → 把这些行全读出来
2. 扔掉前 100000 行
3. 返回最后 10 行

也就是说，**你要跳过 10 万行，但数据库不得不把这 10 万行都读一遍**才能找到你要的那 10 行。越往后翻，需要跳过的行越多，速度越慢。

这个问题叫做**深分页** *此术语见附录E*。

> 为什么不能直接跳到第 100001 行？因为索引是有序的，但物理存储上不是连续的——跳过 10 万页还是需要读取 10 万个索引节点。这个问题要到第 17 章讲索引原理时才能讲清楚，现在你只需要知道现象和解决方案。

## 五、深分页解决方案：游标分页

**游标分页** *此术语见附录E* 的核心思想：利用索引的有序性，直接从你上次停下的地方开始往后读，不需要跳过前面的行。

### 5.1 基本思想

假设你按 `id` 排序分页：

```sql
-- 第 1 页，id 从 1 开始，取 10 条
SELECT * FROM users
WHERE id > 0  -- 从 id=0 之后开始
ORDER BY id
LIMIT 10;

-- 拿到最后一条记录的 id，比如最后一条是 id=10
-- 第 2 页，从 id=10 之后开始，再取 10 条
SELECT * FROM users
WHERE id > 10  -- 直接跳到 id=10 之后，不用跳过前面 10 行
ORDER BY id
LIMIT 10;

-- 拿到最后一条记录的 id，比如最后一条是 id=20
-- 第 3 页，从 id=20 之后开始
SELECT * FROM users
WHERE id > 20
ORDER BY id
LIMIT 10;
```

这种写法无论翻到第几页，速度都一样快——因为 `id` 是主键，有索引，`WHERE id > last_id` 可以直接定位，不需要跳过前面的行。

### 5.2 优缺点对比

| 对比项 | LIMIT OFFSET | 游标分页 |
|--------|-------------|----------|
| 页码跳转 | 支持（直接跳第 N 页） | 不支持（只能翻下一页） |
| 翻页速度 | 越深越慢 | 始终很快 |
| 适用场景 | 小数据量、页数不深 | 大数据量、深度分页、下拉加载 |

很多 App 的"无限下拉"（不断加载更多）用的就是游标分页——用户不需要跳转到第 100 页，只需要不断往下拉，每次加载更多即可。

### 5.3 排序不是主键怎么办？

如果排序不是按主键，而是按其他列（比如 `created_at`）：

```sql
-- 按创建时间倒序，最新的在前
-- 最后一条的 created_at 是 '2025-05-01 12:00:00'，id 是 12345
SELECT * FROM users
WHERE (created_at < '2025-05-01 12:00:00')
   OR (created_at = '2025-05-01 12:00:00' AND id > 12345)
ORDER BY created_at DESC, id ASC
LIMIT 10;
```

原理一样：利用排序列 + 主键（唯一）来定位，避免重复。

```sql
-- ============================================

-- ============================================

-- 0. 确保在正确的数据库
USE my_shop;

SELECT * FROM orders ORDER BY amount DESC;

SELECT * FROM users ORDER BY id LIMIT 2 OFFSET 4;  -- 第 3 页

-- 假设最后一条 id 是 2，第 2 页
SELECT * FROM users WHERE id > 2 ORDER BY id LIMIT 2;
```

## 我做得对不对？

### 验证方法

```sql
-- 1. 测试排序：按 amount 升序
SELECT * FROM orders ORDER BY amount ASC;
-- 金额从小到大：1899 → 2999 → 3499 → 6999 → 12999 ✅

-- 2. 测试排序：按 amount 降序
SELECT * FROM orders ORDER BY amount DESC;
-- 金额从大到小：12999 → 6999 → 3499 → 2999 → 1899 ✅

-- 3. 测试多列排序：先 user_id 升序，再 amount 降序
SELECT * FROM orders ORDER BY user_id ASC, amount DESC;
-- 同一 user_id 的订单按金额降序排列 ✅

-- 4. 测试分页：第 1 页 2 条，第 2 页 2 条
SELECT * FROM users ORDER BY id LIMIT 2 OFFSET 0;  -- id 1, 2
SELECT * FROM users ORDER BY id LIMIT 2 OFFSET 2;  -- id 3, 6 ✅

-- 5. 测试游标分页：id > 0 → 第 1 页；id > 2 → 第 2 页
SELECT * FROM users WHERE id > 0 ORDER BY id LIMIT 2;  -- id 1, 2
SELECT * FROM users WHERE id > 2 ORDER BY id LIMIT 2;  -- id 3, 6 ✅
```

## 不对怎么办？

### 常见错误 1：OFFSET 从 1 开始

❌ 错误做法：

```sql
-- 想取第 1 页，每页 10 条
SELECT * FROM users LIMIT 10 OFFSET 1;
-- 结果：跳过了第 1 行，第 1 页只有 9 条
```

✅ 正确做法：

```sql
-- OFFSET 从 0 开始，不是从 1 开始！
SELECT * FROM users LIMIT 10 OFFSET 0;  -- 第 1 页
SELECT * FROM users LIMIT 10 OFFSET 10; -- 第 2 页
```

### 常见错误 2：LIMIT 简化写法顺序记反

❌ 错误做法：

```sql
-- 想写 LIMIT 10 OFFSET 10 → 每页 10 条，偏移 10
SELECT * FROM users LIMIT 10, 10;
-- 对的，这就是 10 偏移，10 条数，顺序是对的
-- 但如果你写成：
SELECT * FROM users LIMIT 10 OFFSET 10;
-- 和上面写法的意思正好相反！
```

记住两种写法的差异：

```sql
-- 写法 1（标准）：LIMIT 条数 OFFSET 偏移量
SELECT * FROM users LIMIT 10 OFFSET 10;  -- 条数=10，偏移=10 ✅

-- 写法 2（MySQL 简化）：LIMIT 偏移量, 条数
SELECT * FROM users LIMIT 10, 10;  -- 偏移=10，条数=10 ✅

-- 两种写法结果相同，但顺序相反！
```

### 常见错误 3：不排序直接分页

❌ 错误做法：

```sql
SELECT * FROM users LIMIT 10 OFFSET 0;
```

结果：每次翻页可能出现重复或遗漏——因为没有 ORDER BY 的情况下，MySQL 返回数据的顺序不保证固定。

✅ 正确做法：**永远加上 ORDER BY！**

```sql
SELECT * FROM users ORDER BY id LIMIT 10 OFFSET 0;
```

排序稳定了，分页才能稳定。

### 常见错误 4：直接用 LIMIT 大 OFFSET 在生产环境

❌ 错误做法：

```sql
-- 100 万行数据，用户翻到第 1000 页
SELECT * FROM big_table ORDER BY id LIMIT 10 OFFSET 999000;
-- 这会很慢，甚至把数据库拖垮
```

✅ 正确做法：对于"无限下拉"或翻页很深的场景，使用游标分页：

```sql
-- 告诉前端最后一条 id，下次从 id > last_id 开始
SELECT * FROM big_table WHERE id > 999000 ORDER BY id LIMIT 10;
```

| 本章学了什么 | 对应命令/概念 | 注意事项 |
|-------------|-------------|---------|
| 单字段排序 | `ORDER BY 列名 ASC/DESC` | ASC 是默认升序，DESC 是降序 |
| 多列排序 | `ORDER BY col1 ASC, col2 DESC` | 每列可以单独指定 ASC/DESC；先排第一列，再排第二列 |
