## 48.1 栈（Stack）— 后进先出（LIFO）

栈的操作只有两个核心：**push（压入）** 和 **pop（弹出）**。

```
    push(3)
    ┌───┐
    │ 3 │ ← 栈顶
    ├───┤
    │ 2 │
    ├───┤
    │ 1 │
    └───┘
    pop() → 返回3，栈顶变成2
```

| 操作 | 说明 | 时间复杂度 |
|---|---|---|
| `push(e)` | 入栈，放入栈顶 | O(1) |
| `pop()` | 出栈，移除并返回栈顶 | O(1) |
| `peek()` | 查看栈顶但不移除 | O(1) |
| `isEmpty()` | 判断是否为空 | O(1) |

### 基于数组实现栈

```java
public class ArrayStack<E> {
    private Object[] array;
    private int top;       // 栈顶指针（指向下一个空位）
    private static final int DEFAULT_CAPACITY = 10;

    public ArrayStack() {
        array = new Object[DEFAULT_CAPACITY];
        top = 0;
    }

    public void push(E e) {
        if (top == array.length) {
            resize();
        }
        array[top] = e;
        top++;
    }

    @SuppressWarnings("unchecked")
    public E pop() {
        if (top == 0) throw new EmptyStackException();
        top--;
        E e = (E) array[top];
        array[top] = null;  // 防止内存泄漏
        return e;
    }

    @SuppressWarnings("unchecked")
    public E peek() {
        if (top == 0) throw new EmptyStackException();
        return (E) array[top - 1];
    }

    public boolean isEmpty() {
        return top == 0;
    }

    public int size() {
        return top;
    }

    private void resize() {
        int newCapacity = array.length * 2;
        array = Arrays.copyOf(array, newCapacity);
    }
}
```

### Java标准库中的栈

```java
// 方式1：Stack类（较老，继承自Vector，线程安全但慢）
Stack<Integer> stack = new Stack<>();
stack.push(1);
stack.push(2);
int top = stack.pop();  // 2
int peek = stack.peek(); // 1

// 方式2：Deque接口（推荐！官方文档建议用Deque代替Stack）
Deque<Integer> dequeStack = new ArrayDeque<>();
dequeStack.push(1);       // 入栈
dequeStack.push(2);
int top2 = dequeStack.pop();  // 2 — 出栈
int peek2 = dequeStack.peek(); // 1 — 查看栈顶
```

> 🤔 想多一点：为什么推荐ArrayDeque而不是Stack？Stack是Vector的子类，所有方法都被synchronized修饰，在高并发场景下锁粒度太大；在单线程场景下这些同步是不必要的开销。ArrayDeque没有同步，性能更好。

## 48.2 栈的应用场景

### 1. JVM调用栈（最重要的应用）

```java
public class StackTraceDemo {
    public static void main(String[] args) {
        methodA();
    }

    static void methodA() {
        methodB();
    }

    static void methodB() {
        methodC();
    }

    static void methodC() {
        throw new RuntimeException("报错了！");
    }
}

/* 输出（从栈顶到栈底，也就是从出错位置到调用者）：
Exception in thread "main" java.lang.RuntimeException: 报错了！
    at StackTraceDemo.methodC(StackTraceDemo.java:15)
    at StackTraceDemo.methodB(StackTraceDemo.java:11)
    at StackTraceDemo.methodA(StackTraceDemo.java:7)
    at StackTraceDemo.main(StackTraceDemo.java:3)
*/
```

JVM内部维护一个调用栈。每次调用方法，当前方法的"返回地址+局部变量"压栈；方法return时弹栈。这就是为什么无限递归会StackOverflowError——栈空间被撑爆了。

### 2. 括号匹配（经典面试题）

```java
public boolean isValid(String s) {
    Deque<Character> stack = new ArrayDeque<>();
    Map<Character, Character> pairs = Map.of(')', '(', '}', '{', ']', '[');

    for (char c : s.toCharArray()) {
        if (pairs.containsValue(c)) {
            stack.push(c);                           // 左括号入栈
        } else if (pairs.containsKey(c)) {
            if (stack.isEmpty() || stack.pop() != pairs.get(c)) {
                return false;                        // 右括号不匹配
            }
        }
    }
    return stack.isEmpty();  // 栈空了才算全匹配
}

// 测试：
// "()"     → true
// "()[]{}" → true
// "(]"     → false
// "([)]"   → false
// "{[]}"   → true
```

### 3. 浏览器的前进后退

```
浏览A → 浏览B → 浏览C → 点后退
  后退栈: [A, B]       前进栈: [C]

实现：两个栈，一个存后退历史，一个存前进历史。
点后退：从后退栈pop一个页面显示，同时push到前进栈。
点前进：从前进栈pop一个页面显示，同时push到后退栈。
```

## 48.3 队列（Queue）— 先进先出（FIFO）

```
入队(enqueue)             出队(dequeue)
    ↓                         ↑
┌───┬───┬───┬───┬───┐
│ 1 │ 2 │ 3 │ 4 │ 5 │
└───┴───┴───┴───┴───┘
队尾(rear)              队头(front)
```

| 操作 | 说明 | `offer`/`poll`系列 | `add`/`remove`系列 |
|---|---|---|---|
| 入队 | 放入队尾 | `offer(e)` → 失败返回false | `add(e)` → 失败抛异常 |
| 出队 | 从队头取出 | `poll()` → 空返回null | `remove()` → 空抛异常 |
| 查看队头 | 不移除 | `peek()` → 空返回null | `element()` → 空抛异常 |

### 基于链表实现队列

```java
public class LinkedQueue<E> {
    private static class Node<E> {
        E data;
        Node<E> next;
        Node(E data) { this.data = data; }
    }

    private Node<E> front;  // 队头
    private Node<E> rear;   // 队尾
    private int size;

    // 入队 O(1)
    public void offer(E e) {
        Node<E> newNode = new Node<>(e);
        if (rear == null) {
            front = rear = newNode;
        } else {
            rear.next = newNode;
            rear = newNode;
        }
        size++;
    }

    // 出队 O(1)
    public E poll() {
        if (front == null) return null;
        E data = front.data;
        front = front.next;
        if (front == null) rear = null;  // 队列空了
        size--;
        return data;
    }

    public E peek() {
        return front == null ? null : front.data;
    }

    public boolean isEmpty() { return size == 0; }
    public int size() { return size; }
}
```

### Java标准库中的队列

```java
// Queue是一个接口，常用实现：
// 1. LinkedList — 基于双向链表
Queue<String> q1 = new LinkedList<>();
q1.offer("A");
q1.offer("B");
System.out.println(q1.poll());  // "A"

// 2. ArrayDeque — 基于循环数组（通常比LinkedList快）
Queue<String> q2 = new ArrayDeque<>();
q2.offer("X");
q2.offer("Y");
System.out.println(q2.poll());  // "X"
```

## 48.4 BFS（广度优先搜索）— 队列的核心应用

BFS的本质：**先把眼前的所有可能探索完，再深入下一层。** 而"先到先处理"正是队列的特性。

```java
// BFS遍历二叉树的层序遍历
public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;

    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);

    while (!queue.isEmpty()) {
        int levelSize = queue.size();
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.poll();
            level.add(node.val);
            if (node.left != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        result.add(level);
    }
    return result;
}
// 输入:    1
//        /   \
//       2     3
//      / \   / \
//     4   5 6   7
// 输出：[[1], [2,3], [4,5,6,7]]
```

**BFS在后端的实际用途**：
- 网页爬虫：先抓取当前页的所有链接，再逐层深入
- 社交网络"2度好友"：从你出发，BFS两层
- 微服务调用链分析：BFS遍历服务依赖图，找出最短调用路径
- 数据库连接池：用队列管理空闲连接，先归还的先被复用

## 48.5 队列在后端的核心应用：消息队列

虽然你不会在本教程里手写RabbitMQ，但理解队列的数据结构是理解MQ的前提：

```
生产者                                    消费者
  │                                        │
  │  publish("订单创建")                    │  consume()
  ▼                                        ▼
┌─────────────────────────────────────────────┐
│              Queue（队列）                    │
│  [msg1] [msg2] [msg3] [msg4] ...          │
└─────────────────────────────────────────────┘
        队尾 ← ← ← ← ← ← → → → → → 队头
```

消息队列的核心特性直接来自队列数据结构：
- **削峰填谷**：请求瞬间涌入→排队→消费者匀速处理，系统不崩溃
- **解耦**：生产者和消费者不需要知道对方的存在，都只和队列打交道
- **顺序保证**：FIFO（部分MQ保证，如Kafka同一分区内有序）

## 48.6 ❌ 常见错误

| 错误 | 后果 | 正确做法 |
|---|---|---|
| 用`Stack`类（继承Vector） | 不必要的同步开销 | 用`ArrayDeque`实现栈 |
| `pop()`/`poll()`之前不判空 | `EmptyStackException` / NPE | 先`isEmpty()`或`peek()!=null` |
| BFS用栈代替队列 | 变成DFS了 | BFS必须用Queue，DFS用Stack |
| 队列实现中`poll`后忘记更新rear | 出队后rear仍指向被移除的节点 | `front==null`时rear也要置null |
| 递归太深 | StackOverflowError | 改迭代或用尾递归优化 |

## 小结

| 概念 | 规则 | 经典操作 | 后端应用 |
|---|---|---|---|
| 栈 Stack | LIFO 后进先出 | push/pop/peek | JVM调用栈、括号匹配、撤销操作 |
| 队列 Queue | FIFO 先进先出 | offer/poll/peek | 消息队列、BFS、请求排队 |
| Deque | 双端，既是栈也是队列 | push/pop/offer/poll | `ArrayDeque`推荐用于栈和队列 |
| BFS | 用队列逐层展开 | offer→while→poll→offer子节点 | 爬虫、最短路径、层序遍历 |
