# 27-Tab补全

> **本篇属于**：第三篇·Cursor从入门到精通 | **预计阅读**：15分钟 | **难度**：★★☆☆☆
>
> **前置条件**：Cursor已安装并登录，打开了任意Python文件。

Tab补全是Cursor里**用得最多、最省心、最容易上瘾**的功能。你打字的时候，AI在背后实时预测你想写什么，然后弹出灰色建议——你Tab键一按，搞定。这就像你手机上的输入法联想，但要比那个强100倍。

---

## 27.1 Tab补全是什么？

> **📖 术语：Tab补全**——Cursor在你打字时，实时分析你当前文件的内容和上下文，预测你接下来要写什么代码，然后用**灰色文字**显示出来。你按Tab键接受，按Esc键拒绝。

**一个比喻**：你弹钢琴，记得开头几个音符，但后面忘了。这时候有个老师坐在你旁边，听到你弹了一半，就轻声哼出后面的旋律——你顺着他的提示弹下去就行了。Tab补全就是这个"老师"。

---

## 27.2 怎么用：三个操作就够了

### **1.** 接受补全：按 **`Tab`**

AI的建议是灰色的，只要你按一下Tab键，灰色文字就变成你写的代码了。

### **2.** 拒绝补全：按 **`Esc`**

如果AI给的建议不是你想要的，按Esc，灰色文字消失，你继续写。

### **3.** 部分接受：按 **`Ctrl + →`**（方向键右键）

AI可能猜对了大半，但最后几个字不对。你不想全部接受，也不想全部拒绝——这时候用 `Ctrl + →` 可以一个字一个字地接受AI的补全。

---

## 27.3 实际演示：从一个注释开始

打开Cursor，新建一个文件 `fibonacci.py`，然后跟着做：

### **1.** 先写注释，告诉AI你要做什么

输入：

```python
# 计算斐波那契数列第n项
```

### **2.** 换行，开始写def

在新的一行输入 `def`，然后停一下——

AI会立刻弹出灰色补全，大概率是：

```python
def fibonacci(n):
```

✅ 按Tab接受。

### **3.** 继续写函数体

AI会继续补全函数体，你按Tab一步一步接受，最终可能得到：

```python
# 计算斐波那契数列第n项
def fibonacci(n):
    if n <= 0:
        return 0
    elif n == 1:
        return 1
    else:
        return fibonacci(n - 1) + fibonacci(n - 2)
```

### **4.** 验证它能跑

加一行测试代码，AI也会帮你补全 `print` 调用：

```python
print(fibonacci(10))
```

保存，终端运行：

```bash
python fibonacci.py
```

输出：

```
55
```

✅ 斐波那契数列第10项确实是55——AI写的代码是对的！

## 27.4 Tab补全的最佳实践

### 最佳实践1：先写注释，再补全

这是**最核心的技巧**。AI不是读心术——它不知道你想干什么。但如果你先写一行注释说清楚意图，AI就能精准补全。

❌ **错误做法**：直接写 `def`，然后期望AI猜到你要写什么函数。
✅ **正确做法**：先写 `# 计算两个日期的天数差`，然后换行写 `def`。

### 最佳实践2：给AI足够的上下文

AI补全参考的是**你当前文件的内容**。如果你文件里已经有了函数A和函数B，那么AI在补全函数C时，会参考函数A和B的命名风格、参数格式、返回值类型——让补全更准确。

### 最佳实践3：描述性命名

变量名越清晰，AI补全越准。比如：

❌ `def calc(a, b)` —— AI猜不到你要算什么
✅ `def calculate_monthly_payment(principal, annual_rate, months)` —— AI一看就知道这是算月供的

## 27.5 Tab补全的局限性

Tab补全很强，但**不是万能**的。它有这些限制：

| 限制 | 说明 | 怎么办 |
|------|------|--------|
| 只能补一小段 | 最多补充几行代码 | 大任务用Chat或Composer |
| 只看当前文件 | 不跨文件理解 | 跨文件操作用Chat |
| 不一定总是对 | 可能生成有bug的代码 | 运行验证，别盲目信任 |
| 需要已经写了开头 | 你得先写点什么 | 先写注释或函数签名 |

**一句话总结**：Tab补全是"给你搭句子"的，Chat和Composer是"给你写文章"的。

## 27.6 如果出错了怎么办

### 🔴 问题1：灰色的补全根本不出现

**可能原因**：
1. 你没登录Cursor
2. 免费版补全额度用完了（每月2000次）
3. 网络问题

**解决**：
- 检查右上角有没有头像，没登录就登录
- 检查Cursor设置里AI功能是否开启
- 切到GPT-4o-mini（免费无限额度）

### 🔴 问题2：补全的内容和我想的不一样

**这是正常的**。AI不是读心术，它靠上下文猜。你写的越多，它猜的越准。

**解决**：
- 按Esc拒绝，自己写
- 或者先写一行注释，明确告诉AI你要什么，然后再写

### 🔴 问题3：补全的内容有错误

**这也正常**。AI补全是概率预测，不是100%正确。

**解决**：写完后运行验证一下，发现了错误就用Ctrl+K让AI帮你改，或者自己手动改。

## 27.7 练习：用Tab补全写一个函数

**题目**：写一个函数 `is_prime(n)`，判断一个数是不是素数。

**步骤**：
1. 新建文件 `prime.py`
2. 先写注释：`# 判断一个数是不是素数`
3. 换行写 `def is_prime(n):`，然后让AI补全函数体
4. 加测试代码 `print(is_prime(7))` 和 `print(is_prime(10))`
5. 运行验证：7是素数→True / 10不是素数→False

> 提示：如果AI补全的内容不对，按Esc拒绝，自己调整一下函数签名，再让AI补。
