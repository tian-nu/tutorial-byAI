# 32-上下文管理进阶

> **本篇属于**：第三篇·Cursor从入门到精通 | **预计阅读**：20分钟 | **难度**：★★★☆☆
>
> **前置条件**：已掌握Chat模式（第28章）和Composer模式（第29章）。

你在第28章学会了用 `@file` 引用文件。但 `@` 符号的威力远不止于此——这一章，我们深入挖掘Cursor的上下文管理能力，让你精准控制"AI能看到什么"。

---

## 32.1 为什么上下文管理很重要？

一个关键认知：**AI给不给力，60%取决于你给了它什么上下文。**

给太多 → AI记不住，开始乱说
给太少 → AI不了解项目，凭空瞎编
给不对 → AI误解你的意图，改错地方

**一句话**：不是越多越好，而是**只给AI需要的**。

---

## 32.2 @file：引用单个文件

**用法**：在Chat或Composer输入框里输入 `@file`，然后选文件。

**示例**：

```
@file calculator.py 解释一下这个文件的整体结构
```

**什么时候用**：只需要看一个文件时（比如改一个函数、解释一个文件）。

---

## 32.3 @folder：引用整个文件夹

**用法**：在Chat或Composer输入框里输入 `@folder`，然后选文件夹。

**示例**：

```
@folder src/utils/ 帮我梳理一下这个工具模块，每个文件是干什么的，有哪些公共函数
```

**什么时候用**：需要了解一个模块的整体结构时。

**⚠️ 注意**：如果文件夹里有几十个文件，AI可能会被信息淹没——上下文太多，回答质量下降。如果文件超过10个，建议只引用核心文件。

---

## 32.4 @web：搜索网络

**用法**：在Chat或Composer输入框里输入 `@web`，然后输入搜索关键词。

**示例**：

```
@web Python 3.12 match-case 语法
```

**什么时候用**：需要查阅最新文档、API用法、第三方库教程时。

**⚠️ 注意**：`@web` 需要联网，不是所有模型都支持。GPT-4o支持，Claude不一定。

---

## 32.5 @git：引用Git变更

**用法**：在Chat或Composer输入框里输入 `@git`，AI会看到你最近的Git修改。

**示例**：

```
@git 帮我看看我最近的改动，有什么潜在问题吗？
```

**什么时候用**：做好了一个功能，想检查一下改动有没有Bug。

---

## 32.6 Notepads：Cursor的长期记忆

Notepads是Cursor一个隐藏但实用的功能——你可以把它理解为**Cursor的备忘录**。

**怎么用**：
1. 在右侧AI面板，点"Notepads"标签（如果没有，在Chat面板里搜索"Notepads"）
2. 新建一个Notepad，写一些项目的重要信息——比如项目架构、数据库表结构、API接口列表
3. 以后在Chat里，你可以引用这个Notepad，AI会看到里面的内容

**适合存什么**：
- 项目的整体架构描述
- 常用的API接口列表
- 数据库表结构
- 特殊约定和规则（比 `.cursorrules` 更长的内容）

**和 `.cursorrules` 的区别**：
- `.cursorrules`：AI**自动**读取，每次对话都生效
- Notepads：**手动**引用，只在需要时给AI看

## 32.7 上下文预算：怎么看用了多少Tokens？

> **📖 术语：Token**——AI处理文本的最小单位。一个Token大约等于3-4个汉字，或者一个英文单词。AI的上下文窗口是有限的，用完了就"溢出"了。

在Cursor里，你能看到当前用了多少上下文吗？

**当前没有直接的Token计数器**，但你可以通过以下方式判断：
1. 如果AI开始"忘记"你前面说的话 → 上下文快满了
2. 如果AI回答明显变短变敷衍 → 上下文快满了
3. 如果AI突然开始胡说八道 → 上下文已经溢出

**经验判断**：Chat模式大概能记住20-30轮对话（取决于你给的文件大小），超过这个范围，旧的对话会被挤出去。

## 32.8 经验：AI答非所问的时候怎么办？

**这是最常见的问题，值得单独列出来。**

症状：你问AI一个问题，AI的回答和你的问题完全不沾边。

**最可能的三个原因**：

1. **上下文太多**：AI面前堆了太多信息，它挑错了重点
2. **上下文太乱**：你前后给了互相矛盾的信息，AI混乱了
3. **AI猜错了**：AI误解了你的意图

**标准三步解决法**：

**步骤1**：清空上下文
- 点Chat右上角"New Chat"新建对话

**步骤2**：精简上下文
- 只引用AI真正需要的文件（用 `@file` 只给1-2个核心文件，不要一上来就 `@folder` 整个项目）

**步骤3**：重新描述问题
- 比上次更具体、更精确地描述你想要的

## 32.9 上下文管理的黄金法则

| 法则 | 说明 |
|------|------|
| **够用就行** | 只给AI完成当前任务需要的信息，不多给 |
| **先小后大** | 先给核心文件，不够再加，不要一上来就全给 |
| **定期清零** | 对话太长就新建Chat，避免上下文溢出 |
| **明确引用** | 用@file/@folder明确告诉AI看什么，不要指望AI"自己发现" |

---

## 如果出错了怎么办

### 🔴 问题1：AI回答和我的问题完全无关

**最可能的原因**：上下文太多，AI挑错了重点。

**解决**：新建Chat，只引用关键文件，重新提问。

### 🔴 问题2：AI"忘记"了我们之前讨论的内容

**原因**：上下文溢出，旧的对话被挤出去了。

**解决**：新建Chat，把关键信息在新对话里重新告诉AI。

### 🔴 问题3：@file引用了一个大文件，AI直接卡住了

**原因**：文件太大，单个文件太多Token，AI处理不过来。

**解决**：不要引用整个文件。在Chat里把文件的关键部分手动粘贴过去，或者先用 `@file` 引用，然后说"只看第100-200行"。

## 自测

1. 你有一个项目，文件结构是 `src/models/`、`src/views/`、`src/controllers/`。你想让AI理解整个项目的结构，应该怎么做？
2. 你和AI聊了30轮，突然AI开始胡说八道，最可能的原因是什么？怎么解决？
3. @file和@folder有什么区别？什么时候用哪个？

> **答案**：
> 1. 不要一次性 `@folder src/` 整个项目。先新建Chat，用 `@file` 引用3-4个核心文件，描述清楚项目结构。或者用Notepads先写好项目架构，再引用Notepad。
> 2. 上下文溢出——旧的对话被挤出去了。新建Chat，把关键信息重新告诉AI。
> 3. @file引用单个文件，适合只看一个文件时。@folder引用整个文件夹，适合需要了解模块结构时。但@folder如果文件太多AI会处理不过来，慎用。
