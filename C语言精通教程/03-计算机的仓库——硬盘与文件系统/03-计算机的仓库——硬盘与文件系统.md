## 我在做什么？

上一章讲了内存——计算机的"临时记忆"。但内存有个致命缺陷：**断电就忘**。你写的代码、下载的电影、做了一天的 PPT，不能只靠内存存着——它们需要能"永久保存"的地方。

这一章讲硬盘和文件系统。读完你能理解：为什么你的文件能保存到明天，为什么 C 语言读写文件比读写内存慢，以及"绝对路径"和"相对路径"到底怎么区分。

## 一、硬盘和内存的区别

### 1.1 仓库 vs 桌面

想象你在办公室工作：

- **桌面**是你的工作区。你正在处理的文件摊在桌面上，伸手就能拿到。但桌面空间有限，下班后保洁阿姨会清掉。
- **仓库**是文件的永久存放地。文件盒堆在货架上，找起来慢，但不会丢。

```
                   桌面（内存 / RAM）
              +-----------------------+
              |  [当前文档] [计算器]   |  <-- 处理速度快，但断电就清空
              |  [浏览器标签页]       |      空间小（通常 8GB-32GB）
              +-----------------------+
                          |
                          | 保存/加载
                          v
              +-----------------------+
              | 仓库（硬盘 / Disk）   |  <-- 处理速度慢，但断电不丢
              |  [文档] [照片] [代码]  |      空间大（通常 256GB-2TB）
              |  [电影] [游戏] [......]|
              +-----------------------+
```

一句话总结：

| 特性 | 内存（RAM） | 硬盘（HDD/SSD） |
|------|-----------|----------------|
| 速度 | 极快（纳秒级） | 慢（微秒到毫秒级） |
| 断电后 | 数据消失 | 数据保留 |
| 容量 | 小（8-32GB） | 大（256GB-2TB+） |
| 价格 | 贵（每 GB） | 便宜（每 GB） |
| 专业术语 | *易失性* *此术语见附录F* | *持久化* *此术语见附录F* |

### 1.2 两种硬盘：HDD vs SSD

硬盘有两种主流类型：

**HDD（机械硬盘）** *此术语见附录F*：
- 里面有一个旋转的磁盘和一根磁头臂。
- 就像老式唱片机——要读数据，磁头必须物理移动到正确位置。
- 慢，但便宜，容量大。

```
HDD 内部结构（示意）：

        旋转的磁盘
      +------------------------+
      |    ······              |
      |  ···········           |
      | ··············         |  <-- 磁头臂来回移动读数据
      |  ···········           |      就像唱片机的唱针
      |    ······              |
      +------------------------+
              ↑
          磁头臂 (来回移动)
```

**SSD（固态硬盘）** *此术语见附录F*：
- 没有机械部件，纯电子元件。
- 像一个大号的 U 盘。
- 比 HDD 快很多，但比内存仍然慢。
- 现在的笔记本电脑大多用 SSD。

### 1.3 为什么读写文件比读写内存慢？

速度差异来源于两个层面：

1. **物理层面**：内存是纯电路，数据以光速的 2/3 在导线中传输。机械硬盘需要物理移动磁头，这是机械运动，慢好几个数量级。

2. **软件层面**：即使是最快的 SSD，读写文件也要经过操作系统的好几层"关卡"——系统调用、文件系统、设备驱动等。每次读写都要"过安检"。

```text
读内存变量的路径：
  程序 -> CPU -> 内存控制器 -> 内存芯片 -> 数据返回
  (直接，没有中间商)

读文件的路径：
  程序 -> C标准库 -> 操作系统内核 -> 文件系统 -> 设备驱动 -> 硬盘控制器 -> 硬盘 -> 数据返回
  (层层安检，每层都慢一点)
```

## 二、文件是什么？

### 2.1 文件 = 一串字节 + 一个名字

从计算机的视角看，文件极其简单——就是**一串连续的字节，加上一个名字**。

```text
文件 "hello.txt"

名字：hello.txt
内容：一串字节

H  e  l  l  o  \n  W  o  r  l  d
72 101 108 108 111 10 87 111 114 108 100

每个字符对应一个数字（ASCII 码），文件就是这些数字排成一排
```

文件没有"类型"——`.txt`、`.jpg`、`.exe` 只是名字的一部分，用来提示操作系统和人类"这个文件大概是干什么的"。对于计算机来说，所有文件都是字节流。

```c
// C 语言读写文件的核心思想：
// 打开一个字节流，读或写，关闭

FILE *fp = fopen("data.txt", "r");   // 以"读"模式打开
// ... 读或写操作 ...
fclose(fp);                           // 关闭
```

### 2.2 文件是"持久化"的唯一方式

你在程序里创建了一个变量：

```c
int x = 42;
```

这个 `x` 存在内存中。程序一关，`x` 就没了——就像你在桌面上用铅笔写了个数字，下班保洁阿姨擦掉了。

如果你想让 `x` 明天还在，就必须把它**写进文件**：

```c
// 持久化：把数据存到硬盘
FILE *fp = fopen("save.txt", "w");
fprintf(fp, "%d", x);   // 把 x 的值写进文件
fclose(fp);
// 现在 x 的值永久保存在硬盘上了
```

## 三、文件系统

### 3.1 文件系统是什么？

硬盘里可能存了几十万个文件。如果它们只是无序地堆在一起，找起来会是一场噩梦。

*文件系统* *此术语见附录F* 就是硬盘的"目录管理员"——它负责组织文件，让你能用名字找到文件，而不是记住它在硬盘上的物理位置。

```text
硬盘如果没有文件系统：
  +------------------------------------------------+
  | 字节字节字节字节字节字节字节字节字节字节字节字节   |
  | 字节字节字节字节字节字节字节字节字节字节字节字节   |
  | (一个巨大的、无结构的字节堆)                      |
  +------------------------------------------------+

硬盘有了文件系统：
  +------------------------------------------------+
  | /home/user/                                     |
  |   ├── documents/                                |
  |   │   ├── report.docx                           |
  |   │   └── notes.txt                             |
  |   ├── photos/                                   |
  |   │   ├── vacation.jpg                          |
  |   │   └── family.jpg                            |
  |   └── code/                                     |
  |       └── hello.c                               |
  +------------------------------------------------+
```

### 3.2 目录树

文件系统用树状结构组织文件：

```
                    / (根目录)
                      |
        +-------------+-------------+
        |             |             |
      home           etc           usr
        |             |             |
    +---+---+      +--+--+       +--+--+
    |       |      |     |       |     |
  user1   user2  passwd hosts   bin   lib
    |
  +-+-+
  |   |
docs  hello.c
```

- **根目录**：最顶层，用 `/` 表示（Windows 用 `C:\`，Linux/macOS 用 `/`）。
- **目录（文件夹）**：可以包含文件和其他目录的"容器"。
- **文件**：树的叶子节点，存放实际数据。

## 四、绝对路径 vs 相对路径

### 4.1 比喻：两种问路方式

你要找一家餐厅，有两种问路方式：

**绝对路径**：从地球开始描述。
> "地球 -> 亚洲 -> 中国 -> 北京市 -> 朝阳区 -> 幸福路 -> 100 号"

**相对路径**：从你现在的位置开始描述。
> "从你现在站的地方，往前走 200 米，右转就到了。"

```text
假设你的当前位置是 /home/user/

目标文件：/home/user/documents/report.txt

绝对路径：/home/user/documents/report.txt
          (从根目录 / 开始，一步一步走)

相对路径：documents/report.txt
          (从当前位置 /home/user/ 开始，走 documents/ 子目录)
```

### 4.2 两个特殊符号

在相对路径中，有两个极其重要的符号：

| 符号 | 含义 | 比喻 |
|------|------|------|
| `.` | 当前目录 | "我脚下" |
| `..` | 上级目录 | "往上一层楼" |

```text
当前位置：/home/user/documents/

.   = /home/user/documents/       (当前目录)
..  = /home/user/                 (上一级)
../.. = /home/                    (上两级)

./report.txt  = 当前目录下的 report.txt
../photos/vacation.jpg  = 往上一级，进入 photos 目录，找 vacation.jpg
```

### 4.3 C 语言中的路径

```c
// 绝对路径：从根目录开始
FILE *fp1 = fopen("/home/user/data.txt", "r");

// 相对路径：从程序运行时的"当前目录"开始
FILE *fp2 = fopen("data.txt", "r");           // 当前目录下的文件
FILE *fp3 = fopen("./data.txt", "r");         // 同上，显式写"当前目录"
FILE *fp4 = fopen("../data.txt", "r");        // 上一级目录下的文件
FILE *fp5 = fopen("subdir/data.txt", "r");    // 子目录下的文件
```

## 五、文件的"打开"和"关闭"

### 5.1 为什么需要"打开"文件？

你不能直接对着硬盘上的文件操作。必须先"打开"它——这相当于告诉操作系统："我要用这个文件了，请帮我准备好。"

操作系统会做以下事情：
1. 检查文件是否存在，你有没有权限。
2. 在内存中分配一个"缓冲区"，作为中转站。
3. 返回一个"文件指针"（`FILE*`），你通过它来操作文件。

```text
程序                    操作系统                    硬盘
  |                       |                          |
  |--- fopen("a.txt") --->|                          |
  |                       |--- 检查文件是否存在 ----->|
  |                       |<-- 存在，权限OK ---------|
  |                       |                          |
  |<-- 返回 FILE* -------|                          |
  |                       |                          |
  |--- fread(buf) ------>|                          |
  |                       |--- 读数据 -------------->|
  |                       |<-- 数据块 --------------|
  |<-- 数据到buf --------|                          |
  |                       |                          |
  |--- fclose() -------->|                          |
  |                       |--- 清理缓冲区等 ---------|
  |                       |                          |
```

### 5.2 为什么必须"关闭"文件？

1. **数据可能还没真正写入硬盘**：操作系统为了性能，会把数据先攒在缓冲区里，攒够一批再写。关闭文件时，缓冲区会被"强制刷新"到硬盘。不关文件，你的数据可能丢了。
2. **操作系统能同时打开的文件数量有限**：不关文件会耗尽资源。

```c
// 正确做法：打开 -> 使用 -> 关闭
FILE *fp = fopen("data.txt", "r");
if (fp == NULL) {
    // 打开失败！文件可能不存在，或没有权限
    perror("打开文件失败");
    return;
}
// ... 使用 fp ...
fclose(fp);   // 用完必须关！
fp = NULL;    // 好习惯：避免使用已关闭的指针
```

## 本章小结

| 概念 | 一句话解释 |
|------|-----------|
| 硬盘 | 计算机的永久存储设备，断电后数据不丢失 |
| HDD | 机械硬盘，用旋转磁盘和磁头臂读写数据，慢但便宜 |
| SSD | 固态硬盘，纯电子元件，比 HDD 快，比内存慢 |
| 易失性 | 断电后数据消失的特性（内存有，硬盘没有） |
| 持久化 | 把数据从内存保存到硬盘，使其在程序结束后仍存在 |
| 文件系统 | 硬盘上组织和管理文件的"目录管理员" |
| 绝对路径 | 从根目录开始描述的完整文件路径 |
| 相对路径 | 从当前目录开始描述的文件路径 |

## 我做得对不对？

验证方法——你能用自己的话回答以下问题吗？

1. **内存和硬盘的本质区别是什么？** —— 内存快、小、断电就忘；硬盘慢、大、断电不丢。内存是"工作区"，硬盘是"仓库"。
2. **绝对路径和相对路径怎么区分？** —— 绝对路径从根目录 `/` 开始，描述完整路线；相对路径从当前目录开始，用 `.` 和 `..` 导航。
3. **文件是什么？** —— 文件是一串连续的字节加一个名字。文件名后缀（.txt、.jpg）只是给人类和操作系统看的提示，底层都是字节。

## 不对怎么办？

**常见困惑 1**："我用的 Windows 系统，路径是 `C:\Users\...`，不是 `/home/...`。我该学哪个？"

解答：不同操作系统用不同的路径写法。Windows 用反斜杠 `\` 和盘符 `C:`，Linux/macOS 用正斜杠 `/` 和根目录 `/`。但**路径的层级概念是完全一样的**——都是从根出发，一层一层往下找。C 语言中，在 Windows 上写路径时要么用双反斜杠 `"C:\\Users\\..."`，要么用正斜杠 `"C:/Users/..."`（大多数 Windows 编译器支持）。

**常见困惑 2**："我从来没注意过绝对路径和相对路径的区别，这重要吗？"

解答：在写 C 程序时非常重要。如果你写 `fopen("data.txt", "r")`，程序会在**运行时的当前目录**找这个文件，而不是在源代码所在的目录。这两个目录可能完全不同。搞不清这个区别，你就会遇到"文件明明存在，程序却说找不到"的诡异 bug。

**常见困惑 3**："SSD 也是电子的，为什么比内存慢？"

解答：因为接口和协议。内存通过专用的高速总线直接连到 CPU，SSD 要通过 SATA 或 NVMe 接口，走一套更复杂的通信协议。另外，SSD 的存储单元（NAND 闪存）本身读写速度就不如内存的 DRAM 单元快。简单说：内存是直连 CPU 的"VIP 通道"，SSD 走的是"普通通道"。
