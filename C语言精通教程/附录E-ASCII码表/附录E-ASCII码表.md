## 我在做什么？

ASCII（American Standard Code for Information Interchange，美国信息交换标准代码）是计算机中最基础的字符编码。每个英文字母、数字、标点符号对应一个 0-127 的整数。本附录提供完整对照表，方便你查阅字符与数字的对应关系。

## E.5 大小写转换技巧

```c
char to_upper(char c) {
    if (c >= 'a' && c <= 'z') {
        return c - 32;  // 或 c - ('a' - 'A')
    }
    return c;
}

char to_lower(char c) {
    if (c >= 'A' && c <= 'Z') {
        return c + 32;  // 或 c + ('a' - 'A')
    }
    return c;
}
```

> 推荐使用 `<ctype.h>` 中的 `toupper()` 和 `tolower()`，它们更安全、可移植。

## E.6 三问

1. **为什么 `'A'` 是 65？** 这是历史原因。ASCII 编码在 1960 年代制定，65 是设计者选择的编码值。没有特殊的数学原因，就是一个标准约定。

2. **`'\n'` 和 `'\r'` 有什么区别？** `'\n'`（LF，换行）将光标移到下一行同一列；`'\r'`（CR，回车）将光标移到当前行开头。Windows 用 `\r\n` 组合表示换行，Linux/macOS 用 `\n`。

3. **ASCII 能表示中文吗？** 不能。ASCII 只有 128 个字符，只覆盖英文。中文需要 Unicode（UTF-8 编码），每个中文字符占 3-4 字节。
