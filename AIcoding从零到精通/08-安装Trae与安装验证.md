在第07章，你装了 Cursor——一个 AI 编程工具。但 Cursor 不是唯一的选择。这一章，我们装第二个工具：**📖 Trae**。

**为什么装两个？** 因为不同的 AI 编程工具各有擅长。就像你既有菜刀也有水果刀——菜刀切大块，水果刀削细节。Cursor 和 Trae 配合使用，能覆盖 AI 编程的绝大部分场景。

## 8.1 Trae 是什么？——"AI 帮你建房子"

> **📖 术语：Trae**——字节跳动出品的 AI 编程 IDE（集成开发环境）。它和 Cursor 一样，都是"AI 帮你写代码"的工具，但设计理念不同。

**一个比喻帮你理解区别**：

| 场景 | Cursor 的方式 | Trae 的方式 |
|------|-------------|------------|
| 你想建一个房子 | 你告诉工人"这块砖放这里，那块砖放那里"——你一步步指挥 | 你告诉建筑师"我要一个三室两厅"——AI 直接帮你搭好框架 |
| 你想写一个网站 | 你打开文件，Ctrl+K 一步步让 AI 写代码 | 你打开 Builder，一句话描述，AI 生成整个项目结构 |

**Trae 的核心理念是"Builder 模式"**——你告诉 AI 你想要什么，AI 不是只写一行代码，而是帮你**搭建整个项目**。

> **📖 术语：Builder 模式**——Trae 独有的功能。你给出一段自然语言描述（比如"帮我做一个待办事项应用"），Builder 会**自动生成多个文件、自动搭建项目结构、自动安装依赖**。你只需要一句话，它帮你把整个"房子"盖起来。

## 8.2 Cursor vs Trae：两张表看懂区别

### 产品理念对比

| 维度 | Cursor | Trae |
|------|--------|------|
| 出品方 | Cursor Inc.（美国） | 字节跳动（中国） |
| 底层基础 | 基于 VS Code 改造 | 基于 VS Code 改造 |
| 核心优势 | 代码补全极强、行内编辑丝滑 | Builder 模式、中文理解好 |
| 最适合的场景 | 逐行修改代码、代码补全 | 从零搭建项目、快速原型 |
| 国内访问 | 可以访问，但偶尔慢 | 顺畅，几乎不卡 |
| 登录方式 | 邮箱 / Google / GitHub | 国内手机号直接登录 |
| 免费额度 | 2000次/月补全 + 50次高级对话 | 免费额度充足（具体以官网为准） |

### 本书推荐的使用策略

| 场景 | 用什么工具 |
|------|-----------|
| 新建一个项目，从零开始 | **Trae Builder**（一句话生成整个项目） |
| 修改现有代码，微调某个功能 | **Cursor**（Ctrl+K 行内编辑更顺手） |
| 需要让 AI 看大量代码，找 Bug | **Cursor**（Claude 3.5 长文本能力强） |
| 国内网络不稳定，Cursor 卡顿 | **Trae**（国内服务器，访问顺畅） |
| 快速原型，想先看看效果 | **Trae Builder**（几分钟出完整项目） |

**简单总结**：**Trae 负责"从 0 到 1"，Cursor 负责"从 1 到 100"**。

## 8.3 下载安装 Trae

### Step 1：下载

1. 打开浏览器，访问 Trae 官网：**https://www.trae.ai**
2. 页面会自动识别你的操作系统，点击下载按钮
3. 等待下载完成（大约 200-300MB，3-5 分钟）

> Trae 的服务器在国内，下载速度通常比 Cursor 快很多。如果你之前下 Cursor 很慢，Trae 应该会让你感到惊喜。

### Step 2：Windows 安装

1. 双击下载的 `.exe` 安装文件
2. 安装程序会问你是否要安装——点击 **"是"**
3. 安装路径用默认的就行，点击 **"下一步"**
4. 勾选"创建桌面快捷方式"（推荐，方便以后打开）
5. 点击 **"安装"**
6. 等待进度条走完（大约 1 分钟）
7. 安装完成后，勾选"启动 Trae"，点击 **"完成"**

### Step 3：Mac 安装

1. 双击下载的 `.dmg` 文件
2. 把 Trae 图标**拖拽到 Applications（应用程序）文件夹**
3. 等待拷贝完成（大约 10 秒）
4. 打开 Applications 文件夹，双击 Trae 启动

> ⚠️ 和 Cursor 一样，如果 Mac 弹出"无法验证开发者"，去"系统设置" → "隐私与安全性" → 点击"仍要打开"。

## 8.4 首次启动与登录

### 登录方式

Trae 的登录方式对国内用户特别友好：

1. **手机号登录**：输入国内手机号，获取验证码，输入验证码即可登录。**这是最推荐的方式**。
2. **邮箱登录**：如果你不想用手机号，也可以用邮箱注册。
3. **第三方登录**：支持 Google、GitHub 等第三方账号登录。

> **Trae 的最大优势之一**：国内手机号直接登录，不需要任何翻墙工具，不需要 Google 账号，不需要 GitHub 账号。对国内用户来说，这比 Cursor 方便太多了。

### 首次设置向导

登录后，Trae 同样会引导你做初始设置：

| 设置项 | 推荐选择 | 原因 |
|--------|---------|------|
| 键盘快捷键 | Trae 默认 | 和 Cursor 类似，后面统一学 |
| 颜色主题 | 按个人喜好 | 深色/浅色，和 Cursor 选一样的比较统一 |
| AI 模型 | GPT-4o 或默认 | 默认模型通常已经很好用了 |

## 8.5 界面导览：和 Cursor 对比着看

Trae 的界面和 Cursor 很像（因为它们底层都是 VS Code），但有几个关键区别。

```
┌──────────────────────────────────────────────────┐
│ ① 菜单栏                    ② 登录头像/设置      │
├──────────┬───────────────────┬───────────────────┤
│          │                   │                   │
│ ③ 侧边栏 │  ④ 编辑区         │  ⑤ Trae AI 面板   │
│          │                   │  (Chat + Builder) │
│ 文件列表  │  写代码的地方      │                   │
│          │                   │  Chat：和AI对话    │
│          │                   │  Builder：搭项目   │
│          │                   │                   │
├──────────┴───────────────────┴───────────────────┤
│ ⑥ 底部状态栏                                      │
└──────────────────────────────────────────────────┘
```

**和 Cursor 的关键区别**：

| 区域 | Cursor | Trae |
|------|--------|------|
| AI 面板 | 只有一个 Chat 面板 | **两个标签页**：Chat（对话）和 **Builder（建项目）** |
| 行内编辑 | `Ctrl + K` 非常丝滑 | 有类似功能，但不如 Cursor 顺滑 |
| 新建项目 | 需要手动创建文件夹和文件 | Builder 可以一句话生成整个项目 |
| 中文支持 | 好 | **非常好**（中文母语级理解） |

> **Trae 最值得关注的地方就是 Builder 标签页**。这个在 Cursor 里没有。我们下一节就会用到它。

## 8.6 验证安装：你的第一个 Trae AI 操作

现在来做 Trae 上的第一个 AI 操作，顺便验证装成功了没有。

### 方式一：用 Builder 模式（推荐，体验 Trae 的独门绝技）

1. 在 Trae 里，点击右侧 AI 面板的 **"Builder"** 标签页
2. 在 Builder 输入框里输入：

```
帮我创建一个 Python 项目，只有一个文件 hello.py，
运行后打印 "Hello World from Trae!"
```

3. 点击发送（或按回车）
4. **Builder 会自动**：
   - 创建项目文件夹
   - 创建 `hello.py` 文件
   - 写入代码
   - 告诉你如何运行

5. 按 Builder 的提示，在 Trae 的终端里运行：

```bash
python hello.py
```

（Mac 用户用 `python3 hello.py`）

6. 你应该看到：

```
Hello World from Trae!
```

✅ **恭喜！Trae 安装成功，Builder 运行正常！**

### 方式二：用 Chat 模式（和 Cursor 类似的体验）

1. 点击右侧 AI 面板的 **"Chat"** 标签页
2. 在输入框里输入：

```
写一个 Python 程序，计算 1+2+3+...+100 的和，然后打印结果
```

3. 按回车
4. AI 会返回代码和解释
5. 新建一个文件 `sum.py`，把代码复制进去
6. 在终端运行 `python sum.py`，看到 `5050` 就是成功

## 8.7 ⚠️ 安装失败排查清单

### 🔴 问题1：下载 Trae 特别慢

**原因**：虽然 Trae 国内有服务器，但偶尔也会慢。

**解决方案**：
1. 耐心等待，Trae 的下载速度通常比 Cursor 快
2. 如果超过 10 分钟还没下完，试试换个时间段（比如晚上或早上）
3. 检查有没有其他大文件在下载（比如游戏更新、视频下载），暂停它们

### 🔴 问题2：启动后黑屏，什么都看不到

**原因**：可能是显卡驱动问题，或者首次启动加载慢。

**解决方案**：
1. **等 30 秒**——有时候就是加载慢，别急着关
2. 如果 30 秒后还是黑屏，关掉 Trae 重新打开
3. 如果还是黑屏，试试更新显卡驱动（Windows 用户去 NVIDIA/AMD/Intel 官网下载最新驱动）
4. 最后的办法：卸载重装

### 🔴 问题3：Builder 模式生成的代码运行报错

**原因**：Builder 生成的代码偶尔会有小问题。

**解决方案**：
1. 把错误信息复制到 Chat 面板，问 AI："这个报错怎么修？"
2. 或者**切回 Cursor**，用 Cursor 打开同一个文件，按 `Ctrl + K` 让 Cursor 修
3. 这就是为什么我们要装两个工具——互相兜底

### 🔴 问题4：AI 模型加载失败

**症状**：AI 面板显示"模型加载失败"或"服务不可用"。

**原因**：可能是网络问题，或者服务端临时故障。

**解决方案**：
1. 检查网络连接，确保能正常上网
2. 等几分钟再试（服务端临时故障通常几分钟就恢复）
3. 如果一直不行，试试切换模型（在 AI 面板的模型选择下拉菜单里换一个）
4. 如果 Trae 暂时用不了，**切回 Cursor**——这就是装两个工具的好处

### 🔴 问题5：国内用户能直接用吗？

**能。** Trae 是字节跳动（抖音、TikTok 的母公司）的产品，国内访问非常顺畅，不需要任何翻墙工具。这是 Trae 相比 Cursor 的最大优势之一。

如果你用 Cursor 时经常卡顿、连接不稳定，那 Trae 就是你的"备胎"——而且这个备胎可能比正室还好用。

## 自测

1. Cursor 和 Trae 的核心区别是什么？请用一句话概括。
2. 你在什么场景下会选择用 Trae 的 Builder 模式，而不是 Cursor 的 Ctrl+K？
3. 如果你的 Cursor 连接不稳定，但 Trae 正常，你应该怎么做？

> **答案**：
> 1. Cursor 精于逐行修改代码（Ctrl+K 行内编辑丝滑），Trae 精于从零搭建项目（Builder 一句话生成整个项目）。简言之：Cursor 做"修改"，Trae 做"创建"。
> 2. 当你需要从零开始创建一个新项目时（比如"帮我做一个个人主页网站"），用 Trae Builder。它会自动创建多个文件、搭建项目结构。而 Cursor 的 Ctrl+K 更适合在已有文件中修改某一段代码。
> 3. 暂时切换到 Trae 继续工作。Trae 国内访问顺畅，功能也足够强。等 Cursor 恢复后再切回来。这就是为什么我们要装两个工具——互相兜底，不怕一个出问题。
