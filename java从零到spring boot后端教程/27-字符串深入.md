## 27.1 String的不可变性：字符串一旦创建就永不变

String对象创建后，它的内容**永远不能改变**。这叫做**String不可变性**（String Immutability。此术语需进附录）。

```java
String s = "Hello";
s = s + " World";  // 这不是"修改了原来的字符串"！
```

这两行代码发生了什么？

```
步骤1: String s = "Hello";
       堆中创建对象: "Hello"，s指向它

步骤2: s = s + " World";
       创建新对象: "Hello World"，s指向新对象
       旧对象"Hello"还在堆里，但没人引用了（等着GC回收）
```

**所有看似"修改"String的操作，都在创建新的String对象**。这包括：`+` 拼接、`substring()`、`toUpperCase()`、`trim()` 等。

### 为什么设计成不可变？

| 原因 | 说明 |
|------|------|
| **安全性** | 字符串经常用作类名、文件名、URL——不可变意味着不会在运行时被意外篡改 |
| **字符串池** | 不可变使得多个变量可以安全地共享同一个字符串对象（见27.3节） |
| **线程安全** | 不可变对象天然是线程安全的，多线程并发访问不需要加锁 |
| **hashCode缓存** | String经常用作HashMap的key。不可变意味着hasHCode可以缓存，提升性能 |

## 27.2 == vs equals：初学者最大的坑

这是Java面试和实际开发中频率最高的陷阱。**这一节必须彻底理解。**

### == 比较的是"引用地址"

```java
// == 对引用类型比较的是内存地址（是不是同一个对象）
String s1 = new String("hello");
String s2 = new String("hello");
System.out.println(s1 == s2);   // false！两个不同的对象，不同的地址
```

### equals() 比较的是"内容"

```java
System.out.println(s1.equals(s2));   // true！内容相同
```

### 字符串池里的真相

```java
// 直接用字面量——从字符串池取
String a = "hello";
String b = "hello";
System.out.println(a == b);      // true！同一个池里的对象
System.out.println(a.equals(b)); // true

// 用new——强制创建新对象
String c = new String("hello");
String d = new String("hello");
System.out.println(c == d);      // false！
System.out.println(c.equals(d)); // true

// 混用
System.out.println(a == c);      // false！一个池对象，一个堆新对象
System.out.println(a.equals(c)); // true
```

> ❌ **常见错误**：百分之九十的新手都会在这里犯错。**规则**：**比较字符串内容时永远用 `equals()`，不要用 `==`。** `==` 只在你想确认"两个引用是不是指向同一个对象"时才用。

### 图解

```
字符串常量池（String Pool）:
┌──────────────────────────────────┐
│  "hello"  ← a 和 b 都指向这里   │
└──────────────────────────────────┘

堆（Heap）:
┌──────────┐
│ "hello"  │  ← c 指向这里（new创建的）
├──────────┤
│ "hello"  │  ← d 指向这里（另一个new创建的）
└──────────┘
```

## 27.3 字符串池（String Pool）：节省内存的魔法

JVM维护着一个**字符串常量池**（String Constant Pool），存放所有用双引号直接声明的字符串字面量。

```java
String a = "abc";   // 检查池里有没有"abc"→没有→创建并放入池
String b = "abc";   // 检查池里有没有"abc"→有→直接返回池里的引用
// 结果：a和b指向同一个对象。只有1个"abc"对象。

String c = new String("abc");  // 强制在堆上创建新对象，不理池
// 结果：3行代码创建了2个对象——1个在池中，1个在堆中。
```

### intern()方法：手动入池

```java
String s1 = new String("hello");
String s2 = s1.intern();  // intern()：如果池里有就返回池里的引用，没有就放入池
String s3 = "hello";

System.out.println(s1 == s2);  // false（s1是堆对象，s2是池对象）
System.out.println(s2 == s3);  // true（都是池里的）
```

## 27.4 字符串拼接效率对比

### 方案一：+ 运算符（少量拼接够用）

```java
String result = "Hello" + " " + "World" + "!";
// 编译器优化为：
// String result = "Hello World!";
// 字面量拼接在编译时就合并了，效率很高。
```

### 方案二：循环中使用 +（性能杀手！）

```java
// ❌ 低效：每次循环都创建新String，产生大量中间垃圾对象
String result = "";
for (int i = 0; i < 10000; i++) {
    result += i;  // 每次创建新String + 旧String被GC
}
// 10000次循环，创建了10000个临时String对象！
```

这是因为 `result += i` 等价于：
```java
result = new StringBuilder(result).append(i).toString();
// 每次循环创建了3个对象：StringBuilder + String(i) + toString返回的String
```

### 方案三：StringBuilder（循环拼接首选）

```java
// ✅ 高效：只创建一个StringBuilder，内部char数组动态扩容
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 10000; i++) {
    sb.append(i);
}
String result = sb.toString();
```

### 方案四：StringBuffer（线程安全版）

```java
// StringBuffer的方法加了synchronized，线程安全但稍慢
StringBuffer sbf = new StringBuffer();
sbf.append("hello");
```

### 性能对比表

| 方式 | 少量拼接（<5次） | 循环拼接（10000次） | 线程安全 |
|------|------------------|---------------------|----------|
| `+` | ✅ 推荐 | ❌ 极慢 | N/A |
| `StringBuilder` | 可用但不必要 | ✅ 推荐 | ❌ |
| `StringBuffer` | 不必要 | ✅ 可用 | ✅ |

## 27.5 常用String方法

```java
String s = "  Hello, Java World!  ";

// 基本信息
int len = s.length();                    // 字符串长度
char c = s.charAt(1);                   // 获取索引1的字符
boolean isEmpty = s.isEmpty();           // 是否为空字符串

// 查找
int index = s.indexOf("Java");           // "Java"首次出现的位置（-1表示没找到）
int lastIndex = s.lastIndexOf("o");      // "o"最后出现的位置
boolean contains = s.contains("World");  // 是否包含

// 截取
String sub1 = s.substring(7);            // 从索引7到末尾：Java World!
String sub2 = s.substring(7, 11);        // 从索引7到10（不含11）：Java

// 替换
String replaced = s.replace("Java", "Python");  // Hello, Python World!
String trimmed = s.trim();               // 去掉首尾空白：Hello, Java World!

// 大小写
String upper = s.toUpperCase();          //   HELLO, JAVA WORLD!
String lower = s.toLowerCase();          //   hello, java world!

// 拆分与连接
String[] parts = s.trim().split(", ");   // ["Hello", "Java World!"]
String joined = String.join("-", parts); // Hello-Java World!

// 比较
boolean eq = "hello".equals("Hello");            // false
boolean eqIgnore = "hello".equalsIgnoreCase("Hello");  // true
int cmp = "abc".compareTo("abd");                // -1（按字典序）

// 格式化
String formatted = String.format("我叫%s，今年%d岁", "张三", 25);
// "我叫张三，今年25岁"

// 判空安全写法
if (s != null && !s.isEmpty()) { ... }
if (s != null && !s.isBlank()) { ... }  // Java 11+（忽略纯空白字符串）
```

## 本章小结

| 知识点 | 核心要点 |
|--------|----------|
| String不可变性 | 字符串创建后内容不可改。所有"修改"操作都返回新对象。此术语需进附录 |
| == vs equals | `==`比较引用地址（是否是同一个对象）。`equals()`比较内容（值是否相同）。**内容比较永远用equals** |
| 字符串池 | JVM维护的常量池。字面量字符串会被放入池中共享。`new String()` 绕过池 |
| intern() | 手动将字符串放入池中或返回池中已有的引用 |
| StringBuilder | 非线程安全的高效字符串拼接器。循环拼接首选 |
| StringBuffer | 线程安全的StringBuilder。多线程环境使用 |
