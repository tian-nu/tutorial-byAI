## 为什么第一个项目是"个人主页"

想象你第一次学游泳。教练不会把你直接扔进深水区，而是让你先在浅水区踩踩水、感受一下浮力。

**个人主页就是AI编程的"浅水区"**。它只有一个HTML文件，不需要后端、不需要数据库、不需要装任何依赖——你写完就能在浏览器里看到效果。30分钟，从零到有，这是你建立信心的第一步。

> 📊 可视化演示见 [project1_visual.html](../.project/docs/tutorials/ai_coding_project1_visual.html)

## 48.1 你要做什么

做一个能在浏览器中打开的个人介绍网页，包含：

- 你的名字和头像（照片）
- 一段简短的个人介绍
- 你的技能列表
- 联系方式（邮箱、GitHub、微信等）

**技术方案**：纯HTML + CSS + JavaScript，一个文件搞定。不需要装任何东西，你的电脑上已经有浏览器了。

---

## 48.2 第一步：打开工具，新建文件

### **1.** 打开你的AI编程工具（Cursor或Trae）

### **2.** 新建一个文件夹，叫 `my-homepage`

在你的电脑上随便找个地方（比如桌面），新建 `my-homepage` 文件夹。

### **3.** 在工具中打开这个文件夹

Cursor：`File → Open Folder` → 选择 `my-homepage`
Trae：`File → Open Folder` → 选择 `my-homepage`

### **4.** 新建 `index.html` 文件

在工具左侧文件列表中右键 → `New File` → 命名为 `index.html`。

---

## 48.3 第二步：写Prompt，让AI生成代码

### **1.** 打开AI对话面板

Cursor：按 `Ctrl + I`（Composer模式）或 `Ctrl + L`（Chat模式）
Trae：按 `Ctrl + L`（Chat模式）或 `Ctrl + B`（Builder模式）

### **2.** 输入以下Prompt（你可以根据自己的情况修改）：

```
帮我做一个个人主页，要求：
1. 纯HTML + CSS + JavaScript，一个文件搞定
2. 包含：名字（张三）、照片（放在同目录下叫 photo.jpg）、简短介绍、技能列表、联系方式
3. 风格简洁现代，用蓝色调
4. 支持手机端也能看（响应式设计）
5. 页面加载时有一个淡入动画
```

### **3.** 按回车，等待AI生成代码

AI会在几秒到几十秒内生成完整的HTML代码。你会看到代码里有一堆 `<html>`、`<head>`、`<style>`、`<body>` 之类的标签——**不用管它们是什么意思，你只负责验收效果**。

---

## 48.4 第三步：在浏览器打开，看效果

### **1.** 保存文件（`Ctrl + S`）

### **2.** 在文件管理器中找到 `index.html`，双击打开

或者直接在浏览器地址栏输入文件路径，比如：
```
file:///C:/Users/你的用户名/Desktop/my-homepage/index.html
```

### **3.** 看看效果

- 名字显示了吗？
- 颜色好看吗？
- 手机上看（把浏览器窗口缩小）还正常吗？

---

## 48.5 第四步：不满意？改Prompt让AI再改

**这是AI编程最核心的循环**：预览 → 提意见 → AI改 → 再预览。

### 举个例子，你可能想改这些：

**说"颜色太单调了"**
```
我不喜欢蓝色，换成暖色调的，橙色为主。
```

**说"照片太大了"**
```
头像太大了，缩小到直径150像素，改成圆形。
```

**说"加点功能"**
```
在页面右下角加一个"回到顶部"的按钮，点一下平滑滚动到顶部。
```

**说"联系方式太普通了"**
```
联系方式的邮箱和GitHub前面加上图标，让它们看起来更明显。
```

每次说完，AI都会修改代码。你保存 → 刷新浏览器 → 看效果 → 不满意继续改。

**这个循环可能来回3-5轮，完全正常！** 这不是AI不好，而是你对自己想要什么越来越清楚了。

## 48.6 照片怎么放

### 方法一：把照片放到同目录下（推荐）

1. 把你的照片复制到 `my-homepage` 文件夹里
2. 改名为 `photo.jpg`
3. 在HTML中，AI会用这样的路径引用：`<img src="photo.jpg">`

### 方法二：用网络图片URL

如果你没有合适的照片，可以让AI用一个占位图：
```
照片先用一个占位图，用 https://via.placeholder.com/150 这个URL
```

### 注意：路径问题

- ✅ 正确：`photo.jpg`（照片和HTML在同一个文件夹里）
- ❌ 错误：`C:\Users\张三\Desktop\photo.jpg`（绝对路径，换电脑就失效）
- ❌ 错误：`/Users/张三/Desktop/photo.jpg`（Mac路径，Windows上打不开）

完成以下所有检查，你的项目才算通过：

| 序号 | 检查项 | 怎么做 |
|------|--------|--------|
| 1 | 双击 `index.html` 能在浏览器中打开 | 直接双击文件 |
| 2 | 页面上能看到你的名字 | 肉眼检查 |
| 3 | 页面上有你的照片（或占位图） | 肉眼检查，图片不裂 |
| 4 | 有一段个人简介文字 | 肉眼检查 |
| 5 | 有联系方式（至少一项） | 肉眼检查 |
| 6 | 手机浏览器缩小后还能看（不散架） | 把浏览器窗口拖窄 |

## 48.8 排错指南

### 🔴 问题1：中文显示乱码（一堆方框或问号）

**症状**：浏览器里所有中文变成 `□□□□` 或 `????`。

**原因**：HTML文件缺少字符编码声明，浏览器不知道用UTF-8解码中文。

**解决**：让AI在 `<head>` 标签里加上这一行。直接把这句话发给AI：
```
我的网页中文乱码了，帮我在head标签里加上 <meta charset="UTF-8">
```

### 🔴 问题2：图片不显示（页面出现一个裂掉的图片图标）

**症状**：浏览器里图片位置显示一个破碎的图标。

**原因**：图片路径不对。HTML里写的 `photo.jpg`，但实际文件不存在或名字不一样。

**解决**：
1. 检查照片文件是否真的在 `my-homepage` 文件夹里
2. 检查文件名是否完全一致（包括大小写）——`photo.jpg` 和 `Photo.jpg` 在Windows上一样，但到了服务器上就不一样了
3. 如果AI写的是网络图片URL，检查那个URL是否还能访问

### 🔴 问题3：CSS样式没生效（页面白底黑字，没有颜色）

**症状**：文字都显示了，但没有任何颜色、排版。

**原因**：AI生成的CSS可能写在了外部文件里，或者 `<style>` 标签位置不对。

**解决**：告诉AI：
```
样式没有生效，帮我把所有CSS直接写在HTML的<style>标签里，不要用外部CSS文件。
```

### 🔴 问题4：浏览器打开后什么也没有（空白页）

**症状**：双击 `index.html` 后浏览器一片空白。

**原因**：可能是AI生成的代码有语法错误，或者文件是空的。

**解决**：
1. 用文本编辑器（记事本）打开 `index.html`，看看里面有没有内容
2. 如果有内容，按 `F12` 打开浏览器开发者工具，看 Console 面板有没有红色报错
3. 把报错信息发给AI，让它修复

### 🔴 问题5：忘记怎么和AI继续对话了

**解决**：在同一个对话窗口里继续发消息就行。AI会记住之前的上下文，知道你在说什么。

## 48.9 小结

| 你学到了什么 | 一句话 |
|-------------|--------|
| AI编程的最小闭环 | 写Prompt → AI生成 → 预览 → 不满意再改 → 再预览 |
| 纯HTML项目的优势 | 不需要安装任何东西，浏览器就能跑 |
| 迭代是正常的 | 第一次生成的代码大概率需要调整，3-5轮迭代完全正常 |
| 照片路径 | 用相对路径（同目录），不要用绝对路径 |
| 排错思路 | 打开浏览器开发者工具（F12），看Console面板的报错 |

## 48.10 自测

1. 你完成了个人主页后，想把它发给朋友看。朋友直接双击 `index.html` 能看到吗？如果看不到，最可能的原因是什么？
2. 你觉得AI第一次生成的代码不好看，你应该怎么做？请写出具体步骤。
3. 如果你的照片显示不出来，可能是哪三个原因？

> **答案**：
> 1. 能看到，因为HTML文件可以直接在浏览器打开。但如果图片用的是绝对路径（如 `C:\Users\张三\photo.jpg`），朋友电脑上就看不到图片了。解决方法是把照片和HTML放在同一个文件夹，用相对路径 `photo.jpg`。
> 2. 在同一个AI对话窗口里继续发消息，告诉AI具体哪里不好看（比如"颜色太单调，换成暖色调"），AI会修改代码。你保存后刷新浏览器看效果，不满意继续改。
> 3. ①照片文件不在 `my-homepage` 文件夹里；②文件名不一致（大小写或拼写错误）；③HTML里写的是网络图片URL，但那个URL失效了。
