## 让代码"会思考"：条件判断

到目前为止，我们写的代码都是"一条路走到黑"——从上到下，每行都执行一遍。

但真实世界中，你需要做判断：

- **如果**外面下雨 → 带伞
- **否则** → 不带伞

这就是**条件判断（if/else）**📖。代码也能这样：

```python
weather = "下雨"

if weather == "下雨":       # 如果天气是"下雨"
    print("带伞出门")
else:                       # 否则（天气不是下雨）
    print("不用带伞")
```

**逐行讲解**：
- 第3行：`if` 是"如果"。`weather == "下雨"` 是"weather这个变量的值等于'下雨'吗？"。注意是 `==`（两个等号），不是 `=`。
- 第4行：如果条件成立（确实是"下雨"），执行这一行。注意前面缩进了4个空格——Python靠缩进来判断"谁属于谁"。
- 第5行：`else` 是"否则"。如果条件不成立（不是"下雨"），走这里。
- 第6行：否则就打印"不用带伞"。

---

## 三种条件判断：if / elif / else

有时候不止一种情况：

```python
score = 85

if score >= 90:              # 如果 ≥ 90
    grade = "优秀"
elif score >= 75:            # 否则如果 ≥ 75
    grade = "良好"
elif score >= 60:            # 否则如果 ≥ 60
    grade = "及格"
else:                        # 以上都不满足
    grade = "不及格"

print(grade)  # 输出：良好
```

> `elif` = `else + if` = "否则如果"。

**判断顺序很重要**：从上到下，哪个条件先满足就用哪个。如果把 `>= 60` 放最前面，那所有 ≥60 的都会变成"及格"，后面的"良好"和"优秀"永远不会被检查到。

## for循环：把列表里每个元素都处理一遍

假设你有一个名单，要给每个人发消息：

```python
students = ["张三", "李四", "王五"]

for name in students:         # 对于列表里的每一个名字
    print(f"{name}，交作业了！")
```

**输出**：
```
张三，交作业了！
李四，交作业了！
王五，交作业了！
```

**逐行讲解**：
- 第3行：`for` 是"对于"。`name` 是一个临时变量，每次从列表里取一个值。`in students` 是"在列表里"。
- 第4行：对每个取出来的 `name`，打印一句话。注意缩进——缩进的行都在循环里。

**循环（Loop）**📖 = 重复做一件事，直到把所有东西都处理完。

📖 **for循环** = "把列表里每个元素拿出来，都做一遍同样的事"。

## while循环：条件满足就一直做

`for` 循环是"把列表里每个都过一遍"，你知道一共要跑几遍。

`while` 循环是"只要条件满足，就一直跑"，你事先不知道要跑几遍。

📖 **while循环** = "只要这个条件还成立，就继续做"。

> 以下代码由AI生成：

```python
# 猜数字游戏 —— while循环的经典用法
secret = 7               # 秘密数字
guess = 0                # 用户的猜测，初始为0

while guess != secret:           # 只要猜的不等于秘密数字
    guess = int(input("猜一个1到10之间的数字："))  # 让用户输入
    if guess < secret:
        print("太小了，再试试！")
    elif guess > secret:
        print("太大了，再试试！")

print("恭喜你，猜对了！")      # 跳出了while循环，说明猜对了
```

**逐行讲解**：
- 第2行：设定秘密数字是7。
- 第3行：用户的初始猜测是0（肯定不等于7，所以循环一定会开始）。
- 第5行：`while guess != secret:` 是"只要猜测不等于秘密数字，就一直循环"。
- 第6行：`input()` 让用户在键盘上输入，`int()` 把输入转成整数。
- 第7-10行：根据大小给出提示。
- 第12行：当用户终于猜对了，`guess == secret`，`while` 条件变成 `False`，循环结束，打印恭喜。

> ⚠️ **无限循环警告**：如果 `while` 的条件永远为真，代码就会永远跑下去。比如 `while True: print("停不下来了")` —— 这会一直打印直到你强制关闭。AI写代码时一般不会犯这个错，但你得知道这个坑。

## AI生成代码示例：打印1到100所有偶数

> 以下代码由AI生成：

```python
# 方法一：用for循环 + 步长
for num in range(2, 101, 2):    # range(开始, 结束, 步长)
    print(num)                   # 从2开始，每次+2，到100结束

# 方法二：用for循环 + 条件判断
for num in range(1, 101):       # range(1, 101) 产生1到100
    if num % 2 == 0:            # % 是取余数，能被2整除就是偶数
        print(num)
```

**逐行讲解（方法一）**：
- 第2行：`range(2, 101, 2)` 产生一串数字——从2开始，到100结束，每次加2。所以是：2, 4, 6, 8...100。
- 第3行：打印每个数。

**逐行讲解（方法二）**：
- 第6行：`range(1, 101)` 产生1到100全部数字。
- 第7行：`num % 2` 是"num除以2的余数"。如果余数为0，说明能被2整除，就是偶数。
- 第8行：是偶数才打印。

## for vs while 速查

| 用哪个 | 什么时候用 | 比喻 |
|--------|-----------|------|
| `for` | 你知道要处理多少个东西（如列表有10个人） | 按名单点名，点完为止 |
| `while` | 你不知道要跑多少次（如猜数字，猜对才停） | 摇骰子，摇到6才停 |

3. 一个"输错密码最多3次"的功能，应该用 `for` 还是 `while`？为什么？

> **答案**：
> 1. `=` 是赋值（把东西放进盒子），`==` 是判断相等（两边一样吗？）。
> 2. 打印出 2 和 4（1到5中能被2整除的偶数）。
> 3. 用 `while` + 计数器，因为不知道用户第几次会输对（可能第一次就对，也可能第三次才对），while适合这种"条件满足才继续"的场景。当然for也可以（`for attempt in range(3)`），两种都行。
