# 第43章 注解与反射

> 你有没有见过魔术师在不知道卡片内容的情况下，依然能准确"猜"出你选的牌？注解和反射就是 Java 的魔术——**注解**贴在代码上做标记（像贴标签），**反射**在运行时读取这些标记并做出相应行为。Spring Boot 之所以能用 `@Autowired` 自动注入、`@GetMapping` 自动映射 URL，底层全靠这对搭档。

---

## 43.1 注解（Annotation）

> 🏷️ 此术语需进附录：**注解**（Annotation）——Java 5 引入的元数据标记机制。注解本身不改变代码逻辑，但可以被编译器、工具或运行时反射读取，从而实现代码生成、依赖注入、配置简化等功能。

### 43.1.1 内置注解

Java 自带几个常用注解：

```java
// @Override：告诉编译器"我在重写父类方法"
@Override
public String toString() {
    return "MyClass";
}
// 如果方法签名写错了（比如写成 tostring），编译器会报错

// @Deprecated：标记该方法已过时
@Deprecated
public void oldMethod() {
    // 调用时编译器会给出警告
}

// @SuppressWarnings：抑制编译器警告
@SuppressWarnings("unchecked")
public void convertList() {
    List raw = new ArrayList();
    List<String> list = raw;  // 不会产生 unchecked 警告
}

// @FunctionalInterface：检查接口是否是函数式接口（第35章学过）
@FunctionalInterface
interface MyFunc {
    void execute();
    // void execute2(); // 加第二个抽象方法会编译报错
}
```

### 43.1.2 元注解

元注解是"注解的注解"——用来定义其他注解的行为。

```java
import java.lang.annotation.*;

// @Target：注解可以用在哪里
// @Retention：注解保留到什么时候
// @Documented：是否被 javadoc 记录
// @Inherited：子类是否继承父类的注解
```

| 元注解 | 说明 | 常用值 |
|--------|------|--------|
| `@Target` | 注解的适用范围 | `TYPE`(类)、`FIELD`(字段)、`METHOD`(方法)、`PARAMETER`(参数) 等 |
| `@Retention` | 注解的保留阶段 | `SOURCE`(源码)、`CLASS`(字节码)、`RUNTIME`(运行时) |
| `@Documented` | 是否出现在 javadoc 中 | — |
| `@Inherited` | 子类是否继承 | — |

### 43.1.3 自定义注解

```java
import java.lang.annotation.*;

// 定义一个运行时注解
@Target({ElementType.TYPE, ElementType.METHOD})  // 可用于类和方法
@Retention(RetentionPolicy.RUNTIME)              // 保留到运行时（反射能读到）
@Documented
public @interface MyAnnotation {
    // 注解的元素（看起来像方法，实际上是属性）
    String value() default "";       // default 设定默认值
    int priority() default 0;
    String[] tags() default {};      // 数组类型
}

// 使用自定义注解
@MyAnnotation(value = "用户服务", priority = 1, tags = {"核心", "必须"})
public class UserService {

    @MyAnnotation(value = "获取用户", priority = 2)
    public User getUser(Long id) {
        return new User();
    }
}
```

> 🤔 **想多一点**：`@Retention` 的三种保留策略有什么实际区别？
> - `SOURCE`：编译后消失，如 `@Override`、`@SuppressWarnings`（编译器用完就扔）
> - `CLASS`：保留在字节码但运行时不可见（默认值，用得少）
> - `RUNTIME`：运行时可通过反射读取，Spring 的 `@Autowired`、`@Transactional` 都是这个级别

## 43.2 反射（Reflection）

> 🏷️ 此术语需进附录：**反射**（Reflection）——Java 在运行时动态获取类的信息（构造器、字段、方法）并操作它们的能力。反射是框架（Spring、MyBatis 等）的基石，也是注解发挥作用的前提。

### 43.2.1 获取 Class 对象

```java
// 方式1：类名.class
Class<String> cls1 = String.class;

// 方式2：对象.getClass()
String s = "hello";
Class<? extends String> cls2 = s.getClass();

// 方式3：Class.forName()（全限定类名）
Class<?> cls3 = Class.forName("java.lang.String");

// 方式4：基本类型的 Class
Class<Integer> intClass = int.class;          // int
Class<Integer> wrapperClass = Integer.class;   // Integer
System.out.println(intClass == wrapperClass);  // false —— int 和 Integer 是不同的 Class
```

### 43.2.2 操作构造器

```java
import java.lang.reflect.Constructor;

public class Person {
    private String name;
    private int age;

    public Person() {}
    public Person(String name) { this.name = name; }
    public Person(String name, int age) { this.name = name; this.age = age; }
}

// 反射操作
Class<Person> cls = Person.class;

// 获取无参构造器
Constructor<Person> noArgCtor = cls.getConstructor();
Person p1 = noArgCtor.newInstance();

// 获取有参构造器
Constructor<Person> ctor = cls.getConstructor(String.class, int.class);
Person p2 = ctor.newInstance("Alice", 25);

// 获取所有 public 构造器
Constructor<?>[] ctors = cls.getConstructors();

// 获取所有构造器（包括 private）
Constructor<?>[] allCtors = cls.getDeclaredConstructors();
```

### 43.2.3 操作字段

```java
import java.lang.reflect.Field;

public class User {
    private String name = "默认";
    public int age = 0;
}

User user = new User();
Class<? extends User> cls = user.getClass();

// 获取 public 字段
Field ageField = cls.getField("age");
ageField.set(user, 25);
System.out.println(ageField.get(user)); // 25

// 获取 private 字段（需要 setAccessible）
Field nameField = cls.getDeclaredField("name");
nameField.setAccessible(true);  // 暴力访问！
nameField.set(user, "Bob");
System.out.println(nameField.get(user)); // Bob

// 获取所有字段
Field[] fields = cls.getDeclaredFields();
for (Field f : fields) {
    System.out.println(f.getName() + " : " + f.getType());
}
```

> ⚠️ `setAccessible(true)` 可以绕过访问控制，但 Java 9+ 模块化系统中可能受限。这也是为什么很多安全敏感框架会禁用反射。

### 43.2.4 操作方法

```java
import java.lang.reflect.Method;

public class Calculator {
    public int add(int a, int b) { return a + b; }
    private String secret() { return "secret!"; }
}

Calculator calc = new Calculator();
Class<? extends Calculator> cls = calc.getClass();

// 调用 public 方法
Method addMethod = cls.getMethod("add", int.class, int.class);
Object result = addMethod.invoke(calc, 3, 5);
System.out.println(result); // 8

// 调用 private 方法
Method secretMethod = cls.getDeclaredMethod("secret");
secretMethod.setAccessible(true);
String secret = (String) secretMethod.invoke(calc);
System.out.println(secret); // secret!

// 获取所有方法（含继承的）
Method[] methods = cls.getMethods();
for (Method m : methods) {
    System.out.println(m.getName());
}
```

## 43.3 注解 + 反射：实战组合

单独看注解和反射，好像没什么用。但**合在一起**就是框架的基石。下面的例子模拟了 Spring 的 `@Autowired` 逻辑：

```java
import java.lang.annotation.*;
import java.lang.reflect.*;
import java.util.*;

// 1. 定义注解
@Target(ElementType.FIELD)
@Retention(RetentionPolicy.RUNTIME)
@interface MyAutowired {
    String value() default "";
}

// 2. 定义"被注入"的类
class UserRepository {
    public List<String> findAll() {
        return List.of("Alice", "Bob", "Charlie");
    }
}

class OrderRepository {
    public List<String> findAll() {
        return List.of("Order-1", "Order-2");
    }
}

// 3. 使用注解的服务类
class UserService {
    @MyAutowired
    private UserRepository userRepo;

    @MyAutowired
    private OrderRepository orderRepo;

    public void printData() {
        System.out.println("用户: " + userRepo.findAll());
        System.out.println("订单: " + orderRepo.findAll());
    }
}

// 4. 简易 IOC 容器：扫描注解并注入依赖
class SimpleIocContainer {
    private Map<Class<?>, Object> beans = new HashMap<>();

    public void register(Class<?>... classes) throws Exception {
        for (Class<?> cls : classes) {
            beans.put(cls, cls.getDeclaredConstructor().newInstance());
        }
    }

    public <T> T getBean(Class<T> cls) throws Exception {
        T instance = cls.getDeclaredConstructor().newInstance();

        // 反射扫描所有字段
        for (Field field : cls.getDeclaredFields()) {
            if (field.isAnnotationPresent(MyAutowired.class)) {
                Class<?> fieldType = field.getType();
                Object dependency = beans.get(fieldType);
                if (dependency != null) {
                    field.setAccessible(true);
                    field.set(instance, dependency);  // 注入！
                }
            }
        }
        return instance;
    }
}

// 5. 运行
public class Main {
    public static void main(String[] args) throws Exception {
        SimpleIocContainer container = new SimpleIocContainer();
        // 注册"Bean"
        container.register(UserRepository.class, OrderRepository.class);
        // 获取并自动注入
        UserService service = container.getBean(UserService.class);
        service.printData();
    }
}
```

输出：
```
用户: [Alice, Bob, Charlie]
订单: [Order-1, Order-2]
```

这个 50 行的代码已经展示了 Spring 的核心思想：
1. `@MyAutowired` 注解标记需要注入的字段
2. 反射扫描字段上的注解
3. 从容器中查找依赖并 `field.set()` 注入

真实的 Spring 当然复杂得多（支持构造器注入、生命周期回调、AOP 等），但核心原理就是这个。

## 43.4 动态代理

> 🏷️ 此术语需进附录：**动态代理**（Dynamic Proxy）——在运行时动态创建代理对象，拦截方法调用，在调用前后插入额外逻辑。AOP（面向切面编程）的基础，常用于日志、事务、权限控制。

### 43.4.1 JDK 动态代理

JDK 动态代理要求被代理对象**必须实现接口**：

```java
import java.lang.reflect.*;

// 接口
interface UserService {
    void save(String name);
    String find(Long id);
}

// 实现类
class UserServiceImpl implements UserService {
    @Override
    public void save(String name) {
        System.out.println("保存用户: " + name);
    }

    @Override
    public String find(Long id) {
        System.out.println("查询用户: " + id);
        return "User-" + id;
    }
}

// 代理处理器：在这里插入额外逻辑
class LoggingHandler implements InvocationHandler {
    private final Object target;

    LoggingHandler(Object target) {
        this.target = target;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println(">>> [日志] 调用方法: " + method.getName());
        long start = System.currentTimeMillis();

        Object result = method.invoke(target, args);  // 调用真实方法

        long elapsed = System.currentTimeMillis() - start;
        System.out.println("<<< [日志] 方法结束: " + method.getName() + " 耗时 " + elapsed + "ms");
        return result;
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        UserService realService = new UserServiceImpl();

        UserService proxy = (UserService) Proxy.newProxyInstance(
            UserService.class.getClassLoader(),
            new Class<?>[]{UserService.class},
            new LoggingHandler(realService)
        );

        proxy.save("Alice");
        // 输出：
        // >>> [日志] 调用方法: save
        // 保存用户: Alice
        // <<< [日志] 方法结束: save 耗时 0ms

        String user = proxy.find(100L);
        // 输出：
        // >>> [日志] 调用方法: find
        // 查询用户: 100
        // <<< [日志] 方法结束: find 耗时 0ms
    }
}
```

### 43.4.2 动态代理的原理

```
调用 proxy.save("Alice")
       ↓
InvocationHandler.invoke(proxy, save方法, ["Alice"])
       ↓
method.invoke(realService, ["Alice"])   ← 实际执行
       ↓
返回结果（如果有）
```

`Proxy.newProxyInstance()` 在内存中**动态生成了一个 .class 文件**，这个类实现了 `UserService` 接口，每个方法都转发给 `InvocationHandler.invoke()`。可以用以下方式查看生成的类：

```java
// 导出动态代理生成的类
System.setProperty("jdk.proxy.ProxyGenerator.saveGeneratedFiles", "true");
// 或 Java 8：System.setProperty("sun.misc.ProxyGenerator.saveGeneratedFiles", "true");
```

### 43.4.3 CGLib 代理（了解）

JDK 动态代理的局限：只能代理接口。如果类没实现接口，就需要 CGLib（Code Generation Library）：

```
JDK 动态代理： 需要接口 → Proxy.newProxyInstance()
CGLib 代理：   不需要接口 → 通过继承生成子类
```

Spring 中默认 JDK 动态代理，但如果目标类没实现接口则自动切换为 CGLib。`spring-boot-starter-aop` 默认使用 CGLib。

## 43.5 ❌ 常见错误

### 错误1：反射性能滥用

```java
// ❌ 在循环中反射调用
for (int i = 0; i < 1000000; i++) {
    Method m = cls.getMethod("add", int.class, int.class); // 每次都要查方法表
    m.invoke(obj, a, b);
}

// ✅ 方法对象复用
Method m = cls.getMethod("add", int.class, int.class);
for (int i = 0; i < 1000000; i++) {
    m.invoke(obj, a, b);
}
```

反射比直接调用慢（约几倍到几十倍），框架中会在启动时用反射，运行时尽量避免。

### 错误2：忘记 setAccessible

```java
// ❌ 直接访问 private 字段
Field field = cls.getDeclaredField("name");
// field.get(obj); // IllegalAccessException！

// ✅
field.setAccessible(true);
field.get(obj); // OK
```

### 错误3：注解 Retention 设为 SOURCE 却在运行时反射读取

```java
// ❌ 运行时根本读不到！
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.SOURCE)  // 编译后就消失了
@interface MyConfig {
    String value();
}

// 运行时：
cls.isAnnotationPresent(MyConfig.class); // 永远是 false
```

### 错误4：JDK 代理的类没有实现接口

```java
// ❌ UserServiceImpl 没实现接口就无法用 JDK 动态代理
class UserServiceImpl { }
// Proxy.newProxyInstance(...) // IllegalArgumentException: not an interface
```

## 43.6 小结

| 知识点 | 一句话总结 |
|--------|-----------|
| 内置注解 | `@Override`、`@Deprecated`、`@SuppressWarnings` |
| 元注解 | `@Target`(在哪用)、`@Retention`(保留到何时) |
| 自定义注解 | `@interface` 定义，设置 Target 和 Retention |
| 反射获取 Class | `类名.class` / `obj.getClass()` / `Class.forName()` |
| 反射操作 | 获取构造器/字段/方法，`invoke()` 调用 |
| JDK 动态代理 | `Proxy.newProxyInstance()` + `InvocationHandler` |
| 注解+反射 | 框架基石：标记 → 扫描 → 自动处理 |
| 性能注意 | 反射慢，方法对象要复用，启动时用运行时不频繁用 |

## 43.7 自测题

**1.** 定义一个名为 `@TableName` 的运行时注解，只能用在类上，有一个 `String value()` 属性默认为空字符串。写出完整代码。

**2.** 以下代码输出什么？为什么？

```java
@Retention(RetentionPolicy.SOURCE)
@interface DebugOnly {}

@DebugOnly
class MyClass {}

public static void main(String[] args) {
    System.out.println(MyClass.class.isAnnotationPresent(DebugOnly.class));
}
```

**3.** 简述 JDK 动态代理的执行过程：当你调用代理对象的 `save()` 方法时，实际经过了哪些步骤？

> 答案提示：1. `@Target(ElementType.TYPE) @Retention(RetentionPolicy.RUNTIME) @interface TableName { String value() default ""; }`；2. 输出 `false`，因为 `@Retention(RetentionPolicy.SOURCE)` 使注解在编译后消失，运行时反射读不到；3. proxy.save() → InvocationHandler.invoke() → method.invoke(target, args) → 返回结果，即三步转发。
