# 第114章 · 项目五：IM即时通讯

> "前面四个项目都是'一问一答'式的HTTP交互——客户端发请求，服务器回响应，然后断开。但即时通讯需要'服务器主动推送'——张三给你发了一条消息，你的手机不需要刷新就能收到。这就是WebSocket的世界：客户端和服务器建立一条**长连接**（建立后保持不断开的持久网络连接，区别于HTTP短连接，详见附录I），任何一方随时可以发消息。本章带你从零搭建一个支持单聊、群聊、在线状态的IM（即时通讯，支持实时消息收发的通信系统，详见附录I）系统。"

---

## 114.1 HTTP vs WebSocket：从书信到电话

### HTTP的局限

HTTP是"请求-响应"模型：
- 客户端问"有新消息吗？"
- 服务器答"没有"
- 客户端过几秒再问"有新消息吗？"
- 服务器答"有了，张三说你好"

这就像你每分钟给朋友打一次电话："你有事找我吗？"——99%的时间对方说"没有"，浪费了时间（带宽）和精力（CPU）。

### WebSocket的优势

WebSocket建立一条**双向、全双工**的通道：
- 客户端和服务器握手一次，建立持久连接
- 任何一方随时可以向对方发消息
- 不需要反复建立和断开连接

就像打电话——接通后可以直接说，不用每次说话前先拨号。

### 对比表

| 特性 | HTTP | WebSocket |
|------|------|-----------|
| 通信模式 | 请求-响应 | 双向全双工 |
| 连接 | 短连接（每次请求后断开） | 长连接（一直保持） |
| 谁可以主动发 | 只有客户端 | 双方都可以 |
| 协议开销 | 每次请求都有完整HTTP头 | 握手后帧头仅2-14字节 |
| 适用场景 | RESTful API、网页浏览 | 聊天、实时通知、游戏 |

## 114.2 需求分析

### 功能列表

| 序号 | 模块 | 功能 | 说明 |
|------|------|------|------|
| 1 | 连接 | WebSocket连接建立 | 用户登录后建立长连接 |
| 2 | 单聊 | 一对一私聊 | 发送给指定用户 |
| 3 | 群聊 | 群组聊天 | 发送到群组内所有在线成员 |
| 4 | 在线状态 | 上线/下线通知 | 好友列表显示在线状态 |

### 不做什么
- 不做好友管理（只做用户间直接通讯）
- 不做消息已读状态
- 不做文件/图片传输（只做文本消息）
- 不做端到端加密

```
  │ 客户端A   │    │ 客户端B   │    │ 客户端C   │
  └─────┬─────┘    └─────┬─────┘    └─────┬─────┘

        │                │                │
        v                v                v
  ┌─────────────────────────────────────────────────┐
  │                  Connection Manager              │
  │              (sync.Map: userID → Conn)           │
  └───────────────────────┬─────────────────────────┘
                          │
              ┌───────────┼───────────┐
              v           v           v
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │  单聊路由  │ │  群聊广播  │ │  心跳管理  │
        └─────┬─────┘ └─────┬─────┘ └─────┬─────┘
              │             │             │

        │              MySQL                    │

        └──────────────────────────────────────┘
```

核心组件 `Connection Manager` 是一个全局的连接池，用 `sync.Map` 存储所有在线用户的WebSocket连接。每个用户ID对应一个连接对象——就像电话总机，每个分机号码对应一条电话线。

### 表说明

- `messages.msg_type`：1=单聊，2=群聊，3=系统通知
- `messages.to_user_id`：单聊时为目标用户ID，群聊时为0
- `messages.group_id`：群聊时为目标群ID，单聊时为0
- `contacts`：最近联系人表，记录每个用户的最近会话。`last_msg`存储最后一条消息摘要，避免每次列表都要查消息表
- `groups_table`：表名不叫`groups`是因为MySQL中`groups`是保留字

## 114.5 消息协议设计

WebSocket传输的是文本或二进制数据。我们使用JSON格式定义消息协议：

```json
{
    "type": "chat",
    "from": 1,
    "to": 2,
    "content": "你好！",
    "time": "2025-01-01T10:00:00Z"
}
```

### 消息类型定义

| type值 | 含义 | 方向 |
|--------|------|------|
| `chat` | 单聊消息 | 客户端↔服务器 |
| `group_chat` | 群聊消息 | 客户端↔服务器 |
| `system` | 系统通知 | 服务器→客户端 |
| `online` | 上线通知 | 服务器→客户端 |
| `offline` | 下线通知 | 服务器→客户端 |
| `heartbeat` | 心跳包 | 客户端→服务器 |
| `heartbeat_ack` | 心跳应答 | 服务器→客户端 |
| `offline_msgs` | 离线消息推送 | 服务器→客户端 |
| `error` | 错误消息 | 服务器→客户端 |

### ws/hub.go — 连接管理器（核心）

```go
package ws

import (
    "encoding/json"
    "log"
    "sync"

    "github.com/gorilla/websocket"
)

type Client struct {
    UserID uint64
    Conn   *websocket.Conn
    Send   chan []byte
}

type DBInterface interface {
    SaveMessage(msg *WSMessage) error
    GetOfflineMessages(userID uint64) ([]WSMessage, error)
    GetGroupMembers(groupID uint64) ([]uint64, error)
    UpdateContact(userID, contactID uint64, lastMsg string) error
    SetUserOnline(userID uint64, online bool) error
}

type Hub struct {
    clients    sync.Map
    register   chan *Client
    unregister chan *Client
    broadcast  chan []byte
    DB         DBInterface
}

func NewHub(db DBInterface) *Hub {
    return &Hub{
        register:   make(chan *Client, 256),
        unregister: make(chan *Client, 256),
        broadcast:  make(chan []byte, 512),
        DB:         db,
    }
}

func (h *Hub) Run() {
    for {
        select {
        case client := <-h.register:
            h.clients.Store(client.UserID, client)
            log.Printf("用户 %d 上线", client.UserID)
            h.DB.SetUserOnline(client.UserID, true)
            h.pushOfflineMessages(client)

        case client := <-h.unregister:
            if _, ok := h.clients.LoadAndDelete(client.UserID); ok {
                close(client.Send)
                log.Printf("用户 %d 下线", client.UserID)
                h.DB.SetUserOnline(client.UserID, false)
            }

        case message := <-h.broadcast:
            var msg WSMessage
            if err := json.Unmarshal(message, &msg); err != nil {
                continue
            }
            h.routeMessage(&msg)
        }
    }
}

func (h *Hub) routeMessage(msg *WSMessage) {
    switch msg.Type {
    case "chat":
        h.handleChat(msg)
    case "group_chat":
        h.handleGroupChat(msg)
    }
}

func (h *Hub) handleChat(msg *WSMessage) {
    if err := h.DB.SaveMessage(msg); err != nil {
        log.Printf("保存单聊消息失败: %v", err)
    }

    h.DB.UpdateContact(msg.From, msg.To, msg.Content)
    h.DB.UpdateContact(msg.To, msg.From, msg.Content)

    if client, ok := h.clients.Load(msg.To); ok {
        data, _ := json.Marshal(msg)
        select {
        case client.(*Client).Send <- data:
        default:
            // 通道满时消息被丢弃，接收端不会收到本条消息
            log.Printf("用户 %d 发送通道已满，消息丢弃", msg.To)
        }
    }

    if client, ok := h.clients.Load(msg.From); ok {
        data, _ := json.Marshal(msg)
        client.(*Client).Send <- data
    }
}

func (h *Hub) handleGroupChat(msg *WSMessage) {
    if err := h.DB.SaveMessage(msg); err != nil {
        log.Printf("保存群聊消息失败: %v", err)
    }

    memberIDs, err := h.DB.GetGroupMembers(msg.GroupID)
    if err != nil {
        log.Printf("获取群成员失败: %v", err)
        return
    }

    data, _ := json.Marshal(msg)

    for _, userID := range memberIDs {
        if userID == msg.From {
            continue
        }
        if client, ok := h.clients.Load(userID); ok {
            select {
            case client.(*Client).Send <- data:
            default:
            }
        }
    }
}

func (h *Hub) pushOfflineMessages(client *Client) {
    msgs, err := h.DB.GetOfflineMessages(client.UserID)
    if err != nil || len(msgs) == 0 {
        return
    }

    wrapped := map[string]interface{}{
        "type":     "offline_msgs",
        "messages": msgs,
    }
    data, _ := json.Marshal(wrapped)
    client.Send <- data
}

func (h *Hub) SendToUser(userID uint64, msg []byte) {
    if client, ok := h.clients.Load(userID); ok {
        select {
        case client.(*Client).Send <- msg:
        default:
        }
    }
}

func (h *Hub) IsOnline(userID uint64) bool {
    _, ok := h.clients.Load(userID)
    return ok
}
```

`sync.Map` 是Go 1.9引入的并发安全Map。为什么不用普通的`map + sync.RWMutex`？因为IM系统中连接注册/注销非常频繁，`sync.Map`在读写并发场景下性能更好——它针对"key只写一次但读很多次"的场景做了优化。

消息路由的核心逻辑：
1. 单聊：发给目标用户（在线则实时推送），自己也收到一份（多端同步）
2. 群聊：获取群成员列表，排除发送者，逐个推送

### ws/client.go — WebSocket客户端

```go
package ws

import (
    "encoding/json"
    "log"
    "time"

    "github.com/gorilla/websocket"
)

const (
    writeWait      = 10 * time.Second
    pongWait       = 60 * time.Second
    pingPeriod     = 30 * time.Second
    maxMessageSize = 4096
)

var upgrader = websocket.Upgrader{
    ReadBufferSize:  1024,
    WriteBufferSize: 1024,
}

func Upgrade(w http.ResponseWriter, r *http.Request) (*websocket.Conn, error) {
    return upgrader.Upgrade(w, r)
}

func (c *Client) ReadPump(hub *Hub) {
    defer func() {
        hub.unregister <- c
        c.Conn.Close()
    }()

    c.Conn.SetReadLimit(maxMessageSize)
    c.Conn.SetReadDeadline(time.Now().Add(pongWait))
    c.Conn.SetPongHandler(func(string) error {
        c.Conn.SetReadDeadline(time.Now().Add(pongWait))
        return nil
    })

    for {
        _, message, err := c.Conn.ReadMessage()
        if err != nil {
            if websocket.IsUnexpectedCloseError(err,
                websocket.CloseGoingAway, websocket.CloseNormalClosure) {
                log.Printf("WebSocket错误: %v", err)
            }
            break
        }

        var msg WSMessage
        if err := json.Unmarshal(message, &msg); err != nil {
            continue
        }

        msg.From = c.UserID
        msg.Time = time.Now().Format(time.RFC3339)

        if msg.Type == "heartbeat" {
            ack := WSMessage{Type: "heartbeat_ack", Time: msg.Time}
            ackData, _ := json.Marshal(ack)
            c.Send <- ackData
            continue
        }

        data, _ := json.Marshal(msg)
        hub.broadcast <- data
    }
}

func (c *Client) WritePump() {
    ticker := time.NewTicker(pingPeriod)
    defer func() {
        ticker.Stop()
        c.Conn.Close()
    }()

    for {
        select {
        case message, ok := <-c.Send:
            c.Conn.SetWriteDeadline(time.Now().Add(writeWait))
            if !ok {
                c.Conn.WriteMessage(websocket.CloseMessage, []byte{})
                return
            }

            if err := c.Conn.WriteMessage(websocket.TextMessage, message); err != nil {
                return
            }

        case <-ticker.C:
            c.Conn.SetWriteDeadline(time.Now().Add(writeWait))
            if err := c.Conn.WriteMessage(websocket.PingMessage, nil); err != nil {
                return
            }
        }
    }
}
```

每个WebSocket连接启动两个goroutine——一个读一个写。就像电话线路需要同时支持"听"和"说"，两个goroutine各司其职：

- `ReadPump`：读goroutine，监听客户端发来的消息，收到后广播到Hub
- `WritePump`：写goroutine，从Send通道取出消息发送给客户端，同时发送心跳Ping

### 心跳机制详解

为什么需要心跳？因为WebSocket连接可能会"假死"——网络断了但TCP还没超时，服务器以为用户还在线，消息却发不过去。

心跳机制的关键参数：
- `pingPeriod = 30s`：服务器每30秒发送一个Ping帧
- `pongWait = 60s`：如果60秒内没收到Pong回应，认为连接断开
- `SetPongHandler`：客户端自动回复Pong时，刷新读超时计时器

就像两个人打电话时每隔30秒说一声"喂？还在吗？"——如果60秒内对方没回应"在！"，就挂断。

### model/message_repo.go — 消息持久化

```go
func SaveMessage(db *gorm.DB, msg *WSMessage) error {
    msgType := int8(1)
    var toUserID uint64
    var groupID uint64

    if msg.Type == "group_chat" {
        msgType = 2
        groupID = msg.GroupID
    } else {
        toUserID = msg.To
    }

    message := &Message{
        FromUserID: msg.From,
        ToUserID:   toUserID,
        GroupID:    groupID,
        MsgType:    msgType,
        Content:    msg.Content,
    }
    return db.Create(message).Error
}

func GetOfflineMessages(db *gorm.DB, userID uint64) ([]WSMessage, error) {
    var messages []Message

    err := db.Where(
        "(to_user_id = ? AND is_read = 0 AND msg_type = 1) OR "+
            "(group_id IN (SELECT group_id FROM group_members WHERE user_id = ?) AND msg_type = 2)",
        userID, userID,
    ).Order("created_at ASC").Limit(100).Find(&messages).Error
    if err != nil {
        return nil, err
    }

    db.Model(&Message{}).
        Where("to_user_id = ? AND is_read = 0 AND msg_type = 1", userID).
        Update("is_read", 1)

    wsMsgs := make([]WSMessage, len(messages))
    for i, m := range messages {
        wsMsgs[i] = WSMessage{
            Type:    "chat",
            From:    m.FromUserID,
            Content: m.Content,
            Time:    m.CreatedAt.Format(time.RFC3339),
        }
    }
    return wsMsgs, nil
}

func UpdateContact(db *gorm.DB, userID, contactID uint64, lastMsg string) error {
    contact := Contact{
        UserID:    userID,
        ContactID: contactID,
        LastMsg:   lastMsg,
        LastTime:  time.Now(),
    }
    return db.Where("user_id = ? AND contact_id = ?", userID, contactID).
        Assign(contact).FirstOrCreate(&contact).Error
}
```

```go
    Hub *ws.Hub
}

func NewWSHandler(hub *ws.Hub) *WSHandler {
    return &WSHandler{Hub: hub}
}

func (h *WSHandler) Connect(c *gin.Context) {
    userIDStr := c.Query("user_id")
    userID, err := strconv.ParseUint(userIDStr, 10, 64)
    if err != nil || userID == 0 {
        c.JSON(http.StatusBadRequest, gin.H{
            "code": 400, "message": "无效的用户ID", "data": nil,
        })
        return
    }

    conn, err := ws.Upgrade(c.Writer, c.Request)
    if err != nil {
        return
    }

    client := &ws.Client{
        UserID: userID,
        Conn:   conn,
        Send:   make(chan []byte, 256),
    }

    h.Hub.Register(client)

    go client.WritePump()
    go client.ReadPump(h.Hub)
}
```

`Send: make(chan []byte, 256)` 为每个客户端创建一个带缓冲的发送通道——256条消息的缓冲区。如果客户端消费不过来（网络慢），消息不会立即丢弃。但如果满了（客户端彻底断连），发送方会跳过而不是阻塞。

## 本章小结

本章搭建了一个完整的IM即时通讯系统，核心收获：

1. **WebSocket协议**：从HTTP升级到双向长连接，适合实时通信场景
2. **连接管理**：`sync.Map` 实现并发安全的连接池，支持注册/注销/查询
3. **消息路由**：单聊精准投递，群聊广播推送给所有群成员
4. **心跳机制**：Ping/Pong保活，30秒周期检测，60秒超时断连
5. **离线消息**：用户上线时自动推送未读消息，单聊群聊都覆盖
6. **读/写分离**：每个连接两个goroutine，一个读一个写，互不阻塞

IM系统是"实时系统"的入门范本。你学到的WebSocket连接管理、消息路由、心跳保活这些模式，在游戏服务器、实时协作编辑、IoT设备通信中同样适用。
