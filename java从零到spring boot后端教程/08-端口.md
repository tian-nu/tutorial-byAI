# 第08章 · 端口

> "你知道了服务器的IP地址（门牌号），但你敲的是哪扇门？一栋楼有很多房间——端口就是服务器上的'房间号'。"

## 08.1 端口是什么

**端口（Port）** 是一个0~65535的数字，用于区分同一台服务器上的不同服务。

```
同一台服务器 47.xx.xx.xx 上：
  端口22   → SSH远程登录服务
  端口80   → HTTP网站服务
  端口443  → HTTPS加密网站服务
  端口3306 → MySQL数据库服务
  端口6379 → Redis缓存服务
  端口8080 → Spring Boot应用（开发环境）
```

IP地址让你找到服务器，端口让你找到服务器上具体的服务。

```
IP + 端口 = 完整的服务地址
47.xx.xx.xx:8080 → 那台服务器上的Spring Boot应用
47.xx.xx.xx:3306 → 那台服务器上的MySQL数据库
```

## 08.2 端口号范围与分类

| 范围 | 类型 | 说明 |
|------|------|------|
| 0-1023 | 知名端口 | 需要管理员权限才能监听，预留给标准服务 |
| 1024-49151 | 注册端口 | 应用程序常用范围 |
| 49152-65535 | 动态/私有端口 | 操作系统自动分配（客户端连服务器时） |

### 常见知名端口

| 端口 | 服务 | 你会遇到它的场景 |
|------|------|----------------|
| 22 | SSH | 远程登录Linux服务器 |
| 80 | HTTP | 网站的默认端口（浏览器隐藏了它） |
| 443 | HTTPS | 加密网站的默认端口 |
| 3306 | MySQL | 数据库连接 |
| 5432 | PostgreSQL | 另一种数据库连接 |
| 6379 | Redis | 缓存服务 |
| 27017 | MongoDB | 文档数据库 |

## 08.3 Spring Boot默认8080

Spring Boot的默认端口是 `8080`。这不是巧合——1024以上不需要管理员权限，开发时随便用。

```yaml
# application.yml — 默认配置
server:
  port: 8080
```

启动后的日志：

```
Tomcat started on port(s): 8080 (http)
```

然后你访问 `http://localhost:8080` 就能看到你的应用。

### 如果端口被占用

```bash
java -jar myapp.jar
```

结果：

```
***************************
APPLICATION FAILED TO START
***************************

Description:
Web server failed to start. Port 8080 was already in use.
```

这意味着8080端口已经被另一个程序占用了。排查方法：

**Windows:**
```powershell
netstat -ano | findstr :8080
# 找到占用的PID，然后去任务管理器结束它
```

**Linux/macOS:**
```bash
lsof -i :8080
# 或者
netstat -tlnp | grep 8080
```

### 换端口

```yaml
# application.yml
server:
  port: 9090   # 换成9090
```

或者启动时指定：

```bash
java -jar myapp.jar --server.port=9090
```

## 08.4 客户端端口 vs 服务端端口

当浏览器访问 `http://47.xx.xx.xx:8080/api/users` 时：

```
客户端（浏览器）                    服务端（服务器 47.xx.xx.xx）
┌─────────────────┐               ┌─────────────────────────┐
│ 源IP: 192.168.1.5│               │ 目标IP: 47.xx.xx.xx      │
│ 源端口: 52341    │ ────TCP连接──→  │ 目标端口: 8080           │
│ (系统随机分配)   │               │ (Spring Boot监听)        │
└─────────────────┘               └─────────────────────────┘
```

注意：
- **服务端端口**是固定的（8080），你必须在代码或配置里指定
- **客户端端口**是操作系统随机分配的（52341这种），你不需要管

同一个客户端可以同时发起多个连接到同一个服务端端口——操作系统保证每个连接的 `<源IP, 源端口, 目标IP, 目标端口>` 四元组是唯一的。

> 🤔 想多一点：这就是为什么一台服务器能同时处理成千上万个客户端请求——每个连接通过不同的客户端端口区分。Tomcat的200个线程轮流处理这些连接。200个线程处理10000个连接，靠的是非阻塞IO——一个线程不是死等一个慢客户端，而是"你先等着，好了叫我"。

## 08.5 防火墙与端口安全

服务器上的端口不是对全世界开放的——**防火墙**控制哪些端口能被外部访问。

```
公网访问 → 防火墙规则:
  ✅ 允许 80 (HTTP)
  ✅ 允许 443 (HTTPS)
  ✅ 允许 22 (SSH, 但限制来源IP)
  ❌ 拒绝 3306 (MySQL, 不允许外部直接访问数据库)
  ❌ 拒绝 6379 (Redis, 不允许外部直接访问缓存)
```

**安全原则**：只开放必要的端口。数据库端口（3306）绝对不要对公网开放——否则会被扫描到然后暴力破解。

## 本章小结

| 学了什么 | 一句话说明 |
|----------|-----------|
| 端口 | 0-65535的数字，区分同一台服务器上的不同服务 |
