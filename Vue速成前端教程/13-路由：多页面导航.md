## 一、目标与完成效果

**一句话目标**：用 Vue Router 把6个空页面串起来，点击导航栏就能在页面之间跳转。

**完成后的可观测效果**：
- 浏览器地址栏输入 `/` 看到首页，输入 `/login` 看到登录页，输入 `/post/3` 看到「文章3的详情页」
- 页面上有一个导航栏，点击「首页」「登录」等链接，页面内容切换，但页面不刷新（SPA 的核心体验）
- URL 变化时浏览器不会整页重新加载——这就是单页应用（SPA）的魔法

### 步骤1：理解路由——图书馆的索引卡片

#### 我在做什么？

先别急着写代码，理解「路由」这个概念。

**比喻：图书馆的索引卡片柜**

想象你走进一座大图书馆。你不可能把整座图书馆的书一次性全搬到你桌上——太重了，你也不需要。你只需要：
1. 走到索引卡片柜前（**路由表**）
2. 查到你想要的书在哪个书架（**匹配URL**）
3. 走到对应书架，拿出那一本（**显示对应组件**）

Vue Router 就是这座图书馆的**卡片柜**。URL 是你想找的书名，组件是你实际拿到的那本书。

```
URL: /post/42    →   路由表查找    →   显示 PostDetail.vue
URL: /login      →   路由表查找    →   显示 Login.vue
URL: /           →   路由表查找    →   显示 Home.vue
```

#### 关键概念

| 概念 | 图书馆比喻 | Vue 里是什么 |
|------|-----------|-------------|
| **路由表** (routes) | 卡片柜里的所有卡片 | 一个数组，每条记录是 `{path, component}` |
| **路径** (path) | 卡片上写的书架编号 | URL 的路径部分，如 `/post/42` |
| **组件** (component) | 对应编号书架上的那本书 | `.vue` 文件 |
| **路由出口** (router-view) | 你读书的那张桌子 | `<RouterView />` 标签 |

> 📌 **术语标记**：「Vue Router」「router-link」「router-view」「动态路由」「编程式导航」将收录于术语附录。

#### 我做得对不对？

- [ ] 你能用自己的话解释：为什么 SPA 不需要刷新页面就能切换内容？
  - 答案：因为 `<RouterView />` 只是一个「占位框」，Vue Router 根据 URL 把对应的组件塞进这个框里，整个过程不经过服务器，所以不刷新。

### 步骤3：配置 Vue Router 路由表

#### 我在做什么？

创建「卡片柜」——告诉 Vue Router 哪个 URL 对应哪个组件。

**创建 `src/router/index.js`**：

```js
import { createRouter, createWebHistory } from 'vue-router'

// 导入所有页面组件
import Home from '../views/Home.vue'
import PostDetail from '../views/PostDetail.vue'
import Login from '../views/Login.vue'
import Register from '../views/Register.vue'
import CreatePost from '../views/CreatePost.vue'
import EditPost from '../views/EditPost.vue'

// 路由表：每一条记录 = 一张索引卡片
const routes = [
  { path: '/',              name: 'home',        component: Home },
  { path: '/post/:id',      name: 'post-detail',  component: PostDetail },
  { path: '/login',         name: 'login',        component: Login },
  { path: '/register',      name: 'register',     component: Register },
  { path: '/create',        name: 'create-post',  component: CreatePost },
  { path: '/edit/:id',      name: 'edit-post',    component: EditPost },
]

// 创建路由器实例
const router = createRouter({
  history: createWebHistory(),  // 使用 HTML5 History 模式（URL没有#号）
  routes,                        // 简短写法，等价于 routes: routes
})

export default router
```

#### 路由表逐行解释

| 代码 | 含义 |
|------|------|
| `path: '/'` | URL 路径是根路径（首页） |
| `path: '/post/:id'` | `:id` 是动态参数，匹配 `/post/1`、`/post/42`、`/post/abc` 等 |
| `name: 'home'` | 给路由起个名字，后续可以用名字跳转，不用硬编码路径 |
| `component: Home` | 匹配到该路径时，显示 `Home.vue` 组件 |
| `createWebHistory()` | URL 干净模式（没有 `#`），需要服务器配合 |
| `createWebHashHistory()` | URL 带 `#` 的模式（如 `/#/login`），不需要服务器配置 |

#### 我做得对不对？

- [ ] `routes` 数组正好有6条记录？一条对应一个页面？
- [ ] `/post/:id` 和 `/edit/:id` 中的 `:id` 前面有冒号？（冒号表示这是动态参数，不是字面字符）

#### 不对怎么办？

❌ **错误**：把 `path: '/post/:id'` 写成了 `path: '/post/id'`
- 后果：只有访问 `/post/id` 这个字面地址才会匹配，`/post/42` 匹配不到。

✅ **正确**：`path: '/post/:id'`，冒号表示 `id` 是一个变量。

### 步骤4：在 main.js 中注册路由，改造 App.vue

#### 我在做什么？

把路由器「装」到 Vue 应用上，就像给汽车装上方向盘。然后改造 `App.vue`，让它用 `<RouterView />` 来显示当前页。

**修改 `src/main.js`**：

```js
import { createApp } from 'vue'
import App from './App.vue'
import router from './router'    // 新增：导入路由器

const app = createApp(App)
app.use(router)                   // 新增：安装路由插件
app.mount('#app')
```

**修改 `src/App.vue`**：

```vue
<template>
  <div id="app">
    <!-- 导航栏：使用 router-link 做导航 -->
    <nav>
      <router-link to="/">🏠 首页</router-link>
      <router-link to="/create">✏️ 写文章</router-link>
      <router-link to="/login">🔑 登录</router-link>
      <router-link to="/register">📝 注册</router-link>
    </nav>

    <hr />

    <!-- 路由出口：当前页面的内容显示在这里 -->
    <router-view />
  </div>
</template>

<script setup>
// App.vue 作为根组件，不需要额外逻辑
</script>
```

#### 代码解释

| 代码 | 干什么的 |
|------|----------|
| `app.use(router)` | 把路由器注入整个应用，此后所有组件都能用 `useRouter()` 和 `useRoute()` |
| `<router-link to="/">` | 生成一个 `<a>` 标签，点击后不刷新页面，只切换 `<RouterView />` 里的内容 |
| `<router-view />` | 「占位框」，Vue Router 把匹配到的组件塞进这里 |

启动项目：
```bash
npm run dev
```

在浏览器里：

- [ ] 点击「登录」，显示「🔑 登录」？
- [ ] 点击「注册」，显示「📝 注册」？
- [ ] 地址栏的 URL 跟着变化？
- [ ] 页面没有闪烁或整页刷新？

#### 不对怎么办？

如果页面一片空白：
1. 打开浏览器开发者工具（F12），看 Console 是否有红色报错
2. 常见原因：`router/index.js` 中的组件路径写错了（比如 `../views/Home.vue` 少了一个 `.`）
3. 检查 `main.js` 中是否有 `app.use(router)`

如果点击链接后 URL 变了但页面没变：
- 检查 `App.vue` 中是否有 `<router-view />`（注意是小写、带短横线）

🤔 **想多一点**：`<router-link>` 和 `<a href="/login">` 有什么区别？`<a>` 点击后会向服务器发请求，整页刷新。`<router-link>` 点击后只切换组件，不刷新——这才是 SPA 的体验核心。

### 步骤5：动态路由——同一模板，不同内容

#### 我在做什么？

`/post/:id` 中的 `:id` 是动态的——不同文章有不同的ID，但它们共用同一个 `PostDetail.vue` 组件。我们需要在组件里拿到这个ID。

**修改 `src/views/PostDetail.vue`**：

```vue
<template>
  <div>
    <h1>📄 文章详情</h1>
    <p>当前文章ID：<strong>{{ id }}</strong></p>
  </div>
</template>

<script setup>
import { useRoute } from 'vue-router'

const route = useRoute()        // 获取当前路由对象
const id = route.params.id      // 从路径参数中取出 id
</script>
```

#### 代码解释

```
URL: /post/42
     ↓
route.params.id = "42"     ← useRoute() 帮你取出来
     ↓
在组件里直接显示 {{ id }}
```

| 代码 | 含义 |
|------|------|
| `useRoute()` | 获取「当前这次导航」的路由信息（参数、路径等） |
| `route.params` | 路径参数对象，访问 `/post/42` 时 `params.id` 就是 `"42"` |
| `route.params.id` | 取出名为 `id` 的动态参数值 |

#### 验证

在浏览器里手动输入地址：
- `http://localhost:5173/post/1` → 显示「当前文章ID：1」
- `http://localhost:5173/post/42` → 显示「当前文章ID：42」
- `http://localhost:5173/post/hello` → 显示「当前文章ID：hello」

全部正确即通过！

### 步骤6：编程式导航——用代码跳转

#### 我在做什么？

有时候不能用 `<router-link>`——比如「登录成功后自动跳转到首页」，这需要在 JavaScript 代码里执行跳转。这叫「编程式导航」。

**比喻**：`<router-link>` 是楼梯，你自己走过去；编程式导航是电梯，代码把你送过去。

在任意组件的 `<script setup>` 中：

```js
import { useRouter } from 'vue-router'

const router = useRouter()

// 跳转到首页
router.push('/')

// 跳转到文章详情页（动态参数）
router.push('/post/42')

// 用路由名字跳转（更推荐！路径变了名字不变）
router.push({ name: 'home' })
router.push({ name: 'post-detail', params: { id: 42 } })
```

#### 两种跳转方式对比

| 方式 | 写法 | 什么时候用 |
|------|------|-----------|
| `<router-link>` | `<router-link to="/login">登录</router-link>` | HTML 模板里，用户点击链接 |
| `router.push()` | `router.push('/login')` | JS 代码里，比如表单提交后跳转 |

❌ **错误用法**：用 `window.location.href = '/login'` 跳转
- 后果：页面整页刷新，SPA 体验被破坏。

✅ **正确用法**：用 `router.push('/login')`

🤔 **想多一点**：`router.push()` 叫这个名字是因为它把新路径「压」到浏览器历史记录栈里。用户点「后退」按钮时，就像从栈顶「弹出」一条记录，回到上一页。这就是为什么 SPA 能支持浏览器前进/后退——Vue Router 在管理一个虚拟的历史记录栈。

## 四、术语附录

| 术语 | 解释 | 是否项目特有 |
|------|------|:---:|
| **Vue Router** | Vue 官方的路由库。负责管理 URL 与组件之间的映射关系，是 SPA 的核心。 | 否 |
| **router-link** | Vue Router 提供的导航组件，渲染为 `<a>` 标签，但点击不会刷新页面。 | 否 |
| **router-view** | 路由出口组件，根据当前 URL 在此处渲染对应的页面组件。相当于「占位框」。 | 否 |
| **动态路由** | URL 路径中包含可变部分（如 `/post/:id`），同一组件根据参数显示不同内容。 | 否 |
| **编程式导航** | 在 JavaScript 代码中用 `router.push()` 跳转，而非点击 `<router-link>`。 | 否 |
| **SPA（单页应用）** | Single Page Application，整个应用只有一个 HTML 页面，内容通过 JS 动态切换，不刷新浏览器。 | 否 |

## 七、已知坑点与禁止事项

- ⚠️ **`useRoute()` vs `useRouter()` 别搞混**：`useRoute()` 是「读」当前路由信息（参数等），`useRouter()` 是「写」跳转操作。记忆口诀：**带 r 的那个是 Router（路由器），管跳转；不带 r 的是 Route（路由），管信息。**
- ⚠️ **组件导入路径**：`../views/Home.vue` 中的 `..` 表示上级目录。写错了会导致白屏，打开 F12 控制台看报错即可定位。
