## 核心心法：第一版要粗糙

很多新手有一个误区：觉得AI第一版就该写出完美的代码。

**大错特错。**

AI编程的正确流程是：

> **第一版能跑 → 逐轮优化 → 到达目标就停。**

不要追求第一版完美。你追求的是：**第一版能在5分钟内跑起来，然后再花30分钟打磨到生产级。**

打个比方：
> 雕塑家不会一开始就雕眼睛。他先雕出大致的轮廓，再逐步细化。AI编程也是一样——先让AI给你一个"能跑的大轮廓"，再一版一版地精雕细琢。

---

## 逐轮优化模型

| 轮次 | 目标 | 关注点 |
|------|------|--------|
| 第1轮 | 能跑 | 代码能运行，不报错，核心功能实现 |
| 第2轮 | 健壮 | 加错误处理、输入验证、边界条件 |
| 第3轮 | 可读 | 加注释、改命名、提取常量、拆分函数 |
| 第4轮 | 安全 | 防SQL注入、密码加密、权限检查 |
| 第5轮 | 可测 | 加单元测试、加类型注解、加文档 |

**注意：不是每轮都要全做。** 根据你的项目需求，做到第2-3轮就够用了。只有真正要上线的项目才需要做到第5轮。

## 活生生的迭代过程：从垃圾代码到生产级代码

下面展示一次**真实的多轮对话记录**（模拟）。需求是：写一个"文件行数统计工具"。

### 第1轮：能跑

**你的Prompt：**

```
帮我用Python写一个命令行工具，统计一个文件夹里所有Python文件的行数。
```

**AI输出（第1版）：**

```python
import os

def count_lines(path):
    total = 0
    for root, dirs, files in os.walk(path):
        for file in files:
            if file.endswith('.py'):
                with open(os.path.join(root, file)) as f:
                    total += len(f.readlines())
    print(f"总行数: {total}")

count_lines(".")
```

**评价：** 能跑，但是——
- 路径写死了（`.`）
- 没有错误处理
- 没有统计空行/注释行
- 输出太简陋

**✅ 能跑，进入下一轮。**

---

### 第2轮：健壮

**你的Prompt：**

```
这个代码能跑，但需要改进：

1. 让用户通过命令行参数指定路径，而不是写死
2. 加上错误处理：文件读不了的情况、路径不存在的情况
3. 统计时不包含空行
4. 输出更详细：总文件数、总行数、有效代码行数（不含空行）

请修改。
```

**AI输出（第2版）：**

```python
import os
import sys

def count_lines(path):
    if not os.path.exists(path):
        print(f"错误: 路径 '{path}' 不存在")
        return
    
    if not os.path.isdir(path):
        print(f"错误: '{path}' 不是一个目录")
        return
    
    total_files = 0
    total_lines = 0
    effective_lines = 0
    
    for root, dirs, files in os.walk(path):
        for file in files:
            if file.endswith('.py'):
                filepath = os.path.join(root, file)
                try:
                    with open(filepath, 'r', encoding='utf-8') as f:
                        lines = f.readlines()
                        total_files += 1
                        total_lines += len(lines)
                        effective_lines += sum(1 for line in lines if line.strip())
                except Exception as e:
                    print(f"警告: 无法读取文件 {filepath}: {e}")
    
    print(f"统计结果:")
    print(f"  Python文件数: {total_files}")
    print(f"  总行数(含空行): {total_lines}")
    print(f"  有效代码行数: {effective_lines}")

if __name__ == "__main__":
    if len(sys.argv) < 2:
        print("用法: python count_lines.py <目录路径>")
        sys.exit(1)
    
    count_lines(sys.argv[1])
```

**评价：** 健壮多了——命令行参数、错误处理、空行过滤都有了。但代码全在一个函数里，可读性不太好。

**✅ 健壮，进入下一轮。**

---

### 第3轮：可读

**你的Prompt：**

```
代码功能没问题了，但可读性需要提升：

1. 把统计逻辑拆成多个小函数
2. 加类型注解 (typing)
3. 把输出格式改成更美观的表格形式
4. 加docstring注释
```

**AI输出（第3版）：**

```python
"""
文件行数统计工具
用法: python count_lines.py <目录路径>
"""

import os
import sys
from typing import Tuple

def validate_path(path: str) -> str:
    """验证路径是否存在且为目录。"""
    if not os.path.exists(path):
        raise ValueError(f"路径 '{path}' 不存在")
    if not os.path.isdir(path):
        raise ValueError(f"'{path}' 不是一个目录")
    return path

def find_python_files(path: str) -> list[str]:
    """递归查找目录下所有Python文件。"""
    py_files = []
    for root, _, files in os.walk(path):
        for file in files:
            if file.endswith('.py'):
                py_files.append(os.path.join(root, file))
    return py_files

def count_file_lines(filepath: str) -> Tuple[int, int]:
    """
    统计单个文件的行数。
    
    Returns:
        (总行数, 有效代码行数)
    """
    with open(filepath, 'r', encoding='utf-8') as f:
        lines = f.readlines()
    
    total = len(lines)
    effective = sum(1 for line in lines if line.strip())
    return total, effective

def print_report(file_count: int, total_lines: int, effective_lines: int) -> None:
    """打印统计报告。"""
    empty_lines = total_lines - effective_lines
    
    print("=" * 40)
    print("           文件行数统计报告")
    print("=" * 40)
    print(f"  📁 Python文件数:     {file_count:>6}")
    print(f"  📝 总行数:           {total_lines:>6}")
    print(f"  ✍️  有效代码行数:     {effective_lines:>6}")
    print(f"  ⬜ 空行数:           {empty_lines:>6}")
    if total_lines > 0:
        rate = effective_lines / total_lines * 100
        print(f"  📊 有效代码占比:     {rate:>5.1f}%")
    print("=" * 40)

def main():
    if len(sys.argv) < 2:
        print("用法: python count_lines.py <目录路径>")
        print("示例: python count_lines.py ./src")
        sys.exit(1)
    
    try:
        path = validate_path(sys.argv[1])
        py_files = find_python_files(path)
        
        if not py_files:
            print("未找到Python文件")
            return
        
        total_files = 0
        total_lines = 0
        effective_lines = 0
        
        for filepath in py_files:
            try:
                t, e = count_file_lines(filepath)
                total_files += 1
                total_lines += t
                effective_lines += e
            except Exception as e:
                print(f"⚠️  警告: 无法读取 {filepath}: {e}")
        
        print_report(total_files, total_lines, effective_lines)
        
    except ValueError as e:
        print(f"❌ 错误: {e}")
        sys.exit(1)
    except KeyboardInterrupt:
        print("\n\n已取消")
        sys.exit(0)

if __name__ == "__main__":
    main()
```

**评价：** 代码结构清晰，函数拆分合理，有类型注解和文档。但这只是统计行数，连注释行都算进去了。

**☑️ 可读，已经够用。但如果你想让它能区分注释行，进入下一轮。**

### 第4轮：功能增强

**你的Prompt：**

```
现在加一个功能：区分注释行和代码行。
- 统计时把"纯注释行"（井号开头的行）单独统计
- 有效代码行数 = 总行数 - 空行 - 纯注释行
- 报告里加一行显示注释行数
```

**AI输出（第4版）：**

```python
"""文件行数统计工具。"""

import os
import sys
from typing import Tuple
from dataclasses import dataclass

@dataclass
class FileStats:
    """单文件统计结果。"""
    total: int = 0
    code: int = 0
    comment: int = 0
    blank: int = 0

def validate_path(path: str) -> str:
    """验证路径是否存在且为目录。"""
    if not os.path.exists(path):
        raise ValueError(f"路径 '{path}' 不存在")
    if not os.path.isdir(path):
        raise ValueError(f"'{path}' 不是一个目录")
    return path

def find_python_files(path: str) -> list[str]:
    """递归查找目录下所有Python文件。"""
    py_files = []
    for root, _, files in os.walk(path):
        for file in files:
            if file.endswith('.py'):
                py_files.append(os.path.join(root, file))
    return py_files

def classify_line(line: str) -> str:
    """
    分类一行代码。
    
    Returns: 'code', 'comment', 或 'blank'
    """
    stripped = line.strip()
    if not stripped:
        return 'blank'
    if stripped.startswith('#'):
        return 'comment'
    return 'code'

def count_file_lines(filepath: str) -> FileStats:
    """统计单个文件的行数。"""
    with open(filepath, 'r', encoding='utf-8') as f:
        lines = f.readlines()
    
    stats = FileStats()
    stats.total = len(lines)
    
    for line in lines:
        category = classify_line(line)
        if category == 'code':
            stats.code += 1
        elif category == 'comment':
            stats.comment += 1
        else:
            stats.blank += 1
    
    return stats

def print_report(
    file_count: int,
    total_lines: int,
    code_lines: int,
    comment_lines: int,
    blank_lines: int
) -> None:
    """打印统计报告。"""
    print("=" * 45)
    print("          文件行数统计报告")
    print("=" * 45)
    print(f"  📁 Python文件数:    {file_count:>6}")
    print(f"  📝 总行数:          {total_lines:>6}")
    print(f"  ✅ 代码行数:        {code_lines:>6}")
    print(f"  💬 注释行数:        {comment_lines:>6}")
    print(f"  ⬜ 空行数:          {blank_lines:>6}")
    if total_lines > 0:
        print(f"  {'─' * 35}")
        print(f"  📊 代码占比:        {code_lines/total_lines*100:>5.1f}%")
        print(f"  📊 注释占比:        {comment_lines/total_lines*100:>5.1f}%")
    print("=" * 45)

def main():
    if len(sys.argv) < 2:
        print("用法: python count_lines.py <目录路径>")
        print("示例: python count_lines.py ./src")
        sys.exit(1)
    
    try:
        path = validate_path(sys.argv[1])
        py_files = find_python_files(path)
        
        if not py_files:
            print("未找到Python文件")
            return
        
        total_files = 0
        total_lines = 0
        code_lines = 0
        comment_lines = 0
        blank_lines = 0
        
        for filepath in py_files:
            try:
                stats = count_file_lines(filepath)
                total_files += 1
                total_lines += stats.total
                code_lines += stats.code
                comment_lines += stats.comment
                blank_lines += stats.blank
            except Exception as e:
                print(f"⚠️  警告: 无法读取 {filepath}: {e}")
        
        print_report(total_files, total_lines, code_lines, comment_lines, blank_lines)
        
    except ValueError as e:
        print(f"❌ 错误: {e}")
        sys.exit(1)
    except KeyboardInterrupt:
        print("\n\n已取消")
        sys.exit(0)

if __name__ == "__main__":
    main()
```

**评价：** 功能完善，代码清晰，已经是一个拿得出手的工具了。

**☑️ 功能完善，可以停了。**

## 迭代的完整轨迹回顾

| 轮次 | 变化 | 代码行数 | 质量 |
|------|------|---------|------|
| 第1版 | 能跑，但脆弱 | ~10行 | ⭐ |
| 第2版 | 加错误处理 + 命令行参数 | ~35行 | ⭐⭐ |
| 第3版 | 拆分函数 + 类型注解 + 好看输出 | ~80行 | ⭐⭐⭐ |
| 第4版 | 区分注释行 + dataclass | ~100行 | ⭐⭐⭐⭐ |

**从第1版到第4版，你花了大约30分钟，得到了一个结构清晰、功能完善、有文档的工具。** 如果你自己从头写，30分钟可能刚写到第2版。

---

## 什么时候该停止迭代？

**满足以下三个条件，就停：**

1. ✅ **满足需求**：功能实现了，没有遗漏
2. ✅ **能跑**：在你的环境里运行不报错
3. ✅ **没有明显Bug**：主流程没问题，边界条件处理了

**不要追求完美，不要过度优化。** 一个能跑的工具比一个"完美但还没写完"的工具价值大100倍。
