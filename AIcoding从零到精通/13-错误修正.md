## AI会写错代码吗？一定会！

先说一个残酷的事实：

> **AI写代码，第一版就有Bug的概率大约是70%。**

这不是AI不行，这是正常现象。原因很简单：

1. **AI不知道你的完整环境**：你的Python版本、依赖版本、系统配置，AI不可能全部知道。它写出来的代码可能在它的"想象环境"中能跑，但在你的真实环境中就跑不了。

2. **AI在"猜"**：AI本质上是在做概率预测，猜"最可能的代码是什么"。猜对了80%，剩下的20%就是Bug。

3. **AI可能用了过时的API**：AI的训练数据有截止日期，如果你用的库更新了API，AI可能不知道。

4. **AI不知道你的真实数据**：你的数据库表结构、字段名、数据类型——AI不知道，它只能猜。

**所以，遇到AI写错代码，不要慌，不要骂，这是正常的。** 关键是你怎么修正它。

---

## 错误的处理方式："不对，重新写" — 这没用

这是新手最常见的错误。AI写错了，你就说：

```
不对，重新写。
```

或者：

```
这个代码有问题，帮我修一下。
```

**为什么这没用？**

因为AI不知道"哪里有问题"。它只知道"用户不满意"，但不知道具体哪里不满意。它只能重新猜一次，而且大概率猜的和上次差不多——因为同样的Prompt，同样的概率分布，结果不会有太大变化。

**类比：**
> 你让厨师做了道菜，尝了一口说"不好吃，重做"——厨师完全不知道你是不喜欢咸了、淡了、还是辣了，他只能重新做一道一模一样的菜。

---

### ❌ 坏做法：让AI盲猜

```
你：帮我写一个用户登录功能
AI：（写了代码，但密码没加密）
你：不对，这个代码有问题，重新写
AI：（又写了一版，可能还是没加密，或者加密了但其他部分又错了）
你：还是不对，你到底会不会写？
AI：（第三版，可能更差了，因为AI在"乱猜"你到底要什么）
```

> 三轮下来，你什么都没得到，AI也越写越乱。

---

## ✅ 正确处理方式：三步修正法

**三步修正法：**

1. **把完整错误信息复制粘贴给AI**
2. **告诉AI你想要改成什么样**
3. **让AI解释它为什么这么写（可选，但推荐）**

### 三步修正法详解

#### 第一步：复制粘贴完整错误信息

不要只说"报错了"，把**完整的错误信息**贴给AI。包括：

- 错误类型（如 `TypeError`, `AttributeError`, `ImportError`）
- 错误信息（完整的traceback）
- 哪一行报错了
- 你运行的是什么命令

**为什么？** AI看到具体错误信息后，能精确定位问题，而不是盲猜。

#### 第二步：告诉AI你想要改成什么样

不要只说"修一下"，说清楚你期望的行为是什么。

**为什么要说"期望"？** 因为有时候AI写的代码逻辑上能跑，但不是你想要的效果。比如AI用MD5加密密码，但你要的是bcrypt——这不是"错误"，而是"不符合预期"。你必须告诉AI你的期望。

#### 第三步：让AI解释它为什么这么写

这一步是可选的，但强烈推荐。让AI解释它的思路，有两个好处：

1. 你能判断AI的逻辑是否合理
2. 如果AI的逻辑有问题，你能精确指出问题所在

## 案例：一次完整的错误修正过程

### 场景

你让AI写一个从API获取天气数据的功能。

### 你的Prompt（第一步请求）

```
帮我用Python写一个函数，获取北京今天的天气，返回温度和天气描述。
```

### AI的第一版代码

```python
import requests

def get_weather():
    api_key = "your_api_key_here"
    url = f"http://api.openweathermap.org/data/2.5/weather?q=Beijing&appid={api_key}&units=metric"
    
    response = requests.get(url)
    data = response.json()
    
    temperature = data['main']['temp']
    description = data['weather'][0]['description']
    
    return f"北京今天温度 {temperature}°C，天气：{description}"

print(get_weather())
```

### 你运行一下，报错了

```
Traceback (most recent call last):
  File "weather.py", line 12, in <module>
    print(get_weather())
  File "weather.py", line 7, in get_weather
    response = requests.get(url)
  File "/usr/lib/python3/site-packages/requests/api.py", line 75, in get
    return request('get', url, params=params, **kwargs)
requests.exceptions.ConnectionError: HTTPConnectionPool(host='api.openweathermap.org', port=80): 
Max retries exceeded with url: /data/2.5/weather?q=Beijing&appid=your_api_key_here&units=metric
```

### ❌ 错误做法

```
代码报错了，帮我修一下。
```

### ✅ 正确做法：三步修正

```
你写的天气查询代码报错了，下面是完整错误信息：

'''
Traceback (most recent call last):
  File "weather.py", line 12, in <module>
    print(get_weather())
  File "weather.py", line 7, in get_weather
    response = requests.get(url)
requests.exceptions.ConnectionError: HTTPConnectionPool(host='api.openweathermap.org', port=80): 
Max retries exceeded
'''

请帮我修正，要求：
1. 加上错误处理：如果网络不通，返回友好提示而不是崩溃
2. 加上超时设置：5秒超时
3. 加上API返回错误时的处理：如果API返回的不是200，打印具体错误信息
4. 请解释一下你修正的思路
```

### AI修正后的代码

```python
import requests

def get_weather():
    """
    获取北京今天的天气。
    
    Returns:
        str: 天气描述字符串，如果失败返回错误信息
    """
    api_key = "your_api_key_here"
    url = "http://api.openweathermap.org/data/2.5/weather"
    params = {
        "q": "Beijing",
        "appid": api_key,
        "units": "metric"
    }
    
    try:
        # 设置5秒超时
        response = requests.get(url, params=params, timeout=5)
        
        # 检查HTTP状态码
        if response.status_code != 200:
            return f"API请求失败，状态码: {response.status_code}"
        
        data = response.json()
        
        # 检查API返回的响应码
        if data.get("cod") != 200:
            return f"天气API错误: {data.get('message', '未知错误')}"
        
        temperature = data['main']['temp']
        description = data['weather'][0]['description']
        
        return f"北京今天温度 {temperature}°C，天气：{description}"
    
    except requests.exceptions.Timeout:
        return "请求超时，请检查网络连接"
    except requests.exceptions.ConnectionError:
        return "网络连接失败，请检查网络设置"
    except requests.exceptions.RequestException as e:
        return f"请求出错: {e}"
    except (KeyError, TypeError) as e:
        return f"解析天气数据失败: {e}"

print(get_weather())
```

**AI的解释：**

> 修正思路：
> 1. 把URL参数提取到params字典中，代码更清晰
> 2. 添加了timeout=5，防止请求无限等待
> 3. 用try/except捕获三类异常：超时、连接失败、其他请求异常
> 4. 添加了HTTP状态码检查和API响应码检查
> 5. 添加了数据解析异常处理，防止API返回格式变化时崩溃
> 6. 每种异常返回用户友好的中文提示，而不是抛出异常

**修正成功！** 从毫无防护的代码变成了健壮的生产级代码。

## 反复修正的极限：超过3次还不对，怎么办？

**规则：超过3次修正还不对 → 换个思路，重新描述问题，不要死磕。**

### 为什么不能死磕？

- AI可能陷入了"错误路径依赖"——它在一个错误的思路上越走越远
- 你反复修同一段代码，对话历史越来越长，上下文越来越乱
- 你越来越烦躁，AI也越来越"懵"

### 换思路的具体做法

1. **清上下文，开新对话**（参考第11章）
2. **重新描述问题，换一种说法**：不要用同样的Prompt，换个角度
3. **换个AI模型**：如果Cursor的模型不行，试试Trae，或者换GPT-4、Claude
4. **拆成更小的步骤**：如果一个问题修了3次还没好，说明这个问题可能太大了，拆成两个小问题

### 死磕 vs 换思路

| 死磕 | 换思路 |
|------|--------|
| 对话历史30轮，上下文混乱 | 新对话，上下文干净 |
| AI在一个错误方向上反复尝试 | AI从零开始，不受之前错误影响 |
| 你越来越烦躁 | 你心态平和，思路清晰 |
| 修了5次，还是不对 | 换种说法，1次就对了 |
