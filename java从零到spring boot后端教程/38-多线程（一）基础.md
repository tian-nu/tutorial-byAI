# 第38章 多线程（一）：基础

> 你有没有一边听音乐一边写代码的经历？你的大脑同时在处理两件事——这就是"多线程"。在 Java 中，多线程让程序能同时做多件事：一个线程处理用户请求，另一个线程在后台保存日志，互不耽搁。但多线程也是一把双刃剑——用好了效率翻倍，用不好 bug 满天飞。

---

## 38.1 进程与线程

| 概念 | 比喻 | 特点 |
|------|------|------|
| 进程 | 一家独立运营的公司 | 拥有独立内存空间，进程间通信成本高 |
| 线程 | 公司里的员工 | 共享进程内存，通信成本低，是最小调度单位 |

一个 Java 程序至少有一个线程——`main` 线程。此外 JVM 还有 GC 线程、Finalizer 线程等后台线程。

---

## 38.2 创建线程的三种方式

### 38.2.1 方式一：继承 Thread

```java
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println(Thread.currentThread().getName() + " 正在运行");
    }
}

// 使用
MyThread t = new MyThread();
t.start();  // 启动线程，JVM 会调用 run() 方法
```

### 38.2.2 方式二：实现 Runnable（推荐）

```java
class MyTask implements Runnable {
    @Override
    public void run() {
        System.out.println(Thread.currentThread().getName() + " 正在运行");
    }
}

// 使用
Thread t = new Thread(new MyTask());
t.start();
```

**为什么推荐 Runnable？**
- Java 是单继承——如果继承了 Thread 就不能再继承其他类
- Runnable 将"任务"和"线程"解耦，更符合面向对象设计

### 38.2.3 方式三：Lambda 简化

```java
// 最简洁的方式
Thread t = new Thread(() -> {
    System.out.println(Thread.currentThread().getName() + " 正在运行");
});
t.start();
```

## 38.3 start() vs run()

这是一个**极其重要**的区别，也是面试高频题：

```java
Thread t = new Thread(() -> System.out.println("新线程执行"));
t.start();  // ✅ 正确：启动新线程
t.run();    // ❌ 错误：只在当前线程执行 run()，不会启动新线程！
```

| 方法 | 行为 |
|------|------|
| `start()` | 创建新线程，新线程执行 `run()`。一个线程只能 start 一次 |
| `run()` | 在当前线程中执行方法体，**不创建新线程** |

## 38.4 线程的 6 种状态

> 🏷️ 此术语需进附录：**线程生命周期**——Java 线程有 6 种状态：NEW → RUNNABLE → BLOCKED/WAITING/TIMED_WAITING → TERMINATED。状态由 JVM 管理，可以通过 `Thread.getState()` 获取。

```
          start()
 NEW ────────────→ RUNNABLE
                      │    ↑
                      ↓    │ 获得锁
                   BLOCKED─┘
                      │
                      ↓ sleep()时间到 / notify() / unpark()
                 TIMED_WAITING ──────────────┐
                      │                      │
                      ↓ wait()               │
                   WAITING ── notify() ──────┘
                      │
                      ↓ run()结束
                  TERMINATED
```

| 状态 | 含义 | 进入方式 |
|------|------|---------|
| NEW | 线程已创建，未启动 | `new Thread()` |
| RUNNABLE | 正在运行或等待 CPU | `start()` |
| BLOCKED | 等待获取锁 | 进入 `synchronized` 块时锁被占用 |
| WAITING | 无限等待 | `wait()`、`join()`、`LockSupport.park()` |
| TIMED_WAITING | 限时等待 | `sleep(ms)`、`wait(ms)`、`join(ms)` |
| TERMINATED | 线程执行完毕 | `run()` 方法结束 |

**查看线程状态：**

```java
Thread t = new Thread(() -> {
    try { Thread.sleep(1000); } catch (InterruptedException e) {}
});
System.out.println(t.getState()); // NEW
t.start();
System.out.println(t.getState()); // RUNNABLE（可能在运行或就绪）
t.join();  // 等待线程结束
System.out.println(t.getState()); // TERMINATED
```

## 38.5 sleep、yield、join

### 38.5.1 sleep() — 让线程休息

```java
System.out.println("开始");
Thread.sleep(2000);  // 当前线程休眠 2 秒（不释放锁！）
System.out.println("2秒后继续");
```

关键特性：
- **不释放锁**——如果线程持有锁，sleep 期间其他线程仍然拿不到锁
- **可能被中断**——`sleep()` 会抛出 `InterruptedException`，必须捕获
- 精确度不高——操作系统调度最小粒度通常为几毫秒，不能保证精确的毫秒级等待

### 38.5.2 yield() — 谦让 CPU

```java
Thread.yield();  // 暗示调度器："我愿意让出 CPU"
```

`yield()` 是一个**暗示（hint）**，不是强制命令。调度器可能忽略。通常用于调试或测试，实际开发中很少使用。

### 38.5.3 join() — 等待另一个线程

```java
Thread worker = new Thread(() -> {
    System.out.println("我开始干活...");
    try { Thread.sleep(2000); } catch (InterruptedException e) {}
    System.out.println("活干完了！");
});

worker.start();
System.out.println("主线程等着...");
worker.join();  // 主线程阻塞，直到 worker 执行结束
System.out.println("主线程继续");

// 输出：
// 主线程等着...
// 我开始干活...
// 活干完了！
// 主线程继续
```

`join(long millis)` 可以设置超时：

```java
worker.join(1000);  // 最多等 1 秒，超时就继续
```

## 38.6 守护线程（Daemon Thread）

守护线程是"服务型"线程——当所有非守护线程结束时，JVM 自动退出，不管守护线程是否还在运行。

```java
Thread daemon = new Thread(() -> {
    while (true) {
        System.out.println("守护线程工作中...");
        try { Thread.sleep(500); } catch (InterruptedException e) {}
    }
});
daemon.setDaemon(true);  // 必须在线程 start() 之前设置！
daemon.start();

Thread.sleep(2000);
System.out.println("主线程结束，守护线程也会被自动终止");
```

典型守护线程：GC 线程、定时任务线程。

> 🤔 **想多一点**：为什么守护线程说停就停？因为 JVM 退出时，守护线程的 `finally` 块**不保证执行**。所以守护线程不适合做写文件、释放资源等操作。

## 38.7 ❌ 常见错误

### 错误1：调用 run() 而不是 start()

```java
// ❌ 没有启动新线程！
Thread t = new Thread(() -> heavyWork());
t.run();  // 在主线程执行，等于没开线程

// ✅ 正确
t.start(); // 启动新线程
```

### 错误2：重复 start()

```java
Thread t = new Thread(() -> System.out.println("run"));
t.start();
// t.start(); // ❌ IllegalThreadStateException！一个线程只能 start 一次
```

### 错误3：sleep() 在 synchronized 中不释放锁

```java
synchronized (lock) {
    Thread.sleep(5000);  // ❌ 其他等待 lock 的线程也要等 5 秒！
}
// 如果需要等待且释放锁，用 lock.wait()
```

### 错误4：忽略 InterruptedException

```java
// ❌ 吞掉中断信号
try {
    Thread.sleep(1000);
} catch (InterruptedException e) {
    // 什么都不做——其他代码无法知道线程被中断了
}

// ✅ 正确处理
try {
    Thread.sleep(1000);
} catch (InterruptedException e) {
    Thread.currentThread().interrupt();  // 恢复中断状态
    // 或者 return 退出
}
```

### 错误5：对 started 后的线程 setDaemon

```java
Thread t = new Thread(() -> {});
t.start();
// t.setDaemon(true);  // ❌ IllegalThreadStateException
// 必须 start() 之前设置
```

## 38.8 小结

| 知识点 | 一句话总结 |
|--------|-----------|
| 创建线程 | 推荐用 `new Thread(Runnable)` 或 Lambda |
| start vs run | `start()` 启动新线程，`run()` 只是方法调用 |
| 6 种状态 | NEW → RUNNABLE → BLOCKED/WAITING/TIMED_WAITING → TERMINATED |
| sleep() | 休眠指定毫秒，**不释放锁**，可能被中断 |
| join() | 等待另一个线程执行完毕 |
| 守护线程 | 非守护线程全结束时 JVM 退出，守护线程自动终止 |
| yiled() | 谦让 CPU，只是暗示不一定生效 |

## 38.9 自测题

**1.** 以下代码会启动几个线程？

```java
Thread t1 = new Thread(() -> System.out.println("A"));
t1.run();
t1.start();
Thread t2 = new Thread(() -> System.out.println("B"));
t2.start();
```

**2.** 线程在 `sleep()` 期间持有锁吗？线程在 `wait()` 期间持有锁吗？

**3.** 以下代码有什么问题？

```java
Thread t = new Thread(() -> {
    try { Thread.sleep(1000); } catch (InterruptedException e) {}
    System.out.println("done");
});
t.start();
t.setDaemon(true);
```

> 答案提示：1. 两个——`t1.run()` 在主线程执行（不启动新线程），`t1.start()` 和 `t2.start()` 各启动一个；2. sleep 持有锁，wait 释放锁；3. `setDaemon(true)` 必须在 `start()` 之前调用，否则抛 `IllegalThreadStateException`。
