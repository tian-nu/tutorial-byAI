# 第40章 多线程（三）：高级并发

> 前两章我们学了"手动挡"开车（自己创建线程、自己加锁）。这一章学"自动挡"——线程池自动管理线程、CompletableFuture 像搭积木一样编排任务。这些是现代 Java 并发编程的标配，Spring Boot 后端开发中每天都会用到。

---

## 40.1 为什么需要线程池

每次 `new Thread().start()` 的成本相当高：
1. 创建/销毁线程涉及系统调用
2. 线程的栈内存（默认约 1MB）占用资源
3. 线程数不受控可能耗尽系统资源

**线程池**：预创建一批线程，任务来了直接执行，用完回收不销毁。

## 40.2 ThreadPoolExecutor 参数详解

> 🏷️ 此术语需进附录：**线程池**（ThreadPoolExecutor）——维护一组可复用的工作线程，通过核心参数控制并发度、排队策略和拒绝策略。

```java
ThreadPoolExecutor pool = new ThreadPoolExecutor(
    2,                     // corePoolSize：核心线程数（常驻线程）
    5,                     // maximumPoolSize：最大线程数
    60,                    // keepAliveTime：非核心线程空闲存活时间
    TimeUnit.SECONDS,      // 时间单位
    new LinkedBlockingQueue<>(10),  // 工作队列（容量 10）
    Executors.defaultThreadFactory(),
    new ThreadPoolExecutor.CallerRunsPolicy()  // 拒绝策略
);
```

### 参数配置逻辑

```
提交任务 → 核心线程 < corePoolSize？
           ├─ 是 → 创建新线程执行
           └─ 否 → 队列满了？
                   ├─ 否 → 放入队列等待
                   └─ 是 → 线程数 < maximumPoolSize？
                           ├─ 是 → 创建新线程执行
                           └─ 否 → 执行拒绝策略
```

### 四种拒绝策略

| 策略 | 行为 |
|------|------|
| `AbortPolicy`（默认） | 抛 `RejectedExecutionException` |
| `CallerRunsPolicy` | 由提交任务的线程自己执行 |
| `DiscardPolicy` | 静默丢弃 |
| `DiscardOldestPolicy` | 丢弃队列中最老的任务 |

### 常用线程池（通过 Executors 创建）

```java
// ❌ 不推荐的写法——后续会解释原因
ExecutorService pool1 = Executors.newFixedThreadPool(4);   // 固定大小
ExecutorService pool2 = Executors.newCachedThreadPool();   // 弹性伸缩
ExecutorService pool3 = Executors.newSingleThreadExecutor(); // 单线程
ExecutorService pool4 = Executors.newScheduledThreadPool(2); // 定时任务
```

## 40.3 ❌ Executors 的陷阱（重要！）

### 陷阱1：newFixedThreadPool 和 newSingleThreadExecutor 的队列无界

```java
// ❌ 危险！内部使用无界队列 LinkedBlockingQueue
ExecutorService pool = Executors.newFixedThreadPool(4);

// 如果持续提交任务，队列无限增长 → OutOfMemoryError！
for (int i = 0; i < Integer.MAX_VALUE; i++) {
    pool.submit(() -> {
        Thread.sleep(10000);  // 任务执行慢
        return null;
    });
}
```

### 陷阱2：newCachedThreadPool 的线程数无界

```java
// ❌ 危险！最大线程数为 Integer.MAX_VALUE
ExecutorService pool = Executors.newCachedThreadPool();

// 每个任务创建一个新线程 → 创建数百万线程 → OOM！
```

### ✅ 正确做法：手动创建 ThreadPoolExecutor

```java
// ✅ 必须手动指定队列容量和线程上限
ThreadPoolExecutor pool = new ThreadPoolExecutor(
    4,
    8,
    60L, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(100),  // 有界队列
    new ThreadFactoryBuilder().setNameFormat("my-pool-%d").build(),  // 有意义的线程名
    new ThreadPoolExecutor.CallerRunsPolicy()
);
```

> **阿里巴巴 Java 开发手册强制规定**：线程池不允许使用 Executors 去创建，而是通过 ThreadPoolExecutor 的方式。这一规则就是为了避免 OOM。

## 40.4 Callable 与 Future

### 40.4.1 Runnable 的局限

`Runnable` 的 `run()` 没有返回值，也不能抛受检异常：

```java
// Runnable：无返回值，不能抛受检异常
Runnable task = () -> {
    int result = 1 + 1;
    // 拿不到这个 result！
};
```

### 40.4.2 Callable：有返回值

> 🏷️ 此术语需进附录：**Future**——代表异步计算的结果。通过 Future 可以查询任务是否完成、获取计算结果、取消任务。

```java
import java.util.concurrent.*;

Callable<Integer> task = () -> {
    Thread.sleep(1000);  // 模拟耗时计算
    return 1 + 1;
};

ExecutorService pool = Executors.newFixedThreadPool(2);
Future<Integer> future = pool.submit(task);

// 异步做其他事情...
System.out.println("主线程继续工作...");

// 获取结果（阻塞等待）
Integer result = future.get();  // 阻塞直到计算完成
System.out.println("结果：" + result); // 2

pool.shutdown();
```

### 40.4.3 Future 的方法

```java
Future<Integer> future = pool.submit(task);

future.isDone();       // 任务是否完成（正常、异常、取消都是 done）
future.isCancelled();  // 是否被取消
future.cancel(true);   // 取消任务（true 表示可以中断正在执行的任务）

// 带超时的 get
Integer result = future.get(3, TimeUnit.SECONDS);  // 最多等 3 秒，超时抛 TimeoutException
```

## 40.5 CompletableFuture：异步编程利器

> 🏷️ 此术语需进附录：**CompletableFuture**——Java 8 引入的增强版 Future，支持链式调用、组合多个异步任务、异常处理。是现代 Java 异步编程的核心工具。

### 40.5.1 创建异步任务

```java
import java.util.concurrent.CompletableFuture;

// 无返回值的异步任务
CompletableFuture<Void> future1 = CompletableFuture.runAsync(() -> {
    System.out.println("异步任务执行中...");
});

// 有返回值的异步任务
CompletableFuture<String> future2 = CompletableFuture.supplyAsync(() -> {
    return "Hello";
});

// 指定自定义线程池（否则使用 ForkJoinPool.commonPool()）
ExecutorService myPool = Executors.newFixedThreadPool(4);
CompletableFuture<String> future3 = CompletableFuture.supplyAsync(() -> "Hello", myPool);
```

### 40.5.2 链式调用

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> "Hello")                     // 异步任务1：返回 Hello
    .thenApply(s -> s + " World")                    // 转换：Hello World
    .thenApply(String::toUpperCase)                  // 转换：HELLO WORLD
    .thenApply(s -> s + "!")                         // 转换：HELLO WORLD!
    .whenComplete((result, error) -> {               // 完成时回调（无论成功失败）
        if (error == null) {
            System.out.println("结果：" + result);
        } else {
            System.out.println("出错：" + error.getMessage());
        }
    });

System.out.println(future.join());  // HELLO WORLD!
```

### 40.5.3 组合多个异步任务

```java
// thenCompose：一个任务依赖另一个任务的结果（扁平化）
CompletableFuture<String> combined = CompletableFuture
    .supplyAsync(() -> getUserById(1L))        // 查用户
    .thenCompose(user -> CompletableFuture
        .supplyAsync(() -> getOrdersByUser(user)));  // 查该用户的订单

// thenCombine：合并两个独立任务的结果
CompletableFuture<Integer> price1 = CompletableFuture.supplyAsync(() -> getPrice("商品A"));
CompletableFuture<Integer> price2 = CompletableFuture.supplyAsync(() -> getPrice("商品B"));

CompletableFuture<Integer> total = price1.thenCombine(price2, (p1, p2) -> p1 + p2);
System.out.println(total.join());  // 总价

// allOf：等待所有任务完成
CompletableFuture<Void> allDone = CompletableFuture.allOf(
    CompletableFuture.runAsync(() -> saveToDb()),
    CompletableFuture.runAsync(() -> sendEmail()),
    CompletableFuture.runAsync(() -> updateCache())
);
allDone.join();  // 等全部完成

// anyOf：任一任务完成就返回
CompletableFuture<Object> any = CompletableFuture.anyOf(price1, price2);
System.out.println("最先返回：" + any.join());
```

### 40.5.4 异常处理

```java
CompletableFuture<String> future = CompletableFuture
    .supplyAsync(() -> {
        if (Math.random() > 0.5) throw new RuntimeException("出错了！");
        return "成功";
    })
    .exceptionally(error -> {          // 处理异常，返回兜底值
        System.out.println("异常：" + error.getMessage());
        return "兜底默认值";
    })
    .thenApply(String::toUpperCase);

System.out.println(future.join());  // 要么 "成功" 的大写，要么 "兜底默认值" 的大写
```

### 40.5.5 实战：查询用户 → 查订单 → 算金额

```java
public class OrderService {
    public CompletableFuture<BigDecimal> calculateUserTotalSpending(Long userId) {
        return CompletableFuture
            .supplyAsync(() -> userRepo.findById(userId))          // 1. 查用户
            .thenApply(user -> user.orElseThrow(                   // 2. 用户不存在抛异常
                () -> new RuntimeException("用户不存在: " + userId)))
            .thenCompose(user -> CompletableFuture                 // 3. 查订单
                .supplyAsync(() -> orderRepo.findByUserId(user.getId())))
            .thenApply(orders -> orders.stream()                   // 4. 计算总金额
                .map(Order::getAmount)
                .reduce(BigDecimal.ZERO, BigDecimal::add))
            .exceptionally(error -> {                               // 5. 异常兜底
                log.error("计算失败", error);
                return BigDecimal.ZERO;
            });
    }
}
```

## 40.6 线程池的正确关闭

```java
ExecutorService pool = new ThreadPoolExecutor(...);

// 提交任务...
pool.submit(task1);
pool.submit(task2);

// ✅ 优雅关闭
pool.shutdown();  // 不再接受新任务，已提交的任务继续执行
try {
    if (!pool.awaitTermination(30, TimeUnit.SECONDS)) {
        pool.shutdownNow();  // 超时则强制终止
    }
} catch (InterruptedException e) {
    pool.shutdownNow();
    Thread.currentThread().interrupt();
}
```

| 方法 | 行为 |
|------|------|
| `shutdown()` | 平缓关闭，已提交任务执行完 |
| `shutdownNow()` | 立即关闭，尝试中断正在执行的任务 |
| `awaitTermination()` | 等待指定时间，返回是否完全终止 |

## 40.7 ❌ 常见错误

### 错误1：用 Executors 创建线程池导致 OOM

```java
// ❌ 生产环境严禁这样写！
ExecutorService pool = Executors.newFixedThreadPool(10);
// 队列无界 → 任务堆积 → OOM

// ✅ 手动指定参数
ThreadPoolExecutor pool = new ThreadPoolExecutor(
    10, 20, 60L, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(200)  // 有界队列！
);
```

### 错误2：忘记 shutdown 线程池

```java
// ❌ 线程池不关闭，JVM 不会退出
ExecutorService pool = Executors.newFixedThreadPool(2);
pool.submit(() -> System.out.println("done"));
// 程序永远不会退出！线程池中的线程是非守护线程
```

### 错误3：CompletableFuture 使用默认线程池

```java
// ⚠️ 默认使用 ForkJoinPool.commonPool()，核心数 = CPU 核数 - 1
// IO 密集型任务会严重拖慢整个 JVM
CompletableFuture.supplyAsync(() -> blockingIOOperation());

// ✅ 为 IO 密集任务使用自定义线程池
ExecutorService ioPool = Executors.newFixedThreadPool(20);
CompletableFuture.supplyAsync(() -> blockingIOOperation(), ioPool);
```

### 错误4：Future.get() 阻塞主线程

```java
// ❌ 串行调用多个异步任务，等于没异步
for (Future<Integer> future : futures) {
    Integer result = future.get(10, TimeUnit.SECONDS);  // 逐个等待！
    results.add(result);
}

// ✅ 用 CompletableFuture 组合
CompletableFuture.allOf(futures.toArray(new CompletableFuture[0])).join();
```

### 错误5：线程池的线程名称不设置导致排查困难

```java
// ❌ 默认线程名 pool-1-thread-1，不知道哪个池的哪个线程在干什么
ThreadPoolExecutor pool = new ThreadPoolExecutor(...);

// ✅ 设置有意义的线程名
ThreadFactory namedFactory = new ThreadFactoryBuilder()
    .setNameFormat("order-processor-%d")
    .build();
```

## 40.8 小结

| 知识点 | 一句话总结 |
|--------|-----------|
| 线程池参数 | corePoolSize → 队列 → maximumPoolSize → 拒绝策略 |
| Executors 陷阱 | 无界队列或无界线程数，生产环境会 OOM |
| Callable | 类似 Runnable 但有返回值、能抛受检异常 |
| Future | 异步计算结果的占位符，get() 阻塞获取 |
| CompletableFuture | 链式异步编程，支持组合、转换、异常处理 |
| 线程池关闭 | shutdown() + awaitTermination() 优雅关闭 |

## 40.9 自测题

**1.** ThreadPoolExecutor 中，核心线程数 2、最大线程数 5、队列容量 3。当同时提交 10 个任务时，会发生什么？

**2.** 以下代码有什么问题？

```java
ExecutorService pool = Executors.newCachedThreadPool();
for (int i = 0; i < 1_000_000; i++) {
    pool.submit(() -> {
        Thread.sleep(10000);
        return null;
    });
}
```

**3.** 用 CompletableFuture 实现：同时查询用户信息和订单信息（两个异步操作），等两个都完成后合并结果。

> 答案提示：1. 2 个进核心线程 + 3 个进队列 + 3 个创建临时线程（达到最大值 5）= 8 个被接受，剩余 2 个触发拒绝策略；2. CachedThreadPool 最大线程数为 Integer.MAX_VALUE，每个任务创建一个线程，100 万线程会导致 OOM；3. 用 `userFuture.thenCombine(orderFuture, (user, order) -> 合并)`。
