## 7.1 Cursor 是什么？——你的"AI 编程副驾驶"

> **📖 术语：Cursor**——一款"AI 编程工具"。它的底层是基于 VS Code（微软出品的免费代码编辑器）改造的，但在上面**深度集成了 AI 助手**。你可以把它理解成：**VS Code 的身体 + GPT/Claude 的大脑**。

**一个比喻**：

| 工具 | 类比 |
|------|------|
| 记事本 | 一支笔——你只能自己写 |
| VS Code（纯代码编辑器） | 一支高级钢笔——写起来更舒服，但还是你自己写 |
| **Cursor（AI 编程工具）** | 一个会写字的笔——你告诉它写什么，它帮你写 |

Cursor 不是"帮你查资料的 AI"，而是**直接帮你写代码的 AI**。你选中一段代码，告诉它"把这个改成什么什么"，它就改了。你新建一个文件，告诉它"写一个什么什么功能"，它帮你写。

## 7.2 下载安装 Cursor

### 注意：本书使用 Cursor 免费版即可

Cursor 有免费版和付费版。**免费版足够你学完这本书**。付费版的功能我们会在第07章末尾介绍，但现在你不需要花一分钱。

### Step 1：下载

1. 打开浏览器，访问 Cursor 官网：**https://cursor.com**
2. 页面会自动识别你的操作系统（Windows 或 Mac），显示对应的下载按钮
3. 点击 **"Download for Windows"**（Windows）或 **"Download for Mac"**（Mac）
4. 等待下载完成（大约 200MB，根据网络速度可能需要 2-5 分钟）

### Step 2：Windows 安装

1. 双击下载的 `.exe` 安装文件
2. 安装程序会问你是否要安装——点击 **"是"**
3. 等待进度条走完（大约 1 分钟）
4. 安装完成后，Cursor 会自动启动

> ⚠️ 如果 Windows 弹出"Windows 保护你的电脑"（SmartScreen）——这是因为 Cursor 不是从微软商店下载的。点击 **"更多信息"**，然后点击 **"仍要运行"**。这是正常的，Cursor 是安全的。

### Step 3：Mac 安装

1. 双击下载的 `.dmg` 文件，它会挂载成一个虚拟磁盘
2. 把 Cursor 图标**拖拽到 Applications（应用程序）文件夹**（这是 Mac 安装应用的标准方式）
3. 等待拷贝完成（大约 10 秒）
4. 打开 Applications 文件夹，双击 Cursor 启动

> ⚠️ 如果 Mac 弹出"无法验证开发者"——去"系统设置" → "隐私与安全性" → 点击"仍要打开"。这也是正常的，因为 Cursor 不是从 App Store 下载的。

## 7.3 首次启动与登录

Cursor 启动后，你会看到一个欢迎界面。

### 登录方式

Cursor 需要登录才能使用 AI 功能。登录方式有：

1. **邮箱注册**：输入邮箱，验证码登录
2. **Google 账号**：如果你有 Google 账号，直接点 Google 登录
3. **GitHub 账号**：如果你有 GitHub 账号，直接点 GitHub 登录

**推荐**：如果你三种方式都有，用 GitHub 登录最方便（一键登录，不用输入密码）。如果没有，邮箱注册就行。

> 关于"翻墙"：Cursor 的登录和 AI 服务在国内**可以直接访问**，不需要 VPN。但下载速度可能有时候比较慢——如果下载慢，耐心等待，或者试试在非高峰时段（比如早上）下载。

### 首次设置向导

登录后，Cursor 会引导你做几个初始设置。以下是推荐选择：

| 设置项 | 推荐选择 | 原因 |
|--------|---------|------|
| 键盘快捷键 | **Cursor 默认** | 后面会专门学快捷键，现在先用默认的 |
| 颜色主题 | 按个人喜好选（深色/浅色） | 纯个人偏好，深色护眼，浅色明亮 |
| AI 隐私设置 | 默认即可 | 我们没有敏感数据，默认设置足够 |
| 从 VS Code 导入 | 如果你没用过 VS Code，选 **"Skip"** | 零基础读者不需要导入 |

## 7.4 界面导览：每个区域是用来干什么的

Cursor 的界面，"打开后一脸懵"是正常的。下面我们按区域一个个说清楚。

```
┌──────────────────────────────────────────────────┐
│ ① 菜单栏（最顶部）  文件 编辑 选择 查看 转到 ...    │
├──────────┬───────────────────┬───────────────────┤
│          │                   │                   │
│ ② 侧边栏 │  ③ 编辑区         │  ④ AI 面板        │
│ (左侧)    │  (中间，最大区域)  │  (右侧)           │
│          │                   │                   │
│ 文件列表  │  你写代码/看代码   │  和 AI 对话的地方  │
│ 搜索      │  的地方           │                   │
│ 版本管理  │                   │                   │
│          │                   │                   │
├──────────┴───────────────────┴───────────────────┤
│ ⑤ 底部状态栏  Python 3.12.3 | 行 1, 列 1 | UTF-8  │
└──────────────────────────────────────────────────┘
```

**① 菜单栏（最顶部）**：和所有 Windows/Mac 应用一样，菜单栏。你暂时不需要关心这里。

**② 侧边栏（左侧）**：显示你当前打开的项目文件夹里的所有文件。就像 Windows 资源管理器或 Mac 访达的侧边栏一样。你可以在这里新建文件、删除文件、打开文件。

**③ 编辑区（中间，最大区域）**：这是你**写代码的地方**。你打开一个文件，它的内容就显示在这里。这也是你**选中代码让 AI 修改**的地方。

**④ AI 面板（右侧）**：这是 **Cursor 的核心**。这里你可以和 AI 对话：
- 让它帮你写代码
- 让它帮你解释代码
- 让它帮你修改代码
- 让它帮你找 Bug

**⑤ 底部状态栏**：显示当前文件的信息——用的什么语言、光标在第几行第几列、文件编码等。暂时不需要关心。

> 如果你打开后没有看到 AI 面板（右侧那个），按快捷键 **`Ctrl + L`**（Windows）或 **`Command + L`**（Mac）就能打开。

## 7.5 选择 AI 模型：GPT-4o vs Claude 3.5

Cursor 内置了多个 AI 模型供你选择。这就像你请了一个"翻译团队"，不同翻译擅长不同的东西。

> **📖 术语：GPT-4o**——OpenAI 出品的 AI 模型，目前（2026年）最常用的通用模型。速度快、理解能力强、代码生成质量高。**本书推荐的主力模型**。
>
> **📖 术语：Claude 3.5**——Anthropic 出品的 AI 模型。特点是**长文本处理能力极强**，如果你一次要给 AI 看几千行代码，Claude 比 GPT-4o 更适合。本书推荐的**长文本专用模型**。

**怎么选模型？**

在 AI 面板（右侧）的输入框上方，有一个模型选择下拉菜单。点击它，你会看到：

| 模型 | 推荐用途 | 本书推荐 |
|------|---------|---------|
| GPT-4o | 日常写代码、解释代码、修改代码 | ⭐ **主力模型**，90% 的场景用它 |
| Claude 3.5 Sonnet | 处理大量代码（几千行）、长对话 | 遇到长文本时切换 |
| GPT-4o-mini | 简单任务、速度快、免费额度多 | 快速小任务用 |
| 其他模型 | 不同场景 | 暂时不用管 |

**本书建议**：默认用 **GPT-4o**。当你需要一次性给 AI 看很多代码时（比如整个文件几百行），切换到 **Claude 3.5**。

## 7.7 你的第一个 AI 操作：让 Cursor 帮你写代码

终于到了最激动人心的时刻——**让 AI 帮你写第一行代码**。

### Step 1：新建一个文件

1. 在 Cursor 里，按 **`Ctrl + N`**（Windows）或 **`Command + N`**（Mac）新建文件
2. 按 **`Ctrl + S`**（Windows）或 **`Command + S`**（Mac）保存文件
3. 文件名输入：**`hello.py`**（`.py` 是 Python 文件的后缀名）
4. 选择一个文件夹保存（比如桌面或文档文件夹）

### Step 2：用 AI 写代码

1. 在编辑区（中间空白区域），按 **`Ctrl + K`**（Windows）或 **`Command + K`**（Mac）
2. 你会看到编辑区上方出现一个输入框，这就是 Cursor 的 **"行内 AI 指令"** 功能
3. 在输入框里输入以下文字：

```
写一个 Hello World Python 程序，打印后再说一句"这是我用AI写的第一行代码"
```

4. 按回车

**几秒钟后，你会看到**：AI 自动生成了代码，出现在编辑区里！

```python
print("Hello World")
print("这是我用AI写的第一行代码")
```

### Step 3：运行代码

1. 打开终端（在 Cursor 里可以直接打开终端：按 **`Ctrl + `` `**（反引号键，在 ESC 键下方，数字 1 左边））
2. 在终端里输入：

```bash
python hello.py
```

（Mac 用户用 `python3 hello.py`）

3. 按回车，你应该看到：

```
Hello World
这是我用AI写的第一行代码
```

✅ **恭喜！你刚刚完成了人生中第一次 AI 编程！** 你告诉 AI 要什么，AI 写了代码，你运行了代码，代码成功执行了——这就是 AI 编程的核心流程。

## 7.8 ⚠️ 常见问题排错

### 🔴 问题1：`Ctrl + K` 按了没反应

**可能原因 1**：你没有登录 Cursor。AI 功能需要登录才能用。

**解决**：检查右上角有没有你的头像。如果没有，点击登录。

**可能原因 2**：输入法冲突。中文输入法有时候会拦截 `Ctrl + K`。

**解决**：切换到英文输入法再试（按 `Shift` 切换中英文输入法）。

### 🔴 问题2：AI 生成的代码运行报错

**症状**：终端里出现红色的错误信息。

**最常见的情况**：忘记保存文件。AI 生成代码后，记得先按 `Ctrl + S`（Windows）或 `Command + S`（Mac）保存。

**其他情况**：把错误信息复制下来，直接在 AI 面板（右侧）里粘贴给 AI，问它"这个报错是什么意思？怎么修？"

### 🔴 问题3：下载 Cursor 特别慢

**原因**：Cursor 的服务器在国外，国内某些网络环境下下载可能较慢。

**解决方案**：
1. 耐心等待（通常 5-10 分钟内能下完）
2. 试试在早上或深夜下载（网络不那么拥挤的时候）
3. 如果实在太慢，可以考虑使用 VPN（但这不是必须的）

### 🔴 问题4：登录后 AI 面板不显示

**解决**：按 **`Ctrl + L`**（Windows）或 **`Command + L`**（Mac）手动打开 AI 面板。

### 🔴 问题5：AI 面板显示"额度已用完"

**这说明**：你的免费高级模型额度（每月 50 次）用完了。

**解决**：切换到 GPT-4o-mini（免费无限）。在 AI 面板的模型选择下拉菜单里选 "GPT-4o-mini" 即可。虽然响应质量稍低一些，但学习本书完全够用。下个月额度会自动重置。

## 自测

1. `Ctrl + K` 和 `Ctrl + L` 分别是什么功能？在什么场景下用哪个？
2. 如果你想让 AI 帮你写一个"计算 1 到 100 所有数字之和"的 Python 程序，你应该在 Cursor 里按什么键、输入什么？
3. 你发现 AI 生成的代码运行后报错了，你应该怎么做？（至少说出两种方法）

> **答案**：
> 1. `Ctrl + K` 是"行内 AI 指令"——在编辑区直接让 AI 生成或修改代码。`Ctrl + L` 是打开 AI 对话面板——在右侧面板和 AI 聊天。简单需求（"写一段代码"）用 `Ctrl + K`，复杂需求（"帮我解释这段代码、帮我找 Bug"）用 `Ctrl + L`。
> 2. 按 `Ctrl + K`（或 `Command + K`），输入"写一个计算 1 到 100 所有数字之和的 Python 程序"。
> 3. 方法一：把错误信息复制到 AI 面板（Ctrl+L），问 AI"这个报错是什么意思？怎么修？"。方法二：检查是否保存了文件（Ctrl+S），再试一次。方法三：检查 Python 是否安装正确（`python --version`）。
