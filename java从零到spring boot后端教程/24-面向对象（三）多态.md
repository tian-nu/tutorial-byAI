# 第24章 · 面向对象（三）多态

> "你去动物园，管理员对笼子里的各种动物说'表演才艺'——狗汪汪叫，猫喵喵叫，老虎吼。管理员不需要知道笼子里具体是什么动物，只需要说同一句话，每种动物按自己的方式回应。这就是多态——同一个指令，不同对象做出不同的反应。"

---

## 24.1 多态是什么？

**多态**（Polymorphism。此术语需进附录）字面意思是"多种形态"。在Java里，它指的是：**父类类型的变量可以引用子类对象，调用方法时实际执行子类的版本**。

```java
Animal a1 = new Dog();   // 用Animal类型变量引用Dog对象
Animal a2 = new Cat();   // 用Animal类型变量引用Cat对象

a1.makeSound();  // 汪汪汪！  ← 调的是Dog的makeSound
a2.makeSound();  // 喵喵喵！  ← 调的是Cat的makeSound
```

**多态的三个前提**：
1. 要有**继承**（Dog和Cat继承Animal）
2. 要有**方法重写**（子类重写了父类的makeSound方法）
3. 要有**父类引用指向子类对象**（`Animal a = new Dog()`）

---

## 24.2 向上转型：把子类当作父类

**向上转型**（Upcasting）：把子类对象赋值给父类类型的变量。这是**自动**的，不需要强制转换。

```java
// 这些全部合法——向上转型自动完成
Animal animal = new Dog();
Object obj = new Student("张三", 18, "2024001", 85.5);
Person person = new Student("李四", 20, "2024002", 90.0);

// 作为方法参数
public void makeItSound(Animal a) {
    a.makeSound();  // 不管传进来的是什么动物，都会调用自己的makeSound
}

makeItSound(new Dog());  // 汪汪汪！
makeItSound(new Cat());  // 喵喵喵！
```

**向上转型后，只能调用父类里定义的方法**：

```java
Dog dog = new Dog();
dog.makeSound();  // ✅
dog.wagTail();    // ✅ Dog特有的方法

Animal animal = new Dog();
animal.makeSound();  // ✅ Animal里有这个方法
animal.wagTail();    // ❌ 编译错误！Animal类型没有wagTail方法
```

> 🤔 **想多一点**：向上转型丢失了子类特有的方法。那为什么要用？因为灵活性——你可以写一个方法接受 `Animal` 类型，然后不管你传的是Dog、Cat还是Tiger，都能工作。这叫做**面向接口/抽象编程**。

---

## 24.3 动态绑定：编译一种类型，运行另一种类型

多态的底层机制叫做**动态绑定**（Dynamic Binding），也叫**晚期绑定**（Late Binding）。

```java
Animal a = new Dog();
a.makeSound();
```

| 阶段 | 发生了什么 |
|------|-----------|
| **编译时** | 编译器检查 `Animal` 类有没有 `makeSound()` 方法。有→编译通过。没有→编译报错 |
| **运行时** | JVM检查 `a` 实际指向的对象类型（`Dog`），找到Dog的 `makeSound()` 并执行 |

所以：**编译时看左边（声明类型），运行时看右边（实际类型）**。

### 方法查找路径

当调用 `a.makeSound()` 时，JVM从 `a` 的实际类型开始，沿着继承链向上查找：

```
a的实际类型是 Dog
  → Dog有makeSound()吗？有→执行
  → Dog没有的话→找Animal
    → Animal有吗？有→执行
      → Animal没有→找Object
        → 都找不到→编译时就报错了
```

---

## 24.4 instanceof：判断对象的真实类型

有时候你需要知道变量的真实类型，用 `instanceof` 运算符：

```java
public void handleAnimal(Animal a) {
    if (a instanceof Dog) {
        Dog dog = (Dog) a;  // 向下转型（强制）
        dog.wagTail();
    } else if (a instanceof Cat) {
        Cat cat = (Cat) a;
        cat.scratch();
    } else {
        a.makeSound();
    }
}
```

### Java 16+ 模式匹配：一步到位

从Java 16开始，`instanceof` 支持**模式匹配**——判断类型的同时声明变量：

```java
// 老式写法：两步
if (a instanceof Dog) {
    Dog dog = (Dog) a;
    dog.wagTail();
}

// Java 16+：一步
if (a instanceof Dog dog) {
    dog.wagTail();  // 直接用dog！
}

// 也可以用在复杂条件里
if (a instanceof Dog dog && dog.isFriendly()) {
    dog.wagTail();
}
```

> ❌ **常见错误**：在没有instanceof检查的情况下直接向下转型。`Animal a = new Dog(); Cat c = (Cat) a;` → 运行时抛出 `ClassCastException`。**先instanceof再转型，或用模式匹配。**

---

## 24.5 向下转型：把父类强转回子类

**向下转型**（Downcasting）：把父类类型的变量转回子类类型。这是**危险的**，必须手动强制转换，并且可能失败。

```java
Animal a = new Dog();
Dog dog = (Dog) a;     // ✅ a实际就是Dog

Animal a2 = new Cat();
Dog dog2 = (Dog) a2;   // ❌ ClassCastException！Cat不能转Dog
```

**向下转型前必须用instanceof检查**：

```java
if (a instanceof Dog) {
    Dog dog = (Dog) a;   // 安全
    dog.wagTail();
}
```

## 24.6 多态的真正威力：设计模式的基础

如果你不用多态，处理不同动物的代码可能是这样的：

```java
// 没有多态：每个动物都要写一份处理逻辑
public void makeItSound(Object animal) {
    if (animal instanceof Dog) {
        ((Dog) animal).bark();
    } else if (animal instanceof Cat) {
        ((Cat) animal).meow();
    } else if (animal instanceof Bird) {
        ((Bird) animal).chirp();
    }
    // 每加一个新动物，都得改这个方法——噩梦！
}
```

用多态：

```java
// 有多态：一行搞定所有动物
public void makeItSound(Animal animal) {
    animal.makeSound();  // 新动物只需要继承Animal，重写makeSound即可
}

// 新增动物完全不需要改这段代码
class Fish extends Animal {
    @Override
    public void makeSound() {
        System.out.println("咕噜咕噜...");
    }
}
makeItSound(new Fish());  // 咕噜咕噜... ——直接能用！
```

这就是**开闭原则（Open-Closed Principle）**：对扩展开放（可以加新动物），对修改关闭（不需要改makeItSound方法）。

## 本章小结

| 知识点 | 核心要点 |
|--------|----------|
| 多态 | 父类类型变量引用子类对象，调用方法时执行子类版本。此术语需进附录 |
| 向上转型 | 子类→父类，自动发生。但只能调用父类里定义的方法 |
| 向下转型 | 父类→子类，强制 `(类型)`。可能抛ClassCastException |
| 动态绑定 | 编译时检查声明类型的方法是否存在，运行时根据实际类型找到方法执行 |
| instanceof | 判断对象是否是某个类型。Java 16+支持模式匹配 `if(a instanceof Dog d)` |
| ClassCastException | 强转错误类型时抛出 |

## 自测题

**1.** 以下代码输出什么？

```java
class A {
    public String show() { return "A"; }
}
class B extends A {
    public String show() { return "B"; }
}
class C extends B {
    public String show() { return "C"; }
}

A obj = new C();
System.out.println(obj.show());
```

**2.** 以下代码能编译通过吗？如果能，输出什么？如果不能，错在哪？

```java
class Parent {
    public void hello() { System.out.println("Parent"); }
}
class Child extends Parent {
    public void hello() { System.out.println("Child"); }
    public void bye() { System.out.println("Bye"); }
}

Parent p = new Child();
p.bye();
```

**3.** `instanceof` 模式下，以下代码有什么问题？

```java
Animal a = new Dog();
if (a instanceof Cat c) {
    System.out.println("这是一只猫：" + c);
}
```
