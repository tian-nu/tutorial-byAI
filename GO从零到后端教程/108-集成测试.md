# 第108章 · 集成测试

> "单元测试确认了每个零件都能正常工作——螺丝是好的、齿轮是好的、马达是好的。但把它们装在一起之后，整台机器能转起来吗？齿轮咬合对吗？马达的转速传到齿轮后对不对？这就是集成测试（验证多个模块间协作是否正确的测试方法，详见附录I）要回答的问题：各个模块之间的协作是否正确。"

---

## 108.1 集成测试 vs 单元测试

### 不是一回事

| | 单元测试 | 集成测试 |
|---|---|---|
| **测试范围** | 单个函数/方法 | 多个组件协作 |
| **依赖处理** | Mock掉所有依赖 | 使用真实或近似真实的依赖 |
| **速度** | 毫秒级，几千个几秒跑完 | 秒级，几十个可能要几分钟 |
| **失败时定位** | 精确知道哪个函数出错了 | 只知道哪段协作流程错了 |
| **运行频率** | 每次保存都跑 | 提交前跑一次 |
| **问题** | 即使每个函数100%正确，合起来可能全是Bug | 跑得慢，维护成本高 |

### 一个具体的例子

**单元测试**测试 `UserService.Register()` 时Mock了 `UserRepo`——确实验证了"如果Repo返回一个已存在的用户，Register会返回'已被注册'错误"。但问题在于：

- 测试通过了，但真实MySQL的表结构不对（比如少了 `email` 列）
- 测试通过了，但`UserRepo.Create()` 的SQL语句有语法错误
- 测试通过了，但连接池配置不对，并发时会死锁

**集成测试**才能真正发现这些问题。

## 108.2 HTTP接口测试：httptest

Go标准库的 `net/http/httptest` 让你不需要启动真实HTTP服务器就能测试Handler。

### 基本模式

```go
func TestHealthEndpoint(t *testing.T) {
    router := setupRouter()

    req := httptest.NewRequest("GET", "/api/health", nil)
    w := httptest.NewRecorder()

    router.ServeHTTP(w, req)

    assert.Equal(t, 200, w.Code)
    assert.JSONEq(t, `{"status":"ok"}`, w.Body.String())
}
```

`httptest.NewServer` 启动一个真实的测试HTTP服务器（在一个随机端口上）：

```go
func TestFullAPI(t *testing.T) {
    router := setupRouter()
    server := httptest.NewServer(router)
    defer server.Close()

    resp, err := http.Get(server.URL + "/api/health")
    require.NoError(t, err)
    defer resp.Body.Close()

    body, _ := io.ReadAll(resp.Body)

    assert.Equal(t, 200, resp.StatusCode)
    assert.JSONEq(t, `{"status":"ok"}`, string(body))
}
```

`httptest.NewServer` vs `httptest.NewRequest`：
- `NewRequest`：不走网络，直接测试Handler——极快
- `NewServer`：走真实HTTP请求——覆盖更多（序列化、网络层等），但也稍慢

一般推荐用 `NewRequest` + `NewRecorder`，90%的场景够用了。

## 108.3 测试数据库管理

### 三种策略

**策略一：Docker启动临时MySQL（推荐）**

```go
func TestMain(m *testing.M) {
    cmd := exec.Command("docker", "run", "-d",
        "--name", "test-mysql",
        "-e", "MYSQL_ROOT_PASSWORD=testpass",
        "-e", "MYSQL_DATABASE=myapp_test",
        "-p", "3307:3306",
        "mysql:8.0")
    cmd.Run()

    time.Sleep(10 * time.Second)

    code := m.Run()

    exec.Command("docker", "rm", "-f", "test-mysql").Run()

    os.Exit(code)
}

func setupTestDB(t *testing.T) *sql.DB {
    db, err := sql.Open("mysql", "root:testpass@tcp(localhost:3307)/myapp_test?parseTime=true")
    require.NoError(t, err)

    _, err = db.Exec(`
        CREATE TABLE IF NOT EXISTS users (
            id BIGINT AUTO_INCREMENT PRIMARY KEY,
            email VARCHAR(255) NOT NULL UNIQUE,
            password VARCHAR(255) NOT NULL,
            created_at DATETIME DEFAULT CURRENT_TIMESTAMP
        )
    `)
    require.NoError(t, err)

    t.Cleanup(func() {
        db.Exec("DELETE FROM users")
        db.Close()
    })

    return db
}
```

**策略二：testcontainers-go（更优雅）**

```go
import "github.com/testcontainers/testcontainers-go"

func setupMySQLContainer(t *testing.T) (string, func()) {
    ctx := context.Background()

    req := testcontainers.ContainerRequest{
        Image:        "mysql:8.0",
        ExposedPorts: []string{"3306/tcp"},
        Env: map[string]string{
            "MYSQL_ROOT_PASSWORD": "testpass",
            "MYSQL_DATABASE":      "myapp_test",
        },
    }

    container, err := testcontainers.GenericContainer(ctx, testcontainers.GenericContainerRequest{
        ContainerRequest: req,
        Started:          true,
    })
    require.NoError(t, err)

    host, _ := container.Host(ctx)
    port, _ := container.MappedPort(ctx, "3306")
    dsn := fmt.Sprintf("root:testpass@tcp(%s:%s)/myapp_test?parseTime=true", host, port.Port())

    cleanup := func() {
        container.Terminate(ctx)
    }

    return dsn, cleanup
}
```

**策略三：SQLite内存模式**

第106章已经详细展示。速度最快，但SQL语法不完全兼容MySQL（比如AUTOINCREMENT语法、函数差异）。

### 选择建议

| 场景 | 推荐 |
|------|------|
| 单元测试（Repository层） | SQLite内存模式 |
| 集成测试（CI中） | testcontainers-go 或 Docker临时MySQL |
| 本地快速开发测试 | SQLite内存模式 |

## 108.4 实战：为Gin API写集成测试

### 完整的测试辅助函数

```go
package main

import (
    "database/sql"
    "encoding/json"
    "net/http"
    "net/http/httptest"
    "strings"
    "testing"

    "github.com/stretchr/testify/assert"
    "github.com/stretchr/testify/require"
)

func setupTestApp(t *testing.T) (*sql.DB, *gin.Engine) {
    db, err := sql.Open("sqlite3", ":memory:")
    require.NoError(t, err)

    _, err = db.Exec(`
        CREATE TABLE users (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            email TEXT NOT NULL UNIQUE,
            password TEXT NOT NULL
        )
    `)
    require.NoError(t, err)

    userRepo := repository.NewUserRepo(db)
    userService := service.NewUserService(userRepo)
    userHandler := handler.NewUserHandler(userService)

    router := gin.New()
    router.Use(gin.Recovery())

    api := router.Group("/api")
    {
        api.POST("/users", userHandler.Create)
        api.GET("/users/:id", userHandler.GetByID)
    }

    t.Cleanup(func() {
        db.Close()
    })

    return db, router
}
```

### 测试创建用户接口

```go
func TestCreateUser(t *testing.T) {
    _, router := setupTestApp(t)

    body := `{"email":"alice@example.com","password":"secret123"}`
    req := httptest.NewRequest("POST", "/api/users", strings.NewReader(body))
    req.Header.Set("Content-Type", "application/json")
    w := httptest.NewRecorder()

    router.ServeHTTP(w, req)

    assert.Equal(t, http.StatusCreated, w.Code)

    var resp map[string]interface{}
    err := json.Unmarshal(w.Body.Bytes(), &resp)
    require.NoError(t, err)

    assert.Equal(t, "alice@example.com", resp["email"])
    assert.NotEmpty(t, resp["id"])
}

func TestCreateUser_DuplicateEmail(t *testing.T) {
    _, router := setupTestApp(t)

    body := `{"email":"alice@example.com","password":"secret123"}`

    req1 := httptest.NewRequest("POST", "/api/users", strings.NewReader(body))
    req1.Header.Set("Content-Type", "application/json")
    w1 := httptest.NewRecorder()
    router.ServeHTTP(w1, req1)
    assert.Equal(t, http.StatusCreated, w1.Code)

    req2 := httptest.NewRequest("POST", "/api/users", strings.NewReader(body))
    req2.Header.Set("Content-Type", "application/json")
    w2 := httptest.NewRecorder()
    router.ServeHTTP(w2, req2)

    assert.Equal(t, http.StatusConflict, w2.Code)

    var resp map[string]interface{}
    json.Unmarshal(w2.Body.Bytes(), &resp)
    assert.Contains(t, resp["error"], "已存在")
}

func TestGetUser_NotFound(t *testing.T) {
    _, router := setupTestApp(t)

    req := httptest.NewRequest("GET", "/api/users/99999", nil)
    w := httptest.NewRecorder()

    router.ServeHTTP(w, req)

    assert.Equal(t, http.StatusNotFound, w.Code)
}
```

### 测试完整流程（端到端）

从创建到查询的完整链路：

```go
func TestCreateAndGetUser(t *testing.T) {
    _, router := setupTestApp(t)

    createBody := `{"email":"bob@example.com","password":"secret123"}`
    createReq := httptest.NewRequest("POST", "/api/users", strings.NewReader(createBody))
    createReq.Header.Set("Content-Type", "application/json")
    createW := httptest.NewRecorder()
    router.ServeHTTP(createW, createReq)
    require.Equal(t, http.StatusCreated, createW.Code)

    var createResp map[string]interface{}
    json.Unmarshal(createW.Body.Bytes(), &createResp)
    userID := createResp["id"]

    getReq := httptest.NewRequest("GET", "/api/users/"+fmt.Sprintf("%v", userID), nil)
    getW := httptest.NewRecorder()
    router.ServeHTTP(getW, getReq)

    assert.Equal(t, http.StatusOK, getW.Code)

    var getResp map[string]interface{}
    json.Unmarshal(getW.Body.Bytes(), &getResp)
    assert.Equal(t, "bob@example.com", getResp["email"])
}
```

## 🤔 想多一点：测试应该写多少？

这是整个测试领域最常被问到的问题。答案是**看情况**，但有指导原则：

**核心业务逻辑（支付、认证、数据一致性）** → 单元测试覆盖率80%+，加上集成测试覆盖完整业务流程。

**CRUD代码** → 不需要为每个简单的增删改查写单元测试。Repository层用SQLite做一次覆盖即可。

**不稳定的外部依赖** → 重点Mock测试各种失败场景（超时、返回错误、返回异常数据）。这些场景最容易出问题。

**UI/前端** → 单元测试的意义有限。优先做端到端测试（Playwright/Cypress）。

**关键原则：**
1. 不要在测试覆盖率上追求100%——投入产出比会越来越低
2. 优先测试最容易出错的地方（复杂计算、边界条件、异常处理）
3. 测试应该让你有信心重构——如果你重构时不敢改某段代码，说明那段代码缺测试
4. 好的测试 = 下次改代码时，测试能告诉你"你改坏了什么"

## 本章小结

| 知识点 | 要点 |
|--------|------|
| 集成测试 | 验证多个组件协作是否正确 |
| 单元测试 vs 集成测试 | 前者测零件，后者测装配 |
| httptest.NewRequest | 不走网络，直接测试Handler——推荐 |
| httptest.NewServer | 走真实HTTP，测完整链路 |
| httptest.NewRecorder | 捕获Handler的响应 |
| SQLite内存模式 | 快速、干净的测试数据库 |
| Docker临时MySQL | 真实MySQL环境，与生产一致 |
| testcontainers | 代码中管理Docker容器生命周期 |
| 测试策略 | 优先测核心逻辑、边界条件、异常场景 |
| 测试价值 | 让你有信心重构代码 |
