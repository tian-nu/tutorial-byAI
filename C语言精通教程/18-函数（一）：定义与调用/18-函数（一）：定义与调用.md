## 我在做什么？

前面你学会了变量、条件判断、循环——你已经能写一段从头执行到尾的代码了。但如果你碰到这样的情况：同一个计算要在三个地方用，你打算把同一段代码复制粘贴三次吗？

这一章教你**把一段代码打包成一个"加工机器"**——给它原料（参数），它吐出产品（返回值）。这个机器就叫**函数**。读完这一章，你能自己定义函数、调用函数，还能写出递归函数（自己调用自己的函数）。

## 一、函数是什么？

### 1.1 加工机器的比喻

想象一个榨汁机：

```text
        苹果 ──→  ┌──────────┐  ──→ 苹果汁
        橙子 ──→  │  榨汁机   │  ──→ 橙汁
        西瓜 ──→  │ (函数)   │  ──→ 西瓜汁
                  └──────────┘
```

- 你**不需要知道**榨汁机内部齿轮怎么转——你只需要知道"放水果进去，出果汁"。
- 同一个榨汁机，放不同水果，出不同果汁——但榨汁机本身不变。

C语言函数就是这样一个"加工机器"：你给它输入（参数），它执行内部逻辑，然后给你输出（返回值）。你不需要每次重复写榨汁的代码，只需要**调用**这个函数。

### 1.2 你已经用过函数了

回头看你的第一个C程序：

```c
#include <stdio.h>

int main() {
    printf("Hello, World!\n");
    return 0;
}
```

- `main` 就是一个函数——程序的入口函数。
- `printf` 也是一个函数——别人（C标准库）已经帮你写好的"打印机器"。

从这一章开始，你不再只是**用**别人的函数，你要**造**自己的函数。

## 二、函数的定义四要素

定义一个函数，需要四样东西。缺一不可。

```text
      返回值类型     函数名       参数列表
         |            |             |
         v            v             v
       int           add      (int a, int b)
       ~~~           ~~~      ~~~~~~~~~~~~~
         |                          |
         +-- 函数体 { ... }  ───────+
```

### 2.1 要素一：返回值类型

函数干完活之后，交给你什么类型的结果。

- `int` —— 返回一个整数
- `double` —— 返回一个浮点数
- `char` —— 返回一个字符
- `void` —— 什么也不返回（"void"英文意思是"空"）

```c
int add(int a, int b) {     // 返回 int
    return a + b;
}

void say_hello() {           // 不返回任何东西
    printf("Hello!\n");
}
```

### 2.2 要素二：函数名

你给这个机器起的名字。命名规则和变量名一样：字母、数字、下划线，不能以数字开头。

好名字的标准：**看到名字就知道函数干什么**。

| 好名字 | 坏名字 | 为什么坏 |
|--------|--------|----------|
| `calculate_area` | `f` | 不知道f是什么 |
| `print_menu` | `func1` | 不知道func1干什么 |
| `is_prime` | `check` | check什么？ |

### 2.3 要素三：参数列表

放进榨汁机的水果。写在函数名后面的圆括号里。

```c
// 两个参数：a 和 b，类型都是 int
int add(int a, int b) { ... }

// 没有参数：用 void 表示（也可以留空，但写 void 更明确）
void say_hello(void) { ... }
```

### 2.4 要素四：函数体

花括号 `{ }` 里面的代码——榨汁机内部的齿轮。

```c
int add(int a, int b) {
    int result = a + b;   // 函数体：干活的代码
    return result;        // 把结果交出去
}
```

## 三、函数的声明 vs 定义

### 3.1 问题：编译器"从上往下读"

C编译器按顺序读你的代码。如果函数定义在调用之后，编译器会报错：

```c
#include <stdio.h>

int main() {
    int s = add(3, 5);      // 编译器：add是什么？我不认识！
    printf("%d\n", s);
    return 0;
}

int add(int a, int b) {     // 定义在 main 后面
    return a + b;
}
```

编译：
```text
$ gcc test.c
test.c: In function 'main':
test.c:4:13: warning: implicit declaration of function 'add'
```

### 3.2 解决方案：函数声明（也叫"函数原型"）

在调用之前，先告诉编译器"这个函数存在，长这样"——这就是**声明**（declaration）。

```c
#include <stdio.h>

// 函数声明（原型）—— 告诉编译器：有个叫 add 的函数，接收两个 int，返回 int
int add(int a, int b);

int main() {
    int s = add(3, 5);      // 编译器：哦，我知道 add 了
    printf("%d\n", s);
    return 0;
}

// 函数定义 —— 真正实现
int add(int a, int b) {
    return a + b;
}
```

**声明 vs 定义**：

| | 声明（Declaration） | 定义（Definition） |
|---|---|---|
| 做什么 | 告诉编译器"有这个函数" | 真正写出函数体 |
| 有函数体吗？ | 没有（以分号结尾） | 有（花括号） |
| 可以写几次？ | 多次 | 只能一次 |
| 语法 | `int add(int a, int b);` | `int add(int a, int b) { ... }` |

```text
声明：给你一张名片，告诉你"我叫 add，我接收两个 int，返回 int"
定义：把你叫到面前，让你看"我是怎么工作的"
```

## 四、`return` 语句

### 4.1 两个作用

`return` 同时做两件事：

1. **结束函数**：函数执行到 `return` 就立刻停止，后面的代码不会执行。
2. **返回结果**：把 `return` 后面的值交给调用者。

```c
int absolute(int x) {
    if (x < 0) {
        return -x;    // 如果 x 是负数，这里就返回了，不会执行下面的 return
    }
    return x;         // 只有 x >= 0 才执行到这里
}
```

### 4.2 void 函数中的 return

`void` 函数也可以使用 `return;`（不带值），作用是提前结束函数。

```c
void print_positive(int x) {
    if (x <= 0) {
        return;           // 不是正数，不打印，直接走人
    }
    printf("正数：%d\n", x);
}
```

## 五、递归函数

### 5.1 递归是什么？

*递归* *此术语见附录F* 就是函数自己调用自己。就像两面镜子对着照——镜子里有镜子，镜子里的镜子里还有镜子……

```text
factorial(4)
    │
    └── 4 * factorial(3)
                │
                └── 3 * factorial(2)
                            │
                            └── 2 * factorial(1)
                                        │
                                        └── 1   ← 基线条件，停止递归
```

### 5.2 递归的两个必备条件

**条件一：基线条件（Base Case）**——什么时候停下来。

没有基线条件，函数会无限调用自己，直到 *栈溢出* *此术语见附录F*（程序崩溃）。

**条件二：递归条件（Recursive Case）**——每次调用让问题变小。

### 5.3 示例：阶乘

阶乘定义：n! = n × (n-1) × (n-2) × ... × 1，且 0! = 1。

```c
#include <stdio.h>

// 递归实现阶乘
int factorial(int n) {
    // 基线条件：0! = 1，1! = 1
    if (n <= 1) {
        return 1;
    }
    // 递归条件：n! = n * (n-1)!
    return n * factorial(n - 1);
}

int main() {
    printf("5! = %d\n", factorial(5));   // 输出 120
    printf("0! = %d\n", factorial(0));   // 输出 1
    return 0;
}
```

**执行过程（n=5）**：

```text
factorial(5)
= 5 * factorial(4)
= 5 * (4 * factorial(3))
= 5 * (4 * (3 * factorial(2)))
= 5 * (4 * (3 * (2 * factorial(1))))
= 5 * (4 * (3 * (2 * 1)))          ← 碰到基线条件，开始返回
= 5 * (4 * (3 * 2))
= 5 * (4 * 6)
= 5 * 24
= 120
```

### 5.4 示例：斐波那契数列

斐波那契数列：1, 1, 2, 3, 5, 8, 13, 21, ...

规则：第一项 = 1，第二项 = 1，之后每一项 = 前两项之和。

```c
#include <stdio.h>

int fibonacci(int n) {
    // 基线条件
    if (n == 1 || n == 2) {
        return 1;
    }
    // 递归条件
    return fibonacci(n - 1) + fibonacci(n - 2);
}

int main() {
    printf("第 6 项 = %d\n", fibonacci(6));   // 输出 8
    printf("第 10 项 = %d\n", fibonacci(10)); // 输出 55
    return 0;
}
```

### 5.5 递归 vs 迭代

递归不是万能的。同一个问题，通常有两种写法：

```c
// 递归版本：简洁，但效率低（重复计算）
int factorial_recursive(int n) {
    if (n <= 1) return 1;
    return n * factorial_recursive(n - 1);
}

// 迭代版本：用循环，效率高
int factorial_iterative(int n) {
    int result = 1;
    for (int i = 1; i <= n; i++) {
        result *= i;
    }
    return result;
}
```

| 比较 | 递归 | 迭代 |
|------|------|------|
| 代码量 | 少，简洁 | 多，但直观 |
| 效率 | 有函数调用开销 | 通常更快 |
| 内存 | 每次调用占用栈空间 | 不额外占用 |
| 可读性 | 对数学问题友好 | 对线性问题友好 |
| 风险 | 太深会栈溢出 | 无此风险 |

**经验法则**：如果问题天然是递归定义的（如树遍历、分治算法），用递归；如果是线性重复的，用迭代。

## 六、常见错误与排错

### 6.1 忘记声明函数

❌ 错误示例：
```c
int main() {
    int r = multiply(3, 4);  // 编译器不认识 multiply
    return 0;
}

int multiply(int a, int b) {
    return a * b;
}
```

✅ 正确示例：
```c
int multiply(int a, int b);  // 先声明

int main() {
    int r = multiply(3, 4);
    return 0;
}

int multiply(int a, int b) {
    return a * b;
}
```

### 6.2 忘记 return 语句

❌ 错误示例：
```c
int add(int a, int b) {
    int result = a + b;
    // 忘记 return 了！
}
// 编译器警告：control reaches end of non-void function
```

✅ 正确示例：
```c
int add(int a, int b) {
    int result = a + b;
    return result;
}
```

### 6.3 递归缺少基线条件

❌ 错误示例：
```c
int endless(int n) {
    return endless(n - 1);  // 永远停不下来，栈溢出！
}
```

✅ 正确示例：
```c
int safe_recursive(int n) {
    if (n <= 0) return 0;        // 基线条件
    return n + safe_recursive(n - 1);
}
```

### 6.4 void 函数中 return 带值

❌ 错误示例：
```c
void print_number(int n) {
    if (n < 0) {
        return 0;   // 错误：void 函数不能返回 0
    }
    printf("%d\n", n);
}
```

✅ 正确示例：
```c
void print_number(int n) {
    if (n < 0) {
        return;     // 正确：不带值
    }
    printf("%d\n", n);
}
```

## 我做得对不对？

写完本章代码后，用以下程序验证：

```c
#include <stdio.h>

// 声明
int factorial(int n);
int fibonacci(int n);

int main() {
    // 测试阶乘
    printf("0! = %d (应为 1)\n", factorial(0));
    printf("5! = %d (应为 120)\n", factorial(5));

    // 测试斐波那契
    printf("fib(1) = %d (应为 1)\n", fibonacci(1));
    printf("fib(6) = %d (应为 8)\n", fibonacci(6));

    return 0;
}

int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

int fibonacci(int n) {
    if (n == 1 || n == 2) return 1;
    return fibonacci(n - 1) + fibonacci(n - 2);
}
```

编译运行：
```bash
$ gcc -Wall -o test test.c
$ ./test
0! = 1 (应为 1)
5! = 120 (应为 120)
fib(1) = 1 (应为 1)
fib(6) = 8 (应为 8)
```

全部输出与预期一致，说明你学会了。

## 不对怎么办？

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| `implicit declaration of function` | 函数定义在调用之后，且没有声明 | 在调用前加函数声明 |
| `control reaches end of non-void function` | 非 void 函数缺少 return | 检查每个分支是否都有 return |
| 程序跑着跑着崩溃了 | 递归没有基线条件，导致无限递归 | 检查递归函数是否有正确的停止条件 |
| `void function should not return a value` | void 函数中 return 带了值 | 改为 `return;` 或不写 return |
| 结果和预期不一样 | 递归逻辑写错了 | 用小数值（如 n=1, n=2）手算跟踪 |

## 本章小结

| 知识点 | 一句话总结 |
|--------|-----------|
| 函数是什么 | 一段可重复使用的代码，接收输入，返回输出 |
| 定义四要素 | 返回值类型、函数名、参数列表、函数体 |
| 声明 vs 定义 | 声明是"名片"（告诉编译器有这个函数），定义是"实现"（写函数体） |
| return | 结束函数并返回结果 |
| 递归 | 函数自己调用自己 |
| 递归两要素 | 基线条件（停止条件）+ 递归条件（问题缩小） |
| 递归 vs 迭代 | 递归简洁但有栈溢出风险，迭代安全但代码可能更啰嗦 |
