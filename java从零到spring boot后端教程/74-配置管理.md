## 74.1 从 properties 到 yml——告别重复前缀

Spring Boot 默认使用 `application.properties`，但强烈建议改用 `application.yml`。为什么？看对比：

```properties
# application.properties —— 重复的前缀让人抓狂
server.port=8080
server.servlet.context-path=/api
spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=root
spring.datasource.password=your_password_here
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
```

```yaml
# application.yml —— 层级缩进，一目了然
server:
  port: 8080
  servlet:
    context-path: /api

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mydb
    username: root
    password: your_password_here
    driver-class-name: com.mysql.cj.jdbc.Driver
```

### yml 语法规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 缩进 | 用空格（不能用 Tab），通常 2 或 4 空格 | `  port: 8080` |
| 键值对 | `key: value`（冒号后必须空格） | `port: 8080` |
| 列表 | 用 `- ` 开头 | `- value1` |
| 字符串 | 默认不需要引号，特殊字符用 `""` 或 `''` | `name: "Hello World"` |

将一个 `application.properties` 中的配置迁移到 yml，只需将 `application.properties` 重命名为 `application.yml` 并按层级重写即可。两者可以并存，但 yml 优先级更高。

## 74.2 @Value——注入单个属性

最简单的配置注入方式，适合注入一两个属性：

```java
package com.example.demo.config;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

@Component
public class AppInfo {

    @Value("${server.port}")
    private int port;

    @Value("${app.name:Demo App}")
    private String appName;  // 冒号后是默认值：如果配置文件中没有 app.name，就用 "Demo App"

    public void printInfo() {
        System.out.println("应用名：" + appName + "，端口：" + port);
    }
}
```

### @Value 的限制

```java
// ❌ 一次只能注入一个属性，如果要注入 10 个相关属性，需要写 10 个 @Value
@Value("${spring.datasource.url}")
private String url;

@Value("${spring.datasource.username}")
private String username;

@Value("${spring.datasource.password}")
private String password;

// ❌ 不支持复杂类型（List、Map、嵌套对象）
```

> 🤔 想多一点：`@Value` 适合"散装"属性——彼此无关、零星使用。但对于成组的配置（比如 datasource 的 4 个属性），用 `@Value` 既啰嗦又容易漏。这时候就该 `@ConfigurationProperties` 出场了。

## 74.3 @ConfigurationProperties——结构化批量绑定

将一组相关属性映射到一个 Java 类，支持嵌套、List、Map：

```java
package com.example.demo.config;

import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.stereotype.Component;
import java.util.List;
import java.util.Map;

@Component
@ConfigurationProperties(prefix = "app")
public class AppConfig {

    private String name;
    private String version;
    private List<String> authors;
    private Map<String, Integer> limits;
    private Security security = new Security();

    // getter / setter 必须提供
    // （可使用 Lombok @Data 替代）

    public static class Security {
        private String jwtSecret;
        private long jwtExpirationMs;

        // getter / setter
    }

    // ... 所有字段的 getter 和 setter（此处省略以节省篇幅，实际代码中必须提供）
}
```

对应的 `application.yml`：

```yaml
app:
  name: "订单管理系统"
  version: "1.2.0"
  authors:
    - 张三
    - 李四
  limits:
    page-size: 20
    max-upload-mb: 10
  security:
    jwt-secret: your_jwt_secret_here
    jwt-expiration-ms: 86400000
```

### 使用 Lombok 简化

```java
package com.example.demo.config;

import lombok.Data;
import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.stereotype.Component;
import java.util.List;
import java.util.Map;

@Data
@Component
@ConfigurationProperties(prefix = "app")
public class AppConfig {

    private String name;
    private String version;
    private List<String> authors;
    private Map<String, Integer> limits;
    private Security security = new Security();

    @Data
    public static class Security {
        private String jwtSecret;
        private long jwtExpirationMs;
    }
}
```

> ⚠️ 使用 `@ConfigurationProperties` 需要添加依赖：`spring-boot-configuration-processor`（提供 IDE 自动补全提示）。在 pom.xml 中添加：
> ```xml
> <dependency>
>     <groupId>org.springframework.boot</groupId>
>     <artifactId>spring-boot-configuration-processor</artifactId>
>     <optional>true</optional>
> </dependency>
> ```

### @Value vs @ConfigurationProperties 对比

| | @Value | @ConfigurationProperties |
|------|------|------|
| 适合 | 注入少量散装属性 | 注入一组相关属性 |
| 嵌套对象 | ❌ 不支持 | ✅ 支持 |
| List/Map | ❌ 不支持（需 SpEL） | ✅ 原生支持 |
| 松散绑定 | ❌ | ✅（jwt-secret → jwtSecret） |
| JSR-303 校验 | ❌ | ✅（可加 @Validated） |

## 74.4 多环境 Profile——一套代码，多套配置

开发环境连本地库、测试环境连测试库、生产环境连生产库。不能每次切换环境都改代码。

Spring Boot 的 Profile 机制让你为每个环境写一套配置，启动时指定用哪套。

### 方式一：多文件命名

```
application.yml           # 公共配置（所有环境共享）
application-dev.yml       # 开发环境
application-test.yml      # 测试环境
application-prod.yml      # 生产环境
```

`application.yml` 中指定激活哪个：

```yaml
# application.yml——公共配置
spring:
  profiles:
    active: dev            # 激活 dev profile
```

`application-dev.yml`：

```yaml
# application-dev.yml——开发环境专属
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mydb_dev
    username: dev_user
    password: dev_password_here
```

`application-prod.yml`：

```yaml
# application-prod.yml——生产环境专属
server:
  port: 80

spring:
  datasource:
    url: jdbc:mysql://prod-db-server:3306/mydb
    username: prod_user
    password: ${DB_PASSWORD}  # 从环境变量读取，不写死在文件中
```

### 方式二：单文件多段（yml 特有）

```yaml
# application.yml
server:
  port: 8080

---
spring:
  config:
    activate:
      on-profile: dev

server:
  port: 8081
---
spring:
  config:
    activate:
      on-profile: prod

server:
  port: 80
```

`---` 分隔不同 profile 的配置块。

### 指定 Profile 的方式

| 方式 | 命令/配置 |
|------|----------|
| 配置文件 | `spring.profiles.active=prod` |
| 命令行参数 | `java -jar app.jar --spring.profiles.active=prod` |
| 环境变量 | `export SPRING_PROFILES_ACTIVE=prod` |
| IDEA | Run Configuration → Environment variables → `spring.profiles.active=dev` |

> ⚠️ 安全提醒：生产环境的数据库密码**绝对不要**写在 `application-prod.yml` 中。使用环境变量 `${DB_PASSWORD}` 引用，然后在服务器上设置该环境变量。

## 74.5 配置优先级

当同一个属性出现在多个位置时，Spring Boot 按以下优先级（从高到低）决定最终值：

1. 命令行参数（`--server.port=9090`）
2. 操作系统环境变量（`SERVER_PORT=9090`）
3. `application-{profile}.yml`（Profile 特定配置）
4. `application.yml`（公共配置）
5. `@PropertySource` 指定的自定义配置文件
6. 代码中的默认值（`@Value("${xxx:默认值}")`）

> 优先级规则的意义：命令行最高——运维可以在不改变打包文件的情况下修改任何配置。环境变量次之——适合容器化部署（K8s ConfigMap / Secret）。

---

## 74.6 小结

| 知识点 | 核心内容 |
|--------|----------|
| application.yml | 用缩进层级替代重复前缀，比 properties 更清晰 |
| @Value | 注入单个属性，支持默认值 `:默认值` |
| @ConfigurationProperties | 批量绑定结构化配置，支持嵌套、List、Map、校验 |
| Profile 多环境 | 通过 `spring.profiles.active` 切换 dev/test/prod |
| 配置优先级 | 命令行 > 环境变量 > profile 配置 > 公共配置 > 默认值 |

A. `port: 8080`

C. `name: "Hello"`  
D. `items:`（后换行）`  - item1`

**答案提示**：1→B（冒号后缺少空格）。2→用 `@ConfigurationProperties`，因为嵌套结构和多个相关字段用 `@Value` 过于繁琐。3→使用环境变量 `${DB_PASSWORD}` 引用，密码不写死在 yml 文件中；或在 K8s 中使用 Secret；或使用外部配置中心的加密存储。下一章——进入 Spring MVC，写你的第一个 REST 控制器。
