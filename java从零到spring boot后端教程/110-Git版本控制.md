## 110.1 Git 是什么

Git 是一个**分布式版本控制系统**。分布式意味着每个人的电脑上都有完整的仓库副本，不依赖中央服务器也能工作。

### 核心概念三句话

| 概念 | 一句话解释 |
|------|-----------|
| 仓库（Repository） | 一个被 Git 管理的项目文件夹 |
| 提交（Commit） | 一次"快照"——你把当前所有文件的改动打包成一个记录点 |
| 分支（Branch） | 一条独立的开发线，你可以切换过去做实验，不影响主线 |

### 安装验证

```bash
git --version
# 应输出 git version 2.xx.x
```

### 首次配置

```bash
git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"
```

这两个值会出现在你每一次 commit 中，用于追溯谁做了什么。

## 110.2 日常五步工作流

这是你每天都会重复的操作序列：

```
1. git pull       拉取最新代码
2. (写代码...)
3. git add .      暂存所有改动
4. git commit -m "描述你做了什么"    创建一次提交
5. git push       推送到远程仓库
```

### 实际操作

```bash
# 1. 克隆仓库（只需要一次）
git clone https://github.com/yourname/yourproject.git
cd yourproject

# 2. 写代码...创建了一个 UserController.java

# 3. 查看状态（可选但推荐）
git status
# 你会看到：Untracked files: UserController.java

# 4. 暂存
git add UserController.java
# 或者暂存所有改动：git add .

# 5. 提交
git commit -m "feat: add UserController with CRUD endpoints"

# 6. 推送
git push
```

### 验证方法：每做完一步检查

```bash
git status        # 看工作区是否干净
git log --oneline # 看提交历史，确认你的 commit 在上面
```

## 110.3 commit message 规范

好的提交信息是给未来的你和同事看的"线索"。

```bash
# ❌ 坏的信息
git commit -m "修改"
git commit -m "fix bug"
git commit -m "update"

# ✅ 好的信息
git commit -m "feat: add user login with JWT authentication"
git commit -m "fix: resolve NPE when email field is null"
git commit -m "docs: update API documentation for /users endpoint"
```

### 推荐前缀

| 前缀 | 含义 |
|------|------|
| `feat` | 新功能 |
| `fix` | 修复 bug |
| `docs` | 文档变更 |
| `refactor` | 重构（不改变功能） |
| `test` | 测试相关 |
| `chore` | 构建/工具配置 |

## 110.4 分支——并行的开发线

### 你在做什么？
你正在开发一个新功能"用户积分系统"，但主分支上可能还有紧急 bug 要修。你不能让半成品代码混入主分支。

### 操作流程

```bash
# 创建并切换到新分支
git checkout -b feature/points-system

# 写代码...提交...
git add .
git commit -m "feat: add points calculation logic"

# 切换到主分支
git checkout main

# 拉最新代码
git pull

# 合并你的功能分支
git merge feature/points-system

# 推送到远程
git push

# 删除已经合并的分支（可选）
git branch -d feature/points-system
```

### 日常分支操作速查

```bash
git branch                    # 列出本地分支
git branch -a                 # 列出所有分支（含远程）
git checkout -b new-branch    # 创建并切换
git checkout main             # 切换到已有分支
git branch -d old-branch      # 删除本地分支
```

### 验证方法

```bash
git branch          # 确认你在正确的分支上（当前分支前面有 *）
git log --oneline --graph --all   # 可视化查看分支历史
```

## 110.5 合并冲突——当两个人改了同一个文件

冲突不是 bug，是 Git 无法自动决定保留谁的改动时发出的"求救信号"。

### 冲突长什么样

```
<<<<<<< HEAD
System.out.println("Hello from main branch");
=======
System.out.println("Hello from feature branch");
>>>>>>> feature/points-system
```

### 解决步骤

1. 打开冲突文件，找到 `<<<<<<<` 和 `>>>>>>>` 标记
2. 决定保留哪段代码（或合并两段）
3. 删除标记符号
4. 保存文件
5. 执行：
   ```bash
   git add .
   git commit -m "merge: resolve conflict in UserService"
   git push
   ```

### 最可能出错的地方

❌ **没有删除标记符号就提交了**。`<<<<<<<` 和 `>>>>>>>` 会破坏你的代码语法。

✅ 解决完冲突后，用 `grep` 搜索一下：
```bash
grep -r "<<<<<<<" .
```
应该没有输出。

## 110.6 撤销操作——紧急逃生

```bash
# 改坏了文件，想回到上次 commit 的状态
git checkout -- filename.java

# git add 了但还没 commit，想撤回暂存
git reset HEAD filename.java

# commit 了但还没 push，想修改上一次的 commit 信息
git commit --amend -m "new message"

# commit 了但还没 push，想撤销整个 commit（保留代码改动）
git reset --soft HEAD~1

# commit 了但还没 push，想彻底丢弃改动
git reset --hard HEAD~1
```

> ⚠️ `git reset --hard` 不可逆，执行前确认 `git status` 没有未备份的重要改动。

## 110.7 .gitignore——忽略不该提交的文件

在项目根目录创建 `.gitignore`：

```gitignore
# IDE
.idea/
*.iml
.vscode/

# Java
target/
*.class
*.jar
*.war

# 日志
*.log
logs/

# 环境敏感文件
.env
application-local.yml

# OS
.DS_Store
Thumbs.db
```

> ⚠️ `.gitignore` 只对**未跟踪**的文件生效。如果某个文件已经被 Git 跟踪了，先执行 `git rm --cached filename` 停止跟踪。

## 110.8 远程仓库与协作

### 添加远程仓库

```bash
git remote add origin https://github.com/yourname/yourproject.git
git remote -v     # 查看远程仓库地址
```

### 拉取 vs 获取

```bash
git pull          # fetch + merge，一步到位
git fetch         # 只下载远程改动，不合并（更安全，可以先用 git diff 检查）
```

### GitHub/GitLab 协作流

```
main 分支：     始终可部署
dev 分支：      日常开发合并到这里
feature/xxx：   每个功能一个分支
hotfix/xxx：    紧急修复分支
```

## 小结

| 操作 | 命令 |
|------|------|
| 克隆仓库 | `git clone <url>` |
| 查看状态 | `git status` |
| 暂存 | `git add .` |
| 提交 | `git commit -m "message"` |
| 推送 | `git push` |
| 拉取 | `git pull` |
| 创建分支 | `git checkout -b <name>` |
| 切换分支 | `git checkout <name>` |
| 合并分支 | `git merge <name>` |
| 撤销暂存 | `git reset HEAD <file>` |
| 丢弃改动 | `git checkout -- <file>` |

## 自测题

1. 你 `git add` 了一个文件，但还没 `commit`。你想把它从暂存区撤出来，用什么命令？
2. 合并分支时出现冲突，你解决了冲突后，还需要执行哪些命令？
3. `.gitignore` 里加了 `*.log`，但 `app.log` 仍然被 Git 跟踪。为什么？怎么修复？

<details>
<summary>点击查看答案</summary>

1. `git reset HEAD <filename>`
2. `git add .` 然后 `git commit -m "merge: resolve conflict"`，最后 `git push`
3. 因为 `app.log` 在添加到 `.gitignore` 之前已经被 Git 跟踪了。修复：`git rm --cached app.log` 停止跟踪，然后再提交。
</details>
