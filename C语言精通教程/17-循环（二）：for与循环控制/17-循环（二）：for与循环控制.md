## 我在做什么？

上一章你学会了 `while` 和 `do-while`，能写出"重复做一件事"的代码。但 `while` 有一个不便之处：初始化、条件、更新这三要素分散在三个地方，容易漏掉一个。

`for` 循环把三要素集中在一行里，让代码更紧凑、更不容易出错。此外，你还将学会两个控制循环走向的关键字：`break`（立刻跳出循环）和 `continue`（跳过本次循环的剩余部分，开始下一次）。

## 17.1 `for` 循环：三要素合为一体

### 17.1.1 基本语法

```c
for (初始化; 条件; 更新) {
    /* 循环体 */
}
```

**执行流程**：

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  初始化（只执行一次）                                  │
│      │                                              │
│      ▼                                              │
│  ┌───────┐                                          │
│  │ 条件？ │──── 假 ────→ 退出循环                     │
│  └───┬───┘                                          │
│      │ 真                                           │
│      ▼                                              │
│  执行循环体                                          │
│      │                                              │
│      ▼                                              │
│  执行更新                                            │
│      │                                              │
│      └──────→ 回到条件判断                            │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**比喻**：`for` 循环就像操场上的跑步训练。
- 初始化：教练说"站在起跑线上"（`i = 1`）
- 条件：教练说"跑5圈，没到5圈继续跑"（`i <= 5`）
- 循环体：每跑一圈，你自己喊一声"第i圈跑完！"
- 更新：教练在计数器上画一笔（`i++`）
- 退出：跑完5圈，教练说"停！"

### 17.1.2 `for` vs `while`：同一件事的两种写法

```c
/* while 写法：三要素分散 */
int i = 1;          /* 初始化 */
while (i <= 5) {    /* 条件 */
    printf("%d\n", i);
    i++;            /* 更新 */
}

/* for 写法：三要素集中 */
for (int i = 1; i <= 5; i++) {
    printf("%d\n", i);
}
```

`for` 循环的优势一目了然：
- 所有循环控制信息都在第一行，不会漏掉。
- 循环变量 `i` 的作用域限定在 `for` 循环内部（C99起）。
- 代码更紧凑，可读性更高。

### 17.1.3 累加：1到100的和（for版）

```c
#include <stdio.h>

int main(void)
{
    int sum = 0;

    for (int i = 1; i <= 100; i++) {
        sum = sum + i;    /* 或写成 sum += i; */
    }

    printf("1到100的和是：%d\n", sum);   /* 5050 */

    return 0;
}
```

### 17.1.4 `for` 循环的变体

`for` 循环的三个部分都可以省略（但分号不能省）：

```c
/* 变体1：省略初始化（初始化在外面做） */
int i = 1;
for (; i <= 10; i++) {
    printf("%d\n", i);
}

/* 变体2：省略更新（更新在循环体内做） */
for (int i = 1; i <= 10; ) {
    printf("%d\n", i);
    i++;
}

/* 变体3：省略条件 → 死循环 */
for (;;) {   /* 等价于 while(1) */
    printf("永远跑下去...\n");
    /* 需要 break 才能跳出 */
}

/* 变体4：多个变量 */
for (int i = 1, j = 10; i <= 10; i++, j--) {
    printf("i=%d, j=%d\n", i, j);
}
```

### 17.1.5 递减循环

```c
/* 从10倒数到1 */
for (int i = 10; i >= 1; i--) {
    printf("%d\n", i);
}
printf("发射！\n");
```

## 17.2 `break`：立刻跳出循环

`break` 语句让程序立刻跳出当前循环，不再执行循环体的剩余部分，也不再判断条件。

```c
#include <stdio.h>

int main(void)
{
    for (int i = 1; i <= 10; i++) {
        if (i == 5) {
            break;   /* 到了5就跳出，不再继续 */
        }
        printf("%d\n", i);
    }
    printf("循环结束\n");

    return 0;
}
```

输出：
```
1
2
3
4
循环结束
```

**`break` 的执行流程**：

```
for (i=1; i<=10; i++)
    │
    ▼
  输出1, i=2
    │
    ▼
  输出2, i=3
    │
    ▼
  ...
    │
    ▼
  i=5, 遇到 break!
    │
    ▼
  直接跳出循环 ──→ 执行循环后面的代码
```

**比喻**：`break` 就像火灾报警器。你在10层楼里逐层巡查（`for` 循环），查到第5层时，火警响了（`break`）。你立刻停止巡查，冲出大楼。后面的第6到10层，不管了。

### `break` 只跳出最内层循环

```c
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        if (i == 2 && j == 2) {
            break;   /* 只跳出内层 j 循环，外层 i 循环继续 */
        }
        printf("(%d,%d) ", i, j);
    }
    printf("\n");
}
```

输出：
```
(1,1) (1,2) (1,3)
(2,1)
(3,1) (3,2) (3,3)
```

## 17.3 `continue`：跳过本次，继续下一次

`continue` 语句跳过本次循环的剩余部分，直接进入下一次循环的条件判断。

```c
#include <stdio.h>

int main(void)
{
    for (int i = 1; i <= 10; i++) {
        if (i % 2 == 0) {   /* 如果是偶数 */
            continue;        /* 跳过，不打印 */
        }
        printf("%d\n", i);   /* 只打印奇数 */
    }

    return 0;
}
```

输出：
```
1
3
5
7
9
```

**`continue` 的执行流程**：

```
for (i=1; i<=10; i++)
    │
    ▼
  i=1, 奇数 → 打印 1 → i++
    │
    ▼
  i=2, 偶数 → continue → 跳过打印 → i++
    │
    ▼
  i=3, 奇数 → 打印 3 → i++
    │
    ▼
  ...
```

**比喻**：`continue` 就像你在翻一本相册。遇到不喜欢的照片（偶数），你直接翻过去（continue），不细看。遇到喜欢的照片（奇数），你停下来欣赏（执行循环体）。但你不会因为一张不喜欢的照片就把整本相册合上（`break` 才这样）。

### `break` vs `continue` 对比

```c
/* break：遇到5就停止，后面的都不打印 */
for (int i = 1; i <= 10; i++) {
    if (i == 5) break;
    printf("%d ", i);
}
/* 输出：1 2 3 4 */

/* continue：遇到5就跳过，后面的继续打印 */
for (int i = 1; i <= 10; i++) {
    if (i == 5) continue;
    printf("%d ", i);
}
/* 输出：1 2 3 4 6 7 8 9 10 */
```

## 17.4 循环嵌套

循环可以嵌套：一个循环里面再套一个循环。类比生活中的"钟表"：秒针转一圈（内层循环），分针走一格（外层循环）。

### 17.4.1 嵌套循环示例：打印矩形

```c
#include <stdio.h>

int main(void)
{
    int rows = 4;
    int cols = 6;

    for (int i = 1; i <= rows; i++) {      /* 外层：控制行 */
        for (int j = 1; j <= cols; j++) {   /* 内层：控制列 */
            printf("* ");
        }
        printf("\n");   /* 每行结束后换行 */
    }

    return 0;
}
```

输出：
```
* * * * * *
* * * * * *
* * * * * *
* * * * * *
```

### 17.4.2 嵌套循环示例：九九乘法表

```c
#include <stdio.h>

int main(void)
{
    for (int i = 1; i <= 9; i++) {
        for (int j = 1; j <= i; j++) {
            printf("%d×%d=%-2d ", j, i, i * j);
        }
        printf("\n");
    }

    return 0;
}
```

输出：
```
1×1=1
1×2=2  2×2=4
1×3=3  2×3=6  3×3=9
1×4=4  2×4=8  3×4=12 4×4=16
...
```

### 17.4.3 嵌套循环的执行顺序

```
外层 i=1：
  内层 j=1,2,3,...,9  → 内层跑完
  输出换行
外层 i=2：
  内层 j=1,2,3,...,9  → 内层又跑完
  输出换行
...
外层 i=9：
  内层 j=1,2,3,...,9  → 内层再跑完
  输出换行
```

**比喻**：嵌套循环就像时钟。时针（外层）走一格，分针（内层）转一整圈。时针走完12格，分针已经转了12圈。

## 17.5 时间复杂度初探

循环的次数直接影响程序的运行速度。当你写嵌套循环时，要注意"这个程序要跑多少次"。

```c
/* 单层循环：跑 N 次 */
for (int i = 0; i < N; i++) { ... }
/* 时间复杂度：O(N) */

/* 双层循环：跑 N × N 次 */
for (int i = 0; i < N; i++) {
    for (int j = 0; j < N; j++) { ... }
}
/* 时间复杂度：O(N^2) */
```

当 N = 1000 时：
- O(N) = 1000 次（很快）
- O(N^2) = 1,000,000 次（开始慢了）
- O(N^3) = 1,000,000,000 次（等很久）

**这不是本章重点**，但你需要有这个意识：嵌套循环的次数是乘积关系，层数越多，增长速度越快。

## 17.6 完整示例：打印素数

```c
#include <stdio.h>
#include <stdbool.h>

int main(void)
{
    int limit;

    printf("请输入上限：");
    scanf("%d", &limit);

    printf("2到%d之间的素数有：\n", limit);

    int count = 0;

    for (int num = 2; num <= limit; num++) {
        bool is_prime = true;

        /* 检查 num 是否是素数 */
        for (int div = 2; div * div <= num; div++) {
            if (num % div == 0) {
                is_prime = false;
                break;   /* 找到一个因子就够了，跳出内层循环 */
            }
        }

        if (is_prime) {
            printf("%d ", num);
            count++;

            /* 每输出10个素数换一行 */
            if (count % 10 == 0) {
                printf("\n");
            }
        }
    }

    printf("\n共找到 %d 个素数。\n", count);

    return 0;
}
```

## 17.7 常见错误

### 错误1：`for` 循环条件用 `=` 而不是 `<=` 或 `<`

```c
/* ❌ 错误示例 */
for (int i = 1; i = 10; i++) {   /* 条件是赋值，永远为真，死循环！ */
    printf("%d\n", i);
}

/* ✅ 正确示例 */
for (int i = 1; i <= 10; i++) {
    printf("%d\n", i);
}
```

### 错误2：`for` 循环头后加分号

```c
/* ❌ 错误示例 */
for (int i = 1; i <= 10; i++); {   /* 分号让循环体为空 */
    printf("%d\n", i);              /* 这行只执行一次 */
}

/* ✅ 正确示例 */
for (int i = 1; i <= 10; i++) {    /* 没有分号 */
    printf("%d\n", i);
}
```

### 错误3：`continue` 在 `while` 中跳过更新

```c
/* ❌ 错误示例 */
int i = 1;
while (i <= 10) {
    if (i % 2 == 0) {
        continue;   /* 跳过了 i++，死循环！ */
    }
    printf("%d\n", i);
    i++;
}

/* ✅ 正确示例：把更新放在 continue 之前 */
int i = 0;
while (i < 10) {
    i++;
    if (i % 2 == 0) {
        continue;
    }
    printf("%d\n", i);
}

/* ✅ 更好的写法：用 for 循环 */
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) {
        continue;   /* for 的 i++ 在 continue 之前执行，安全 */
    }
    printf("%d\n", i);
}
```

### 错误4：`for` 循环中分号用逗号代替

```c
/* ❌ 错误示例 */
for (int i = 1, i <= 10, i++) {   /* 分号不能写成逗号！编译报错 */
    ...
}

/* ✅ 正确示例 */
for (int i = 1; i <= 10; i++) {
    ...
}
```

## 17.8 循环选择指南

| 场景 | 推荐循环 | 原因 |
|------|----------|------|
| 已知循环次数 | `for` | 三要素集中，清晰 |
| 未知循环次数（如输入验证） | `while` | 条件驱动 |
| 至少执行一次 | `do-while` | 语义匹配 |
| 死循环 + 内部跳出 | `while(1)` 或 `for(;;)` | 风格选择 |

## 17.9 我在做什么？做得对不对？不对怎么办？

### 我在做什么？

我在学习 `for` 循环（三要素集中在一行的紧凑循环），以及 `break`（跳出循环）和 `continue`（跳过本次）两个循环控制语句。我还理解了循环嵌套的执行顺序和简单的时间复杂度概念。

### 我做得对不对？

- 你能把 `while` 循环改写为等价的 `for` 循环 → for循环过关。
- 你能解释 `break` 和 `continue` 的区别 → 循环控制过关。
- 你能写出一个嵌套循环打印乘法表 → 嵌套循环过关。
- 你知道 `continue` 在 `while` 中可能跳过更新导致死循环 → 安全意识过关。

### 不对怎么办？

| 症状 | 第一步检查 |
|------|-----------|
| `for` 死循环 | 条件部分是否用了 `=`？分号是否写成了逗号？ |
| `for` 循环体只执行一次 | `for(...)` 后是否误加了分号？ |
| `while` + `continue` 死循环 | 更新语句是否在 `continue` 之后？改用 `for` |
| 嵌套循环输出不对 | 内层和外层的循环变量是否写反了？ |
| `break` 没有跳出外层循环 | `break` 只跳出最内层，需要跳出外层需用 `goto` 或标志变量 |

## 本章小结

| 知识点 | 说明 |
|--------|------|
| `for` 循环 | `for(初始化; 条件; 更新)`，三要素集中 |
| `for` vs `while` | `for` 适合已知次数，`while` 适合条件驱动 |
| `break` | 立刻跳出当前循环，不再执行循环体剩余部分 |
| `continue` | 跳过本次循环剩余部分，进入下一次迭代 |
| 嵌套循环 | 外层每走一步，内层走完一整圈 |
| 时间复杂度 | 嵌套循环次数 = 各层次数的乘积 |
| `for` 头后分号 | 误加分号导致空循环体 |
| `continue` 陷阱 | 在 `while` 中可能跳过更新语句，导致死循环 |
