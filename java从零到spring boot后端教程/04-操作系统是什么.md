# 第04章 · 操作系统是什么

> "你在键盘上敲了 `java Hello` 然后回车——谁把这几个字母显示在屏幕上？谁找到 `Hello.class` 文件？谁分配了内存给JVM？这些都是操作系统干的。"

## 04.1 操作系统：计算机的"大管家"

操作系统（OS，Operating System）是硬件和应用软件之间的中间层。你的Java程序不是直接跟CPU、内存、硬盘对话的——而是通过操作系统。

```
┌─────────────────────────────┐
│       你的Java程序            │  ← 应用层
├─────────────────────────────┤
│       JVM（Java虚拟机）       │
├─────────────────────────────┤
│   操作系统（Windows/Linux）   │  ← 系统层
├─────────────────────────────┤
│  CPU / 内存 / 硬盘 / 网卡     │  ← 硬件层
└─────────────────────────────┘
```

关键职责：
- **进程管理**：每个Java程序运行在一个进程中，OS负责创建、调度、终止进程
- **内存管理**：OS给每个进程分配独立的内存空间，进程A不能偷看进程B的内存
- **文件系统**：你看到的C盘、D盘、文件夹，都是OS抽象出来的，底层是硬盘上的0和1
- **网络管理**：你的Java程序发HTTP请求，OS负责把数据打包成TCP包通过网卡发出去
- **设备驱动**：鼠标、键盘、打印机……OS统一管理

## 04.2 进程与线程

这是后端开发中最核心的OS概念之一。

### 进程

当你执行 `java -jar myapp.jar`，操作系统创建一个**进程**。这个进程就是你的Spring Boot应用在OS层面的"身份"。

- 每个进程有独立的PID（进程ID）
- 每个进程有独立的内存空间（进程A不能直接访问进程B的内存）
- 进程之间通信需要特殊机制（如Socket、管道）

```bash
# Linux下查看Java进程
ps aux | grep java
# 输出：root  12345  1.2  35.6  ...  java -jar myapp.jar
#             ↑ PID
```

### 线程

一个进程内部可以有多个**线程**。线程是进程中的"执行流"。一个Java程序启动后，至少有一个 `main` 线程。

```java
public class ThreadDemo {
    public static void main(String[] args) {
        // main线程：JVM启动时自动创建
        
        // 打印当前线程名
        System.out.println(Thread.currentThread().getName());  // 输出: main
        
        // 创建新线程
        Thread t = new Thread(() -> {
            System.out.println("我在新线程: " + Thread.currentThread().getName());
        });
        t.start();
    }
}
```

| | 进程 | 线程 |
|---|------|------|
| 资源 | 独立内存空间 | 共享进程的内存空间 |
| 创建开销 | 大（需要分配内存等） | 小 |
| 通信 | 复杂（IPC机制） | 简单（共享变量即可） |
| 崩溃影响 | 不影响其他进程 | 可能导致整个进程崩溃 |

## 04.3 JVM的进程模型

当你启动一个Spring Boot应用：

```bash
java -jar myapp.jar
```

操作系统创建一个**进程**。JVM在这个进程内部启动多个线程：

```
操作系统进程（myapp.jar, PID=12345）
├── main线程          ← 执行Spring Boot的启动逻辑
├── GC线程            ← 垃圾回收，后台默默运行
├── JIT编译线程       ← 热点代码编译
├── Tomcat线程-01     ← 处理HTTP请求
├── Tomcat线程-02     ← 处理HTTP请求
├── Tomcat线程-03     ← 处理HTTP请求
├── ...               ← 默认200个Tomcat线程
└── 其他守护线程      ← 各种后台任务
```

> 🤔 想多一点：你做 `jstack 12345`（Java自带的线程堆栈工具），能看到这个进程里所有线程在干什么。这个命令在排查"线程卡死"问题时非常有用——你能看到哪个线程卡在等锁，哪个线程在等数据库返回。

## 04.4 操作系统与Java部署的关系

最终你的Spring Boot应用会部署在Linux服务器上。你需要了解：

```bash
# 查看CPU核心数（决定你能同时跑多少线程）
nproc

# 查看内存总量（决定 -Xmx 设多大）
free -h

# 查看磁盘剩余空间（日志别把硬盘写满了）
df -h

# 查看某个进程的状态
top -p 12345
```

Windows用于开发，Linux用于部署——这是Java后端的常态。第05章我们会开始熟悉命令行，第108章会深入Linux运维。
