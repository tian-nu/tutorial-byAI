# 第23章 · 面向对象（二）继承

> "学生和老师都是'人'——都有姓名、年龄、身份证号，都需要吃饭睡觉。如果你为Student类和Teacher类各写一遍这些相同的代码，你就是一个低效的程序员。Java的解决方案叫继承：把共同的代码提到父类里，子类自动拥有父类的一切。但同时有一个铁律——Java只允许单继承，一个类只能有一个爹。"

---

## 23.1 继承的概念：is-a关系

**继承**（Inheritance。此术语需进附录）描述的是"是一个（is-a）"的关系：
- 学生是（is-a）一个人 → Student继承Person
- 猫是（is-a）一个动物 → Cat继承Animal
- 正方形是（is-a）一个形状 → Square继承Shape

```java
// 父类（Parent Class），也叫基类（Base Class）、超类（Superclass）
public class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() { return name; }
    public int getAge() { return age; }

    public void eat() {
        System.out.println(name + "正在吃饭");
    }

    public void sleep() {
        System.out.println(name + "正在睡觉");
    }
}

// 子类（Child Class），也叫派生类（Derived Class）、子类（Subclass）
public class Student extends Person {  // extends = "继承自"
    private String studentId;  // Student特有的字段
    private double score;

    public Student(String name, int age, String studentId, double score) {
        super(name, age);  // 调用父类构造器 ← 必须第一行
        this.studentId = studentId;
        this.score = score;
    }

    public void study() {
        System.out.println(getName() + "正在学习");
    }

    public String getStudentId() { return studentId; }
    public double getScore() { return score; }
}
```

### 比喻：父子继承

就像孩子继承父母的DNA：孩子自然拥有父母的外貌特征（name和age），但也有自己独有的特征（studentId和score）。Java里用关键字 `extends` 完成这个"继承"。

### 继承了什么？没继承什么？

| 继承 | 不继承 |
|------|--------|
| public的字段和方法 ✅ | private字段（存在但不直接访问）❌ |
| protected的字段和方法 ✅ | 构造器 ❌（但可以通过super调用） |
| 默认（同包）的字段和方法 ✅ | |

## 23.2 super关键字：父类的"代言人"

`super` 在子类里代表父类，它有三大用途：

### 用途一：调用父类构造器

```java
public Student(String name, int age, String studentId, double score) {
    super(name, age);  // 必须放在子类构造器的第一行
    this.studentId = studentId;
    this.score = score;
}
```

**铁律**：`super(...)` 必须是子类构造器的**第一条语句**。如果父类只有有参构造器，子类必须显式调用 `super(参数)`，否则编译报错。

```java
public class Person {
    public Person(String name) { ... }  // 父类只有有参构造器
}

public class Student extends Person {
    public Student(String name, String studentId) {
        super(name);  // 必须显式调用！不写就报错
        this.studentId = studentId;
    }
}
```

### 用途二：访问父类被遮蔽的字段/方法

```java
public class Parent {
    protected String address = "北京";
}

public class Child extends Parent {
    private String address = "上海";  // 子类定义了自己的address

    public void showAddress() {
        System.out.println("父类的address：" + super.address);  // 北京
        System.out.println("子类的address：" + this.address);   // 上海
    }
}
```

### 用途三：调用父类方法

```java
public class Student extends Person {
    @Override
    public void eat() {
        super.eat();  // 先执行父类的eat()
        System.out.println("吃完后还要学习");  // 再加自己的逻辑
    }
}
```

## 23.3 @Override：方法重写

**方法重写**（Override）：子类定义一个与父类**签名完全一样**的方法，覆盖父类的实现。

```java
public class Animal {
    public void makeSound() {
        System.out.println("动物发出叫声");
    }
}

public class Dog extends Animal {
    @Override                       // ← 这个注解告诉编译器："我在重写"
    public void makeSound() {
        System.out.println("汪汪汪！");
    }
}

public class Cat extends Animal {
    @Override
    public void makeSound() {
        System.out.println("喵喵喵！");
    }
}
```

```java
Animal a1 = new Dog();
Animal a2 = new Cat();
a1.makeSound();  // 汪汪汪！
a2.makeSound();  // 喵喵喵！
```

### @Override注解的三个作用

1. **编译器检查**：如果你写错了方法签名，编译器报错（比如 `public void makesound()` ——少了个大写S）
2. **代码可读性**：一眼看出这个方法是在重写父类
3. **反后悔**：父类哪天改了方法名，子类编译就报错，提醒你去改

> ❌ **常见错误**：重写时方法签名写错一点，比如参数类型不同、方法名大小写不同——这就不是重写，而是定义了一个新方法。**永远加 `@Override` 注解**，让编译器帮你把关。

## 23.4 方法重写 vs 方法重载：一字之差，天壤之别

初学Java最容易混淆的两个概念：

| 对比维度 | 方法重写（Override） | 方法重载（Overload） |
|----------|---------------------|---------------------|
| 发生位置 | 父子类之间 | 同一个类内部 |
| 方法名 | 完全相同 | 完全相同 |
| 参数列表 | 完全相同 | **必须不同**（个数/类型/顺序） |
| 返回类型 | 相同或其子类（协变） | 可以不同 |
| 访问修饰符 | 不能比父类更严格 | 无限制 |
| 注解 | `@Override` | 不需注解 |
| 多态 | 运行时多态 | 编译时多态 |

```java
// 重载（Overload）——在同一个类里
public class Calculator {
    public int add(int a, int b) { return a + b; }
    public int add(int a, int b, int c) { return a + b + c; }          // 参数个数不同
    public double add(double a, double b) { return a + b; }            // 参数类型不同
    public double add(int a, double b) { return a + b; }               // 参数顺序不同
}

// 重写（Override）——在子类里
public class Animal {
    public void makeSound() { ... }
}
public class Dog extends Animal {
    @Override
    public void makeSound() { ... }  // 重写父类方法
}
```

## 23.5 Object类：万类之祖

Java里**所有类都默认继承 `Object` 类**。你不写 `extends`，编译器自动加上 `extends Object`。所以 `java.lang.Object` 是Java类型体系的根。

Object提供了几个所有类都拥有的方法：

| 方法 | 作用 | 何时重写 |
|------|------|----------|
| `toString()` | 返回对象的字符串表示 | 几乎所有类都应该重写 |
| `equals(Object obj)` | 判断两个对象是否"相等" | 需要比较内容而不是地址时重写 |
| `hashCode()` | 返回对象的哈希码 | 重写equals时必须重写hasHCode |
| `getClass()` | 返回对象的类信息 | 通常不需要重写 |

```java
public class Student {
    private String name;
    private String studentId;

    @Override
    public String toString() {
        return "Student{name='" + name + "', studentId='" + studentId + "'}";
    }
}

Student s = new Student("张三", "2024001");
System.out.println(s);  // 不重写的话输出：Student@15db9742
                        // 重写后输出：Student{name='张三', studentId='2024001'}
```

## 23.6 Java为什么只支持单继承？

Java**不允许**一个类同时继承多个类：

```java
// ❌ 编译错误！Java不支持多继承
public class C extends A, B { ... }
```

为什么？因为**菱形问题**（Diamond Problem）：

```
     A (有方法 speak())
    / \
   B   C  (B和C各自重写了speak())
    \ /
     D  → D到底用哪个speak()？B的还是C的？
```

C++支持多继承，但引入了虚继承（virtual inheritance）来解决菱形问题，比较复杂。Java选择了更简单的方案：**单继承 + 多接口实现**（第25章讲）。

但Java支持**多层继承**：
```java
public class Animal { }
public class Mammal extends Animal { }
public class Dog extends Mammal { }    // Dog → Mammal → Animal → Object
```

## 本章小结

| 知识点 | 核心要点 |
|--------|----------|
| 继承 | `class A extends B`。is-a关系。子类拥有父类的public/protected成员 |
| super | 调用父类构造器（必须放第一行）、访问父类被遮蔽的字段、调用父类方法 |
| 构造器链 | 子类构造器隐式或显式调用父类构造器，一路向上直到Object |
| @Override | 重写父类方法的注解。编译器帮你检查签名是否正确 |
| 重写vs重载 | 重写=子类覆盖父类方法（运行期）。重载=同类内同名不同参（编译期） |
| Object | 所有类的终极父类。toString()/equals()/hasHCode() 是标配 |
| 单继承 | Java一个类只能extends一个父类。原因是避免菱形问题 |
