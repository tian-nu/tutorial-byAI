# 第28章 · 包装类与自动装箱

> "Java里的 `ArrayList` 不能存 `int`，只能存对象。那我们怎么把基本类型放进集合里？答案：包装类。Java给每种基本类型都配备了一个'对象版本'——`int` 变 `Integer`、`double` 变 `Double`。更神奇的是，Java会自动在两者之间转换，但这背后藏着陷阱。"

---

## 28.1 为什么需要包装类？

Java的集合框架（`ArrayList`、`HashMap` 等）只能存储对象，不能存储基本类型（`int`、`double` 等）。此外，有些场景需要"没有值"的表示——基本类型没有 `null`，但对象可以。

**包装类**（Wrapper Class。此术语需进附录）把基本类型"包"成一个对象：

| 基本类型 | 包装类 | 大小 |
|----------|--------|------|
| `byte` | `Byte` | 8位 |
| `short` | `Short` | 16位 |
| `int` | `Integer` | 32位 |
| `long` | `Long` | 64位 |
| `float` | `Float` | 32位 |
| `double` | `Double` | 64位 |
| `char` | `Character` | 16位 |
| `boolean` | `Boolean` | 1位 |

### 基本用法

```java
// 创建包装类对象
Integer i1 = Integer.valueOf(10);  // 推荐：可能使用缓存
Integer i2 = new Integer(10);      // 不推荐：Java 9+已废弃

// 类型转换
String numStr = "123";
int num = Integer.parseInt(numStr);      // String → int
double d = Double.parseDouble("3.14");   // String → double
String s = Integer.toString(456);        // int → String

// 边界值常量
int maxInt = Integer.MAX_VALUE;  // 2147483647
int minInt = Integer.MIN_VALUE;  // -2147483648
```

## 28.2 自动装箱与自动拆箱

Java 5引入了**自动装箱**（Autoboxing）和**自动拆箱**（Unboxing），让基本类型和包装类之间的转换变得透明。此术语需进附录。

```java
// 自动装箱：基本类型自动转为包装类
Integer num = 100;  // 编译器自动变成：Integer num = Integer.valueOf(100);

// 自动拆箱：包装类自动转为基本类型
int value = num;    // 编译器自动变成：int value = num.intValue();

// 在表达式中混合使用——完全自动
Integer a = 10;
Integer b = 20;
Integer result = a + b;  // 自动拆箱→int加法→自动装箱回Integer
// 等价于：Integer result = Integer.valueOf(a.intValue() + b.intValue());

// 方法调用中自动转换
public void process(int x) { ... }
process(new Integer(42));  // 自动拆箱为int

// 集合中使用
List<Integer> list = new ArrayList<>();
list.add(5);               // 自动装箱：int 5 → Integer.valueOf(5)
int first = list.get(0);   // 自动拆箱：Integer → int
```

## 28.3 Integer缓存陷阱（-128 ~ 127）：面试必考

这是Java面试面到包装类时必问的题。

```java
Integer a = 100;
Integer b = 100;
System.out.println(a == b);   // true？还是false？

Integer c = 200;
Integer d = 200;
System.out.println(c == d);   // true？还是false？
```

**答案**：第一个 `true`，第二个 `false`。

**原因**：`Integer.valueOf()` 内部维护了一个缓存，默认缓存了 **-128 到 127** 之间的Integer对象。当自动装箱的值在这个范围内时，它直接返回缓存的同一个对象。

```java
// Integer.valueOf() 的简化源码逻辑：
public static Integer valueOf(int i) {
    if (i >= -128 && i <= 127) {
        return IntegerCache.cache[i + 128];  // 返回缓存对象
    }
    return new Integer(i);  // 创建新对象
}
```

所以：
```java
Integer a = 100;  // 自动装箱→Integer.valueOf(100)→返回缓存对象
Integer b = 100;  // 自动装箱→Integer.valueOf(100)→返回**同一个**缓存对象
// a == b → true（指向同一个对象）

Integer c = 200;  // 自动装箱→Integer.valueOf(200)→200超出缓存→new Integer(200)
Integer d = 200;  // 自动装箱→Integer.valueOf(200)→200超出缓存→new Integer(200)
// c == d → false（两个不同的对象！）
```

### 其他包装类的缓存范围

| 包装类 | 缓存范围 |
|--------|----------|
| `Boolean` | `TRUE` 和 `FALSE` 两个值都被缓存 |
| `Byte` | -128 ~ 127（全部） |
| `Short` | -128 ~ 127 |
| `Integer` | -128 ~ 127 |
| `Long` | -128 ~ 127 |
| `Character` | 0 ~ 127 |
| `Float` | **没有缓存** |
| `Double` | **没有缓存** |

> ❌ **常见错误**：用 `==` 比较两个Integer对象的值。记住铁律：**比较两个包装类对象的数值时，永远用 `equals()`，不要用 `==`！**

```java
Integer a = 100;
Integer b = 100;
Integer c = 200;
Integer d = 200;

System.out.println(a.equals(b));  // ✅ true（比较值）
System.out.println(c.equals(d));  // ✅ true（比较值）
System.out.println(a == b);       // ❌ 有时true有时false，不要用
System.out.println(c == d);       // ❌ false
```

## 28.4 自动拆箱的NullPointerException

自动拆箱时，如果包装类对象是 `null`，会抛出 `NullPointerException`：

```java
Integer x = null;
int y = x;  // 自动拆箱→x.intValue()→NullPointerException！

// 最常见的触发场景
Map<String, Integer> map = new HashMap<>();
map.put("score", null);
int score = map.get("score");  // NullPointerException！

// 安全写法
Integer scoreObj = map.get("score");
if (scoreObj != null) {
    int score = scoreObj;
}
// 或者
int score = map.getOrDefault("score", 0);  // 有null就用默认值0
```

## 28.5 包装类的实用方法

```java
// === Integer ===
int num = Integer.parseInt("123");         // 字符串→int
Integer numObj = Integer.valueOf("456");   // 字符串→Integer
String binary = Integer.toBinaryString(10); // "1010"
String hex = Integer.toHexString(255);      // "ff"
int max = Integer.max(10, 20);              // 20
int min = Integer.min(10, 20);              // 10

// === Double ===
double d = Double.parseDouble("3.14");
boolean isNaN = Double.isNaN(0.0 / 0.0);          // true（NaN）
boolean isInfinite = Double.isInfinite(1.0 / 0.0); // true

// === Boolean ===
boolean b = Boolean.parseBoolean("true");  // "true"→true，其他→false
boolean b2 = Boolean.parseBoolean("YES");  // false！（只有"true"才返回true）

// === Character ===
boolean isDigit = Character.isDigit('5');       // true
boolean isLetter = Character.isLetter('A');     // true
boolean isUpperCase = Character.isUpperCase('A'); // true
char lower = Character.toLowerCase('A');         // 'a'

// === Long ===
long big = Long.parseLong("9223372036854775807");
```

## 本章小结

| 知识点 | 核心要点 |
|--------|----------|
| 包装类 | 8种基本类型对应的对象类型。用于集合框架和需要null的场景 |
| 自动装箱 | 基本类型→包装类，编译器自动调用 `valueOf()`。此术语需进附录 |
| 自动拆箱 | 包装类→基本类型，编译器自动调用 `xxxValue()`。此术语需进附录 |
| Integer缓存 | -128~127的Integer对象被缓存。`==`比较在范围内true，范围外false |
| 比较规则 | 包装类数值比较永远用 `equals()`，不要用 `==` |
| 拆箱NPE | 自动拆箱时如果包装类是null，抛NullPointerException |
