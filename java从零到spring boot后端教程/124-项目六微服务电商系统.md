# 124-项目六：微服务电商系统

> 💡 本章你会做出一个**微服务架构的电商系统**，将单体拆分为用户服务、商品服务、订单服务三个独立微服务，使用 Nacos 做注册中心、OpenFeign 做远程调用、Spring Cloud Gateway 做网关。这是从单体到分布式的里程碑跳跃。

---

## 本章目标
- 将一个电商系统拆分为 3 个微服务
- 使用 Nacos 做服务注册与发现
- 使用 OpenFeign 实现服务间调用
- 使用 Spring Cloud Gateway 做统一入口
- 理解 Seata 分布式事务的基本概念

```java
package com.example.user.controller;

import com.example.common.Result;
import com.example.common.UserDTO;
import org.springframework.web.bind.annotation.*;

import java.util.Map;

@RestController
@RequestMapping("/api/users")
public class UserController {

    private final Map<Long, UserDTO> userStore = new java.util.concurrent.ConcurrentHashMap<>();

    @PostMapping
    public Result<UserDTO> create(@RequestBody UserDTO user) {
        user.setId((long) (userStore.size() + 1));
        userStore.put(user.getId(), user);
        return Result.success(user);
    }

    @GetMapping("/{id}")
    public Result<UserDTO> getById(@PathVariable Long id) {
        UserDTO user = userStore.get(id);
        if (user == null) {
            return Result.error(404, "用户不存在");
        }
        return Result.success(user);
    }
}
```

### ProductController

```java
package com.example.product.controller;

import com.example.common.ProductDTO;
import com.example.common.Result;
import org.springframework.web.bind.annotation.*;

import java.math.BigDecimal;
import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;

@RestController
@RequestMapping("/api/products")
public class ProductController {

    private final Map<Long, ProductDTO> productStore = new ConcurrentHashMap<>();

    public ProductController() {
        productStore.put(1L, new ProductDTO(1L, "iPhone 15", new BigDecimal("6999.00"), 100));
        productStore.put(2L, new ProductDTO(2L, "MacBook Pro", new BigDecimal("12999.00"), 50));
    }

    @GetMapping("/{id}")
    public Result<ProductDTO> getById(@PathVariable Long id) {
        ProductDTO product = productStore.get(id);
        if (product == null) {
            return Result.error(404, "商品不存在");
        }
        return Result.success(product);
    }

    @PutMapping("/{id}/stock")
    public Result<String> deductStock(@PathVariable Long id,
                                       @RequestParam int quantity) {
        ProductDTO product = productStore.get(id);
        if (product == null) {
            return Result.error(404, "商品不存在");
        }
        if (product.getStock() < quantity) {
            return Result.error(400, "库存不足");
        }
        product.setStock(product.getStock() - quantity);
        return Result.success("扣减成功");
    }
}
```

### OrderService

```java
package com.example.order.service;

import com.example.common.ProductDTO;
import com.example.common.Result;
import com.example.common.UserDTO;
import com.example.order.feign.ProductFeignClient;
import com.example.order.feign.UserFeignClient;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;

@Service
@RequiredArgsConstructor
public class OrderService {

    private final UserFeignClient userFeignClient;
    private final ProductFeignClient productFeignClient;

    public String createOrder(Long userId, Long productId, int quantity) {
        Result<UserDTO> userResult = userFeignClient.getUserById(userId);
        if (userResult.getCode() != 200) {
            return "下单失败：用户不存在";
        }

        Result<ProductDTO> productResult = productFeignClient.getProductById(productId);
        if (productResult.getCode() != 200) {
            return "下单失败：商品不存在";
        }

        Result<String> stockResult = productFeignClient.deductStock(productId, quantity);
        if (stockResult.getCode() != 200) {
            return "下单失败：" + stockResult.getMessage();
        }

        return String.format("订单创建成功！用户%s 购买了 %s x%d，总价 %.2f",
                userResult.getData().getUsername(),
                productResult.getData().getName(),
                quantity,
                productResult.getData().getPrice().multiply(
                        new java.math.BigDecimal(quantity)));
    }
}
```

## 124.10 Seata 分布式事务简介

### 问题

订单服务调用了两个服务：用户服务（查用户）+ 商品服务（扣库存）。如果扣库存成功后，订单自身写入数据库失败，库存已经扣了无法回滚——这就是分布式事务问题。

### Seata AT 模式

Seata 通过**全局事务协调器（TC）** + **undo_log 表**实现自动回滚：

```
订单服务：                               Seata TC：
  ↓                                      ┌───────────┐
  @GlobalTransactional                   │ 全局事务ID  │
  ├─ 查用户 (user-service)   ──────────▶│ 记录分支1   │
  ├─ 扣库存 (product-service) ──────────▶│ 记录分支2   │
  └─ 写订单 (order-service)   ──失败──▶ │ 通知分支1+2 │
                                         │ 执行回滚    │
                                         └───────────┘
```

> 💡 Seata 的 AT 模式对业务代码零侵入，只需要加 `@GlobalTransactional` 注解和 undo_log 表，框架自动管理全局事务。

### Seata 快速启动（可选）

```bash
docker run -d --name seata-server -p 8091:8091 -p 7091:7091 \
  seataio/seata-server:1.8.0
```

各微服务添加依赖：

```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-seata</artifactId>
</dependency>
```

`application.yml` 添加配置：

```yaml
seata:
  tx-service-group: my_tx_group
  service:
    vgroup-mapping:
      my_tx_group: default
```

在需要分布式事务的方法上加：

```java
@GlobalTransactional
public String createOrder(Long userId, Long productId, int quantity) {
    // ... 跨服务调用
}
```

## 自测题

1. `@FeignClient("user-service")` 中的 `"user-service"` 是什么？OpenFeign 怎么找到对应的服务地址？
2. Spring Cloud Gateway 的 `lb://user-service` 和直接写 `http://localhost:8081` 有什么区别？
3. 分布式事务中，Seata 的 AT 模式和 TCC 模式的核心区别是什么？

<details>
<summary>点击查看答案</summary>

1. `"user-service"` 是 Nacos 中注册的服务名。OpenFeign 从 Nacos 获取该服务的所有实例列表（IP:Port），通过 LoadBalancer 选择一个实例发起调用。
2. `lb://user-service` 是负载均衡地址：Gateway 从 Nacos 发现 user-service 的所有实例，动态选择一个转发。`http://localhost:8081` 是硬编码地址：单点故障、无法扩展。微服务必须用前者。
3. AT 模式对业务代码零侵入，Seata 通过解析 SQL 自动生成 undo_log 实现回滚；TCC 需要业务方实现 Try/Confirm/Cancel 三个方法，侵入性强但性能更好、适合高并发场景。
</details>
