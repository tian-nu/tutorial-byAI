## 52.1 为什么需要学多种排序算法？

你可能会说：一行`Arrays.sort(arr)`不就行了？

是的，工作中直接用这个。但你必须理解它内部做了什么，才能：
- **分析数据库`ORDER BY`的性能**：数据量小时用快速排序，大时用归并排序
- **理解为什么索引能加速排序**：B+树本身就是有序的
- **面试**：手写快排/归并是后端面试的"入场券"
- **极端场景选择合适方案**：几乎有序的数据用插入排序O(n)，随机数据用快排O(n log n)

## 52.3 基础排序（O(n²)三兄弟）

### 冒泡排序

像气泡一样，大的往后浮。每轮把未排序区最大的元素"冒"到最后。

```java
public void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        boolean swapped = false;          // 优化：如果一轮没交换，说明已有序
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = true;
            }
        }
        if (!swapped) break;  // 提前终止
    }
}
```

### 选择排序

每轮从未排序区选出最小的，放到已排序区的末尾。

```java
public void selectionSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        int minIndex = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }
        int temp = arr[i];
        arr[i] = arr[minIndex];
        arr[minIndex] = temp;
    }
}
// 特点：无论数据是否有序，都是O(n²)。交换次数最少（每轮最多一次）。
```

### 插入排序

像打牌时理牌，拿一张插到已排序区中正确的位置。

```java
public void insertionSort(int[] arr) {
    int n = arr.length;
    for (int i = 1; i < n; i++) {
        int key = arr[i];      // 当前要插入的牌
        int j = i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];    // 大于key的元素右移
            j--;
        }
        arr[j + 1] = key;           // 插入到正确位置
    }
}
// 特点：数据几乎有序时接近O(n)！TimSort就利用了这一点。
```

## 52.4 快速排序（QuickSort — 最重要的排序算法）

**分治思想**：选一个基准值（pivot），比它小的放左边，比它大的放右边，然后递归排序左右两部分。

```java
public void quickSort(int[] arr, int left, int right) {
    if (left >= right) return;

    int pivotIndex = partition(arr, left, right);
    quickSort(arr, left, pivotIndex - 1);
    quickSort(arr, pivotIndex + 1, right);
}

private int partition(int[] arr, int left, int right) {
    // 选最右边元素为基准
    int pivot = arr[right];
    int i = left;  // i指向小于pivot区的下一个位置

    for (int j = left; j < right; j++) {
        if (arr[j] < pivot) {
            swap(arr, i, j);
            i++;
        }
    }
    swap(arr, i, right);  // 把pivot放到正确位置
    return i;
}

private void swap(int[] arr, int i, int j) {
    int temp = arr[i];
    arr[i] = arr[j];
    arr[j] = temp;
}
```

**partition过程演示（[5,2,8,1,9,3]，pivot=3）：**

```
初始: [5, 2, 8, 1, 9, 3]  pivot=3, i=0, j=0
j=0: 5<3? No  → [5, 2, 8, 1, 9, 3]  i=0
j=1: 2<3? Yes → swap(arr,0,1) → [2, 5, 8, 1, 9, 3]  i=1
j=2: 8<3? No  → [2, 5, 8, 1, 9, 3]  i=1
j=3: 1<3? Yes → swap(arr,1,3) → [2, 1, 8, 5, 9, 3]  i=2
j=4: 9<3? No  → [2, 1, 8, 5, 9, 3]  i=2
最后: swap(arr,2,5) → [2, 1, 3, 5, 9, 8]
pivot在索引2，左边都<3，右边都>3
```

> 🤔 想多一点：快速排序的最坏情况是**每次pivot都是最小或最大**（如已排序数组选最后一个为pivot），退化为O(n²)。优化方法：随机选pivot、三数取中法。JDK的`Arrays.sort(int[])`使用**Dual-Pivot QuickSort**（双轴快排），用两个pivot分成三段，更快。

## 52.5 归并排序（MergeSort — 稳定排序的王者）

**分治思想**：把数组分成两半，分别排序，然后合并两个有序数组。

```java
public void mergeSort(int[] arr, int left, int right) {
    if (left >= right) return;

    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);       // 排序左半
    mergeSort(arr, mid + 1, right);  // 排序右半
    merge(arr, left, mid, right);    // 合并两个有序半区
}

private void merge(int[] arr, int left, int mid, int right) {
    int[] temp = new int[right - left + 1];
    int i = left;       // 左半区指针
    int j = mid + 1;    // 右半区指针
    int k = 0;          // 临时数组指针

    while (i <= mid && j <= right) {
        if (arr[i] <= arr[j]) {
            temp[k++] = arr[i++];
        } else {
            temp[k++] = arr[j++];
        }
    }
    // 把剩余元素拷贝过去
    while (i <= mid) temp[k++] = arr[i++];
    while (j <= right) temp[k++] = arr[j++];

    // 拷贝回原数组
    for (int p = 0; p < temp.length; p++) {
        arr[left + p] = temp[p];
    }
}
```

**归并过程演示（合并 [2,5,8] 和 [3,6,7]）：**

```
左: [2, 5, 8]    右: [3, 6, 7]    temp: []
2<3 → temp:[2]    左进1
5>3 → temp:[2,3]  右进1
5<6 → temp:[2,3,5] 左进1
8>6 → temp:[2,3,5,6] 右进1
8>7 → temp:[2,3,5,6,7] 右进1
右空 → temp:[2,3,5,6,7,8] 左剩余
```

**为什么归并排序是O(n log n)且稳定？**
- 树高log n层，每层合并O(n) → O(n log n)
- 相等元素合并时左半区优先 → 稳定（相对顺序不变）
- 但需要额外O(n)的空间

## 52.6 Java的 Arrays.sort() 内部做了什么？

**JDK对排序的实现是双轨制**，非常精巧：

```java
// 对原始类型（int[], long[], double[] 等）
Arrays.sort(intArr);
// → 使用 Dual-Pivot QuickSort（双轴快速排序）
// 原因：原始类型不需要稳定性，Dual-Pivot比普通快排更快


// 对对象类型（String[], User[] 等）
Arrays.sort(objArr);
// → 使用 TimSort（归并排序的优化版）
// 原因：对象排序需要稳定性（equals相等的元素保持原顺序）
```

### Dual-Pivot QuickSort（JDK 7+，作者Vladimir Yaroslavskiy）

用两个pivot（p1 < p2），把数组分成三部分：
- < p1 的部分
- p1 ≤ x ≤ p2 的部分
- \> p2 的部分

比传统快排选择更均衡，缓存友好性更好。

### TimSort（JDK 7+，作者Tim Peters，原为Python设计）

TimSort的核心洞察：**现实中大部分数据都是"部分有序"的**。它：
1. 扫描数组，找出所有自然有序的run（连续升序或降序段）
2. 短run用插入排序扩展
3. 用归并排序合并这些run

**TimSort的稳定性保证了**：`Arrays.sort(users, comparing(User::getAge))`——同龄用户保持原始顺序。

## 52.7 算法选择的实战决策树

```
需要排序 →
  │
  ├── 数据量 < 100 → 插入排序（常数因子小）
  │
  ├── 对象数组 → TimSort / 归并排序（需要稳定）
  │
  ├── 原始类型 → Dual-Pivot QuickSort
  │
  ├── 数据几乎有序 → 插入排序 O(n)
  │
  ├── 内存有限 → 快速排序（O(log n)额外空间）
  │
  └── 需要外部排序（数据太大，内存放不下）→ 归并排序分块
```

## 52.8 ❌ 常见错误

| 错误 | 后果 | 正确做法 |
|---|---|---|
| 快排partition索引越界 | ArrayIndexOutOfBounds | j只循环到right-1（不含pivot） |
| 归并排序merge时忘记拷贝回原数组 | 排序无效 | merge末尾必须`arr[left+p] = temp[p]` |
| 以为`Arrays.sort(int[])`稳定 | 期望同龄用户保持原顺序 | 对对象用`Arrays.sort(T[])`才有TimSort |
| 快排递归base case不对 | 无限递归/StackOverflow | `if (left >= right) return;` |
| 冒泡排序不带swapped优化 | 已有序还跑满O(n²) | 加swapped标志提前终止 |

## 小结

| 算法 | 时间复杂度 | 空间 | 稳定 | 核心思想 |
|---|---|---|---|---|
| 冒泡 | O(n²) | O(1) | ✅ | 大的往后冒 |
| 选择 | O(n²) | O(1) | ❌ | 每轮选最小的 |
| 插入 | O(n²) / 几乎有序O(n) | O(1) | ✅ | 打牌理牌 |
| 快排 | O(n log n) / 最坏O(n²) | O(log n) | ❌ | 分治，pivot划分 |
| 归并 | O(n log n) | O(n) | ✅ | 分治，合并有序 |
| TimSort | O(n log n) / 部分有序O(n) | O(n) | ✅ | 找run+归并 |
| Dual-Pivot | O(n log n) | O(log n) | ❌ | 双轴三区快排 |
