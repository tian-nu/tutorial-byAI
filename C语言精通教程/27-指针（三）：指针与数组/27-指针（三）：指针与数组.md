## 我在做什么？

前两章你学会了指针的基本操作和指针与函数配合。这一章解开 C 语言中一个最核心的秘密：**数组和指针的关系**。

在 C 语言中，数组名几乎等同于指针。`arr[i]` 和 `*(arr + i)` 是等价的。这不是巧合，而是 C 语言设计的基石。读完这一章，你会理解指针算术，知道为什么 `arr[i]` 等价于 `*(arr + i)`，并且能用指针遍历数组。

## 一、数组名就是指针

### 1.1 数组名是首元素地址

```c
#include <stdio.h>

int main() {
    int arr[] = {10, 20, 30, 40, 50};

    printf("arr     = %p\n", arr);       // 数组名的值
    printf("&arr[0] = %p\n", &arr[0]);   // 首元素地址
    // 两个地址一模一样！

    return 0;
}
```

输出：
```text
arr     = 0x7ffd12345670
&arr[0] = 0x7ffd12345670
```

数组名 `arr` 的值就是数组首元素的地址。所以可以直接把数组名赋给指针：

```c
int *p = arr;     // 等价于 int *p = &arr[0];
```

### 1.2 数组名和指针的区别

虽然数组名在大多数情况下表现得像指针，但它们不是同一个东西：

```c
int arr[5] = {1, 2, 3, 4, 5};
int *p = arr;

p = p + 1;        // 正确：指针可以改变指向
// arr = arr + 1; // 错误：数组名是常量，不能改变
```

```text
arr：是一个标签，永远贴在数组首元素的地址上，不可移动。
p：  是一个变量，可以指向任何地方。
```

## 二、指针算术

### 2.1 指针加减整数

指针加 1，不是加 1 个字节，而是**跳过一个元素**。

```c
#include <stdio.h>

int main() {
    int arr[] = {10, 20, 30, 40, 50};
    int *p = arr;          // p 指向 arr[0]

    printf("*p     = %d\n", *p);       // 10
    printf("*p + 1 = %d\n", *p + 1);   // 11（先取 *p=10，再加 1）
    printf("*(p+1) = %d\n", *(p + 1)); // 20（p+1 指向 arr[1]，再解引用）

    return 0;
}
```

**关键区别**：`*p + 1` 和 `*(p + 1)` 完全不同。

```text
int arr[] = {10, 20, 30, 40, 50};
int *p = arr;

  p     p+1   p+2   p+3   p+4
  ↓     ↓     ↓     ↓     ↓
┌────┬────┬────┬────┬────┐
│ 10 │ 20 │ 30 │ 40 │ 50 │
└────┴────┴────┴────┴────┘
  [0]  [1]  [2]  [3]  [4]

p + 1 的地址 = p 的地址 + sizeof(int) × 1
               = p 的地址 + 4 字节 × 1
```

### 2.2 指针减法

```c
int arr[] = {10, 20, 30, 40, 50};
int *p1 = &arr[0];
int *p2 = &arr[3];

printf("p2 - p1 = %ld\n", p2 - p1);  // 输出 3（相差 3 个元素）
```

两个指针相减得到的是**元素个数**，不是字节数。

### 2.3 指针算术的实际步长

```c
#include <stdio.h>

int main() {
    int    iarr[] = {1, 2, 3};
    double darr[] = {1.0, 2.0, 3.0};
    char   carr[] = {'a', 'b', 'c'};

    printf("int*    +1 移动 %zu 字节\n", sizeof(int));
    printf("double* +1 移动 %zu 字节\n", sizeof(double));
    printf("char*   +1 移动 %zu 字节\n", sizeof(char));

    return 0;
}
```

输出：
```text
int*    +1 移动 4 字节
double* +1 移动 8 字节
char*   +1 移动 1 字节
```

指针算术的步长由指针类型决定。这是编译器自动处理的——你不需要手动乘以 `sizeof`。

## 三、`arr[i]` 等价于 `*(arr + i)`

### 3.1 核心等价关系

```c
arr[i]   ←→   *(arr + i)
&arr[i]  ←→   arr + i
```

```c
#include <stdio.h>

int main() {
    int arr[] = {10, 20, 30, 40, 50};

    printf("arr[2]   = %d\n", arr[2]);      // 30
    printf("*(arr+2) = %d\n", *(arr + 2));  // 30

    printf("&arr[2]  = %p\n", &arr[2]);     // 某个地址
    printf("arr+2    = %p\n", arr + 2);      // 同一个地址

    return 0;
}
```

### 3.2 为什么等价

```text
arr = 数组首地址 = 0x100（假设）

arr[0] = *(arr + 0) = *(0x100 + 0×4) = *0x100 = 10
arr[1] = *(arr + 1) = *(0x100 + 1×4) = *0x104 = 20
arr[2] = *(arr + 2) = *(0x100 + 2×4) = *0x108 = 30
```

在 C 语言设计者看来，`arr[i]` 就是 `*(arr + i)` 的语法糖。

### 3.3 一个有趣的推论

因为 `arr[i]` 等于 `*(arr + i)`，而加法是可交换的，所以：

```c
arr[i]  ←→  *(arr + i)  ←→  *(i + arr)  ←→  i[arr]
```

是的，`i[arr]` 也是合法的 C 语言！不过没人这么写，因为太反直觉。

```c
#include <stdio.h>

int main() {
    int arr[] = {10, 20, 30};
    printf("%d\n", 2[arr]);  // 输出 30，等价于 arr[2]
    return 0;
}
```

## 四、指针遍历 vs 下标遍历

### 4.1 两种遍历方式

```c
#include <stdio.h>

int main() {
    int arr[] = {10, 20, 30, 40, 50};
    int size = 5;

    // 方式一：下标遍历（你熟悉的）
    printf("下标遍历：");
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");

    // 方式二：指针遍历
    printf("指针遍历：");
    for (int *p = arr; p < arr + size; p++) {
        printf("%d ", *p);
    }
    printf("\n");

    return 0;
}
```

输出：
```text
下标遍历：10 20 30 40 50
指针遍历：10 20 30 40 50
```

### 4.2 指针遍历的详细过程

```text
初始化：p = arr
        p
        ↓
      ┌────┬────┬────┬────┬────┐
      │ 10 │ 20 │ 30 │ 40 │ 50 │
      └────┴────┴────┴────┴────┘

第一次循环：*p = 10, p++（p 右移）
            p
            ↓
      ┌────┬────┬────┬────┬────┐
      │ 10 │ 20 │ 30 │ 40 │ 50 │
      └────┴────┴────┴────┴────┘

第二次循环：*p = 20, p++
                p
                ↓
      ┌────┬────┬────┬────┬────┐
      │ 10 │ 20 │ 30 │ 40 │ 50 │
      └────┴────┴────┴────┴────┘

...直到 p == arr + size（越界），循环结束
```

### 4.3 哪种更好？

| | 下标遍历 | 指针遍历 |
|---|---|---|
| 可读性 | 更直观 | 稍难读 |
| 速度 | 每次要计算 `arr + i` | 只需 `p++`，略快 |
| 灵活性 | 需要知道索引 | 不依赖索引 |
| 使用场景 | 大多数情况 | 追求极致性能，或处理不完整数组 |

**经验法则**：除非有特殊理由，用下标遍历更清晰。现代编译器优化后两者性能差异极小。

## 五、指针与字符串

### 5.1 字符串就是 char 指针

```c
char *str = "hello";    // str 指向字符串字面量
char str2[] = "hello";  // str2 是数组

// 两种方式都可以用指针遍历
char *p = str;
while (*p != '\0') {
    printf("%c", *p);
    p++;
}
```

### 5.2 字符串遍历的经典写法

```c
#include <stdio.h>

// 自己实现 strlen
size_t my_strlen(const char *s) {
    const char *p = s;
    while (*p != '\0') {
        p++;
    }
    return p - s;   // 结束地址 - 起始地址 = 字符个数
}

int main() {
    printf("长度：%zu\n", my_strlen("hello"));  // 输出 5
    return 0;
}
```

```text
s = "hello"
  s
  ↓
┌───┬───┬───┬───┬───┬───┐
│ h │ e │ l │ l │ o │\0 │
└───┴───┴───┴───┴───┴───┘
                      ↑
                      p（循环结束后）
p - s = 5
```

## 六、指针与数组作为函数参数

### 6.1 三种等价的写法

```c
void func1(int arr[]);       // 写法一：数组形式
void func2(int arr[10]);     // 写法二：带大小（编译器忽略数字）
void func3(int *arr);        // 写法三：指针形式（本质）
```

函数参数中的 `int arr[]` 和 `int *arr` 完全等价。它们都是指针。

### 6.2 为什么函数内 sizeof 失效

```c
#include <stdio.h>

void print_size(int arr[]) {
    printf("函数内 sizeof(arr) = %zu\n", sizeof(arr));  // 总是 8
}

int main() {
    int nums[] = {1, 2, 3, 4, 5};
    printf("main 中 sizeof(nums) = %zu\n", sizeof(nums));  // 输出 20
    print_size(nums);
    return 0;
}
```

输出：
```text
main 中 sizeof(nums) = 20
函数内 sizeof(arr) = 8
```

原因：函数参数中的 `arr` 实际上是指针，`sizeof(arr)` 求的是指针大小。

## 七、常见错误与排错

### 7.1 混淆 `*p + 1` 和 `*(p + 1)`

❌ 错误示例：
```c
int arr[] = {10, 20, 30};
int *p = arr;
printf("%d\n", *p + 1);   // 输出 11，不是 20
```

✅ 正确示例：
```c
printf("%d\n", *(p + 1)); // 输出 20
```

### 7.2 指针遍历越界

❌ 错误示例：
```c
int arr[5] = {1, 2, 3, 4, 5};
int *p;
for (p = arr; p <= arr + 5; p++) {  // 用了 <=，最后一次 p 指向 arr[5]，越界
    printf("%d\n", *p);
}
```

✅ 正确示例：
```c
for (p = arr; p < arr + 5; p++) {   // 用 <，p 最多到 arr+4
    printf("%d\n", *p);
}
```

### 7.3 对数组名做 `++`

❌ 错误示例：
```c
int arr[5] = {1, 2, 3, 4, 5};
arr++;  // 错误！数组名是常量
```

✅ 正确示例：
```c
int *p = arr;
p++;    // 正确：p 是指针变量
```

## 我做得对不对？

用以下程序验证：

```c
#include <stdio.h>

// 用指针遍历求和
int sum_pointer(int *arr, int size) {
    int total = 0;
    for (int *p = arr; p < arr + size; p++) {
        total += *p;
    }
    return total;
}

// 自己实现 strlen
size_t my_strlen(const char *s) {
    const char *p = s;
    while (*p) p++;   // *p != '\0' 的简写
    return p - s;
}

int main() {
    // 测试1：arr[i] == *(arr+i)
    int arr[] = {10, 20, 30, 40, 50};
    printf("测试1：arr[3] = %d (应为 40)\n", arr[3]);
    printf("测试1：*(arr+3) = %d (应为 40)\n", *(arr + 3));

    // 测试2：指针遍历求和
    printf("测试2：sum = %d (应为 150)\n", sum_pointer(arr, 5));

    // 测试3：指针减法
    int *p1 = &arr[1];
    int *p2 = &arr[4];
    printf("测试3：p2 - p1 = %ld (应为 3)\n", p2 - p1);

    // 测试4：自己实现 strlen
    printf("测试4：my_strlen(\"hello\") = %zu (应为 5)\n", my_strlen("hello"));

    return 0;
}
```

编译运行：
```bash
$ gcc -Wall -o test test.c
$ ./test
测试1：arr[3] = 40 (应为 40)
测试1：*(arr+3) = 40 (应为 40)
测试2：sum = 150 (应为 150)
测试3：p2 - p1 = 3 (应为 3)
测试4：my_strlen("hello") = 5 (应为 5)
```

## 不对怎么办？

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| 指针遍历输出乱码 | 越界了（用了 `<=` 而不是 `<`） | 循环条件改为 `p < arr + size` |
| 指针加减结果不对 | 混淆了元素个数和字节数 | 记住：`p+1` 跳过 1 个元素，不是 1 字节 |
| 编译错误 `lvalue required` | 对数组名做了 `++` 或赋值 | 用指针变量替代数组名 |
| 指针相减得到负数 | 指针顺序搞反了 | 确保被减指针指向更后面的元素 |
| 函数内 `sizeof(arr)` 不是预期值 | 忘记函数参数中 `arr` 是指针 | 在函数外计算大小，或额外传 size |

## 本章小结

| 知识点 | 一句话总结 |
|--------|-----------|
| 数组名即指针 | 数组名是首元素地址，可赋给同类型指针 |
| 指针算术 | `p + n` 跳过 n 个元素，步长由类型决定 |
| `arr[i] == *(arr+i)` | 下标访问本质是指针运算的语法糖 |
| 指针遍历 | `for (p = arr; p < arr + size; p++)` |
| 指针相减 | 得到两个指针之间的元素个数 |
| 字符串指针遍历 | `while (*p) p++;` 遍历到 `\0` |
| 函数参数中的数组 | `int arr[]` 和 `int *arr` 等价 |
