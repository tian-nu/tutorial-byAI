# 第25章 · 面向对象（四）抽象类与接口

> "你去动物园，见过一只纯粹的'动物'吗？没有。你见的要么是狗，要么是猫，要么是老虎。'动物'只是一个抽象概念——它定义了所有动物的共性，但你永远无法创建一个'动物'实例。抽象类就是这种角色。接口更进一步：它只定义'能做什么'，完全不关心'怎么做'。"

---

## 25.1 抽象类：不能new的类

**抽象类**（Abstract Class。此术语需进附录）用 `abstract` 关键字修饰，它不能直接被实例化（不能用 `new`），只能被继承。

```java
// 抽象类——不能new它
public abstract class Animal {
    protected String name;

    public Animal(String name) {
        this.name = name;
    }

    // 抽象方法——没有方法体，子类必须实现
    public abstract void makeSound();

    // 普通方法——有具体实现，子类可以直接用或重写
    public void sleep() {
        System.out.println(name + "正在睡觉");
    }
}

// 子类必须实现所有抽象方法
public class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }

    @Override
    public void makeSound() {   // 必须实现！
        System.out.println(name + "汪汪汪！");
    }
}
```

```java
Animal a = new Animal("某动物");  // ❌ 编译错误！抽象类不能实例化
Animal a = new Dog("大黄");       // ✅ 用子类
a.makeSound();  // 大黄汪汪汪！
a.sleep();      // 大黄正在睡觉（继承来的普通方法）
```

### 抽象类的规则

| 规则 | 说明 |
|------|------|
| 抽象类的构造器 | 可以有构造器，供子类通过 `super()` 调用 |
| 抽象方法 | 只有声明，没有方法体。子类必须实现（除非子类也是抽象类） |
| 抽象类中可以有普通方法 | 抽象类 = 普通类 + 抽象方法。可以有字段、普通方法 |
| 有抽象方法的类**必须**声明为抽象类 | 反过来不成立：抽象类可以不包含任何抽象方法 |

## 25.2 接口：只定义"能做什么"

**接口**（Interface。此术语需进附录）是一种比抽象类更"纯"的抽象——在Java 8之前，接口里所有方法都是抽象的（没有方法体）。

接口描述的是"能做什么"（has-a能力），而不是"是什么"（is-a身份）。

```java
// 定义一个接口
public interface Flyable {
    void fly();  // 默认为 public abstract（即使你不写）
}

public interface Swimable {
    void swim();
}

// 一个类可以实现多个接口（弥补了单继承的缺陷）
public class Duck implements Flyable, Swimable {
    @Override
    public void fly() {
        System.out.println("鸭子扑腾扑腾飞");
    }

    @Override
    public void swim() {
        System.out.println("鸭子悠闲地游");
    }
}
```

### 比喻：USB接口

USB接口只定义规范——形状、引脚、电压。任何实现了USB规范的设备（鼠标、键盘、U盘）都能插到USB口上工作。电脑不关心你是什么设备，只关心你是不是实现了USB接口。

这就对应Java里：`USB usb = new Mouse(); usb.connect();` ——只要实现了USB接口，就能用。

### 接口 vs 抽象类

| 对比维度 | 抽象类 | 接口 |
|----------|--------|------|
| 关键字 | `abstract class` | `interface` |
| 继承/实现 | 只能extends一个 | 可以实现（implements）多个 |
| 构造器 | 可以有 | **不能有** |
| 字段 | 可以有实例字段 | **只能有常量**（`public static final`） |
| 方法 | 可以有抽象方法 + 普通方法 | Java 8+可以有default/static方法；Java 9+可以有private方法 |
| 访问修饰符 | 任意 | 方法默认 `public`，字段默认 `public static final` |
| 用途 | "是什么"——is-a关系 | "能做什么"——has-a能力 |

## 25.3 Java 8的接口革命：default方法

Java 8之前，给接口加一个新方法意味着所有实现类都要改——这是灾难性的。Java 8引入了**default方法**：接口方法可以有默认实现。

```java
public interface Flyable {
    void fly();

    // default方法——有默认实现，实现类可以不改
    default void land() {
        System.out.println("安全着陆");
    }
}

public class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟在飞");
    }
    // land()不需要重写！自动继承默认实现
}

Bird b = new Bird();
b.fly();   // 鸟在飞
b.land();  // 安全着陆（默认实现）
```

### 多继承的"菱形问题"在接口中重新出现

如果一个类实现了两个接口，它们有同名的default方法：

```java
public interface A {
    default void hello() { System.out.println("A的hello"); }
}

public interface B {
    default void hello() { System.out.println("B的hello"); }
}

public class C implements A, B {
    // 必须重写hello()来解决冲突！
    @Override
    public void hello() {
        A.super.hello();  // 可以选择调用A的版本
    }
}
```

### 接口的static方法（Java 8+）

```java
public interface MathUtils {
    static int max(int a, int b) {
        return a > b ? a : b;
    }
}

int result = MathUtils.max(10, 20);  // 通过接口名直接调用
```

## 25.4 函数式接口（Functional Interface）

**函数式接口**是**只包含一个抽象方法**的接口。Java 8的Lambda表达式就是基于函数式接口实现的。

```java
// 函数式接口——只有一个抽象方法
@FunctionalInterface    // 这个注解让编译器检查：是不是只有一个抽象方法？
public interface Calculator {
    int calculate(int a, int b);  // 唯一的抽象方法
}

// 使用Lambda表达式
Calculator add = (a, b) -> a + b;
Calculator sub = (a, b) -> a - b;

System.out.println(add.calculate(10, 5));  // 15
System.out.println(sub.calculate(10, 5));  // 5
```

Java标准库提供了几个常用的函数式接口：

| 接口 | 抽象方法 | 用途 |
|------|----------|------|
| `Function<T,R>` | `R apply(T t)` | 接收T，返回R |
| `Predicate<T>` | `boolean test(T t)` | 接收T，返回boolean（条件判断） |
| `Consumer<T>` | `void accept(T t)` | 接收T，无返回值（消费） |
| `Supplier<T>` | `T get()` | 无参数，返回T（供应） |

## 25.5 抽象类还是接口？选择指南

| 场景 | 推荐 |
|------|------|
| 需要共享**状态**（字段） | 抽象类（接口不能有实例字段） |
| 需要共享**默认行为**（方法实现） | 都可以（抽象类的方法 + 接口的default方法） |
| 需要**多继承**（实现多个"角色"） | 接口（类只能单继承） |
| 定义**规范/协议** | 接口（比如 `Comparable`、`Serializable`） |
| 既有状态又有行为 | 抽象类 |
| 只是增加"某某能力" | 接口 |

> 🤔 **想多一点**：很多人纠结于"用抽象类还是用接口"。一个简单的决策框架：如果子类之间是**"是一种"**关系且共享状态（字段），用抽象类。如果只是**"能做某件事"**（多种不同身份的对象都能做这件事），用接口。Spring框架里大量使用接口来定义服务契约。

## 本章小结

| 知识点 | 核心要点 |
|--------|----------|
| 抽象类 | `abstract class`。不能new。可以有抽象方法（无方法体）+ 普通方法（有实现） |
| 接口 | `interface`。纯行为契约。Java 8+: default方法、static方法 |
| 实现多个接口 | `class A implements B, C`。弥补了单继承的缺陷 |
| default方法 | Java 8引入，接口方法有默认实现。多接口同名时子类必须解决冲突 |
| 函数式接口 | 只有一个抽象方法的接口。`@FunctionalInterface`注解。配合Lambda使用 |
| 抽象类 vs 接口 | 抽象类=is-a+共享状态。接口=has-a能力+多实现 |
