> "图形界面让你舒服，命令行让你强大。当你通过 SSH 连上一台远程 Linux 服务器，没有桌面，没有鼠标——只有一个闪烁的光标。那是后端工程师真正的主场。"

## 05.1 终端是什么

**终端（Terminal）** 是一个文本界面，你敲命令，操作系统执行，返回结果。没有图标、没有菜单、没有滚动条——只有文字。

- Windows上：PowerShell 或 CMD 或 Windows Terminal
- macOS上：Terminal.app 或 iTerm2
- Linux上：gnome-terminal 或直接用 SSH 连上去

```bash
# 你敲的命令                   # 注释（不会执行）
whoami                        # 显示当前用户名
pwd                           # 显示当前目录路径
ls                            # 列出当前目录的文件（Windows用 dir）
cd /home/user                 # 切换到指定目录
mkdir myproject               # 创建文件夹
rm file.txt                   # 删除文件（小心！没有回收站）
```

## 05.2 环境变量：JAVA_HOME 与 PATH

这是新手最容易卡住的地方。

### 环境变量是什么？

环境变量是操作系统级别的"全局变量"。任何程序都能读取它们。

### JAVA_HOME

`JAVA_HOME` 告诉系统"JDK装在哪里"。

```bash
# Windows PowerShell 查看
echo $env:JAVA_HOME
# 输出: C:\Program Files\Java\jdk-21

# Linux/macOS 查看
echo $JAVA_HOME
# 输出: /usr/lib/jvm/java-21-openjdk
```

如果不设 `JAVA_HOME`，很多工具（如Maven、Gradle、IDE）找不到JDK在哪里。

### PATH

`PATH` 告诉系统"去哪里找可执行程序"。当你敲 `java` 或 `javac` 时，操作系统会去 `PATH` 里列出的目录中依次查找。

```bash
# 查看PATH
echo $env:PATH    # Windows PowerShell
echo $PATH        # Linux/macOS
```

你可能会看到类似 `/usr/lib/jvm/java-21/bin` 这样的路径。进去看看：

```bash
ls /usr/lib/jvm/java-21/bin
# java  javac  jar  javadoc  jshell  jstack  ...
```

这些就是JDK自带的命令行工具。

### 怎么设置？

**Windows（系统环境变量）：**
1. 搜索"编辑系统环境变量" → 环境变量
2. 新建系统变量：`JAVA_HOME = C:\Program Files\Java\jdk-21`
3. 编辑 `Path`，添加：`%JAVA_HOME%\bin`

**Linux/macOS（shell配置文件）：**
```bash
# 在 ~/.bashrc 或 ~/.zshrc 末尾添加
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk
export PATH=$JAVA_HOME/bin:$PATH
```

设置完后验证：

```bash
java -version
# openjdk version "21.0.1" 2023-10-17
# OpenJDK Runtime Environment (build 21.0.1+12-LTS)
# OpenJDK 64-Bit Server VM (build 21.0.1+12-LTS, mixed mode, sharing)

javac -version
# javac 21.0.1
```

两条命令都有输出且版本号一致 → 环境配置成功！

## 05.3 Maven命令初体验

Maven是Java项目的构建工具（相当于Java世界的"make"）。它会帮你：
- 管理依赖（下载第三方库）
- 编译代码
- 运行测试
- 打包成 `.jar` 文件

```bash
# 创建一个新的Maven项目（快速体验）
mvn archetype:generate \
  -DgroupId=com.example \
  -DartifactId=myapp \
  -DarchetypeArtifactId=maven-archetype-quickstart \
  -DinteractiveMode=false

# 进入项目目录
cd myapp

# 编译
mvn compile
# 输出末尾看到 BUILD SUCCESS 就是成了

# 运行测试
mvn test

# 打包成jar
mvn package
# 会在 target/ 目录生成 myapp-1.0-SNAPSHOT.jar
```

生成的目录结构：

```
myapp/
├── pom.xml              ← Maven配置文件（定义依赖、插件）
└── src/
    ├── main/
    │   └── java/
    │       └── com/
    │           └── example/
    │               └── App.java   ← 主类
    └── test/
        └── java/
            └── com/
                └── example/
                    └── AppTest.java  ← 测试类
```

> 🤔 想多一点：Maven下载的依赖（jar包）存在哪里？在 `~/.m2/repository/`（用户目录下的 `.m2` 文件夹）。你可以在那里找到Spring Boot、MySQL驱动等所有下载过的jar包。如果你发现Maven下载特别慢，通常是因为默认从国外Maven中央仓库下载——换成阿里云镜像就快了。

## 05.4 常用终端命令速查

| 命令 | Windows PowerShell | Linux/macOS | 说明 |
|------|-------------------|-------------|------|
| 当前目录 | `pwd` 或 `Get-Location` | `pwd` | print working directory |
| 列出文件 | `ls` 或 `dir` | `ls` | list |
| 切换目录 | `cd 路径` | `cd 路径` | change directory |
| 创建目录 | `mkdir 名称` | `mkdir 名称` | make directory |
| 删除文件 | `rm 文件名` | `rm 文件名` | remove（小心！） |
| 复制文件 | `cp 源 目标` | `cp 源 目标` | copy |
| 移动文件 | `mv 源 目标` | `mv 源 目标` | move |

## 05.5 第一个Java命令行程序

好了，动手写你第一个Java程序。不用IDE，就用终端+记事本。

**1. 创建文件 `Hello.java`：**

```java
public class Hello {
    public static void main(String[] args) {
        System.out.println("你好，命令行！");
    }
}
```

**2. 打开终端，进入文件所在目录，执行：**

```bash
# 编译
javac Hello.java

# 查看——多了一个 Hello.class
ls
# Hello.java  Hello.class

# 运行
java Hello
# 输出: 你好，命令行！
```

**3. 试试故意搞错：**

```bash
# 把 .java 文件删掉，只剩 .class
rm Hello.java
java Hello
# 输出: 你好，命令行！
# 即使没有源码，.class也能运行！这就是"一次编译到处运行"的体现
```

> 🎉 恭喜！你刚才完成了Java程序员最经典的"仪式"——在命令行里编译并运行了第一个Java程序。后面所有的Spring Boot、微服务、分布式系统，本质上都是这个过程的放大版。
