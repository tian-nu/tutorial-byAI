# 第26章 · 内部类与枚举

> "函数里能定义局部变量，类里能不能内嵌一个'小类'？能——这叫内部类。再想一个问题：星期一到星期日、性别男和女、订单状态待付款/已付款/已发货……这些'固定取值'的常量怎么组织？枚举就是为这种场景设计的。"

---

## 26.1 内部类的四种类型

**内部类**（Inner Class。此术语需进附录）就是定义在类内部的类。Java有四种内部类：

| 类型 | 位置 | 能访问外部类的什么 | 是否需要外部类实例 |
|------|------|-------------------|--------------------|
| 成员内部类 | 类内部（字段级） | 所有（含private） | 是 |
| 静态内部类 | 类内部，带static | 只能访问静态成员 | 否 |
| 局部内部类 | 方法内部 | 局部变量+所有外部类成员 | 是 |
| 匿名内部类 | 表达式位置 | 局部变量+所有外部类成员 | 是 |

---

## 26.2 静态内部类（最常用）

静态内部类用 `static` 修饰，**不依赖外部类实例，可以独立使用**。这是最常用的一种，因为它简单。

```java
public class Outer {
    private static String msg = "Hello";

    // 静态内部类
    public static class Inner {
        public void print() {
            System.out.println(msg);  // 可以访问外部类的静态成员
            // System.out.println(Outer.this.name);  // ❌ 不能访问实例成员
        }
    }
}

// 使用：不需要创建Outer实例
Outer.Inner inner = new Outer.Inner();
inner.print();  // Hello
```

**最常见的应用场景**：工具类里放Builder（建造者模式）：

```java
public class User {
    private String name;
    private int age;

    private User(Builder builder) {
        this.name = builder.name;
        this.age = builder.age;
    }

    // 静态内部类Builder
    public static class Builder {
        private String name;
        private int age;

        public Builder name(String name) { this.name = name; return this; }
        public Builder age(int age) { this.age = age; return this; }
        public User build() { return new User(this); }
    }
}

// 使用
User user = new User.Builder()
    .name("张三")
    .age(25)
    .build();
```

## 26.3 成员内部类

成员内部类（不带static）**必须通过外部类实例才能创建**：

```java
public class Outer {
    private String name = "外部";

    public class Inner {
        public void print() {
            System.out.println("我是内部类，我的外部类是：" + name);
            System.out.println("引用方式：" + Outer.this.name);  // 显式引用外部类
        }
    }
}

// 使用：必须先创建外部类实例
Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();  // 注意语法：outer.new Inner()
inner.print();
```

> ❌ **常见错误**：试图 `new Outer.Inner()`——编译器报错"No enclosing instance"。成员内部类必须先有外部类对象才能创建。

## 26.4 匿名内部类：一次性使用

**匿名内部类**（Anonymous Inner Class。此术语需进附录）没有名字，直接在代码里定义并实例化。通常用来简写接口的实现：

```java
// 不用匿名内部类：要单独定义一个类
class MyRunnable implements Runnable {
    @Override
    public void run() {
        System.out.println("执行任务");
    }
}
Runnable r = new MyRunnable();

// 用匿名内部类：就在使用的地方直接写
Runnable r = new Runnable() {
    @Override
    public void run() {
        System.out.println("执行任务");
    }
};

// 匿名内部类也可以重写父类的方法
Animal dog = new Animal() {
    @Override
    public void makeSound() {
        System.out.println("汪汪汪");
    }
};
```

匿名内部类在Java 8之前大量用于事件监听。Lambda表达式出现后，函数式接口的场景大多改用Lambda。

## 26.5 枚举（enum）：有名字的常量集合

**枚举**（Enum。此术语需进附录）是Java 5引入的特殊类型，用于定义**有限的、固定的一组常量**。

### 最基础的枚举

```java
public enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}

// 使用
Day today = Day.MONDAY;
System.out.println(today);  // MONDAY

// 通用的方法
Day[] values = Day.values();                // 获取所有枚举值
Day monday = Day.valueOf("MONDAY");         // 根据字符串获取枚举
int ordinal = Day.WEDNESDAY.ordinal();      // 获取序号（0开始）
String name = Day.FRIDAY.name();            // 获取名字
```

### 带字段和方法的枚举

枚举不只是常量，它可以有字段、构造器和方法：

```java
public enum Season {
    SPRING("春天", "温暖"),
    SUMMER("夏天", "炎热"),
    AUTUMN("秋天", "凉爽"),
    WINTER("冬天", "寒冷");

    private final String chineseName;
    private final String description;

    // 枚举构造器默认就是private，只能内部使用
    Season(String chineseName, String description) {
        this.chineseName = chineseName;
        this.description = description;
    }

    public String getChineseName() { return chineseName; }
    public String getDescription() { return description; }
}

// 使用
Season s = Season.SUMMER;
System.out.println(s.getChineseName());   // 夏天
System.out.println(s.getDescription());   // 炎热
```

### 枚举搭配switch

```java
public enum OrderStatus {
    PENDING, PAID, SHIPPED, DELIVERED, CANCELLED
}

OrderStatus status = OrderStatus.PAID;
String action = switch (status) {
    case PENDING -> "请付款";
    case PAID -> "准备发货";
    case SHIPPED -> "已发货，等待收货";
    case DELIVERED -> "已签收";
    case CANCELLED -> "已取消";
};
System.out.println(action);  // 准备发货
```

### 为什么用枚举而不是常量？

```java
// 方案一：用int常量（不推荐）
public static final int STATUS_PENDING = 0;
public static final int STATUS_PAID = 1;
// 问题：int类型不限制取值范围，可以传999。没有命名空间。

// 方案二：用String常量（不推荐）
public static final String STATUS_PENDING = "PENDING";
// 问题：容易写错，大小写、拼写错误编译器不会提示。

// 方案三：用枚举（推荐）
public enum Status { PENDING, PAID }
// 优势：类型安全、编译器检查、IDE自动补全、限定取值
```

## 26.6 record：数据类一行搞定（Java 14+）

`record`（此术语需进附录）是Java 14引入的新类型，用于创建**不可变的数据类**。之前写一个简单的数据类要写几十行（字段、构造器、equals、hasHCode、toString），用record一行就够了：

```java
// 传统写法：约40行
public class Point {
    private final int x;
    private final int y;

    public Point(int x, int y) { this.x = x; this.y = y; }
    public int x() { return x; }
    public int y() { return y; }
    public boolean equals(Object o) { ... }
    public int hashCode() { ... }
    public String toString() { ... }
}

// record写法：1行！
public record Point(int x, int y) { }
```

record自动生成：
- 所有字段的**Getter**（注意：叫 `x()` 而不是 `getX()`）
- `equals()`、`hashCode()`、`toString()`
- 一个**全参构造器**（叫正则构造器）

```java
Point p = new Point(3, 4);
System.out.println(p.x());   // 3  （注意是x()不是getX()）
System.out.println(p.y());   // 4
System.out.println(p);       // Point[x=3, y=4]  （自动toString）

// record是不可变的——没有setter
// p.x = 5;  // ❌ 编译错误！
```

### record可以添加额外构造器和方法

```java
public record Person(String name, int age) {
    // 紧凑构造器（验证参数）
    public Person {
        if (age < 0) {
            throw new IllegalArgumentException("年龄不能为负");
        }
    }

    // 额外方法
    public boolean isAdult() {
        return age >= 18;
    }
}
```

## 本章小结

| 知识点 | 核心要点 |
|--------|----------|
| 静态内部类 | `static class`，不依赖外部实例，最常用。常见于Builder模式 |
| 成员内部类 | 非static，需要 `outer.new Inner()`。能访问外部类的private成员 |
| 匿名内部类 | 没名字，直接在new处定义。简化接口实现。常被Lambda替代 |
| 枚举 | `enum`定义有限常量集。可带字段/构造器/方法。比int/String常量更安全 |
| record | Java 14+。一行定义不可变数据类。自动生成Getter/equals/hashCode/toString |
