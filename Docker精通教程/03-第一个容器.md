## 我在做什么？

你装好了 Docker，现在该真正用它了。这一章，你会执行你的第一条 `docker run` 命令，亲眼看到一个容器从无到有、从运行到停止的全过程。

更重要的是，你会理解 `docker run` 背后到底发生了什么——不是"魔法"，而是可以拆解成一步步的机械过程。

学完这一章，你能：
- 用 `docker run` 启动一个容器
- 进入容器内部，像操作一台 Linux 一样操作它
- 查看容器的状态（运行中 / 已停止）
- 理解 `docker run hello-world` 背后发生了什么

---

## 一、你的第一个容器：Hello World

### 执行命令

打开终端（PowerShell / Terminal），输入：

```bash
docker run hello-world
```

### 你会看到什么

```
Unable to find image 'hello-world:latest' locally
latest: Pulling from library/hello-world
c1ec31eb5944: Pull complete
Digest: sha256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
Status: Downloaded newer image for hello-world:latest

Hello from Docker!
This message shows that your installation appears to be working correctly.

To generate this message, Docker took the following steps:
 1. The Docker client contacted the Docker daemon.
 2. The Docker daemon pulled the "hello-world" image from the Docker Hub.
 3. The Docker daemon created a new container from that image which runs the
    executable that produces the output you are currently reading.
 4. The Docker daemon streamed that output to the Docker client, which sent it
    to your terminal.
```

### 背后发生了什么？—— 一步步拆解

Docker 的输出其实已经告诉你答案了，但让我们把它翻译成"人话"：

```
你敲下 docker run hello-world
           │
           ▼
┌─────────────────────────────────────────────┐
│ 步骤 1：Docker 客户端找到 Docker 守护进程     │
│         (docker CLI → dockerd)              │
│         告诉它："我要跑 hello-world 镜像"     │
└──────────────────────┬──────────────────────┘
                       ▼
┌─────────────────────────────────────────────┐
│ 步骤 2：Docker 守护进程检查本地有没有这个镜像  │
│         本地没有 → 去 Docker Hub 找          │
│         本地有   → 跳过拉取，直接创建容器      │
└──────────────────────┬──────────────────────┘
                       ▼
┌─────────────────────────────────────────────┐
│ 步骤 3：从 Docker Hub 拉取镜像               │
│         Pulling from library/hello-world    │
│         下载到本地，存起来                    │
└──────────────────────┬──────────────────────┘
                       ▼
┌─────────────────────────────────────────────┐
│ 步骤 4：用这个镜像创建一个容器               │
│         分配资源、设置隔离环境                │
└──────────────────────┬──────────────────────┘
                       ▼
┌─────────────────────────────────────────────┐
│ 步骤 5：运行容器里的程序                     │
│         hello-world 镜像里只有一个可执行程序   │
│         它打印 Hello from Docker! 然后退出    │
└──────────────────────┬──────────────────────┘
                       ▼
┌─────────────────────────────────────────────┐
│ 步骤 6：程序退出 → 容器自动停止              │
│         容器还在（状态变为 Exited），           │
│         但里面的程序已经运行完了              │
└─────────────────────────────────────────────┘
```

> **想多一点**：你可能会好奇："为什么镜像叫 `hello-world`，不是 `ubuntu` 或 `nginx`？"`hello-world` 是 Docker 官方维护的一个**最小镜像**，里面只有一个几 KB 的可执行程序，唯一的功能就是打印一段欢迎信息然后退出。它的存在就是为了验证"你的 Docker 安装成功了"。它不能做任何别的事，它甚至连 shell 都没有。

## 二、真正的容器：进入 Ubuntu 内部

`hello-world` 只是"撒个花就跑"，你想要的是一个能真正交互的容器。接下来，我们启动一个 Ubuntu 容器，**进入它的内部**，像操作一台 Linux 虚拟机一样操作它。

### 启动一个交互式 Ubuntu 容器

```bash
docker run -it ubuntu bash
```

你会看到终端提示符从你的电脑变成了容器内部：

```
root@a1b2c3d4e5f6:/#
```

恭喜——你现在正站在一个 Ubuntu 容器的"肚子"里。

### 你在里面能做什么？

```bash
# 看看当前系统信息
cat /etc/os-release
# 输出：NAME="Ubuntu" VERSION="22.04.x LTS" ...

# 看看当前用户是谁
whoami
# 输出：root

# 更新软件包列表
apt update

# 装一个工具试试
apt install -y curl

# 用 curl 请求一个网页
curl http://example.com
```

你在容器里做的任何操作，都和在一台真实的 Ubuntu 机器上一样。但这一切，只占用了你宿主机几 MB 的内存。

### 退出容器

```bash
exit
```

或者按 `Ctrl + D`。

退出后，终端提示符回到你的宿主机的样子。容器停止了。

## 三、`docker run -it` 到底是什么意思？

上一节你用了 `docker run -it ubuntu bash`，每个部分都有含义：

```bash
docker run -it ubuntu bash
│      │   │   │      │
│      │   │   │      └── 在容器里执行的命令（启动 bash shell）
│      │   │   └── 镜像名（ubuntu）
│      │   └── 两个选项的组合
│      └── docker 的子命令（创建并启动容器）
└── Docker 客户端
```

### `-i` 和 `-t` 分别是什么？

| 选项 | 全称 | 作用 | 不写会怎样 |
|------|------|------|-----------|
| `-i` | `--interactive` | 保持 STDIN（标准输入）打开 | 你无法在容器里输入任何东西 |
| `-t` | `--tty` | 分配一个伪终端（pseudo-TTY）*此术语见附录C* | 你看到的输出没有格式、没有颜色、没有提示符 |

**两个必须一起用。** `-i` 让你能说话，`-t` 让容器能"好好说话"（有格式的输出）。

### 忘记 `-it` 会怎样？—— 经典错误演示

❌ 错误做法：

```bash
docker run ubuntu bash
```

你敲下回车，什么都没有发生。终端一闪，容器已经退出。

**为什么？** 因为你没有 `-i`，bash 的标准输入没有连接到你键盘，bash 发现自己"没人跟我说话"，于是退出。没有 `-t`，就算有输出也没格式。

✅ 正确做法：

```bash
docker run -it ubuntu bash
```

## 四、查看容器：`docker ps` 和 `docker ps -a`

### 查看正在运行的容器

```bash
docker ps
```

输出示例：

```
CONTAINER ID   IMAGE     COMMAND   CREATED         STATUS         PORTS     NAMES
a1b2c3d4e5f6   ubuntu    "bash"    2 minutes ago   Up 2 minutes             quirky_bose
```

列的含义：

| 列名 | 含义 |
|------|------|
| CONTAINER ID | 容器的唯一 ID（前 12 位） |
| IMAGE | 创建这个容器用的镜像 |
| COMMAND | 容器启动时执行的命令 |
| CREATED | 容器创建时间 |
| STATUS | 当前状态（Up = 运行中，Exited = 已停止） |
| PORTS | 端口映射（后面章节会讲） |
| NAMES | 容器的名字（不指定时 Docker 自动生成一个随机名字） |

### 查看所有容器（包括已停止的）

```bash
docker ps -a
```

加上 `-a`（all），你会看到你的 `hello-world` 和 `ubuntu` 容器都在列表里，状态是 `Exited`。

> 刚才的 `hello-world` 容器去哪了？它还在！`docker ps -a` 就能看到。容器停止后不会自动删除，除非你加了 `--rm` 选项。

### 删除已经停止的容器

```bash
# 删除指定容器
docker rm quirky_bose
# 或者用容器 ID
docker rm a1b2c3d4e5f6
```

## 五、启动已停止的容器

容器停止后，你不需要重新 `docker run`。你可以用 `docker start` 把它重新启动：

```bash
# 先看看有哪些已停止的容器
docker ps -a

# 假设你有一个名为 quirky_bose 的容器
docker start quirky_bose
```

但 `docker start` 默认是**后台启动**，你不会自动进入容器内部。要重新进入交互模式：

```bash
# 启动容器，然后附加到它的终端
docker start -ai quirky_bose
```

或者分两步：

```bash
docker start quirky_bose    # 启动容器（后台）
docker attach quirky_bose   # 附加到容器的终端
```

> `docker run` = `docker create`（创建容器）+ `docker start`（启动容器）+ `docker attach`（附加终端）。`docker run` 是一条龙服务，`docker start` 只负责重新启动一个已经创建好的容器。

---

## 六、`docker run` 的常用选项速览

你已经学了两个关键选项，这里再补充几个后面会经常用到的：

| 选项 | 全称 | 作用 | 示例 |
|------|------|------|------|
| `-i` | `--interactive` | 保持标准输入打开 | `docker run -i ubuntu` |
| `-t` | `--tty` | 分配伪终端 | `docker run -t ubuntu` |
| `-d` | `--detach` | 后台运行 | `docker run -d nginx` |
| `--name` | — | 给容器起个名字 | `docker run --name my-ubuntu ubuntu` |
| `--rm` | — | 容器退出后自动删除 | `docker run --rm -it ubuntu bash` |
| `-p` | `--publish` | 端口映射 | `docker run -p 8080:80 nginx` |
| `-v` | `--volume` | 挂载数据卷 | `docker run -v /data:/app/data ubuntu` |

> 现在不用全部记住。`-it`、`-d`、`--name` 这三个先记牢，后面的章节会逐一展开讲 `-p`、`-v` 等。

## 我做得对不对？

### 验证方法

请按顺序执行以下操作，每一步都成功才算通过：

```bash
# 1. 启动一个交互式 Ubuntu 容器
docker run -it ubuntu bash

# 2. 在容器里执行：更新软件包列表
apt update

# 3. 在容器里执行：输出系统信息
cat /etc/os-release

# 4. 退出容器
exit

# 5. 查看所有容器（包括已停止的）
docker ps -a

# 6. 找到你刚才的 ubuntu 容器，确认它的状态是 "Exited"
```

**预期结果**：
- 步骤 2：`apt update` 成功从软件源获取更新信息（不需要全部成功，有网络响应就行）
- 步骤 3：输出包含 `NAME="Ubuntu"` 和版本号
- 步骤 5-6：`docker ps -a` 的输出中能看到你的 ubuntu 容器，`STATUS` 列显示 `Exited`

## 不对怎么办？

### 常见错误 1：`docker run ubuntu` 进去又马上退出

❌ 错误做法：

```bash
docker run ubuntu
```

或者：

```bash
docker run ubuntu bash
```

✅ 正确做法：

```bash
docker run -it ubuntu bash
```

**原因**：容器需要一个**前台进程**来维持运行。如果你不加 `-it`，bash 启动后检测到没有终端连接，就退出了。bash 一退出，容器的主进程就没了，容器也就停止了。

> 这是一个非常核心的概念：**容器的生命周期 = 它的主进程（PID 1）的生命周期**。主进程退出，容器就停止。`hello-world` 镜像的主进程是打印消息的可执行程序，打印完就退出，所以容器只活了 1 秒。`nginx` 镜像的主进程是 Web 服务器，它会一直运行，所以 `nginx` 容器可以一直运行。

### 常见错误 2：`docker start` 后看不到容器

```bash
docker start quirky_bose
# 终端没有进入容器，也没看到输出
```

这是正常的。`docker start` 默认以后台模式启动容器，不会自动附加到终端。你需要在后面加 `-a`（attach）：

```bash
docker start -ai quirky_bose
```

### 常见错误 3：`apt update` 在容器里失败

```
Err:1 http://archive.ubuntu.com/ubuntu jammy InRelease
  Could not connect to archive.ubuntu.com:80
```

**原因**：容器内的网络可能有问题（尤其是 Docker Desktop 初次启动时）。尝试以下方法：

```bash
# 方法 1：重启 Docker Desktop
# 方法 2：检查宿主机能否上网
ping google.com
# 方法 3：如果宿主机能上网但容器不行，重启 Docker 服务
# Linux 上：
sudo systemctl restart docker
```

### 常见错误 4：删除容器提示"容器正在运行"

```bash
docker rm quirky_bose
# Error: You cannot remove a running container
```

**原因**：不能删除正在运行的容器。先停再删：

```bash
docker stop quirky_bose
docker rm quirky_bose
```

或者强制删除（不推荐，除非你确定）：

```bash
docker rm -f quirky_bose
```

### 常见错误 5：镜像拉取慢导致 `docker run ubuntu` 卡住

和 02 章一样，你需要配置镜像加速器。如果还没配，翻回 02 章的"排错预案 2"。

## 本章小结

| 本章学了什么 | 对应命令/概念 | 注意事项 |
|-------------|-------------|---------|
| 第一个容器 | `docker run hello-world` | 验证安装的经典命令，Docker 输出里已经写了背后的步骤 |
| `docker run` 背后的 6 步 | 客户端 → 守护进程 → 检查本地 → 拉取 → 创建容器 → 运行 | 理解这个流程，排错时才知道卡在哪一步 |
| 交互式容器 | `docker run -it ubuntu bash` | `-i` 保持输入 + `-t` 分配伪终端，必须一起用 |
| 退出容器 | `exit` 或 `Ctrl+D` | 退出后容器停止，但不会自动删除 |
| 查看容器 | `docker ps`（运行中）/ `docker ps -a`（全部） | 每个容器有唯一 ID 和随机名字 |
| 启动已停止的容器 | `docker start` / `docker start -ai` | `docker start` 默认后台，加 `-ai` 才进入交互 |
| 删除容器 | `docker rm` 容器名 | 不能删除正在运行的容器，先 `docker stop` |
| 容器生命周期 | 容器 = 主进程，主进程退出 = 容器停止 | 这是 Docker 最核心的运行时原理，理解它排错事半功倍 |
| 常用选项 | `-it`、`-d`、`--name`、`--rm` | 先记 `-it` 和 `--name`，后面章节逐步展开其他 |

## 本篇最可能出错的地方及原因

### 1. 忘记 `-it` 导致容器一闪而过（最常见的坑）

**原因**：新手最容易犯的错误。不加 `-it`，bash 启动后没有终端连接，检测到不是交互环境就自动退出。容器的主进程退出 = 容器停止。

**排查**：`docker ps -a` 看到容器状态是 `Exited (0)`，说明正常退出，不是你环境有问题，只是你忘了加 `-it`。

### 2. 镜像拉取卡住或超时（中国大陆用户必遇）

**原因**：Docker Hub 的服务器在海外，国内直连速度极慢甚至超时。

**排查**：`docker pull ubuntu` 或 `docker run ubuntu` 长时间没有进度条，或者报 `net/http: TLS handshake timeout`。

**解决**：配置国内镜像加速器（见 02 章排错预案 2）。这是中国大陆用户安装 Docker 后**必须做的一步**，否则后续所有拉取镜像的操作都会卡住。

### 3. Linux 用户忘记 `usermod -aG docker` 后重新登录

**原因**：执行 `sudo usermod -aG docker $USER` 后，组权限只在**登录时**加载。如果不重新登录，`docker` 命令仍然提示 `permission denied`。

**排查**：执行 `groups` 命令，看输出里有没有 `docker` 组。如果没有，说明你没重新登录。

**解决**：注销当前会话，重新登录（或 `newgrp docker` 临时生效）。

### 4. 把 `docker run` 和 `docker start` 搞混

**原因**：`docker run` = 创建新容器 + 启动；`docker start` = 启动已存在的容器。新手容易每次都 `docker run`，导致创建了一堆重复容器。

**排查**：`docker ps -a` 看到一堆同名的容器，状态都是 `Exited`。

**解决**：用 `docker start -ai 容器名` 重新进入已存在的容器，而不是每次都 `docker run`。用 `docker rm` 清理多余的已停止容器。
