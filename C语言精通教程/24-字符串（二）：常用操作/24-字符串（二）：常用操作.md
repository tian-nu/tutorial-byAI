## 我在做什么？

上一章你知道了 C 语言字符串的本质——以 `\0` 结尾的 `char` 数组。但光知道本质还不够，你得会操作它：复制、拼接、比较、查找。

C 语言没有 `.copy()` 或 `.concat()` 这样的方法。一切操作都靠 `<string.h>` 里的函数。这些函数用起来不难，但背后藏着 C 语言最著名的安全漏洞：*缓冲区溢出* *此术语见附录F*。读完这一章，你会安全地使用 `strcpy`、`strcat`、`strcmp`、`strchr`、`strstr`，并理解为什么它们都"不安全"版本和"安全"版本。

## 一、字符串复制：`strcpy` 与 `strncpy`

### 1.1 `strcpy`：复制字符串

`strcpy` 把源字符串的内容（包括 `\0`）复制到目标数组中。

```c
#include <stdio.h>
#include <string.h>

int main() {
    char src[] = "Hello";
    char dest[20];                    // 目标数组必须足够大

    strcpy(dest, src);                // 把 src 复制到 dest
    printf("dest = %s\n", dest);      // 输出 Hello
    printf("src  = %s\n", src);       // 输出 Hello（src 没变）

    return 0;
}
```

```text
strcpy(dest, src) 的工作过程：

src:
┌───┬───┬───┬───┬───┬───┐
│ H │ e │ l │ l │ o │ \0│
└───┴───┴───┴───┴───┴───┘
  ↓   ↓   ↓   ↓   ↓   ↓   逐个复制
┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐
│ H │ e │ l │ l │ o │ \0│ ? │ ? │ ? │ ? │ ? │ ? │ ? │ ? │ ? │ ? │ ? │ ? │ ? │ ? │
└───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘
dest
```

### 1.2 `strcpy` 的危险：不检查目标大小

`strcpy` 不会检查目标数组是否够大。如果源比目标长，它照样复制——结果就是越界写入。

```c
char short_buf[5];             // 只有 5 个字节
strcpy(short_buf, "Hello, World!");  // 源有 14 个字节！灾难！
```

```text
short_buf 只有 5 字节，但 strcpy 写了 14 字节：

┌────┬────┬────┬────┬────┬────┬────┬────┬────┬────┬────┬────┬────┬────┐
│ H  │ e  │ l  │ l  │ o  │ ,  │    │ W  │ o  │ r  │ l  │ d  │ !  │ \0 │
└────┴────┴────┴────┴────┴────┴────┴────┴────┴────┴────┴────┴────┴────┘
  ←── short_buf 的合法范围 ──→   ←── 越界写入，覆盖了其他变量的内存 ──→
```

这就是 *缓冲区溢出* *此术语见附录F*——C 语言最臭名昭著的安全漏洞，也是无数黑客攻击的入口。

### 1.3 `strncpy`：安全版本

`strncpy` 多了一个参数 `n`：最多复制 `n` 个字符。

```c
#include <stdio.h>
#include <string.h>

int main() {
    char src[] = "Hello, World!";
    char dest[6];                       // 只有 6 个字节

    strncpy(dest, src, sizeof(dest) - 1);  // 最多复制 5 个字符
    dest[sizeof(dest) - 1] = '\0';         // 手动加 \0（重要！）

    printf("dest = %s\n", dest);        // 输出 Hello
    return 0;
}
```

**`strncpy` 的陷阱**：如果 `n` 小于源长度，`strncpy` 不会自动加 `\0`。你必须手动加！

```c
char dest[5];
strncpy(dest, "Hello World", 5);
// dest 现在是 {'H','e','l','l','o'}，没有 \0！
// printf("%s", dest);  危险！
```

## 二、字符串拼接：`strcat` 与 `strncat`

### 2.1 `strcat`：拼接字符串

`strcat` 把源字符串追加到目标字符串的末尾（从目标的 `\0` 位置开始复制）。

```c
#include <stdio.h>
#include <string.h>

int main() {
    char dest[30] = "Hello";     // 目标有足够空间
    char src[] = " World";

    strcat(dest, src);            // 把 src 追加到 dest 后面
    printf("%s\n", dest);         // 输出 Hello World
    return 0;
}
```

```text
strcat(dest, src) 的工作过程：

执行前：
dest: ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐
      │ H │ e │ l │ l │ o │ \0│ ? │ ? │ ? │ ? │ ? │ ? │ ? │
      └───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘
                              ↑
                         从这个 \0 开始覆盖

执行后：
dest: ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐
      │ H │ e │ l │ l │ o │   │ W │ o │ r │ l │ d │ \0│ ? │
      └───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘
```

### 2.2 `strcat` 的危险：同样不检查大小

如果目标数组不够大，`strcat` 照样越界写入。

### 2.3 `strncat`：安全版本

`strncat` 多一个参数 `n`：最多追加 `n` 个字符。**它总是自动加 `\0`**（比 `strncpy` 好）。

```c
#include <stdio.h>
#include <string.h>

int main() {
    char dest[15] = "Hello";

    // 最多追加 7 个字符（留 1 个给 \0，所以 dest 总大小 15 够用）
    strncat(dest, " Beautiful World", 7);
    printf("%s\n", dest);  // 输出 Hello Beauti
    return 0;
}
```

## 三、字符串比较：`strcmp` 与 `strncmp`

### 3.1 为什么不能用 `==`

字符串是数组，数组名是指针。`==` 比较的是两个指针的地址，不是内容。

```c
char s1[] = "abc";
char s2[] = "abc";
if (s1 == s2) {           // 比较的是地址，永远不相等！
    printf("相等\n");
}
```

### 3.2 `strcmp`：比较字符串内容

`strcmp` 逐个字符比较两个字符串的 ASCII 码值。

```c
#include <stdio.h>
#include <string.h>

int main() {
    printf("%d\n", strcmp("abc", "abc"));   // 输出 0（相等）
    printf("%d\n", strcmp("abc", "abd"));   // 输出负数（abc < abd）
    printf("%d\n", strcmp("abd", "abc"));   // 输出正数（abd > abc）
    printf("%d\n", strcmp("abc", "ab"));    // 输出正数（abc > ab）
    printf("%d\n", strcmp("AB", "ab"));     // 输出负数（A=65, a=97）
    return 0;
}
```

**返回值规则**：
- 返回 `0`：两个字符串相等。
- 返回负数：第一个字符串"小于"第二个。
- 返回正数：第一个字符串"大于"第二个。

"大小"不是长度，而是**字典序**（按 ASCII 码逐个比较）。

```text
"abc" vs "abd"：
  a==a, b==b, c(99) < d(100) → "abc" < "abd"

"abc" vs "ab"：
  a==a, b==b, c(99) vs \0(0) → c > \0 → "abc" > "ab"
```

### 3.3 `strncmp`：比较前 N 个字符

```c
#include <stdio.h>
#include <string.h>

int main() {
    printf("%d\n", strncmp("hello", "help", 3));    // 输出 0（前 3 个字符相同）
    printf("%d\n", strncmp("hello", "help", 4));    // 输出负数（第 4 个字符 l < p）
    return 0;
}
```

## 四、字符串查找：`strchr` 与 `strstr`

### 4.1 `strchr`：查找字符

`strchr` 在字符串中查找一个字符，返回该字符**第一次出现的位置**（指针）。如果找不到，返回 `NULL`。

```c
#include <stdio.h>
#include <string.h>

int main() {
    char str[] = "Hello, World!";
    char *pos = strchr(str, 'W');   // 查找 'W'

    if (pos != NULL) {
        printf("找到了 'W'，位置索引 = %ld\n", pos - str);  // 输出 7
        printf("从 'W' 开始：%s\n", pos);                  // 输出 World!
    } else {
        printf("没找到\n");
    }

    return 0;
}
```

```text
strchr(str, 'W') 的工作过程：

str: ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐
     │ H │ e │ l │ l │ o │ , │   │ W │ o │ r │ l │ d │ ! │
     └───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘
       ↑   ↑   ↑   ↑   ↑   ↑   ↑   ↑
       检查 检查 检查 检查 检查 检查 检查 找到了！返回这个地址
```

### 4.2 `strstr`：查找子字符串

`strstr` 在字符串中查找一个子字符串，返回子字符串**第一次出现的位置**。

```c
#include <stdio.h>
#include <string.h>

int main() {
    char str[] = "The quick brown fox jumps over the lazy dog";
    char *pos = strstr(str, "fox");   // 查找 "fox"

    if (pos != NULL) {
        printf("找到了 'fox'，位置索引 = %ld\n", pos - str);  // 输出 16
        printf("从 'fox' 开始：%s\n", pos);  // 输出 fox jumps over the lazy dog
    } else {
        printf("没找到\n");
    }

    return 0;
}
```

### 4.3 查找所有出现位置

```c
#include <stdio.h>
#include <string.h>

int main() {
    char str[] = "abcabcabc";
    char *pos = str;
    int count = 0;

    while ((pos = strstr(pos, "abc")) != NULL) {
        count++;
        printf("第 %d 次出现，位置 = %ld\n", count, pos - str);
        pos++;  // 往后移一位，继续查找
    }
    printf("共找到 %d 次\n", count);

    return 0;
}
```

## 五、其他常用字符串函数

### 5.1 `sprintf`：格式化输出到字符串

```c
#include <stdio.h>

int main() {
    char buffer[100];
    int age = 25;
    double height = 1.75;

    sprintf(buffer, "年龄：%d，身高：%.2f 米", age, height);
    printf("%s\n", buffer);  // 输出 年龄：25，身高：1.75 米
    return 0;
}
```

### 5.2 `sscanf`：从字符串格式化读取

```c
#include <stdio.h>

int main() {
    char input[] = "25 1.75";
    int age;
    double height;

    sscanf(input, "%d %lf", &age, &height);
    printf("年龄：%d，身高：%.2f\n", age, height);  // 输出 年龄：25，身高：1.75
    return 0;
}
```

### 5.3 `strtok`：分割字符串

```c
#include <stdio.h>
#include <string.h>

int main() {
    char str[] = "apple,banana,orange,grape";
    char *token = strtok(str, ",");       // 第一次调用，传字符串

    while (token != NULL) {
        printf("%s\n", token);
        token = strtok(NULL, ",");        // 后续调用，传 NULL
    }

    return 0;
}
```

输出：
```text
apple
banana
orange
grape
```

**注意**：`strtok` 会修改原字符串（把分隔符替换为 `\0`），不是线程安全的。

## 六、缓冲区溢出：C语言的安全噩梦

### 6.1 什么是缓冲区溢出

当程序向一个固定大小的缓冲区写入超过其容量的数据时，多余的数据会覆盖相邻内存。这就是缓冲区溢出。

```c
#include <stdio.h>
#include <string.h>

int main() {
    char password[8] = "secret";   // 密码
    char input[8];                  // 用户输入缓冲区

    printf("请输入：");
    gets(input);   // 危险！gets 不检查边界，C11 已移除

    // 如果用户输入超过 7 个字符，会覆盖 password
    printf("password = %s\n", password);
    return 0;
}
```

### 6.2 历史上的缓冲区溢出攻击

1988 年，Morris 蠕虫利用 Unix 系统 `fingerd` 服务的缓冲区溢出漏洞，感染了当时互联网上约 10% 的计算机。这是历史上第一个引起广泛关注的缓冲区溢出攻击。

此后，缓冲区溢出一直是黑客攻击的主要手段之一。Windows 的"蓝屏死机"、Linux 的"段错误"，很多都源于此。

### 6.3 安全函数速查表

| 危险函数 | 安全替代 | 关键区别 |
|----------|----------|----------|
| `gets()` | `fgets()` | `fgets` 有大小限制 |
| `strcpy()` | `strncpy()` | `strncpy` 限制复制长度 |
| `strcat()` | `strncat()` | `strncat` 限制追加长度 |
| `sprintf()` | `snprintf()` | `snprintf` 限制输出长度 |
| `scanf("%s")` | `fgets()` 或 `scanf("%19s")` | 指定宽度 |

## 七、常见错误与排错

### 7.1 目标数组不够大

❌ 错误示例：
```c
char dest[5] = "Hi";
strcat(dest, " there");  // dest 只有 5 字节，放不下 "Hi there"
```

✅ 正确示例：
```c
char dest[20] = "Hi";
strncat(dest, " there", sizeof(dest) - strlen(dest) - 1);
```

### 7.2 `strncpy` 忘记加 `\0`

❌ 错误示例：
```c
char dest[5];
strncpy(dest, "Hello World", 5);  // dest 没有 \0
printf("%s", dest);                // 危险！
```

✅ 正确示例：
```c
char dest[5];
strncpy(dest, "Hello World", sizeof(dest) - 1);
dest[sizeof(dest) - 1] = '\0';     // 手动加 \0
```

### 7.3 `strcmp` 返回值当作布尔值

❌ 错误示例：
```c
if (strcmp(s1, s2)) {           // 如果相等，返回 0（假）；不相等反而进 if
    printf("相等\n");           // 逻辑反了！
}
```

✅ 正确示例：
```c
if (strcmp(s1, s2) == 0) {     // 明确检查是否等于 0
    printf("相等\n");
}
```

### 7.4 字符串用 `==` 比较

❌ 错误示例：
```c
char *s = get_input();
if (s == "yes") { ... }  // 比较的是地址
```

✅ 正确示例：
```c
if (strcmp(s, "yes") == 0) { ... }
```

## 我做得对不对？

用以下程序验证：

```c
#include <stdio.h>
#include <string.h>

int main() {
    // 测试1：strcpy
    char dest1[20];
    strcpy(dest1, "Hello");
    printf("测试1：%s (应为 Hello)\n", dest1);

    // 测试2：strncpy 安全复制
    char dest2[6];
    strncpy(dest2, "HelloWorld", sizeof(dest2) - 1);
    dest2[sizeof(dest2) - 1] = '\0';
    printf("测试2：%s (应为 Hello)\n", dest2);

    // 测试3：strcat
    char dest3[30] = "Hello";
    strcat(dest3, " World");
    printf("测试3：%s (应为 Hello World)\n", dest3);

    // 测试4：strcmp
    printf("测试4：strcmp(\"abc\",\"abc\") = %d (应为 0)\n", strcmp("abc", "abc"));
    printf("测试4：strcmp(\"abc\",\"abd\") < 0 (应为 1)\n", strcmp("abc", "abd") < 0);

    // 测试5：strchr
    char str5[] = "Hello World";
    char *p = strchr(str5, 'W');
    printf("测试5：'W'的位置 = %ld (应为 6)\n", p - str5);

    // 测试6：strstr
    char *q = strstr(str5, "World");
    printf("测试6：'World'的位置 = %ld (应为 6)\n", q - str5);

    return 0;
}
```

## 不对怎么办？

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| 程序崩溃或输出乱码 | 缓冲区溢出 | 使用安全版本函数，确保目标数组足够大 |
| `strncpy` 后打印出多余字符 | 忘记手动加 `\0` | `strncpy` 后手动加 `\0` |
| 字符串比较逻辑反了 | `strcmp` 相等返回 0 | 用 `== 0` 判断相等 |
| `strtok` 分割后原字符串变了 | `strtok` 会修改原字符串 | 如果需要保留原字符串，先复制一份 |
| 编译警告 `gets is dangerous` | 使用了 `gets` | 改用 `fgets` |

## 本章小结

| 知识点 | 一句话总结 |
|--------|-----------|
| `strcpy` | 复制字符串，不检查目标大小，危险 |
| `strncpy` | 限制长度的复制，但可能不加 `\0`，需手动处理 |
| `strcat` | 拼接字符串到末尾，不检查目标大小 |
| `strncat` | 限制长度的拼接，自动加 `\0` |
| `strcmp` | 比较两个字符串，相等返回 0 |
| `strncmp` | 比较前 N 个字符 |
| `strchr` | 查找字符第一次出现的位置 |
| `strstr` | 查找子字符串第一次出现的位置 |
| 缓冲区溢出 | 写入超过缓冲区容量的数据，覆盖相邻内存 |
| 安全函数 | 使用带 `n` 的版本（`strncpy`、`strncat`、`snprintf`） |
