## 三、组件 = 乐高积木

先建立直觉。

想象你要拼一艘乐高船。你不会从零捏塑料，而是用现成的积木块：船身、桅杆、船帆，一块一块拼起来。

Vue 组件就是这样的积木块。每个 `.vue` 文件就是一块积木，自带结构（HTML）、行为（JS）、样式（CSS）。搭页面 = 把组件拼在一起。

| 乐高世界 | Vue 世界 |
|---|---|
| 一块积木 | 一个 `.vue` 文件（组件） |
| 积木有形状、颜色、功能 | 组件有模板（结构）、脚本（逻辑）、样式（外观） |
| 拼积木 = 组装成船 | 拼组件 = 组装成页面 |
| 同一种积木可以复用多次 | 同一个组件可以在多个页面里用 |

> 🤔 **想多一点**：为什么非要用组件？因为真实项目里一个页面可能有几千行代码。如果不拆组件，所有代码堆在一个文件里，改一个小按钮要找半天。拆成组件后，每个文件只管自己的一亩三分地，出 Bug 了也知道去哪个文件找。

## 四、分步操作

### 步骤 1：SFC 三部分详解

> **我在做什么？** 深入理解 `.vue` 文件的三个区块分别怎么写。

每个 `.vue` 文件包含三个部分，各司其职：

```vue
<!-- ========== ① Template：写 HTML 结构 ========== -->
<template>
  <!-- 这里写的和普通 HTML 基本一样，但可以用 Vue 的模板语法增强 -->
  <div class="blog-header">
    <h1>{{ blogTitle }}</h1>
    <nav>
      <a href="/">首页</a>
      <a href="/about">关于</a>
    </nav>
  </div>
</template>

<!-- ========== ② Script setup：写 JS 逻辑 ========== -->
<script setup>
// 这里写数据、方法、计算属性、生命周期钩子...
// 顶层变量自动暴露给 template 使用
const blogTitle = '我的博客'
</script>

<!-- ========== ③ Style scoped：写 CSS 样式 ========== -->
<style scoped>
/* scoped 表示这里的样式只在这个组件里生效 */
.blog-header {
  background: #42b883;
  padding: 20px;
}
.blog-header h1 {
  color: white;
}
</style>
```

| 区块 | 写什么 | 对应传统开发 | 关键规则 |
|---|---|---|---|
| `<template>` | HTML 结构 | `.html` 文件 | 必须有**一个根元素**包裹所有内容 |
| `<script setup>` | JS 数据+逻辑 | `.js` 文件 | 顶层变量自动暴露给模板，无需 `return` |
| `<style scoped>` | CSS 样式 | `.css` 文件 | `scoped` 让样式只影响当前组件 |

      <a href="#">归档</a>

<style scoped>

  background: linear-gradient(135deg, #42b883, #35495e);
  color: white;
  padding: 20px 40px;
  display: flex;
  justify-content: space-between;
  align-items: center;

  color: white;
  text-decoration: none;
  margin-left: 20px;
}
.blog-header nav a:hover {
  text-decoration: underline;
}
</style>

> ✅ **做得对不对？** 文件创建在 `src/components/BlogHeader.vue`，路径和大小写完全一致。

> 🤔 **想多一点**：Vue 官方强烈建议组件名用**多单词**（如 `BlogHeader` 而不是 `Header`），因为 HTML 标准里已经有 `<header>`、`<footer>` 等元素，单单词组件名会跟原生元素冲突。驼峰命名 `BlogHeader` 在模板里可以写成 `<BlogHeader />` 或 `<blog-header />`，Vue 两种都认。

### 步骤 3：导入和使用组件

> **我在做什么？** 在 `App.vue` 中导入 `BlogHeader` 并显示它。

打开 `src/App.vue`，清空所有内容，写入：

```vue
<template>
  <div class="app">
    <BlogHeader />
  </div>
</template>

<script setup>
// 导入组件
import BlogHeader from './components/BlogHeader.vue'
</script>

<style scoped>
.app {
  min-height: 100vh;
}
</style>
```

**关键点**：`<script setup>` 里 `import` 的组件**自动注册**，可以直接在 `<template>` 里使用，不需要手动写 `components: { BlogHeader }`。这就是 `<script setup>` 的便利之处。

保存后看浏览器，你应该看到博客顶部的绿色导航栏了。

> ✅ **做得对不对？** 浏览器显示一个绿色渐变的长条，左边写着「🖊️ 我的技术博客」，右边有三个链接。

> ❌ **不对怎么办？**
>
> **错误：页面一片空白，控制台报 `Failed to resolve component`**
> - 原因 1：`import` 路径写错了。`./components/BlogHeader.vue` 的 `./` 不能省
> - 原因 2：文件名大小写不对。`BlogHeader.vue` 和 `blogheader.vue` 是不同的
> - 解决：逐字校对文件名和 import 路径

### 步骤 5：组件嵌套 —— 在 App.vue 中拼装

> **我在做什么？** 把三个组件在 `App.vue` 里像搭积木一样拼起来。

修改 `src/App.vue`：

```vue
<template>
  <div class="app">
    <BlogHeader />
    <BlogList />
    <BlogFooter />
  </div>
</template>

<script setup>
import BlogHeader from './components/BlogHeader.vue'
import BlogList from './components/BlogList.vue'
import BlogFooter from './components/BlogFooter.vue'
</script>

<style scoped>
.app {
  min-height: 100vh;
  background: #f5f5f5;
}
</style>
```

保存，看浏览器。你应该看到一个完整的博客页面：顶部导航 → 文章列表 → 底部页脚。

> ✅ **做得对不对？** 浏览器里三部分完整显示，布局从上到下依次是 Header、文章卡片列表、Footer。

这就是**组件嵌套**：`App.vue` 是根组件，里面嵌了三个子组件。子组件里以后还可以再嵌子组件，一层一层往下。整个应用就是一个组件树。

```
App.vue（根组件）
├── BlogHeader.vue（子组件）
├── BlogList.vue（子组件）
└── BlogFooter.vue（子组件）
```

> 🤔 **想多一点**：你会发现 `App.vue` 现在的代码非常「清爽」——它不关心 Header 长什么样、List 里有什么文章、Footer 写什么文字。它只负责「拼装」，具体细节由各自组件管。这就是**关注点分离**，前端开发的核心理念。

### 步骤 6：把 05 静态博客拆成 Vue 组件（练习）

> **我在做什么？** 回顾 05 教程里的静态博客，看看它和现在的 Vue 组件化有什么区别。

05 教程的静态博客是一个大 HTML 文件，所有代码堆在一起：

```
❌ 05 版本：一个 index.html 里塞了全部代码
┌──────── index.html ──────────┐
│  <header> ... </header>      │
│  <main> ... </main>          │
│  <footer> ... </footer>      │
│  <style> ... </style>        │
│  <script> ... </script>      │
└──────────────────────────────┘
```

现在拆成了 Vue 组件：

```
✅ Vue 版本：每个组件一个文件，各自独立
src/
├── App.vue           → 只管拼装
├── components/
│   ├── BlogHeader.vue → 只管导航栏
│   ├── BlogList.vue   → 只管文章列表
│   └── BlogFooter.vue → 只管页脚
```

**如果你要改页脚的版权文字**：
- 05 版本：在 `index.html` 里找到 `<footer>` 区域（可能混在几百行代码里）
- Vue 版本：直接打开 `BlogFooter.vue`，改一行就行

**如果你要在另一个页面也显示相同的导航栏**：
- 05 版本：复制粘贴 `<header>` 代码到新页面 → 以后改导航栏要改两处
- Vue 版本：直接 `<BlogHeader />` 一行搞定 → 改一处，处处生效

## 六、已知坑点与禁止事项

| 坑点 | 现象 | 解决 |
|---|---|---|
| 组件名只有一个单词 | 页面不显示，控制台警告 `"Header" should be multi-word` | 改成 `BlogHeader`、`AppFooter` 等多单词名 |
| `import` 路径缺 `./` | 控制台报 `Failed to resolve component` | 自己目录下的文件用 `./` 开头，如 `./components/xxx.vue` |
| `<template>` 里有两个根元素 | 编译报错或显示异常 | `<template>` 下只能有一个根元素，用 `<div>` 包起来 |
| 忘了 `import` | 页面空白，组件不渲染 | 在 `<script setup>` 里 import 每个要用的组件 |
| 文件名大小写不一致 | Windows 可能不报错，Linux/macOS 部署时炸 | 严格按驼峰命名，文件名和 import 路径大小写完全一致 |

## 七、术语附录

| 术语 | 解释 |
|---|---|
| **SFC** | Single File Component，单文件组件。一个 `.vue` 文件包含 `<template>`（结构）、`<script setup>`（逻辑）、`<style scoped>`（样式） |
| **组件（Component）** | Vue 应用的基本构造单元。一个 `.vue` 文件就是一个组件，类比乐高的一块积木。组件可以嵌套、复用 |
| **`<script setup>`** | Vue3 的语法糖，写在 `<script>` 标签里的 `setup` 属性。`import` 的组件自动注册，顶层变量自动暴露给模板 |
| **`scoped`** | `<style>` 标签的属性，让当前组件写的 CSS 只影响当前组件，不会污染其他组件 |
