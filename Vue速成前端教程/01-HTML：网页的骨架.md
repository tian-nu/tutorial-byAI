## 📖 分步操作

# 步骤 1：HTML 是什么——浏览器能看懂的语言

想象你在一家餐厅。你（浏览器）坐在桌前，厨师（服务器）递给你一张写了菜名的纸条（HTML 文件）。纸条上用一套约定好的符号标记哪些是标题、哪些是正文、哪些是图片。这套符号就是 **HTML**。

**HTML** = HyperText Markup Language，超文本标记语言。它不是"编程语言"——它不会算数、不会判断、不会循环。它只是一个**标记系统**，告诉浏览器"这里放标题、这里放段落、这里放图片"。

一个最简单的 HTML 文件长这样：

```html
<!DOCTYPE html>
<html>
  <head>
    <title>我的第一个网页</title>
  </head>
  <body>
    <h1>欢迎！</h1>
    <p>这是我的第一个网页。</p>
  </body>
</html>
```

每一对尖括号 `<>` 包起来的东西叫一个**标签**（Tag）。标签通常是成对出现的：开始标签 `<p>` 和结束标签 `</p>`，中间夹着内容。

> 🤔 **想多一点**：为什么叫"超文本"？因为 HTML 里可以放链接（Hyperlink），点击就能跳到另一个页面——文本不再是线性的，而是"超"越了普通文本的限制。

### 我做得对不对？

- 在脑海里确认：HTML 是一堆标签组成的标记文档，不是编程语言。
- 能默写出一个页面最基本的骨架结构（`<html>` → `<head>` + `<body>`）。

# 步骤 2：用 VS Code 创建 index.html，双击浏览器打开

### 2.1 创建文件

1. 在桌面上新建一个文件夹，名叫 `my-first-webpage`。
2. 打开 VS Code，点击 `File → Open Folder...`，选中刚创建的文件夹。
3. 在 VS Code 左侧资源管理器中，点击"新建文件"图标（一张纸带个加号），命名为 `index.html`。
4. 在 `index.html` 中输入一个英文感叹号 `!`，然后按 `Tab` 键。VS Code 会自动帮你生成 HTML 骨架。

你会看到类似这样的内容：

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body>
  
</body>
</html>
```

把 `<html lang="en">` 改成 `<html lang="zh-CN">`，把 `<title>Document</title>` 改成 `<title>我的第一个网页</title>`。

然后在 `<body>` 和 `</body>` 之间写下：

```html
<h1>Hello，世界！</h1>
<p>这是我的第一个 HTML 页面。</p>
```

5. 保存文件：`Ctrl + S`。

### 2.2 在浏览器中打开

在文件资源管理器中找到 `index.html`，双击它——浏览器会自动打开，你会看到一个大标题"Hello，世界！"和一段文字"这是我的第一个 HTML 页面。"

**恭喜！你刚刚创建了人生第一个网页。**

### 我做得对不对？

- 浏览器地址栏显示的路径是本地文件路径（如 `file:///C:/Users/...`）。
- 页面上能看到你写的标题和段落。

### 不对怎么办？

- ❌ 双击后打开的是 VS Code 而不是浏览器 → 右键文件 → "打开方式" → 选择 Chrome。
- ❌ 浏览器显示一片空白 → 检查是否保存了文件（`Ctrl + S`），检查内容是否确实写在了 `<body>` 标签之间。

# 步骤 3：常用标签——搭网页的积木

把 HTML 标签想象成乐高积木，每个标签是一种形状，组合起来就是一个完整的网页。

### 3.1 标题标签 `<h1>` ~ `<h6>`

```html
<h1>一级标题：最大最醒目</h1>
<h2>二级标题</h2>
<h3>三级标题</h3>
<h4>四级标题</h4>
<h5>五级标题</h5>
<h6>六级标题：最小的标题</h6>
```

类比：一本书的章节标题。`h1` 是书名，`h2` 是章，`h3` 是节。**一个页面通常只有一个 `<h1>`**，就像一本书只有一个书名。

### 3.2 段落标签 `<p>`

```html
<p>这是一段文字。浏览器会自动在段落之间加上间距。</p>
<p>这是第二段。每个 p 标签独占一行。</p>
```

### 3.3 链接标签 `<a>`

```html
<a href="https://www.baidu.com">点我去百度</a>
```

`a` 是 anchor（锚）的缩写。把链接想象成船锚——它把"当前页面"锚定到"目标页面"，点击就顺着锚链跳过去。

### 3.4 图片标签 `<img>`

```html
<img src="https://via.placeholder.com/200" alt="一张占位图">
```

`img` 是**自闭合标签**——它不需要 `</img>` 结束标签。就像一个直接贴在墙上的便利贴，不需要"合上"。

### 3.5 列表标签 `<ul>` / `<ol>` + `<li>`

```html
<!-- 无序列表：前面是圆点 -->
<ul>
  <li>苹果</li>
  <li>香蕉</li>
  <li>橘子</li>
</ul>

<!-- 有序列表：前面是数字 -->
<ol>
  <li>第一步：打开冰箱</li>
  <li>第二步：放入大象</li>
  <li>第三步：关上冰箱</li>
</ol>
```

- `ul` = Unordered List（无序列表），就像备忘录上用圆点列的事项。
- `ol` = Ordered List（有序列表），就像菜谱上的步骤编号。
- `li` = List Item，列表中每一项都必须用 `<li>` 包起来。

### 3.6 容器标签 `<div>` 和 `<span>`

```html
<div>
  <h2>这是一个区块</h2>
  <p>div 就像一个收纳盒，把所有相关的东西装在一起。</p>
</div>

<p>这段话里有一个<span style="color:red;">红色文字</span>，因为 span 包住了它。</p>
```

| 标签 | 比喻 | 特点 |
|------|------|------|
| `<div>` | 收纳箱 | 独占一行（块级元素），用来划分大的区域 |
| `<span>` | 荧光笔 | 不换行（行内元素），用来标记段落中的一小段文字 |

> 🤔 **想多一点**：`div` 和 `span` 本身没有任何外观效果。它们就像透明的塑料袋——袋子本身看不见，但你可以给袋子贴上标签（class / id），或者往袋子里装东西。它们最大的作用是为 CSS 和 JavaScript 提供"抓手"。

### 我做得对不对？

在 `index.html` 的 `<body>` 中把上面每个标签都试一遍，保存后在浏览器中查看效果。你应该能看到：
- 6 个大小不同的标题
- 2 段文字之间有间距
- 一个蓝色的可点击链接
- 一张图片
- 一个带圆点的列表和一个带编号的列表
- 一个 div 包着的区域

# 步骤 4：属性——给标签加参数

属性 = HTML 标签的"设置项"。写在开始标签里，格式是 `属性名="值"`。

### 4.1 五大常用属性

| 属性 | 用于 | 示例 | 说明 |
|------|------|------|------|
| `href` | `<a>` | `href="https://example.com"` | 链接目标地址 |
| `src` | `<img>` | `src="photo.jpg"` | 图片来源路径 |
| `alt` | `<img>` | `alt="一张风景照"` | 图片加载失败时显示的替代文字；读屏软件会朗读它 |
| `class` | 任意标签 | `class="card highlight"` | 给标签分类/分组（可以有多个，空格分隔），CSS 和 JS 的主要操作入口 |
| `id` | 任意标签 | `id="main-title"` | 给标签一个唯一身份标识（一个页面中每个 id 只能出现一次） |

### 4.2 完整示例

```html
<a href="https://www.example.com" class="nav-link">首页</a>
<img src="cat.jpg" alt="一只橘猫" class="pet-photo" id="hero-image">
<div class="container" id="wrapper">
  <p>这个 div 同时有 class 和 id。</p>
</div>
```

### 4.3 class 和 id 的区别（重点！）

| | class | id |
|------|-------|-----|
| 唯一性 | 可以在页面中出现多次 | 一个页面中只能出现一次 |
| 一个标签可以有多个？ | ✅ 可以，用空格分隔 | ❌ 不可以，只能一个 |
| 比喻 | 班级里的"男生"——很多人共用这个标签 | 身份证号——每个人唯一 |
| CSS 选择符 | `.` 开头（如 `.card`） | `#` 开头（如 `#hero`） |
| 使用建议 | 大部分场景用 class | 的唯一操作或锚点跳转时用 id |

> 🤔 **想多一点**：什么时候用 id？当你要在页面内做一个"跳转到此处"的锚点链接时。比如 `#hero-image` 可以让 `<a href="#hero-image">` 点击后滚动到这张图片的位置。除此之外，**优先用 class**——它更灵活、可复用。

### 我做得对不对？

- 在 html 中给一个标签同时加 class 和 id，保存后在 F12 的 Elements 面板中确认能看到这两个属性。
- 尝试写 `<a href="#hero-image">跳到图片</a>`，点击后浏览器地址栏会出现 `#hero-image`。

# 步骤 5：嵌套规则——标签的"俄罗斯套娃"

HTML 标签可以像俄罗斯套娃一样一层套一层，但有严格的规则。

### 5.1 嵌套规则速查表

| 规则 | ✅ 正确 | ❌ 错误 |
|------|--------|--------|
| 结束标签顺序必须与开始标签相反 | `<p><strong>粗体</strong></p>` | `<p><strong>粗体</p></strong>` |
| 块级元素（div, p, h1）可以包含行内元素（span, a, strong） | `<p>点击<a href="#">这里</a></p>` | — |
| 行内元素不能包含块级元素 | `<span>文字</span>` | `<span><div>不行！</div></span>` |
| `<p>` 不能嵌套 `<p>` | — | `<p>段落1<p>段落2</p></p>` |
| `<ul>` / `<ol>` 的直接子元素只能是 `<li>` | `<ul><li>项目</li></ul>` | `<ul><div>项目</div></ul>` |
| `<li>` 内部可以放任何东西 | `<li><a href="#">链接</a></li>` | — |

### 5.2 嵌套类比

把 HTML 嵌套想象成寄快递：
- `<div>` 是最大的纸箱
- `<ul>` 是里面的分类盒
- `<li>` 是分类盒里的每个格子
- `<a>` 是贴在格子上的一张便签

你不能把纸箱（div）塞进便签（a）里——大小不对。

### 我做得对不对？

- 在 VS Code 中故意写 `<p><div>测试</div></p>`，观察 VS Code 是否会给出红色波浪线警告。

# 步骤 6：语义化——让标签"说话"

### 6.1 什么是语义化

语义化 = 用正确的标签描述内容的**含义**，而不是只用 `<div>` 套一切。

**非语义化（不要这样写）**：
```html
<div class="header">网站标题</div>
<div class="nav">导航栏</div>
<div class="main">主要内容</div>
<div class="footer">底部信息</div>
```

**语义化（推荐这样写）**：
```html
<header>网站标题</header>
<nav>导航栏</nav>
<main>主要内容</main>
<footer>底部信息</footer>
```

### 6.2 常用语义化标签

| 标签 | 含义 | 比喻 |
|------|------|------|
| `<header>` | 页眉/头部区域 | 报纸顶端的刊头 |
| `<nav>` | 导航链接区域 | 商场的楼层指引牌 |
| `<main>` | 页面主要内容 | 书的正文（每页只有一个 main） |
| `<section>` | 一个独立的内容区块 | 报纸的一个版面 |
| `<article>` | 一篇独立完整的内容 | 报纸上的一篇报道 |
| `<aside>` | 侧边栏/附加内容 | 杂志侧边的批注 |
| `<footer>` | 页脚/底部区域 | 报纸底部的版权声明 |

### 6.3 为什么语义化重要

1. **搜索引擎**：Google 的爬虫会优先看 `<main>`、`<article>` 里的内容来理解你的页面。
2. **屏幕阅读器**：视障用户使用的读屏软件靠语义标签来导航——"跳到 main 区域"、"阅读 article"。
3. **队友读代码**：看到 `<nav>` 就知道这是导航，不用猜 `<div class="some-weird-name">` 是干嘛的。

> 🤔 **想多一点**：语义化不是"装饰"——它是网页的"公民义务"。你写的不只是一堆标签，而是一份要让世界上任何人都能访问的文档。把 `<div>` 滥用在所有地方，就好比一本书既没有目录、也没有章节标题、也没有段落间距——技术上可以读，但实际上是一场灾难。

### 我做得对不对？

- 能闭卷写出 `<header>`、`<nav>`、`<main>`、`<footer>` 四个标签及各自含义。
- 理解 `<article>` 和 `<section>` 的区别：article 是"独立可发表的文章"，section 是"页面内部的一个分区"。

# 步骤 7：F12 → Elements 看 DOM 树

浏览器拿到 HTML 文本后，会在内部构建一棵树状结构，这叫 **DOM**（Document Object Model，文档对象模型）。你可以把它想象成HTML的"骨架X光片"。

### 7.1 打开开发者工具

1. 在浏览器中打开你的 `index.html`。
2. 按 `F12`（或 `Ctrl + Shift + I`）。
3. 点击 **Elements**（元素）标签页。

你会看到类似这样的结构：

```
▼ <html lang="zh-CN">
  ▼ <head>
      <meta charset="UTF-8">
      <title>我的第一个网页</title>
    </head>
  ▼ <body>
      <h1>Hello，世界！</h1>
      <p>这是我的第一个 HTML 页面。</p>
    </body>
  </html>
```

前面有 `▼` 的表示这个节点可以展开。展开 `<body>` 后，你看到的 `h1` 和 `p` 就是 `<body>` 的**子节点**，它们处于同一层级，是**兄弟节点**。

### 7.2 DOM 树的直觉理解

```
        html
       /    \
    head    body
    /  \     /  \
 meta title h1   p
```

每个标签都是一个**节点**（Node），标签之间的包含关系变成了树的**父子关系**。

### 7.3 试试看

- 在 Elements 面板中，把鼠标悬停在 `<h1>` 标签上，浏览器页面里的对应标题会被高亮出来。
- 双击 `<h1>Hello，世界！</h1>` 中的 "Hello，世界！"，直接改成 "Hello，DOM！"，页面上的文字会实时改变。

> ⚠️ 注意：这只是浏览器内存里的临时修改，刷新页面后就会恢复原样。要永久修改必须改源文件。

### 我做得对不对？

- 按下 F12 能看到 Elements 面板。
- 能展开 `<body>` 节点看到你写的所有标签。
- 能在 Elements 中临时修改文字，并在页面上看到即时变化。

# 步骤 8：常见错误排查

### ❌ 错误 1：标签没闭合

```html
<!-- ❌ 错误 -->
<p>这是一段文字
<p>另一段文字</p>

<!-- ✅ 正确 -->
<p>这是一段文字</p>
<p>另一段文字</p>
```

**后果**：浏览器会尽力"猜测"你的意图自动闭合，但不同浏览器的猜测不同，导致页面在不同浏览器里表现不一致。

### ❌ 错误 2：属性值忘加引号

```html
<!-- ❌ 错误 -->
<a href=https://example.com>链接</a>
<img src=cat.jpg alt=猫>

<!-- ✅ 正确 -->
<a href="https://example.com">链接</a>
<img src="cat.jpg" alt="猫">
```

**为什么必须加引号**：属性值里可能有空格或特殊字符，不加引号浏览器就会误解。比如 `alt=一只 猫` 会被理解成 `alt="一只"` 外加一个不存在的 `猫` 属性。

### ❌ 错误 3：`<img>` 忘了 `alt`

```html
<!-- ❌ 不推荐 -->
<img src="photo.jpg">

<!-- ✅ 推荐 -->
<img src="photo.jpg" alt="日落时的海滩">
```

没有 `alt` 的话，图片加载失败时用户看到的就是一片空白，屏幕阅读器也什么都读不出来。

### ❌ 错误 4：嵌套顺序混乱

```html
<!-- ❌ 错误：先开的门后关 -->
<p><strong>文字</p></strong>

<!-- ✅ 正确：先开的门先关，后开的门后关 -->
<p><strong>文字</strong></p>
```

**记忆口诀**：标签嵌套像穿衣服——先穿内衣再穿外套，脱的时候先脱外套再脱内衣。

> 🤔 **想多一点**：HTML 的容错能力是一把双刃剑。浏览器不会因为你忘了闭合标签就崩溃——它会尽力渲染。但"能显示"不代表"没问题"。养成写标签就立刻写闭合的习惯，可以帮你避免大量诡异 bug。

```html
  </header>

  <main>
    <article>
      <h3>一篇示例文章</h3>

    </article>
  </main>

  <footer>
    <p>&copy; 2026 我的网站</p>
  </footer>
</body>
</html>
```

## 🧠 术语附录

| 术语 | 解释 |
|------|------|
| **HTML** | 超文本标记语言，用标签描述网页结构和内容。不是编程语言。 |
| **标签（Tag）** | 用 `<>` 包起来的关键词，如 `<p>`、`<h1>`。大部分成对出现，少数自闭合。 |
| **属性（Attribute）** | 写在开始标签里的 `键="值"` 对，给标签附加额外信息（如 `href`、`src`、`class`）。 |
| **DOM** | 文档对象模型。浏览器把 HTML 解析后生成的一棵树状结构，每个标签是一个节点。 |
| **语义化（Semantic HTML）** | 用正确的标签描述内容的含义而不是外观，如用 `<nav>` 而不是 `<div class="nav">`。 |
| **块级元素** | 默认独占一行的标签，如 `<div>`、`<p>`、`<h1>`。 |
| **行内元素** | 默认不换行、与其他行内元素同处一行的标签，如 `<span>`、`<a>`、`<strong>`。 |
| **自闭合标签** | 不需要结束标签的标签，如 `<img>`、`<br>`、`<hr>`。 |

---

## 🚧 已知坑点与禁止事项

- **不要用 `<div>` 包一切**：能用语义标签就用语义标签，`<div>` 是最后选择。
- **`<p>` 标签不能嵌套 `<p>`**：浏览器会自动在第一个 `<p>` 后面偷偷插入一个 `</p>`。
- **id 必须唯一**：一个页面里同一个 id 只能出现一次，否则 JavaScript 操作会出意外。
- **属性值必须加引号**：尤其是包含空格或特殊字符时。
