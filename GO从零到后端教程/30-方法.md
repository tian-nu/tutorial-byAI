# 第30章 · 方法

> "函数就像快递员——谁都可以找他寄快递。方法就像你的私人管家——他认识你，知道你家在哪，只能为你服务。方法把行为和数据类型绑在了一起。"

---

## 30.1 方法是什么

### 函数 vs 方法

```go
func add(a, b int) int {
    return a + b
}
```

这是函数——独立的，不属于任何人。

```go
type Calculator struct {
    history []string
}

func (c Calculator) add(a, b int) int {
    result := a + b
    c.history = append(c.history, fmt.Sprintf("%d + %d = %d", a, b, result))
    return result
}
```

这是方法——绑在 `Calculator` 类型上的函数。

注意那个 `(c Calculator)` 的位置，它叫**接收者（Receiver）**。语法是：

```
func (接收者变量 接收者类型) 方法名(参数) 返回值 {
    方法体
}
```

`(c Calculator)` 表示：这个方法属于 `Calculator` 类型，方法内部可以通过 `c` 访问属于这个 `Calculator` 实例的数据。

### 比喻：遥控器

普通函数像一个万能遥控器——谁拿都能用，不知道控制的是哪台电视。

方法是每台电视自带的遥控器——它知道自己控制的是这台电视，按"音量+"就会调这台电视的音量。

### 方法的调用

```go
calc := Calculator{}
result := calc.add(3, 4)
```

`calc.add(3, 4)` 的读法是："用calc自己的add方法，计算3加4"。`.` 操作符在这里的意思是"调用属于calc的方法"。

## 30.2 值接收者 vs 指针接收者

这是Go方法里最重要的选择题。两种接收者，两种截然不同的行为。

### 值接收者：操作的是副本

```go
type Counter struct {
    count int
}

func (c Counter) increment() {
    c.count++
}

func main() {
    c := Counter{count: 0}
    c.increment()
    fmt.Println(c.count)
}
```

输出：`0`

为什么没变？因为 `func (c Counter) increment()` 里的 `c` 是 `Counter` 的**副本**。方法里把副本的 `count` 从0改到1，但原始的 `c` 纹丝不动。

**比喻**：你对着镜子里的自己做了一个鬼脸——你确实动了，但动的只是镜像，真实的你没变。

### 指针接收者：操作的是原件

```go
func (c *Counter) increment() {
    c.count++
}

func main() {
    c := Counter{count: 0}
    c.increment()
    fmt.Println(c.count)
}
```

输出：`1`

接收者类型从 `Counter` 变成了 `*Counter`。方法拿到的不是副本，而是原始 `c` 的指针。通过指针修改，改的是原件。

### 语法糖：Go自动帮你转换

```go
c := Counter{}
c.increment()
```

你可能注意到了：`c` 是 `Counter` 类型（不是 `*Counter`），但方法签名是 `func (c *Counter) increment()`——接收者要求指针。为什么代码能编译？

Go的编译器帮你做了隐式转换：`c.increment()` 等价于 `(&c).increment()`。因为 `c` 是可以取地址的变量，编译器自动取了它的地址，然后调用指针接收者的方法。

同理，反过来也行：

```go
func (c Counter) show() {
    fmt.Println(c.count)
}

cp := &Counter{count: 5}
cp.show()
```

`cp` 是 `*Counter`，但 `show` 的接收者是 `Counter`（值类型）。编译器自动解引用：`cp.show()` 等价于 `(*cp).show()`。

但有一个例外：**不能取地址的值，不能调用指针接收者的方法**。

```go
Counter{count: 5}.increment()
```

`Counter{count: 5}` 是一个临时值，没有固定的内存地址，编译器无法取它的地址。所以如果 `increment` 是指针接收者，这行会编译报错。

## 30.3 如何选择值接收者还是指针接收者

### 决策流程

**第一问：方法需要修改接收者的数据吗？**
- 需要 → 指针接收者
- 不需要 → 继续下一问

**第二问：接收者是大型结构体吗？**
- 是（比如超过64字节） → 指针接收者（避免复制开销）
- 不是 → 继续下一问

**第三问：接收者是不可变类型吗（如 Map、Slice、func、chan）？**
- 是 → 值接收者（它们本身已经是引用语义）
- 不是 → 看情况

**第四问：接收者是基本类型吗（int、string 等）？**
- 是，且不需要修改 → 值接收者
- 是，且需要修改 → 指针接收者

### 一致性原则

**同一个类型的所有方法，接收者类型应该一致。** 如果一个类型有的方法用值接收者，有的用指针接收者，代码会很混乱：

```go
type User struct { ... }

func (u User) Name() string { ... }       // 值接收者
func (u *User) SetName(name string) { ... }  // 指针接收者
```

这其实是可以的，Go允许混用。但大多数代码规范建议：**如果一个类型有任何一个方法需要指针接收者，那么所有方法都用指针接收者**。这样保持一致性，减少混乱。

## 30.4 方法只能定义在本包的类型上

Go不允许给其他包的类型定义方法：

```go
func (s string) reverse() string {
}
```

编译报错。`string` 是Go内置类型，不属于你的包。你不能给它添加方法。

但你可以基于内置类型创建新类型，然后给新类型添加方法：

```go
type MyString string

func (ms MyString) reverse() string {
    runes := []rune(ms)
    for i, j := 0, len(runes)-1; i < j; i, j = i+1, j-1 {
        runes[i], runes[j] = runes[j], runes[i]
    }
    return string(runes)
}

var s MyString = "你好世界"
fmt.Println(s.reverse())
```

`type MyString string` 定义了一个新类型，底层是 `string`，但它是独立的类型，Go允许你为它定义方法。

## 30.5 方法集

方法集是理解接口（下一章）的前置知识。

```go
type Dog struct {
    Name string
}

func (d Dog) Speak() string {
    return "汪汪"
}

func (d *Dog) Run() {
    fmt.Println(d.Name, "在跑")
}
```

`Dog` 有两个方法：`Speak`（值接收者）和 `Run`（指针接收者）。

**方法集的规则：**

| 类型 | 方法集包含 |
|------|----------|
| `Dog`（值类型） | 只包含值接收者的方法：`Speak` |
| `*Dog`（指针类型） | 包含所有方法：`Speak` + `Run` |

因此：

```go
var d Dog
d.Speak()
d.Run()
```

`d.Run()` 能编译——因为编译器自动取了 `&d`，调用了 `(*Dog).Run()`。这里编译器帮了你。

但在接口的场景下，编译器的帮助有限：

```go
type Runner interface {
    Run()
}

func startRunning(r Runner) {
    r.Run()
}

d := Dog{Name: "旺财"}
startRunning(d)
```

**编译报错！** 因为 `Runner` 接口要求 `Run()` 方法，而 `Dog` 的方法集里不包含 `Run()`（那是 `*Dog` 的方法集）。如果传指针：

```go
startRunning(&d)
```

就能编译通过。

这个规则在下一章接口中会反复出现，现在先记住结论：**值接收者的方法可以被值和指针调用；指针接收者的方法只能被指针调用（在接口场景下不可逾越）。**

## 🤔 想多一点：为什么不叫"this"或"self"？

Java用 `this`，Python用 `self`，Go用的是你自己定义的变量名（惯例用类型首字母小写，如 `u User`、`c *Counter`）。

这不是Go开发者忘了，而是故意的——**Go追求简洁，不引入额外关键字**。`this` 或 `self` 本质就是参数，没必要搞特殊。而且你可以起任何名字：

```go
func (thisDoesntHaveToBeOneChar User) DoSomething() { }
```

虽然没人这么写，但Go允许。这种设计的另一个好处：没有"继承链中this指向谁"的歧义。

## 本章小结

| 知识点 | 要点 |
|--------|------|
| 方法定义 | `func (r Receiver) Method(params) returns { ... }` |
| 值接收者 | 操作副本，不修改原值 |
| 指针接收者 | 操作原值，能修改接收者 |
| 语法糖 | 编译器自动在值和指针间转换（取地址/解引用） |
| 选择原则 | 需修改→指针，大结构体→指针，一致性→统一 |
| 方法集 | 值类型只含值接收者方法，指针类型含全部 |
| 跨包限制 | 不能给其他包的类型定义方法 |
