# 114-云服务部署实战

> 💡 你的 Spring Boot 应用在本机跑得很完美，Docker 镜像也打包好了，GitHub Actions 也通了。现在只剩最后一步：把它放到互联网上，让全世界都能访问。本章带你从零开始，在云服务器上部署一个完整的 Spring Boot 应用，配置域名和 HTTPS，让它像一个真正的"产品"。

---

## 本章目标
- 购买并初始化一台云服务器
- 在服务器上安装 Docker 和必要工具
- 用 docker-compose 部署 Spring Boot + MySQL + Redis
- 配置 Nginx 反向代理
- 绑定域名并启用 HTTPS（Let's Encrypt 免费证书）

---

## 114.1 购买云服务器

### 推荐配置

| 项目 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 1 核 | 2 核 |
| 内存 | 2 GB | 4 GB |
| 系统盘 | 40 GB | 60 GB |
| 操作系统 | Ubuntu 22.04 LTS | Ubuntu 22.04 LTS |
| 带宽 | 1 Mbps | 3 Mbps+ |

### 云服务商选择

- **阿里云 ECS**：国内首选，学生优惠
- **腾讯云 CVM**：轻量应用服务器性价比高
- **AWS EC2**：国际首选，t2.micro 免费套餐

> 💡 入门推荐各家的"轻量应用服务器"，预装了常用软件，价格更低。

购买时注意：
1. 选择离用户最近的**地域**
2. **安全组**（防火墙）中开放 22（SSH）、80（HTTP）、443（HTTPS）端口
3. 设置 root 密码或添加 SSH 密钥

## 114.2 服务器初始化

### SSH 登录

```bash
ssh root@your_server_ip
```

### 创建非 root 用户

```bash
adduser ubuntu
usermod -aG sudo ubuntu
```

退出后用 `ubuntu` 用户登录：

```bash
ssh ubuntu@your_server_ip
```

### 安装 Docker

```bash
sudo apt update
sudo apt install docker.io docker-compose-v2 -y
sudo systemctl enable docker --now
sudo usermod -aG docker $USER
```

退出 SSH 重新登录，然后验证：

```bash
docker run hello-world
```

### 安装其他工具

```bash
sudo apt install nginx curl wget vim -y
```

## 114.3 部署应用

### 创建目录结构

```bash
mkdir -p /opt/myapp
cd /opt/myapp
```

### 上传文件

在**本地项目**中，确保有以下文件：
- `Dockerfile`
- `docker-compose.yml`（使用第112章的模板）

```bash
# 在本地执行——上传到服务器
scp Dockerfile docker-compose.yml ubuntu@your_server_ip:/opt/myapp/
scp -r sql/ ubuntu@your_server_ip:/opt/myapp/
```

### 服务器上的 docker-compose.yml

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: prod-mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: your_root_password_here
      MYSQL_DATABASE: myapp
      MYSQL_USER: appuser
      MYSQL_PASSWORD: your_app_password_here
    ports:
      - "127.0.0.1:3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./sql/init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: prod-redis
    restart: always
    command: redis-server --requirepass your_redis_password_here
    ports:
      - "127.0.0.1:6379:6379"
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "your_redis_password_here", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  app:
    build: .
    container_name: prod-app
    restart: always
    ports:
      - "127.0.0.1:8080:8080"
    environment:
      SPRING_PROFILES_ACTIVE: prod
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/myapp?useSSL=false&allowPublicKeyRetrieval=true
      SPRING_DATASOURCE_USERNAME: appuser
      SPRING_DATASOURCE_PASSWORD: your_app_password_here
      SPRING_DATA_REDIS_HOST: redis
      SPRING_DATA_REDIS_PASSWORD: your_redis_password_here
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy

volumes:
  mysql_data:
  redis_data:
```

> ⚠️ `127.0.0.1:3306:3306` 表示 MySQL 端口只绑定到本地回环地址，外部无法直接访问。这是安全措施——外部只通过 Nginx 访问应用。

### 启动

```bash
cd /opt/myapp
docker compose up -d
docker compose ps
docker compose logs -f app
```

### 验证

```bash
curl http://localhost:8080/health
```

## 114.4 配置 Nginx 反向代理

### 你在做什么？
外部用户访问 `http://yourdomain.com`，Nginx 把请求转发给 `localhost:8080` 的 Spring Boot 应用。

### 配置 Nginx

```bash
sudo vim /etc/nginx/sites-available/myapp
```

```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    client_max_body_size 10M;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
    }

    location /health {
        proxy_pass http://127.0.0.1:8080/health;
        proxy_set_header Host $host;
        access_log off;
    }
}
```

### 启用配置

```bash
sudo ln -s /etc/nginx/sites-available/myapp /etc/nginx/sites-enabled/
sudo nginx -t                      # 测试配置语法
sudo systemctl reload nginx        # 重载配置
```

### 验证方法

```bash
curl -I http://yourdomain.com
# 应返回 HTTP/1.1 200 OK
```

## 114.5 绑定域名

### 域名 DNS 配置

在域名服务商（阿里云/腾讯云/Cloudflare）的后台添加 DNS 记录：

| 类型 | 主机记录 | 记录值 | TTL |
|------|----------|--------|-----|
| A | @ | `你的服务器IP` | 600 |
| A | www | `你的服务器IP` | 600 |

### 验证 DNS

```bash
# 在本地执行
ping yourdomain.com
# 应返回你的服务器 IP

nslookup yourdomain.com
# 应看到 A 记录指向你的 IP
```

DNS 生效需要 1-10 分钟（取决于 TTL 和 DNS 传播速度）。

## 114.6 HTTPS——Let's Encrypt 免费证书

### 安装 certbot

```bash
sudo apt install certbot python3-certbot-nginx -y
```

### 申请并自动配置

```bash
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
```

按提示输入邮箱、同意协议。certbot 会自动修改 Nginx 配置，添加 SSL 证书。

### 验证

```bash
curl -I https://yourdomain.com
# 应返回 HTTP/2 200

sudo certbot renew --dry-run
# 测试自动续期（Let's Encrypt 证书 90 天有效，certbot 会自动续期）
```

## 114.7 安全加固

### 防火墙

```bash
sudo ufw allow 22
sudo ufw allow 80
sudo ufw allow 443
sudo ufw enable
sudo ufw status
```

### 禁止 root SSH 登录

```bash
sudo vim /etc/ssh/sshd_config
```

修改：

```
PermitRootLogin no
PasswordAuthentication no
```

然后：

```bash
sudo systemctl reload sshd
```

## 自测题

1. 为什么 docker-compose.yml 中 MySQL 端口写的是 `127.0.0.1:3306:3306` 而不是 `3306:3306`？
2. Nginx 反向代理中 `proxy_set_header Host $host` 的作用是什么？
3. Let's Encrypt 证书有效期是多少天？怎么自动续期？

<details>
<summary>点击查看答案</summary>

1. `127.0.0.1:3306:3306` 只绑定到本地回环接口，外部 IP 无法访问 MySQL。如果写 `3306:3306`，MySQL 会对公网开放，有安全隐患。
2. 把客户端请求的原始 Host 头（域名）传递给后端 Spring Boot 应用，而不是 Nginx 的 localhost。这对于应用内部的 URL 生成和虚拟主机路由很重要。
3. 90 天。certbot 安装时自动创建了 systemd timer（`systemctl list-timers | grep certbot`），每天检查并自动续期即将过期的证书。
</details>
