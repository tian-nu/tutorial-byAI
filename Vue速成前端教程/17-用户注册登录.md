## 一、目标与完成效果

**一句话目标**：实现注册和登录功能，接收后端返回的 token 存入 localStorage，导航栏根据登录状态显示不同内容。

**完成后的可观测效果**：
- `/register` 页面有注册表单，填写后调用 `POST /api/users/register`，成功后跳转登录页
- `/login` 页面有登录表单，填写后调用 `POST /api/users/login`，拿到 token 存 localStorage，跳转首页
- 导航栏：未登录时显示「登录」「注册」；登录后显示「写文章」「退出」
- 点击「退出」清除 token，回到未登录状态

## 三、分步操作

### 步骤1：理解认证流程——Token 是什么

#### 我在做什么？

在写代码之前，先理解「注册 → 登录 → 存 token → 用 token」这条链路。

**比喻：酒店入住流程**

```
1. 注册 = 第一次来酒店，前台登记你的身份证，给你建档
2. 登录 = 下次来酒店，报名字+密码，前台验证身份
3. Token = 前台给你的房卡（一个加密字符串）
4. 用 Token = 每次进房间、去健身房、点餐，刷房卡就行
5. 退出 = 退房，房卡失效
```

**技术流程**：

```
注册：
  浏览器 ──POST /api/users/register {username, password}──→ 后端
  浏览器 ←────── {message: "注册成功"} ────────────── 后端

登录：
  浏览器 ──POST /api/users/login {username, password}──→ 后端
  浏览器 ←────── {token: "eyJhbG...", user: {...}} ── 后端
  浏览器：localStorage.setItem('token', token)
  浏览器：router.push('/')  跳转首页

访问受保护资源（如发布文章）：
  浏览器 ──POST /api/posts  + Authorization: Bearer <token>──→ 后端
  后端：解密 token → 验证身份 → 处理请求 → 返回数据
```

> 📌 **术语标记**：「localStorage」「token」「CORS」「跨域」将收录于术语附录。

#### 我做得对不对？

- [ ] 你能用自己的话解释：为什么有了用户名密码，还需要 token？
  - 答案：用户名密码是「身份证」，只在登录时用一次。token 是「房卡」，每次请求都带着它，不用反复输入密码。而且 token 有过期时间，房卡丢了可以挂失。

### 步骤2：实现注册页面

#### 我在做什么？

注册页面=表单 + v-model 绑定数据 + 点击提交调用 API。

**修改 `src/views/Register.vue`**：

```vue
<template>
  <div class="register-page">
    <h1>📝 注册</h1>

    <form @submit.prevent="handleRegister" class="form">
      <div class="form-group">
        <label>用户名</label>
        <input
          v-model="username"
          type="text"
          placeholder="请输入用户名（至少3个字符）"
          required
        />
      </div>

      <div class="form-group">
        <label>密码</label>
        <input
          v-model="password"
          type="password"
          placeholder="请输入密码（至少6个字符）"
          required
        />
      </div>

      <div class="form-group">
        <label>确认密码</label>
        <input
          v-model="confirmPassword"
          type="password"
          placeholder="请再次输入密码"
          required
        />
      </div>

      <!-- 错误提示 -->
      <p v-if="errorMsg" class="error">{{ errorMsg }}</p>

      <button type="submit" :disabled="loading">
        {{ loading ? '注册中...' : '注册' }}
      </button>
    </form>

    <p class="switch-link">
      已有账号？<router-link to="/login">去登录</router-link>
    </p>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { useRouter } from 'vue-router'
import { registerUser } from '../api/index.js'

const router = useRouter()
const username = ref('')
const password = ref('')
const confirmPassword = ref('')
const errorMsg = ref('')
const loading = ref(false)

async function handleRegister() {
  // 1. 表单验证
  errorMsg.value = ''

  if (username.value.length < 3) {
    errorMsg.value = '用户名至少3个字符'
    return
  }
  if (password.value.length < 6) {
    errorMsg.value = '密码至少6个字符'
    return
  }
  if (password.value !== confirmPassword.value) {
    errorMsg.value = '两次输入的密码不一致'
    return
  }

  // 2. 发送注册请求
  loading.value = true
  try {
    await registerUser({
      username: username.value,
      password: password.value,
    })
    // 3. 注册成功 → 跳转登录页
    router.push('/login')
  } catch (err) {
    errorMsg.value = err.message || '注册失败，请稍后重试'
  } finally {
    loading.value = false
  }
}
</script>
```

#### 代码逐行解释

| 代码 | 含义 |
|------|------|
| `@submit.prevent="handleRegister"` | 表单提交时执行 `handleRegister`，`.prevent` 阻止默认刷新页面 |
| `v-model="username"` | 双向绑定：输入框的值 ↔ `username` 变量 |
| `:disabled="loading"` | 请求中时按钮变灰，防止重复提交 |
| `errorMsg.value = ''` | 每次提交前先清空旧错误信息 |
| `await registerUser({...})` | 调用 API 发送注册请求 |
| `router.push('/login')` | 注册成功后跳转到登录页 |

❌ **错误**：忘记 `@submit.prevent`，只写 `@submit`
- 后果：表单提交后页面会整页刷新（浏览器默认行为），SPA 体验被破坏。

✅ **正确**：`@submit.prevent="handleRegister"`

### 步骤3：实现登录页面

#### 我在做什么？

登录页面结构与注册相似，区别在于：登录成功后拿到 token，存到 localStorage，然后跳转首页。

**修改 `src/views/Login.vue`**：

```vue
<template>
  <div class="login-page">
    <h1>🔑 登录</h1>

    <form @submit.prevent="handleLogin" class="form">
      <div class="form-group">
        <label>用户名</label>
        <input
          v-model="username"
          type="text"
          placeholder="请输入用户名"
          required
        />
      </div>

      <div class="form-group">
        <label>密码</label>
        <input
          v-model="password"
          type="password"
          placeholder="请输入密码"
          required
        />
      </div>

      <p v-if="errorMsg" class="error">{{ errorMsg }}</p>

      <button type="submit" :disabled="loading">
        {{ loading ? '登录中...' : '登录' }}
      </button>
    </form>

    <p class="switch-link">
      没有账号？<router-link to="/register">去注册</router-link>
    </p>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { useRouter } from 'vue-router'
import { loginUser } from '../api/index.js'

const router = useRouter()
const username = ref('')
const password = ref('')
const errorMsg = ref('')
const loading = ref(false)

async function handleLogin() {
  errorMsg.value = ''

  if (!username.value || !password.value) {
    errorMsg.value = '请填写用户名和密码'
    return
  }

  loading.value = true
  try {
    const data = await loginUser({
      username: username.value,
      password: password.value,
    })

    // 登录成功：保存 token 和用户信息
    localStorage.setItem('token', data.token)
    localStorage.setItem('user', JSON.stringify(data.user))

    // 跳转首页
    router.push('/')
  } catch (err) {
    errorMsg.value = err.message || '用户名或密码错误'
  } finally {
    loading.value = false
  }
}
</script>
```

#### 关键区别：登录 vs 注册

| | 注册 | 登录 |
|------|------|------|
| API 调用 | `registerUser({...})` | `loginUser({...})` |
| 后端返回 | `{ message: "注册成功" }` | `{ token: "...", user: {...} }` |
| 成功后 | 跳转 `/login` | 存 token → 跳转 `/` |
| 额外验证 | 确认密码一致 | 无 |

#### 关于 localStorage

```js
// 存
localStorage.setItem('token', 'eyJhbGciOi...')
localStorage.setItem('user', JSON.stringify({ id: 1, username: '小明' }))

// 取
const token = localStorage.getItem('token')
const user = JSON.parse(localStorage.getItem('user'))

// 删
localStorage.removeItem('token')
localStorage.removeItem('user')
```

| 操作 | 代码 | 说明 |
|------|------|------|
| 存 | `localStorage.setItem(key, value)` | key 和 value 都是字符串 |
| 取 | `localStorage.getItem(key)` | 返回字符串或 `null` |
| 删 | `localStorage.removeItem(key)` | 删除指定 key |
| 清空 | `localStorage.clear()` | 删除所有（慎用） |

**注意**：localStorage 只能存字符串。存对象时用 `JSON.stringify()`，取对象时用 `JSON.parse()`。

🤔 **想多一点**：为什么不用 Cookie 存 token？localStorage 不会被自动附加到每个 HTTP 请求中，而 Cookie 会。这意味着用 localStorage 存 token 可以防止 CSRF 攻击（跨站请求伪造）。但 XSS 攻击（跨站脚本注入）仍能读取 localStorage，所以两种方案各有优劣。教程选 localStorage 是因为它操作更简单，适合学习。

### 步骤4：改造导航栏——根据登录状态显示不同内容

#### 我在做什么？

导航栏需要知道「用户是否已登录」，然后显示不同的按钮。

**比喻**：就像商场门卫——没会员卡的人看到「办卡」「登录」入口，有会员卡的人看到「会员专区」「退出」。

**修改 `src/components/BlogHeader.vue`**：

```vue
<template>
  <header class="blog-header">
    <div class="header-inner">
      <router-link to="/" class="logo">📝 MyBlog</router-link>

      <nav class="nav-links">
        <router-link to="/">🏠 首页</router-link>

        <!-- 登录后才显示 -->
        <template v-if="isLoggedIn">
          <router-link to="/create">✏️ 写文章</router-link>
          <a href="#" @click.prevent="handleLogout">🚪 退出</a>
        </template>

        <!-- 未登录时显示 -->
        <template v-else>
          <router-link to="/login">🔑 登录</router-link>
          <router-link to="/register">📝 注册</router-link>
        </template>
      </nav>
    </div>
  </header>
</template>

<script setup>
import { ref } from 'vue'
import { useRouter } from 'vue-router'

const router = useRouter()
const isLoggedIn = ref(!!localStorage.getItem('token'))

function handleLogout() {
  localStorage.removeItem('token')
  localStorage.removeItem('user')
  isLoggedIn.value = false
  router.push('/')
}
</script>
```

| 代码 | 含义 |
|------|------|

### 步骤5：解决 CORS 跨域问题

#### 我在做什么？

当你从 `http://localhost:5173`（前端）请求 `http://localhost:8000`（后端）时，浏览器会因为「跨域」而阻止请求。

**什么是 CORS（跨域资源共享）？**

浏览器有一道安全门禁：如果你的前端页面在 `localhost:5173`，你只能请求 `localhost:5173` 上的资源。任何不同域名、不同端口、不同协议的请求都会被拦截——除非后端明确「允许」你访问。

```
前端 (localhost:5173)  ──请求──→  后端 (localhost:8000)
                              ❌ 被浏览器拦截！
                              原因：端口不同（5173 ≠ 8000）
```

**解决方案：后端配置 CORS**

如果你用的是 Flask（Python）后端：
```python
from flask_cors import CORS
app = Flask(__name__)
CORS(app)  # 允许所有来源跨域访问
```

如果你用的是 Express（Node.js）后端：
```js
const cors = require('cors')
app.use(cors())
```

**Vite 代理方案（前端临时解决，开发阶段用）**

如果后端暂时不方便改，可以在 Vite 配置里做代理。

**修改 `vite.config.js`**：

```js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8000',  // 后端地址
        changeOrigin: true,
      }
    }
  }
})
```

配置代理后，前端请求 `/api/posts` 会被 Vite 转发到 `http://localhost:8000/api/posts`。浏览器看到的是同源请求（都是 `localhost:5173`），所以不会触发 CORS 拦截。

**对比**：

| 方案 | 谁改 | 适用场景 |
|------|------|----------|
| 后端加 CORS 头 | 后端 | 正式项目，推荐 |
| Vite 代理 | 前端 | 开发阶段临时方案，上线后无效 |

> 📌 **术语标记**：「CORS」「跨域」——不是说前端代码跨不过去，而是**浏览器**的安全策略拦住了。

## 五、术语附录

| 术语 | 解释 | 是否项目特有 |
|------|------|:---:|
| **localStorage** | 浏览器提供的本地存储，数据持久保存在用户电脑上，关闭浏览器也不会丢失。最多存 5MB。 | 否 |
| **token** | 一段加密字符串，代表「你已经登录了」。每次请求带着它，后端就知道你是谁。相当于酒店的房卡。 | 否 |
| **CORS** | Cross-Origin Resource Sharing，跨域资源共享。浏览器安全策略，阻止不同源之间的请求，除非后端明确允许。 | 否 |
| **跨域** | 协议、域名、端口三者中任意一个不同，就是跨域。`localhost:5173` 和 `localhost:8000` 端口不同，算跨域。 | 否 |

| 表单验证 | 前端验证（客户端）+ 后端验证（服务端），双重保险 |

## 八、已知坑点与禁止事项

- ⚠️ **localStorage 只能存字符串**：存对象必须 `JSON.stringify()`，取对象必须 `JSON.parse()`。忘了转换会导致 `[object Object]` 或 `null`。
- ⚠️ **token 不要存 Cookie**（除非你理解 CSRF 并做了防护）：本教程用 localStorage + 手动加 Authorization 头，更安全。
- ⚠️ **退出登录后必须更新 `isLoggedIn` 状态**：只删 localStorage 不更新 `ref`，导航栏不会变。
- ⚠️ **CORS 是浏览器行为，不是后端行为**：用 `curl` 或 Postman 请求不会触发 CORS，只有浏览器会拦截。这就是为什么后端同学说「我测试没问题啊」——他们用的是 Postman，不是浏览器。
