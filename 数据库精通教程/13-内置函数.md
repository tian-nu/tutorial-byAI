## 我在做什么？

你已经会写 SELECT、WHERE、JOIN、GROUP BY——这些是 SQL 的骨架。但光有骨架不够，你需要**血肉**：内置函数。

内置函数就是 MySQL 自带的"工具函数"，帮你做字符串拼接、日期格式化、数学计算、条件判断。这些函数不需要你写复杂逻辑，一行 SQL 就能搞定。

学完这一章，你能用 `CONCAT` 拼接字符串、`DATE_FORMAT` 格式化日期、`CASE WHEN` 做条件统计、`COALESCE` 处理 NULL，并认识窗口函数的存在。

## 一、字符串函数

### 1.1 CONCAT：拼接字符串

```sql
SELECT CONCAT('Hello', ' ', 'World');     -- Hello World
SELECT CONCAT(first_name, ' ', last_name) AS full_name FROM users;
```

### 1.2 CONCAT_WS：带分隔符的拼接

```sql
-- CONCAT_WS = CONCAT With Separator
SELECT CONCAT_WS('-', '2024', '01', '15');  -- 2024-01-15
SELECT CONCAT_WS(', ', '北京', '上海', '广州');  -- 北京, 上海, 广州
```

比 `CONCAT` 的好处是：不用手动在每两个参数之间加分隔符，而且**自动跳过 NULL**。

### 1.3 SUBSTRING：截取子串

```sql
-- SUBSTRING(str, start, length)
-- 注意：MySQL 中 start 从 1 开始（不是 0！）
SELECT SUBSTRING('Hello World', 1, 5);    -- Hello
SELECT SUBSTRING('Hello World', 7);       -- World（省略 length 取到末尾）
SELECT SUBSTRING('Hello World', -5);      -- World（负数从末尾倒数）
```

### 1.4 REPLACE：替换

```sql
SELECT REPLACE('Hello World', 'World', 'MySQL');  -- Hello MySQL
SELECT REPLACE('138-0000-1111', '-', '');          -- 13800001111（去掉横线）
```

### 1.5 TRIM / LTRIM / RTRIM：去空格

```sql
SELECT TRIM('  hello  ');    -- 'hello'（去掉两边空格）
SELECT LTRIM('  hello  ');   -- 'hello  '（去掉左边空格）
SELECT RTRIM('  hello  ');   -- '  hello'（去掉右边空格）
```

### 1.6 UPPER / LOWER：大小写转换

```sql
SELECT UPPER('hello');  -- HELLO
SELECT LOWER('HELLO');  -- hello
```

### 1.7 LENGTH vs CHAR_LENGTH（重点！）

```sql
-- CHAR_LENGTH：字符数
-- LENGTH：字节数

SELECT CHAR_LENGTH('hello');   -- 5（5 个字符）
SELECT LENGTH('hello');        -- 5（5 字节，英文每字符 1 字节）

SELECT CHAR_LENGTH('你好');     -- 2（2 个字符）
SELECT LENGTH('你好');          -- 6（utf8mb4 下中文每字符 3 字节）
```

**关键区别**：`CHAR_LENGTH` 返回的是"你能数出来的字符个数"，`LENGTH` 返回的是"计算机实际存储的字节数"。在 utf8mb4 编码下，一个中文字符占 3 字节，一个 emoji 占 4 字节。

## 二、日期函数

### 2.1 NOW / CURDATE / CURTIME

```sql
SELECT NOW();      -- 2024-01-15 14:30:00（当前日期+时间）
SELECT CURDATE();  -- 2024-01-15（当前日期）
SELECT CURTIME();  -- 14:30:00（当前时间）
```

### 2.2 DATE_ADD / DATE_SUB：日期加减

```sql
-- 加 7 天
SELECT DATE_ADD('2024-01-15', INTERVAL 7 DAY);     -- 2024-01-22

-- 减 1 个月
SELECT DATE_SUB('2024-01-15', INTERVAL 1 MONTH);   -- 2023-12-15

-- 加 1 年
SELECT DATE_ADD('2024-01-15', INTERVAL 1 YEAR);    -- 2025-01-15

-- 查"最近 7 天的订单"
SELECT * FROM orders
WHERE created_at >= DATE_SUB(NOW(), INTERVAL 7 DAY);
```

### 2.3 DATEDIFF：日期差（天数）

```sql
SELECT DATEDIFF('2024-01-15', '2024-01-01');  -- 14（相差 14 天）
SELECT DATEDIFF(NOW(), '2024-01-01');         -- 今天距离元旦多少天
```

### 2.4 DATE_FORMAT：格式化日期（重点！）

```sql
-- DATE_FORMAT(date, format_string)
-- 这个函数高频使用，但格式符容易记混

SELECT DATE_FORMAT('2024-01-15 14:30:00', '%Y-%m-%d');        -- 2024-01-15
SELECT DATE_FORMAT('2024-01-15 14:30:00', '%Y年%m月%d日');     -- 2024年01月15日
SELECT DATE_FORMAT('2024-01-15 14:30:00', '%Y-%m-%d %H:%i:%s'); -- 2024-01-15 14:30:00
SELECT DATE_FORMAT('2024-01-15 14:30:00', '%W');               -- Monday（星期几）
```

**常用格式符**：

| 格式符 | 含义 | 示例 |
|--------|------|------|
| `%Y` | 四位年份 | 2024 |
| `%y` | 两位年份 | 24 |
| `%m` | 月份（01-12） | 01 |
| `%d` | 日期（01-31） | 15 |
| `%H` | 小时（00-23） | 14 |
| `%i` | 分钟（00-59） | 30 |
| `%s` | 秒（00-59） | 00 |
| `%W` | 星期名称 | Monday |
| `%M` | 月份名称 | January |

> **记忆技巧**：`%m` 是 month，`%i` 是 minutes（因为 `%m` 已经被 month 占了），`%d` 是 day。

## 三、数学函数

### 3.1 ROUND / CEIL / FLOOR：取整

```sql
SELECT ROUND(3.14159, 2);    -- 3.14（四舍五入到 2 位小数）
SELECT ROUND(3.14159, 0);    -- 3（四舍五入到整数）
SELECT ROUND(3.5);           -- 4

SELECT CEIL(3.1);            -- 4（向上取整，天花板）
SELECT CEIL(3.9);            -- 4

SELECT FLOOR(3.1);           -- 3（向下取整，地板）
SELECT FLOOR(3.9);           -- 3
```

### 3.2 ABS / RAND

```sql
SELECT ABS(-10);             -- 10（绝对值）
SELECT RAND();               -- 0~1 之间的随机数，如 0.734529
SELECT FLOOR(RAND() * 100);  -- 0~99 之间的随机整数
```

## 四、条件函数

### 4.1 IF：简单的条件判断

```sql
-- IF(condition, value_if_true, value_if_false)
SELECT
    username,
    IF(age >= 18, '成年', '未成年') AS age_group
FROM users;
```

### 4.2 CASE WHEN：多条件判断（重点！）

```sql
-- 场景：按订单金额分级
SELECT
    id,
    amount,
    CASE
        WHEN amount < 100 THEN '小额'
        WHEN amount < 500 THEN '中额'
        WHEN amount < 1000 THEN '大额'
        ELSE '超大额'
    END AS level
FROM orders;
```

**CASE WHEN 的两种写法**：

```sql
-- 写法 A：搜索式（CASE WHEN ... THEN ... ELSE ... END）
SELECT
    CASE
        WHEN score >= 90 THEN 'A'
        WHEN score >= 80 THEN 'B'
        WHEN score >= 70 THEN 'C'
        ELSE 'D'
    END AS grade
FROM students;

-- 写法 B：简单式（CASE column WHEN value THEN ...）
SELECT
    CASE status
        WHEN 1 THEN '待审核'
        WHEN 2 THEN '已通过'
        WHEN 3 THEN '已拒绝'
        ELSE '未知'
    END AS status_name
FROM orders;
```

### 4.3 CASE WHEN 在聚合中的应用

```sql
-- 场景：统计"每个月的订单总数、大额订单数、小额订单数"
SELECT
    DATE_FORMAT(created_at, '%Y-%m') AS month,
    COUNT(*) AS total_orders,
    COUNT(CASE WHEN amount >= 500 THEN 1 END) AS high_value_orders,
    COUNT(CASE WHEN amount < 100 THEN 1 END) AS low_value_orders
FROM orders
GROUP BY DATE_FORMAT(created_at, '%Y-%m');
```

**工作原理**：`CASE WHEN amount >= 500 THEN 1 END` 满足条件返回 `1`，不满足返回 `NULL`。`COUNT(1)` 会计数，`COUNT(NULL)` 不计数。所以 `COUNT(CASE WHEN ...)` 就是"条件计数"。

## 五、类型转换：CAST 和 CONVERT

```sql
-- CAST(value AS type)
SELECT CAST('123' AS SIGNED);           -- 123（字符串转整数）
SELECT CAST('2024-01-15' AS DATE);      -- 2024-01-15（字符串转日期）
SELECT CAST(123 AS CHAR);               -- '123'（整数转字符串）

-- CONVERT(value, type)
SELECT CONVERT('123', SIGNED);          -- 123

-- 隐式转换的陷阱（回顾第 07 章）
SELECT '10' > '9';                      -- 0（字符串比较：'1' < '9'）
SELECT CAST('10' AS SIGNED) > 9;        -- 1（数值比较：10 > 9）
```

## 六、COALESCE 和 IFNULL：处理 NULL

### 6.1 COALESCE：返回第一个非 NULL 的值

**COALESCE** *此术语见附录E* 接受多个参数，返回第一个**不是 NULL** 的参数：

```sql
-- COALESCE(val1, val2, val3, ...)
SELECT COALESCE(NULL, NULL, 'hello', 'world');  -- 'hello'（跳过前两个 NULL）
SELECT COALESCE(NULL, 0);                        -- 0
SELECT COALESCE(NULL, NULL);                     -- NULL（全 NULL）

-- 实战：给没有订单总金额的用户显示 0
SELECT
    u.username,
    COALESCE(SUM(o.amount), 0) AS total_amount
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
GROUP BY u.id, u.username;
```

### 6.2 IFNULL：COALESCE 的简化版

```sql
-- IFNULL(val, default)：如果 val 是 NULL，返回 default
SELECT IFNULL(NULL, '默认值');  -- '默认值'
SELECT IFNULL('hello', '默认值');  -- 'hello'

-- 等价于 COALESCE(val, default)，但 IFNULL 只能接受两个参数
```

## 七、窗口函数：进阶选读

> **本节标注"进阶选读"**。窗口函数是 SQL 中比较高级的话题，本教程不深入展开。这里只给你一个"开窗"的感觉，让你知道有这个东西存在，以后遇到时不会一脸懵。

### 7.1 什么是窗口函数？

普通聚合（`GROUP BY`）会把多行**压缩成一行**。窗口函数不压缩行，而是**给每一行计算一个值**，这个值基于"窗口"（一组行）计算。

**窗口函数** *此术语见附录E* 的核心语法：

```sql
函数名() OVER (PARTITION BY 分组列 ORDER BY 排序列)
```

### 7.2 ROW_NUMBER：给每行编号

**ROW_NUMBER** *此术语见附录E* 给每一行一个序号：

```sql
-- 场景：给每个用户的订单按金额从大到小编号
SELECT
    user_id,
    amount,
    ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY amount DESC) AS rank_num
FROM orders;
```

结果：
```
+---------+--------+----------+
| user_id | amount | rank_num |
+---------+--------+----------+
|       1 | 500.00 |        1 |
|       1 | 300.00 |        2 |
|       1 | 100.00 |        3 |
|       2 | 800.00 |        1 |
|       2 | 200.00 |        2 |
+---------+--------+----------+
```

`PARTITION BY user_id` 表示"按 user_id 分组"，`ORDER BY amount DESC` 表示"组内按金额降序排列"。每个用户组内独立编号。

### 7.3 常用窗口函数一览

| 函数 | 作用 | 示例 |
|------|------|------|
| ROW_NUMBER() | 行号（不重复） | 1, 2, 3, 4... |
| RANK() | 排名（有并列，跳号） | 1, 1, 3, 4... |
| DENSE_RANK() | 排名（有并列，不跳号） | 1, 1, 2, 3... |
| SUM() OVER() | 窗口内累计和 | 跑动求和 |
| LAG() | 取前一行值 | 环比计算 |

### 7.4 实战：找每个用户最大的订单

```sql
-- 用窗口函数
SELECT user_id, amount, created_at
FROM (
    SELECT *,
        ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY amount DESC) AS rn
    FROM orders
) AS t
WHERE rn = 1;
```

如果你觉得窗口函数太难，没关系——这个需求也可以用子查询或 JOIN 实现：

```sql
-- 用子查询（回顾第 10a 章）
SELECT * FROM orders o1
WHERE amount = (
    SELECT MAX(amount) FROM orders o2 WHERE o2.user_id = o1.user_id
);
```

> **想多一点**：窗口函数是 SQL 的"瑞士军刀"——功能强大，但学习曲线陡峭。如果你现在觉得不理解，完全正常。等你把基础 SQL 写熟练了，自然会遇到"每组分前 3 名""跑动求和"这类需求，那时候再回来学窗口函数，会有豁然开朗的感觉。

## 我做得对不对？

### 验证方法

```sql
-- 1. 字符串函数
SELECT CONCAT('Hello', ' ', 'World') AS concat_test;
SELECT CONCAT_WS('-', '2024', '01', '15') AS concat_ws_test;
SELECT SUBSTRING('Hello World', 1, 5) AS sub_test;
SELECT REPLACE('Hello World', 'World', 'MySQL') AS replace_test;
SELECT CHAR_LENGTH('你好') AS char_len, LENGTH('你好') AS byte_len;

-- 2. 日期函数
SELECT NOW() AS now_val;

SELECT DATE_FORMAT(NOW(), '%Y年%m月%d日 %H:%i:%s') AS formatted;

-- 3. 条件函数
SELECT
    IF(1 > 0, 'true', 'false') AS if_test,

        WHEN 85 >= 90 THEN 'A'

-- 4. COALESCE
SELECT
    COALESCE(NULL, NULL, 'first_not_null') AS coalesce_test,

    COUNT(*) AS total,

-- 6. 窗口函数（进阶选读）

    user_id,
    amount,

FROM orders;
```

## 不对怎么办？

### 常见错误 1：CHAR_LENGTH 和 LENGTH 混淆

```sql
-- ❌ 误以为 LENGTH 是字符数
SELECT LENGTH('你好');  -- 返回 6（字节数），不是 2！

-- ✅ 正确：字符数用 CHAR_LENGTH
SELECT CHAR_LENGTH('你好');  -- 返回 2（字符数）
```

**记住**：`CHAR_LENGTH` = 你数出来的字符个数，`LENGTH` = 计算机存的字节数。

### 常见错误 2：DATE_FORMAT 格式符混淆

```sql
-- ❌ 错误：用 %M（月份名称）而不是 %m（数字月份）
SELECT DATE_FORMAT(NOW(), '%Y-%M-%d');  -- 2024-January-15（英文月份名）

-- ✅ 正确：数字月份用 %m
SELECT DATE_FORMAT(NOW(), '%Y-%m-%d');  -- 2024-01-15

-- ❌ 错误：用 %m 表示分钟
SELECT DATE_FORMAT(NOW(), '%H:%m');  -- 14:01（把月份当分钟了！）

-- ✅ 正确：分钟用 %i
SELECT DATE_FORMAT(NOW(), '%H:%i');  -- 14:30
```

**记忆技巧**：`%M`（大写）= 月份名称（January），`%m`（小写）= 数字月份（01）。`%i` = 分钟（因为 `%m` 已经被月份占了）。

### 常见错误 3：CASE WHEN 顺序错误

```sql
-- ❌ 错误：条件顺序反了
SELECT
    CASE
        WHEN score >= 70 THEN 'C'   -- 先判断这个
        WHEN score >= 80 THEN 'B'   -- 永远不会执行到！
        WHEN score >= 90 THEN 'A'   -- 永远不会执行到！
    END AS grade
FROM students;
-- 85 分的学生会得到 'C' 而不是 'B'！

-- ✅ 正确：从大到小（或从小到大）按顺序写
SELECT
    CASE
        WHEN score >= 90 THEN 'A'
        WHEN score >= 80 THEN 'B'
        WHEN score >= 70 THEN 'C'
        ELSE 'D'
    END AS grade
FROM students;
```

**原因**：CASE WHEN 按顺序评估，第一个满足的条件就返回，后面的不再判断。

### 常见错误 4：CAST 时遇到非法值

```sql
-- ❌ 字符串转整数，但字符串不是纯数字
SELECT CAST('abc' AS SIGNED);  -- 返回 0（非严格模式）或报错（严格模式）

-- ✅ 防御：先用正则或函数检查
-- 如果需要安全转换，在应用层处理
```

## 本章小结

| 本章学了什么 | 对应命令/概念 | 注意事项 |
|-------------|-------------|---------|
| 字符串拼接 | CONCAT / CONCAT_WS | CONCAT_WS 自动跳过 NULL |
