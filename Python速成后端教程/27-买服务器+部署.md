## 一、目标与完成效果

**一句话目标**：在云服务商买一台最便宜的 Linux 服务器，把你的博客项目部署上去，让全世界都能通过公网 IP 访问。

**完成后的可观测效果**：
- 你拥有了一台云服务器（阿里云/腾讯云轻量应用服务器，最低配），知道它的公网 IP 和 root 密码。
- 你能通过 `ssh root@你的IP` 远程登录服务器，不用再碰本地终端。
- 服务器上安装了 Python 3.11+、pip、Git。
- 你的博客项目代码被 `git clone` 到了服务器上，依赖安装完毕，服务已启动。
- 浏览器访问 `http://你的IP:8000/docs` 能看到 Swagger 文档页面。
- 服务在后台运行（用 `nohup`），你关掉 SSH 终端也不会停。

## 三、分步操作

### 步骤 1：选购云服务器

**我在做什么？** 在云服务商那里买一台 Linux 服务器。

**重要提示**：你不需要一台高配服务器。最低配（1 核 CPU、1GB 内存）足够跑你的博客项目。学完可以随时销毁，不会一直扣钱。

#### 推荐方案

| 云服务商 | 产品名称 | 推荐配置 | 参考价格 |
|----------|----------|----------|----------|
| 阿里云 | 轻量应用服务器 | 1 核 1G，20GB SSD | ~30-50 元/月 |
| 腾讯云 | 轻量应用服务器 | 1 核 1G，20GB SSD | ~30-50 元/月 |
| 华为云 | 云耀云服务器 | 1 核 1G，20GB SSD | ~30-50 元/月 |

**三家的"轻量应用服务器"本质一样**：CPU、内存、硬盘、带宽打包卖，比传统 ECS 便宜，适合个人项目和学习。此术语需进附录：轻量应用服务器。

**比喻**：云服务器就像"租一间公寓"——你不用自己盖楼，只需要付租金，就能用里面的水电（CPU、内存、网络）。最低配的"单间"足够你一个人住（跑博客），等你公司做大再换"大平层"（高配服务器）。

#### 购买步骤（以阿里云为例，其他平台类似）

1. 打开 [阿里云官网](https://www.aliyun.com/)，注册/登录。
2. 搜索"轻量应用服务器"，进入购买页面。
3. **地域**：选择离你最近的（如"华东 1（杭州）"或"华南 1（深圳）"）。地域影响访问速度，但对你个人学习来说差别不大。此术语需进附录：地域。
4. **镜像类型**：选择 **"系统镜像"** → **Ubuntu 22.04 LTS**。**不要选"应用镜像"（那些预装了 WordPress 等，不是我们需要的）。**
5. **套餐**：选择最低配（1 核 1G 内存，20GB 硬盘，3Mbps 带宽）。
6. **时长**：先买 1 个月试试。学完可以销毁，或者继续用。
7. 支付，等待创建完成。

**我做得对不对？** 在云服务商控制台 → 轻量应用服务器列表，能看到一台状态为"运行中"的服务器。

**不对怎么办？**

| 问题 | 解决 |
|------|------|
| 选错了镜像（选了 Windows 或 CentOS） | 在控制台"重置系统"，选择 Ubuntu 22.04 LTS |
| 地域选错了 | 轻量应用服务器创建后**不能换地域**，只能销毁重建。好在数据不多，销毁重建不心疼 |
| 买了高配，发现用不上 | 可以"降配"（部分平台支持），或者销毁重新买最低配 |

### 步骤 2：获取公网 IP 和 root 密码

**我在做什么？** 拿到连接服务器所需的"地址"和"钥匙"。

在云服务商控制台 → 轻量应用服务器列表 → 点击你的服务器，你会看到：

| 信息 | 示例 | 用途 |
|------|------|------|
| **公网 IP** | `47.96.123.45` | 这就是服务器的"门牌号"，全世界都能通过这个地址访问 |
| **root 密码** | 初始随机密码，或你设置的密码 | 登录服务器的"钥匙" |

**如果没看到密码**：
- 阿里云：点击"重置密码"，设置一个 8-30 位的密码（包含大小写字母和数字）。
- 腾讯云：点击"重置密码"。
- 记下密码！用笔记软件保存，不要只记在脑子里。

**比喻**：公网 IP = 家庭住址，root 密码 = 家门钥匙。有了这两个，你就能从任何地方"回家"（登录服务器）。

### 步骤 3：SSH 登录服务器

**我在做什么？** 用 SSH 从你的电脑远程连接到服务器。此术语需进附录：SSH。

**SSH**（Secure Shell）是一种加密的远程连接协议——你可以把它理解为"在本地打开的终端窗口，但实际运行的是远程服务器上的命令"。

#### Windows 用户（用 PowerShell 或 WSL 终端）

```powershell
ssh root@你的公网IP
```

**示例**：

```powershell
ssh root@47.96.123.45
```

#### macOS / Linux 用户

```bash
ssh root@你的公网IP
```

**第一次连接时，会看到这个提示**：

```
The authenticity of host '47.96.123.45 (47.96.123.45)' can't be established.
ECDSA key fingerprint is SHA256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

**输入 `yes`，然后回车。** 这是 SSH 在确认"你确定要连接这台你从来没连过的服务器吗？"——只需要在第一次连接时确认一次。

**然后输入你的 root 密码**（输入时不显示，盲打后回车）。

**成功登录的标志**：终端提示符变成了类似这样：

```
root@你的服务器名:~#
```

**注意**：提示符从 `$` 变成了 `#`——`#` 表示你是 root（超级管理员），权限最高，操作需谨慎。

**我做得对不对？** 登录后输入 `whoami`，输出 `root`。输入 `hostname`，输出你的服务器名。

**不对怎么办？**

| 问题 | 现象 | 解决 |
|------|------|------|
| `Connection refused` | 连不上 | 服务器还没完全启动，等 1-2 分钟再试。或者安全组没开 22 端口（见步骤 10） |
| `Permission denied` | 密码错误 | 去控制台重置密码，再试 |
| `Connection timed out` | 超时 | 检查公网 IP 是否正确。某些公司 WiFi 可能封了 22 端口，换手机热点试试 |
| Windows 找不到 `ssh` 命令 | — | Windows 10 1803 以上自带 SSH。如果太旧，用 WSL 终端里的 `ssh` 命令 |

### 步骤 4：服务器基础配置

**我在做什么？** 更新系统软件包，确保服务器是最新状态。

```bash
# 更新软件包列表
apt update

# 升级所有已安装的软件包（-y 表示自动回答 yes）
apt upgrade -y
```

**这一步需要 2-5 分钟**，取决于网络速度和需要更新的包数量。你会看到终端滚屏输出下载和安装进度。

**比喻**：`apt update` = 刷新"超市商品清单"（看看有哪些新版本），`apt upgrade` = 把家里所有东西都换成最新版。新服务器就像刚租的空房子，先做一次大扫除（更新系统），再搬东西进来。

**我做得对不对？** 命令执行完后，再次运行 `apt update`，应该看到 `All packages are up to date.`。

### 步骤 5：安装 Python 3.11+

**我在做什么？** 在服务器上安装 Python 运行环境。

```bash
# 安装 Python 3、pip、虚拟环境工具
apt install python3 python3-pip python3-venv -y
```

**验证安装**：

```bash
python3 --version
# 预期输出：Python 3.10.x 或 3.11.x（Ubuntu 22.04 默认带 3.10）

pip3 --version
# 预期输出：pip 22.x from /usr/lib/python3/dist-packages/pip (python 3.10)
```

**Ubuntu 22.04 默认 Python 是 3.10，够用了。** 如果你需要 3.11+，可以用 `deadsnakes` PPA：

```bash
# 可选：安装 Python 3.11
apt install software-properties-common -y
add-apt-repository ppa:deadsnakes/ppa -y
apt update
apt install python3.11 python3.11-venv -y
```

**不过对于本教程，3.10 完全够用，跳过这一步也行。**

### 步骤 6：安装 Git 并克隆项目

**我在做什么？** 把 GitHub 上的博客项目代码下载到服务器上。

```bash
# 安装 Git
apt install git -y

# 验证
git --version
# 预期输出：git version 2.34.x
```

**克隆你的项目**：

```bash
# 进入用户主目录
cd ~

# 克隆项目（替换为你的 GitHub 仓库地址）
git clone https://github.com/你的用户名/blog_backend.git

# 进入项目目录
cd blog_backend
```

**如果你的仓库是私有的**，需要配置 SSH 密钥或者用 GitHub Token。对于学习项目，建议设为公开仓库——更方便。

**我做得对不对？** `ls` 能看到 `blog_backend` 目录，里面有 `main.py`、`requirements.txt` 等文件。

**不对怎么办？**

| 问题 | 解决 |
|------|------|
| 克隆失败，提示 `repository not found` | 检查仓库地址是否正确。如果是私有仓库，需要先配置认证 |
| 服务器上没用过 GitHub，克隆时提示 SSL 错误 | `apt install ca-certificates -y` 更新证书 |
| 忘记 GitHub 仓库地址 | 在本地项目目录执行 `git remote -v` 查看 |

### 步骤 7：创建虚拟环境并安装依赖

**我在做什么？** 在服务器上创建 Python 虚拟环境，安装项目依赖。

```bash
# 确保在项目目录中
cd ~/blog_backend

# 创建虚拟环境
python3 -m venv .venv

# 激活虚拟环境
source .venv/bin/activate

# 现在提示符前应该出现 (.venv) 字样
# 安装依赖
pip install -r requirements.txt
```

**注意事项**：
- 激活虚拟环境用 `source .venv/bin/activate`，不是 Windows 的 `.venv\Scripts\activate`。
- 如果你在创建 `requirements.txt` 时没有包含所有依赖，现在可以手动补装：`pip install fastapi uvicorn sqlalchemy python-jose passlib python-multipart`。

**我做得对不对？** `pip list` 能看到 FastAPI、uvicorn、SQLAlchemy 等包。

**不对怎么办？**

| 问题 | 解决 |
|------|------|
| `pip install` 报错 `command not found` | 先确认虚拟环境已激活（提示符前有 `(.venv)`） |
| 安装依赖时某个包失败 | 大概率是网络问题，重试 `pip install -r requirements.txt` |
| `python-jose` 安装失败 | 需要先安装编译工具：`apt install build-essential -y` |

### 步骤 8：启动服务

**我在做什么？** 启动 FastAPI 服务，让它监听来自外网的请求。

```bash
# 确保在项目目录，虚拟环境已激活
cd ~/blog_backend
source .venv/bin/activate

# 启动服务（关键：--host 0.0.0.0）
uvicorn blog_backend.main:app --host 0.0.0.0 --port 8000
```

**`--host 0.0.0.0` 是什么意思？** 此术语需进附录。

FastAPI 默认只监听 `127.0.0.1`（本机），这意味着只有服务器自己（或 SSH 连接）能访问，外网用户访问不了。`--host 0.0.0.0` 告诉 uvicorn："监听所有网络接口，不管是本机、内网还是外网，只要有请求来就接收。"

**比喻**：`127.0.0.1` = "只接熟人电话"（本机），`0.0.0.0` = "接所有人的电话"（开放给外网）。部署时必须用 `0.0.0.0`。

**启动后你会看到**：

```
INFO:     Started server process [12345]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

**现在测试**：在浏览器中访问 `http://你的公网IP:8000/docs`。

**如果看到 Swagger 文档页面，恭喜！你的博客已经部署到服务器上了！** 🎉

**如果看不到**，继续步骤 9——大概率是防火墙（安全组）的问题。

> ❌ 常见错误：服务只在 127.0.0.1 监听

```bash
# ❌ 错误做法：没有 --host 参数
uvicorn blog_backend.main:app --port 8000
# 默认监听 127.0.0.1，外网访问不到

# ✅ 正确做法
uvicorn blog_backend.main:app --host 0.0.0.0 --port 8000
```

### 步骤 9：让服务在后台运行——`nohup` 或 `screen`

**我在做什么？** 当你按 `Ctrl+C` 或关闭 SSH 终端时，服务会停止。我需要让服务在后台持续运行。

#### 方案 A：`nohup`（最简单）此术语需进附录

```bash
# 先按 Ctrl+C 停掉当前运行的服务

# 用 nohup 在后台启动
nohup uvicorn blog_backend.main:app --host 0.0.0.0 --port 8000 > app.log 2>&1 &

# 解释：
# nohup           = no hangup，即使终端关了也不停
# > app.log       = 把输出写到 app.log 文件
# 2>&1            = 把错误输出也合并到 app.log
# &               = 放在后台运行
```

**查看日志**：

```bash
tail -f app.log     # 实时查看日志（按 Ctrl+C 退出查看，不影响服务）
```

**查看服务是否在运行**：

```bash
ps aux | grep uvicorn
```

**停止服务**：

```bash
# 找到 uvicorn 的 PID
ps aux | grep uvicorn
# 输出类似：root  12345  ...  uvicorn blog_backend.main:app

# 终止它
kill 12345    # 替换为实际的 PID
```

#### 方案 B：`screen`（推荐，更灵活）此术语需进附录

```bash
# 安装 screen
apt install screen -y

# 创建一个名为 "blog" 的 screen 会话
screen -S blog

# 现在进入了一个新的终端窗口，启动服务
cd ~/blog_backend
source .venv/bin/activate
uvicorn blog_backend.main:app --host 0.0.0.0 --port 8000

# 按 Ctrl+A，然后按 D —— 从 screen 中"脱离"（detach），服务继续运行
# 你会回到原来的终端

# 重新连接（reattach）screen 会话
screen -r blog

# 查看所有 screen 会话
screen -ls

# 在 screen 会话中按 Ctrl+C 停止服务，输入 exit 退出 screen
```

**比喻**：`nohup` = 让程序"自己悄悄跑，别管终端关了没"；`screen` = 给程序开了一个"独立终端小窗口"，你可以随时进去看看，退出来服务继续跑。

**推荐用 `screen`**——你可以随时 `screen -r blog` 回到服务终端，看实时日志，改代码，重启服务，非常方便。

### 步骤 10：防火墙设置——开放 8000 端口

**我在做什么？** 云服务商默认只开放了少数端口（22 SSH、80 HTTP、443 HTTPS），8000 端口需要手动开放。

**比喻**：安全组就像服务器的"门禁系统"——默认所有门都锁着，你需要告诉保安："请把 8000 号门打开，允许客人进来。"

#### 云服务商安全组设置

**阿里云**：
1. 控制台 → 轻量应用服务器 → 点击你的服务器 → "防火墙" 标签
2. 点击"添加规则"
3. 端口：`8000`，协议：`TCP`，来源：`0.0.0.0/0`（允许所有 IP）
4. 备注：`FastAPI Blog`
5. 点击确定

**腾讯云**：
1. 控制台 → 轻量应用服务器 → 点击你的服务器 → "防火墙" 标签
2. 点击"添加规则"
3. 端口：`8000`，协议：`TCP`，策略：`允许`
4. 备注：`FastAPI Blog`
5. 点击确定

**添加后，再次访问 `http://你的公网IP:8000/docs`，应该能看到了！**

**我做得对不对？** 浏览器打开 `http://你的公网IP:8000/docs`，能看到 Swagger 文档页面。

**不对怎么办？**

| 问题 | 解决 |
|------|------|
| 开了安全组还是访问不了 | 服务器上可能有本地防火墙（`ufw`）。运行 `ufw allow 8000` 放行 |
| 安全问题：`0.0.0.0/0` 所有人都能访问 | 对于学习阶段没问题。生产环境可以通过 Nginx 反向代理，只开放 80/443 端口（下章讲） |
| 8000 端口被其他程序占用 | `lsof -i:8000` 查看是谁占用了 8000 端口 |

### 步骤 11：验证——完整功能测试

**我在做什么？** 确认部署成功，所有接口都能正常工作。

在浏览器中打开 `http://你的公网IP:8000/docs`，逐项测试：

1. **获取文章列表**：`GET /posts` → 返回文章列表（可能是空的，正常）
2. **注册用户**：`POST /users` → 输入用户名密码，注册成功
3. **登录**：`POST /login` → 获取 token
4. **创建文章**：`POST /posts` → 粘贴 token，创建一篇文章
5. **获取文章列表**：`GET /posts` → 能看到刚创建的文章

**全部通过？你的博客系统正式上线了！** 🎉

## 八、已知坑点与禁止事项

| 坑点 | 现象 | 原因 | 解决 |
|------|------|------|------|
| 忘记 `--host 0.0.0.0` | 外网访问 `http://IP:8000` 无响应 | uvicorn 默认监听 `127.0.0.1`，只接受本机请求 | 加 `--host 0.0.0.0` |
| 安全组没开 8000 端口 | 外网访问不了，但服务器上 `curl localhost:8000` 正常 | 云服务商默认拦截非标准端口 | 在安全组/防火墙中添加 8000 端口规则 |
| 关掉 SSH 终端，服务停了 | 重新登录后 `ps aux \| grep uvicorn` 找不到进程 | 直接运行 `uvicorn` 是前台进程，终端关闭 → 进程结束 | 用 `nohup` 或 `screen` 启动 |
| `screen -r` 提示 "No screen to be resumed" | 找不到 screen 会话 | 可能之前在另一个用户下创建的，或者会话已终止 | `screen -ls` 查看所有会话，用 `screen -r 会话ID` 连接 |
| 忘记激活虚拟环境 | 启动报错 `ModuleNotFoundError: No module named 'fastapi'` | 依赖安装在虚拟环境里，没激活就用系统 Python | `source .venv/bin/activate` |
| 在服务器上直接 `pip install` 而不是在 venv 里 | 污染系统 Python，不同项目依赖冲突 | 服务器上可能有多个项目，直接安装会导致冲突 | 始终在虚拟环境中安装依赖 |
| 在服务器上存明文密码/密钥 | 代码里的 `SECRET_KEY` 是硬编码的 | 本教程阶段为了简单，密钥写在代码里 | 生产环境用环境变量：`export SECRET_KEY=xxx`，然后 `os.getenv("SECRET_KEY")` |
