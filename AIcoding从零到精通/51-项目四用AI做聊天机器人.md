## 为什么第四个项目是"聊天机器人"

前三个项目，你都是"自己用"——个人主页给别人看，Todo给自己用，仪表盘给自己看。但聊天机器人不同——**你调用的是AI的能力**。

这是实战篇的**终极项目**。难度最高，但也最酷——你写一个网页，用户在浏览器里说话，你的程序调用AI API，AI回复，显示在网页上。**你用AI做了一个AI**。

---

## 51.1 你要做什么

做一个网页聊天机器人，支持：
- 在浏览器里和AI聊天（类似ChatGPT网页版）
- AI记住上下文（你说"我叫张三"，下一句问"我叫什么？"，AI能答出来）
- 可以设定AI的角色（比如"你是一个毒舌的吐槽大师"）
- 对话界面美观，像真正的聊天应用

**最终效果**：浏览器打开 `http://localhost:5000`，看到一个聊天界面。输入"你好"，AI回复。输入"给我讲个笑话"，AI讲笑话。设定角色为"你是一个毒舌的吐槽大师"，AI的回复风格变了。

**技术方案**：Python Flask（后端） + OpenAI/DeepSeek API（AI能力） + HTML/CSS/JS（前端聊天界面）。

## 51.2 技术名词速览

| 名词 | 白话解释 |
|------|---------|
| **📖 OpenAI API** | OpenAI公司提供的"AI大脑租赁服务"——你发文字过去，它回复文字过来。按用量收费，但有免费额度 |
| **📖 API Key** | 你的"门禁卡"——调用API时必须带上它，证明"是你本人在用，而且你有额度"。**绝对不能泄露！** |
| **📖 .env** | 一个专门存放私密信息的文件（环境变量文件）。数据库密码、API Key等敏感信息放这里，不写进代码 |
| **📖 CORS** | "跨域资源共享"——当你前端（浏览器）和后端（服务器）不在同一个域名时，浏览器默认会阻止请求。CORS是告诉浏览器"这个可以放行" |

## 51.3 ⚠️ 安全第一：API Key绝对不能泄露

在开始之前，请把这句话刻在脑子里：

> **API Key 就像你的银行卡密码。把代码发到GitHub时，如果代码里有API Key，就等于把你的银行卡密码贴在了公告栏上。**

**后果有多严重？**
- 别人拿到你的API Key，可以疯狂调用API
- 你的账户额度可能几分钟内被刷光
- 如果绑定了信用卡，可能产生巨额账单

**怎么做才安全？**

✅ **正确做法**：把API Key放在 `.env` 文件里，用 `python-dotenv` 库读取，并且把 `.env` 加入 `.gitignore`。

❌ **错误做法**：把API Key直接写在代码里：
```python
# ❌ 绝对不要这样！
OPENAI_API_KEY = "sk-abc123def456..."  # 一旦提交到GitHub就泄露了
```

✅ **正确做法**：
```python
# ✅ 正确：从环境变量读取
import os
from dotenv import load_dotenv
load_dotenv()
OPENAI_API_KEY = os.getenv("OPENAI_API_KEY")
```

然后 `.env` 文件里写：
```
OPENAI_API_KEY=your_api_key_here
```

并在 `.gitignore` 里加上：
```
.env
```

## 51.6 第三步：让AI写后端代码

### **1.** 打开AI对话，输入以下Prompt：

```
我要做一个网页聊天机器人，技术栈：Python Flask + OpenAI API + HTML/CSS/JS。

功能需求：
1. 网页上有一个聊天界面，类似ChatGPT
2. 用户输入文字，发送给后端，后端调用OpenAI API，返回AI回复
3. 支持多轮对话——AI能记住之前聊过的内容（保存对话历史）
4. 支持设定AI角色——页面上有一个输入框可以设定系统提示词，比如"你是一个毒舌的吐槽大师"
5. 前端用EventSource或fetch实现流式输出（AI回复一个字一个字地显示）

安全要求：
- API Key从.env文件读取，用python-dotenv
- .env文件已加入.gitignore
- 代码中不要硬编码API Key

项目结构：
chatbot/
├── app.py           # 后端Flask应用
├── .env             # API Key（不提交到Git）
├── .gitignore
└── templates/
    └── index.html   # 前端聊天界面

请先写后端app.py，再写前端index.html。
```

### **2.** AI会生成 `app.py`，代码大概长这样：

```python
# app.py —— 聊天机器人后端
from flask import Flask, render_template, request, jsonify, Response
from openai import OpenAI
from dotenv import load_dotenv
import os
import json

load_dotenv()

app = Flask(__name__)
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

# 存储对话历史（实际生产环境应该用数据库或Redis）
conversations = {}

@app.route('/')
def index():
    return render_template('index.html')

@app.route('/api/chat', methods=['POST'])
def chat():
    data = request.get_json()
    user_message = data.get('message', '')
    session_id = data.get('session_id', 'default')
    system_prompt = data.get('system_prompt', '你是一个有用的AI助手。')
    
    # 初始化或获取对话历史
    if session_id not in conversations:
        conversations[session_id] = [{"role": "system", "content": system_prompt}]
    
    # 添加用户消息
    conversations[session_id].append({"role": "user", "content": user_message})
    
    # 调用OpenAI API
    response = client.chat.completions.create(
        model="gpt-3.5-turbo",
        messages=conversations[session_id],
        stream=True
    )
    
    # ... 流式返回逻辑

# ... 完整代码由AI生成
```

> ⚠️ **同样，不要自己手打这段代码**。让AI生成完整代码。AI生成的代码会包含流式输出、错误处理等完整逻辑。

## 51.9 第六步：调试迭代

聊天机器人是最容易出Bug的项目，因为涉及前端、后端、外部API三层。典型的调试过程：

**第一次运行**：`401 Unauthorized`
→ API Key错了或没设置。检查 `.env` 文件里的Key是否正确。

**第二次运行**：能回复了，但每次回复都是完整的一段，不是流式
→ 告诉AI："前端没有流式显示，AI回复是一下子全出来的，请改成SSE流式输出"

**第三次运行**：流式OK了，但AI不记得上下文
→ 告诉AI："AI记不住之前聊过的内容，每次都是新对话，请检查对话历史的管理逻辑"

**第四次运行**：角色设定改了但没生效
→ 告诉AI："修改系统提示词后，需要清空之前的对话历史，因为旧消息里system prompt是旧的"

**第五次运行**：✅ 全部功能正常！

## 51.11 排错指南

### 🔴 问题1：`401 Unauthorized` 或 `Incorrect API key provided`

**原因**：API Key错误——可能是没设置、拼错、过期、或者没有额度。

**解决**：
1. 检查 `.env` 文件是否在 `chatbot` 文件夹里
2. 检查 `.env` 里的Key是否完整（以 `sk-` 开头，没有多余空格或引号）
3. 登录API平台确认Key是否有效
4. 确认账户有余额/额度

### 🔴 问题2：`429 Too Many Requests` 或 `Rate limit exceeded`

**原因**：调用太频繁，或者免费额度用完了。

**解决**：
1. 等一会儿再试（通常1分钟后自动恢复）
2. 登录API平台查看额度使用情况
3. 如果是免费额度用完，需要充值

### 🔴 问题3：前端发送消息后没反应，F12 Console报 `CORS error`

**原因**：📖 CORS（跨域）问题——浏览器阻止了前端请求后端API。

**症状**：Console报错类似 `Access to fetch at 'http://localhost:5000/api/chat' from origin 'null' has been blocked by CORS policy`。

**解决**：告诉AI：
```
前端请求后端API时出现了CORS跨域错误，请在后端Flask中加上 flask-cors：
1. pip install flask-cors
2. 在app.py中添加 from flask_cors import CORS; CORS(app)
```

### 🔴 问题4：AI记不住上下文

**原因**：对话历史管理有问题——每次请求创建了新的对话，或者历史没有正确追加。

**解决**：告诉AI"AI记不住之前聊过的内容，请检查对话历史的管理逻辑，确保每次请求都把之前的消息带上"。

### 🔴 问题5：流式输出不生效

**原因**：前端没有正确处理SSE（Server-Sent Events）流式响应。

**解决**：告诉AI"前端没有流式显示，请检查前端JavaScript是否正确处理了SSE流式响应"。

### 🔴 问题6：`ModuleNotFoundError: No module named 'openai'`

**原因**：没装依赖。

**解决**：
```bash
pip install openai python-dotenv flask
```

### 🔴 问题7：修改角色后，旧对话还在

**原因**：修改系统提示词后，对话历史里还有旧的system prompt。

**解决**：告诉AI"修改系统提示词后，请清空之前的对话历史，用新的system prompt重新开始"。
