Spring Security 过滤器链

HTTP 请求依次穿过 7 个过滤器，每个负责不同的安全检查

↑ HTTP 请求进入
│
1
SecurityContextPersistenceFilter
恢复上一次请求的 SecurityContext（从 HttpSession 读取）
请求到达时，尝试从 Session 中恢复已认证用户信息存入 SecurityContextHolder。请求结束后，清空 SecurityContextHolder 防止内存泄漏。
│
2
HeaderWriterFilter
添加安全响应头（X-Frame-Options、X-XSS-Protection 等）
自动在响应中添加安全头部：X-Frame-Options: DENY（防点击劫持）、X-Content-Type-Options: nosniff（防 MIME 嗅探）、X-XSS-Protection: 1; mode=block。
│
3
CsrfFilter
检查 CSRF Token（POST/PUT/DELETE 必须携带有效 token）
跨站请求伪造防护。检查非 GET 请求是否携带合法的 _csrf token。匹配失败抛 AccessDeniedException。前后端分离项目通常关闭。
│
4
LogoutFilter
拦截 /logout 请求，清除认证信息并重定向
匹配 /logout → 调用 LogoutHandler 链（清除 SecurityContext、使 Session 失效、清除 RememberMe）→ 重定向到登录页。
│
5
UsernamePasswordAuthenticationFilter ⭐
核心登录过滤器 — 处理 POST /login 表单提交
这是最核心的认证过滤器。从请求中提取 username 和 password → 封装成 UsernamePasswordAuthenticationToken → 交给 AuthenticationManager 认证 → 成功后存入 SecurityContextHolder。
│
6
ExceptionTranslationFilter
捕获下游抛出的安全异常，翻译为 HTTP 响应
捕获 AuthenticationException → 触发 AuthenticationEntryPoint（未登录 → 302 到登录页 / 401）。捕获 AccessDeniedException → 触发 AccessDeniedHandler（权限不足 → 403）。
│
7
FilterSecurityInterceptor ⭐
最终守门员 — 检查当前请求是否有权访问目标资源
根据配置的权限规则（authorizeHttpRequests）和当前用户的角色/权限，决定是否放行。这是过滤器链的最后一环，通过后请求到达 Controller。
│
↓ 到达 Controller（你的业务代码）
