## 一、目标与完成效果

**一句话目标**：实现文章封面图上传功能——用户上传图片，后端保存到服务器，返回可访问的 URL，并把 URL 关联到文章。

**完成后的可观测效果**：
- `POST /posts/upload-image` 接收图片文件，保存到 `uploads/` 目录，文件名用 UUID 避免冲突。
- 上传成功后返回 `{"url": "/static/abc123.jpg", "filename": "abc123.jpg"}`。
- 浏览器访问 `http://127.0.0.1:8000/static/abc123.jpg` 能看到刚才上传的图片。
- 创建文章时可以传 `cover_image` 字段，把封面图 URL 关联到文章。
- 上传超过 5MB 的文件会被拒绝（返回 413）。
- 上传非图片文件（如 `.exe`）会被拒绝。
- 你能说出为什么不能信任用户上传的文件名，以及什么是"路径遍历攻击"。

## 三、分步操作

### 步骤 1：安装依赖——`python-multipart`

FastAPI 处理文件上传需要 `python-multipart` 这个库。没有它，上传接口会报错。

```powershell
pip install python-multipart
```

**验证**：

```powershell
python -c "import multipart; print('python-multipart 安装成功')"
```

**为什么需要这个库？** 文件上传用的是 `multipart/form-data` 编码格式（和普通 JSON 请求的 `application/json` 不同）。此术语需进附录。`python-multipart` 负责解析这种格式的请求体，把文件数据从原始字节流中提取出来。

**比喻**：普通 JSON 请求就像寄一封信——信纸（请求体）上直接写内容。文件上传就像寄一个包裹——包裹里有好几样东西（文件 + 表单字段），需要专门的"拆包裹工具"（`python-multipart`）来分别取出。

### 步骤 2：创建上传目录

我们先把文件存到 `blog_backend/uploads/` 目录下。这个目录需要手动创建，并且要在服务器启动时自动检测——如果不存在就创建，防止因为目录不存在而报错。

```powershell
mkdir blog_backend\uploads
```

但更好的做法是**在代码中自动创建**——这样不管谁部署你的项目，都不用担心忘记创建目录。

打开 `blog_backend/main.py`，在 `app = FastAPI()` 之后加上启动事件：

```python
# blog_backend/main.py —— 在 app = FastAPI() 之后追加

import os

UPLOAD_DIR = "uploads"

# 启动时自动创建上传目录
@app.on_event("startup")
def create_upload_dir():
    os.makedirs(UPLOAD_DIR, exist_ok=True)
```

**逐行解释**：
- `UPLOAD_DIR = "uploads"`：定义上传目录路径。相对于服务器启动时的工作目录（通常是 `blog_backend/` 的父目录）。
- `@app.on_event("startup")`：FastAPI 的启动事件钩子。服务器启动时自动执行这个函数。
- `os.makedirs(UPLOAD_DIR, exist_ok=True)`：创建目录。`exist_ok=True` 的意思是：如果目录已经存在，不报错，直接跳过。

**比喻**：`on_event("startup")` 就像酒店开业前，经理检查所有房间是否准备好——缺椅子补椅子，但不会因为"椅子已摆好"而发脾气。

❌ 常见错误 → ✅ 正确示例：

❌ 错误：忘记创建目录，上传时直接报错

```python
# 错误：没有创建 uploads/ 目录
with open(f"uploads/{filename}", "wb") as f:
    f.write(data)
# → FileNotFoundError: uploads/ 目录不存在！
```

✅ 正确：启动时自动创建

```python
@app.on_event("startup")
def create_upload_dir():
    os.makedirs("uploads", exist_ok=True)
```

### 步骤 3：实现文件上传接口

现在写核心的上传接口。打开 `blog_backend/main.py`，在文件顶部追加导入，然后添加接口：

```python
# blog_backend/main.py —— 在文件顶部追加导入

import uuid
from fastapi import UploadFile, File


# === 上传封面图 ===
@app.post("/posts/upload-image")
async def upload_image(file: UploadFile = File(...)):
    # 1. 检查文件类型
    allowed_types = ["image/jpeg", "image/png", "image/gif", "image/webp"]
    if file.content_type not in allowed_types:
        raise HTTPException(
            status_code=400,
            detail=f"不支持的文件类型：{file.content_type}。仅支持 JPEG、PNG、GIF、WebP。"
        )

    # 2. 读取文件内容
    contents = await file.read()

    # 3. 检查文件大小（限制 5MB）
    max_size = 5 * 1024 * 1024  # 5MB
    if len(contents) > max_size:
        raise HTTPException(
            status_code=413,
            detail=f"文件过大：{len(contents) / 1024 / 1024:.1f}MB。最大允许 5MB。"
        )

    # 4. 生成唯一文件名（用 UUID 避免冲突）
    ext = os.path.splitext(file.filename)[1]  # 获取扩展名，如 ".jpg"
    unique_name = f"{uuid.uuid4()}{ext}"

    # 5. 保存文件
    file_path = os.path.join(UPLOAD_DIR, unique_name)
    with open(file_path, "wb") as f:
        f.write(contents)

    # 6. 返回可访问的 URL
    return {
        "url": f"/static/{unique_name}",
        "filename": unique_name,
    }
```

**逐行解释**：

- `file: UploadFile = File(...)`：FastAPI 的文件上传参数。`UploadFile` 是 FastAPI 提供的文件类型，`File(...)` 表示这个参数是必需的（`...` 是 Python 的 `Ellipsis`，相当于"必填"）。此术语需进附录：UploadFile。
- `file.content_type`：文件的 MIME 类型，由浏览器在上传时提供。比如 JPEG 图片的 `content_type` 是 `"image/jpeg"`。此术语需进附录：MIME 类型。
- `await file.read()`：读取文件的所有字节内容。`UploadFile` 的方法是异步的，所以前加 `await`，函数定义要加 `async`。
- `len(contents)`：文件的实际字节数，用于检查文件大小。5MB = 5 × 1024 × 1024 字节。
- `uuid.uuid4()`：生成一个随机的 UUID（通用唯一标识符），比如 `a1b2c3d4-e5f6-7890-abcd-ef1234567890`。用 UUID 做文件名，保证永远不会重名。此术语需进附录：UUID。
- `os.path.splitext(file.filename)[1]`：从原始文件名中提取扩展名。`"photo.jpg"` → `".jpg"`。保留扩展名是为了让浏览器知道这是什么类型的文件。
- `file.filename`：用户上传时的原始文件名。**注意**：我们只用它来提取扩展名，不直接用它做存储文件名——因为用户可能上传 `../../../etc/passwd` 这样的恶意文件名（路径遍历攻击）。
- `with open(file_path, "wb") as f`：`"wb"` = 二进制写入模式。图片是二进制数据，不能用文本模式 `"w"`。

**六个步骤的流程图**：

```
用户上传图片 → ① 检查文件类型 → ② 读取文件内容 → ③ 检查文件大小
→ ④ 生成 UUID 文件名 → ⑤ 保存到 uploads/ → ⑥ 返回 /static/xxx.jpg
```

**比喻**：这个接口就像一个快递收发室：
1. 先看包裹里是什么（检查文件类型——只收图片，拒收炸弹）
2. 打开包裹取出内容（读取文件）
3. 称重（检查大小——超过 5kg 不收）
4. 贴上新的标签（生成 UUID 文件名——不信任寄件人写的名字）
5. 放到仓库货架上（保存到 uploads/）
6. 告诉你货架编号（返回 URL）

### 步骤 4：静态文件服务——让上传的图片能被访问

文件存到 `uploads/` 目录了，但浏览器怎么访问它？你需要配置**静态文件服务**——告诉 FastAPI 把 `uploads/` 目录下的文件直接暴露给 HTTP 访问。

在 `blog_backend/main.py` 中，`app = FastAPI()` 之后追加：

```python
# blog_backend/main.py —— 在 app = FastAPI() 之后追加

from fastapi.staticfiles import StaticFiles

# 静态文件服务：把 uploads/ 目录映射到 /static 路径
app.mount("/static", StaticFiles(directory="uploads"), name="static")
```

**逐行解释**：
- `StaticFiles(directory="uploads")`：告诉 FastAPI，`uploads/` 目录里的文件可以作为静态文件直接访问。
- `app.mount("/static", ...)`：把 `/static` 这个 URL 路径"挂载"到 `uploads/` 目录。访问 `http://127.0.0.1:8000/static/abc.jpg` 时，FastAPI 会去 `uploads/abc.jpg` 找文件并返回。
- `name="static"`：内部标识名称，随便取。

**比喻**：`app.mount` 就像在网站上开了一个"展示窗口"——`/static` 是窗口地址，`uploads/` 是仓库。客户说"我要看 `/static/abc.jpg`"，工作人员就从仓库里取出 `abc.jpg` 展示。

**验证**：上传一张图片后，用浏览器访问 `http://127.0.0.1:8000/static/<文件名>`，应该能看到图片。

### 步骤 5：封面图关联到文章——Post 模型加字段

现在图片能上传了，但上传的图片和文章还没关联——我们需要在 `Post` 模型中加一个 `cover_image` 字段来存储封面图 URL。

打开 `blog_backend/models.py`，在 `Post` 类中追加：

```python
# blog_backend/models.py —— 在 Post 类中追加

from sqlalchemy import Column, Integer, String, Boolean, DateTime, ForeignKey, Text


class Post(Base):
    __tablename__ = "posts"

    id = Column(Integer, primary_key=True, index=True)
    title = Column(String, nullable=False)
    content = Column(Text, nullable=False)
    published = Column(Boolean, default=False)
    summary = Column(String, nullable=True)
    cover_image = Column(String, nullable=True)  # 新增：封面图 URL
    author_id = Column(Integer, ForeignKey("users.id"), nullable=True)
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
```

**`cover_image = Column(String, nullable=True)`**：存储封面图的 URL（如 `/static/abc.jpg`）。`nullable=True` 表示文章不一定要有封面图。

**⚠️ 这是修改了数据库表结构**——如果你之前已经创建过 `blog.db`，需要重建数据库（删除 `blog.db` 后重新运行服务器，FastAPI + SQLAlchemy 会自动建表）。或者手动执行：

```sql
ALTER TABLE posts ADD COLUMN cover_image VARCHAR;
```

**建议**：直接在项目目录下删除 `blog.db`，重启服务器让它自动重建。**注意：这会丢失所有数据，仅开发阶段这样做。**

```python
# === 创建文章（支持封面图） ===
@app.post("/posts")
def create_post(
    blog: BlogCreate,
    db: Session = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    new_post = Post(
        title=blog.title,
        content=blog.content,
        published=blog.published,
        summary=blog.summary,
        cover_image=blog.cover_image,  # 新增：封面图 URL
        author_id=current_user.id,
    )
    db.add(new_post)
    db.commit()
    db.refresh(new_post)
    return new_post
```

### 步骤 7：⚠️ 安全警告——不能信任用户上传的文件名

你可能会想：为什么不用用户上传的原始文件名？`cover.jpg` 不是比 `a1b2c3d4-e5f6-7890-abcd-ef1234567890.jpg` 更好看吗？

**因为恶意用户可以利用文件名攻击你的服务器。**

**路径遍历攻击**（Path Traversal Attack）：此术语需进附录。

用户上传一个文件，文件名是：

```
../../../etc/passwd
```

如果你直接用这个文件名保存：

```python
# ❌ 危险：直接用用户传来的文件名
with open(f"uploads/{file.filename}", "wb") as f:
    f.write(contents)
```

`os.path.join("uploads", "../../../etc/passwd")` 会解析成 `/etc/passwd`——你的服务器系统文件被覆盖了！

**`..` 在路径中表示"上一级目录"**。攻击者可以用多个 `../` 跳出 `uploads/` 目录，访问系统任意文件。

**比喻**：你开了一家快递代收点，规定所有包裹放仓库 A 区。有人寄了一个包裹，收件地址写的是"仓库 A 区 / ../../经理办公室 / 保险柜"——如果你不检查地址，包裹就放到经理办公室了。

**防御措施**：

1. **用 UUID 重命名**（我们已经在做了）：不信任用户传来的文件名，自己生成一个完全随机的名字。
2. **限制文件类型**：只接受图片类型（`image/jpeg`、`image/png` 等），拒绝 `.exe`、`.php`、`.sh` 等可执行文件。
3. **限制文件大小**：防止用户上传超大文件耗尽服务器磁盘空间。

❌ 常见错误 → ✅ 正确示例：

❌ 错误：直接使用用户文件名

```python
# 危险！用户文件名可能是 "../../../etc/passwd"
file_path = f"uploads/{file.filename}"
with open(file_path, "wb") as f:
    f.write(contents)
```

✅ 正确：用 UUID 生成安全的文件名

```python
ext = os.path.splitext(file.filename)[1]  # 只取扩展名
safe_name = f"{uuid.uuid4()}{ext}"        # 随机 UUID + 扩展名
file_path = os.path.join("uploads", safe_name)
with open(file_path, "wb") as f:
    f.write(contents)
```

#### 8.4 上传超大文件 → 应被拒绝

```bash

$f = [System.IO.File]::Create("dummy.bin"); $f.SetLength(6 * 1024 * 1024); $f.Close()
```

> 实际上纯随机字节的 `.jpg` 可能通不过类型检查（因为 `content_type` 不会被伪造的扩展名改变）。真正的测试可以跳过类型检查单独测试大小限制，或者用一个真实的超大 JPEG 图片。

```python
@app.post("/posts/upload-image")
async def upload_image(file: UploadFile = File(...)):
    # 检查文件类型
    allowed_types = ["image/jpeg", "image/png", "image/gif", "image/webp"]
    if file.content_type not in allowed_types:
        raise HTTPException(
            status_code=400,
            detail=f"不支持的文件类型：{file.content_type}。仅支持 JPEG、PNG、GIF、WebP。"
        )

    # 读取文件内容
    contents = await file.read()

    # 检查文件大小
    max_size = 5 * 1024 * 1024
    if len(contents) > max_size:
        raise HTTPException(
            status_code=413,
            detail=f"文件过大：{len(contents) / 1024 / 1024:.1f}MB。最大允许 5MB。"
        )

    # 生成唯一文件名
    ext = os.path.splitext(file.filename)[1]
    unique_name = f"{uuid.uuid4()}{ext}"

    # 保存文件
    file_path = os.path.join(UPLOAD_DIR, unique_name)
    with open(file_path, "wb") as f:
        f.write(contents)

    return {
        "url": f"/static/{unique_name}",
        "filename": unique_name,
    }


# === 创建文章（支持封面图） ===
@app.post("/posts")
def create_post(
    blog: BlogCreate,
    db: Session = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    new_post = Post(
        title=blog.title,
        content=blog.content,

        summary=blog.summary,
        cover_image=blog.cover_image,  # 新增
        author_id=current_user.id,
    )
    db.add(new_post)
    db.commit()
    db.refresh(new_post)
    return new_post
```

## 八、已知坑点与禁止事项

| 坑点 | 现象 | 原因 | 解决 |
|------|------|------|------|
| 忘记安装 `python-multipart` | 上传接口报错 `RuntimeError: File field is not supported` | FastAPI 文件上传依赖 `python-multipart` 解析请求体 | `pip install python-multipart` |
| 上传目录不存在 | 保存文件时报 `FileNotFoundError` | `uploads/` 目录没创建 | 用 `@app.on_event("startup")` + `os.makedirs(exist_ok=True)` 自动创建 |
| 直接使用用户文件名 | 恶意文件名 `../../../etc/passwd` 覆盖系统文件 | 没有过滤用户输入的文件名 | 用 `uuid.uuid4()` 生成唯一文件名，只用原始文件名的扩展名 |
| 忘记 `async` 和 `await` | 报错 `coroutine object is not callable` | `UploadFile.read()` 是异步方法 | 函数声明加 `async def`，读取加 `await file.read()` |
| `StaticFiles` 路径不对 | 浏览器访问 `/static/xxx.jpg` 返回 404 | `directory` 参数指定的路径不存在或不在当前工作目录 | 检查 `uploads/` 目录是否在服务器启动目录下，`app.mount` 和 `UPLOAD_DIR` 路径是否一致 |
| 文件类型检查被绕过 | 攻击者上传 `.exe` 文件但 `content_type` 是 `image/jpeg` | 客户端可以伪造 `Content-Type` 头 | 生产环境应检查文件内容的魔术字节（magic bytes），不只是 `content_type` |
| 更新文章时忘记传 `cover_image` | 更新文章后封面图变成 `null` | `PUT /posts/{id}` 用的是 `BlogCreate` Schema，如果没传 `cover_image` 就覆盖为 `null` | 更新接口中先检查 `blog.cover_image` 是否有值，有值才更新 |
