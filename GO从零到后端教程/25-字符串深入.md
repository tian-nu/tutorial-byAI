## 25.1 字符串的底层结构

在Go的运行时（runtime）源码里，字符串的内部结构定义如下：

```go
type stringStruct struct {
    str unsafe.Pointer
    len int
}
```

就两个字段：
- **str**：一个指针，指向存储实际字符数据的连续内存区域
- **len**：字符串的**字节长度**（不是字符数！）

比如字符串 `"Hello"` 在内存中大致长这样：

```
┌─────────────────┐
│ str ──────────→ │ H │ e │ l │ l │ o │
│ len = 5        │
└─────────────────┘
```

字符串 `"你好"` 在内存中：

```
┌─────────────────┐
│ str ──────────→ │ E4 BD A0 │ E5 A5 BD │ (UTF-8编码：3+3=6字节)
│ len = 6        │
└─────────────────┘
```

由于字符串底层用指针指向数据，所以：
- 赋值字符串（`s2 := s1`）只复制指针和len（16字节），不复制底层字符数据
- 但字符串是不可变的，所以你不用担心s2改了会影响s1——反正谁也改不了

### 为什么字符串不可变

字符串不可变有两个好处：

1. **安全**：一个字符串可以被多处引用，不用担心某处悄悄改了它。这在并发编程中特别重要。
2. **高效**：字符串比较时可以先比较len，不等就直接返回false；相等再比较内容。哈希计算也可以缓存。

如果你需要修改字符串，转成 `[]byte` 或 `[]rune`，改完再转回来：

```go
s := "Hello"
b := []byte(s)
b[0] = 'h'
s = string(b)
fmt.Println(s)  // "hello"
```

## 25.2 strings包：字符串操作的瑞士军刀

`strings` 包是Go标准库中专门处理字符串的包，里面装满了实用函数。下面按类别逐一介绍。

### 包含判断

```go
strings.Contains("hello world", "world")   // true
strings.Contains("hello world", "goodbye") // false

strings.ContainsAny("hello", "aeiou")      // true（包含aeiou中任意一个）
strings.ContainsRune("你好", '你')          // true（包含某个rune）
```

### 前缀和后缀

```go
strings.HasPrefix("hello world", "hello")  // true
strings.HasPrefix("hello world", "heaven") // false

strings.HasSuffix("hello.txt", ".txt")     // true
strings.HasSuffix("hello.txt", ".go")      // false
```

### 查找位置

```go
strings.Index("hello world", "world")      // 6（从0开始数，首次出现的位置）
strings.Index("hello world", "x")          // -1（没找到）

strings.LastIndex("go gopher", "go")       // 3（最后一次出现的位置）
```

### 切割和拼接

```go
parts := strings.Split("a,b,c,d", ",")
fmt.Println(parts)  // ["a" "b" "c" "d"]

parts = strings.SplitN("a,b,c,d", ",", 2)
fmt.Println(parts)  // ["a" "b,c,d"]（最多切成2段）

joined := strings.Join([]string{"a", "b", "c"}, "-")
fmt.Println(joined)  // "a-b-c"
```

### 大小写转换

```go
strings.ToUpper("hello")  // "HELLO"
strings.ToLower("HELLO")  // "hello"
strings.Title("hello world")  // "Hello World"（每个单词首字母大写）
```

### 去空白

```go
strings.TrimSpace("  hello  ")   // "hello"（去掉首尾空白）
strings.Trim("###hello###", "#") // "hello"（去掉首尾指定的字符）
strings.TrimLeft("###hello###", "#")  // "hello###"
strings.TrimRight("###hello###", "#") // "###hello"
```

### 替换

```go
strings.Replace("hello hello", "hello", "hi", 1)   // "hi hello"（只换第一个）
strings.Replace("hello hello", "hello", "hi", -1)  // "hi hi"（-1表示全换）
strings.ReplaceAll("hello hello", "hello", "hi")   // "hi hi"（Go 1.12+）
```

### 重复

```go
strings.Repeat("Go! ", 3)  // "Go! Go! Go! "
```

### 比较（忽略大小写）

```go
strings.EqualFold("Go", "go")  // true（不区分大小写比较）
```

### strings.Builder：高效拼接字符串

```go
var builder strings.Builder

builder.WriteString("Hello")
builder.WriteString(", ")
builder.WriteString("World!")

result := builder.String()
fmt.Println(result)  // "Hello, World!"
```

`strings.Builder` 内部用一个 `[]byte` 来存数据，`String()` 时才转成不可变的 `string`。这是官方推荐的高效拼接方式。

`Builder` 还可以预分配内存避免扩容：

```go
var builder strings.Builder
builder.Grow(100)  // 预分配100字节，避免写的时候扩容
```

> 🤔 **想多一点**：你可能会发现很多函数都能用其他函数组合出来。比如 `strings.Contains(s, sub)` 本质上就是 `strings.Index(s, sub) >= 0`。那为什么Go要单独提供 `Contains`？因为代码可读性——`if strings.Contains(name, "@")` 比 `if strings.Index(name, "@") >= 0` 更好理解。标准库多做一个函数，换来的是所有Go程序员少想一步。

## 25.3 strconv包：字符串与数字互转

`strconv`（String Conversion）是字符串与各种数值类型互转的专用包。第18章我们提到了 `Itoa` 和 `Atoi`，这里做系统性的介绍。

### 整数 ⇄ 字符串

```go
s := strconv.Itoa(42)
fmt.Println(s)    // "42"
fmt.Printf("%T\n", s)  // string

i, err := strconv.Atoi("42")
if err == nil {
    fmt.Println(i)     // 42
    fmt.Printf("%T\n", i)  // int
}

i, err = strconv.Atoi("abc")
fmt.Println(err)  // strconv.Atoi: parsing "abc": invalid syntax
```

`Itoa` = Integer to ASCII，`Atoi` = ASCII to Integer。

### FormatInt / ParseInt：带进制的转换

```go
s := strconv.FormatInt(255, 16)   // 255转成16进制字符串
fmt.Println(s)  // "ff"

s = strconv.FormatInt(255, 2)     // 255转成2进制
fmt.Println(s)  // "11111111"

i, err := strconv.ParseInt("ff", 16, 64)  // 16进制"ff"转成int64
fmt.Println(i)  // 255
```

第三个参数 `64` 表示结果类型是 `int64`（可以是0/8/16/32/64）。`ParseInt` 返回 `int64`，但你可以转成需要的大小。

### 浮点数 ⇄ 字符串

```go
s := strconv.FormatFloat(3.14159, 'f', 2, 64)
fmt.Println(s)  // "3.14"（保留2位小数）

f, err := strconv.ParseFloat("3.14", 64)
fmt.Println(f)  // 3.14
```

`FormatFloat` 的参数：
- `'f'`：格式（f=普通小数，e=科学计数法，g=自动选择）
- `2`：小数点后保留几位
- `64`：输入是float64

### 布尔 ⇄ 字符串

```go
s := strconv.FormatBool(true)
fmt.Println(s)  // "true"

b, err := strconv.ParseBool("true")
fmt.Println(b)  // true
```

`ParseBool` 接受 `"true"`、`"false"`、`"1"`、`"0"` 等写法。

## 25.4 拼接效率：四种方式的性能对决

在Go里拼接字符串至少有四种方式。它们的性能差距非常大，尤其是在大数据量时。

### 方式一：`+` 运算符

```go
s := ""
for i := 0; i < 10000; i++ {
    s += "a"
}
```

**效率：最低。**

每次 `+=` 都会创建一个全新的字符串（因为字符串不可变），旧字符串的内容被完整复制到新字符串中。循环10000次就是10000次分配+复制，时间复杂度是 O(n²)。

### 方式二：`fmt.Sprintf`

```go
s := fmt.Sprintf("%s%s%s", "Hello", ", ", "World!")
```

**效率：低。**

`fmt.Sprintf` 底层用了反射机制，有额外的解析开销。适合写少量格式化字符串，不适合大量拼接。

### 方式三：`bytes.Buffer`

```go
var buf bytes.Buffer
for i := 0; i < 10000; i++ {
    buf.WriteString("a")
}
s := buf.String()
```

**效率：高。**

`bytes.Buffer` 内部维护一个可增长的 `[]byte`，`WriteString` 时直接append，最后 `String()` 时才转成不可变的string。少了大量不必要的内存分配。

### 方式四：`strings.Builder`（推荐）

```go
var builder strings.Builder
for i := 0; i < 10000; i++ {
    builder.WriteString("a")
}
s := builder.String()
```

**效率：最高，Go 1.10+官方推荐。**

`strings.Builder` 内部也是 `[]byte`，但它比 `bytes.Buffer` 做了更多优化——比如 `String()` 时用 `unsafe` 避免了一次拷贝。而且语义更纯粹：它就是用来"构建字符串"的。

### 性能对比（定性）

| 方式 | 速度 | 内存分配 | 适用场景 |
|------|------|----------|----------|
| `+` | 🐌 极慢（O(n²)） | 极多 | 只拼少数几个字符串 |
| `fmt.Sprintf` | 🐢 慢 | 多 | 需要格式化时 |
| `bytes.Buffer` | 🐇 快 | 少 | 大量拼接（Go 1.9及以下） |
| `strings.Builder` | 🚀 最快 | 最少 | 大量拼接（Go 1.10+，推荐） |

### 经验法则

- 只拼**3~5个**字符串 → 用 `+`，简单直观
- 需要**格式化**（插入数字、浮点数等） → `fmt.Sprintf`
- **循环中大量拼接**（几百、几千次以上） → `strings.Builder`
- **不确定** → `strings.Builder`，永远不会错

```go
// ✅ 好的写法：少量就用+
greeting := "你好，" + name + "，今天天气" + weather

// ✅ 好的写法：大量就用Builder
var b strings.Builder
b.Grow(estimatedSize)
for _, item := range items {
    b.WriteString(item.Name)
    b.WriteString(",")
}
result := b.String()
```

### 为什么Builder比Buffer快

`bytes.Buffer` 的 `String()` 方法会把内部的 `[]byte` 强制复制一份，生成一个新的不可变string。而 `strings.Builder` 的 `String()` 方法用了 `unsafe.Pointer`——直接在原地把 `[]byte` 转成string，避免了字节复制。

不过这个优化对初学者来说是透明的——你不需要知道怎么用 `unsafe`，只需要知道结论：**大量拼接时用 `strings.Builder`。**

> 🤔 **想多一点**：一个简单的 `+=` 和 `Builder` 的差距能有多大？如果你在循环里拼10000次，`+=` 可能需要几百毫秒，`Builder` 只需要几毫秒。这不是"差不多"，这是"差两个数量级"。Go团队专门开发了 `strings.Builder` 就是为了解决这个性能问题。所以当一个Go程序员说"拼接字符串慢"时，老手的第一反应就是——"你是不是在用 `+=`？"

## 本章小结

| 知识点 | 核心要点 |
|--------|----------|
| 底层结构 | 指针+长度（不存容量）；不可变，赋值只拷贝指针和len |
| strings.Contains | 判断子串是否存在 |
| strings.HasPrefix/Suffix | 判断前缀/后缀 |
| strings.Split/Join | 切割拼回，SplitN限制段数 |
| strings.Replace/ReplaceAll | 替换子串，-1表示全替换 |
| strings.Builder | 高效拼接，WriteString + String()；可Grow预分配 |
| strconv.Itoa/Atoi | 整数 ⇄ 字符串 |
| strconv.FormatInt/ParseInt | 带进制的转换（二进制、十六进制等） |
| strconv.ParseFloat | 字符串 → 浮点数 |
| 拼接效率 | Builder > Buffer > Sprintf > +；循环中大量拼接必须用Builder |
