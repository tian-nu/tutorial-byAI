# 第6章 · IP地址

> "互联网是怎么找到你的电脑的？答案是IP地址。它就像每家每户的门牌号。但为什么又有公网IP和内网IP？127.0.0.1是什么？"

---

## 6.1 IP地址是门牌号

互联网由无数设备组成——你的电脑、你的手机、百度服务器、微信服务器、你家的路由器...

数据在互联网上传输，就像快递在城市里运输。如果快递单上没有地址，快递员怎么送？

**IP地址 = 互联网上的门牌号**，每一台连到互联网的设备都必须有一个IP地址。

IP地址（IPv4）的格式：`192.168.1.100`

4个数字，每个数字0~255，用点隔开。

为什么是0~255？因为每个数字占用**8个二进制位**（1字节），2⁸ = 256种可能（0到255）。

`192.168.1.100` 在计算机眼里其实就是32个0和1：
```
11000000.10101000.00000001.01100100
```

---

## 6.2 IPv4 vs IPv6

### IPv4：不够用了

IPv4有32位，总共能表示约**43亿**个地址。

43亿听起来很多，但地球上的人有80亿，每个人还不止一个设备（手机、电脑、平板、智能电视...）。

解决方案之一是NAT（后面讲），让多台设备共享一个公网IP。但根本解决方案是IPv6。

### IPv6：几乎无限

IPv6有128位，能表示的地址数量是：

> 2¹²⁸ ≈ 340,282,366,920,938,463,463,374,607,431,768,211,456 个

大约是**340万亿亿亿**个。地球上每一粒沙子都能分配一个IPv6地址还有多。

IPv6地址长这样：
```
2001:0db8:85a3:0000:0000:8a2e:0370:7334
```

用冒号分隔的8组十六进制数（可以简写，连续的0省略）。

为什么IPv6还没全面普及？
- 升级成本：所有网络设备（路由器、交换机）都要支持
- 兼容性：老应用只认识IPv4
- 但趋势是明确的，全球IPv6使用率已经超过40%

---

## 6.3 公网IP vs 内网IP

### 全家用一个公网IP（NAT）

你家里的情况大概是这样的：

```
互联网 ─── [你家的路由器: 公网IP 123.45.67.89]
                 │
      ┌──────────┼──────────┐
      │          │          │
  你的电脑    你的手机    智能电视
  192.168.1.2 192.168.1.3 192.168.1.4
```

- 路由器有一个**公网IP**（`123.45.67.89`），这个IP在全球互联网上独一无二
- 你家里的设备都有**内网IP**（`192.168.1.x`），只在你自己家这个小网络里有效

当你用电脑访问百度时：
1. 数据包从电脑发出：源地址 `192.168.1.2`
2. 经过路由器时，路由器把源地址改成 `123.45.67.89`（NAT转换）
3. 百度收到请求，看到的是路由器的公网IP
4. 百度回复给 `123.45.67.89`
5. 路由器收到回复，根据记录判断"这是刚才电脑发的请求"，转发给 `192.168.1.2`

**这就是NAT（Network Address Translation，网络地址转换）**——让整个家庭共用一个公网IP上网。

### 内网IP段

以下IP段被保留为内网专用，不会出现在公网上：

```
10.0.0.0 ~ 10.255.255.255      （A类内网）
172.16.0.0 ~ 172.31.255.255    （B类内网）
192.168.0.0 ~ 192.168.255.255  （C类内网，最常用）
```

如果你看到自己的IP是 `192.168.x.x`，你就在内网里。

### 比喻

- **公网IP** = 你小区的地址："阳光花园 3号楼"
- **内网IP** = 你的房间号："502室"
- **NAT** = 快递送到小区门口（公网IP），门卫（路由器）根据记录帮你送到502室

## 6.4 localhost 和 127.0.0.1

### 这是"我自己"

`127.0.0.1` 是一个特殊的IP地址——它永远指向**本机**。

`localhost` 是 `127.0.0.1` 的域名（更可读的名字）。

```bash
$ ping 127.0.0.1     # ping自己
$ ping localhost     # 同上
```

当你启动一个Go Web服务器：

```go
r := gin.Default()
r.Run("127.0.0.1:8080")  // 只能本机访问
r.Run("0.0.0.0:8080")    // 任何IP都能访问
```

- `127.0.0.1:8080`：只有你自己这台电脑能访问
- `0.0.0.0:8080`：局域网里任何人都能通过你的IP访问

### 为什么开发时用 localhost

- 不需要网络，数据在操作系统内部就转了一圈
- 速度快（不走网卡）
- 安全（外部访问不到）

整个 `127.x.x.x` 网段（127.0.0.0 ~ 127.255.255.255）都是回环地址（Loopback），都指向本机。

## 6.5 子网掩码

### 判断两个IP在不在同一个局域网

子网掩码（Subnet Mask）用于判断两个IP地址是否在**同一个局域网**内。

```
IP地址：      192.168.1.100
子网掩码：    255.255.255.0
```

做"按位与"运算：
- `192.168.1.100` 和 `255.255.255.0` 做AND → `192.168.1.0`（网络号）

如果另一个IP `192.168.1.200` 和子网掩码做AND也得到 `192.168.1.0` → 同一个网络，直接通信。

如果 `10.0.5.3` 做AND得到 `10.0.5.0` → 不同网络，需要通过路由器。

### CIDR表示法

`192.168.1.0/24`

- `/24` = 子网掩码前24位是1，即 `255.255.255.0`
- `/16` = `255.255.0.0`
- `/8` = `255.0.0.0`

理解了就行，实际配置网络时偶尔会用到。

## 本章小结

- IP地址 = 互联网上的门牌号
- IPv4 不够用了，IPv6 是未来
- 公网IP全球唯一，内网IP只在局域网有效
- NAT让多个设备共享一个公网IP
- `127.0.0.1` / `localhost` = 我自己
