## 我在做什么？

上一章我们学会了 `fork()` —— 创建独立的进程。但进程有一个问题：**太重了**。创建进程需要复制整个地址空间，进程间通信（IPC）也很复杂。如果只是想让程序"同时做多件事"（比如一个线程处理网络请求，一个线程更新界面），进程的开销太大了。

**线程**是进程内部的"轻量级执行流"。多个线程共享同一个进程的内存空间，创建和切换的开销远小于进程。但代价是：共享内存带来了**竞态条件**——多个线程同时修改同一份数据，结果不可预测。

想象一个厨房（进程）。如果多个厨师（线程）共用同一个灶台、同一把刀、同一个冰箱，他们必须协调好——"我先拿刀，你等一下"——否则就会出事故。这就是线程同步要解决的问题。

`*此术语见附录F*`：线程、pthread、POSIX线程、竞态条件、临界区、互斥锁、mutex、死锁。

## 54.1 线程 vs 进程

```
┌─────────────────────────────────────────────────────────┐
│                    进程 (Process)                        │
│  ┌─────────────────────────────────────────────────┐    │
│  │ 代码段  │ 数据段  │ 堆      │ 文件描述符          │    │
│  └─────────────────────────────────────────────────┘    │
│                                                         │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │ 线程 1   │  │ 线程 2   │  │ 线程 3   │              │
│  │ 自己的栈 │  │ 自己的栈 │  │ 自己的栈 │              │
│  │ 自己的寄存器│ │ 自己的寄存器│ │ 自己的寄存器│             │
│  └──────────┘  └──────────┘  └──────────┘              │
│       │              │              │                   │
│       └──────────────┼──────────────┘                   │
│                      │                                  │
│              共享：代码段、数据段、堆、文件描述符            │
└─────────────────────────────────────────────────────────┘
```

| 对比维度 | 进程 | 线程 |
|----------|------|------|
| 创建开销 | 高（复制整个地址空间） | 低（只需栈和寄存器） |
| 切换开销 | 高（切换页表等） | 低（同进程内切换） |
| 内存空间 | 独立 | 共享（除栈外） |
| 通信方式 | IPC（管道、消息队列、共享内存） | 直接读写共享变量 |
| 独立性 | 一个崩溃不影响其他 | 一个崩溃可能拖垮整个进程 |
| 创建函数 | `fork()` | `pthread_create()` |

## 54.2 pthread_create：创建线程

POSIX 线程（pthread）是 Linux/macOS 上的标准线程库。编译时需要链接 `-pthread`：

```
gcc -std=c99 -Wall -pthread -o thread_demo thread_demo.c
```

### 第一个多线程程序

```c
#include <stdio.h>
#include <pthread.h>
#include <unistd.h>  /* sleep() */

/* 线程函数：签名必须是 void* func(void*) */
void* thread_worker(void *arg) {
    int thread_id = *(int*)arg;
    printf("[线程 %d] 开始工作\n", thread_id);
    sleep(2);
    printf("[线程 %d] 工作完成\n", thread_id);
    return NULL;
}

int main(void) {
    pthread_t t1, t2;
    int id1 = 1, id2 = 2;

    printf("[主线程] 创建线程 1 和 2\n");

    /* pthread_create(线程ID指针, 属性(通常NULL), 线程函数, 参数) */
    pthread_create(&t1, NULL, thread_worker, &id1);
    pthread_create(&t2, NULL, thread_worker, &id2);

    printf("[主线程] 等待线程结束...\n");

    /* pthread_join: 等待线程结束（类似 wait 之于进程） */
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);

    printf("[主线程] 所有线程已结束\n");
    return 0;
}
```

运行输出（顺序可能不同）：
```
[主线程] 创建线程 1 和 2
[主线程] 等待线程结束...
[线程 1] 开始工作
[线程 2] 开始工作
[线程 1] 工作完成
[线程 2] 工作完成
[主线程] 所有线程已结束
```

### 线程函数签名

```c
void* thread_function(void *arg) {
    /* arg 是 pthread_create 传入的第四个参数 */
    /* 返回 void* 指针，pthread_join 可以获取这个返回值 */
    return (void*)42;  /* 返回一个值 */
}
```

### 线程传递参数

```c
#include <stdio.h>
#include <pthread.h>
#include <stdlib.h>

typedef struct {
    int  start;
    int  end;
    long result;
} TaskData;

void* compute_sum(void *arg) {
    TaskData *data = (TaskData*)arg;
    data->result = 0;
    for (int i = data->start; i <= data->end; i++) {
        data->result += i;
    }
    return NULL;
}

int main(void) {
    pthread_t t1, t2;
    TaskData d1 = {1, 500000, 0};
    TaskData d2 = {500001, 1000000, 0};

    pthread_create(&t1, NULL, compute_sum, &d1);
    pthread_create(&t2, NULL, compute_sum, &d2);

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);

    printf("总和: %ld\n", d1.result + d2.result);
    return 0;
}
```

## 54.3 竞态条件：多线程的噩梦

当多个线程同时读写同一变量，且结果依赖于执行顺序时，就产生了**竞态条件**：

```c
#include <stdio.h>
#include <pthread.h>

#define INCREMENTS 1000000

int counter = 0;  /* 全局变量，所有线程共享 */

void* increment(void *arg) {
    for (int i = 0; i < INCREMENTS; i++) {
        counter++;  /* 不是原子操作！ */
    }
    return NULL;
}

int main(void) {
    pthread_t t1, t2;

    pthread_create(&t1, NULL, increment, NULL);
    pthread_create(&t2, NULL, increment, NULL);

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);

    printf("预期: %d, 实际: %d\n", 2 * INCREMENTS, counter);
    return 0;
}
```

运行输出（实际值每次不同）：
```
预期: 2000000, 实际: 1523784    <-- 丢失了约 47 万次操作！
```

### 为什么 counter++ 不是原子的？

```
counter++ 在 CPU 层面分解为三步：

线程1                          线程2
  │                               │
  │ 1. 从内存读取 counter 到寄存器 │
  │    (counter=100)              │
  │                               │ 1. 从内存读取 counter 到寄存器
  │                               │    (counter=100)  ← 也是100！
  │                               │
  │ 2. 寄存器 +1 → 101            │
  │                               │ 2. 寄存器 +1 → 101
  │                               │
  │ 3. 写回内存 (counter=101)     │
  │                               │ 3. 写回内存 (counter=101)
  │                               │
  ▼                               ▼
 结果：两次 counter++ 只增加了 1！
```

这就是竞态条件：两个线程同时读-改-写，后写入的覆盖了前一个的结果。

## 54.4 互斥锁（Mutex）：保护临界区

**互斥锁**（Mutual Exclusion Lock）确保同一时刻只有一个线程能进入**临界区**——被保护的代码段。

```
互斥锁就像厕所门上的锁：

线程1:  lock(mutex)  ──>  进入临界区...  ──>  unlock(mutex)
线程2:  lock(mutex)  ──>  (等待...)      ──>  进入临界区...  ──>  unlock(mutex)
                           ↑ 线程2 被阻塞在线程1释放锁之前
```

```c
#include <stdio.h>
#include <pthread.h>

#define INCREMENTS 1000000

int counter = 0;
pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;  /* 静态初始化 */

void* increment(void *arg) {
    for (int i = 0; i < INCREMENTS; i++) {
        pthread_mutex_lock(&mutex);    /* 加锁 */
        counter++;                     /* 临界区：受保护 */
        pthread_mutex_unlock(&mutex);  /* 解锁 */
    }
    return NULL;
}

int main(void) {
    pthread_t t1, t2;

    pthread_create(&t1, NULL, increment, NULL);
    pthread_create(&t2, NULL, increment, NULL);

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);

    pthread_mutex_destroy(&mutex);  /* 销毁互斥锁 */

    printf("预期: %d, 实际: %d\n", 2 * INCREMENTS, counter);
    return 0;
}
```

现在输出：
```
预期: 2000000, 实际: 2000000    <-- 完全正确！
```

### 互斥锁的完整生命周期

```c
/* 方式1: 静态初始化 */
pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;

/* 方式2: 动态初始化 */
pthread_mutex_t mutex;
pthread_mutex_init(&mutex, NULL);  /* NULL = 默认属性 */

/* 使用 */
pthread_mutex_lock(&mutex);
/* ... 临界区代码 ... */
pthread_mutex_unlock(&mutex);

/* 销毁 */
pthread_mutex_destroy(&mutex);
```

### 锁的粒度

锁的粒度越细，并发性越好，但代码越复杂：

```c
/* ❌ 粗粒度锁：整个循环被锁住，线程变成串行执行 */
void* increment(void *arg) {
    pthread_mutex_lock(&mutex);
    for (int i = 0; i < INCREMENTS; i++) {
        counter++;
    }
    pthread_mutex_unlock(&mutex);
    return NULL;
}

/* ✅ 细粒度锁：只在修改共享变量时加锁 */
void* increment(void *arg) {
    for (int i = 0; i < INCREMENTS; i++) {
        pthread_mutex_lock(&mutex);
        counter++;
        pthread_mutex_unlock(&mutex);
    }
    return NULL;
}
```

## 54.5 死锁：锁的陷阱

**死锁**发生在两个线程互相等待对方释放锁，导致永远无法继续执行：

```
线程1                          线程2
  │                               │
  │ lock(锁A) 成功                │ lock(锁B) 成功
  │                               │
  │ lock(锁B) 等待...             │ lock(锁A) 等待...
  │ (等线程2释放B)                │ (等线程1释放A)
  │                               │
  ▼                               ▼
  死锁！两个线程互相等待，永远无法继续
```

```c
#include <stdio.h>
#include <pthread.h>
#include <unistd.h>

pthread_mutex_t lock_a = PTHREAD_MUTEX_INITIALIZER;
pthread_mutex_t lock_b = PTHREAD_MUTEX_INITIALIZER;

void* thread1_work(void *arg) {
    pthread_mutex_lock(&lock_a);
    printf("[线程1] 获取锁A\n");
    sleep(1);  /* 增加死锁概率 */

    pthread_mutex_lock(&lock_b);  /* 等待锁B */
    printf("[线程1] 获取锁B\n");

    pthread_mutex_unlock(&lock_b);
    pthread_mutex_unlock(&lock_a);
    return NULL;
}

void* thread2_work(void *arg) {
    pthread_mutex_lock(&lock_b);
    printf("[线程2] 获取锁B\n");
    sleep(1);

    pthread_mutex_lock(&lock_a);  /* 等待锁A */
    printf("[线程2] 获取锁A\n");

    pthread_mutex_unlock(&lock_a);
    pthread_mutex_unlock(&lock_b);
    return NULL;
}

int main(void) {
    pthread_t t1, t2;
    pthread_create(&t1, NULL, thread1_work, NULL);
    pthread_create(&t2, NULL, thread2_work, NULL);

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);

    printf("程序结束（如果没有死锁的话）\n");
    return 0;
}
```

这个程序会死锁——永远看不到"程序结束"。

### 死锁的四个必要条件

1. **互斥**：资源不能被共享，一次只能被一个线程使用
2. **持有并等待**：线程持有至少一个资源，同时在等待其他资源
3. **不可剥夺**：资源只能由持有者主动释放
4. **循环等待**：存在线程-资源的循环等待链

### 预防死锁

```c
/* 策略1: 固定锁的顺序 —— 所有线程以相同顺序获取锁 */
void* safe_thread_work(void *arg) {
    /* 总是先锁A，再锁B */
    pthread_mutex_lock(&lock_a);
    pthread_mutex_lock(&lock_b);

    /* 临界区... */

    pthread_mutex_unlock(&lock_b);
    pthread_mutex_unlock(&lock_a);
    return NULL;
}

/* 策略2: trylock —— 尝试获取锁，失败则释放已持有的锁 */
void* trylock_work(void *arg) {
    while (1) {
        pthread_mutex_lock(&lock_a);
        if (pthread_mutex_trylock(&lock_b) == 0) {
            /* 两个锁都获取成功 */
            break;
        }
        /* 获取锁B失败，释放锁A，稍后重试 */
        pthread_mutex_unlock(&lock_a);
        usleep(1000);  /* 等待1毫秒 */
    }

    /* 临界区... */

    pthread_mutex_unlock(&lock_b);
    pthread_mutex_unlock(&lock_a);
    return NULL;
}
```

## 54.6 完整代码清单

```c
/*
 * thread_demo.c —— 线程编程完整演示
 * 编译: gcc -std=c99 -Wall -pthread -o thread_demo thread_demo.c
 * 注意: 仅在 Linux/macOS 上运行，需要 POSIX 线程支持
 */

#include <stdio.h>
#include <pthread.h>
#include <unistd.h>
#include <stdlib.h>

/* ==================== 演示1: 基本线程创建 ==================== */
void* basic_worker(void *arg) {
    int id = *(int*)arg;
    printf("  [线程 %d] 开始，ID=%lu\n", id, pthread_self());
    sleep(1);
    printf("  [线程 %d] 结束\n", id);
    return (void*)(long)(id * 100);  /* 返回一个值 */
}

void demo_basic_threads(void) {
    printf("===== 1. 基本线程创建与 join =====\n");
    pthread_t t1, t2;
    int id1 = 1, id2 = 2;

    pthread_create(&t1, NULL, basic_worker, &id1);
    pthread_create(&t2, NULL, basic_worker, &id2);

    void *ret1, *ret2;
    pthread_join(t1, &ret1);
    pthread_join(t2, &ret2);

    printf("  线程1 返回值: %ld\n", (long)ret1);
    printf("  线程2 返回值: %ld\n", (long)ret2);
}

/* ==================== 演示2: 竞态条件 ==================== */
#define COUNT 500000
int race_counter = 0;

void* race_increment(void *arg) {
    for (int i = 0; i < COUNT; i++) {
        race_counter++;  /* 不加锁，产生竞态 */
    }
    return NULL;
}

void demo_race_condition(void) {
    printf("\n===== 2. 竞态条件演示 =====\n");
    race_counter = 0;
    pthread_t t1, t2;

    pthread_create(&t1, NULL, race_increment, NULL);
    pthread_create(&t2, NULL, race_increment, NULL);

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);

    printf("  预期值: %d\n", 2 * COUNT);
    printf("  实际值: %d\n", race_counter);
    printf("  丢失了 %d 次操作\n", 2 * COUNT - race_counter);
}

/* ==================== 演示3: 互斥锁解决竞态 ==================== */
int safe_counter = 0;
pthread_mutex_t safe_mutex = PTHREAD_MUTEX_INITIALIZER;

void* safe_increment(void *arg) {
    for (int i = 0; i < COUNT; i++) {
        pthread_mutex_lock(&safe_mutex);
        safe_counter++;
        pthread_mutex_unlock(&safe_mutex);
    }
    return NULL;
}

void demo_mutex(void) {
    printf("\n===== 3. 互斥锁保护 =====\n");
    safe_counter = 0;
    pthread_t t1, t2;

    pthread_create(&t1, NULL, safe_increment, NULL);
    pthread_create(&t2, NULL, safe_increment, NULL);

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);

    pthread_mutex_destroy(&safe_mutex);

    printf("  预期值: %d\n", 2 * COUNT);
    printf("  实际值: %d\n", safe_counter);
    printf("  结果: %s\n", safe_counter == 2 * COUNT ? "正确" : "错误");
}

/* ==================== 演示4: 计算任务并行化 ==================== */
typedef struct {
    long start;
    long end;
    long result;
} RangeSum;

void* compute_range_sum(void *arg) {
    RangeSum *rs = (RangeSum*)arg;
    rs->result = 0;
    for (long i = rs->start; i <= rs->end; i++) {
        rs->result += i;
    }
    return NULL;
}

void demo_parallel_computation(void) {
    printf("\n===== 4. 并行计算 =====\n");
    pthread_t t1, t2;
    RangeSum r1 = {1, 50000000, 0};
    RangeSum r2 = {50000001, 100000000, 0};

    pthread_create(&t1, NULL, compute_range_sum, &r1);
    pthread_create(&t2, NULL, compute_range_sum, &r2);

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);

    printf("  1 到 1亿 的和 = %ld\n", r1.result + r2.result);
}

/* ==================== 演示5: 死锁演示 ==================== */
pthread_mutex_t dead_a = PTHREAD_MUTEX_INITIALIZER;
pthread_mutex_t dead_b = PTHREAD_MUTEX_INITIALIZER;

void* deadlock_thread1(void *arg) {
    pthread_mutex_lock(&dead_a);
    printf("  [线程A] 获取锁A, 尝试获取锁B...\n");
    usleep(100000);
    pthread_mutex_lock(&dead_b);
    printf("  [线程A] 获取锁B\n");
    pthread_mutex_unlock(&dead_b);
    pthread_mutex_unlock(&dead_a);
    return NULL;
}

void* deadlock_thread2(void *arg) {
    /* 同样的顺序，不会死锁 */
    pthread_mutex_lock(&dead_a);
    printf("  [线程B] 获取锁A, 尝试获取锁B...\n");
    usleep(100000);
    pthread_mutex_lock(&dead_b);
    printf("  [线程B] 获取锁B\n");
    pthread_mutex_unlock(&dead_b);
    pthread_mutex_unlock(&dead_a);
    return NULL;
}

void demo_deadlock_free(void) {
    printf("\n===== 5. 固定锁顺序避免死锁 =====\n");
    printf("  (两个线程都以 A→B 的顺序获取锁，不会死锁)\n");

    pthread_t t1, t2;
    pthread_create(&t1, NULL, deadlock_thread1, NULL);
    pthread_create(&t2, NULL, deadlock_thread2, NULL);

    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    printf("  死锁避免成功！\n");
}

int main(void) {
    demo_basic_threads();
    demo_race_condition();
    demo_mutex();
    demo_parallel_computation();
    demo_deadlock_free();

    printf("\n===== 全部演示完成 =====\n");
    return 0;
}
```

## 我做得对不对？

1. `demo_basic_threads()`：两个线程各自执行，`pthread_join` 获取返回值（100 和 200）。
2. `demo_race_condition()`：实际值小于预期值，证明竞态条件存在。
3. `demo_mutex()`：加锁后实际值等于预期值。
4. `demo_parallel_computation()`：分两段并行计算，结果与单线程一致。
5. `demo_deadlock_free()`：固定锁顺序后程序正常结束，不会死锁。

**验证竞态**：多次运行 `demo_race_condition()`，每次结果不同且小于预期值。

## 不对怎么办？

### 常见Bug 1：忘记 -pthread 编译选项

```bash
# ❌ 忘记 -pthread，链接错误
gcc -std=c99 -Wall -o thread thread.c
# undefined reference to `pthread_create'
```

✅ 编译命令必须包含 `-pthread`：`gcc -std=c99 -Wall -pthread -o thread thread.c`

### 常见Bug 2：向线程传递栈上变量的地址

```c
/* ❌ 传递局部变量地址，线程启动时可能已被销毁 */
void create_threads(void) {
    for (int i = 0; i < 5; i++) {
        pthread_create(&t, NULL, worker, &i);  /* i 的地址！ */
        /* 循环结束后 i 的值可能不是线程看到的值 */
    }
}
```

✅ 每个线程分配独立的参数内存（`malloc` 或静态数组）。

### 常见Bug 3：忘记销毁互斥锁

```c
/* ❌ 互斥锁未销毁，在某些系统上会造成资源泄漏 */
pthread_mutex_t mutex;
pthread_mutex_init(&mutex, NULL);
/* ... 使用 ... */
/* 忘记 pthread_mutex_destroy(&mutex); */
```

✅ 使用完后调用 `pthread_mutex_destroy(&mutex)`。

### 常见Bug 4：锁了但忘记解锁

```c
/* ❌ 函数中途 return 但没有解锁 */
void* worker(void *arg) {
    pthread_mutex_lock(&mutex);
    if (some_error) {
        return NULL;  /* 忘记解锁！其他线程永远等不到锁 */
    }
    pthread_mutex_unlock(&mutex);
    return NULL;
}
```

✅ 任何退出路径都要解锁，或使用 `goto cleanup` 模式。

### 常见Bug 5：对已销毁的锁再次加锁

```c
/* ❌ 销毁锁后其他线程可能还在尝试加锁 */
pthread_mutex_destroy(&mutex);
/* 此时另一个线程调用 pthread_mutex_lock(&mutex) → 未定义行为 */
```

✅ 确保所有线程都不再使用锁之后再销毁。

## 本章小结

| 要点 | 说明 |
|------|------|
| 线程 vs 进程 | 线程共享地址空间，轻量；进程地址空间独立，重量 |
| pthread_create | 创建线程，参数：线程ID、属性、函数、参数 |
| pthread_join | 等待线程结束，获取返回值 |
| pthread_self | 获取当前线程ID |
| 竞态条件 | 多线程同时读写共享变量，结果不可预测 |
| 临界区 | 访问共享资源的代码段，必须互斥访问 |
| 互斥锁(mutex) | 确保同一时刻只有一个线程进入临界区 |
| pthread_mutex_lock | 加锁，如果已被锁则阻塞等待 |
| pthread_mutex_unlock | 解锁 |
| pthread_mutex_trylock | 尝试加锁，失败立即返回而不是阻塞 |
| 死锁 | 两个线程互相等待对方释放锁，永远阻塞 |
| 死锁预防 | 固定锁顺序、trylock 重试、减少锁持有时间 |
| 编译选项 | 需要 `-pthread` 链接 pthread 库 |
