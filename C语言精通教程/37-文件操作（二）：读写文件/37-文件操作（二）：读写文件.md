## 我在做什么？

第 36 章你学会了打开和关闭文件的门，但门打开之后呢？你需要真正往文件里写东西、从文件里读东西。这一章你将掌握四种核心读写方式：格式化读写（像 printf/scanf 一样方便）、行读写（一行行处理文本）、二进制读写（直接搬运内存数据）、文件定位（在文件中随意跳转）。最后还会揭露 feof 的经典陷阱——无数 C 程序员都踩过的坑。

读完本章你能：用 fprintf/fscanf 做格式化文件 IO，用 fgets/fputs 逐行处理文本，用 fread/fwrite 高效复制二进制文件，用 fseek/ftell 在文件中定位，以及正确判断文件是否读完（避免 feof 陷阱）。

## 一、格式化读写：fprintf 与 fscanf

### 1.1 fprintf：往文件里写格式化数据

`fprintf` 和 `printf` 几乎一模一样，唯一的区别是多了一个参数——文件指针。`printf` 是往屏幕（stdout）写，`fprintf` 是往你指定的文件写。

```text
printf("Hello %d", 42);
         |
         v
       stdout（屏幕）

fprintf(fp, "Hello %d", 42);
         ||
         vv
       fp 指向的文件
```

语法：

```c
int fprintf(FILE *fp, const char *format, ...);
```

返回值：成功返回写入的字符数，失败返回负数。

```c
#include <stdio.h>

int main() {
    FILE *fp = fopen("students.txt", "w");
    if (fp == NULL) {
        fprintf(stderr, "无法创建文件\n");
        return 1;
    }

    fprintf(fp, "姓名: %s, 年龄: %d, 成绩: %.1f\n", "张三", 20, 92.5);
    fprintf(fp, "姓名: %s, 年龄: %d, 成绩: %.1f\n", "李四", 21, 85.0);
    fprintf(fp, "姓名: %s, 年龄: %d, 成绩: %.1f\n", "王五", 19, 78.5);

    fclose(fp);
    return 0;
}
```

运行后 `students.txt` 的内容：

```
姓名: 张三, 年龄: 20, 成绩: 92.5
姓名: 李四, 年龄: 21, 成绩: 85.0
姓名: 王五, 年龄: 19, 成绩: 78.5
```

比喻：`fprintf` 就像你用同样的语法写作文，只是换了一本本子。`printf` 写在黑板上（所有人看得到），`fprintf` 写在你的笔记本里（只有打开文件的人才看得到）。

### 1.2 fscanf：从文件中按格式读取

`fscanf` 和 `scanf` 也是一对亲兄弟。`scanf` 从键盘（stdin）读，`fscanf` 从你指定的文件读。

```c
int fscanf(FILE *fp, const char *format, ...);
```

返回值：成功读取并赋值的项数，到达文件末尾或出错返回 `EOF`。

读取刚才写入的学生信息：

```c
#include <stdio.h>

int main() {
    FILE *fp = fopen("students.txt", "r");
    if (fp == NULL) {
        fprintf(stderr, "无法打开文件\n");
        return 1;
    }

    char name[50];
    int age;
    float score;

    // 按写入时的格式原样读取
    while (fscanf(fp, "姓名: %s, 年龄: %d, 成绩: %f\n", name, &age, &score) == 3) {
        printf("读到: %s, %d 岁, %.1f 分\n", name, age, score);
    }

    fclose(fp);
    return 0;
}
```

输出：

```
读到: 张三, 20 岁, 92.5 分
读到: 李四, 21 岁, 85.0 分
读到: 王五, 19 岁, 78.5 分
```

关键点：`fscanf` 的返回值是**成功读取的项数**。上面代码中 `fscanf` 试图读取 3 项（`%s`、`%d`、`%f`），所以当返回值 `== 3` 时才说明全部读成功了。如果文件格式不匹配，返回值可能小于 3，说明读取失败。

### 1.3 fscanf 的局限性

`fscanf` 有一个巨大的弱点：它要求文件格式**精确匹配**格式字符串中的每一个字符。看下面的坑：

```c
// 文件内容：42
// 格式字符串："%d"
// fscanf 读到 42，成功 ✓

// 文件内容：age=42
// 格式字符串："%d"
// fscanf 读到 'a'，不是数字，失败 ✗
```

而且 `fscanf` 读到空白字符（空格、换行、制表符）时会自动跳过，这让它在处理复杂格式时行为难以预测。对于文本文件，更推荐使用下一节介绍的 `fgets` + `sscanf` 组合。

## 二、行读写：fgets 与 fputs

### 2.1 fgets：安全地读一行

`fgets` 是读取文本文件的"瑞士军刀"。它每次读取一行（包括换行符），并且永远不会越界——因为你必须告诉它缓冲区的大小。

```c
char *fgets(char *buffer, int size, FILE *fp);
```

参数：
- `buffer`：存放读取内容的字符数组
- `size`：buffer 的最大容量（`fgets` 最多读 `size-1` 个字符，留一位放 `\0`）
- `fp`：文件指针

返回值：成功返回 `buffer`，到达文件末尾或出错返回 `NULL`。

```c
#include <stdio.h>

int main() {
    FILE *fp = fopen("students.txt", "r");
    if (fp == NULL) {
        fprintf(stderr, "无法打开文件\n");
        return 1;
    }

    char line[256];  // 每一行最多 255 个字符

    while (fgets(line, sizeof(line), fp) != NULL) {
        printf("读到一行: %s", line);  // line 本身已含换行符，printf 不用再加 \n
    }

    fclose(fp);
    return 0;
}
```

**fgets 会保留换行符**：如果一行的内容是 `"hello\n"`，fgets 读到的字符串就是 `"hello\n\0"`。如果你不需要换行符，可以手动去掉：

```c
#include <string.h>

// 查找最后一个字符，如果是换行符就替换为 \0
line[strcspn(line, "\n")] = '\0';
```

### 2.2 fgets vs gets：为什么永远不要用 gets

你可能在老旧教材中见过 `gets()`：

```c
// ❌ 永远不要用！
char buffer[10];
gets(buffer);  // 如果用户输入了 100 个字符，全部写入 buffer，越界崩溃
```

`gets` 不检查缓冲区大小，是 C 语言已知最危险的函数之一。C11 标准已将 `gets` 移除。始终使用 `fgets`：

```c
// ✅ 安全
char buffer[10];
fgets(buffer, sizeof(buffer), stdin);  // 最多读 9 个字符，安全
```

### 2.3 fputs：写一行

```c
int fputs(const char *str, FILE *fp);
```

`fputs` 把字符串写入文件，不会自动追加换行符。如果需要换行，自己加 `\n`。

```c
fputs("Hello World\n", fp);   // 写入 "Hello World" + 换行
fputs("No newline", fp);      // 写入 "No newline"，没有换行
```

`fputs` 比 `printf` 简单、快速，当你不需要格式化时优先用它。

### 2.4 sscanf：内存中的 fscanf

有时你不想直接从文件解析，而是先读一行到缓冲区，再从缓冲区解析。这就是 `sscanf` 的用途：

```c
int sscanf(const char *str, const char *format, ...);
```

和 `fscanf` 一样，但数据来源是字符串而不是文件。

```c
char line[] = "姓名: 张三, 年龄: 20, 成绩: 92.5";
char name[50];
int age;
float score;

// 从字符串 line 中解析
sscanf(line, "姓名: %s, 年龄: %d, 成绩: %f", name, &age, &score);
printf("%s %d %.1f\n", name, age, score);  // 张三 20 92.5
```

**推荐模式**：`fgets` + `sscanf` 组合比直接用 `fscanf` 安全得多，因为 fgets 保证了不越界。

```c
char line[256];
while (fgets(line, sizeof(line), fp) != NULL) {
    char name[50];
    int age;
    float score;
    if (sscanf(line, "姓名: %s, 年龄: %d, 成绩: %f", name, &age, &score) == 3) {
        printf("解析成功: %s\n", name);
    }
}
```

## 三、二进制读写：fread 与 fwrite

### 3.1 文本读写的局限

到目前为止，你读写文件都要经过"格式化"——整数 `42` 在文件中是字符 `'4'` 和 `'2'`，占 2 字节。但 `int` 型 `42` 在内存中只占 4 字节。当你需要高效存储或复制数据时，来回转换既慢又浪费空间。

```text
文本存储：整数 42 → 写入文件 "42"   （2 字节，可读）
二进制存储：整数 42 → 写入文件 0x2A 00 00 00（4 字节，不可读但精确）

文本存储：整数 123456 → 写入文件 "123456"（6 字节）
二进制存储：整数 123456 → 写入文件（4 字节）
```

### 3.2 fwrite：把内存原样写入文件

```c
size_t fwrite(const void *ptr, size_t size, size_t count, FILE *fp);
```

参数逐个解释：
- `ptr`：指向要写入的数据的指针（`void*` 意味着可以传任何类型的指针）
- `size`：每个元素的字节大小
- `count`：要写入的元素个数
- `fp`：目标文件指针

返回值：成功写入的元素个数（应该等于 `count`）。

```c
#include <stdio.h>

int main() {
    int scores[] = {95, 87, 92, 78, 88};
    int count = 5;

    // 注意模式是 "wb"（write binary）
    FILE *fp = fopen("scores.bin", "wb");
    if (fp == NULL) {
        fprintf(stderr, "无法创建文件\n");
        return 1;
    }

    size_t written = fwrite(scores, sizeof(int), count, fp);
    printf("写入了 %zu 个整数，共 %zu 字节\n",
           written, written * sizeof(int));

    fclose(fp);
    return 0;
}
```

输出：

```
写入了 5 个整数，共 20 字节
```

比喻：`fwrite` 就像把整盒积木（内存中的数据）连盒子一起搬到仓库（文件）里，不拆开、不重组。`fprintf` 则像把积木一个个拆开，在上面贴好标签再放进去。

### 3.3 fread：从文件中原样读回内存

```c
size_t fread(void *ptr, size_t size, size_t count, FILE *fp);
```

参数和 `fwrite` 完全对应。

```c
#include <stdio.h>

int main() {
    int scores[5];

    FILE *fp = fopen("scores.bin", "rb");  // "rb" = read binary
    if (fp == NULL) {
        fprintf(stderr, "无法打开文件\n");
        return 1;
    }

    size_t read_count = fread(scores, sizeof(int), 5, fp);
    printf("读到了 %zu 个整数:\n", read_count);
    for (size_t i = 0; i < read_count; i++) {
        printf("  scores[%zu] = %d\n", i, scores[i]);
    }

    fclose(fp);
    return 0;
}
```

输出：

```
读到了 5 个整数:
  scores[0] = 95
  scores[1] = 87
  scores[2] = 92
  scores[3] = 78
  scores[4] = 88
```

### 3.4 结构体的二进制读写

二进制 IO 的真正威力在于可以直接读写结构体：

```c
#include <stdio.h>
#include <string.h>

typedef struct {
    char name[32];
    int age;
    float score;
} Student;

int main() {
    // === 写入 ===
    Student s1 = {"张三", 20, 92.5};
    Student s2 = {"李四", 21, 85.0};

    FILE *fp = fopen("students.bin", "wb");
    if (fp == NULL) {
        fprintf(stderr, "无法创建文件\n");
        return 1;
    }

    fwrite(&s1, sizeof(Student), 1, fp);
    fwrite(&s2, sizeof(Student), 1, fp);
    fclose(fp);

    // === 读取 ===
    fp = fopen("students.bin", "rb");
    if (fp == NULL) {
        fprintf(stderr, "无法打开文件\n");
        return 1;
    }

    Student s;
    while (fread(&s, sizeof(Student), 1, fp) == 1) {
        printf("姓名: %s, 年龄: %d, 成绩: %.1f\n", s.name, s.age, s.score);
    }

    fclose(fp);
    return 0;
}
```

输出：

```
姓名: 张三, 年龄: 20, 成绩: 92.5
姓名: 李四, 年龄: 21, 成绩: 85.0
```

**重要警告**：用 `fwrite` 直接写结构体存在可移植性问题。不同编译器、不同平台对结构体的内存布局可能不同（内边距、字节序等）。在 A 机器上写出的 `students.bin`，到 B 机器上可能读出来的数据是乱的。仅在**同一平台同一编译器**下可放心使用，跨平台传输请用文本格式或序列化协议。

### 3.5 文件复制实战（二进制模式）

用 `fread` + `fwrite` 可以复制任意文件：

```c
#include <stdio.h>

#define BUFFER_SIZE 4096  // 一次读 4KB

int main(int argc, char *argv[]) {
    if (argc != 3) {
        fprintf(stderr, "用法: %s <源文件> <目标文件>\n", argv[0]);
        return 1;
    }

    FILE *src = fopen(argv[1], "rb");
    if (src == NULL) {
        fprintf(stderr, "无法打开源文件: %s\n", argv[1]);
        return 1;
    }

    FILE *dst = fopen(argv[2], "wb");
    if (dst == NULL) {
        fprintf(stderr, "无法创建目标文件: %s\n", argv[2]);
        fclose(src);
        return 1;
    }

    char buffer[BUFFER_SIZE];
    size_t bytes_read;

    while ((bytes_read = fread(buffer, 1, BUFFER_SIZE, src)) > 0) {
        fwrite(buffer, 1, bytes_read, dst);
    }

    printf("复制完成\n");

    fclose(src);
    fclose(dst);
    return 0;
}
```

关键细节：`fread(buffer, 1, BUFFER_SIZE, src)` 中，第二个参数是 `1`（每个元素 1 字节），第三个参数是 `BUFFER_SIZE`（最多读 4096 个元素）。这样返回值 `bytes_read` 就是**实际读到的字节数**，而不是元素个数。这种写法在复制文件时最常用。

## 四、文件定位：fseek 与 ftell

### 4.1 文件的"读头"

当你打开一个文件开始读，C 在内部维护了一个"读头"（也叫文件位置指示器），指向你接下来要读/写的字节位置。每次读写后，读头自动前移。

```text
文件内容: H e l l o   W o r l d \n
            ^
            读头（刚打开文件时在开头）

读完 5 个字节后:
文件内容: H e l l o   W o r l d \n
                      ^
                      读头移到这里
```

`fseek` 让你手动移动这个读头，`ftell` 告诉你读头现在在哪里。

### 4.2 fseek：移动文件位置

```c
int fseek(FILE *fp, long offset, int whence);
```

- `fp`：文件指针
- `offset`：偏移量（可正可负）
- `whence`：参考点

| whence 值 | 含义 |
|-----------|------|
| `SEEK_SET` | 从文件开头算起 |
| `SEEK_CUR` | 从当前位置算起 |
| `SEEK_END` | 从文件末尾算起 |

返回值：成功返回 `0`，失败返回非 `0`。

```c
// 跳到文件开头
fseek(fp, 0, SEEK_SET);

// 跳到文件末尾
fseek(fp, 0, SEEK_END);

// 从当前位置向后跳 100 字节
fseek(fp, 100, SEEK_CUR);

// 从文件末尾向前 50 字节（offset 为负）
fseek(fp, -50, SEEK_END);
```

### 4.3 ftell：获取当前位置

```c
long ftell(FILE *fp);
```

返回值：当前读头位置（从文件开头算起的字节偏移），失败返回 `-1L`。

```c
fseek(fp, 0, SEEK_END);      // 跳到末尾
long size = ftell(fp);       // 获取文件大小（字节数）
printf("文件大小: %ld 字节\n", size);
```

### 4.4 实用示例：读取文件中的第 N 个结构体

```c
#include <stdio.h>

typedef struct {
    char name[32];
    int age;
    float score;
} Student;

int main() {
    FILE *fp = fopen("students.bin", "rb");
    if (fp == NULL) {
        fprintf(stderr, "无法打开文件\n");
        return 1;
    }

    int index;
    printf("你想看第几个学生？(从 0 开始): ");
    scanf("%d", &index);

    // 跳到第 index 个结构体的位置
    if (fseek(fp, index * sizeof(Student), SEEK_SET) != 0) {
        fprintf(stderr, "fseek 失败\n");
        fclose(fp);
        return 1;
    }

    Student s;
    if (fread(&s, sizeof(Student), 1, fp) == 1) {
        printf("姓名: %s, 年龄: %d, 成绩: %.1f\n", s.name, s.age, s.score);
    } else {
        printf("该位置没有数据\n");
    }

    fclose(fp);
    return 0;
}
```

这就叫**随机访问**——不需要从头读到尾，直接跳到目标位置读取。

### 4.5 rewind：快速回到开头

```c
void rewind(FILE *fp);
```

等价于：

```c
fseek(fp, 0, SEEK_SET);
```

但 `rewind` 还会清除文件错误标志。当你读完一遍文件想再读一遍时很有用：

```c
while (fgets(line, sizeof(line), fp) != NULL) {
    printf("%s", line);
}
// 读完了，想再读一遍
rewind(fp);
while (fgets(line, sizeof(line), fp) != NULL) {
    printf("%s", line);  // 又从头读一遍
}
```

## 五、feof 的陷阱

### 5.1 feof 是"迟到"的

`feof(fp)` 的功能是：检查文件流是否已经到达末尾。注意**已经**二字——`feof` 不会预知未来，它只是在某次读取操作**已经**触碰到文件末尾之后，才会返回真。

```c
int feof(FILE *fp);
```

返回值：如果之前的一次读取操作遇到了文件末尾，返回非 0；否则返回 0。

### 5.2 经典错误：用 feof 控制循环

无数 C 初学者——以及不少老手——都写过这样的代码：

```c
// ❌ 经典错误！
while (!feof(fp)) {
    fgets(line, sizeof(line), fp);
    printf("%s", line);
}
```

为什么这是错的？因为 `feof` 在 `fgets` 触碰到文件末尾**之前**始终返回 0。执行流程：

```text
1. 检查 feof(fp) → 0（还没到末尾）
2. fgets 读到最后一行的数据 → 成功
3. 检查 feof(fp) → 0（还是没到末尾，因为上次 fgets 成功了）
4. fgets 再次读取 → 没有数据了，返回 NULL，但 line 的内容未定义
5. 检查 feof(fp) → 1（到了！但已经太晚了——你已经用了一次无效数据）
```

具体后果：**最后一行数据被打印了两次**。因为第 4 步 fgets 失败后没有改 line 的内容，line 还保留着上一轮读到的数据，然后 printf 又把它输出了一遍。

### 5.3 正确写法

```c
// ✅ 正确：用读取函数的返回值控制循环
while (fgets(line, sizeof(line), fp) != NULL) {
    printf("%s", line);
}
```

同理，对于其他读取函数：

```c
// fgetc：读到文件末尾返回 EOF
int ch;
while ((ch = fgetc(fp)) != EOF) {
    putchar(ch);
}

// fread：返回实际读取的元素数
int data[100];
size_t n;
while ((n = fread(data, sizeof(int), 100, fp)) > 0) {
    // 处理 n 个元素
}

// fscanf：返回成功匹配的项数
int a, b;
while (fscanf(fp, "%d %d", &a, &b) == 2) {
    printf("%d + %d = %d\n", a, b, a + b);
}
```

### 5.4 feof 的正确使用场景

`feof` 不是完全没用，它适用于以下场景：**在读取循环结束后，判断是因为读完而退出，还是因为出错而退出**。

```c
while (fgets(line, sizeof(line), fp) != NULL) {
    // 处理每一行
}

if (feof(fp)) {
    printf("正常读到文件末尾，数据处理完成\n");
} else if (ferror(fp)) {
    fprintf(stderr, "读取过程中发生错误！\n");
}
```

`ferror(fp)` 检查文件流是否处于错误状态。

## 六、完整示例：一个简单的日志查看器

综合运用本章学到的所有知识，写一个简易的日志查看器，支持从头查看、翻页、跳转到末尾：

```c
#include <stdio.h>
#include <string.h>

void show_help() {
    printf("命令:\n");
    printf("  n    - 下一页 (10行)\n");
    printf("  p    - 上一页 (10行)\n");
    printf("  e    - 跳到末尾\n");
    printf("  s    - 回到开头\n");
    printf("  q    - 退出\n");
}

int main(int argc, char *argv[]) {
    if (argc != 2) {
        fprintf(stderr, "用法: %s <日志文件>\n", argv[0]);
        return 1;
    }

    FILE *fp = fopen(argv[1], "r");
    if (fp == NULL) {
        fprintf(stderr, "无法打开文件: %s\n", argv[1]);
        return 1;
    }

    char line[1024];
    int line_count;
    char cmd[10];

    show_help();

    while (1) {
        printf("> ");
        fgets(cmd, sizeof(cmd), stdin);

        if (cmd[0] == 'q') {
            break;
        } else if (cmd[0] == 'n') {
            // 读下 10 行
            line_count = 0;
            while (line_count < 10 && fgets(line, sizeof(line), fp) != NULL) {
                printf("%s", line);
                line_count++;
            }
            if (line_count == 0) {
                printf("(已到文件末尾)\n");
            }
        } else if (cmd[0] == 'p') {
            // 上一页：先获取当前位置，回退大约 20 行，再读 10 行
            long current = ftell(fp);
            long target = current - 20 * 80;  // 假设每行最多 80 字符
            if (target < 0) target = 0;
            fseek(fp, target, SEEK_SET);
            // 跳过可能的不完整行
            fgets(line, sizeof(line), fp);
            // 之后再读 10 行显示
            int display = 0;
            while (display < 10 && fgets(line, sizeof(line), fp) != NULL) {
                long pos = ftell(fp);
                if (pos > current) break;  // 不要超过原来的位置
                printf("%s", line);
                display++;
            }
            // 恢复到原来的位置
            fseek(fp, current, SEEK_SET);
        } else if (cmd[0] == 'e') {
            fseek(fp, 0, SEEK_END);
            printf("(已跳到文件末尾)\n");
        } else if (cmd[0] == 's') {
            rewind(fp);
            printf("(已回到文件开头)\n");
        } else {
            show_help();
        }
    }

    fclose(fp);
    return 0;
}
```

#include <stdio.h>

### 7.3 验证 feof 陷阱

```c
#include <stdio.h>

int main() {
    FILE *fp = fopen("test.txt", "w");
    fprintf(fp, "line1\nline2\nline3\n");
    fclose(fp);

    fp = fopen("test.txt", "r");
    char line[100];

    printf("=== 错误写法（feof 循环）===\n");
    while (!feof(fp)) {
        if (fgets(line, sizeof(line), fp) != NULL) {
            printf("读到: %s", line);
        }
    }
    printf("(观察上面是否出现重复行)\n");

    rewind(fp);
    printf("\n=== 正确写法（fgets 返回值）===\n");
    while (fgets(line, sizeof(line), fp) != NULL) {
        printf("读到: %s", line);
    }

    fclose(fp);
    return 0;
}
```

实际运行你会发现，错误写法可能不会出现重复行——这是因为现代 gcc 中 `feof` 的实现有时刚好能在此场景下正常工作。但**不要依赖巧合**，始终用读取函数的返回值判断。

## 八、不对怎么办？

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| fscanf 读不到数据 | 格式字符串与文件内容不匹配 | 先用 fgets 读一行打印出来，确认文件格式 |
| fscanf 读到乱码 | 文件中包含中文或特殊字符 | 确认文件编码为 UTF-8，或用 fgets + sscanf |
| fread 返回 0 | 文件为空或到达末尾；打开模式错误 | 检查文件大小；确认用了 "rb" 而非 "r" |
| fwrite 写出的文件比预期大 | 结构体内存对齐导致实际大小大于字段之和 | 使用 `sizeof(结构体名)` 获取实际大小 |
| fseek 后读取的数据不对 | 偏移量计算错误；文本模式下 fseek 行为受限 | 二进制文件用 "rb"，文本文件注意换行符转换 |
| ftell 返回 -1 | 文件流不支持定位（如管道） | 检查是否试图对 stdin、管道等使用 ftell |
| 最后一被行重复打印 | 使用了 `while(!feof(fp))` | 改用读取函数的返回值控制循环 |
| fclose 返回非 0 | 磁盘满；网络文件系统错误 | 检查磁盘空间，处理 fclose 的错误返回值 |

## 本章小结

| 知识点 | 要点 |
|--------|------|
| fprintf | 格式化写入文件，用法同 printf，多一个 FILE* 参数 |
| fscanf | 格式化读取文件，返回成功读取的项数 |
| fgets | 安全读一行，必须指定缓冲区大小，保留换行符 |
| fputs | 写字符串到文件，不自动加换行符 |
| sscanf | 从内存字符串中解析，常与 fgets 配合使用 |
| fread | 二进制读取，从文件原样复制数据到内存 |
| fwrite | 二进制写入，从内存原样复制数据到文件 |
| fseek | 移动文件读头，支持 SEEK_SET/SEEK_CUR/SEEK_END |
| ftell | 获取当前读头位置（字节偏移） |
| rewind | 回到文件开头，等价于 `fseek(fp, 0, SEEK_SET)` |
| feof | 不要用在 while 条件中！用读取函数的返回值替代 |
