## 一、目标与完成效果

**一句话目标**：在文章详情页增加「编辑」和「删除」按钮，实现文章的修改（PUT）和删除（DELETE）功能，并确保只能操作自己的文章。

**完成后的可观测效果**：
- 文章详情页出现「编辑」和「删除」按钮（仅文章作者可见）
- 点击「编辑」→ 跳转 `/edit/:id` → 表单自动填充文章原有内容 → 修改后提交 → 跳转首页
- 点击「删除」→ 弹出确认框 → 确认后删除 → 跳转首页
- 非作者看不到编辑/删除按钮
- 删除或编辑后首页自动刷新

## 三、分步操作

### 步骤1：理解 HTTP 方法——PUT 和 DELETE

#### 我在做什么？

HTTP 协议用不同的「动词」表示不同的操作意图。就像你在餐厅，说「我要点菜」和「我要退菜」是不同的动词。

| HTTP 方法 | 餐厅比喻 | CRUD 操作 | 本教程用途 |
|-----------|----------|-----------|-----------|
| **GET** | 看菜单 | Read（读） | 获取文章列表/详情 |
| **POST** | 点菜 | Create（创建） | 发布文章 |
| **PUT** | 换菜 | Update（更新） | 编辑文章 |
| **DELETE** | 退菜 | Delete（删除） | 删除文章 |

**完整请求对比**：

```
GET    /api/posts/1         → 获取文章1
POST   /api/posts           → 创建文章（body: {title, content, ...}）
PUT    /api/posts/1         → 更新文章1（body: {title, content, ...}）
DELETE /api/posts/1         → 删除文章1
```

**注意**：PUT 和 DELETE 也需要携带 `Authorization` 头，否则后端会拒绝请求。

在 `<script setup>` 中增加相关逻辑：

```js
import { ref, computed, onMounted } from 'vue'
import { useRoute, useRouter } from 'vue-router'
import { getPost, deletePost } from '../api/index.js'

const route = useRoute()
const router = useRouter()
const id = Number(route.params.id)
const post = ref(null)
const loading = ref(true)
const error = ref(null)
const deleting = ref(false)          // 新增：删除中状态

// 判断当前用户是否是文章作者
const isAuthor = computed(() => {
  const userStr = localStorage.getItem('user')
  if (!userStr || !post.value) return false
  const user = JSON.parse(userStr)
  return user.username === post.value.author
})

// 删除文章
async function handleDelete() {
  const confirmed = confirm('确定要删除这篇文章吗？此操作不可恢复！')
  if (!confirmed) return

  const token = localStorage.getItem('token')
  if (!token) {
    router.push('/login')
    return
  }

  deleting.value = true
  try {
    await deletePost(id, token)
    router.push('/')        // 删除成功 → 跳转首页
  } catch (err) {
    alert('删除失败：' + (err.message || '请稍后重试'))
  } finally {
    deleting.value = false
  }
}

// ... fetchPost() 和 onMounted 保持不变 ...
```

#### 代码逐行解释

| 代码 | 含义 |
|------|------|
| `computed(() => { ... })` | 计算属性，根据 `post.author` 和 `localStorage` 中的用户名判断是否是作者 |
| `v-if="isAuthor"` | 只有作者才能看到编辑/删除按钮 |
| `confirm('确定要删除...')` | 浏览器原生确认对话框，返回 `true`（确定）或 `false`（取消） |
| `await deletePost(id, token)` | 调用 API 删除文章 |

#### 判断作者身份的流程

```
当前用户                   文章作者
    │                        │
    │  localStorage.getItem('user')  │
    │  → JSON.parse → { username: '小明' }
    │                        │
    │              post.author = '小明'
    │                        │
    └──────── 比较 ──────────┘
              │
        相等 → isAuthor = true（显示按钮）
        不等 → isAuthor = false（隐藏按钮）
```

❌ **错误**：用 `post.authorId === userId` 判断
- 问题：mock 数据中没有 `authorId` 字段，只有 `author` 用户名。如果是真实后端，应该用 `authorId`（数字），因为用户名可能重复。这里用 `username` 是简化写法。

✅ **正确**：根据实际数据结构判断。教程用 `username` 匹配，正式项目用 `authorId`。

🤔 **想多一点**：`confirm()` 这个弹窗真的能拦住误删吗？用户可能顺手就点了「确定」。更安全的做法是让用户输入「DELETE」确认，或使用二次确认模态框。但 `confirm()` 是学习阶段最简洁的方案，够用就行。

### 步骤3：实现编辑页面

#### 我在做什么？

编辑页面与发布页面类似，但需要先「加载旧数据填充表单」，然后发送 PUT 请求。

**比喻**：发布是「写新作业」，编辑是「改已交的作业」——你先把作业本拿回来（GET），改完再交上去（PUT）。

**修改 `src/views/EditPost.vue`**：

```vue
<template>
  <div class="edit-post">
    <h1>🛠️ 编辑文章</h1>

    <!-- 加载中 -->
    <div v-if="loading">⏳ 加载文章数据...</div>

    <!-- 加载失败 -->
    <div v-else-if="loadError" class="error">
      <p>😵 加载失败：{{ loadError }}</p>
      <router-link to="/">← 返回首页</router-link>
    </div>

    <!-- 编辑表单 -->
    <form v-else @submit.prevent="handleUpdate" class="form">
      <div class="form-group">
        <label>标题 <span class="required">*</span></label>
        <input v-model="title" type="text" placeholder="请输入文章标题" maxlength="100" />
      </div>

      <div class="form-group">
        <label>摘要</label>
        <textarea v-model="summary" rows="3" placeholder="文章摘要" maxlength="200"></textarea>
      </div>

      <div class="form-group">
        <label>正文 <span class="required">*</span></label>
        <textarea v-model="content" rows="12" placeholder="请输入文章正文"></textarea>
      </div>

      <p v-if="errorMsg" class="error">{{ errorMsg }}</p>

      <div class="form-actions">
        <button type="submit" :disabled="submitting">
          {{ submitting ? '保存中...' : '💾 保存修改' }}
        </button>
        <router-link to="/" class="cancel-btn">取消</router-link>
      </div>
    </form>
  </div>
</template>

<script setup>
import { ref, onMounted } from 'vue'
import { useRoute, useRouter } from 'vue-router'
import { getPost, updatePost } from '../api/index.js'

const route = useRoute()
const router = useRouter()
const id = Number(route.params.id)

const title = ref('')
const summary = ref('')
const content = ref('')
const loading = ref(true)
const loadError = ref('')
const errorMsg = ref('')
const submitting = ref(false)

// 页面加载时获取文章数据并填充表单
onMounted(async () => {
  try {
    const data = await getPost(id)
    title.value = data.title
    summary.value = data.summary || ''
    content.value = data.content
  } catch (err) {
    loadError.value = err.message || '加载文章失败'
  } finally {
    loading.value = false
  }
})

// 提交更新
async function handleUpdate() {
  errorMsg.value = ''

  if (!title.value.trim()) {
    errorMsg.value = '标题不能为空'
    return
  }
  if (content.value.trim().length < 10) {
    errorMsg.value = '正文至少需要10个字符'
    return
  }

  const token = localStorage.getItem('token')
  if (!token) {
    router.push('/login')
    return
  }

  submitting.value = true
  try {
    await updatePost(
      id,
      {
        title: title.value.trim(),
        content: content.value.trim(),
        summary: summary.value.trim() || content.value.trim().slice(0, 100),
      },
      token
    )
    router.push('/')           // 保存成功 → 跳转首页
  } catch (err) {
    errorMsg.value = err.message || '保存失败，请稍后重试'
  } finally {
    submitting.value = false
  }
}
</script>
```

#### 编辑 vs 发布 对比

| | 发布页面（CreatePost） | 编辑页面（EditPost） |
|------|------|------|
| 页面加载时 | 表单为空 | 先 GET 获取旧数据，填充表单 |
| HTTP 方法 | `POST /api/posts` | `PUT /api/posts/:id` |
| 权限 | 需要登录 | 需要登录 + 是作者 |
| 成功后 | 跳转首页 | 跳转首页 |
| 加载状态 | 无（表单为空不需要加载） | 有（`loading` / `loadError`） |

## 五、小结

| 我学了什么 | 关键点 |
|------------|--------|
| HTTP 方法 | GET（读）/ POST（创）/ PUT（改）/ DELETE（删） |
| 编辑 = GET + PUT | 先获取旧数据填充表单，修改后 PUT 更新 |
| 删除 = confirm + DELETE | 确认弹窗 → DELETE 请求 → 跳转首页 |
| 权限控制 | `computed` 判断 `isAuthor`，`v-if` 控制按钮显示 |
| 编辑页面状态 | 三种状态：加载中 / 加载失败 / 编辑表单 |
| 数据回填 | `onMounted` 中 GET 数据，赋值给 `v-model` 绑定的变量 |

## 七、已知坑点与禁止事项

- ⚠️ **`confirm()` 不要滥用**：学习阶段够用，但生产环境建议用自定义模态框，体验更好。
- ⚠️ **编辑页面必须先 GET 再 PUT**：如果直接 PUT 而不先 GET，用户看不到原文，改起来很痛苦。
- ⚠️ **删除前必须确认**：`DELETE` 是破坏性操作，不可撤销。永远不要直接执行删除，先让用户确认。
- ⚠️ **权限判断逻辑要准确**：用 `post.author` 比较用户名是简化方案，正式项目用 `post.authorId` 更可靠。
