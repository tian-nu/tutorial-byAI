## 我在做什么？

从这一章开始，你要从"观众席"走到"操作台"前面。你将在自己的电脑上安装C语言编译器，亲手写出人生第一个C程序，让它打印出"Hello, World!"，然后逐行理解这五行代码到底在说什么。

如果把编程比作做饭，编译器就是你的灶台。没有灶台，你有再多食材（代码）也做不熟。本章就是帮你把灶台搭起来。

全章分三个部分：
1. 安装编译器（选你的操作系统那一节即可）
2. 写第一个程序，逐行讲解
3. 编译、运行、看结果

## 6.1 安装编译器

C语言的代码是人能读懂的文本，但计算机只认识二进制的机器指令。编译器的作用就是把这二者"翻译"过去。我们使用的编译器叫 **GCC**（GNU Compiler Collection），它是开源且跨平台的。

先把你的操作系统对号入座：

### 6.1.1 Windows 用户

Windows 本身不自带GCC，你需要安装 **MinGW-w64**（Minimalist GNU for Windows）。

**安装步骤：**

**第1步**：打开浏览器，访问 `https://winlibs.com`，下载最新版本的 `Win64 Zip archive`（选择不带-llvm的版本），例如 `gcc-14.2.0-mingw-w64-ucrt-x86_64.zip`。

**第2步**：将下载的zip文件解压到你选择的目录，建议放在 `C:\mingw64\`，解压后的目录结构应该类似：

```
C:\mingw64\
  ├── bin\
  │   ├── gcc.exe      <-- 这就是我们要用的编译器
  │   ├── g++.exe
  │   └── ...
  ├── include\
  └── lib\
```

**第3步**：把 `C:\mingw64\bin` 加入系统的PATH环境变量：
  - 按 `Win + R`，输入 `sysdm.cpl`，回车。
  - 点击"高级"标签 → "环境变量"。
  - 在"系统变量"中找到 `Path`，双击。
  - 点击"新建"，输入 `C:\mingw64\bin`。
  - 一路点"确定"关闭所有窗口。

**第4步**：验证安装。打开一个新的命令提示符（`Win + R` → `cmd`），输入：

```cmd
gcc --version
```

如果看到类似 `gcc (MinGW-W64 x86_64-ucrt-mcf) 14.2.0` 的输出，安装成功。

如果提示"'gcc' 不是内部或外部命令"，说明PATH没设对。回到第3步重新检查。

### 6.1.2 macOS 用户

macOS 不自带GCC（系统自带的 `gcc` 实际上是Clang的别名），你需要安装真正的GCC。

**方式一（推荐）**：使用 Homebrew。如果没有安装Homebrew，先在终端运行：

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

然后安装GCC：

```bash
brew install gcc
```

安装后验证：

```bash
gcc-14 --version
```

Homebrew安装的GCC默认带版本号后缀。如果你希望直接用 `gcc` 命令，可以创建一个别名（非必须）：

```bash
echo 'alias gcc="gcc-14"' >> ~/.zshrc
source ~/.zshrc
```

**方式二**：安装Xcode Command Line Tools：

```bash
xcode-select --install
```

这会安装Apple的Clang编译器，它不是GCC，但基本兼容。如果你采用此方式，后续所有 `gcc` 命令改为 `clang` 即可。

### 6.1.3 Linux 用户

大多数Linux发行版自带GCC或可一键安装。

**Debian/Ubuntu**：

```bash
sudo apt update
sudo apt install gcc
```

**Fedora**：

```bash
sudo dnf install gcc
```

**Arch**：

```bash
sudo pacman -S gcc
```

验证安装：

```bash
gcc --version
```

## 6.2 第一个程序：Hello, World!

编译器就绪后，我们来写程序。全世界所有程序员的第一行C代码，几乎都是这段。

用任意文本编辑器（Windows用记事本、macOS用文本编辑、或用VS Code等）新建文件，命名为 `hello.c`，输入以下内容：

```c
#include <stdio.h>

int main(void)
{
    printf("Hello, World!\n");
    return 0;
}
```

保存后，我们来逐行理解这五行代码。

---

## 6.3 逐行讲解

### 第1行：`#include <stdio.h>`

想象你要写一封信，信封上需要写收件人地址。你不需要自己发明"地址格式"这个概念——邮局早就定义好了。

`#include <stdio.h>` 就相当于你去邮局窗口喊一声："把标准地址格式给我拿来！"

- `#include` 是一条 `*预处理指令*（此术语见附录F）`。它在编译之前生效，相当于"复制-粘贴"：把指定文件的内容粘贴到当前位置。
- `<stdio.h>` 是一个 `*头文件*（此术语见附录F）`。头文件就像一份"服务菜单"，告诉你有哪些现成的函数可以用。
- `stdio` 是 Standard Input/Output 的缩写，即 `*标准库*（此术语见附录F）`中的输入输出模块。
- 尖括号 `<>` 告诉编译器："去系统标准路径找这个头文件"。

**比喻**：`#include` 像是在图书馆办理借书证。没有它，你甚至进不了图书馆的门，更别说借书了。`stdio.h` 是你借的第一本书，因为你要用到 `printf` 这个函数。

### 第2行：空行

它不是必须的，但让代码更易读。就像写文章分段——编译器完全忽略空行和多余空格。

### 第3行：`int main(void)`

每一栋房子必须有一个大门。无论房子多豪华，客人总得从大门进来。C程序的"大门"就是 `main` 函数。

- `main` 是 `*main函数*（此术语见附录F）`，它是程序的入口。程序启动时，操作系统会调用这个函数。
- `int` 表示 `main` 最终要向操作系统"汇报"一个整数。这个整数的中文名叫 `*返回值*（此术语见附录F）`。
- `(void)` 表示 `main` 不接收任何参数（你可以先理解为"不需要任何输入"）。

**比喻**：`main` 就像一个餐厅的"入口大门"。客人（操作系统）从这道门进来，吃完饭走的时候给前台（return）一个反馈："0表示一切都好，1表示出问题了。"

这里有一个初学者最常犯的错误——把 `main` 写成 `mian`。少了这一个字母，编译器就不认识这个入口了，程序跑不起来。打字的时候请多看两眼。

### 第4行：`printf("Hello, World!\n");`

这一行是真正"做事情"的代码。它调用了一个叫 `printf` 的函数，把括号里的内容输出到屏幕上。

- `printf` = print formatted，即"格式化打印"。
- `"Hello, World!\n"` 是一个字符串（双引号括起来的一串字符）。
- `\n` 是一个特殊的转义字符，表示"换行"（newline）。如果没有它，光标会停在 `!` 后面，下一个输出会挤在同一行。
- 末尾的 `;`（分号）表示这一条语句结束。C语言中，一句话说完必须打分号，就像中文句子以句号结束。

**比喻**：`printf` 像一个扩音器。你把话写在小纸条上（括号里的字符串），塞给扩音器，它就帮你在屏幕上大声朗读出来。`\n` 则是扩音器读完一句话后自动换一口气。

### 第5行：`return 0;`

程序执行完毕，要"关门"了。`return 0;` 告诉操作系统："一切正常，我跑完了。"

- `0` 是约定中的"成功"信号。非零值（如 `1`）通常表示出错。
- 这个值可以被其他程序（比如命令行脚本）检测到，用来判断你的程序是否正常结束。

## 6.4 编译与运行

代码写好了，但它是给人看的，计算机不认识。你需要用编译器把它翻译成计算机能执行的二进制文件。

### 6.4.1 编译

打开终端（命令提示符），进入 `hello.c` 所在目录，执行：

```bash
gcc hello.c -o hello
```

分解这条命令：
- `gcc`：调用编译器。
- `hello.c`：源文件名。
- `-o hello`：指定输出文件名（`-o` = output）。如果不指定，默认输出 `a.exe`（Windows）或 `a.out`（macOS/Linux）。

如果一切正常，终端不会有任何输出，但你会看到多了一个新文件：
- Windows 上：`hello.exe`
- macOS/Linux 上：`hello`

如果编译出错，屏幕会显示错误信息。常见错误及处理见下一节。

### 6.4.2 运行

**Windows**：

```cmd
hello.exe
```

或者直接：

```cmd
hello
```

**macOS / Linux**：

```bash
./hello
```

注意 `./` 必须写——它告诉系统"在当前目录下找这个文件"。不加 `.` 的话，系统会去系统路径里找。

你应该看到输出：

```
Hello, World!
```

如果你看到了这行字，恭喜！你的C语言环境已经搭好，第一个程序成功运行。

## 6.5 常见错误与排查

### 错误1：`gcc: command not found` 或 `'gcc' 不是内部或外部命令`

**原因**：编译器没装好，或者PATH没配置对。

**解决**：回到6.1节重新检查安装步骤。运行 `gcc --version` 验证是否可以看到版本号。

### 错误2：编译时报 `hello.c:1:10: fatal error: stdio.h: No such file or directory`

**原因**：编译器找不到标准库头文件。通常是MinGW安装不完整。

**解决**：重新安装完整的MinGW-w64，不要只安装基础包。

### 错误3：编译时报 `error: expected ';' before 'return'`

**原因**：`printf` 那一行末尾忘了打分号。

```c
/* ❌ 错误示例 */
printf("Hello, World!\n")
return 0;

/* ✅ 正确示例 */
printf("Hello, World!\n");
return 0;
```

### 错误4：`warning: control reaches end of non-void function`

**原因**：忘了写 `return 0;`。编译器警告你：`main` 函数应该返回一个 `int`，但你什么都没返回。

```c
/* ❌ 错误示例 */
int main(void)
{
    printf("Hello!\n");
}  /* 没有 return */

/* ✅ 正确示例 */
int main(void)
{
    printf("Hello!\n");
    return 0;
}
```

### 错误5：把 `main` 写成 `mian`

```c
/* ❌ 错误示例 */
int mian(void) { ... }

/* ✅ 正确示例 */
int main(void) { ... }
```

这个错误编译器通常不会报具体错，而是提示 `undefined reference to 'WinMain'` 之类让你摸不着头脑的信息。如果你看到看不懂的链接错误，**先检查 `main` 拼写**。

### 错误6：运行时报 `'hello' is not recognized` 或 `bash: hello: command not found`

**原因**：你可能只编译但忘了指定输出文件名，然后去找了一个叫 `hello` 的文件（实际叫 `a.exe` 或 `a.out`）。

**解决**：用 `gcc hello.c -o hello` 重新编译，或者直接运行 `a.exe` / `./a.out`。

## 6.7 我在做什么？做得对不对？不对怎么办？

### 我在做什么？

我在安装C语言编译器（gcc），写出一个最简单的C程序（Hello World），通过 `printf` 向屏幕输出一行文字，并用 `return 0` 向操作系统汇报成功。

### 我做得对不对？

- 在终端输入 `gcc --version` 能看到版本号 → 编译器装好了。
- 编译 `hello.c` 没有报错 → 代码语法正确。
- 运行后屏幕出现 `Hello, World!` → 程序正确执行。

### 不对怎么办？

| 症状 | 第一步检查 |
|------|-----------|
| gcc命令不识别 | PATH环境变量是否设置 |
| 编译报错 | 分号是否遗漏？`main` 是否拼错？`#include` 首字母大写没？ |
| 运行没输出 | 是否真的运行了可执行文件？编译是否成功？ |
| 乱码或奇怪字符 | 源文件编码是否为UTF-8？Windows终端是否支持UTF-8？ |

## 本章小结

| 知识点 | 说明 |
|--------|------|
| GCC编译器 | 将C源码翻译成机器码的工具 |
| 预处理指令 `#include` | 在编译前复制指定头文件内容 |
| 头文件 `stdio.h` | 标准输入输出函数声明 |
| `main` 函数 | 程序入口，必须有且仅有一个 |
| `printf` 函数 | 输出格式化字符串到屏幕 |
| `return 0` | 向操作系统返回成功状态 |
| 分号 `;` | C语句结束符，不可省略 |
| `\n` | 换行转义字符 |
| `gcc -o` | 指定输出文件名 |

## 练习题

1. **修改输出**：将 `hello.c` 中的 `"Hello, World!\n"` 改为 `"你好，世界！\n"`，重新编译运行，观察输出。

2. **多行输出**：在 `return 0;` 之前再加一行 `printf("I am learning C.\n");`，编译运行，看看两行文字是否各自换行。

3. **去掉 `\n`**：把程序中的 `\n` 删除，编译运行。观察光标停留位置的变化，理解 `\n` 的作用。

4. **验证返回值**（可选挑战）：在终端运行你的程序后，立即输入以下命令查看返回值：
   - Windows：`echo %errorlevel%`
   - macOS/Linux：`echo $?`
   
   把 `return 0;` 改成 `return 7;`，重新编译运行，再查看返回值。体会 `main` 返回值如何传递到操作系统。

5. **故意犯错**：去掉 `printf` 后面的分号，编译看报错信息。把 `main` 写成 `mian`，编译看报错信息。通过阅读报错信息，学会快速定位错误。
