## 我在做什么？

前几章你学会了声明变量、存数据，但程序的核心是"计算"。计算靠什么？靠运算符。

运算符就像数学里的加减乘除号，但C语言中的运算符比数学课上丰富得多，而且带着一堆"坑"。这一章，你会在C语言的运算符"工具箱"里找到前两个抽屉：

1. **算术运算符**：`+` `-` `*` `/` `%` —— 做加减乘除取余。
2. **赋值运算符**：`=` `+=` `-=` `*=` `/=` `%=` —— 给变量改值。
3. **自增自减**：`++` `--` —— 这个看起来简单，但有个经典陷阱 `++i` vs `i++`。

## 11.1 算术运算符

### 11.1.1 五个基本运算符

| 运算符 | 名称 | 示例 | 结果 |
|--------|------|------|------|
| `+` | 加法 | `5 + 3` | `8` |
| `-` | 减法 | `5 - 3` | `2` |
| `*` | 乘法 | `5 * 3` | `15` |
| `/` | 除法 | `5 / 2` | `2`（整数除法，截断！） |
| `%` | 取模（取余） | `5 % 2` | `1`（5除以2的余数） |

**比喻**：算术运算符就像厨房里的基础厨具。`+` 和 `-` 是菜刀和砧板，`*` 是搅拌机，`/` 是分菜勺，`%` 是量杯——专门量"剩多少"。

### 11.1.2 整数除法的截断（再强调一遍）

这是C语言新手踩得最多的坑，第9章提过，这里再深入讲一次。

```c
#include <stdio.h>

int main(void)
{
    printf("5 / 2   = %d\n", 5 / 2);     /* 输出 2 */
    printf("5 / -2  = %d\n", 5 / -2);    /* 输出 -2（C99起向零舍入）*/
    printf("-5 / 2  = %d\n", -5 / 2);    /* 输出 -2 */
    printf("-5 / -2 = %d\n", -5 / -2);   /* 输出 2 */

    return 0;
}
```

**规则**：C99起，整数除法**向零截断**（C89的实现定义行为已被C99修正）。简单说，就是"直接把小数部分切掉"。正数向下切，负数向上切，目标是靠近零。

```
         向零截断示意
         │
    -2.5 │  2.5
      ↓   │   ↓
  ... -3 -2 -1 0 1 2 3 ...
      ↑       │       ↑
  -5/2 = -2  │   5/2 = 2
             │
         向零靠近
```

### 11.1.3 取模运算符 `%`

`%` 是"取余数"运算符，只能用于整数。

```c
int a = 17 % 5;   /* a = 2，因为 17 ÷ 5 = 3 余 2 */
int b = 10 % 3;   /* b = 1 */
int c = 10 % 2;   /* c = 0（偶数取模2余0，奇数取模2余1）*/
int d = 3 % 5;    /* d = 3（被除数比除数小时，余数就是被除数）*/
```

**取模的经典用途**：
- 判断奇偶：`n % 2 == 0` 是偶数，`n % 2 == 1` 是奇数。
- 循环计数：`index % 7` 可以让数字在0到6之间循环。
- 提取个位数：`n % 10` 得到数字的个位。

```c
/* 判断奇偶 */
int num = 7;
if (num % 2 == 0) {
    printf("偶数\n");
} else {
    printf("奇数\n");
}

/* 提取个位 */
int n = 12345;
printf("个位 = %d\n", n % 10);       /* 5 */
printf("十位 = %d\n", (n / 10) % 10); /* 4 */
printf("百位 = %d\n", (n / 100) % 10); /* 3 */
```

### 11.1.4 运算符优先级

回想小学数学："先乘除，后加减"。C语言也是这样：

```c
int result = 5 + 3 * 4;   /* result = 17，不是 32 */
/* 等价于 5 + (3 * 4) */
```

如果想让加法先算，用小括号：

```c
int result = (5 + 3) * 4;   /* result = 32 */
```

**小括号永远是最优先的**。不确定优先级时，加括号。多写一对括号不会影响性能，但少写一对可能让你调试一整天。

### 11.1.5 负数取模的陷阱

```c
int a = -10 % 3;   /* a = -1（C99起，结果符号与被除数相同） */
int b = 10 % -3;   /* b = 1 */
```

不同语言对负数取模的处理不同。C语言中，结果的符号跟被除数（`%`左边的数）一致。Python则跟除数一致。跨语言时要小心。

## 11.2 赋值运算符

### 11.2.1 基本赋值 `=`

`=` 不是"等于"，是"把右边的值放进左边的变量"。

```c
int x;
x = 10;         /* 把10放进x */
x = x + 5;      /* 取出x的值(10)，加5，再放回x（现在x=15） */
```

### 11.2.2 复合赋值运算符

当你想对变量本身做运算时，复合赋值运算符更简洁：

| 写法 | 等价于 | 含义 |
|------|--------|------|
| `x += 5` | `x = x + 5` | x增加5 |
| `x -= 5` | `x = x - 5` | x减少5 |
| `x *= 5` | `x = x * 5` | x乘以5 |
| `x /= 5` | `x = x / 5` | x除以5 |
| `x %= 5` | `x = x % 5` | x取模5 |

**比喻**：复合赋值就像"原地操作"。你不必把东西拿出来、算好、再放回去——直接在原地就搞定了。

```c
int score = 100;
score += 20;   /* score = 120（加20分） */
score -= 30;   /* score = 90（扣30分） */
score *= 2;    /* score = 180（翻倍） */
score /= 3;    /* score = 60（除以3） */
score %= 7;    /* score = 4（60除以7的余数） */
```

### 11.2.3 赋值表达式的值

在C语言中，赋值本身是一个表达式，它有一个值——就是被赋的那个值。

```c
int x, y;
x = (y = 10);  /* y被赋值为10，然后(y=10)这个表达式的值也是10，赋给x */
/* 现在 x=10, y=10 */
```

这也可以用来做"链式赋值"：

```c
int a, b, c;
a = b = c = 0;   /* 从右往左：c=0, b=0, a=0 */
```

但这也可以写出危险的代码：

```c
/* ❌ 危险示例 */
if (x = 5) {     /* 本意是 if (x == 5)，但写成了赋值 */
    /* 这里永远会执行，因为 x=5 的值是5（非0，即真） */
}
```

## 11.3 自增与自减：`++` 和 `--`

### 11.3.1 基本用法

`++` 让变量增加1，`--` 让变量减少1。它们是C语言中最常用的运算符之一。

```c
int i = 0;
i++;    /* i = i + 1; 现在 i = 1 */
++i;    /* i = i + 1; 现在 i = 2 */
i--;    /* i = i - 1; 现在 i = 1 */
--i;    /* i = i - 1; 现在 i = 0 */
```

### 11.3.2 前缀 vs 后缀：经典陷阱

`++i`（前缀）和 `i++`（后缀）在**单独使用时**效果一样，但在**作为表达式的一部分时**，有本质区别：

```c
#include <stdio.h>

int main(void)
{
    int a, b;
    int i = 5;

    /* 后缀：先使用，再自增 */
    a = i++;   /* a = 5, i = 6 */
    printf("i++: a = %d, i = %d\n", a, i);

    /* 前缀：先自增，再使用 */
    i = 5;     /* 重置 */
    b = ++i;   /* b = 6, i = 6 */
    printf("++i: b = %d, i = %d\n", b, i);

    return 0;
}
```

**比喻**：想象你排队买奶茶。
- `i++`（后缀）：你先买奶茶，然后服务员在你编号上画一笔（+1）。你拿到的是原编号。
- `++i`（前缀）：服务员先在你编号上画一笔（+1），然后你再买奶茶。你拿到的是新编号。

**记忆口诀**：
- `i++`：先用后加（先返回 `i` 的旧值，再 `i = i + 1`）
- `++i`：先加后用（先 `i = i + 1`，再返回 `i` 的新值）

```c
int i = 1;
printf("%d\n", i++);   /* 输出 1，i变成2 */
printf("%d\n", ++i);   /* 输出 3，i变成3 */

/* 更复杂的例子 */
int x = 5;
int y = ++x * 2;   /* x先自增为6, 然后 6*2=12, y=12 */
int z = x++ * 2;   /* x=6, 6*2=12, z=12, 然后x自增为7 */
```

### 11.3.3 不要在同一表达式里多次使用 `++`/`--`

```c
/* ❌ 危险：UB！ */
int i = 5;
int result = i++ + ++i;   /* 未定义行为！结果完全不可预测 */

/* ❌ 同样危险 */
int arr[10];
int i = 0;
arr[i] = i++;   /* 未定义行为！ */
```

C标准规定，在同一个表达式中多次修改同一个变量是 `*未定义行为*`。不同编译器可能给出不同结果。**永远不要这样做**。

## 11.4 完整示例：收银机

```c
#include <stdio.h>

int main(void)
{
    /* 商品价格 */
    double apple_price = 5.50;
    double banana_price = 3.20;
    double milk_price = 8.90;

    /* 购买数量 */
    int apples = 3;
    int bananas = 2;
    int milk = 1;

    /* 计算总价 */
    double subtotal = 0.0;
    subtotal += apple_price * apples;    /* 苹果小计 */
    subtotal += banana_price * bananas;  /* 香蕉小计 */
    subtotal += milk_price * milk;       /* 牛奶小计 */

    /* 折扣：满50减10 */
    double discount = 0.0;
    if (subtotal >= 50.0) {
        discount = 10.0;
    }

    double total = subtotal - discount;

    printf("小计: %.2f\n", subtotal);
    printf("折扣: %.2f\n", discount);
    printf("总计: %.2f\n", total);

    /* 取模示例：判断总价是不是整数 */
    int total_cents = (int)(total * 100);  /* 转成分 */
    if (total_cents % 100 == 0) {
        printf("总价是整数元\n");
    } else {
        printf("总价有零头\n");
    }

    return 0;
}
```

## 11.5 常见错误

### 错误1：整数除法丢失精度

```c
/* ❌ 错误示例 */
double result = 1 / 2;   /* result = 0.0（整数除法，截断后才是0，再转double） */

/* ✅ 正确示例 */
double result = 1.0 / 2;    /* result = 0.5 */
double result = 1 / 2.0;    /* result = 0.5 */
```

### 错误2：`++i` vs `i++` 混淆

```c
/* ❌ 错误示例 */
int i = 3;
int result = i++ * 2;   /* 你以为 result = 8？实际是 6 */
/* 因为 i++ 返回 3（旧值），3*2=6 */

/* ✅ 正确示例 */
int i = 3;
int result = ++i * 2;   /* result = 8，因为 ++i 返回 4 */
```

### 错误3：`%` 用于浮点数

```c
/* ❌ 错误示例 */
double x = 10.5 % 3.2;   /* 编译报错！% 只能用于整数 */

/* ✅ 正确示例 */
double x = fmod(10.5, 3.2);   /* 用 math.h 中的 fmod 函数 */
```

### 错误4：`=` 和 `==` 混淆（预告）

```c
/* ❌ 错误示例 */
int x = 5;
if (x = 10) {   /* 赋值，不是比较！x被改成10，条件永远为真 */
    printf("x等于10\n");
}

/* ✅ 正确示例 */
int x = 5;
if (x == 10) {   /* 这才是比较 */
    printf("x等于10\n");
}
```

## 11.6 我在做什么？做得对不对？不对怎么办？

### 我在做什么？

我在学习C语言的算术运算符（`+` `-` `*` `/` `%`）、赋值运算符（`=` `+=` 等）、自增自减运算符（`++` `--`），以及它们的优先级和常见陷阱。

### 我做得对不对？

- 你能解释 `5 / 2` 为什么等于 `2` 而不是 `2.5` → 整数除法过关。
- 你能说出 `i++` 和 `++i` 的区别 → 自增自减过关。
- 你能用 `%` 判断一个数是奇数还是偶数 → 取模过关。
- 你能写出至少三种复合赋值运算符 → 赋值过关。

### 不对怎么办？

| 症状 | 第一步检查 |
|------|-----------|
| 除法结果不对（小数丢失） | 是否用了整数除法？改成 `5.0 / 2` |
| `++`/`--` 结果与预期不符 | 检查是前缀还是后缀？是否在同一表达式里改了多次？ |
| `%` 编译报错 | 操作数是否为浮点类型？`%` 只能用于整数 |
| 赋值后值不对 | 是否用了复合赋值？`x += 5` 是 `x = x + 5`，不是 `x = 5` |

## 本章小结

| 知识点 | 说明 |
|--------|------|
| `+` `-` `*` `/` `%` | 五个基本算术运算符 |
| 整数除法截断 | 两整数相除，小数部分被丢弃（向零截断） |
| 取模 `%` | 只能用于整数，得余数。常用于奇偶判断、循环计数 |
| 优先级 | 乘除取模 > 加减，有疑问加括号 |
| 复合赋值 `+=` `-=` 等 | `x += 5` 等价于 `x = x + 5` |
| 自增 `++` | `i++` 先用后加，`++i` 先加后用 |
| 自减 `--` | `i--` 先用后减，`--i` 先减后用 |
| 同一表达式多次修改 | 未定义行为，禁止 |

## 练习题

1. **基础运算**：写程序计算 `(8 + 4) * 3 - 6 / 2`，输出结果并验证。

2. **整数除法 vs 浮点除法**：分别计算 `7 / 2`、`7.0 / 2`、`7 / 2.0`、`(double)7 / 2`，输出四种结果，解释差异。

3. **取模练习**：输入一个三位数（如 `123`），用取模和除法分别提取百位、十位、个位。

4. **自增自减对比**：写程序输出 `i++` 和 `++i` 在表达式中的不同行为。用 `int i = 10;` 分别测试 `printf("%d\n", i++);` 和 `printf("%d\n", ++i);`。

5. **复合赋值**：声明 `int x = 100;`，依次执行 `x += 50; x -= 30; x *= 2; x /= 3; x %= 7;`，每一步后输出 `x` 的值。
