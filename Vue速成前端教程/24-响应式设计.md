## 三、分步操作

### 步骤 1：什么是响应式设计？

**一句话**：同一个网页，在不同屏幕尺寸下自动调整布局，让手机、平板、电脑的用户都能舒服地浏览。

**比喻**：响应式设计就像水——倒进杯子里是杯子的形状，倒进碗里是碗的形状，倒进河里是河的形状。你只写一套 HTML，CSS 负责"变形"。

**核心工具只有两个**：
1. **媒体查询（Media Query）**：当屏幕宽度 ≤ 768px 时，执行某段 CSS。
2. **弹性布局（Flexbox / Grid）**：元素自动换行，不写死宽度。

---

### 步骤 2：媒体查询 — 屏幕宽度的"开关"

**我在做什么？**
写 CSS 规则，告诉浏览器："当屏幕宽度小于等于 768px 时，用这一套样式；大于 768px 时，用另一套。"

**基础语法**：

```css
/* 电脑端默认样式（大于 768px） */
.blog-header {
  display: flex;
  justify-content: space-between;
  padding: 20px 40px;
}

/* 手机端（≤ 768px）覆写 */
@media (max-width: 768px) {
  .blog-header {
    flex-direction: column;
    padding: 10px 16px;
  }
}
```

**三个常用断点**（业界约定俗成，不必死记，但好用）：

| 断点 | 覆盖设备 | 用法 |
|------|----------|------|
| `max-width: 480px` | 小手机 | 最紧凑的布局 |
| `max-width: 768px` | 手机横屏 + 小平板 | **最常用的移动端断点** |
| `max-width: 1024px` | 大平板 + 小笔记本 | 中等屏幕调整 |

🤔 **想多一点**：为什么用 `max-width` 而不是 `min-width`？`max-width: 768px` 的意思是"屏幕宽度 ≤ 768px 时生效"。这遵循"移动优先"的反向思路——先写电脑端的大屏样式作为默认，再用 `max-width` 覆写小屏。如果你更喜欢"移动优先"（先写手机样式，再用 `min-width` 覆写大屏），两种都可以，但本项目统一用 `max-width`。

---

### 步骤 3：移动端汉堡菜单

**我在做什么？**
手机上导航栏如果还是横着排一排链接，文字会挤成一团甚至溢出。解决方案是**汉堡菜单**（☰ 图标）：默认隐藏导航链接，点击 ☰ 后展开。

**改造 BlogHeader.vue**：

```html
<template>
  <header class="blog-header">
    <div class="header-top">
      <h1>我的博客</h1>
      <!-- ☰ 只在手机上显示 -->
      <button class="hamburger" @click="menuOpen = !menuOpen">
        {{ menuOpen ? '✕' : '☰' }}
      </button>
    </div>

    <!-- 导航链接：电脑端始终显示，手机端根据 menuOpen 切换 -->
    <nav :class="['nav-links', { 'nav-open': menuOpen }]">
      <router-link to="/" @click="menuOpen = false">首页</router-link>
      <router-link to="/create" @click="menuOpen = false">写文章</router-link>
      <template v-if="userStore.isLoggedIn">
        <span>👋 {{ userStore.username }}</span>
        <button @click="handleLogout">退出</button>
      </template>
      <template v-else>
        <router-link to="/login" @click="menuOpen = false">登录</router-link>
      </template>
    </nav>
  </header>
</template>

<script setup>
import { ref } from 'vue'
import { useUserStore } from '@/stores/user'

const userStore = useUserStore()
const menuOpen = ref(false)

function handleLogout() {
  userStore.logout()
  menuOpen.value = false
}
</script>
```

**CSS 样式**：

```css
/* ===== 电脑端默认 ===== */
.blog-header {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 16px 40px;
  background: #fff;
  box-shadow: 0 2px 8px rgba(0,0,0,0.06);
}

.hamburger {
  display: none;  /* 电脑端隐藏汉堡按钮 */
}

.nav-links {
  display: flex;
  align-items: center;
  gap: 20px;
}

/* ===== 手机端（≤ 768px）覆写 ===== */
@media (max-width: 768px) {
  .blog-header {
    flex-direction: column;
    padding: 12px 16px;
  }

  .header-top {
    display: flex;
    justify-content: space-between;
    align-items: center;
    width: 100%;
  }

  .hamburger {
    display: block;  /* 👈 手机端显示汉堡按钮 */
    background: none;
    border: none;
    font-size: 24px;
    cursor: pointer;
  }

  .nav-links {
    display: none;          /* 默认隐藏 */
    flex-direction: column; /* 纵向排列 */
    width: 100%;
    padding-top: 12px;
    gap: 12px;
  }

  .nav-links.nav-open {
    display: flex;  /* 点击 ☰ 后展开 */
  }
}
```

❌ **错误示例**：汉堡菜单展开后，点击导航链接跳转了但菜单还开着。
✅ **正确示例**：每个 `<router-link>` 的 `@click` 中设置 `menuOpen = false`。

**我做得对不对？**
- 电脑端：导航栏横向排列，看不到 ☰ 按钮。
- 手机端（DevTools 切换到 375px 宽）：只看到标题和 ☰ 按钮，点击 ☰ 后导航链接纵向展开，再点 ✕ 消失。

### 步骤 4：Flexbox 自动换行 — 文章列表网格

**我在做什么？**
电脑上文章列表可以 3 列并排，但手机上 3 列会挤成面条。用 Flexbox 的 `flex-wrap` 让卡片自动换行。

**改造 Home.vue 的文章列表容器**：

```html
<template>
  <div class="home">
    <!-- loading / error / empty 状态... -->

    <div v-else class="post-grid">
      <article v-for="post in posts" :key="post.id" class="post-card">
        <h2>{{ post.title }}</h2>
        <p class="post-excerpt">{{ post.excerpt }}</p>
        <router-link :to="`/post/${post.id}`">阅读更多</router-link>
      </article>
    </div>
  </div>
</template>
```

```css
.post-grid {
  display: flex;
  flex-wrap: wrap;    /* 🎯 一行放不下就自动换行 */
  gap: 20px;          /* 卡片间距 */
  padding: 20px;
}

/* 每张卡片的基础宽度 */
.post-card {
  flex: 1 1 300px;    /* 最小 300px，剩余空间均分 */
  /* 等价于：flex-grow: 1; flex-shrink: 1; flex-basis: 300px; */
  background: #fff;
  border-radius: 8px;
  padding: 20px;
  box-shadow: 0 2px 8px rgba(0,0,0,0.06);
}
```

**效果解释**：
- 屏幕宽度 ≥ 940px 时：`(300px × 3) + (20px × 2) = 940px`，刚好 3 列。
- 屏幕宽度 600px ~ 940px 时：2 列，每列自动拉伸填满剩余空间。
- 屏幕宽度 < 600px 时：1 列，每列占满整行。

🤔 **想多一点**：`flex: 1 1 300px` 这三个数字各代表什么？记忆口诀——"**长大、缩小、起点**"：第一位 `flex-grow` 是"有多想长大"，第二位 `flex-shrink` 是"有多想缩小"，第三位 `flex-basis` 是"起步宽度"。这里 `1 1 300px` 的意思是"我愿意长大也愿意缩小，但我至少 300px 宽"。

❌ **错误示例**：给每张卡片写死 `width: 300px`，手机上就溢出了。
✅ **正确示例**：用 `flex: 1 1 300px`，让浏览器自动分配。

### 步骤 5：图片不溢出

**我在做什么？**
文章里插入的图片如果宽度超过屏幕，会出现横向滚动条——手机端的噩梦。加一条全局规则解决。

**全局样式**（在 `src/assets/main.css` 或 `index.html` 的 `<style>` 中）：

```css
img {
  max-width: 100%;
  height: auto;
}
```

**效果**：图片永远不会超过它所在容器的宽度。如果图片原本 2000px 宽但容器只有 320px，图片会自动缩到 320px。

---

### 步骤 6：rem/vw 相对单位 — 告别写死的 px

**我在做什么？**
`px` 是绝对单位，14px 在手机上可能偏大，在 4K 显示器上又偏小。用相对单位让字体等元素随屏幕等比缩放。

**单位对比**：

| 单位 | 含义 | 适用场景 |
|------|------|----------|
| `px` | 绝对像素 | 边框、阴影等不需要缩放的元素 |
| `rem` | 相对于根元素 `<html>` 的 `font-size` | 字体大小、间距、整体排版 |
| `vw` | 视口宽度的 1% | 全宽容器、大标题 |
| `vh` | 视口高度的 1% | 全高容器、首屏占位 |
| `%` | 相对于父元素 | 宽度、内边距 |

**实用技巧**：

```css
/* 设置根字体大小，方便 rem 计算 */
html {
  font-size: 16px;  /* 1rem = 16px */
}

/* 标题用 vw 保证在手机上也不换行 */
.hero-title {
  font-size: clamp(24px, 5vw, 48px);  /* 最小 24px，最大 48px，中间按 5vw */
}

/* 文章内容用 rem */
.post-content {
  font-size: 1rem;      /* = 16px */
  line-height: 1.75;
  max-width: 720px;     /* 阅读舒适宽度，用 px 固定 */
  margin: 0 auto;
}
```

🤔 **想多一点**：`clamp(最小值, 理想值, 最大值)` 是响应式排版的神器。`clamp(24px, 5vw, 48px)` 的意思是：平时按 5vw（屏幕宽度的 5%）计算，但不得小于 24px 也不得大于 48px。这样在 320px 宽的手机上字体是 24px（因为 5vw×320=16px < 24px），在 1920px 宽的显示器上是 48px（因为 5vw×1920=96px > 48px）。

## 六、小结

| 概念 | 比喻 | 关键代码 |
|------|------|----------|
| **响应式设计** | 水——倒进什么容器就是什么形状 | 一套 HTML，CSS 自适应 |
| **媒体查询 (@media)** | 屏幕宽度的"开关" | `@media (max-width: 768px) { ... }` |
| **汉堡菜单** | 手机端的收纳盒 | `menuOpen` 控制 `v-if` / `:class` |
| **Flexbox 换行** | 自动换行的书架 | `flex-wrap: wrap` + `flex: 1 1 300px` |
| **rem** | 相对于根字体的大小 | `1rem = 16px`（默认） |
| **vw** | 屏幕宽度的 1% | `5vw` = 屏幕宽度的 5% |
| **clamp()** | 带上下限的弹性值 | `clamp(24px, 5vw, 48px)` |

## 七、术语附录

| 术语 | 解释 |
|------|------|
| **响应式设计 (Responsive Design)** | 同一套 HTML/CSS 代码在不同屏幕尺寸（手机、平板、电脑）下自动调整布局，无需为每种设备单独开发。 |
| **媒体查询 (Media Query)** | CSS 的条件语句，根据屏幕宽度、分辨率、方向等条件应用或覆盖样式。写法：`@media (条件) { 样式 }`。 |
| **汉堡菜单 (Hamburger Menu)** | 移动端常见的三横线图标（☰），点击后展开导航菜单。因图标像汉堡而得名。 |
| **rem (Root EM)** | 相对于 HTML 根元素 `font-size` 的单位。如果 `<html>` 的 `font-size` 是 16px，则 `1rem = 16px`。 |
| **vw (Viewport Width)** | 视口宽度单位。`1vw` = 浏览器窗口宽度的 1%。`100vw` = 全屏宽。 |
| **vh (Viewport Height)** | 视口高度单位。`1vh` = 浏览器窗口高度的 1%。`100vh` = 全屏高。 |
| **clamp()** | CSS 函数，接收三个参数：最小值、理想值、最大值。浏览器在理想值基础上自动限制在最小值和最大值之间。 |

## 八、已知坑点与禁止事项

- **禁止在移动端使用 `hover` 效果**：手机没有鼠标，`:hover` 在移动端会出现"点一下有效果，再点一下才触发"的诡异行为。用 `@media (hover: hover)` 包裹 hover 样式，或直接去掉。
- **禁止写死 `width`**：`width: 1200px` 在手机上必然溢出，一律用 `max-width` 或 `%`。
- **`@media` 的断点不要太多**：2-3 个断点足够，5 个以上说明设计思路有问题。
- **测试真机**：DevTools 模拟不完全等于真机，条件允许时用真机扫一遍。
