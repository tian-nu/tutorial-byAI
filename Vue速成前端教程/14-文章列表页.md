### 步骤2：创建 BlogCard 组件

#### 我在做什么？

每篇文章在首页上显示为一张「卡片」。卡片是公共组件，因为它会在首页列表中出现多次，也可能在搜索结果页等地方复用。

**创建 `src/components/BlogCard.vue`**：

```vue
<template>
  <div class="blog-card">
    <router-link :to="`/post/${post.id}`" class="card-title">
      {{ post.title }}
    </router-link>
    <p class="card-summary">{{ post.summary }}</p>
    <div class="card-meta">
      <span>👤 {{ post.author }}</span>
      <span>📅 {{ post.createdAt }}</span>
    </div>
  </div>
</template>

<script setup>
// 接收父组件传入的文章数据
defineProps({
  post: {
    type: Object,
    required: true   // 必须传入，否则控制台警告
  }
})
</script>

<style scoped>
.blog-card {
  border: 1px solid #e0e0e0;
  border-radius: 8px;
  padding: 16px;
  margin-bottom: 12px;
  transition: box-shadow 0.2s;
}
.blog-card:hover {
  box-shadow: 0 2px 12px rgba(0, 0, 0, 0.1);
}
.card-title {
  font-size: 18px;
  font-weight: bold;
  color: #2c3e50;
  text-decoration: none;
}
.card-title:hover {
  color: #42b983;
}
.card-summary {
  color: #666;
  margin: 8px 0;
  line-height: 1.6;
}
.card-meta {
  font-size: 13px;
  color: #999;
  display: flex;
  gap: 16px;
}
</style>
```

#### 代码逐行解释

| 代码 | 含义 |
|------|------|
| `defineProps({ post: ... })` | 声明这个组件接收一个叫 `post` 的 prop（属性） |
| `required: true` | 父组件必须传 `post`，不传就报警告 |
| `:to="\`/post/${post.id}\`"` | 动态拼接跳转链接，`post.id=1` 时生成 `/post/1` |
| `scoped` | 样式只作用于当前组件，不会污染其他组件 |

❌ **错误**：`to="/post/post.id"` — 引号内是字符串，`post.id` 不会被替换成数字。
✅ **正确**：`:to="`/post/${post.id}`"` — 冒号开头表示这是 JavaScript 表达式，模板字符串 `${}` 会替换。

🤔 **想多一点**：`defineProps` 是 Vue 3 的编译宏，不需要手动 import。你在 `<script setup>` 里直接写就行，Vue 编译器会自动处理。这就像你写 `ref()` 不需要 import 一样——但 `ref` 其实需要 `import { ref } from 'vue'`，而 `defineProps` 是真·不需要 import。

### 步骤3：改造 Home.vue 渲染文章列表

#### 我在做什么？

把 `v-for` 循环和 `BlogCard` 组件结合起来，让首页渲染出文章列表。

**修改 `src/views/Home.vue`**：

```vue
<template>
  <div class="home">
    <h1>📚 最新文章</h1>

    <!-- v-for 循环渲染文章卡片 -->
    <BlogCard
      v-for="post in posts"
      :key="post.id"
      :post="post"
    />
  </div>
</template>

<script setup>
import { posts } from '../mock/posts.js'   // 导入假数据
import BlogCard from '../components/BlogCard.vue'
</script>

<style scoped>
.home {
  max-width: 720px;
  margin: 0 auto;
  padding: 20px;
}
h1 {
  margin-bottom: 20px;
}
</style>
```

#### 代码逐行解释

| 代码 | 含义 |
|------|------|
| `v-for="post in posts"` | 遍历 `posts` 数组，每次循环把当前元素赋值给 `post` |
| `:key="post.id"` | 给每个循环项一个唯一标识，Vue 用它来高效更新 DOM |
| `:post="post"` | 把当前文章对象传给 `BlogCard` 组件的 `post` prop |
| `import { posts } from '../mock/posts.js'` | 导入假数据，注意 `../` 是从 `views/` 回到 `src/` |

❌ **错误**：忘记写 `:key="post.id"`
- 后果：Vue 会在控制台警告，且列表更新时可能出现渲染错乱。

✅ **正确**：`v-for="post in posts" :key="post.id"`

### 步骤5：改造 App.vue 使用 BlogHeader

#### 我在做什么？

把 `App.vue` 里手写的导航栏替换成 `BlogHeader` 组件，让结构更干净。

**修改 `src/App.vue`**：

```vue
<template>
  <div id="app">
    <!-- 使用封装的导航栏组件 -->
    <BlogHeader />

    <!-- 路由出口 -->
    <router-view />
  </div>
</template>

import BlogHeader from './components/BlogHeader.vue'
</script>

<style>
/* 全局样式：去掉默认边距 */

  margin: 0;
  padding: 0;
  box-sizing: border-box;
}
body {
  font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif;
  background: #f5f5f5;
  color: #333;
}
</style>
```

#### 对比改造前后

| | 改造前 | 改造后 |
|------|--------|--------|
| 导航栏 | 手写 `<nav>` 在 App.vue 里 | 独立 `BlogHeader.vue` 组件 |
| App.vue 复杂度 | 模板里混着导航和路由出口 | 只有两行：`<BlogHeader />` + `<router-view />` |
| 可维护性 | 改导航要改 App.vue | 改导航只改 BlogHeader.vue |

## 七、已知坑点与禁止事项

- ⚠️ **`v-for` 必须配 `:key`**：忘记写 key 会导致列表更新时出现渲染错乱，Vue 会在控制台警告。
- ⚠️ **`import` 路径注意 `../` 层级**：`views/Home.vue` 引用 `mock/posts.js` 需要 `../mock/posts.js`（回退一级到 `src/`），而 `App.vue` 引用 `components/BlogHeader.vue` 只需要 `./components/BlogHeader.vue`（同级）。
- ⚠️ **模板字符串中 `:to` 的冒号不能丢**：`to="/post/${post.id}"` 是错的（当成普通字符串），`:to="`/post/${post.id}`"` 才是对的（JS 表达式）。
