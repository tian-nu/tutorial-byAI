## 26.1 Cursor 是什么？——带AI大脑的编辑器

> **📖 术语：Cursor**——它是基于 VS Code（微软的代码编辑器）改造出来的一款编辑器，但**原生就把AI深度集成进去了**。你可以把它理解成："VS Code的身体 + GPT/Claude的大脑"。

想象一下：你用普通编辑器（比如纯VS Code），你得自己写每一行代码。有了Cursor，AI就坐在你旁边——你打字它帮你补，你有问题它帮你答，你要改代码它帮你改，甚至复杂需求它能自己找文件自己改完。

一个简单的比喻：

| 工具 | 像什么 |
|------|--------|
| 记事本 | 一支铅笔，你自己写 |
| VS Code（纯） | 一支高级钢笔，写起来舒服，但还是你自己写 |
| **Cursor** | 一支会写字的钢笔——你告诉它写什么，它帮你写 |

Cursor 有四个核心功能，这也是我们这一篇要全部讲完的：
- **Tab补全**：你打字，AI实时预测下一步
- **Chat模式**：你和AI对话，问问题、改代码
- **Composer模式**：给AI一个需求，它一次性生成/修改多个文件
- **Agent模式**：AI自动驾驶——它自己读项目、改代码、运行终端、循环修复

## 26.2 界面导览：每个区域是干什么的

第一次打开Cursor，你可能会觉得"这不就是VS Code吗"——没错，大部分区域和VS Code一样，但右边多了一块AI专属区域。

```
┌─────────────────────────────────────────────────────┐
│ ① 菜单栏（最顶部） 文件 编辑 选择 查看 转到 ...      │
├──────────┬───────────────────┬─────────────────────┤
│          │                   │                     │
│ ② 侧边栏 │  ③ 编辑区         │  ④ AI面板           │
│ (左侧)    │  (中间，最大区域)  │  (右侧)             │
│          │                   │                     │
│ 文件列表 │  你写代码/看代码   │  Chat/Composer在这里 │
│ 搜索      │  的地方           │                     │
│ 扩展管理  │                   │                     │
│          │                   │                     │
├──────────┴───────────────────┴─────────────────────┤
│ ⑤ 底部状态栏  Python 3.12.3 | 行 1, 列 1 | UTF-8   │
└─────────────────────────────────────────────────────┘
```

**① 菜单栏（最顶部）**：和所有桌面应用一样，各种功能入口。你刚开始用，这里基本不用碰。

**② 侧边栏（左侧）**：显示你当前打开的项目文件夹里的所有文件，就像Windows资源管理器。你可以在这里新建、删除、打开文件，管理扩展，看Git版本。

**③ 编辑区（中间最大块）**：这是你的主战场——你在这里看代码、改代码、选中代码让AI修改。

**④ AI面板（右侧）**：这是**Cursor的灵魂**。这里有两个标签页：
- **Chat**：和AI聊天对话（我们第28章讲）
- **Composer**：批量生成多个文件（我们第29章讲）

如果你打开后右侧AI面板不见了，别急——按快捷键 **`Ctrl + L`**（Windows）或 **`Command + L`**（Mac）就能打开。

**⑤ 底部状态栏**：显示一些状态信息——当前用的什么Python版本、光标在哪一行、文件编码是什么。刚开始不用关心。

## 26.3 导入VS Code配置（如果你之前用VS Code）

如果你之前一直在用VS Code，Cursor非常贴心——**它能直接导入你VS Code的所有配置**：
- 快捷键设置
- 已安装的扩展
- 颜色主题
- 用户设置

### 怎么导入：

**1.** 第一次启动Cursor的时候，它会自动问你："是否从VS Code导入配置？"

**2.** 如果它问了，而且你确实想用VS Code的配置——直接点"导入"，等着它完成就行。

**3.** 如果启动的时候你跳过了，后面还能导入：
- 点击左下角齿轮（设置）
- 选择"Import from VS Code"
- 按照提示操作就行

✅ 如果你是零基础，之前从来没用过VS Code——直接选"Skip跳过"就好，Cursor会给你默认配置，够用了。

## 26.4 选择AI模型：哪个模型适合我？

Cursor内置了好几个AI模型，就像你车里有不同档位——不同场景用不同档位。

最常用的三个：

| 模型 | 特点 | 推荐场景 |
|------|------|----------|
| **GPT-4o** | 速度快、理解准、代码质量高 | ⭐ **默认主力**，90%的场景用它 |
| **Claude 3.5 Sonnet** | 超长上下文处理能力强，能一次看几千行代码 | 需要处理大量代码时切换到它 |
| **本地模型（Ollama等）** | 代码不上传云端，隐私性好 | 对隐私要求极高，代码不能出内网 |

### 怎么切换模型：

在右侧AI面板的输入框上方，有个下拉菜单，点击就能选。

**本书建议**：默认就用 **GPT-4o**，遇到大文件（几百行）再切Claude就好。

## 26.5 你的第一个AI操作：让Cursor写Hello World

准备好了吗？我们来跑通完整流程——让AI写一个Python程序，然后运行它。

### **1.** 新建文件
- 按 **`Ctrl + N`**（Windows）或 **`Command + N`**（Mac）新建一个空白文件
- 按 **`Ctrl + S`** 保存，文件名输入 `hello.py`

### **2.** 召唤AI写代码
- 在编辑区按 **`Ctrl + K`**——这会弹出一个行内AI输入框
- 输入这句话：

```
写一个Hello World Python程序，打印'Hello AI Coding'
```

- 按回车

几秒钟后，AI就会生成代码：

```python
print("Hello AI Coding")
```

### **3.** 运行代码验证
- 按 **`Ctrl + `` `**（反引号键，ESC下面那个）打开终端
- 在终端输入：

```bash
python hello.py
```

（Mac用户用 `python3 hello.py`）

- 按回车，你应该看到：

```
Hello AI Coding
```

✅ **恭喜！** 你已经完成了在Cursor里的第一次AI编程——从需求到代码到运行，全程不到一分钟。

## 26.6 如果出错了怎么办

### 🔴 问题1：Cursor打不开，双击没反应

**可能原因**：安装包下载不完整。

**解决**：重新去官网下载一遍，换个浏览器下载试试。

### 🔴 问题2：Ctrl+K按了没反应

**可能原因1**：你没登录。AI功能需要登录才能用。
**解决**：检查右上角有没有头像，没有就点登录。

**可能原因2**：中文输入法冲突。
**解决**：按 `Shift` 切换到英文输入法再试。

### 🔴 问题3：AI说"额度用完了"

这是免费版的限制——免费版每月有50次高级模型（GPT-4o/Claude）请求。用完了怎么办？

**解决**：切到 GPT-4o-mini 继续用，免费无限额度。虽然质量稍差，但学习完全够用，下个月额度会自动重置。

### 🔴 问题4：模型加载失败，AI不回答

**可能原因**：网络问题。Cursor的AI服务在国内可以直接用，但偶尔会抽风。

**解决**：
1. 关掉Cursor再重新打开
2. 切个模型试试（比如从GPT-4o切到Claude）
3. 如果还是不行，等几分钟再试，一般是临时网络波动。
