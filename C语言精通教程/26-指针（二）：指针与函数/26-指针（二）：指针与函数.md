## 我在做什么？

上一章你知道了指针是什么——存储地址的变量。这一章把指针和函数结合起来，回答两个关键问题：

1. 指针作为函数参数，怎么让函数"真正"修改外部变量？
2. 函数返回指针，有什么坑？

读完这一章，你会写经典的 `swap` 函数，理解为什么不能返回局部变量的地址，以及避免 *野指针* *此术语见附录F* 和 *悬空指针* *此术语见附录F* 的陷阱。

## 一、指针作为函数参数：真正修改外部变量

### 1.1 回顾：传值调用的局限

第 19 章讲过，C 语言只有传值调用。函数收到的是实参的拷贝，修改形参不影响实参。

```c
void bad_swap(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
    // a 和 b 是拷贝，外面的变量没变
}
```

### 1.2 传指针：把地址告诉函数

既然函数不能修改外部变量，那我们把外部变量的**地址**传进去，让函数通过地址去修改。

```c
#include <stdio.h>

void swap(int *a, int *b) {
    int temp = *a;   // 取出 a 指向的值
    *a = *b;         // 把 b 指向的值赋给 a 指向的位置
    *b = temp;       // 把暂存的值赋给 b 指向的位置
}

int main() {
    int x = 5, y = 10;
    printf("交换前：x = %d, y = %d\n", x, y);
    swap(&x, &y);
    printf("交换后：x = %d, y = %d\n", x, y);
    return 0;
}
```

输出：
```text
交换前：x = 5, y = 10
交换后：x = 10, y = 5
```

### 1.3 执行过程图解

```text
调用 swap(&x, &y) 时：

main 中：                     swap 中：
┌──────────┐ ┌──────────┐    ┌──────────┐ ┌──────────┐
│ x = 5    │ │ y = 10   │    │ a = &x   │ │ b = &y   │
│ 地址 0x100│ │ 地址 0x104│    │ 值 0x100 │ │ 值 0x104 │
└──────────┘ └──────────┘    └─────│─────┘ └─────│─────┘
      ↑             ↑              │             │
      └─────────────┴──────────────┘             │
                  └───────────────────────────────┘

执行 *a = *b 后：
┌──────────┐ ┌──────────┐
│ x = 10   │ │ y = 10   │    temp 中暂存着 5
│ 地址 0x100│ │ 地址 0x104│
└──────────┘ └──────────┘

执行 *b = temp 后：
┌──────────┐ ┌──────────┐
│ x = 10   │ │ y = 5    │
│ 地址 0x100│ │ 地址 0x104│
└──────────┘ └──────────┘
```

### 1.4 比喻：远程遥控

```text
传值调用：我把我的文件复印一份给你，你在复印件上改，我原件不变。
传指针：   我把我的文件柜钥匙给你，你打开柜子直接在原件上改。

指针就是钥匙。
```

### 1.5 一个函数返回多个值

C 语言函数只能返回一个值。但通过指针参数，可以"返回"多个值：

```c
#include <stdio.h>
#include <math.h>

// 计算一个数的平方和立方，通过指针参数返回
void square_and_cube(double x, double *square, double *cube) {
    *square = x * x;
    *cube = x * x * x;
}

int main() {
    double s, c;
    square_and_cube(3.0, &s, &c);
    printf("3^2 = %.0f, 3^3 = %.0f\n", s, c);  // 输出 3^2 = 9, 3^3 = 27
    return 0;
}
```

## 二、返回指针的函数

### 2.1 函数可以返回指针

```c
int* find_max(int *arr, int size) {
    int *max = &arr[0];
    for (int i = 1; i < size; i++) {
        if (arr[i] > *max) {
            max = &arr[i];
        }
    }
    return max;   // 返回指向最大值的指针
}

int main() {
    int nums[] = {3, 7, 1, 9, 4};
    int *p = find_max(nums, 5);
    printf("最大值：%d\n", *p);  // 输出 9
    return 0;
}
```

这里返回的是 `arr` 中的地址，`arr` 是 `main` 中的数组，函数返回后 `arr` 仍然存在，所以安全。

### 2.2 致命错误：返回局部变量的地址

```c
int* create_number() {
    int n = 100;     // 局部变量，在栈上
    return &n;       // 危险！返回局部变量的地址
}

int main() {
    int *p = create_number();
    printf("%d\n", *p);  // 可能输出 100，也可能输出垃圾值
    return 0;
}
```

```text
create_number 被调用时：
┌──────────────────────┐
│ 栈帧：create_number  │
│ ┌──────────────────┐ │
│ │ n = 100          │ │ ← 返回 &n
│ └──────────────────┘ │
└──────────────────────┘
create_number 返回后：
┌──────────────────────┐
│ 栈帧被回收            │
│ ┌──────────────────┐ │
│ │ 垃圾值            │ │ ← p 还指向这里！
│ └──────────────────┘ │
└──────────────────────┘
```

函数返回后，局部变量 `n` 的内存被回收。`p` 变成了 *悬空指针* *此术语见附录F*——指向已经释放的内存。

### 2.3 三种安全的返回指针方式

**方式一**：返回传入的指针参数（如 `find_max` 返回 `arr` 中的地址）。

**方式二**：返回静态局部变量的地址：

```c
int* get_counter() {
    static int counter = 0;  // 静态变量，程序结束才释放
    counter++;
    return &counter;         // 安全
}
```

**方式三**：返回动态分配的内存地址（第 30 章讲 `malloc`）：

```c
int* create_array(int size) {
    int *arr = malloc(size * sizeof(int));  // 堆上分配，手动释放
    return arr;                              // 安全
}
```

## 三、指针的 `const` 限定

### 3.1 指向常量的指针

```c
const int *p;   // p 指向的内容不能通过 p 修改
int const *p;   // 同上，写法不同，意思一样
```

```c
void print_array(const int *arr, int size) {
    // arr[0] = 100;  // 错误！不能通过 const 指针修改
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);  // 读取没问题
    }
}
```

`const` 在这里的意思是"我承诺不会通过这个指针修改数据"。它让调用者放心——传进去的数据不会被函数修改。

### 3.2 常量指针（指针本身不可变）

```c
int *const p = &a;  // p 永远指向 a，不能再指向别处
// p = &b;          // 错误！p 是常量指针
*p = 20;            // 但可以修改 p 指向的值
```

### 3.3 指向常量的常量指针

```c
const int *const p = &a;  // 既不能改 p 指向谁，也不能通过 p 修改值
```

### 3.4 记忆技巧

```text
const int *p     → 从右往左读：p 是一个指针，指向 int 常量
int *const p     → 从右往左读：p 是一个常量指针，指向 int
const int *const p → p 是一个常量指针，指向 int 常量
```

## 四、野指针与悬空指针

### 4.1 野指针

*野指针* *此术语见附录F* 是指**未初始化的指针**。它指向的内存地址是随机的，解引用它会导致不可预测的行为。

```c
int *p;      // 未初始化，p 的值是随机的
*p = 10;     // 危险！写到了随机内存地址
```

**预防**：永远初始化指针。如果暂时不知道指向谁，初始化为 `NULL`。

```c
int *p = NULL;  // 明确表示"不指向任何地方"
```

### 4.2 悬空指针

*悬空指针* *此术语见附录F* 是指**指向已释放内存的指针**。它不指向随机地址（和野指针不同），而是指向曾经有效但现在已经无效的内存。

```c
int *p;
{
    int x = 10;
    p = &x;     // p 指向 x
}               // x 离开作用域，被销毁
// p 现在是悬空指针
*p = 20;       // 危险！x 已经不存在了
```

另一个常见场景：

```c
int *p = malloc(sizeof(int));
free(p);       // 释放内存
// p 现在是悬空指针
*p = 10;       // 危险！use after free
```

**预防**：释放后立即置为 NULL。

```c
free(p);
p = NULL;      // 明确标记为无效
```

### 4.3 对比

| | 野指针 | 悬空指针 |
|---|---|---|
| 成因 | 未初始化 | 指向已释放的内存 |
| 指向哪里 | 随机地址 | 曾经有效的地址 |
| 预防 | 初始化为 NULL | free 后置 NULL |
| 共同点 | 解引用都会导致未定义行为 | |

## 五、常见错误与排错

### 5.1 返回局部变量地址

❌ 错误示例：
```c
int* bad_function() {
    int result = 42;
    return &result;   // 返回局部变量的地址
}
```

编译警告（gcc）：
```text
warning: function returns address of local variable
```

✅ 正确示例：
```c
int* good_function(int *result) {
    *result = 42;
    return result;    // 返回传入的指针，安全
}
```

### 5.2 忘记解引用

❌ 错误示例：
```c
void swap(int *a, int *b) {
    int temp = a;   // 错误：把地址赋给了 temp，不是值
    a = b;          // 错误：改了指针的指向，没改指向的值
    b = temp;
}
```

✅ 正确示例：
```c
void swap(int *a, int *b) {
    int temp = *a;   // 取 a 指向的值
    *a = *b;         // 取 b 指向的值，赋给 a 指向的位置
    *b = temp;
}
```

### 5.3 忘记传地址

❌ 错误示例：
```c
int x = 5, y = 10;
swap(x, y);       // 传的是值，不是地址
```

✅ 正确示例：
```c
swap(&x, &y);     // 传地址
```

## 我做得对不对？

用以下程序验证：

```c
#include <stdio.h>

// 正确的 swap
void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

// 通过指针返回多个值
void divide(int a, int b, int *quotient, int *remainder) {
    *quotient = a / b;
    *remainder = a % b;
}

// 返回数组中的最大值地址
int* find_max(int *arr, int size) {
    int *max = &arr[0];
    for (int i = 1; i < size; i++) {
        if (arr[i] > *max) {
            max = &arr[i];
        }
    }
    return max;
}

int main() {
    // 测试1：swap
    int x = 5, y = 10;
    swap(&x, &y);
    printf("测试1：x = %d (应为 10), y = %d (应为 5)\n", x, y);

    // 测试2：多返回值
    int q, r;
    divide(17, 5, &q, &r);
    printf("测试2：商 = %d (应为 3), 余数 = %d (应为 2)\n", q, r);

    // 测试3：返回指针
    int nums[] = {3, 7, 1, 9, 4};
    int *max_ptr = find_max(nums, 5);
    printf("测试3：最大值 = %d (应为 9), 索引 = %ld (应为 3)\n",
           *max_ptr, max_ptr - nums);

    return 0;
}
```

编译运行：
```bash
$ gcc -Wall -o test test.c
$ ./test
测试1：x = 10 (应为 10), y = 5 (应为 5)
测试2：商 = 3 (应为 3), 余数 = 2 (应为 2)
测试3：最大值 = 9 (应为 9), 索引 = 3 (应为 3)
```

## 不对怎么办？

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| `swap` 函数没效果 | 传了值而不是地址 | 确保调用时用 `&` 取地址 |
| 函数返回后指针指向的值变了 | 返回了局部变量的地址 | 不要返回局部变量的地址 |
| 编译警告 `returns address of local variable` | 同上 | 改用静态变量或传入指针参数 |
| 段错误 | 解引用了野指针或悬空指针 | 初始化指针，free 后置 NULL |
| 函数本应修改外部变量但没修改 | 函数内改了指针的指向而非解引用 | 检查是否写了 `*p = ...` 而不是 `p = ...` |

## 本章小结

| 知识点 | 一句话总结 |
|--------|-----------|
| 指针作为函数参数 | 传地址让函数能修改外部变量 |
| 返回指针 | 可以返回指针，但绝不能返回局部变量的地址 |
| 野指针 | 未初始化的指针，指向随机地址 |
| 悬空指针 | 指向已释放内存的指针 |
| `const` 指针 | 承诺不通过该指针修改数据，提高代码安全性 |
| 多返回值 | 通过指针参数让函数"返回"多个值 |
| 安全返回指针 | 返回传入的指针、静态变量地址、或动态分配的内存 |
