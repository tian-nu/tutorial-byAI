# 30-Agent模式

> **本篇属于**：第三篇·Cursor从入门到精通 | **预计阅读**：25分钟 | **难度**：★★★★☆
>
> **前置条件**：Git已安装，会基本的Git分支操作，项目已初始化Git仓库。
>
> 📊 可视化演示见 [ai_coding_agent_visual.html](../.project/docs/tutorials/ai_coding_agent_visual.html)

如果说Tab补全是"帮你写半句话"，Chat是"你和AI一对一对话"，Composer是"帮你生成多个文件"——那么**Agent模式**就是**AI自动驾驶**。

你只需要说清楚你要做什么，AI会：
1. 自己读你的项目文件
2. 自己理解项目结构
3. 自己找到需要改的地方
4. 自己修改代码
5. 自己运行终端命令
6. 自己看运行结果
7. 如果不对，自己修复，循环往复

直到它觉得任务完成了。

---

## 30.1 Agent模式是什么？

> **📖 术语：Agent模式（智能体）**——一种AI自动驾驶模式。不需要你一步步指挥，AI自己探索项目、改代码、测试结果、循环修复，直到完成你要求的需求。

**一个比喻**：
- **Tab补全** = 打车时，司机开一段你指挥一段
- **Chat模式** = 打车时，你告诉司机去哪里，司机开（还是需要你指路）
- **Agent模式** = 你上车，告诉司机"我要去XX"，然后闭眼睡觉，到了司机叫你

---

## 30.2 ⚠️ 重要安全提醒：先开Git分支再用Agent

**这是红线，必须遵守**：

> Agent可以帮你自动改代码，也能**一次性改乱你的整个项目**。

所以，**永远记住**：
1. ✅ 用Agent前，先提交所有已有的改动
2. ✅ 切到一个新分支：`git checkout -b feature/xxx`
3. ✅ 如果AI改乱了，直接切回主分支删掉这个分支就行
4. ❌ 永远不要在主分支直接用Agent改一堆东西

## 30.3 完整流程演示：给Todo项目加一个删除功能

我们用一个实际例子走一遍流程。假设你已经有一个Todo应用：
- `index.html`：页面结构
- `style.css`：样式
- `app.js`：JS逻辑

目前Todo应用只有**添加任务**功能，没有**删除任务**功能。我们让Agent帮我们加上。

### 步骤**1.** 打开Agent模式

- 按 `Ctrl + I` 打开Composer
- 在Composer上方选择 **"Agent"** 模式（默认是Normal，点一下切换成Agent）

### 步骤**2.** 描述你的需求

在输入框里输入：

```
我这个项目现在有一个Todo应用，目前可以添加任务。请帮我加一个删除任务功能：
1. 每个任务后面加一个删除按钮
2. 点击删除按钮，删除对应的任务
3. 在列表底部显示当前还有几个未完成的任务

请你自己看一下现有的三个文件：index.html, style.css, app.js，找到需要改的地方，修改后给我看。
```

### 步骤**3.** Agent开始自动工作

Agent会：

1. **自动找文件** → 找到 `index.html`、`style.css`、`app.js`
2. **自动读代码** → 理解每个文件里写了什么
3. **分析结构** → 理解任务列表的DOM结构、JS数据结构
4. **制定修改计划** → 哪里要改HTML，哪里要改CSS，哪里要改JS
5. **修改代码** → 逐个文件修改
6. **权限确认** → AI修改每个文件前，会问你允不允许
7. **如果你都同意** → AI改完后，还会建议你运行一下测试
8. **如果测试出问题** → AI会自己看错误，自己修复，循环

## 30.4 权限控制：AI不会随便改东西

Agent有严格的权限控制：

| 操作 | 是否需要你同意 | 说明 |
|------|----------------|------|
| 读取文件 | 不需要 | AI只读，不写 |
| 修改文件 | 需要 | 每次改文件，都会弹框问你"允许修改这个文件吗？" |
| 执行终端命令 | 需要 | AI要运行终端命令（比如测试、安装依赖），必须问你 |
| 删除文件 | 需要 | 要删文件也必须问你 |

你永远掌握着最终控制权——**AI只建议，你说了算**。

## 30.5 什么时候用Agent模式？

Agent模式很强，但**不是所有场景都适合**。

| 适合用Agent的场景 | 为什么 |
|-------------------|--------|
| **给大型项目加功能** | 涉及多个文件，Agent自己找文件自己改 |
| **重构代码** | 把旧代码重构成新结构，需要理解整个模块 |
| **修复Bug** | 你只说清楚Bug现象，Agent自己找原因自己修 |
| **代码审查** | 让Agent帮你扫一遍代码，找潜在问题 |

| 不适合用Agent的场景 | 为什么 |
|-------------------|--------|
| **简单改一个函数** | 大材小用，Chat更快 |
| **关键业务代码** | AI可能理解错需求，改了不该改的地方 |
| **没有Git备份** | 一旦改乱了，无法恢复，风险大 |

**一句话经验**：大任务让Agent干，小任务自己动手。

## 30.6 如果出错了怎么办

### 🔴 问题1：Agent改了很多不该改的文件

**原因**：Agent可能理解错了你的需求范围。

**解决**：
- 立即停止，用Git回滚到之前的分支
- 重新描述需求，更精确地说清楚"只改哪些文件，不要碰哪些文件"
- 比如："只修改app.js里的逻辑，不要碰index.html和style.css"

### 🔴 问题2：Agent一直循环修改，但改不对

**原因**：需求描述太模糊，或者Agent理解错了。

**解决**：
- 停止Agent，切换到Chat模式
- 手动把问题说清楚，一步一步改
- 不要让Agent猜，直接告诉它改哪行、改成什么样

### 🔴 问题3：Agent执行终端命令把项目搞坏了

**原因**：Agent可能运行了错误的命令（比如 `rm -rf` 删了整个目录）。

**解决**：
- 切回主分支，从主分支重新开分支
- 下次遇到AI要执行危险命令（比如删除文件、强制覆盖），直接拒绝

### 🔴 问题4：Agent说"我无法完成这个任务"

**说明**：任务太复杂或者太模糊，Agent能力不够。

**解决**：把大任务拆成小任务，一个一个让Agent做。比如：
1. 先让Agent只改HTML加删除按钮
2. 改完了再让Agent改JS实现删除逻辑
3. 改完了再让Agent改CSS调样式

## 自测

1. 你想让Agent帮你在现有项目上加一个功能，第一步应该做什么？
2. Agent修改文件和执行终端命令之前，会做什么？
3. Agent改完代码后，发现改乱了整个项目，你怎么恢复？

> **答案**：
> 1. 第一步：提交所有现有改动，然后切到一个新Git分支 `git checkout -b feature/xxx`。这是安全红线，必须遵守。
> 2. Agent每次修改文件和执行终端命令之前，都会问你"允许吗？"，你同意了才会做。你永远掌握控制权。
> 3. 切回主分支，删掉这个功能分支，从干净状态重新开始。这就是为什么我们要求提前切分支。
