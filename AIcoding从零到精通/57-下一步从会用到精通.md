## 推荐进阶学习路径

根据你的兴趣，这里有四条路可以走：

### 路径一：想学后端开发

你想做"真正的软件工程师"——写API、设计数据库、部署服务。

**推荐：** 本仓库的《Go从零到后端教程》或《Java从零到Spring Boot后端教程》

```
Go 路径：Go语言 → Gin框架 → MySQL → Redis → 项目实战 → 部署
Java 路径：Java基础 → Spring Boot → MyBatis → MySQL → 项目实战 → 部署
```

**配AI使用**：用你在这本书里学到的Prompt Engineering技巧，让AI帮你写Go或Java代码。你会发现——换了语言，但和AI对话的方法是一样的。

### 路径二：想做自己的项目

你有一个想法——"我想做一个记账App"、"我想做一个自动发微博的机器人"、"我想做一个帮我整理书签的工具"。

**推荐：** 直接动手。

```
你不需要再学更多的东西。你已经有足够的能力了。
1. 用你学的Prompt Engineering技巧，描述清楚你的项目
2. 用Cursor或Trae的Agent模式，让AI帮你搭项目骨架
3. 遇到问题？翻回第52-53章，避开陷阱
4. 担心安全？翻回第54章，检查密钥
5. 代码质量不满意？翻回第55章，逐条检查
```

**最好的学习方式，就是做你自己想用的东西。** 教程和课程给你的是"别人的项目"——你亲自做出来的，才是"你的项目"。

### 路径三：想深入AI工具

你不想做传统后端，你想成为"AI编程工具专家"。

**推荐：**
- 深入学习Cursor的 `.cursorrules` 和Rules系统（回看第31章）
- 研究Agent模式的高级用法（多Agent协作、自定义Agent）
- 关注AI编程工具的最新动态（见下方"社区与资源"）

### 路径四：想学Git版本控制

你在第53章看到了Git的重要性，但只学了最基础的 `add` / `commit` / `checkout`。

**推荐：** 本仓库的《Git到GitHub教程》

```
Git基础 → 分支管理 → 合并与冲突解决 → GitHub协作 → Pull Request
```

**Git是每个程序员都必须掌握的工具，无论你写不写代码。**

## 最后的建议

这本书的最后一句话，我想留给你：

> **每天用AI写一点代码，三个月后你会感谢今天的自己。**

这不是鸡汤。这是事实。

三个月是什么概念？如果你每天用AI写30分钟代码：

- 第1周：你还在熟悉工具，Prompt写得磕磕绊绊
- 第2周：你开始能一次写出不错的Prompt，AI的代码质量明显提升
- 第1个月：你已经能独立完成小项目了，会有一种"原来我也可以"的成就感
- 第2个月：你开始注意到代码质量、安全性、性能——这些"高级话题"
- 第3个月：你回头看第1周写的代码，会觉得"我当时怎么写成这样"——这是进步最真实的标志

**三个月后，你会从一个"用AI帮忙写代码的人"，变成一个"用AI写代码的人"。** 这个转变不是学出来的，是练出来的。
