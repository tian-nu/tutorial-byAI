## 38.1 读取文件

### 方式一：os.ReadFile（最简单）

Go 1.16引入的快捷方式——一次性读取整个文件：

```go
data, err := os.ReadFile("config.json")
if err != nil {
    fmt.Println("读取失败:", err)
    return
}
fmt.Println(string(data))
```

**适合**：小文件（配置、模板等）。文件内容全部加载到内存。

**不适合**：大文件（几百MB或更大），会把内存撑爆。

### 方式二：os.Open + 逐块读取

逐块读取，不会一次性加载整个文件：

```go
file, err := os.Open("large_file.txt")
if err != nil {
    fmt.Println("打开失败:", err)
    return
}
defer file.Close()

buffer := make([]byte, 1024)
for {
    n, err := file.Read(buffer)
    if err == io.EOF {
        break
    }
    if err != nil {
        fmt.Println("读取失败:", err)
        return
    }
    fmt.Print(string(buffer[:n]))
}
```

`file.Read(buffer)` 每次最多读 `len(buffer)` 字节到buffer里。返回值 `n` 是实际读取的字节数。读到文件末尾时返回 `io.EOF` 错误。

`defer file.Close()` 保证文件被关闭——及时释放操作系统资源。

### 方式三：bufio.Scanner（按行读取）

处理文本文件最常见的方式：

```go
file, err := os.Open("data.txt")
if err != nil {
    fmt.Println("打开失败:", err)
    return
}
defer file.Close()

scanner := bufio.NewScanner(file)
lineNumber := 1
for scanner.Scan() {
    line := scanner.Text()
    fmt.Printf("第%d行: %s\n", lineNumber, line)
    lineNumber++
}

if err := scanner.Err(); err != nil {
    fmt.Println("扫描失败:", err)
}
```

**比喻**：`os.ReadFile` 是一次性把整本书复印下来。`os.Open + file.Read` 是一次读一页。`bufio.Scanner` 是一次读一行——最适合处理日志、CSV等按行组织的文件。

### 方式四：io.ReadAll

把整个 `io.Reader` 读到内存：

```go
file, _ := os.Open("data.txt")
defer file.Close()

data, err := io.ReadAll(file)
if err != nil {
    fmt.Println("读取失败:", err)
    return
}
fmt.Println(string(data))
```

本质和 `os.ReadFile` 一样，但用已有的 `file` 对象。

## 38.2 写入文件

### 方式一：os.WriteFile（最简单）

```go
content := []byte("Hello, 世界")
err := os.WriteFile("output.txt", content, 0644)
if err != nil {
    fmt.Println("写入失败:", err)
    return
}
```

`0644` 是Unix文件权限：所有者可读写（6）、同组可读（4）、其他人可读（4）。

**注意**：`os.WriteFile` 会**覆盖**已有文件。如果文件不存在则创建。

### 方式二：os.Create + file.Write

```go
file, err := os.Create("output.txt")
if err != nil {
    fmt.Println("创建失败:", err)
    return
}
defer file.Close()

n, err := file.Write([]byte("第一部分内容\n"))
if err != nil {
    fmt.Println("写入失败:", err)
    return
}
fmt.Println("写入了", n, "字节")

n, err = file.WriteString("第二部分内容\n")
if err != nil {
    fmt.Println("写入失败:", err)
    return
}
```

`file.Write` 接收 `[]byte`，`file.WriteString` 接收 `string`——效果一样。

**注意**：`os.Create` 会**清空**已有文件从头写，相当于"新建 + 打开"。

### 方式三：os.OpenFile（灵活控制）

```go
file, err := os.OpenFile("output.txt", os.O_APPEND|os.O_CREATE|os.O_WRONLY, 0644)
if err != nil {
    fmt.Println("打开失败:", err)
    return
}
defer file.Close()

file.WriteString("追加的内容\n")
```

`os.OpenFile` 的三个参数：
1. 文件路径
2. 打开标志（组合使用 `|`）
3. 文件权限

常用的打开标志：

| 标志 | 含义 |
|------|------|
| `os.O_RDONLY` | 只读 |
| `os.O_WRONLY` | 只写 |
| `os.O_RDWR` | 读写 |
| `os.O_CREATE` | 不存在则创建 |
| `os.O_APPEND` | 追加到末尾 |
| `os.O_TRUNC` | 清空已有内容 |
| `os.O_EXCL` | 和 `O_CREATE` 一起用，文件已存在则报错 |

### 方式四：bufio.Writer（缓冲写）

```go
file, _ := os.Create("output.txt")
defer file.Close()

writer := bufio.NewWriter(file)
writer.WriteString("第一行\n")
writer.WriteString("第二行\n")
writer.Flush()
```

数据先写入内存缓冲区，调用 `Flush()` 才真正写入磁盘。**必须调 `Flush()`**，否则数据可能丢失。

缓冲写的优势：减少了系统调用次数。你写1000次小数据，bufio帮你合并成几次大写入——对性能敏感的场景很有用。

## 38.3 文件信息

### os.Stat：获取文件信息

```go
info, err := os.Stat("output.txt")
if err != nil {
    if os.IsNotExist(err) {
        fmt.Println("文件不存在")
    } else {
        fmt.Println("其他错误:", err)
    }
    return
}

fmt.Println("文件名:", info.Name())
fmt.Println("大小:", info.Size(), "字节")
fmt.Println("权限:", info.Mode())
fmt.Println("修改时间:", info.ModTime())
fmt.Println("是目录吗?", info.IsDir())
```

### os.IsNotExist：判断文件不存在

```go
_, err := os.Stat("notfound.txt")
if os.IsNotExist(err) {
    fmt.Println("文件确实不存在")
}
```

**注意**：不要用 `err != nil` 判断文件不存在——可能是其他错误（权限不足、磁盘错误等）。**用 `os.IsNotExist(err)` 明确判断。**

## 38.4 路径操作：path/filepath

手动拼接路径字符串容易出错（Windows用 `\`，Linux用 `/`）。用 `filepath` 包来处理：

```go
import "path/filepath"

fullPath := filepath.Join("data", "users", "profile.json")
fmt.Println(fullPath)
```

`filepath.Join` 自动处理跨平台路径分隔符。

### 常用函数

```go
path := "/home/user/documents/report.pdf"

dir := filepath.Dir(path)
file := filepath.Base(path)
ext := filepath.Ext(path)
name := filepath.Base(path[:len(path)-len(ext)])

fmt.Println("目录:", dir)
fmt.Println("文件名:", file)
fmt.Println("扩展名:", ext)
fmt.Println("不带扩展名:", name)
```

输出：
```
目录: /home/user/documents
文件名: report.pdf
扩展名: .pdf
不带扩展名: report
```

### 其他实用函数

```go
filepath.Abs(".")                      // 绝对路径
filepath.Rel("/a/b", "/a/b/c/d")       // 相对路径 → "c/d"
filepath.Glob("*.txt")                 // 通配符匹配文件列表
filepath.Walk(".", func(path string, info os.FileInfo, err error) error {
    fmt.Println(path)
    return nil
})                                      // 递归遍历目录
```

## 🤔 想多一点：为什么不用相对路径？

Go程序运行时，相对路径是相对于**当前工作目录**（working directory）的，而不是相对于源代码文件的。

```go
data, _ := os.ReadFile("config.json")
```

如果你在项目根目录运行 `go run main.go`，它会找根目录下的 `config.json`。但如果你在另一个目录运行时，它会在那个目录找——可能找不到。

**解决方案**：
- 使用绝对路径（通过命令行参数或环境变量传入）
- 在 `init` 中切换到源码所在目录
- 使用 `embed`（Go 1.16+，把文件打包进二进制）
