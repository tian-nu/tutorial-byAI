## 幻觉是什么：AI会"编造"不存在的东西

想象你问朋友："重庆的市花是什么？"

朋友理直气壮地回答："芙蓉花！重庆的市花是芙蓉花，1987年确定的，寓意着……"

听起来很有道理，对吧？问题是——**重庆的市花是山茶花，不是芙蓉花**。你朋友一本正经地编了一个细节丰富但完全错误的答案。

这就是**幻觉（📖 术语：Hallucination）**——AI会编造不存在的事实、不存在的API、不存在的函数参数，而且说得像真的一样。

> **幻觉** = AI用非常自信的语气，输出了一个完全错误的信息。它不是在"骗你"——它自己也不知道自己在胡说。

在AI编程中，幻觉特别危险，因为：

- AI会编造**不存在的API**：`requests.get_json_from_url()` ← 这个函数根本不存在
- AI会虚构**不存在的函数参数**：`print("hello", encoding="utf-8", mode="strict")` ← `print()` 根本没有 `mode` 参数
- AI会说**不存在的功能**："Python标准库有一个 `data_analyzer` 模块，可以直接分析CSV" ← 没有这个模块

## 根因：AI为什么会胡说八道？

回到第02章的核心结论：**LLM是一个"猜下一个字"的概率游戏**。

AI不会验证真假，它只会根据训练数据，猜"下一个最可能的字是什么"。所以：

1. **训练数据不完整**：AI读过的数据里，关于"重庆的市花"的信息可能很少、不准确，或者根本没有。但AI还是会猜一个答案——因为它的任务就是"猜"，不是"说不知道"。

2. **概率推断的边界**：当你问AI一个它没见过的问题时，AI不会说"我不知道"——它会从已有的知识里"拼凑"一个看起来最合理的答案。就像你考试遇到不会的题，瞎蒙一个总比空着强。

3. **模式匹配的陷阱**：AI见过太多"XX的市花是YY，ZZZZ年确定的"这种句式。所以当你问"重庆的市花是什么"，它自动套用了这个句式，然后随便填了一个"听起来对"的花名。

**AI的幻觉不是Bug，是概率预测的必然副产品。** 就像你问一个只会猜下一个字的人"圆周率第100位是什么"，他只能猜——因为他没有"计算"的能力，只有"猜"的能力。

## 实战识别：3个真实幻觉案例

下面是AI编程中三种最常见的幻觉类型。每种都给你看AI的"幻觉输出"和"真相"。

### 案例一：编造不存在的API

**❌ AI的幻觉输出：**

```python
# 以下代码由AI生成 —— 但它是错的！
import requests

def get_user_data(user_id):
    # requests.get_json()？这个函数根本不存在！
    response = requests.get("https://api.example.com/users/" + user_id)
    data = response.get_json()  # ← 幻觉！requests.Response 没有 get_json() 方法
    return data
```

**✅ 真相：**

```python
# requests 的正确用法是 .json() 方法，不是 .get_json()
import requests

def get_user_data(user_id):
    response = requests.get("https://api.example.com/users/" + user_id)
    data = response.json()  # ← 正确！调用 .json() 方法
    return data
```

**为什么会这样？** AI见过太多类似 `get_json()` 的命名模式（比如 `get_data()`、`get_text()`），它"猜" `requests` 也应该有 `get_json()`。但实际上，`requests` 用的是 `.json()`。

---

### 案例二：虚构函数参数

**❌ AI的幻觉输出：**

```python
# 以下代码由AI生成 —— 但它是错的！
print("Hello World", encoding="utf-8", flush=True, mode="strict")
# ← 幻觉！print() 没有 encoding 参数，也没有 mode 参数
```

**✅ 真相：**

```python
# print() 只有这些参数：sep, end, file, flush
print("Hello World", flush=True)  # ← 正确
# 如果需要指定编码，应该用 sys.stdout 或 open()
import sys
sys.stdout.reconfigure(encoding='utf-8')
print("Hello World")
```

**为什么会这样？** AI见过太多函数有 `encoding` 参数（比如 `open(file, encoding="utf-8")`），它"猜" `print()` 也应该有。但实际上 `print()` 的编码是通过 `sys.stdout` 控制的。

---

### 案例三：说不存在的功能

**❌ AI的幻觉输出：**

```
你：Python怎么读取Excel文件？

AI：Python标准库自带一个 excel_reader 模块，你只需要：
    from excel_reader import ExcelFile
    data = ExcelFile("data.xlsx").read_all()
    
    ← 幻觉！Python标准库根本没有 excel_reader 模块！
```

**✅ 真相：**

```python
# Python读取Excel需要用第三方库，比如 openpyxl 或 pandas
# 先安装：pip install openpyxl
from openpyxl import load_workbook

wb = load_workbook("data.xlsx")
ws = wb.active
for row in ws.iter_rows(values_only=True):
    print(row)
```

**为什么会这样？** AI知道"Python能读Excel"，但它"猜"了一个不存在的标准库模块名。实际上，这个功能需要安装第三方库。

## 应对策略：四招对付幻觉

幻觉不可能完全消除，但你可以用下面四招把它的影响降到最低。

### 策略一：让AI自查

AI虽然会胡说，但如果你让它**再检查一遍**，它有概率发现自己错了。

```
你：帮我写一个Python函数，用requests库发POST请求，带JSON body。
AI：（生成代码，可能包含幻觉）

你：请你检查一下上面的代码，确认所有调用的方法和参数都是真实存在的。
    如果有不确定的地方，请标注出来。
AI：（重新检查，有概率发现并修正幻觉）
```

**这个方法不是100%有效，但能过滤掉一部分幻觉。**

---

### 策略二：限定来源

在Prompt里明确告诉AI"只使用XX库的标准方法"。

```
❌ 坏Prompt："帮我写一个读取CSV文件的Python函数"
   → AI可能编造不存在的库

✅ 好Prompt："用Python标准库的csv模块，帮我写一个读取CSV文件的函数。
    只使用csv模块的官方文档里的方法，不要用任何第三方库或不存在的函数。"
   → AI的幻觉概率大幅降低
```

---

### 策略三：分步验证

不要让AI一次性写完所有代码。拆成小步骤，每一步跑一下验证。

```
第1步：让AI写函数签名 → 你检查参数对不对
第2步：让AI写函数体 → 你运行，看有没有报错
第3步：让AI写测试用例 → 你跑测试，看结果对不对
```

**每一步都验证，幻觉在第一步就会被发现，不会扩散到整个项目。**

---

### 策略四：跑起来看结果

**这是最简单也最有效的方法**：AI写的代码，跑一下就知道对不对。

```
AI写了代码 → 你运行 → 报错 "AttributeError: 'Response' object has no attribute 'get_json'"
→ 把错误信息复制给AI → AI修正 → 再运行 → 成功
```

> **90%的幻觉会在你第一次运行代码的时候暴露出来**。所以，**永远不要信任AI写的代码，跑一遍再说**。

## 自测

1. 用你自己的话解释什么是"幻觉"，为什么AI会产生幻觉？
2. 你让AI写了一段代码，运行时报错 `AttributeError: 'list' object has no attribute 'append_all'`。这最可能是什么问题？你应该怎么处理？
3. 四个应对策略中，哪一个是"最简单但最有效"的？为什么？

> **答案**：
> 1. 幻觉是AI编造不存在的事实/API/参数的行为。原因是AI的本质是"猜下一个字"的概率游戏，它不会验证真假，只会从训练数据中拼凑出最可能的答案。
> 2. 这很可能是AI的幻觉——`list` 没有 `append_all()` 方法，正确的方法可能是 `extend()`。你应该把错误信息复制给AI，让它修正代码。
> 3. "跑起来看结果"是最简单但最有效的。因为90%的幻觉会在运行时报错直接暴露出来，不需要任何技巧，只需要运行代码然后把错误信息给AI。
