## 为什么AI编程还是需要懂一点编程

你可能心里在嘀咕："不是说AI会帮我写代码吗？那我还学编程概念干嘛？"

好问题。我们用做饭的比方来回答。

你能指挥厨师做菜，但至少得知道**盐和糖的区别**。

你跟厨师说："这道菜放点盐。"厨师照做了。但如果厨师搞错了，放了糖呢？你尝一口，应该能发现"不对，这味道是甜的，不是咸的"——**你得有基本的判断力**。

AI也是同理：

- AI帮你写代码，但你得能**看懂**它写的是"盐"还是"糖"
- AI帮你改代码，但你得能**验证**改完之后功能对不对
- AI帮你排错，但你得能**读懂**报错信息，知道问题大概出在哪

> **不是让你手写代码，而是让你能看懂AI写的代码。** 就像你不是厨师，但你得知道菜里放了什么。

## Python vs JavaScript：为什么本书选Python

市面上编程语言有几百种，但最流行的就那几个。本书选**Python**，原因很简单：

| 语言 | 优点 | 缺点 | 适合谁 |
|------|------|------|--------|
| **Python** 🏆 | 语法像英文，读起来像看文章；AI最擅长；库最多 | 运行速度不如C/Java | **新手首选，本书使用** |
| **JavaScript** | 网页开发必备；浏览器里就能跑 | 语法有些反直觉；坑多 | 想做网页前端的 |
| **Java** | 企业级开发标配；稳定 | 啰嗦，一个Hello World要写5行 | 想做后端开发的 |

看一个对比你就明白了。同一个功能——"打印Hello World"：

**Python（本书使用）：**
```python
print("Hello World")
```
就一行。你一眼就能看懂：`print` 是"打印"，括号里是要打印的内容。

**Java：**
```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello World");
    }
}
```
五行代码，里面全是看不懂的单词。对于新手来说，这叫"劝退式入门"。

**Python是AI最擅长写的语言**。ChatGPT、Claude、Cursor、Trae，它们生成Python代码的准确率是最高的。因为：
1. Python语法简洁，AI出错的概率低
2. Python是全球AI/ML（📖 术语：人工智能/机器学习）领域的标配语言，训练数据最多
3. Python社区庞大，遇到问题网上搜一下就有答案

## 我们的目标：不是手写，而是看懂、验证、改需求

读第二篇（第17-25章），你的目标**不是**学完就能手写代码。那是传统编程教程的目标，不是本书的。

**你的三个目标，按优先级排序：**

```
┌─────────────────────────────────────────┐
│  目标① 能看懂                            │
│  AI写的代码，你能大致说出它在干什么       │
│  ─────────────────────────────────────  │
│  目标② 能验证                            │
│  运行AI的代码，你能判断结果对不对          │
│  ─────────────────────────────────────  │
│  目标③ 能改需求                          │
│  发现AI写的不对，你能用中文说清楚怎么改    │
└─────────────────────────────────────────┘
```

举个例子：

AI给你写了这样一段代码：

```python
def calculate_average(scores):
    total = sum(scores)
    count = len(scores)
    return total / count
```

- **能看懂**：哦，这是在计算平均分——先把所有分数加起来，再除以人数
- **能验证**：我输入 `[80, 90, 100]`，应该得到 `90`，跑一下确认
- **能改需求**：如果AI忘了处理空列表（没有分数的情况），你能说："如果列表是空的，别让它报错，返回0"

**这就是你学完本篇能达到的水平。不是让你自己写，而是让你和AI配合得更顺畅。**

## 学习方法：看不懂就选中，问AI

本篇每一章都有AI生成的代码示例。你看代码的时候，大概率会遇到看不懂的地方。

**这时候不要慌，不要跳过，不要觉得自己笨。** 你的应对方法只有一个：

> 🔥 **看不懂的代码 → 用鼠标选中 → 按Ctrl+L（Cursor）→ 输入"解释这段代码做了什么，用大白话讲" → AI会给你解释**

这是AI编程时代最大的红利：**你有一个24小时在线的私人教师。**

比如你看这段代码：

```python
result = [x * 2 for x in numbers if x > 0]
```

看不懂？选中，问AI。AI会告诉你："这行代码的意思是：从 `numbers` 里挑出所有大于0的数，每个数乘以2，放进一个叫 `result` 的新列表里。"

传统学编程，这种语法你得翻书、查文档、上论坛问——可能要花半小时。现在，5秒钟搞定。

**请把"选中→问AI"这个动作，变成你的肌肉记忆。**

## 回答初学者最大的焦虑

我知道你在想什么。

> "道理我都懂，但我还是不会写代码。看完这几章，我就能写了吗？"

**坦白告诉你：不能。** 看完这9章，你还是不会手写代码。

但这**不是问题**——因为我们的目标本来就不是让你手写。

想想看：你会开车，但你会造发动机吗？你会用手机，但你会设计芯片吗？

**你用AI写代码，就像你开车——你不需要会造发动机，你只需要会握方向盘、踩油门、看路况。**

这9章教会你的就是"驾驶技能"：
- 第18章：认识仪表盘（变量和数据类型）
- 第19章：学会换挡（函数）
- 第20章：看懂路标（条件与循环）
- 第21章：学会整理后备箱（数据结构）
- 第22章：存取东西（文件操作）
- 第23章：学会用导航（API）
- 第24章：知道车开在哪（本地 vs 线上）
- 第25章：终极拷问——会看仪表盘就够了吗？

**你不会手写代码，但你能看懂AI写的代码——这才是AI时代的核心能力。**
