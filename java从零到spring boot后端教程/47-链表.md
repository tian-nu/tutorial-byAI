# 47. 链表

> 🔗 你和朋友玩"传话游戏"。一排人，每人只知道自己的话和下一人是谁。你想给第10个人传话，不能直接跳到第10个——必须从第1个开始，依次往下传。你想在中间加一个人也非常简单：让前面的人指向新人，新人指向后一个人就行。——这就是链表：**节点散落内存各处，靠指针串联，插入删除极快，但访问只能顺藤摸瓜**。

---

## 47.1 链表 vs 数组：本质差异

```
数组（连续内存）：
┌───┬───┬───┬───┐
│ A │ B │ C │ D │
└───┴───┴───┴───┘
100 104 108 112      ← 连续地址，可以跳

链表（分散内存）：
┌───┬───┐    ┌───┬───┐    ┌───┬───┐    ┌───┬───┐
│ A │ ●─┼───→│ B │ ●─┼───→│ C │ ●─┼───→│ D │ / │
└───┴───┘    └───┴───┘    └───┴───┘    └───┴───┘
 800          200          500          150          ← 地址不连续，必须逐个访问
```

| 维度 | 数组/ArrayList | 链表/LinkedList |
|---|---|---|
| 内存 | 连续一块 | 分散多块，每个节点多存一个指针 |
| 随机访问 `get(i)` | O(1) | O(n) — 必须从head走到第i个 |
| 头部插入 | O(n) — 全部后移 | O(1) |
| 中间插入 | O(n) — 后移一半 | O(n) — 先走到位置(走n/2步)，再O(1)改指针 |
| 尾部插入 | ArrayList均摊O(1) | LinkedList O(1)（如果维护tail指针） |
| 删除 | O(n) | O(n) — 同理，要先找到 |
| 缓存友好 | ✅ 连续内存，CPU缓存命中率高 | ❌ 跳跃访问，缓存命中率低 |

## 47.2 单向链表（完整Java实现）

```java
public class SinglyLinkedList<E> {

    private static class Node<E> {
        E data;
        Node<E> next;

        Node(E data) {
            this.data = data;
            this.next = null;
        }
    }

    private Node<E> head;
    private int size;

    public SinglyLinkedList() {
        head = null;
        size = 0;
    }

    public int size() {
        return size;
    }

    public boolean isEmpty() {
        return size == 0;
    }

    // 头部添加 O(1)
    public void addFirst(E e) {
        Node<E> newNode = new Node<>(e);
        newNode.next = head;   // 新节点指向旧head
        head = newNode;         // head改为新节点
        size++;
    }

    // 尾部添加 O(n) — 因为要走到末尾
    public void addLast(E e) {
        Node<E> newNode = new Node<>(e);
        if (head == null) {
            head = newNode;
        } else {
            Node<E> cur = head;
            while (cur.next != null) {
                cur = cur.next;
            }
            cur.next = newNode;
        }
        size++;
    }

    // 在指定索引插入 O(n)
    public void add(int index, E e) {
        if (index < 0 || index > size) {
            throw new IndexOutOfBoundsException("index: " + index + ", size: " + size);
        }
        if (index == 0) {
            addFirst(e);
            return;
        }
        Node<E> newNode = new Node<>(e);
        Node<E> prev = head;
        for (int i = 0; i < index - 1; i++) {
            prev = prev.next;           // 走到插入位置的前一个节点
        }
        newNode.next = prev.next;       // 新节点指向后节点
        prev.next = newNode;            // 前节点指向新节点
        size++;
    }

    // 获取索引元素 O(n)
    public E get(int index) {
        checkIndex(index);
        Node<E> cur = head;
        for (int i = 0; i < index; i++) {
            cur = cur.next;
        }
        return cur.data;
    }

    // 删除索引元素 O(n)
    public E remove(int index) {
        checkIndex(index);
        if (index == 0) {
            return removeFirst();
        }
        Node<E> prev = head;
        for (int i = 0; i < index - 1; i++) {
            prev = prev.next;
        }
        Node<E> toRemove = prev.next;
        prev.next = toRemove.next;      // 跳过要删除的节点
        size--;
        return toRemove.data;
    }

    public E removeFirst() {
        if (head == null) throw new NoSuchElementException();
        E data = head.data;
        head = head.next;
        size--;
        return data;
    }

    // 判断是否包含某元素 O(n)
    public boolean contains(E e) {
        Node<E> cur = head;
        while (cur != null) {
            if (Objects.equals(cur.data, e)) return true;
            cur = cur.next;
        }
        return false;
    }

    private void checkIndex(int index) {
        if (index < 0 || index >= size) {
            throw new IndexOutOfBoundsException("index: " + index + ", size: " + size);
        }
    }

    @Override
    public String toString() {
        StringBuilder sb = new StringBuilder("[");
        Node<E> cur = head;
        while (cur != null) {
            sb.append(cur.data);
            if (cur.next != null) sb.append(" -> ");
            cur = cur.next;
        }
        sb.append("]");
        return sb.toString();
    }
}
```

**使用示例**：

```java
SinglyLinkedList<Integer> list = new SinglyLinkedList<>();
list.addFirst(1);   // [1]
list.addFirst(2);   // [2 -> 1]
list.addLast(3);    // [2 -> 1 -> 3]
list.add(1, 99);    // [2 -> 99 -> 1 -> 3]
list.remove(2);     // [2 -> 99 -> 3]
System.out.println(list); // [2 -> 99 -> 3]
```

## 47.3 双向链表

Java的`LinkedList`实际上是一个**双向链表**，每个节点既指向前一个又指向后一个：

```
     ┌───────┐    ┌───────┐    ┌───────┐
null ←│ A │ ●─┼───→│ B │ ●─┼───→│ C │ ●→│ null
     └───┼───┘    └───┼───┘    └───┼───┘
         └────────────┘────────────┘
         前驱指针prev，指向前一个节点
```

好处：尾部删除从O(n)降到O(1)，因为可以从tail直接找到前一个节点。

**简化版双向链表实现**：

```java
public class DoublyLinkedList<E> {

    private static class Node<E> {
        E data;
        Node<E> prev;
        Node<E> next;

        Node(E data) {
            this.data = data;
        }
    }

    private Node<E> head;
    private Node<E> tail;
    private int size;

    public int size() { return size; }
    public boolean isEmpty() { return size == 0; }

    // 头部添加 O(1)
    public void addFirst(E e) {
        Node<E> newNode = new Node<>(e);
        if (head == null) {
            head = tail = newNode;
        } else {
            newNode.next = head;
            head.prev = newNode;
            head = newNode;
        }
        size++;
    }

    // 尾部添加 O(1) — 有tail指针！
    public void addLast(E e) {
        Node<E> newNode = new Node<>(e);
        if (tail == null) {
            head = tail = newNode;
        } else {
            tail.next = newNode;
            newNode.prev = tail;
            tail = newNode;
        }
        size++;
    }

    // 头部删除 O(1)
    public E removeFirst() {
        if (head == null) throw new NoSuchElementException();
        E data = head.data;
        head = head.next;
        if (head != null) {
            head.prev = null;
        } else {
            tail = null;  // 删光了
        }
        size--;
        return data;
    }

    // 尾部删除 O(1) — 有prev指针回溯
    public E removeLast() {
        if (tail == null) throw new NoSuchElementException();
        E data = tail.data;
        tail = tail.prev;
        if (tail != null) {
            tail.next = null;
        } else {
            head = null;
        }
        size--;
        return data;
    }
}
```

## 47.4 经典链表算法题（必会，面试高频）

### 反转链表

```java
// 迭代法：三个指针 prev, cur, next
public Node reverse(Node head) {
    Node prev = null;
    Node cur = head;
    while (cur != null) {
        Node next = cur.next;   // 1. 先记住下一个节点
        cur.next = prev;         // 2. 当前节点反转指向前一个
        prev = cur;              // 3. prev前进
        cur = next;              // 4. cur前进
    }
    return prev;                 // 新的head
}

// 过程演示：
// 初始: 1→2→3→null
// 一轮: null←1  2→3→null  (prev=1, cur=2)
// 二轮: null←1←2  3→null   (prev=2, cur=3)
// 三轮: null←1←2←3          (prev=3, cur=null)
// 返回: prev=3 → 3→2→1→null
```

### 找链表中间节点（快慢指针）

```java
public Node findMiddle(Node head) {
    Node slow = head;
    Node fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;        // 慢指针每次走1步
        fast = fast.next.next;   // 快指针每次走2步
    }
    return slow;  // 快指针到末尾时，慢指针恰好在中间
}
// 链表: 1→2→3→4→5
// slow: 1→2→3     fast: 1→3→5→null
// fast到null时，slow=3，正是中点
```

### 检测环（Floyd判圈算法）

```java
public boolean hasCycle(Node head) {
    Node slow = head;
    Node fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) {   // 快慢指针相遇 → 有环！
            return true;
        }
    }
    return false;
}
// 原理：如果有环，快指针最终会"套圈"慢指针
```

## 47.5 LinkedList 在实际开发中的典型使用

### LRU缓存（Least Recently Used — 最近最少使用）

```java
// LinkedHashMap可以轻松实现LRU
LinkedHashMap<String, String> cache = new LinkedHashMap<>(16, 0.75f, true) {
    @Override
    protected boolean removeEldestEntry(Map.Entry<String, String> eldest) {
        return size() > 100;   // 超过100条，删除最老的
    }
};
```

`LinkedHashMap`内部用双向链表维护插入/访问顺序，这就是LRU缓存的核心数据结构。

### 用LinkedList当队列使用

```java
// Java的LinkedList同时实现了List和Deque接口
LinkedList<String> queue = new LinkedList<>();
queue.offer("请求1");     // 入队（尾部）
queue.offer("请求2");
String req = queue.poll(); // 出队（头部），"请求1"
// poll() O(1) — 因为LinkedList有head和tail指针
```

## 47.6 ❌ 常见错误

| 错误 | 后果 | 正确做法 |
|---|---|---|
| 用`list.get(i)`遍历LinkedList | O(n²)！每次get都是O(n) | 用增强for或Iterator，内部维护了游标 |
| 反转链表时忘记保存next | 链表断开丢失 | `Node next = cur.next` 必须写在修改之前 |
| 判环时`fast.next.next`不判空 | NPE | 先判`fast != null && fast.next != null` |
| `head == null`时不更新tail | 双向链表状态不一致 | 同步更新head和tail |
| 认为LinkedList中间插入比ArrayList快 | 只快在改指针这一步，但找到插入位置还是要O(n) | 准确理解：插入本身O(1)，找到位置O(n) |

## 小结

| 概念 | 解释 |
|---|---|
| 链表物理结构 | 节点分散，靠next指针串联，不是连续内存 |
| 单向 vs 双向 | 双向多了prev指针，可从尾部反向遍历，尾部删除O(1) |
| get访问 | O(n) — 必须从head逐个走 |
| 头部插入/删除 | O(1) — 改head指针即可 |
| 中间插入 | O(n) — 找到位置是主要开销，改指针是O(1) |
| 快慢指针 | 找中点、判环的经典技巧 |
| 反转链表 | 三指针prev/cur/next迭代 |
