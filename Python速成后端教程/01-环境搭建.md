## 01.1 你需要的最少 Python 知识

在开始之前，确认一下你至少会这些：

```python
# 变量
name = "Alice"
age = 25

# 函数
def greet(name):
    return f"你好，{name}"

# if/else
if age >= 18:
    print("成年")
else:
    print("未成年")

# 列表
fruits = ["苹果", "香蕉", "橘子"]
fruits.append("西瓜")

# 字典

# class 基础

        self.name = name

    def bark(self):

```

## 01.2 安装 Python 3.11+

### 为什么要 Python 3.11+？

FastAPI 利用了一些 Python 3.10 和 3.11 的新特性。3.11 比 3.10 快 10-60%，免费的性能提升不要白不要。

### Windows 安装

1. 打开浏览器，访问 https://www.python.org/downloads/
2. 点击黄色的 "Download Python 3.11.x" 按钮（版本号可能更新，3.11 以上就行）
3. 运行下载的安装程序
4. **关键步骤**：勾选底部的 **"Add Python to PATH"**（把 Python 加入系统路径）

   > 这一步极其重要。如果不勾选，后面在终端里输入 `python` 会提示"找不到命令"。如果你已经装了但没勾选，重新运行安装程序，选 "Modify" 然后勾上。

5. 点击 "Install Now"
6. 等待安装完成

### macOS 安装

```bash
# 方法一：官网下载安装包（推荐）
# 访问 https://www.python.org/downloads/ 下载 macOS 安装包

# 方法二：用 Homebrew（如果你装了 Homebrew）
brew install python@3.11
```

### Linux 安装（Ubuntu/Debian）

```bash
sudo apt update
sudo apt install python3.11 python3.11-venv python3-pip -y
```

### 验证安装

打开终端（Windows 按 Win+R 输入 `cmd` 或 `powershell`；macOS 打开 Terminal；Linux 打开终端），输入：

```bash
python --version
```

你应该看到类似这样的输出：

```
Python 3.11.7
```

如果版本号是 3.11 或更高，安装成功。

再验证 pip（Python 的包管理器，后面用它装 FastAPI 等库）：

```bash
pip --version
```

应该看到类似：

```
pip 24.0 from C:\Users\你的用户名\AppData\Local\Programs\Python\Python311\Lib\site-packages\pip (python 3.11)
```

## 01.3 安装 VS Code

VS Code 是目前最流行的代码编辑器，免费、轻量、插件丰富。

1. 访问 https://code.visualstudio.com/
2. 下载对应系统的安装包
3. 安装（一路默认即可）
4. 打开 VS Code
5. 点击左侧的扩展图标（四个方块），搜索 **Python**
6. 安装微软官方的 **Python** 扩展（作者：Microsoft）

验证：在 VS Code 里按 `Ctrl+Shift+P`（macOS 按 `Cmd+Shift+P`），输入 `Python: Select Interpreter`，应该能看到你刚装的 Python 3.11。

## 01.5 什么是虚拟环境？为什么需要它？

### 比喻：给每个项目一个独立厨房

想象你开了一家餐厅，每个厨师做菜需要不同的调料：
- 川菜厨师要用豆瓣酱 v2.0
- 粤菜厨师要用豆瓣酱 v1.5
- 如果只有一个公共调料架，两个厨师会打架

**虚拟环境就是给每个项目一个独立的"调料架"**。你装在这个项目里的 Python 包，不会影响其他项目。

### 创建虚拟环境

在项目文件夹里执行：

```bash
python -m venv .venv
```

这条命令做了什么？
- `python -m venv`：调用 Python 自带的 venv 模块
- `.venv`：虚拟环境文件夹的名字（前面加 `.` 表示隐藏文件夹，约定俗成）

### 激活虚拟环境

```bash
# Windows（PowerShell）
.venv\Scripts\Activate.ps1

# Windows（CMD）
.venv\Scripts\activate.bat

# macOS / Linux
source .venv/bin/activate
```

激活成功后，终端提示符前面会出现 `(.venv)`：

```
(.venv) C:\Users\你的用户名\blog_backend>
```

这表示你现在在虚拟环境里。

### ❌ 常见错误：PowerShell 执行策略

Windows 用户可能遇到这个错误：

```
无法加载文件 .venv\Scripts\Activate.ps1，因为在此系统上禁止运行脚本。
```

**解决方法**：以管理员身份打开 PowerShell，执行：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

然后输入 `Y` 确认。关闭 PowerShell，重新打开普通终端，再激活虚拟环境。

### 验证虚拟环境

```bash
# 激活后，检查 Python 路径
where python    # Windows
which python    # macOS / Linux
```

应该显示虚拟环境里的 Python 路径（包含 `.venv`），而不是系统全局的 Python。

## 01.6 安装 FastAPI 和 Uvicorn

### 它们是什么？

- **FastAPI**：写接口的框架。你告诉它"访问 `/hello` 时返回一段文字"，其余的它帮你搞定。
- **Uvicorn**：把 FastAPI 写的程序跑起来的服务器。类比：FastAPI 是厨师，Uvicorn 是端菜的服务员。

### pip：Python 的"应用商店"

`pip` 是 Python 的包管理器。你告诉它你要装什么，它自动下载、安装、处理好依赖关系。

确保虚拟环境已激活（提示符前有 `(.venv)`），然后：

```bash
pip install fastapi uvicorn
```

输出类似：

```
Collecting fastapi
  Downloading fastapi-0.115.0-py3-none-any.whl ...
Collecting uvicorn
  ...
Successfully installed fastapi-0.115.0 uvicorn-0.30.0 ...
```

### 验证安装

```bash
python -c "import fastapi; print(fastapi.__version__)"
```

应该输出 FastAPI 的版本号，例如 `0.115.0`。

如果报错 `ModuleNotFoundError: No module named 'fastapi'`，说明虚拟环境没激活或安装失败。重新激活虚拟环境再试。

## 01.7 创建 requirements.txt

把项目依赖记录下来，方便以后部署或分享给其他人：

```bash
pip freeze > requirements.txt
```

这会生成一个 `requirements.txt` 文件，内容类似：

```
fastapi==0.115.0
uvicorn==0.30.0
...
```

以后在新环境装依赖，只需要：

```bash
pip install -r requirements.txt
```

## 01.9 常见错误排查

| 错误信息 | 原因 | 解决 |
|----------|------|------|
| `'python' 不是内部或外部命令` | 没装 Python 或没加入 PATH | 重新安装 Python，勾选 "Add Python to PATH" |
| `pip 不是内部或外部命令` | pip 没装或 PATH 问题 | `python -m pip --version` 试试 |
| `无法加载 Activate.ps1` | PowerShell 执行策略限制 | 管理员运行 `Set-ExecutionPolicy RemoteSigned` |
| `ModuleNotFoundError: No module named 'fastapi'` | 虚拟环境没激活 | 确认终端提示符前有 `(.venv)` |
| `pip install` 报 `SSL` 错误 | 网络问题或代理 | 换网络，或加 `--trusted-host pypi.org` |
