# 第19章 · 运算符

> "运算符就是程序里的'动作'——加、减、比较、判断。大部分运算符和数学课上学的差不多，但Go有几个独门暗器，比如位运算符在某些场景下特别好用。"

---

## 19.1 算术运算符：+-*/%

五个基本算术运算符：

```go
a := 10
b := 3

fmt.Println(a + b)  // 13  加法
fmt.Println(a - b)  // 7   减法
fmt.Println(a * b)  // 30  乘法
fmt.Println(a / b)  // 3   除法（整数除法，截断）
fmt.Println(a % b)  // 1   取余（模运算）
```

### 需要特别注意的点

**1. 整数除法截断**

```go
fmt.Println(10 / 3)    // 3（不是3.333...）
fmt.Println(10.0 / 3)  // 3.3333333333333335（浮点数除法）
```

两个整数相除，结果还是整数（小数部分直接丢掉，不是四舍五入）。

**2. 取余运算只能用于整数**

```go
fmt.Println(10 % 3)     // 1 ✅
fmt.Println(10.5 % 3)   // ❌ 编译错误！浮点数不能取余
```

**3. 不同类型不能直接运算**

```go
var a int = 10
var b float64 = 3.0
fmt.Println(a + b)  // ❌ 编译错误！int和float64不能直接相加
```

必须显式转换：

```go
fmt.Println(float64(a) + b)  // 13.0 ✅
```

**4. 自增自减（只有后置，没有前置）**

```go
i := 0
i++  // i = 1 ✅
i--  // i = 0 ✅

j := i++  // ❌ 编译错误！i++是一个语句，不是表达式，没有返回值
++i       // ❌ 编译错误！Go没有前置自增
```

Go从设计上就去掉了 `++i` 和 `--i`，也去掉了自增自减作为表达式的用法。为什么？因为前置和后置混用、自增作为表达式的一部分，在很多语言里是常见的Bug来源（比如C语言里 `arr[i++]` 的求值顺序是未定义的）。

## 19.2 比较运算符：== != < > <= >=

六个比较运算符，结果是 `bool` 类型：

```go
a := 10
b := 3

fmt.Println(a == b)  // false  等于
fmt.Println(a != b)  // true   不等于
fmt.Println(a < b)   // false  大于
fmt.Println(a > b)   // true   小于
fmt.Println(a <= b)  // false  大于等于
fmt.Println(a >= b)  // true   小于等于
```

比较运算符常用于 `if` 条件判断中（下一章会大量用到）。

比较的结果可以直接赋给 `bool` 变量：

```go
isGreater := a > b
fmt.Println(isGreater)  // true
```

## 19.3 逻辑运算符：&& || ! + 短路求值

### 三个逻辑运算符

```go
x := true
y := false

fmt.Println(x && y)  // false（AND 与：两者都为true才是true）
fmt.Println(x || y)  // true （OR 或：有一个为true就是true）
fmt.Println(!x)      // false（NOT 非：取反）
```

逻辑运算的"与或非"真值表：

| x | y | x && y (与) | x \|\| y (或) | !x (非) |
|---|---|-----------|------------|------|
| T | T | T | T | F |
| T | F | F | T | F |
| F | T | F | T | T |
| F | F | F | F | T |

### 短路求值（重要！）

Go的逻辑运算符有**短路求值**特性——如果根据左边的结果已经能确定整个表达式的值，右边就不会被执行。

```go
x := 10

if x > 5 && expensiveCheck() {
    fmt.Println("进去了")
}

if x < 5 && expensiveCheck() {
    fmt.Println("不会执行")
}
```

第一个 `if`：`x > 5` 是 `true`，所以必须执行 `expensiveCheck()` 才能确定最终结果。

第二个 `if`：`x < 5` 是 `false`，`false && anything` 永远是 `false`，所以 `expensiveCheck()` **根本不会被调用**。

`||` 同理：如果左边是 `true`，`true || anything` 永远是 `true`，右边不会被调用。

### 利用短路求值做简化

短路求值可以用来避免空指针访问：

```go
if user != nil && user.IsAdmin() {
    fmt.Println("管理员登录")
}
```

如果 `user` 是 `nil`，`user.IsAdmin()` 会导致程序崩溃。但因为有短路求值，`user != nil` 为 `false` 时，Go直接跳过右边的判断。这是Go里非常常见的模式。

## 19.4 位运算符：计算机底层的超能力

位运算是直接对二进制位进行操作的运算。日常写业务代码可能用不到，但在权限系统、标志位、底层优化中非常有用。

### 六种位运算符

| 运算符 | 名称 | 说明 |
|--------|------|------|
| `&` | 按位与 | 两个位都是1，结果才是1 |
| `\|` | 按位或 | 两个位有一个是1，结果就是1 |
| `^` | 按位异或 | 两个位不同，结果才是1 |
| `<<` | 左移 | 全部位向左移动n位，右边补0 |
| `>>` | 右移 | 全部位向右移动n位，左边补符号位 |
| `&^` | 位清空 | 将左边数中右边数为1的位清0 |

### 逐一演示

```go
a := 0b1100  // 12（二进制1100）
b := 0b1010  // 10（二进制1010）

fmt.Printf("%04b\n", a & b)   // 1000（8）  — 按位与
fmt.Printf("%04b\n", a | b)   // 1110（14） — 按位或
fmt.Printf("%04b\n", a ^ b)   // 0110（6）  — 按位异或
fmt.Printf("%04b\n", a &^ b)  // 0100（4）  — 位清空（b中为1的位在a中被清零）
fmt.Printf("%04b\n", a << 1)  // 11000（24）— 左移1位=乘以2
fmt.Printf("%04b\n", a >> 2)  // 0011（3）  — 右移2位=除以4
```

左移一位等于乘以2，右移一位等于除以2（整数除法）。位运算比乘除法快得多。

### 实战应用：权限管理

位运算是权限系统的最佳选择。假设一个文件有三种权限：读（R）、写（W）、执行（X）。

```go
const (
    Read    = 1 << iota  // 1 (001)
    Write                // 2 (010)
    Execute              // 4 (100)
)

myPermission := Read | Write  // 001 | 010 = 011 (3)，同时拥有读和写权限

hasRead := myPermission & Read == Read   // true
hasExecute := myPermission & Execute == Execute  // false

myPermission &^= Write  // 把Write位清零（011 &^ 010 = 001），只剩下Read
```

用三个位（3 bits），就能管理三种权限的任意组合，判断是否拥有某权限只需一次位与运算。这是最简的方案。

### 实战应用：标志位

```go
const (
    FlagA = 1 << iota  // 1
    FlagB               // 2
    FlagC               // 4
    FlagD               // 8
)

flags := FlagA | FlagC | FlagD  // 1 | 4 | 8 = 13

if flags & FlagC != 0 {
    fmt.Println("FlagC被设置了")
}
if flags & FlagB == 0 {
    fmt.Println("FlagB没有被设置")
}
```

一个int变量就能存储32个（int32）或64个（int64）开关，每个开关用某一位表示。

> 🤔 **想多一点**：你可能会想"平时写业务代码真的能用到位运算吗？"——比你想象的多。比如Redis里每个key的过期策略、HTTP/2的帧格式、protobuf的编码、数据库的bitmap索引，背后全是位运算。就算你不直接写位运算，理解它在干什么是后端工程师的基本素养。

## 19.5 赋值运算符：= += -= 等

赋值运算符就是把运算和赋值合并在一起：

```go
x := 10

x += 3   // x = x + 3 → 13
x -= 2   // x = x - 2 → 11
x *= 4   // x = x * 4 → 44
x /= 2   // x = x / 2 → 22
x %= 5   // x = x % 5 → 2

x &= 3   // x = x & 3
x |= 1   // x = x | 1
x ^= 2   // x = x ^ 2
x <<= 1  // x = x << 1
x >>= 1  // x = x >> 1
x &^= 1  // x = x &^ 1
```

本质上就是 `x = x 运算符 值` 的简写。

## 19.6 运算符优先级

当多个运算符同时出现时，Go按照下面的优先级顺序计算（从高到低）：

| 优先级 | 运算符 | 说明 |
|--------|--------|------|
| 1（最高） | `*` `/` `%` `<<` `>>` `&` `&^` | 乘除、位运算 |
| 2 | `+` `-` `\|` `^` | 加减、位或和异或 |
| 3 | `==` `!=` `<` `<=` `>` `>=` | 比较 |
| 4 | `&&` | 逻辑与 |
| 5（最低） | `\|\|` | 逻辑或 |

举例：

```go
result := 2 + 3*4       // 2 + 12 = 14（*优先级高于+）
result := (2 + 3) * 4   // 5 * 4 = 20（括号改变优先级）

result := 5 > 3 && 2 < 4  // 等价于 (5 > 3) && (2 < 4) = true
```

**一句话总结：记不住优先级就用括号。** 代码是写给人看的，多余的括号比"我猜不出优先级"好一万倍。

## 本章小结

| 知识点 | 核心要点 |
|--------|----------|
| 算术 `+ - * / %` | `/` 整数除法截断；`%` 只能用于整数；不同类型不能直接运算 |
| 自增自减 `i++` `i--` | 只有后置，不是表达式，不能 `j := i++` |
| 比较 `== != < > <= >=` | 结果都是 `bool` 类型 |
| 逻辑 `&& \|\| !` | 短路求值：左边能确定结果就不看右边 |
| 位运算 `& \| ^ << >> &^` | 直接操作二进制位；权限管理和标志位的利器 |
| 赋值 `= += -= *=` 等 | `x += 3` 等价于 `x = x + 3` |
| 优先级 | 记不住就用括号！多写括号比猜优先级好一万倍 |
