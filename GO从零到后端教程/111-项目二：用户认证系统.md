# 第111章 · 项目二：用户认证系统

> "Todo项目是'所有人共用同一个任务列表'——这在现实中根本不成立。每个正经应用都需要知道'你是谁'。本章我们要搭建一个用户认证系统：注册、登录、JWT令牌、密码哈希、头像上传。认证系统就像大楼的门禁——没有它，任何人都能在你的系统里乱逛。"

---

## 111.1 需求分析

### 这个项目要做什么

一个独立的用户认证服务，提供：

- **注册**：用户提交邮箱+密码，系统存储哈希后的密码
- **登录**：验证邮箱+密码，返回JWT令牌
- **JWT认证中间件**：保护需要登录才能访问的接口
- **个人信息**：已登录用户查看和修改自己的资料
- **密码重置**：通过邮箱发送重置链接（简化版：直接重置）
- **头像上传**：上传图片文件，存储到本地磁盘

### 功能列表

| 序号 | 功能 | HTTP方法 | 路径 | 认证 |
|------|------|---------|------|------|
| 1 | 注册 | POST | /api/register | 否 |
| 2 | 登录 | POST | /api/login | 否 |
| 3 | 获取个人信息 | GET | /api/user/profile | 是 |
| 4 | 更新个人信息 | PUT | /api/user/profile | 是 |
| 5 | 修改密码 | PUT | /api/user/password | 是 |
| 6 | 上传头像 | POST | /api/user/avatar | 是 |
| 7 | 密码重置 | POST | /api/password-reset | 否 |

### 不做什么

- 不做邮箱验证（发送真实邮件需要SMTP服务）
- 不做OAuth第三方登录（微信/GitHub登录）
- 不做角色权限管理（RBAC）
- 不做短信验证码登录

这些留到实际工作中根据需求再加。本章的目标是把认证的**核心骨架**搭起来。

## 111.2 核心技术选型

### 密码存储：bcrypt

为什么用bcrypt（一种故意慢速的密码哈希算法，防暴力破解，详见附录I）而不是SHA256？回顾第77章和第83章：

- SHA256是一般哈希——快，但正因为它快，攻击者可以用GPU每秒尝试数十亿个密码
- bcrypt是**慢哈希**——故意慢（可配置cost因子），让暴力破解变得昂贵

bcrypt的cost=10意味着每个密码哈希大约需要100毫秒。对单个用户来说感知不到，但对攻击者来说，每秒只能尝试10个密码——从"秒破"变成了"百年破"。

### 令牌机制：JWT

JWT（JSON Web Token）就像一张游乐园的门票：
- 你买票（登录），售票处给你一张盖章的门票（JWT）
- 之后每玩一个项目（访问接口），出示门票即可
- 门票有有效期（过期时间），过期需要重新买
- 门票上的印章无法伪造（HMAC签名）

JWT结构：`Header.Payload.Signature`，用Base64编码，用`.`分隔。

## 111.3 数据库设计

只有一张用户表，但字段比Todo那章多不少：

```sql
CREATE DATABASE IF NOT EXISTS auth_db
    DEFAULT CHARACTER SET utf8mb4
    DEFAULT COLLATE utf8mb4_unicode_ci;

USE auth_db;

CREATE TABLE users (
    id              BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    email           VARCHAR(255)    NOT NULL,
    password_hash   VARCHAR(255)    NOT NULL,
    nickname        VARCHAR(100)    NOT NULL DEFAULT '',
    avatar          VARCHAR(500)    NOT NULL DEFAULT '',
    bio             VARCHAR(500)    NOT NULL DEFAULT '',
    is_active       TINYINT         NOT NULL DEFAULT 1,
    created_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    UNIQUE INDEX idx_email (email),
    INDEX idx_is_active (is_active)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 字段说明

- `email`：唯一索引，保证不能重复注册。邮箱是用户身份的唯一标识
- `password_hash`：存储bcrypt哈希后的密码，**永远不存明文密码**
- `nickname`：用户昵称，默认空
- `avatar`：头像文件路径，不是存图片二进制数据。像图书馆存书的位置（书架号），而不是把整本书搬进来
- `bio`：个人简介
- `is_active`：是否启用，可以用来禁用账号而不是删除
- 注意：没有 `deleted_at` 软删除字段——用户数据比较敏感，不轻易删

## 111.4 API设计

### 统一响应格式

```json
{
    "code": 200,
    "message": "success",
    "data": { ... }
}
```

### 1. 注册 — POST /api/register

请求体：
```json
{
    "email": "user@example.com",
    "password": "MyPass123!",
    "nickname": "张三"
}
```

响应（201）：
```json
{
    "code": 201,
    "message": "注册成功",
    "data": {
        "id": 1,
        "email": "user@example.com",
        "nickname": "张三"
    }
}
```

验证规则：
- email格式必须合法（有@有域名）
- 密码至少8位
- 邮箱不能重复注册

请求体：
```json
{
    "email": "user@example.com",
    "password": "MyPass123!"
}
```

响应（200）：
```json
{
    "code": 200,
    "message": "登录成功",
    "data": {
        "token": "eyJhbGciOiJIUzI1NiIs...",
        "expires_at": "2025-01-02T10:00:00Z"
    }

```

请求头：`Authorization: Bearer <token>`

### model/user.go

```go
package model

import (
    "errors"
    "time"

    "golang.org/x/crypto/bcrypt"
    "gorm.io/gorm"
)

type User struct {
    ID           uint64    `gorm:"primaryKey" json:"id"`
    Email        string    `gorm:"size:255;not null;uniqueIndex" json:"email"`
    PasswordHash string    `gorm:"size:255;not null" json:"-"`
    Nickname     string    `gorm:"size:100" json:"nickname"`
    Avatar       string    `gorm:"size:500" json:"avatar"`
    Bio          string    `gorm:"size:500" json:"bio"`
    IsActive     int8      `gorm:"default:1" json:"-"`
    CreatedAt    time.Time `json:"created_at"`
    UpdatedAt    time.Time `json:"updated_at"`
}

func (User) TableName() string {
    return "users"
}

type RegisterReq struct {
    Email    string `json:"email" binding:"required,email"`
    Password string `json:"password" binding:"required,min=8"`
    Nickname string `json:"nickname"`
}

type LoginReq struct {
    Email    string `json:"email" binding:"required,email"`
    Password string `json:"password" binding:"required"`
}

type UpdateProfileReq struct {
    Nickname string `json:"nickname"`
    Bio      string `json:"bio"`
}

type ChangePasswordReq struct {
    OldPassword string `json:"old_password" binding:"required"`
    NewPassword string `json:"new_password" binding:"required,min=8"`
}

var (
    ErrEmailExists    = errors.New("邮箱已被注册")
    ErrUserNotFound   = errors.New("用户不存在")
    ErrWrongPassword  = errors.New("密码错误")
    ErrUserInactive   = errors.New("账号已被禁用")
)
```

`json:"-"` 在 `PasswordHash` 上非常重要——这意味着序列化JSON时不会输出密码哈希。即使代码出了Bug泄露了用户数据，至少不会把哈希后的密码也泄露出去。

### model/user.go — 数据库操作

```go
func Register(db *gorm.DB, req *RegisterReq) (*User, error) {
    var count int64
    db.Model(&User{}).Where("email = ?", req.Email).Count(&count)
    if count > 0 {
        return nil, ErrEmailExists
    }

    hash, err := bcrypt.GenerateFromPassword([]byte(req.Password), bcrypt.DefaultCost)
    if err != nil {
        return nil, err
    }

    user := &User{
        Email:        req.Email,
        PasswordHash: string(hash),
        Nickname:     req.Nickname,
    }

    if err := db.Create(user).Error; err != nil {
        return nil, err
    }
    return user, nil
}

func Login(db *gorm.DB, req *LoginReq) (*User, error) {
    var user User
    err := db.Where("email = ?", req.Email).First(&user).Error
    if err != nil {
        if errors.Is(err, gorm.ErrRecordNotFound) {
            return nil, ErrUserNotFound
        }
        return nil, err
    }

    if user.IsActive == 0 {
        return nil, ErrUserInactive
    }

    err = bcrypt.CompareHashAndPassword([]byte(user.PasswordHash), []byte(req.Password))
    if err != nil {
        return nil, ErrWrongPassword
    }

    return &user, nil
}

func GetUserByID(db *gorm.DB, id uint64) (*User, error) {
    var user User
    err := db.First(&user, id).Error
    if err != nil {
        return nil, ErrUserNotFound
    }
    return &user, nil
}

func UpdateProfile(db *gorm.DB, id uint64, req *UpdateProfileReq) (*User, error) {
    var user User
    if err := db.First(&user, id).Error; err != nil {
        return nil, ErrUserNotFound
    }

    updates := map[string]interface{}{}
    if req.Nickname != "" {
        updates["nickname"] = req.Nickname
    }
    if req.Bio != "" {
        updates["bio"] = req.Bio
    }
    if len(updates) > 0 {
        db.Model(&user).Updates(updates)
    }

    db.First(&user, id)
    return &user, nil
}

func ChangePassword(db *gorm.DB, id uint64, req *ChangePasswordReq) error {
    var user User
    if err := db.First(&user, id).Error; err != nil {
        return ErrUserNotFound
    }

    err := bcrypt.CompareHashAndPassword([]byte(user.PasswordHash), []byte(req.OldPassword))
    if err != nil {
        return ErrWrongPassword
    }

    newHash, err := bcrypt.GenerateFromPassword([]byte(req.NewPassword), bcrypt.DefaultCost)
    if err != nil {
        return err
    }

    return db.Model(&user).Update("password_hash", string(newHash)).Error
}

func UpdateAvatar(db *gorm.DB, id uint64, avatarPath string) error {
    return db.Model(&User{}).Where("id = ?", id).
        Update("avatar", avatarPath).Error
}

func ResetPassword(db *gorm.DB, email, newPassword string) error {
    var user User
    if err := db.Where("email = ?", email).First(&user).Error; err != nil {
        return ErrUserNotFound
    }

    newHash, err := bcrypt.GenerateFromPassword([]byte(newPassword), bcrypt.DefaultCost)
    if err != nil {
        return err
    }

    return db.Model(&user).Update("password_hash", string(newHash)).Error
}
```

### middleware/jwt.go — JWT认证中间件

```go
package middleware

import (
    "net/http"
    "strings"
    "time"

    "github.com/gin-gonic/gin"
    "github.com/golang-jwt/jwt/v5"
)

type Claims struct {
    UserID uint64 `json:"user_id"`
    Email  string `json:"email"`
    jwt.RegisteredClaims
}

var jwtSecret []byte

func SetJWTSecret(secret string) {
    jwtSecret = []byte(secret)
}

func GenerateToken(userID uint64, email string) (string, time.Time, error) {
    expiresAt := time.Now().Add(24 * time.Hour)

    claims := Claims{
        UserID: userID,
        Email:  email,
        RegisteredClaims: jwt.RegisteredClaims{
            ExpiresAt: jwt.NewNumericDate(expiresAt),
            IssuedAt:  jwt.NewNumericDate(time.Now()),
            Issuer:    "auth-api",
        },
    }

    token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
    tokenString, err := token.SignedString(jwtSecret)
    if err != nil {
        return "", time.Time{}, err
    }

    return tokenString, expiresAt, nil
}

func AuthRequired() gin.HandlerFunc {
    return func(c *gin.Context) {
        authHeader := c.GetHeader("Authorization")
        if authHeader == "" {
            c.JSON(http.StatusUnauthorized, gin.H{
                "code":    401,
                "message": "缺少认证令牌",
                "data":    nil,
            })
            c.Abort()
            return
        }

        parts := strings.SplitN(authHeader, " ", 2)
        if len(parts) != 2 || parts[0] != "Bearer" {
            c.JSON(http.StatusUnauthorized, gin.H{
                "code":    401,
                "message": "认证格式错误，需要 Bearer token",
                "data":    nil,
            })
            c.Abort()
            return
        }

        tokenString := parts[1]
        claims := &Claims{}

        token, err := jwt.ParseWithClaims(tokenString, claims, func(token *jwt.Token) (interface{}, error) {
            return jwtSecret, nil
        })

        if err != nil || !token.Valid {
            c.JSON(http.StatusUnauthorized, gin.H{
                "code":    401,
                "message": "令牌无效或已过期",
                "data":    nil,
            })
            c.Abort()
            return
        }

        c.Set("user_id", claims.UserID)
        c.Set("email", claims.Email)
        c.Next()
    }
}
```

JWT中间件的核心流程：
1. 从请求头 `Authorization` 中取出 `Bearer xxx`
2. 用密钥验证签名
3. 检查过期时间
4. 把 `user_id` 和 `email` 注入到上下文（后续handler可以直接用）

就像大楼门禁——刷工牌（验证token），确认是本公司员工（签名有效），记录是谁（存入上下文），然后放行。

### handler/auth.go — 注册和登录

```go
package handler

import (
    "net/http"
    "time"

    "auth-api/middleware"
    "auth-api/model"

    "github.com/gin-gonic/gin"
    "gorm.io/gorm"
)

type AuthHandler struct {
    DB *gorm.DB
}

func NewAuthHandler(db *gorm.DB) *AuthHandler {
    return &AuthHandler{DB: db}
}

func (h *AuthHandler) Register(c *gin.Context) {
    var req model.RegisterReq
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(http.StatusBadRequest, gin.H{
            "code":    400,
            "message": "参数错误：" + err.Error(),
            "data":    nil,
        })
        return
    }

    user, err := model.Register(h.DB, &req)
    if err != nil {
        if err == model.ErrEmailExists {
            c.JSON(http.StatusConflict, gin.H{
                "code":    409,
                "message": "该邮箱已被注册",
                "data":    nil,
            })
            return
        }
        c.JSON(http.StatusInternalServerError, gin.H{
            "code":    500,
            "message": "注册失败",
            "data":    nil,
        })
        return
    }

    c.JSON(http.StatusCreated, gin.H{
        "code":    201,
        "message": "注册成功",
        "data": gin.H{
            "id":       user.ID,
            "email":    user.Email,
            "nickname": user.Nickname,
        },
    })
}

func (h *AuthHandler) Login(c *gin.Context) {
    var req model.LoginReq
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(http.StatusBadRequest, gin.H{
            "code":    400,
            "message": "参数错误",
            "data":    nil,
        })
        return
    }

    user, err := model.Login(h.DB, &req)
    if err != nil {
        switch err {
        case model.ErrUserNotFound:
            c.JSON(http.StatusUnauthorized, gin.H{
                "code":    401,
                "message": "邮箱或密码错误",
                "data":    nil,
            })
        case model.ErrWrongPassword:
            c.JSON(http.StatusUnauthorized, gin.H{
                "code":    401,
                "message": "邮箱或密码错误",
                "data":    nil,
            })
        case model.ErrUserInactive:
            c.JSON(http.StatusForbidden, gin.H{
                "code":    403,
                "message": "账号已被禁用",
                "data":    nil,
            })
        default:
            c.JSON(http.StatusInternalServerError, gin.H{
                "code":    500,
                "message": "登录失败",
                "data":    nil,
            })
        }
        return
    }

    token, expiresAt, err := middleware.GenerateToken(user.ID, user.Email)
    if err != nil {
        c.JSON(http.StatusInternalServerError, gin.H{
            "code":    500,
            "message": "生成令牌失败",
            "data":    nil,
        })
        return
    }

    c.JSON(http.StatusOK, gin.H{
        "code":    200,
        "message": "登录成功",
        "data": gin.H{
            "token":      token,
            "expires_at": expiresAt.Format(time.RFC3339),
        },
    })
}
```

注意：登录失败时，不管"用户不存在"还是"密码错误"，返回的消息**统一**为"邮箱或密码错误"。为什么？如果你区分提示（"用户不存在" vs "密码错误"），攻击者就能用这个信息来枚举哪些邮箱注册了——这是安全漏洞。

```go
        c.JSON(http.StatusInternalServerError, gin.H{
            "code":    500,
            "message": "更新失败",
            "data":    nil,
        })
        return
    }

    c.JSON(http.StatusOK, gin.H{
        "code":    200,
        "message": "更新成功",
        "data":    user,
    })
}

func (h *UserHandler) ChangePassword(c *gin.Context) {
    userID := c.GetUint64("user_id")

    var req model.ChangePasswordReq
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(http.StatusBadRequest, gin.H{
            "code":    400,

            })
            return
        }
        c.JSON(http.StatusInternalServerError, gin.H{
            "code":    500,
            "message": "修改失败",
            "data":    nil,
        })
        return
    }

    c.JSON(http.StatusOK, gin.H{
        "code":    200,
        "message": "密码修改成功",
        "data":    nil,
    })
}

func (h *UserHandler) UploadAvatar(c *gin.Context) {
    userID := c.GetUint64("user_id")

    file, err := c.FormFile("avatar")
    if err != nil {
        c.JSON(http.StatusBadRequest, gin.H{
            "code":    400,
            "message": "请选择要上传的文件",
            "data":    nil,
        })
        return
    }

    ext := filepath.Ext(file.Filename)
    allowedExts := map[string]bool{".jpg": true, ".jpeg": true, ".png": true, ".gif": true}
    if !allowedExts[ext] {
        c.JSON(http.StatusBadRequest, gin.H{
            "code":    400,
            "message": "不支持的图片格式，仅支持 jpg/jpeg/png/gif",
            "data":    nil,
        })
        return
    }

    if file.Size > 5*1024*1024 {
        c.JSON(http.StatusBadRequest, gin.H{
            "code":    400,
            "message": "图片大小不能超过5MB",
            "data":    nil,
        })
        return
    }

    newFileName := uuid.New().String() + ext
    savePath := filepath.Join(h.UploadDir, newFileName)

    if err := c.SaveUploadedFile(file, savePath); err != nil {
        c.JSON(http.StatusInternalServerError, gin.H{
            "code":    500,
            "message": "文件保存失败",
            "data":    nil,
        })
        return
    }

    avatarURL := "/uploads/" + newFileName
    if err := model.UpdateAvatar(h.DB, userID, avatarURL); err != nil {
        c.JSON(http.StatusInternalServerError, gin.H{
            "code":    500,
            "message": "更新头像失败",
            "data":    nil,
        })
        return
    }

    c.JSON(http.StatusOK, gin.H{
        "code":    200,
        "message": "上传成功",
        "data": gin.H{
            "avatar_url": avatarURL,
        },
    })
}

func (h *UserHandler) ResetPassword(c *gin.Context) {
    var req struct {
        Email       string `json:"email" binding:"required,email"`
        NewPassword string `json:"new_password" binding:"required,min=8"`
    }
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(http.StatusBadRequest, gin.H{
            "code":    400,
            "message": "参数错误",
            "data":    nil,
        })
        return
    }

    err := model.ResetPassword(h.DB, req.Email, req.NewPassword)
    if err != nil {
        if err == model.ErrUserNotFound {
            c.JSON(http.StatusOK, gin.H{
                "code":    200,
                "message": "如果该邮箱已注册，密码重置链接已发送",
                "data":    nil,
            })
            return
        }
        c.JSON(http.StatusInternalServerError, gin.H{
            "code":    500,
            "message": "重置失败",
            "data":    nil,
        })
        return
    }

    c.JSON(http.StatusOK, gin.H{
        "code":    200,
        "message": "密码重置成功",
        "data":    nil,
    })
}
```

头像上传的几个安全要点：
1. **白名单验证文件扩展名**——只允许 `.jpg`、`.jpeg`、`.png`、`.gif`
2. **限制文件大小**——这里限制5MB
3. **用UUID重命名**——防止用户上传 `../../etc/passwd` 这样的路径穿越攻击
4. **文件扩展名用白名单，不要用黑名单**——白名单是"只允许这些"，黑名单是"禁止这些"。攻击者总能找到黑名单漏掉的后缀

密码重置接口的响应也做了安全处理：不管邮箱存不存在，都返回"如果该邮箱已注册，密码重置链接已发送"。防止攻击者通过这个接口枚举用户邮箱。

### router/router.go

```go
package router

import (
    "auth-api/handler"
    "auth-api/middleware"

    "github.com/gin-gonic/gin"
    "gorm.io/gorm"
)

func Setup(db *gorm.DB, uploadDir string) *gin.Engine {
    r := gin.Default()

    authH := handler.NewAuthHandler(db)
    userH := handler.NewUserHandler(db, uploadDir)

    r.Static("/uploads", uploadDir)

    api := r.Group("/api")
    {
        api.POST("/register", authH.Register)
        api.POST("/login", authH.Login)
        api.POST("/password-reset", userH.ResetPassword)

        authorized := api.Group("/user")
        authorized.Use(middleware.AuthRequired())
        {
            authorized.GET("/profile", userH.GetProfile)
            authorized.PUT("/profile", userH.UpdateProfile)
            authorized.PUT("/password", userH.ChangePassword)
            authorized.POST("/avatar", userH.UploadAvatar)
        }
    }

    return r
}
```

`r.Static("/uploads", uploadDir)` 让上传的图片可以通过HTTP直接访问。比如 `avatar` 字段存了 `/uploads/abc.jpg`，浏览器就能通过 `http://localhost:8080/uploads/abc.jpg` 看到图片。

## 本章小结

本章搭建了一个完整的用户认证系统，你学到了：

1. **bcrypt密码哈希**：永远不存明文密码，bcrypt故意慢以防暴力破解
2. **JWT令牌**：无状态认证，服务器不存session，靠密钥验签来确认身份
3. **认证中间件**：保护需要登录的接口，把用户信息注入上下文
4. **文件上传安全**：扩展名白名单、大小限制、UUID重命名防路径穿越
5. **安全响应策略**：登录失败不区分原因，密码重置不暴露是否存在用户
