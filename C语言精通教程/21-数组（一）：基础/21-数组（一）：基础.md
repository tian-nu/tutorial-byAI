## 我在做什么？

目前为止，你操作的都是单个变量——一个 `int`、一个 `double`、一个 `char`。但如果你要存一个班 50 个学生的成绩，你准备定义 50 个变量吗？

```c
int score1, score2, score3, ..., score50;  // 灾难！
```

这就是数组要解决的问题：**用一个名字，管理一组相同类型的数据。** 读完这一章，你能声明数组、初始化数组、用下标访问元素，并且理解数组越界的危险。

## 一、数组是什么？

### 1.1 一排连续的内存盒子

想象一排储物柜，编号从 0 开始：

```text
int arr[5] = {10, 20, 30, 40, 50};

内存中的样子：
┌──────┬──────┬──────┬──────┬──────┐
│  10  │  20  │  30  │  40  │  50  │
└──────┴──────┴──────┴──────┴──────┘
  arr[0] arr[1] arr[2] arr[3] arr[4]
    ↑                          ↑
  第一个盒子                 最后一个盒子
  (索引从0开始)             (索引 = 长度-1)
```

一个数组有三个关键特征：
1. **所有元素类型相同**（都是 `int`，或都是 `double`）
2. **元素在内存中连续排列**（一个紧挨着一个）
3. **用索引（下标）访问元素**，从 0 开始

### 1.2 为什么索引从 0 开始？

这不是 C 语言故意为难你。原因和内存有关：

```text
arr[0] 的地址 = 数组起始地址 + 0 × sizeof(int)
arr[1] 的地址 = 数组起始地址 + 1 × sizeof(int)
arr[2] 的地址 = 数组起始地址 + 2 × sizeof(int)
```

索引就是"偏移量"——从起始位置往后跳几个元素。第一个元素偏移 0 个位置，所以索引是 0。

## 二、声明与初始化

### 2.1 声明数组

```c
类型 数组名[大小];
```

```c
int scores[5];          // 5 个 int 的数组
double prices[10];      // 10 个 double 的数组
char letters[26];       // 26 个 char 的数组
```

注意：**大小必须是常量**（C99 之前），或者是常量表达式。C99 允许 *VLA（变长数组）* *此术语见附录F*，下一章会讲。

```c
int n = 5;
int arr[n];    // VLA，C99 允许，但 C11 变成了可选特性
```

### 2.2 初始化数组

方式一：完整初始化

```c
int arr[5] = {10, 20, 30, 40, 50};
// arr[0]=10, arr[1]=20, arr[2]=30, arr[3]=40, arr[4]=50
```

方式二：部分初始化（剩余元素自动补 0）

```c
int arr[5] = {10, 20};
// arr[0]=10, arr[1]=20, arr[2]=0, arr[3]=0, arr[4]=0
```

方式三：自动推断大小

```c
int arr[] = {10, 20, 30, 40, 50};  // 编译器自动算出大小是 5
```

方式四：全部初始化为 0

```c
int arr[100] = {0};   // 第一个元素是 0，其余 99 个自动补 0
```

### 2.3 数组大小的计算

在定义数组的同一个作用域内，可以用 `sizeof` 计算数组大小：

```c
int arr[] = {10, 20, 30, 40, 50};
int size = sizeof(arr) / sizeof(arr[0]);  // 5

// sizeof(arr)    = 5 × 4 字节 = 20 字节（假设 int 是 4 字节）
// sizeof(arr[0]) = 4 字节
// 20 / 4 = 5 个元素
```

**警告**：这个方法只在数组定义的作用域内有效。数组传进函数后，`sizeof` 得到的是指针大小（第19章讲过）。

## 三、访问数组元素

### 3.1 下标访问

```c
int arr[5] = {10, 20, 30, 40, 50};

arr[0] = 100;          // 修改第一个元素
int x = arr[2];        // 读取第三个元素，x = 30
arr[4] = arr[0] + 5;   // 把第一个元素的值加 5 赋给最后一个元素
```

### 3.2 遍历数组

```c
#include <stdio.h>

int main() {
    int scores[] = {85, 92, 78, 95, 88};
    int size = sizeof(scores) / sizeof(scores[0]);

    printf("成绩列表：\n");
    for (int i = 0; i < size; i++) {
        printf("第 %d 个学生：%d 分\n", i + 1, scores[i]);
    }

    return 0;
}
```

输出：
```text
成绩列表：
第 1 个学生：85 分
第 2 个学生：92 分
第 3 个学生：78 分
第 4 个学生：95 分
第 5 个学生：88 分
```

### 3.3 数组元素的常见操作

```c
#include <stdio.h>

int main() {
    int arr[] = {3, 1, 7, 5, 9};
    int size = 5;

    // 求和
    int sum = 0;
    for (int i = 0; i < size; i++) {
        sum += arr[i];
    }
    printf("和：%d\n", sum);  // 25

    // 找最大值
    int max = arr[0];
    for (int i = 1; i < size; i++) {
        if (arr[i] > max) {
            max = arr[i];
        }
    }
    printf("最大值：%d\n", max);  // 9

    // 查找
    int target = 7;
    int found = -1;
    for (int i = 0; i < size; i++) {
        if (arr[i] == target) {
            found = i;
            break;
        }
    }
    if (found != -1) {
        printf("找到了，索引是 %d\n", found);  // 索引是 2
    } else {
        printf("没找到\n");
    }

    return 0;
}
```

## 四、数组越界：C语言最臭名昭著的陷阱

### 4.1 C语言不检查数组越界

这是 C 语言和 Java/Python 等语言最大的区别之一：

```c
int arr[5] = {10, 20, 30, 40, 50};

arr[5] = 999;    // 越界了！但 C 编译器不报错！
arr[100] = 888;  // 严重越界！编译器也不管！
```

```text
合法范围：arr[0] ~ arr[4]

┌──────┬──────┬──────┬──────┬──────┬──────┬──────┐
│  10  │  20  │  30  │  40  │  50  │  ??  │  ??  │
└──────┴──────┴──────┴──────┴──────┴──────┴──────┘
  [0]    [1]    [2]    [3]    [4]    [5]    [6]
   ↑                             ↑       ↑
  合法                          越界    严重越界
```

### 4.2 越界的后果

*数组越界* *此术语见附录F* 的后果取决于你踩到了什么：

1. **什么也没发生**（最危险）：你踩到了没被使用的内存，程序正常运行——但这是一个定时炸弹。
2. **数据损坏**：你踩到了其他变量的内存，那个变量的值被莫名其妙改了。
3. *段错误（Segmentation Fault）* *此术语见附录F*：你踩到了操作系统保护的内存，程序崩溃。

```text
内存布局：
┌──────────────────────────────────────┐
│ 变量 a  │ 变量 b  │ arr[0..4] │ 变量 c  │
└──────────────────────────────────────┘

如果写 arr[5]，可能踩到变量 c，把 c 的值改了
如果写 arr[-1]，可能踩到变量 b，把 b 的值改了
```

### 4.3 如何避免越界

```c
#include <stdio.h>

#define SIZE 5        // 用宏定义数组大小，避免魔法数字

int main() {
    int arr[SIZE] = {10, 20, 30, 40, 50};

    // 遍历时用 SIZE 控制边界
    for (int i = 0; i < SIZE; i++) {   // 不是 i <= SIZE
        printf("%d ", arr[i]);
    }
    printf("\n");

    return 0;
}
```

**经验法则**：
- 遍历数组时，循环条件永远是 `i < size`，不是 `i <= size`。
- 尽量用 `sizeof(arr) / sizeof(arr[0])` 计算大小，而不是硬编码数字。
- 用 `#define` 或 `const` 定义数组大小，用到的地方都引用它。

## 五、常见错误与排错

### 5.1 用变量作为数组大小（C99 之前）

❌ 错误示例（C89/C90 中）：
```c
int n;
scanf("%d", &n);
int arr[n];   // C89 中不允许，C99 允许但 C11 可选
```

✅ 安全做法：
```c
#define MAX 100
int arr[MAX];   // 用常量
```

### 5.2 初始化时越界

❌ 错误示例：
```c
int arr[3] = {1, 2, 3, 4};  // 初始化元素多于数组大小
```

编译错误：
```text
error: excess elements in array initializer
```

### 5.3 忘记索引从 0 开始

❌ 错误示例：
```c
int arr[5] = {10, 20, 30, 40, 50};
for (int i = 1; i <= 5; i++) {    // 从 1 开始，到 5 结束
    printf("%d ", arr[i]);          // arr[0] 没打印，arr[5] 越界
}
```

✅ 正确示例：
```c
int arr[5] = {10, 20, 30, 40, 50};
for (int i = 0; i < 5; i++) {      // 从 0 开始，到 4 结束
    printf("%d ", arr[i]);
}
```

### 5.4 数组整体赋值

❌ 错误示例：
```c
int a[5] = {1, 2, 3, 4, 5};
int b[5];
b = a;          // 错误！数组不能整体赋值
```

✅ 正确示例：
```c
int a[5] = {1, 2, 3, 4, 5};
int b[5];
for (int i = 0; i < 5; i++) {
    b[i] = a[i];   // 逐个元素复制
}
```

## 我做得对不对？

用以下程序验证：

```c
#include <stdio.h>

int main() {
    // 测试1：初始化
    int arr1[5] = {10, 20, 30, 40, 50};
    printf("测试1：arr1[0] = %d (应为 10), arr1[4] = %d (应为 50)\n",
           arr1[0], arr1[4]);

    // 测试2：部分初始化
    int arr2[5] = {1, 2};
    printf("测试2：arr2[2] = %d (应为 0), arr2[4] = %d (应为 0)\n",
           arr2[2], arr2[4]);

    // 测试3：sizeof 计算大小
    int arr3[] = {1, 2, 3, 4, 5, 6, 7};
    int size = sizeof(arr3) / sizeof(arr3[0]);
    printf("测试3：数组大小 = %d (应为 7)\n", size);

    // 测试4：求和
    int sum = 0;
    for (int i = 0; i < size; i++) {
        sum += arr3[i];
    }
    printf("测试4：和 = %d (应为 28)\n", sum);

    return 0;
}
```

编译运行：
```bash
$ gcc -Wall -o test test.c
$ ./test
测试1：arr1[0] = 10 (应为 10), arr1[4] = 50 (应为 50)
测试2：arr2[2] = 0 (应为 0), arr2[4] = 0 (应为 0)
测试3：数组大小 = 7 (应为 7)
测试4：和 = 28 (应为 28)
```

## 不对怎么办？

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| 程序崩溃（段错误） | 数组越界访问 | 检查循环条件，确保 `i < size` 而不是 `i <= size` |
| 输出乱码或奇怪数字 | 访问了未初始化的数组元素 | 初始化所有元素，或用 `{0}` 全部置零 |
| `excess elements in initializer` | 初始化列表元素多于数组大小 | 减少初始化元素，或增大数组 |
| 数组大小算出来不对 | 在函数参数中用 sizeof | 数组传参后 sizeof 返回指针大小，需额外传 size |
| 程序正常运行但数据不对 | 可能越界覆盖了其他变量 | 检查所有数组访问，确保索引在合法范围内 |

## 本章小结

| 知识点 | 一句话总结 |
|--------|-----------|
| 数组是什么 | 一组相同类型、连续存储的元素，用索引访问 |
| 索引从 0 开始 | 索引表示从起始地址的偏移量 |
| 声明 | `类型 数组名[大小]` |
| 初始化 | `{值1, 值2, ...}`，未指定部分自动补 0 |
| 数组大小 | `sizeof(arr) / sizeof(arr[0])`（仅定义处有效） |
| 数组越界 | C 语言不检查，后果不可预测，是严重 bug 来源 |
| 段错误 | 访问操作系统保护的内存时程序崩溃 |
