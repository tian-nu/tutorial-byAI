### 组件拆解

| 区域 | HTML 标签 | 布局方式 | 数据来源 |
|------|----------|---------|---------|
| 导航栏 | `<nav>` | Flexbox 横排 | 静态 HTML |
| 文章列表 | `<main>` + `<article>` | 纵向排列 | JS 动态渲染 |
| 侧边栏 | `<aside>` | Flexbox 右侧 | 静态 HTML |
| 页脚 | `<footer>` | 居中 | 静态 HTML |
| 整体 | `<div class="page">` | Flexbox 纵向 | — |
| 内容区 | `<div class="page-body">` | Flexbox 横向（main+aside） | — |

> 🤔 **想多一点**：拿到一个页面设计时，先别急着写代码。花 5 分钟在纸上（或脑海中）画一个结构图，把每个区域对应的 HTML 标签标出来，把数据流理清楚。这个习惯会让你在后续学 Vue 时过渡得非常平滑——因为 Vue 的组件拆分本质上就是这件事。

```html
        <!-- 文章卡片将由 JavaScript 动态生成 -->

      <!-- 侧边栏 -->

        <div class="sidebar-card">
          <h3>📌 关于我</h3>
          <p>一名前端开发者，热爱技术分享。</p>
        </div>
        <div class="sidebar-card">
          <h3>🏷️ 标签云</h3>
          <div class="tags">
            <span class="tag">HTML</span>
            <span class="tag">CSS</span>
            <span class="tag">JavaScript</span>
            <span class="tag">Flexbox</span>
            <span class="tag">Vue</span>
          </div>
        </div>
      </aside>
    </div>

    <!-- 页脚 -->
    <footer class="page-footer">
      <p>&copy; 2026 我的技术博客 · 用 ❤️ 和 HTML+CSS+JS 构建</p>
    </footer>

  </div>

  <script src="app.js"></script>

```

### 关键点

- 文章列表区 `<main class="post-list" id="post-list">` 是空的——里面没有文章卡片。文章卡片将由 JS 动态生成。
- 侧边栏是静态 HTML，因为个人简介和标签云是固定的。
- `<script src="app.js"></script>` 放在 `</body>` 之前，确保 DOM 加载完毕后再执行。

- `mockPosts` 是一个对象数组，模拟从后端 API 返回的数据。在真实项目中，你会用 `fetch()` 从服务器获取这些数据。

- 使用了**模板字符串**（反引号 `` ` `` + `${}`）来拼接 HTML。这比用 `+` 拼接字符串可读得多。

> 🤔 **想多一点**：这里的 `mockPosts` 数组就是"数据"，`renderPosts` 函数就是"渲染逻辑"。数据变了，重新调用 `renderPosts` 就能更新页面。但注意——每次更新都是**全量替换**（`innerHTML = ''` 然后重新创建所有元素），性能低且会丢失用户交互状态（如滚动位置、输入框内容）。这就是为什么我们需要 Vue 的**虚拟 DOM**和**响应式系统**——它能精确知道哪些数据变了，只更新对应的 DOM 节点。

# 步骤 5：点击文章——alert 模拟跳转

在 `app.js` 中，给 `renderPosts` 函数末尾添加事件绑定：

```javascript
function renderPosts(posts) {
  const postList = document.querySelector('#post-list');
  postList.innerHTML = '';

  posts.forEach(post => {
    const card = document.createElement('article');
    card.className = 'post-card';
    card.innerHTML = `
      <h2 class="post-card-title">${post.title}</h2>
      <div class="post-card-meta">
        <span>📅 ${post.date}</span>
        <span class="tag-label">${post.tag}</span>
        <span>❤️ ${post.likes} 赞</span>
      </div>
      <p class="post-card-summary">${post.summary}</p>
      <a href="#" class="post-card-link" data-post-id="${post.id}">阅读全文 →</a>
    `;

    // ========== 点击事件绑定 ==========

    // 点击整张卡片 → alert 模拟跳转
    card.addEventListener('click', function() {
      alert(`即将跳转到文章详情页：\n"${post.title}"\n\n（在真实项目中，这里会跳转到 /post/${post.id}）`);
    });

    // 点击"阅读全文"链接 → 阻止默认跳转，同样 alert
    const link = card.querySelector('.post-card-link');
    link.addEventListener('click', function(event) {
      event.stopPropagation();  // 阻止事件冒泡，避免触发卡片的 click
      alert(`点击了"阅读全文"：\n"${post.title}"\n\n（这里会跳转到文章详情页）`);
    });

    postList.appendChild(card);
  });
}
```

### 事件冒泡（Event Bubbling）解释

当你点击"阅读全文"链接时，点击事件会先触发链接的 `click`，然后**向上冒泡**到卡片的 `click`。如果不禁用冒泡，两个 alert 都会弹出。

`event.stopPropagation()` 的作用就是**阻止事件向上冒泡**——让链接的点击只触发链接的处理函数，不触发卡片父元素的处理函数。

**类比**：你在一个房间里（链接）按了灯开关，正常情况下只有这个房间的灯会亮。但如果没有 `stopPropagation`，整栋楼（卡片）的灯都会亮。

# 步骤 7：反思——手动 DOM 的痛点，引出框架

### 7.1 你刚刚做了什么

回顾一下，为了实现"渲染 5 篇文章"这个功能，你写了：

1. 一个 mock 数据数组
2. 一个 `renderPosts` 函数，里面用 `createElement`、`innerHTML`、`appendChild` 手动创建和插入 DOM
3. 手动给每个元素绑定事件

### 7.2 当需求变复杂时

想象一下，如果产品经理提了这些需求：

| 需求 | 手动 DOM 的做法 | 工作量 |
|------|----------------|--------|
| 按点赞数排序 | 排序数组 → 全量重渲染 → 重新绑定事件 | 改数组 + 重新调用 renderPosts |
| 用户点赞某篇文章 | 找到对应 DOM 节点 → 更新点赞数 → 更新样式 | 需要 `querySelector` 精确定位 |
| 筛选标签（只显示 CSS 文章） | 过滤数组 → 全量重渲染 → 重新绑定事件 | 过滤 + 重新调用 renderPosts |
| 搜索框输入实时过滤 | 每次输入都需要过滤 → 全量重渲染 | 性能问题：每次输入都重建所有 DOM |
| 文章列表分页 | 维护当前页码 → 计算切片 → 渲染 → 分页按钮事件 | 大量状态管理代码 |

**核心痛点**：数据和 DOM 是**分离的**。数据变了，你需要手动告诉 DOM 怎么变。而这个"手动告诉"的过程，代码量巨大、容易出错、难以维护。

### 7.3 Vue 的解决方案

```javascript
// Vue 中你只需要关注数据
const posts = ref([...]);  // 响应式数据

// 模板中直接写
<article v-for="post in posts" :key="post.id" @click="goToPost(post)">
  <h2>{{ post.title }}</h2>
  <p>{{ post.summary }}</p>
</article>
```

- 数据变了 → DOM 自动更新。你不需要手动 `querySelector`、不需要手动 `innerHTML`。
- 事件绑定写在模板里，不需要手动 `addEventListener`。
- 列表渲染用 `v-for`，不需要手动 `forEach` + `createElement`。

**Vue 的核心价值**：让你从"手动操作 DOM"的体力活中解放出来，专注于"数据是什么"和"数据怎么变"。

### 小结：本教程系列的定位

| 教程 | 学到的 | 对应 Vue 中的概念 |
|------|--------|-----------------|
| HTML | 标签、语义化 | Vue 模板语法 |
| CSS | 选择器、盒模型、Flexbox | 和 Vue 完全兼容，直接使用 |
| JavaScript | DOM 操作、事件、数组、函数 | Vue 帮你自动完成这些 |
| 本项目 | 手动 DOM 渲染的痛点 | 理解"为什么需要框架" |

## 🧠 术语附录

| 术语 | 解释 |
|------|------|
| **静态页面（Static Page）** | 每次请求返回完全相同的 HTML 文件，没有后端动态生成内容。本教程的博客就是静态页面。 |
| **mock 数据（Mock Data）** | 模拟真实后端 API 返回的假数据，用于前端开发和测试。 |
| **动态渲染（Dynamic Rendering）** | 用 JavaScript 在浏览器中根据数据动态生成 HTML，而不是服务器返回现成的 HTML。 |
| **事件冒泡（Event Bubbling）** | 事件从触发元素向上传播到父元素、祖父元素……直到 `document`。`stopPropagation()` 可以阻止。 |
| **模板字符串（Template Literal）** | 反引号 `` ` `` 包裹的字符串，支持 `${变量}` 语法嵌入变量和多行文本。 |
| **响应式（Reactive）** | 数据变化时，依赖该数据的 UI 自动更新。Vue 的核心特性之一。 |
| **虚拟 DOM（Virtual DOM）** | 框架在内存中维护的轻量 DOM 副本，通过对比差异（diff）来精确更新真实 DOM，提高性能。 |

---

## 🚧 已知坑点与禁止事项

- **`innerHTML` 会清空所有子元素**：每次调用 `innerHTML = ''` 会销毁所有旧的 DOM 节点，包括上面绑定的事件。这就是为什么 Vue 不这样做的原因。
- **事件冒泡需要小心**：嵌套元素的事件会向上冒泡，不想触发父元素事件时记得 `event.stopPropagation()`。
- **mock 数据不是真实数据**：`jsonplaceholder.typicode.com` 等免费 API 可以用于练习，但生产环境需要真实后端。
- **不要在生产环境用 `alert`**：`alert` 会阻塞整个页面，用户体验极差。真实项目中用模态框（Modal）或路由跳转。
