**一句话目标**：学会在 Linux 命令行里"活下来"——不要求精通，但遇到 `Permission denied` 要知道怎么查，能用终端完成文件操作、权限修改、进程管理。

## 三、分步操作

### 步骤 1：为什么学 Linux？——服务器 99% 是 Linux

**我在做什么？** 理解为什么后端工程师必须会 Linux。

你写的博客 API 运行在自己的电脑上（`127.0.0.1:8000`），只有你自己能访问。要让全世界的人都能访问，你需要把它部署到**服务器**上。

**服务器是什么？** 服务器就是一台永远不关机的电脑，放在数据中心里，有公网 IP。你把自己写的程序放上去，它 24 小时运行，任何人都能访问。

**关键事实**：全世界 99% 的服务器运行的是 Linux，不是 Windows。你买的云服务器，默认系统就是 Linux。

**为什么服务器用 Linux 而不是 Windows？**
- **免费**：Linux 开源免费，Windows Server 要买许可证。
- **轻量**：Linux 没有图形界面，省内存、省 CPU，一台 1 核 1G 的机器就能跑你的博客。
- **稳定**：Linux 服务器可以连续运行好几年不重启。
- **生态**：Docker、Nginx、数据库、各种后端工具，都是为 Linux 优先设计的。

**比喻**：Linux 是后端世界的"通用语言"。你去任何一个机房，随便打开一台服务器，里面跑的都是 Linux。你可以不会焊电路板，但你必须会拧螺丝——Linux 命令行就是后端工程师的"螺丝刀"。

> 🤔 想多一点：我不用学 Linux，用宝塔面板行不行？

宝塔面板是一个 Linux 的图形化管理工具（网页版），可以让你用鼠标点击来管理服务器。**对于快速部署，宝塔没问题。** 但它的局限性在于：
- 宝塔封装了底层操作，出了问题你不知道怎么排查。
- 面试时面试官不会问你"会不会用宝塔"，他会问"Linux 权限数字 755 是什么意思"。
- 很多公司服务器不允许安装第三方面板。

**本教程教你最基础的 Linux 命令行，30 分钟，够你部署项目用一辈子。**

### 步骤 2：安装 WSL2——在 Windows 里跑 Linux

**我在做什么？** 在 Windows 上安装一个"内置的 Linux 子系统"，不需要虚拟机、不需要双系统。

**WSL**（Windows Subsystem for Linux，此术语需进附录）是微软官方提供的功能——在 Windows 里直接跑一个真正的 Linux 内核，比虚拟机快得多，文件互通。

**比喻**：WSL 就像在 Windows 这个大房子里，隔了一个"Linux 小房间"。你在这个小房间里可以做任何 Linux 的事，但随时可以回到 Windows 大房间拿东西。

#### 安装步骤

**第一步：启用 WSL 功能**

以**管理员身份**打开 PowerShell（右键开始菜单 → "终端(管理员)" 或 "Windows PowerShell(管理员)"），输入：

```powershell
wsl --install
```

这条命令会自动完成以下操作：
1. 启用"适用于 Linux 的 Windows 子系统"功能
2. 启用"虚拟机平台"功能
3. 下载并安装 Linux 内核更新包
4. 安装 Ubuntu（默认发行版）

**如果提示需要重启，重启电脑后再继续。**

**第二步：设置 Ubuntu 用户名和密码**

重启后，系统会自动打开一个 Ubuntu 终端窗口。第一次启动会提示你创建用户名和密码：

```
Installing, this may take a few minutes...
Please create a default UNIX user account. The username does not need to match your Windows username.
Enter new UNIX username: your_username_here
New password: your_password_here
Retype new password: your_password_here
```

**注意**：
- 用户名用小写英文，不要有空格。比如 `zhangsan`。
- 密码输入时**屏幕上不会显示任何字符**（甚至不会显示 `***`），这是 Linux 的安全设计，不是键盘坏了。盲打密码，然后按回车。
- 记住这个密码！后面用 `sudo` 命令时需要输入。

**第三步：验证安装**

打开 Ubuntu 终端（从开始菜单搜索 "Ubuntu"），输入：

```bash
lsb_release -a
```

**预期输出**：

```
No LSB modules are available.
Distributor ID: Ubuntu
Description:    Ubuntu 22.04 LTS
Release:        22.04
Codename:       jammy
```

看到 `Ubuntu 22.04 LTS`，安装成功 ✅

**我做得对不对？** 打开 Ubuntu 终端，能看到 `your_username@你的电脑名:~$` 这样的提示符，说明一切正常。

**不对怎么办？**

| 问题 | 解决 |
|------|------|
| PowerShell 提示 `wsl` 不是命令 | 你的 Windows 版本太旧（需要 Win10 2004+ 或 Win11）。更新 Windows 或手动安装 WSL：参考 [微软官方文档](https://learn.microsoft.com/zh-cn/windows/wsl/install-manual) |
| 重启后 Ubuntu 没自动打开 | 从开始菜单搜索 "Ubuntu"，手动打开 |
| `wsl --install` 报错 `0x800701bc` | 需要手动下载 WSL 内核更新包：去微软官网下载 `wsl_update_x64.msi` 安装 |
| 忘记 Ubuntu 密码 | 在 PowerShell 中运行 `wsl --user root`，然后 `passwd 你的用户名` 重置密码 |

### 步骤 3：保命四件套——`ls` / `cd` / `mkdir` / `vim`

**我在做什么？** 学习四个最核心的命令，覆盖 80% 的日常操作。

打开 Ubuntu 终端，我们开始。

#### 3.1 `ls`——看看这地方有什么

```bash
ls
```

**效果**：列出当前目录下的所有文件和文件夹。

```bash
ls -l     # 详细列表模式（显示权限、大小、时间）
ls -a     # 显示隐藏文件（以 . 开头的文件）
ls -la    # 两个参数组合：详细 + 隐藏
```

**比喻**：`ls` 就是你在文件夹里"看一眼"——`ls` 是瞥一眼，`ls -l` 是仔细看标签。

#### 3.2 `cd`——去别的地方

```bash
cd /home/你的用户名    # 去指定目录（绝对路径）
cd ~                   # 回家（用户主目录）
cd ..                  # 回上一级目录
cd -                   # 回上一个待过的目录
cd /                   # 去根目录（整个文件系统的最顶层）
```

**比喻**：`cd` 是传送门——你可以在文件系统的任何位置之间瞬间移动。

#### 3.3 `mkdir`——建个文件夹

```bash
mkdir my_project        # 创建一个文件夹
mkdir -p a/b/c          # 一次性创建多层文件夹（a/b/c，即使 a 和 b 不存在）
```

#### 3.4 `vim`——终端里的编辑神器

**vim 是 Linux 上最常用的文本编辑器。** 但它有一个"臭名昭著"的特点：**刚打开时不知道怎么打字，也不知道怎么退出。** 没关系，你只需要记住三个操作：

```bash
vim hello.txt     # 用 vim 打开（或创建）hello.txt 文件
```

**vim 的三个模式**：

| 操作 | 效果 | 什么时候用 |
|------|------|-----------|
| 按 `i` | 进入**编辑模式**（左下角出现 `-- INSERT --`） | 想打字的时候 |
| 按 `Esc` | 退出编辑模式，回到**命令模式** | 打完了，想保存或退出 |
| 输入 `:wq` 然后回车 | **保存并退出**（`w` = write 写入，`q` = quit 退出） | 编辑完成，保存离开 |
| 输入 `:q!` 然后回车 | **不保存强制退出**（`q` = quit，`!` = 强制执行） | 改错了，不想保存 |

**完整操作流程**：

```bash
# 1. 创建并打开文件
vim hello.txt

# 2. 按 i 键 → 进入编辑模式
# 3. 输入内容：Hello, Linux!
# 4. 按 Esc 键 → 退出编辑模式
# 5. 输入 :wq → 回车 → 保存并退出
```

**比喻**：vim 就像一把"安全锁"——你不按 `i` 就不能编辑，防止你误触键盘把文件改坏。`Esc` 是"退出编辑"，`:wq` 是"关门锁好"。

**我做得对不对？** 退出 vim 后，用 `cat hello.txt` 查看文件内容，应该看到你刚才输入的 `Hello, Linux!`。

**不对怎么办？**

| 问题 | 解决 |
|------|------|
| 按 `i` 没反应，打不了字 | 确保键盘输入法是英文状态。中文输入法下 vim 可能不响应 |
| 按 `Esc` 再按 `:wq` 没反应 | 确认 `:` 是英文冒号。如果还是不行，按几次 `Esc`，再输入 `:wq!`（加感叹号强制） |
| 不小心按了 `Ctrl+S`，终端卡住了 | 按 `Ctrl+Q` 恢复（`Ctrl+S` 是古老的"暂停输出"快捷键） |

### 步骤 4：其他常用命令——`cp` / `mv` / `rm` / `cat`

**我在做什么？** 学习文件复制、移动、删除、查看的命令。

#### 4.1 `cp`——复制文件

```bash
cp hello.txt hello_backup.txt        # 复制文件
cp -r my_project my_project_backup   # 复制整个文件夹（-r = 递归复制）
```

#### 4.2 `mv`——移动/重命名文件

```bash
mv hello.txt Documents/        # 把 hello.txt 移到 Documents 文件夹
mv old_name.txt new_name.txt   # 重命名文件（mv 没有"重命名"命令，移动就是重命名）
```

**比喻**：Linux 里没有"重命名"这个操作——`mv` 既负责"搬东西"也负责"改名字"。把文件从"旧名字"搬到"新名字"，就等于重命名。

#### 4.3 `rm`——删除文件（⚠️ 高危操作）

```bash
rm hello.txt           # 删除一个文件
rm -r my_project/      # 删除整个文件夹（-r = 递归）
rm -rf my_project/     # 强制删除，不提示确认（⚠️ 极其危险）
```

**⚠️ Linux 的 `rm` 没有"回收站"！删了就没了，永远找不回来！**

**安全原则**：
- 删除前先用 `ls` 确认当前目录。
- 尽量用完整路径，不要用模糊的 `rm -rf *`。
- 不确定时，先用 `ls` 代替 `rm` 看看会匹配到什么文件。

```bash
# ❌ 危险做法：rm -rf /（← 这会删掉整个系统！）
# ❌ 危险做法：rm -rf *（← 当前目录所有文件全部消失）
# ✅ 安全做法：rm -rf my_project/（← 明确指定要删的文件夹）
```

#### 4.4 `cat`——快速查看文件内容

```bash
cat hello.txt         # 把文件内容全部输出到屏幕
cat -n hello.txt      # 带行号显示
```

**比喻**：`cat` 就像把文件内容"倒"在屏幕上。适合看小文件，大文件建议用 `less`（`less hello.txt`，按 `q` 退出）。

### 步骤 5：权限——`chmod` / `chown` / `ls -l`

**我在做什么？** 理解 Linux 的权限系统，这是 "Permission denied" 错误的根源。

#### 5.1 看懂 `ls -l` 的权限字符串

```bash
ls -l
```

你会看到类似这样的输出：

```
drwxr-xr-x  2 user user  4096 Jun  5 10:00 my_project/
-rw-r--r--  1 user user   123 Jun  5 10:01 hello.txt
```

**聚焦第一列，那个神秘字符串**：

```
drwxr-xr-x
-rw-r--r--
```

**拆解权限字符串**（以 `-rw-r--r--` 为例）：

```
-   rw-   r--   r--
①   ②     ③     ④
```

| 位置 | 含义 | 示例值 | 解释 |
|------|------|--------|------|
| ① 第 1 位 | 文件类型 | `-` | `-` = 普通文件，`d` = 目录，`l` = 链接 |
| ② 第 2-4 位 | 所有者权限 | `rw-` | 文件主人可以读(r)和写(w)，不能执行(-) |
| ③ 第 5-7 位 | 同组用户权限 | `r--` | 同组的人只能读(r)，不能写不能执行 |
| ④ 第 8-10 位 | 其他人权限 | `r--` | 其他所有人只能读(r) |

**三种权限字母**：

| 字母 | 含义 | 对文件 | 对目录 |
|------|------|--------|--------|
| `r` | read（读） | 可以查看文件内容 | 可以列出目录内容（`ls`） |
| `w` | write（写） | 可以修改文件内容 | 可以在目录中创建/删除文件 |
| `x` | execute（执行） | 可以运行这个文件（如脚本） | 可以进入这个目录（`cd`） |
| `-` | 没有此权限 | — | — |

**比喻**：权限就像"公寓门禁卡"——`r` 是"能看房号"，`w` 是"能开门进去"，`x` 是"能进这栋楼"。你自己的房间（owner）可以全开，邻居（group）只能看，陌生人（others）只能看。

#### 5.2 `chmod`——改权限（此术语需进附录）

**数字法**（最常用）：

| 权限 | 数字 | 计算方式 |
|------|------|----------|
| `---` | 0 | 0+0+0 |
| `--x` | 1 | 0+0+1 |
| `-w-` | 2 | 0+2+0 |
| `-wx` | 3 | 0+2+1 |
| `r--` | 4 | 4+0+0 |
| `r-x` | 5 | 4+0+1 |
| `rw-` | 6 | 4+2+0 |
| `rwx` | 7 | 4+2+1 |

**记忆口诀**：读=4，写=2，执行=1。要什么权限就加什么数字。

**三个数字分别代表**：所有者、同组、其他人。

```bash
chmod 755 script.sh     # 所有者 rwx，同组 r-x，其他人 r-x
chmod 644 hello.txt     # 所有者 rw-，同组 r--，其他人 r--
chmod 777 public/       # 所有人 rwx（⚠️ 不安全，少用）
```

**常用组合**：

| 数字 | 权限字符串 | 适用场景 |
|------|-----------|----------|
| `755` | `rwxr-xr-x` | 目录、可执行脚本 |
| `644` | `rw-r--r--` | 普通文件（代码、配置） |
| `600` | `rw-------` | 敏感文件（密钥、私钥） |

**字母法**（适合小改）：

```bash
chmod +x script.sh       # 给所有人加执行权限
chmod -w hello.txt       # 去掉所有人的写权限
chmod u+x script.sh      # 只给所有者(u)加执行权限
```

#### 5.3 `chown`——改所有者（此术语需进附录）

```bash
chown 用户名 文件名            # 把文件所有者改为指定用户
chown 用户名:组名 文件名       # 同时改所有者和组
chown -R 用户名:组名 目录/     # 递归修改整个目录
```

**什么时候用？** 当你从别人那里复制了文件，或者用 `sudo` 创建了文件，文件所有者变成了 `root`，你需要改回来。

### 步骤 6：进程——`ps aux` / `kill` / `htop`

**我在做什么？** 学习如何查看和管理正在运行的程序（进程）。

#### 6.1 `ps aux`——查看所有进程

```bash
ps aux
```

**输出解读**：

| 列 | 含义 | 示例 |
|----|------|------|
| USER | 哪个用户启动的 | root, zhangsan |
| PID | 进程 ID（唯一编号） | 1234 |
| %CPU | CPU 占用率 | 2.5 |
| %MEM | 内存占用率 | 1.2 |
| COMMAND | 启动命令 | `python3 main.py` |

**常用组合**：

```bash
ps aux | grep python        # 只显示包含 "python" 的进程
ps aux | grep uvicorn       # 查找你的 FastAPI 服务是否在运行
```

**管道的比喻**：`|`（管道符）像一个水管——把左边命令的输出，接到右边命令的输入。`ps aux` 输出所有进程 → 经过管道 → `grep python` 只保留含 "python" 的行。

#### 6.2 `kill`——终止进程

```bash
kill 1234               # 礼貌地请进程 1234 退出（发送 TERM 信号）
kill -9 1234            # 强制杀死进程 1234（发送 KILL 信号，不给它收拾的机会）
```

**比喻**：`kill` = 通知进程"你可以下班了"，`kill -9` = 保安把进程"架出去"。

**什么时候用 `-9`？** 只有当普通 `kill` 无效（进程卡死、无响应）时才用。`-9` 是强制终止，进程来不及保存数据。

#### 6.3 `htop`——可视化的进程管理器（此术语需进附录）

```bash
# 先安装 htop
sudo apt install htop -y

# 运行
htop
```

**效果**：一个彩色的、可以上下滚动的进程列表，CPU 和内存使用率以进度条显示。按 `F10` 或 `q` 退出。

**比喻**：`ps aux` 是"拍一张照片"（静态快照），`htop` 是"看监控摄像头"（实时刷新）。`htop` 比 `ps aux` 直观得多，但 `ps aux` 不需要额外安装。

### 步骤 7：练习——完整走一遍

**我在做什么？** 把前面学的命令串起来，做一个完整的练习。

在 Ubuntu 终端中依次执行：

```bash
# 1. 创建目录
mkdir ~/linux_practice
cd ~/linux_practice

# 2. 创建文件并用 vim 编辑
vim diary.txt
# 按 i → 输入 "Day 1: I learned Linux basics!" → 按 Esc → 输入 :wq → 回车

# 3. 用 cat 查看
cat diary.txt
# 预期输出：Day 1: I learned Linux basics!

# 4. 复制文件
cp diary.txt diary_backup.txt

# 5. 查看权限
ls -l
# 预期看到两个文件，权限类似 -rw-r--r--

# 6. 修改权限
chmod 600 diary.txt
ls -l diary.txt
# 预期看到权限变成了 -rw-------

# 7. 查看进程
ps aux | grep bash
# 预期看到你的 bash 终端进程

# 8. 清理（删除练习目录）
cd ~
rm -rf ~/linux_practice
```

## 七、已知坑点与禁止事项

| 坑点 | 现象 | 原因 | 解决 |
|------|------|------|------|
| WSL 安装后忘记密码 | 无法用 `sudo` | 新用户密码输入时不显示，可能打错了 | `wsl --user root` 进入 root，`passwd 用户名` 重置 |
| `rm -rf /` | 系统被删光 | `/` 是根目录，包含了整个系统 | **永远不要执行 `rm -rf /` 或 `rm -rf /*`** |
| `rm -rf *` | 当前目录所有文件消失 | `*` 匹配所有文件 | 删除前用 `ls` 确认，尽量用具体路径 |
| vim 输完 `:wq` 还在编辑模式 | 命令没生效 | 必须先在命令模式下输入 `:wq`。如果还在编辑模式（左下角 `-- INSERT --`），按 `Esc` 退出编辑模式再输入 `:wq` | 确认左下角没有 `-- INSERT --`，然后输入 `:wq` |
| `sudo` 提示不在 sudoers 中 | 当前用户没有管理员权限 | WSL 安装时创建的用户默认有 sudo 权限。如果是新增用户，需要手动加入 sudo 组 | `usermod -aG sudo 用户名`（需要先用 root 登录） |
| `Permission denied` | 无法操作文件 | 权限不够 | 先 `ls -l` 看权限，然后用 `chmod` 改权限或 `sudo` 提权 |
| 脚本文件无法执行 | `./script.sh` 提示 Permission denied | 文件没有执行权限（`x`） | `chmod +x script.sh` 加执行权限 |
