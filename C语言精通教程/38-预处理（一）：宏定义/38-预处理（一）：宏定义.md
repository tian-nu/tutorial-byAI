## 我在做什么？

在你写的代码交给编译器之前，C 语言还有一个"幕后工作者"——*预处理器* *此术语见附录F*。它在编译之前对代码做**文本替换**。`#define` 是预处理器最常用的指令，它能定义常量、简化代码，但如果用不好也会制造难以排查的 bug。这一章你将学会：如何用 `#define` 定义常量，如何写宏函数，以及"括号地狱"和"副作用"两大经典陷阱。

读完本章你能：用 `#define` 定义符号常量，写出安全的多行宏函数，理解宏展开的括号地狱并避免它，区分宏函数和真实函数的适用场景，使用预定义宏了解编译环境。

## 一、什么是预处理？

### 1.1 预处理器在编译流程中的位置

```text
源文件 (.c)  →  预处理器  →  展开后的代码  →  编译器  →  目标文件 (.o)
               (cpp)                           (gcc)
```

预处理器在编译器之前工作，它只做**文本替换**，不理解 C 语言语法。你可以想象预处理器是一个"查找替换"机器人——它扫描源代码，找到所有 `#` 开头的指令，执行替换，然后把处理后的代码交给编译器。

### 1.2 演示：用 gcc -E 看到预处理结果

```bash
gcc -E source.c -o source.i
```

`-E` 选项告诉 gcc 只做预处理，不编译。`source.i` 就是预处理后的代码，你可以用文本编辑器打开，看到所有 `#include` 的文件内容被复制进来，所有 `#define` 被替换。

## 二、#define：文本替换

### 2.1 基本语法

```c
#define 标识符 替换文本
```

例如：

```c
#define PI 3.14159
#define MAX_SIZE 100
#define GREETING "Hello, World!"
```

当预处理器遇到 `PI`，就把它替换成 `3.14159`；遇到 `MAX_SIZE`，替换成 `100`；遇到 `GREETING`，替换成 `"Hello, World!"`。

```c
#include <stdio.h>

#define PI 3.14159
#define MAX_SIZE 100

int main() {
    double area = PI * 10 * 10;          // 变成: double area = 3.14159 * 10 * 10;
    int arr[MAX_SIZE];                   // 变成: int arr[100];
    printf("PI = %f, MAX_SIZE = %d\n", PI, MAX_SIZE);

    return 0;
}
```

### 2.2 宏的名字是"原样替换"

关键理解：`#define` 做的是**纯文本替换**，不是计算。看这个例子：

```c
#define THREE 1 + 2

int result = THREE * 3;
```

你以为 `result` 是 `(1+2)*3 = 9`？错！预处理器做的是：

```c
// 替换后
int result = 1 + 2 * 3;
```

乘法优先级高于加法，所以实际结果是 `1 + 6 = 7`。这就是为什么定义宏常量时务必加括号：

```c
#define THREE (1 + 2)  // 正确：加括号
```

### 2.3 宏命名惯例

C 语言中，宏常量通常用**全大写 + 下划线**命名，以区分变量：

```c
#define BUFFER_SIZE 4096
#define MAX_PATH_LENGTH 260
#define TRUE 1
#define FALSE 0
```

这不是语法要求，而是广泛遵守的编码惯例。看到全大写标识符，你就知道它是宏。

### 2.4 宏可以跨多行

如果替换文本太长，用 `\` 续行：

```c
#define LONG_STRING "This is a very long string \
that spans multiple lines."

#define HEADER \
    "================================\n" \
    "  Welcome to My Program        \n" \
    "================================\n"
```

注意 `\` 必须是该行的最后一个字符，后面不能有空格。

### 2.5 取消宏定义：#undef

```c
#define TEMP 100
// ... TEMP 在这里是 100 ...
#undef TEMP
// ... TEMP 在这里不再是宏，使用它会导致编译错误（除非有同名变量）...
```

`#undef` 在一个宏的定义不再需要时使用，或者你想重新定义它之前先取消旧定义。

## 三、宏函数：带参数的宏

### 3.1 基本语法

```c
#define 宏名(参数) 替换文本
```

例如：

```c
#define SQUARE(x) x * x
#define MAX(a, b) a > b ? a : b
```

使用时看起来像函数调用，但**本质仍然是文本替换**：

```c
int s = SQUARE(5);   // 替换为: int s = 5 * 5;
int m = MAX(3, 7);   // 替换为: int m = 3 > 7 ? 3 : 7;
```

### 3.2 宏函数的展开过程

```c
#define SQUARE(x) x * x

int result = SQUARE(3 + 2);
```

预处理器展开：

```c
int result = 3 + 2 * 3 + 2;
```

由于乘法优先级高于加法，实际计算结果是 `3 + 6 + 2 = 11`，而不是你期望的 `25`。这是宏函数最臭名昭著的陷阱——**括号地狱**。

---

## 四、括号地狱：宏函数的第一大陷阱

### 4.1 问题所在

```c
#define SQUARE(x) x * x

SQUARE(5)       → 5 * 5       → 25   ✓ 看起来正确
SQUARE(1+2)     → 1 + 2 * 1 + 2 → 5  ✗ 期望 9
SQUARE(a+b)     → a + b * a + b   ✗ 期望 (a+b)*(a+b)
```

问题的根源：宏参数 `x` 被替换为 `1+2`，但 `1+2` 不是一个整体，它是一串字符。预处理器不理解运算优先级，它只是机械地替换。

### 4.2 第一次修复：给参数加括号

```c
#define SQUARE(x) (x) * (x)

SQUARE(1+2)     → (1+2) * (1+2) → 9   ✓ 正确了！
```

这样够了吗？看下一个例子：

```c
int result = 100 / SQUARE(2);
```

展开：

```c
int result = 100 / (2) * (2);
```

`/` 和 `*` 同优先级，从左到右结合：`100 / 2 * 2 = 100`。但你可能期望的是 `100 / 4 = 25`。

### 4.3 第二次修复：给整个表达式加括号

```c
#define SQUARE(x) ((x) * (x))  // 里外都加括号！

100 / SQUARE(2)    → 100 / ((2) * (2))  → 100 / 4 → 25   ✓
100 / SQUARE(1+2)  → 100 / ((1+2) * (1+2)) → 100 / 9 → 11 ✓
```

### 4.4 通用规则

**定义宏函数时，给每个参数加上括号，给整个替换表达式也加上括号。**

```c
// ❌ 危险
#define MAX(a, b) a > b ? a : b

// ✅ 安全
#define MAX(a, b) ((a) > (b) ? (a) : (b))
```

### 4.5 多语句宏函数的陷阱

如果你需要宏函数包含多条语句：

```c
#define SWAP(a, b) tmp = a; a = b; b = tmp

// 在 if 语句中使用
if (x > y)
    SWAP(x, y);
else
    printf("no swap\n");
```

展开后：

```c
if (x > y)
    tmp = x; x = y; y = tmp;  // 只有 tmp = x 属于 if！
else
    printf("no swap\n");       // else 找不到匹配的 if，编译错误！
```

解决方案：用 `do { ... } while(0)` 包裹多条语句：

```c
#define SWAP(a, b, tmp) do { \
    tmp = a;                  \
    a = b;                    \
    b = tmp;                  \
} while(0)
```

`do { ... } while(0)` 是一个"假循环"——它只执行一次，但能把多条语句打包成一个整体。在 `if` 后面使用时，分号正好结束整个语句。

## 五、副作用：宏函数的第二大陷阱

### 5.1 什么是副作用

如果宏参数中包含**带副作用的表达式**（比如 `++`、`--` 等会改变变量值的操作），那么由于宏展开是纯文本替换，参数可能被计算多次。

```c
#define SQUARE(x) ((x) * (x))

int i = 5;
int result = SQUARE(i++);
```

展开：

```c
int result = ((i++) * (i++));
```

`i++` 被执行了两次！结果是 `5 * 6 = 30`（两次 `i++` 的求值顺序是未定义行为，不同编译器结果可能不同），而 `i` 变成了 `7`。你期望的是 `5 * 5 = 25`。

### 5.2 用 MAX 宏看副作用

```c
#define MAX(a, b) ((a) > (b) ? (a) : (b))

int x = 5, y = 10;
int m = MAX(x++, y++);
```

展开：

```c
int m = ((x++) > (y++) ? (x++) : (y++));
```

执行过程：
1. `x++` 返回 5，x 变成 6
2. `y++` 返回 10，y 变成 11
3. 比较：`5 > 10` 为假
4. 执行 `y++`：返回 11，y 变成 12
5. `m = 11`

你看，`x` 只自增了一次，`y` 自增了两次，`m` 的值是 `11` 而不是预期的 `10`。

### 5.3 如何避免副作用

**永远不要在宏函数参数中使用带副作用的表达式**。如果你需要自增/自减，先单独完成：

```c
// ❌
int m = MAX(x++, y++);

// ✅
int a = x++;
int b = y++;
int m = MAX(a, b);
```

或者说，当你需要处理有副作用的参数时，就应该用真正的函数，而不是宏。

## 六、宏函数 vs 函数

### 6.1 对比表

| 特性 | 宏函数 | 函数 |
|------|--------|------|
| 实现方式 | 文本替换，编译前展开 | 代码生成，运行时调用 |
| 参数类型 | 无类型检查，接受任意类型 | 有类型检查，参数必须匹配 |
| 求值次数 | 参数可能被求值多次（副作用） | 参数只求值一次 |
| 执行开销 | 无函数调用开销（调用/返回） | 有函数调用开销 |
| 调试 | 难以调试，调试器看不到宏 | 可以单步调试 |
| 代码体积 | 每次使用都展开，可能增大代码 | 只有一份代码，体积小 |
| 递归 | 不支持 | 支持 |

### 6.2 什么时候用宏函数

- **极短小、高频调用**的操作，如 `MAX`、`MIN`、`ABS`。
- 需要**类型无关**的操作（C 没有模板，宏可以接受任意类型）。
- 在旧代码中为了性能优化（现代编译器内联函数 `inline` 已基本取代这个用途）。

### 6.3 什么时候用函数

- 超过 3 行的逻辑。
- 参数包含副作用风险。
- 需要类型安全。
- 需要调试。

现代 C 语言中，推荐使用 `static inline` 函数替代宏函数：

```c
// 推荐：用 inline 函数替代宏
static inline int max_int(int a, int b) {
    return a > b ? a : b;
}
```

`inline` 函数既保留了宏的高效（无调用开销），又避免了宏的副作用和括号地狱。

## 七、预定义宏：编译器内置的宏

### 7.1 常用预定义宏

C 标准预定义了一些宏，你不需要 `#define` 就能使用：

| 宏 | 含义 | 示例值 |
|----|------|--------|
| `__FILE__` | 当前源文件名（字符串） | `"main.c"` |
| `__LINE__` | 当前行号（整数） | `42` |
| `__DATE__` | 编译日期（字符串） | `"Jun  7 2026"` |
| `__TIME__` | 编译时间（字符串） | `"14:30:00"` |
| `__STDC__` | 是否符合 ANSI C 标准 | `1` |
| `__STDC_VERSION__` | C 标准版本 | `201112L`（C11） |

### 7.2 实用示例：编译日志

```c
#include <stdio.h>

#define LOG(msg) printf("[%s:%d] %s\n", __FILE__, __LINE__, msg)

int main() {
    LOG("程序启动");
    int x = 42;
    LOG("x 已初始化");
    printf("x = %d\n", x);
    LOG("程序结束");

    return 0;
}
```

输出：

```
[main.c:7] 程序启动
[main.c:9] x 已初始化
x = 42
[main.c:11] 程序结束
```

### 7.3 打印编译信息

```c
#include <stdio.h>

int main() {
    printf("此程序由 %s 编译\n", __FILE__);
    printf("编译时间: %s %s\n", __DATE__, __TIME__);
    printf("C 标准版本: %ld\n", __STDC_VERSION__);

    return 0;
}
```

输出：

```
此程序由 main.c 编译
编译时间: Jun  7 2026 14:30:00
C 标准版本: 201112
```

### 7.4 断言宏：利用 __LINE__ 和 __FILE__

很多库的断言宏就利用了这两个预定义宏：

```c
#define MY_ASSERT(condition) \
    do { \
        if (!(condition)) { \
            fprintf(stderr, "断言失败: %s, 文件 %s, 第 %d 行\n", \
                    #condition, __FILE__, __LINE__); \
        } \
    } while(0)

int main() {
    int x = 5;
    MY_ASSERT(x > 0);   // 通过，不输出
    MY_ASSERT(x == 3);  // 失败，输出错误信息

    return 0;
}
```

输出：

```
断言失败: x == 3, 文件 main.c, 第 15 行
```

注意 `#condition` 是宏的**字符串化操作符**——它把宏参数变成字符串字面量。`#x` 展开为 `"x"`，`#(x > 0)` 展开为 `"x > 0"`。

## 八、# 和 ##：宏的两个特殊操作符

### 8.1 #：字符串化（Stringizing）

把宏参数转换为字符串字面量：

```c
#define STRINGIFY(x) #x

printf("%s\n", STRINGIFY(hello));        // 输出: hello
printf("%s\n", STRINGIFY(3.14));         // 输出: 3.14
printf("%s\n", STRINGIFY(1 + 2));        // 输出: 1 + 2（不是 3！）
```

### 8.2 ##：连接（Token Pasting）

把两个符号连接成一个：

```c
#define CONCAT(a, b) a ## b

int CONCAT(my, Var) = 42;    // 展开为: int myVar = 42;
int CONCAT(x, 1) = 10;       // 展开为: int x1 = 10;
int CONCAT(x, 2) = 20;       // 展开为: int x2 = 20;
```

### 8.3 实用示例：自动生成变量名

```c
#include <stdio.h>

#define MAKE_VAR(name, num) int name##num

#define PRINT_VAR(name, num) printf(#name #num " = %d\n", name##num)

int main() {
    MAKE_VAR(counter, 1) = 100;   // int counter1 = 100;
    MAKE_VAR(counter, 2) = 200;   // int counter2 = 200;
    MAKE_VAR(counter, 3) = 300;   // int counter3 = 300;

    PRINT_VAR(counter, 1);        // printf("counter1 = %d\n", counter1);
    PRINT_VAR(counter, 2);
    PRINT_VAR(counter, 3);

    return 0;
}
```

输出：

```
counter1 = 100
counter2 = 200
counter3 = 300
```

## 九、常见错误

### 9.1 宏定义中忘记括号

```c
// ❌
#define DOUBLE(x) x + x
int r = 3 * DOUBLE(2);  // 展开: 3 * 2 + 2 = 8，期望 3 * 4 = 12

// ✅
#define DOUBLE(x) ((x) + (x))
```

### 9.2 宏定义末尾加分号

```c
// ❌
#define PI 3.14159;
int main() {
    double x = PI * 2;  // 展开: double x = 3.14159; * 2;  → 编译错误！
}

// ✅
#define PI 3.14159
```

### 9.3 宏参数中的副作用

```c
// ❌
#define ABS(x) ((x) < 0 ? -(x) : (x))
int result = ABS(a++);  // a++ 被求值两次！
```

### 9.4 宏函数名和括号之间加空格

```c
#define SQUARE (x) ((x) * (x))  // ❌ 括号和 SQUARE 之间有空格
// 这定义的是 SQUARE 替换为 (x) ((x) * (x))，不是带参数的宏！
```

### 9.5 把宏当函数调试

```c
#define MAX(a, b) ((a) > (b) ? (a) : (b))

// 调试器里你无法"步入" MAX，因为它不是函数，只是一段文本
// 想看展开结果，用 gcc -E
```

## 十、我做得对不对？

### 10.1 验证宏展开

创建以下程序，用 `gcc -E` 查看预处理结果：

```c
#include <stdio.h>

#define PI 3.14159
#define SQUARE(x) ((x) * (x))
#define MAX(a, b) ((a) > (b) ? (a) : (b))

int main() {
    double area = PI * SQUARE(5);
    int m = MAX(10, 20);
    printf("area = %f, m = %d\n", area, m);
    return 0;
}
```

运行：

```bash
gcc -E main.c -o main.i
```

打开 `main.i`，找到 `main` 函数部分，你会看到：

```c
int main() {
    double area = 3.14159 * ((5) * (5));
    int m = ((10) > (20) ? (10) : (20));
    printf("area = %f, m = %d\n", area, m);
    return 0;
}
```

### 10.2 验证括号地狱

```c
#include <stdio.h>

// 有问题的宏
#define BAD_SQUARE(x) x * x

// 正确的宏
#define GOOD_SQUARE(x) ((x) * (x))

int main() {
    printf("BAD_SQUARE(1+2)  = %d (期望 9)\n", BAD_SQUARE(1+2));
    printf("GOOD_SQUARE(1+2) = %d (期望 9)\n", GOOD_SQUARE(1+2));

    printf("100 / BAD_SQUARE(2)  = %d (期望 25)\n", 100 / BAD_SQUARE(2));
    printf("100 / GOOD_SQUARE(2) = %d (期望 25)\n", 100 / GOOD_SQUARE(2));

    return 0;
}
```

预期输出：

```
BAD_SQUARE(1+2)  = 5 (期望 9)
GOOD_SQUARE(1+2) = 9 (期望 9)
100 / BAD_SQUARE(2)  = 100 (期望 25)
100 / GOOD_SQUARE(2) = 25 (期望 25)
```

### 10.3 验证副作用

```c
#include <stdio.h>

#define MAX(a, b) ((a) > (b) ? (a) : (b))

int main() {
    int x = 5, y = 10;
    int m = MAX(x++, y++);
    printf("m = %d\n", m);
    printf("x = %d (期望 6)\n", x);
    printf("y = %d (期望 11，但实际可能更大)\n", y);
    return 0;
}
```

运行后观察 `y` 的值——它可能被自增了两次，而不是一次。

## 十一、不对怎么办？

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| 编译错误：`expected ';'` 在宏附近 | 宏定义末尾多加了分号 | 去掉 `#define` 行末尾的分号 |
| 宏展开后结果不符合预期 | 参数缺少括号或表达式缺少括号 | 给每个参数加括号，整个表达式也加括号 |
| 变量值异常变化 | 宏参数中有 `++`/`--` 等副作用 | 避免在宏参数中使用副作用表达式 |
| if 语句后面的宏不按预期执行 | 多语句宏未用 `do { } while(0)` 包裹 | 用 `do { ... } while(0)` 包裹多语句宏 |
| 宏函数定义后无法使用 | 宏名和括号之间有空格 | 去掉空格：`#define FOO(x)` 而非 `#define FOO (x)` |
| 调试时看不到宏展开 | 宏在预处理阶段已被替换 | 用 `gcc -E` 查看预处理结果 |
| 大项目中宏名冲突 | 多个头文件定义了同名宏 | 先用 `#undef` 取消，再重新定义；或用命名空间前缀 |

| #define 常量 | `#define PI 3.14159`，纯文本替换，全大写命名 |
| 宏函数 | `#define MAX(a,b) ((a)>(b)?(a):(b))`，参数和整体都要加括号 |
| 括号地狱 | `SQUARE(1+2)` 展开为 `1+2*1+2`，必须写成 `((x)*(x))` |
| 副作用 | 宏参数可能被求值多次，不要传 `++`/`--` 等表达式 |
| do-while(0) | 包裹多语句宏，确保在 if/else 中正确运行 |
| # 操作符 | 字符串化参数，`#x` → `"x"` |
| ## 操作符 | 连接符号，`a##b` → `ab` |
| 预定义宏 | `__FILE__`、`__LINE__`、`__DATE__`、`__TIME__` |
| 宏 vs 函数 | 宏无类型检查、无调用开销、有副作用；函数反之 |
| inline 函数 | 现代 C 推荐用 `static inline` 替代宏函数 |
