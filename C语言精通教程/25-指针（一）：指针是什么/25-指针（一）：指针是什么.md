## 我在做什么？

指针是 C 语言最核心、最强大、也最让人害怕的概念。如果你问十个程序员"学 C 语言最难的是什么"，九个会回答"指针"。

但指针其实不神秘。它就是一个**存储地址的变量**。就像你家的门牌号告诉你"家在哪里"，指针告诉你"数据在哪里"。读完这一章，你能理解指针的本质，会用 `&` 取地址和 `*` 解引用，理解 `NULL` 空指针的作用，以及为什么指针是 C 语言强大之源。

## 一、指针是什么？

### 1.1 门牌号比喻

想象你住在一个小区里，每栋楼都有一个门牌号：

```text
小区（内存）：
┌─────────────────────────────────────────────────┐
│ 门牌号 1001 │ 门牌号 1002 │ 门牌号 1003 │ ...   │
│ ┌────────┐  │ ┌────────┐  │ ┌────────┐  │       │
│ │ 张三   │  │ │ 李四   │  │ │ 王五   │  │       │
│ │ 25岁   │  │ │ 30岁   │  │ │ 28岁   │  │       │
│ └────────┘  │ └────────┘  │ └────────┘  │       │
└─────────────────────────────────────────────────┘
```

- 门牌号 = 内存地址
- 房间里的内容 = 变量的值
- **指针 = 一张写着门牌号的纸条**

```c
int person = 25;      // 这个人（变量）住在内存里，值为 25
int *p = &person;     // p 是一张纸条，写着这个人的门牌号（地址）
```

`&` 是"取地址"运算符——问"这个变量住哪个门牌号？"。
`*` 是"解引用"运算符——"按门牌号去敲门，看看里面是谁？"

### 1.2 第一个指针程序

```c
#include <stdio.h>

int main() {
    int a = 42;
    int *p = &a;          // p 是一个指针，存储 a 的地址

    printf("a 的值：%d\n", a);         // 输出 42
    printf("a 的地址：%p\n", &a);      // 输出类似 0x7ffd12345678
    printf("p 的值（即 a 的地址）：%p\n", p);
    printf("p 指向的值：%d\n", *p);    // 输出 42

    return 0;
}
```

输出示例：
```text
a 的值：42
a 的地址：0x7ffd12345678
p 的值（即 a 的地址）：0x7ffd12345678
p 指向的值：42
```

### 1.3 图示

```text
变量 a：                     指针 p：
┌──────────────┐            ┌──────────────────┐
│ 地址：0x100  │            │ 地址：0x200      │
│ 值：42       │ ←───────── │ 值：0x100        │
└──────────────┘            └──────────────────┘
      ↑                            │
      │                            │
      └────────────────────────────┘
        p 存着 a 的地址，所以 p "指向" a
```

## 二、声明指针

### 2.1 语法

```c
int *p;        // p 是一个指向 int 的指针
double *q;     // q 是一个指向 double 的指针
char *r;       // r 是一个指向 char 的指针
```

`*` 靠近谁都可以，功能一样：

```c
int *p;    // 推荐：强调 p 是指针类型
int* p;    // 也可以：强调"指向 int 的指针"
int * p;   // 也可以：但少见
```

**陷阱**：一行声明多个变量时，`*` 只作用于它右边的变量名：

```c
int *p, q;   // p 是指针，但 q 是普通的 int！不是指针！
```

所以推荐每行只声明一个指针变量。

### 2.2 指针类型为什么重要

指针不仅存地址，还存"这个地址上存的是什么类型"。类型决定了**解引用时读几个字节、怎么解释这些字节**。

```c
int a = 0x12345678;
int *p = &a;       // p 知道：解引用时读 4 个字节，按 int 解释
char *q = (char*)&a; // q 也指向同一个地址，但 q 只读 1 个字节
```

```text
a 在内存中（小端序）：
地址：  0x100   0x101   0x102   0x103
      ┌──────┬──────┬──────┬──────┐
      │ 0x78 │ 0x56 │ 0x34 │ 0x12 │
      └──────┴──────┴──────┴──────┘
        ↑                 ↑
        q 读这 1 字节     p 读这 4 字节
        *q = 0x78         *p = 0x12345678
```

## 三、`&` 和 `*`：一对互逆操作

### 3.1 `&`：取地址

```c
int a = 10;
int *p = &a;   // &a 返回 a 的地址，类型是 int*
```

`&` 后面只能跟变量，不能跟表达式或常量：

```c
int *p = &(a + 1);  // 错误！a+1 是一个临时值，没有地址
int *q = &10;       // 错误！10 是常量，没有地址
```

### 3.2 `*`：解引用

```c
int a = 10;
int *p = &a;
*p = 20;        // 通过 p 修改 a 的值
printf("%d\n", a);  // 输出 20
```

### 3.3 `&` 和 `*` 互逆

```c
int a = 10;
int *p = &a;     // p  = &a
printf("%d\n", *(&a));  // *(&a) = a = 10
printf("%d\n", *p);      // *p = a = 10
```

```text
&a ──→ 取 a 的地址 → 得到一个指针
*指针 ──→ 取指针指向的值 → 得到原来的变量

所以 *(&a) = a
```

## 四、通过指针修改外部的值

回到第 19 章留下的悬念：怎么在函数内修改外部变量？

```c
#include <stdio.h>

void double_it(int *p) {
    *p = *p * 2;    // 通过指针修改 p 指向的变量
}

int main() {
    int a = 10;
    printf("调用前：a = %d\n", a);   // 10
    double_it(&a);                    // 传 a 的地址
    printf("调用后：a = %d\n", a);   // 20
    return 0;
}
```

```text
调用 double_it(&a) 时：

main 中的 a               double_it 中的 p
┌──────────────┐         ┌──────────────────┐
│ 地址：0x100  │         │ 地址：0x200      │
│ 值：10       │ ←────── │ 值：0x100        │
│      ↓       │         └──────────────────┘
│  *p = 20     │               p 指向 a
│ 值：20       │         所以 *p 就是 a
└──────────────┘
```

这就是指针的第一个真正用途：**让函数能修改外部变量**。

## 五、NULL 空指针

### 5.1 什么是 NULL

*NULL* *此术语见附录F* 是一个特殊的指针值，表示"不指向任何地方"。它通常定义为 `(void*)0`。

```c
int *p = NULL;   // p 不指向任何有效内存
```

### 5.2 为什么需要 NULL

NULL 有三个重要作用：

**作用一：初始化指针**。未初始化的指针可能指向任何地方，访问它会产生不可预测的后果：

```c
int *p;         // 未初始化，p 的值是随机的
*p = 10;        // 危险！写到了随机内存地址
```

```c
int *p = NULL;  // 初始化为 NULL
// 在解引用前检查
if (p != NULL) {
    *p = 10;    // 安全
}
```

**作用二：函数返回"没找到"的信号**。就像 `strchr` 找不到字符时返回 NULL。

**作用三：标记数组/链表的结束**。

### 5.3 解引用 NULL 的后果

```c
int *p = NULL;
*p = 10;   // 段错误！程序崩溃
```

NULL 指向的地址（通常是 0）被操作系统保护，任何访问都会触发 *段错误* *此术语见附录F*。

## 六、指针的大小

### 6.1 指针大小取决于系统

```c
#include <stdio.h>

int main() {
    printf("int* 的大小：%zu 字节\n", sizeof(int*));
    printf("char* 的大小：%zu 字节\n", sizeof(char*));
    printf("double* 的大小：%zu 字节\n", sizeof(double*));
    printf("void* 的大小：%zu 字节\n", sizeof(void*));
    return 0;
}
```

输出（64 位系统）：
```text
int* 的大小：8 字节
char* 的大小：8 字节
double* 的大小：8 字节
void* 的大小：8 字节
```

**所有指针类型大小相同**——因为在同一系统上，地址总线的宽度是固定的（32 位系统 4 字节，64 位系统 8 字节）。

### 6.2 指针类型不影响指针自身大小

```text
32 位系统：地址范围 0x00000000 ~ 0xFFFFFFFF，需要 4 字节存地址
64 位系统：地址范围 0x0000000000000000 ~ 0xFFFFFFFFFFFFFFFF，需要 8 字节存地址
```

## 七、`void*` 通用指针

### 7.1 可以指向任何类型

```c
void *p;           // 通用指针，可以指向任何类型
int a = 10;
double b = 3.14;

p = &a;            // 指向 int
p = &b;            // 指向 double，没问题
```

### 7.2 不能直接解引用

```c
void *p = &a;
// printf("%d\n", *p);   // 错误！编译器不知道要读几个字节
printf("%d\n", *(int*)p);  // 正确：先转换为具体类型指针再解引用
```

`void*` 就像一个"只知道门牌号，不知道里面住的是谁"的纸条。

## 八、常见错误与排错

### 8.1 未初始化指针

❌ 错误示例：
```c
int *p;
*p = 10;         // p 指向随机地址，危险！
```

✅ 正确示例：
```c
int *p = NULL;   // 或 int *p = &某个变量;
// 使用前检查
if (p != NULL) {
    *p = 10;
}
```

### 8.2 解引用前不检查 NULL

❌ 错误示例：
```c
char *result = strchr(str, 'x');
printf("%c\n", *result);   // 如果 result 是 NULL，崩溃！
```

✅ 正确示例：
```c
char *result = strchr(str, 'x');
if (result != NULL) {
    printf("%c\n", *result);
} else {
    printf("没找到\n");
}
```

### 8.3 混淆 `*` 的两种含义

```c
int *p;      // 这里的 * 是类型声明的一部分：p 是指向 int 的指针
*p = 10;     // 这里的 * 是解引用运算符：取 p 指向的值
```

声明中的 `*` 和表达式中的 `*` 是两回事。

### 8.4 指针类型不匹配

❌ 错误示例：
```c
int a = 10;
double *p = &a;   // 警告：int* 赋值给 double*
```

✅ 正确示例：
```c
int a = 10;
int *p = &a;      // 类型匹配
```

## 我做得对不对？

用以下程序验证：

```c
#include <stdio.h>

void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main() {
    // 测试1：基本指针操作
    int x = 10;
    int *p = &x;
    printf("测试1：*p = %d (应为 10)\n", *p);
    *p = 20;
    printf("测试1：x = %d (应为 20)\n", x);

    // 测试2：swap 函数
    int m = 5, n = 8;
    swap(&m, &n);
    printf("测试2：m = %d (应为 8), n = %d (应为 5)\n", m, n);

    // 测试3：NULL 指针
    int *np = NULL;
    printf("测试3：np == NULL ? %d (应为 1)\n", np == NULL);

    // 测试4：指针大小
    printf("测试4：sizeof(int*) = %zu (应为 8，64位系统)\n", sizeof(int*));
    printf("测试4：sizeof(char*) = %zu (应为 8，64位系统)\n", sizeof(char*));

    return 0;
}
```

编译运行：
```bash
$ gcc -Wall -o test test.c
$ ./test
测试1：*p = 10 (应为 10)
测试1：x = 20 (应为 20)
测试2：m = 8 (应为 8), n = 5 (应为 5)
测试3：np == NULL ? 1 (应为 1)
测试4：sizeof(int*) = 8 (应为 8，64位系统)
测试4：sizeof(char*) = 8 (应为 8，64位系统)
```

## 不对怎么办？

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| 段错误（Segmentation Fault） | 解引用了 NULL 或未初始化的指针 | 初始化指针，解引用前检查 NULL |
| 程序输出随机值 | 指针未初始化，指向了随机地址 | 总是初始化指针 |
| 编译警告 `incompatible pointer type` | 指针类型不匹配 | 确保指针类型和指向的变量类型一致 |
| 修改指针指向的值，原变量没变 | 修改的是指针本身，不是解引用 | 检查是否写了 `p = ...` 而不是 `*p = ...` |
| `*p` 打印出的值和预期不同 | 指针类型错误，读的字节数不对 | 检查指针类型是否匹配 |

## 本章小结

| 知识点 | 一句话总结 |
|--------|-----------|
| 指针 | 存储内存地址的变量 |
| `&` 取地址 | 获取变量的内存地址，返回指针 |
| `*` 解引用 | 通过指针访问其指向的值 |
| 指针声明 | `类型 *变量名`，如 `int *p` |
| 指针类型 | 决定解引用时读几个字节、怎么解释 |
| NULL | 空指针，表示不指向任何有效内存 |
| 指针大小 | 32 位系统 4 字节，64 位系统 8 字节，所有类型指针大小相同 |
| `void*` | 通用指针，可以指向任何类型，但不能直接解引用 |
| 指针的核心用途 | 让函数能修改外部变量，高效传递大数据 |
