## 看懂 ≠ 会写 ≠ 能排错

恭喜你，读完了编程基本概念篇的全部9章。现在你面对一段AI生成的Python代码，应该不再是"天书"了。

但这里有一个重要的认知，我希望你在进入下一篇之前就想清楚：

```
看懂代码  ≠  会写代码  ≠  能排错
```

这三个能力是**递进**的，不是并列的：

| 能力 | 含义 | 你能做到吗 |
|------|------|-----------|
| **看懂** | 读一段代码，大致知道它在干什么 | 本篇的目标，读完你应该能了 |
| **会写** | 自己在空白文件里写出代码 | 读完本篇还做不到，而且**不需要** |
| **能排错** | 代码出问题了，能找到原因并修好 | 需要经验积累，但AI可以帮你 |

## AI编程时代的能力模型（优先级排序）

在传统编程时代，能力模型是这样的：

```
传统编程：
  ① 会写代码  →  ② 会排错  →  ③ 会读代码  →  ④ 会说需求
  （最重要的能力）（次要能力）（有了前两个自然就会）（基本没人提）
```

在AI编程时代，这个顺序**完全反过来了**：

```
AI编程时代：
  ① 描述需求  →  ② 阅读代码  →  ③ 验证逻辑  →  ④ 定位问题
  ↑ 最重要       ↑ 第二重要     ↑ 第三重要     ↑ 进阶能力
```

**解释一下每个能力**：

### ① 描述需求（最重要）

你能不能用中文（或英文）把你要的东西说清楚？这是AI编程最核心的能力。

- ❌ "帮我写个登录" → AI不知道你要什么
- ✅ "帮我写一个Python Flask的用户登录接口，用户名和密码登录，密码用bcrypt加密，登录成功返回JSON格式的token"

**这个能力，第一篇（Prompt Engineering）已经教过你了。**

### ② 阅读代码（第二重要）

AI写了代码，你能不能看懂？这是**本篇9章的核心目标**。

- 看到 `name = "张三"`，你知道这是字符串变量
- 看到 `def calculate(x, y):`，你知道这是定义了一个函数
- 看到 `for item in items:`，你知道这是循环遍历列表
- 看到 `response.json()`，你知道这是在解析API返回的JSON

**你现在应该能做到了。**

### ③ 验证逻辑（第三重要）

AI写的代码，你怎么知道它写得对不对？"能跑"不等于"正确"。

```python
# AI写了这段代码，说"计算平均分"
def average(scores):
    return sum(scores) / len(scores)
```

- 你输入 `[80, 90, 100]`，得到 `90.0` ✅ 正确
- 你输入 `[]`（空列表），会怎样？❌ 报错！除零错误！

**你能发现第二个场景的问题吗？** 这就是验证——不是看代码能不能跑，而是看它在各种情况下对不对。

## 能力自测：给你一段AI代码，你能答出这三问吗？

下面是一段AI生成的代码。读一遍，然后回答三个问题：

> 以下代码由AI生成：

```python
def process_orders(orders):
    result = []
    for order in orders:
        if order["status"] == "paid":
            total = order["price"] * order["quantity"]
            result.append({
                "id": order["id"],
                "total": total
            })
    return result
```

**三问**：

1. **它做了什么？**（阅读代码）—— 你能用大白话说出来吗？
2. **哪里可能出错？**（验证逻辑）—— 如果 `order` 字典里没有 `"price"` 这个键，会怎样？
3. **怎么改？**（改需求）—— 如果要求"只返回总额超过100的订单"，你要怎么跟AI说？

**答案（先自己思考，再看）：**

> 1. 它接收一个订单列表，筛选出状态为"paid"（已支付）的订单，计算每个订单的总价（单价×数量），返回一个包含订单ID和总价的新列表。
> 2. 如果某个订单字典里缺少 `"price"` 或 `"quantity"` 键，程序会报 `KeyError`。另外如果 `orders` 是空列表，返回空列表，这个倒没问题。
> 3. 对AI说："在 `result.append` 那行之前加一个判断：如果 `total > 100`，才加进去。低于等于100的跳过。"

## 一句话总结

> **AI时代，"会和AI合作"比"会自己写代码"更重要。**

你不需要成为一个能从零手写代码的程序员。你需要成为一个：
- 能说清楚需求的人
- 能看懂AI写了什么的人
- 能判断AI写得对不对的人
- 能在AI写错时告诉它怎么改的人

## 自测

1. AI编程时代的四项核心能力，按优先级排序是什么？
2. 下面代码如果输入 `scores = []`（空列表），会出什么问题？
   ```python
   def average(scores):
       return sum(scores) / len(scores)
   ```
3. 用一句话总结：AI时代，什么比"会自己写代码"更重要？

> **答案**：
> 1. ①描述需求 ②阅读代码 ③验证逻辑 ④定位问题。
> 2. 会报 `ZeroDivisionError`（除零错误），因为 `len([])` 是0，任何数除以0都会报错。需要加判断：如果列表为空，返回0或提示。
> 3. "会和AI合作"比"会自己写代码"更重要。
