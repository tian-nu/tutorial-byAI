# 第71章 · 参数校验

> "你不会让客人不付钱就进餐厅吧？后端也一样——请求来了不等于请求是合法的。用户可能不小心把邮箱写成了'abc'，把年龄写成了'-5'，或者干脆性别字段是空的。参数校验就是把垃圾请求挡在门外的第一道防线。"

---

## 71.1 binding标签

Gin集成了 `go-playground/validator` 库，通过在结构体tag中写 `binding` 规则来实现参数校验。

### 基本校验规则

```go
type CreateUserRequest struct {
    Name     string `json:"name" binding:"required"`
    Password string `json:"password" binding:"required,min=6,max=20"`
    Email    string `json:"email" binding:"required,email"`
    Age      int    `json:"age" binding:"required,min=1,max=150"`
    Phone    string `json:"phone" binding:"required,len=11"`
    Gender   string `json:"gender" binding:"required,oneof=male female other"`
    Website  string `json:"website" binding:"omitempty,url"`
}
```

### 常用binding规则速查

| 规则 | 含义 | 示例 |
|------|------|------|
| `required` | 必填，零值报错 | `binding:"required"` |
| `min=N` | 最小值（数字）或最小长度（字符串） | `binding:"min=6"` |
| `max=N` | 最大值或最大长度 | `binding:"max=100"` |
| `len=N` | 精确长度 | `binding:"len=11"` |
| `eq=N` | 等于某值 | `binding:"eq=1"` |
| `ne=N` | 不等于某值 | `binding:"ne=0"` |
| `gt=N` | 大于（greater than） | `binding:"gt=0"` |
| `gte=N` | 大于等于 | `binding:"gte=1"` |
| `lt=N` | 小于（less than） | `binding:"lt=100"` |
| `lte=N` | 小于等于 | `binding:"lte=99"` |
| `oneof=a b c` | 必须是枚举值之一 | `binding:"oneof=male female"` |
| `email` | 邮箱格式 | `binding:"email"` |
| `url` | URL格式 | `binding:"url"` |
| `ip` | IP地址 | `binding:"ip"` |
| `ipv4` | IPv4地址 | `binding:"ipv4"` |
| `alpha` | 纯字母 | `binding:"alpha"` |
| `alphanum` | 字母+数字 | `binding:"alphanum"` |
| `numeric` | 数字字符串 | `binding:"numeric"` |
| `omitempty` | 零值时跳过验证 | `binding:"omitempty,email"` |

### 组合校验

多个规则用逗号分隔：

```go
Password string `json:"password" binding:"required,min=8,max=32,alphanum"`
```

### 针对数字和字符串

同一个 `min` 规则，对数字和对字符串含义不同：

```go
Age  int    `json:"age" binding:"min=18"`        // 数字≥18
Name string `json:"name" binding:"min=2"`         // 字符串长度≥2
```

### 针对切片和Map

```go
Tags  []string          `json:"tags" binding:"required,min=1"`
Extra map[string]string `json:"extra" binding:"max=10"`
```

切片和Map的 `min`/`max` 检查的是元素数量。

## 71.2 自定义校验器

内置规则不够用？注册你自己的校验函数。

### 场景：校验手机号格式

```go
import (
    "regexp"

    "github.com/gin-gonic/gin/binding"
    "github.com/go-playground/validator/v10"
)

var phoneRegex = regexp.MustCompile(`^1[3-9]\d{9}$`)

func validatePhone(fl validator.FieldLevel) bool {
    phone, ok := fl.Field().Interface().(string)
    if !ok {
        return false
    }
    return phoneRegex.MatchString(phone)
}
```

### 注册自定义校验器

```go
if v, ok := binding.Validator.Engine().(*validator.Validate); ok {
    v.RegisterValidation("phone", validatePhone)
}
```

类型断言检查后，`v.RegisterValidation("phone", validatePhone)` 把函数注册为名为 `phone` 的规则。

### 使用

```go
type Request struct {
    Mobile string `json:"mobile" binding:"required,phone"`
}
```

现在 `binding:"phone"` 会调用你写的 `validatePhone` 函数。

### 初始化位置

自定义校验器的注册必须在 `r.Run()` 之前完成。通常在 `main` 函数的开头：

```go
func main() {
    initValidator()

    r := gin.Default()
    r.Run()
}

func initValidator() {
    if v, ok := binding.Validator.Engine().(*validator.Validate); ok {
        v.RegisterValidation("phone", validatePhone)
    }
}
```

## 71.3 错误信息翻译成中文

默认的校验错误返回的是英文，比如：

```
Key: 'Request.Name' Error:Field validation for 'Name' failed on the 'required' tag
```

这对调用方非常不友好。我们需要把它翻译成中文。

### 提取校验错误

```go
r.POST("/users", func(c *gin.Context) {
    var req CreateUserRequest

    if err := c.ShouldBindJSON(&req); err != nil {
        errs, ok := err.(validator.ValidationErrors)
        if !ok {
            c.JSON(400, gin.H{"msg": "参数错误"})
            return
        }

        msg := translateErrors(errs)
        c.JSON(400, gin.H{"msg": msg})
        return
    }
    c.JSON(200, gin.H{"result": "ok"})
})
```

### 翻译函数

```go
func translateErrors(errs validator.ValidationErrors) string {
    for _, err := range errs {
        switch err.Tag() {
        case "required":
            return fmt.Sprintf("%s 是必填项", err.Field())
        case "min":
            return fmt.Sprintf("%s 不能小于 %s", err.Field(), err.Param())
        case "max":
            return fmt.Sprintf("%s 不能大于 %s", err.Field(), err.Param())
        case "len":
            return fmt.Sprintf("%s 长度必须为 %s", err.Field(), err.Param())
        case "email":
            return fmt.Sprintf("%s 格式不正确", err.Field())
        case "oneof":
            return fmt.Sprintf("%s 必须是 [%s] 之一", err.Field(), err.Param())
        default:
            return fmt.Sprintf("%s 验证失败: %s", err.Field(), err.Tag())
        }
    }
    return "参数验证失败"
}
```

`err.Field()` 返回结构体的字段名（英文），`err.Tag()` 返回校验规则名（如 `required`），`err.Param()` 返回规则的参数（如 `min=6` 中的 `6`）。

### 中文字段名

用结构体tag自定义字段的中文名：

```go
type CreateUserRequest struct {
    Name     string `json:"name" binding:"required" zh:"用户姓名"`
    Password string `json:"password" binding:"required,min=6" zh:"密码"`
}
```

在翻译函数中读取 `zh` tag：

```go
func getFieldName(err validator.FieldError) string {
    return err.StructField()
}
```

完整的中文翻译方案请参考本章的 `🤔 想多一点` 部分。

## 71.4 ShouldBind vs MustBind

Gin提供了两组绑定函数：

| Should系列 | Must系列 | 区别 |
|-----------|---------|------|
| `c.ShouldBindJSON(&obj)` | `c.BindJSON(&obj)` | BindJSON出错自动返回400+Abort |
| `c.ShouldBind(&obj)` | `c.Bind(&obj)` | Bind自动检测Content-Type |
| `c.ShouldBindQuery(&obj)` | `c.BindQuery(&obj)` | 绑定Query参数 |
| `c.ShouldBindUri(&obj)` | `c.BindUri(&obj)` | 绑定路径参数 |

### 为什么推荐Should系列

```go
r.POST("/users", func(c *gin.Context) {
    var req Request

    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(400, gin.H{
            "code": 10400,
            "msg":  translateErrors(err.(validator.ValidationErrors)),
        })
        return
    }

    c.JSON(200, Response{Code: 0, Msg: "success", Data: req})
})
```

你可以完全控制错误响应格式——业务错误码、中文错误提示、统一响应结构。

如果用 `c.BindJSON`，Gin帮你返回的400响应格式跟你自己的统一格式不一致，调用方会困惑。

### ShouldBindQuery：绑定查询参数

```go
type SearchRequest struct {
    Keyword string `form:"keyword" binding:"required"`
    Page    int    `form:"page" binding:"min=1"`
    Size    int    `form:"size" binding:"min=1,max=100"`
}

r.GET("/search", func(c *gin.Context) {
    var req SearchRequest
    if err := c.ShouldBindQuery(&req); err != nil {
        c.JSON(400, gin.H{"msg": "参数错误"})
        return
    }
    c.JSON(200, req)
})
```

注意这里用的是 `form` tag（不是 `json`），因为查询参数是通过 `?key=value` 形式传递的。

### ShouldBindUri：绑定路径参数

```go
type UserRequest struct {
    ID string `uri:"id" binding:"required,uuid"`
}

r.GET("/users/:id", func(c *gin.Context) {
    var req UserRequest
    if err := c.ShouldBindUri(&req); err != nil {
        c.JSON(400, gin.H{"msg": "ID格式错误"})
        return
    }
    c.JSON(200, gin.H{"id": req.ID})
})
```

使用 `uri` tag，校验规则同样生效。

## 🤔 想多一点：全局错误翻译器

上面的翻译函数写得简单，但生产项目中需要一个**完整的英文→中文错误翻译系统**。以下是一个生产级的方案：

```go
import (
    "fmt"
    "reflect"
    "strings"

    "github.com/gin-gonic/gin/binding"
    "github.com/go-playground/locales/zh"
    ut "github.com/go-playground/universal-translator"
    "github.com/go-playground/validator/v10"
    zhTranslations "github.com/go-playground/validator/v10/translations/zh"
)

var trans ut.Translator

func InitTrans() {
    zhLocale := zh.New()
    uni := ut.New(zhLocale, zhLocale)

    trans, _ = uni.GetTranslator("zh")

    if v, ok := binding.Validator.Engine().(*validator.Validate); ok {
        zhTranslations.RegisterDefaultTranslations(v, trans)
    }
}

func Translate(err error) string {
    errs, ok := err.(validator.ValidationErrors)
    if !ok {
        return err.Error()
    }

    var msgs []string
    for _, e := range errs {
        msgs = append(msgs, e.Translate(trans))
    }
    return strings.Join(msgs, "; ")
}
```

`zhTranslations.RegisterDefaultTranslations` 自动注册了所有内置规则的中文翻译。`required` 对应"是必填字段"，`min` 对应"长度不能小于..."。自定义字段名通过 `RegisterTagNameFunc` 让翻译器读JSON tag而非字段名。

## 本章小结

| 知识点 | 要点 |
|--------|------|
| binding标签 | 结构体tag中写 `binding:"required,min=6"` |
| 常用规则 | required / min / max / len / email / url / oneof / omitempty |
| 结构体绑定 | `c.ShouldBindJSON(&req)` 自动校验 |
| 自定义校验器 | `v.RegisterValidation("name", fn)` |
| 校验函数签名 | `func(fl validator.FieldLevel) bool` |
| 错误翻译 | 遍历 `validator.ValidationErrors`，映射tag到中文文案 |
| ShouldBindQuery | 绑定Query参数，用 `form` tag |
| ShouldBindUri | 绑定路径参数，用 `uri` tag |
| Should vs Must | Should返回error（推荐），Must自动返回400+Abort |
| 生产级翻译 | `go-playground/validator/translations/zh` + `ut.Translator` |
