# 第47章 · 堆与优先队列

> "你要从10亿条日志中找出访问次数最多的100个IP地址——你怎么做？把10亿条全排序再取前100？那排序就要O(n log n)，内存都装不下。堆说：'把所有数据都给我，不需要排序，我帮你随时维护前100名。'每次新来一条日志，O(log 100)就能更新排名——这就是堆的威力。"

## 47.1 堆是什么

堆（Heap）是一种特殊的**完全二叉树**，同时满足**堆性质**：

- **最小堆**：每个节点的值 ≤ 所有子节点的值（根最小）
- **最大堆**：每个节点的值 ≥ 所有子节点的值（根最大）

```
最小堆：                     最大堆：
       [1]                       [9]
      /   \                     /   \
    [3]   [2]                [7]   [8]
    / \   / \                / \   / \
  [7] [4][5][6]           [3][1][4][2]

每一条从上到下的路径上，值都是递增的    每条路径上值都是递减的
```

**注意**：堆只保证父子之间的关系，不保证兄弟之间的关系。在最小堆中，[3]和[2]谁大谁小不确定——只要都大于它们的父节点[1]就行。

### 堆的数组存储

堆通常用切片（数组）存储，因为完全二叉树有个神奇的性质：

```
索引为 i 的节点：
  父节点索引 = (i - 1) / 2
  左子节点索引 = 2*i + 1
  右子节点索引 = 2*i + 2
```

```
        [1]           索引0
       /   \
    [3]   [2]       索引1  索引2
    / \   / \
  [7] [4][5][6]    索引3 4 5 6

数组: [1, 3, 2, 7, 4, 5, 6]
索引:  0  1  2  3  4  5  6

验证: 索引1(值3)的左子是索引3(值7)，右子是索引4(值4) ✓
      2*1+1=3, 2*1+2=4 ✓
```

这个性质让堆的实现极其简单——不需要指针，直接操作切片索引即可。

## 47.2 container/heap：Go的堆实现

Go标准库提供了`container/heap`包。使用它的前提是实现`heap.Interface`接口：

```go
type Interface interface {
    sort.Interface
    Push(x any)
    Pop() any
}

type Interface interface {
    Len() int
    Less(i, j int) bool
    Swap(i, j int)
}
```

所以你需要实现5个方法。来看一个最小堆的完整实现：

```go
type MinHeap []int

func (h MinHeap) Len() int {
    return len(h)
}

func (h MinHeap) Less(i, j int) bool {
    return h[i] < h[j]
}

func (h MinHeap) Swap(i, j int) {
    h[i], h[j] = h[j], h[i]
}

func (h *MinHeap) Push(x any) {
    *h = append(*h, x.(int))
}

func (h *MinHeap) Pop() any {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[:n-1]
    return x
}

func main() {
    h := &MinHeap{5, 3, 7, 1, 4, 2, 6}
    heap.Init(h)
    fmt.Println(*h)

    heap.Push(h, 0)
    fmt.Println(*h)

    for h.Len() > 0 {
        fmt.Print(heap.Pop(h), " ")
    }
}
```

输出：
```
[1 3 2 7 4 5 6]
[0 3 1 7 4 5 6 2]
0 1 2 3 4 5 6 7   ← 每次Pop拿出来的都是当前最小的
```

### Push和Pop的底层：上浮和下沉

堆的核心操作只有两个：

**上浮（sift-up）**：新元素加在数组末尾（最后一个位置），然后和父节点比较——如果比父节点小（最小堆），就交换位置。一直往上浮，直到不再比父节点小或到根节点。

```
添加 0 到最小堆 [1,3,2,7,4,5,6]:

         [1]               [1]              [0]
        /   \             /   \            /   \
      [3]   [2]        [3]   [2]       [3]   [1]
      / \   / \        / \   / \       / \   / \
    [7][4][5][6]    [7][4][5][6]   [7][4][5][2]
    /                  /               /
  [0]←新加          [0]→上浮         [6]
```

**下沉（sift-down）**：Pop时把最后一个元素移到根位置，然后和较小的子节点比较——如果比子节点大（最小堆），就交换位置。一直往下沉，直到不再比子节点大或到叶子。

```
Pop出0后，把最后一个元素[6]移到根：

         [6]               [1]              [1]
        /   \             /   \            /   \
      [3]   [1]        [3]   [6]       [3]   [2]
      / \   /          / \   /         / \   /
    [7][4][5]       [7][4][5]       [7][4][5]
                                        /
                                      [6]

移[6]到根 → 6>1,交换 → 6>2,交换 → 停止
```

上浮和下沉都是O(log n)——因为堆的高度是log n。

## 47.3 堆排序

堆排序利用堆的性质：**不断从最大堆中Pop堆顶，得到的就是从大到小的序列**。

```go
func heapSort(arr []int) []int {
    h := &MaxHeap{}
    *h = arr // 注意：*h 与 arr 共享底层数组，后续 Pop 会修改 arr 的内容
    heap.Init(h)

    result := make([]int, 0, len(arr))
    for h.Len() > 0 {
        result = append(result, heap.Pop(h).(int))
    }
    return result
}

type MaxHeap []int

func (h MaxHeap) Len() int           { return len(h) }
func (h MaxHeap) Less(i, j int) bool { return h[i] > h[j] }
func (h MaxHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }
func (h *MaxHeap) Push(x any)        { *h = append(*h, x.(int)) }
func (h *MaxHeap) Pop() any {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[:n-1]
    return x
}
```

堆排序的时间复杂度：建堆O(n) + 每次Pop O(log n) × n次 = O(n log n)。空间复杂度O(1)（原地排序）或O(n)（如果创建新切片）。

### 手写堆排序（不依赖container/heap）

面试中可能要求你手写堆排序，不依赖标准库：

```go
func heapSortManual(arr []int) []int {
    n := len(arr)

    for i := n/2 - 1; i >= 0; i-- {
        heapify(arr, n, i)
    }

    for i := n - 1; i > 0; i-- {
        arr[0], arr[i] = arr[i], arr[0]
        heapify(arr, i, 0)
    }
    return arr
}

func heapify(arr []int, n int, i int) {
    largest := i
    left := 2*i + 1
    right := 2*i + 2

    if left < n && arr[left] > arr[largest] {
        largest = left
    }
    if right < n && arr[right] > arr[largest] {
        largest = right
    }
    if largest != i {
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)
    }
}
```

步骤分解：
1. **建堆**：从最后一个非叶子节点开始（索引n/2-1），自底向上做heapify——确保每个子树都是最大堆
2. **排序**：把堆顶（最大值）和最后一个元素交换→堆的大小减1→对新的堆顶做heapify→重复

```
[4,10,3,5,1]  初始数组
     4
   /   \
  10    3
 /  \
5    1

建堆后：
     10
   /   \
  5     3
 /  \
4    1

取顶(10)→换到最后→heapify...
经过n次后：数组变成[1,3,4,5,10]（升序）
```

## 47.4 堆的四大经典应用

### 应用一：Top K问题

这是堆最经典的应用场景——从海量数据中找出最大（或最小）的K个元素。

**思路**：维护一个大小为K的最小堆。遍历所有数据——如果新元素比堆顶（K个中最小的）大，就弹出堆顶、插入新元素。遍历完之后，堆里就是最大的K个。

```go
func topK(arr []int, k int) []int {
    if k <= 0 || len(arr) == 0 {
        return nil
    }
    h := &MinHeap{}
    heap.Init(h)
    for i := 0; i < k && i < len(arr); i++ {
        heap.Push(h, arr[i])
    }

    for i := k; i < len(arr); i++ {
        if arr[i] > (*h)[0] {
            heap.Pop(h)
            heap.Push(h, arr[i])
        }
    }

    result := make([]int, k)
    for i := k - 1; i >= 0; i-- {
        result[i] = heap.Pop(h).(int)
    }
    return result
}

arr := []int{3, 1, 5, 7, 2, 4, 9, 6, 8}
fmt.Println(topK(arr, 3))
```

输出：`[7 8 9]`

复杂度分析：
- 如果全排序取前K：O(n log n)
- 用堆：O(n log K)
- 当n=10亿、K=100时，log n ≈ 30，log K ≈ 7——快了4倍以上，而且内存只占K个位置（不是n个）。

### 应用二：Dijkstra最短路径

Dijkstra算法的核心就是"每次选距离最短的未访问节点"。用最小堆来维护候选节点：

```go
type Item struct {
    node     int
    distance int
}

type PriorityQueue []Item

func (pq PriorityQueue) Len() int           { return len(pq) }
func (pq PriorityQueue) Less(i, j int) bool { return pq[i].distance < pq[j].distance }
func (pq PriorityQueue) Swap(i, j int)      { pq[i], pq[j] = pq[j], pq[i] }
func (pq *PriorityQueue) Push(x any)        { *pq = append(*pq, x.(Item)) }
func (pq *PriorityQueue) Pop() any {
    old := *pq
    n := len(old)
    x := old[n-1]
    *pq = old[:n-1]
    return x
}

func dijkstra(graph [][]int, start int) []int {
    n := len(graph)
    dist := make([]int, n)
    for i := range dist {
        dist[i] = 1<<31 - 1
    }
    dist[start] = 0

    pq := &PriorityQueue{{start, 0}}
    heap.Init(pq)

    for pq.Len() > 0 {
        item := heap.Pop(pq).(Item)
        u := item.node
        if item.distance > dist[u] {
            continue
        }
        for v := 0; v < n; v++ {
            if graph[u][v] > 0 {
                newDist := dist[u] + graph[u][v]
                if newDist < dist[v] {
                    dist[v] = newDist
                    heap.Push(pq, Item{v, newDist})
                }
            }
        }
    }
    return dist
}
```

每次从堆顶取最短距离节点，O(log n)；总共有n个节点需要取出，每条边可能触发一次Push——总复杂度O((n+m) log n)。

### 应用三：合并K个有序链表

有K个升序链表，合并成一个升序链表。

```go
func mergeKLists(lists []*ListNode) *ListNode {
    h := &MinHeapNode{}
    heap.Init(h)

    for _, list := range lists {
        if list != nil {
            heap.Push(h, list)
        }
    }

    dummy := &ListNode{}
    cur := dummy
    for h.Len() > 0 {
        node := heap.Pop(h).(*ListNode)
        cur.Next = node
        cur = cur.Next
        if node.Next != nil {
            heap.Push(h, node.Next)
        }
    }
    return dummy.Next
}

type MinHeapNode []*ListNode

func (h MinHeapNode) Len() int           { return len(h) }
func (h MinHeapNode) Less(i, j int) bool { return h[i].Val < h[j].Val }
func (h MinHeapNode) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }
func (h *MinHeapNode) Push(x any)        { *h = append(*h, x.(*ListNode)) }
func (h *MinHeapNode) Pop() any {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[:n-1]
    return x
}
```

思路：把所有链表的头节点放进最小堆。每次Pop出最小的节点接到结果上，然后把这个节点的Next推入堆。重复直到堆空。

时间复杂度：O(N log K)，其中N是所有节点的总数，K是链表数量。

### 应用四：中位数维护（数据流中位数）

持续接收数字流，随时查询中位数。**用两个堆**——一个最大堆存较小的一半，一个最小堆存较大的一半。

```go
type MedianFinder struct {
    small *MaxHeap
    large *MinHeap
}

func NewMedianFinder() *MedianFinder {
    return &MedianFinder{
        small: &MaxHeap{},
        large: &MinHeap{},
    }
}

func (mf *MedianFinder) AddNum(num int) {
    heap.Push(mf.small, num)

    if mf.small.Len() > 0 && mf.large.Len() > 0 && (*mf.small)[0] > (*mf.large)[0] {
        heap.Push(mf.large, heap.Pop(mf.small))
    }

    if mf.small.Len() > mf.large.Len()+1 {
        heap.Push(mf.large, heap.Pop(mf.small))
    }
    if mf.large.Len() > mf.small.Len() {
        heap.Push(mf.small, heap.Pop(mf.large))
    }
}

func (mf *MedianFinder) FindMedian() float64 {
    if mf.small.Len() > mf.large.Len() {
        return float64((*mf.small)[0])
    }
    return float64((*mf.small)[0]+(*mf.large)[0]) / 2.0
}
```

`small`（最大堆）存较小的一半，`large`（最小堆）存较大的一半。中位数要么是small的堆顶（奇数），要么是两堆顶平均（偶数）。每次插入O(log n)，查询O(1)。

## 🤔 想多一点：堆和排序有什么区别？

初学者容易混淆堆和排序。关键区别：

| | 堆 | 排序 |
|---|----|------|
| 目的 | 快速获取极值 | 全局有序 |
| 内部状态 | 部分有序（只保证父子） | 全局有序 |
| 获取第K元素 | O(K log n) | O(1)（已排序） |
| 维护代价 | 插入O(log n) | 插入O(n) |
| 适用场景 | 动态数据，关心极值 | 静态数据，需要全局有序 |

打个比方：排序是把全班同学按身高排成一列——你想问"第5矮的是谁？"可以直接看第5个。堆是一个"领奖台"——"谁是第一"一眼就能看到，但你想知道"第5名是谁"就没那么容易了。

**堆的核心价值在于"动态"**。数据在不停变化——新数据进来，旧数据过时——用排序要每次全排（O(n log n)），用堆只需每次调整（O(log n)）。这就是为什么Top K问题、优先级队列、事件调度器全都用堆。
