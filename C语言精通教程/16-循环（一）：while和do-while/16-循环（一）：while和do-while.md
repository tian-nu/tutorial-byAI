## 我在做什么？

上一章你学会了"判断"——在岔路口选一条路走。但有时候，你需要"反复走同一条路"。比如打印1到100的所有数字，你不可能写100行 `printf`。比如验证用户输入，密码不对就让他一直重试，直到正确为止。

这种"重复做一件事"的能力，在编程中叫**循环**。C语言有三种循环结构，本章讲前两种：
1. **`while` 循环**：最通用的循环，先判断再执行。
2. **`do-while` 循环**：先执行再判断，至少执行一次。
3. **死循环**：`while(1)` 永远跑下去（以及什么时候用它）。

## 16.1 `while` 循环：先判断，再执行

### 16.1.1 基本语法

```c
while (条件) {
    /* 循环体：条件为真时反复执行 */
}
```

**执行流程**：

```
    判断条件
      │
  ┌───┴───┐
  │ 真    │ 假
  │       │
  ▼       ▼
执行循环体   退出循环
  │
  └──→ 回到判断条件
```

**比喻**：`while` 循环就像自动门。你站在门口（条件），门检测到有人（条件为真），就打开让你进去。你进去后（执行循环体），自动门又回到了"检测"状态。如果你还在门口，它再打开。你走了（条件为假），门就关上了。

### 16.1.2 第一个例子：打印1到5

```c
#include <stdio.h>

int main(void)
{
    int count = 1;   /* 初始化 */

    while (count <= 5) {   /* 条件 */
        printf("%d\n", count);
        count = count + 1;   /* 更新 */
    }

    return 0;
}
```

输出：
```
1
2
3
4
5
```

**拆解三个要素**：

```
┌─────────────────────────────────────────────────┐
│              循环的三要素                          │
├─────────────────────────────────────────────────┤
│                                                  │
│  int count = 1;        ← 要素1：初始化            │
│  while (count <= 5) {  ← 要素2：条件（边界）       │
│      printf("%d\n", count);                      │
│      count = count + 1; ← 要素3：更新（向边界靠近） │
│  }                                               │
│                                                  │
└─────────────────────────────────────────────────┘
```

**三要素缺一不可**：
- 没有初始化 → 未定义行为（变量值随机）。
- 没有条件 → 不知道什么时候停。
- 没有更新 → 条件永远为真，死循环。

### 16.1.3 循环的四个阶段

让我们用 `count` 的值来追踪程序的执行：

```
count=1 → 判断 1<=5？是 → 打印1 → count=2
count=2 → 判断 2<=5？是 → 打印2 → count=3
count=3 → 判断 3<=5？是 → 打印3 → count=4
count=4 → 判断 4<=5？是 → 打印4 → count=5
count=5 → 判断 5<=5？是 → 打印5 → count=6
count=6 → 判断 6<=5？否 → 跳出循环
```

**比喻**：循环就像爬楼梯。
- 初始化：你站在第1级台阶。
- 条件：还没到顶楼（5楼）？继续爬。
- 循环体：每爬一级，喊一声"到了第X楼"。
- 更新：向上迈一步。
- 退出：到了6楼（超过5楼），不爬了。

### 16.1.4 累加：1到100的和

```c
#include <stdio.h>

int main(void)
{
    int i = 1;
    int sum = 0;

    while (i <= 100) {
        sum = sum + i;   /* 累加 */
        i = i + 1;       /* 或写成 i++ */
    }

    printf("1到100的和是：%d\n", sum);   /* 输出 5050 */

    return 0;
}
```

## 16.2 `do-while` 循环：先执行，再判断

### 16.2.1 基本语法

```c
do {
    /* 循环体：先执行一次 */
} while (条件);
```

**执行流程**：

```
    执行循环体（至少一次）
      │
      ▼
    判断条件
      │
  ┌───┴───┐
  │ 真    │ 假
  │       │
  ▼       ▼
回到循环体   退出循环
```

**比喻**：`do-while` 就像游乐园的过山车。你坐上去（先执行），到了终点，工作人员问："还想再坐一次吗？"（判断条件）。你说"想"，就再坐一圈。你说"不想"，就下车。但不管怎样，你至少坐了一次。

### 16.2.2 `while` vs `do-while` 的区别

这是两者最本质的区别：

```c
/* while：条件为假时，一次都不执行 */
int x = 10;
while (x < 5) {
    printf("这句话不会被打印\n");   /* 条件为假，跳过 */
}

/* do-while：条件为假时，至少执行一次 */
int y = 10;
do {
    printf("这句话会被打印一次\n");  /* 先执行，再判断 */
} while (y < 5);
```

### 16.2.3 典型应用：输入验证

`do-while` 最常见的用途是验证用户输入——至少问一次，如果输入不合法就继续问。

```c
#include <stdio.h>

int main(void)
{
    int score;

    do {
        printf("请输入分数（0-100）：");
        scanf("%d", &score);

        if (score < 0 || score > 100) {
            printf("分数无效，请重新输入！\n");
        }
    } while (score < 0 || score > 100);

    printf("你输入的分数是：%d\n", score);

    return 0;
}
```

**为什么用 `do-while` 而不是 `while`？** 因为"询问用户"这个操作至少要执行一次。如果用 `while`，你需要先给 `score` 一个初始值（比如 `-1`），然后 `while (score < 0 || score > 100)`——这也能工作，但 `do-while` 更自然。

## 16.3 死循环：`while(1)`

### 16.3.1 什么是死循环？

```c
while (1) {
    /* 永远执行，不会停 */
}
```

`1` 永远为真，所以这个循环永远不会自己停下来。它需要一个 `break` 语句（下一章讲）来手动跳出。

### 16.3.2 什么时候用死循环？

死循环不是bug——它是很多程序的核心设计模式。比如：

- **操作系统**：开机后就是一个大死循环，等待用户操作。
- **游戏主循环**：渲染画面 → 处理输入 → 更新状态 → 回到开头。
- **服务器**：监听端口 → 接收请求 → 处理 → 回到监听。

```c
#include <stdio.h>

int main(void)
{
    int password = 1234;
    int input;

    while (1) {   /* 死循环：一直问，直到密码正确 */
        printf("请输入密码：");
        scanf("%d", &input);

        if (input == password) {
            printf("密码正确！欢迎！\n");
            break;   /* 跳出死循环 */
        } else {
            printf("密码错误，请重试。\n");
        }
    }

    return 0;
}
```

### 16.3.3 不小心写出的死循环

```c
/* ❌ 忘记更新循环变量 */
int i = 1;
while (i <= 10) {
    printf("%d\n", i);
    /* 忘了 i = i + 1！i 永远是 1，条件永远为真 */
}

/* ❌ 条件永远为真 */
int x = 5;
while (x = 5) {   /* 赋值，不是比较！条件永远为 5（非0，真） */
    printf("被困住了...\n");
}

/* ❌ 分号陷阱 */
int i = 1;
while (i <= 10); {   /* 注意分号！循环体是空的 */
    printf("%d\n", i);
    i++;
}
/* 这相当于：
while (i <= 10) {
    ;  // 空循环体，什么都不做，i 永远不变
}
// 下面的花括号是一个独立的代码块，只执行一次
*/
```

## 16.4 完整示例：猜数字游戏

```c
#include <stdio.h>
#include <stdlib.h>
#include <time.h>

int main(void)
{
    int secret, guess, attempts = 0;

    /* 初始化随机数种子 */
    srand((unsigned int)time(NULL));

    /* 生成 1-100 的随机数 */
    secret = rand() % 100 + 1;

    printf("=== 猜数字游戏 ===\n");
    printf("我已经想好了一个1到100之间的数字。\n");

    /* 用 do-while：至少猜一次 */
    do {
        printf("请输入你的猜测：");
        scanf("%d", &guess);
        attempts = attempts + 1;

        if (guess > secret) {
            printf("太大了！再试试。\n");
        } else if (guess < secret) {
            printf("太小了！再试试。\n");
        } else {
            printf("恭喜你猜对了！用了 %d 次。\n", attempts);
        }
    } while (guess != secret);

    return 0;
}
```

## 16.5 常见错误

### 错误1：忘记更新循环变量

```c
/* ❌ 错误示例 */
int i = 1;
while (i <= 10) {
    printf("%d\n", i);
    /* 忘了 i++，死循环！ */
}

/* ✅ 正确示例 */
int i = 1;
while (i <= 10) {
    printf("%d\n", i);
    i++;
}
```

### 错误2：`while` 后加分号

```c
/* ❌ 错误示例 */
int i = 1;
while (i <= 10); {   /* 分号让循环体为空，死循环 */
    printf("%d\n", i);
    i++;
}

/* ✅ 正确示例 */
int i = 1;
while (i <= 10) {    /* 没有分号 */
    printf("%d\n", i);
    i++;
}
```

### 错误3：`do-while` 后忘分号

```c
/* ❌ 错误示例 */
do {
    printf("Hello\n");
} while (x < 5)   /* 缺少分号！编译报错 */

/* ✅ 正确示例 */
do {
    printf("Hello\n");
} while (x < 5);   /* 分号不能少 */
```

### 错误4：循环条件用 `=` 而不是 `==`

```c
/* ❌ 错误示例 */
int x = 5;
while (x = 5) {   /* 赋值，条件永远为真，死循环 */
    printf("...\n");
}

/* ✅ 正确示例 */
int x = 5;
while (x == 5) {   /* 比较，但这里 x 不变，还是死循环 */
    printf("...\n");
    x = 0;         /* 需要让条件最终为假 */
}
```

## 16.6 我在做什么？做得对不对？不对怎么办？

### 我在做什么？

我在学习C语言的循环结构：`while`（先判断再执行）和 `do-while`（先执行再判断），理解循环的三要素（初始化、条件、更新），以及死循环的用途和危险。

### 我做得对不对？

- 你能写出一个循环打印1到100的数字 → while过关。
- 你能解释 `while` 和 `do-while` 的区别 → 概念过关。
- 你能说出循环三要素，并知道缺少任何一个的后果 → 安全意识过关。
- 你能用 `do-while` 实现输入验证 → 实战过关。

### 不对怎么办？

| 症状 | 第一步检查 |
|------|-----------|
| 程序卡住不动（死循环） | 更新语句是否写了？循环变量是否在变？ |
| 循环一次都不执行 | `while` 的条件是否一开始就是假？ |
| 循环多执行了一次 | 条件边界是否写错了？`<=` 和 `<` 是否搞混？ |
| `do-while` 编译报错 | `while(条件)` 后面是否忘了分号？ |
| `while` 后面的代码不执行 | `while(...)` 后面是否误加了分号？ |

## 本章小结

| 知识点 | 说明 |
|--------|------|
| `while` | 先判断条件，条件为真才执行循环体，可能一次都不执行 |
| `do-while` | 先执行一次循环体，再判断条件，至少执行一次 |
| 循环三要素 | 初始化、条件（边界）、更新（向边界靠近），缺一不可 |
| 死循环 `while(1)` | 永远为真，需要 `break` 跳出。常用于服务器、游戏主循环 |
| `while` 后的分号 | 误加分号导致空循环体 |
| `do-while` 后的分号 | 必须加分号，否则编译报错 |

## 练习题

1. **1到N的累加**：输入一个正整数N，用 `while` 循环计算1到N的和，输出结果。

2. **乘法表**：用 `while` 循环打印9×9乘法表的前5行。格式：`1×1=1 1×2=2 ... 1×9=9`。

3. **倒计时**：输入一个正整数N，用 `while` 循环从N倒数到1，每行输出一个数字，最后输出"发射！"。

4. **输入验证**：用 `do-while` 循环要求用户输入一个1-7之间的整数。如果输入不合法，提示重新输入，直到合法为止。

5. **猜数字改进**：扩展本章的猜数字游戏，增加"最多10次机会"的限制。如果10次没猜对，显示"游戏结束"并公布答案。
