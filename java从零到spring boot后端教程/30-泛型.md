# 第30章 泛型

> 你有没有遇到过这样的尴尬：买了一个"万能收纳盒"，结果往里放袜子、放钥匙、放零食都行，但每次取东西时都要确认"这到底是什么"？泛型（Generics）就是给这个收纳盒贴上标签——**收纳盒<袜子>**、**收纳盒<钥匙>**——让你在放入和取出时心里有数，不用猜。

---

## 30.1 为什么需要泛型

Java 5 之前，集合里什么都能塞：

```java
List list = new ArrayList();
list.add("hello");
list.add(123);       // 可以放入整数
list.add(new Date()); // 可以放入日期对象

// 取出来时必须强制转型，而且你得"记住"第0个是String
String s = (String) list.get(0); // 必须强转
Integer i = (Integer) list.get(1); // 如果记错了类型，运行时报错 ClassCastException
```

两个致命问题：
1. **编译器不帮你检查**——放错类型不会有任何警告
2. **取数据必须强制转型**——代码又丑又危险

泛型解决的就是这两个问题：**编译时类型检查 + 消除强制转型**。

```java
List<String> list = new ArrayList<>();
list.add("hello");
// list.add(123);  // ❌ 编译错误！编译器直接拦住
String s = list.get(0); // ✅ 不需要强转，编译器知道是String
```

## 30.2 泛型类

### 30.2.1 基本语法

在类名后加 `<T>`，T 是**类型参数**（Type Parameter），你可以在类内部把它当作一个真实类型来用。

```java
// 定义一个泛型"盒子"
public class Box<T> {
    private T content;

    public void put(T content) {
        this.content = content;
    }

    public T get() {
        return content;
    }
}

// 使用
Box<String> stringBox = new Box<>();
stringBox.put("一本书");
String book = stringBox.get();  // 不需要强转

Box<Integer> intBox = new Box<>();
intBox.put(42);
int value = intBox.get();       // 同样不需要强转
```

> 🤔 **想多一点**：泛型参数命名不是强制的，但业界惯例是：
> - `T` — Type（类型）
> - `E` — Element（集合元素）
> - `K`, `V` — Key, Value（键值对）
> - `?` — 通配符（未知类型）
>
> 你也可以写 `Box<MyType>`，但用单字母更简洁。

### 30.2.2 多个类型参数

```java
public class Pair<K, V> {
    private K key;
    private V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() { return key; }
    public V getValue() { return value; }
}

// 使用
Pair<String, Integer> pair = new Pair<>("年龄", 25);
```

## 30.3 泛型方法

即使类本身不是泛型类，方法也可以独立声明泛型参数：

```java
public class Utils {
    // 泛型方法：在返回值前声明 <T>
    public static <T> T getMiddle(T[] array) {
        return array[array.length / 2];
    }

    // 交换数组中两个元素的位置
    public static <T> void swap(T[] array, int i, int j) {
        T temp = array[i];
        array[i] = array[j];
        array[j] = temp;
    }
}

// 调用时，编译器自动推断类型
String[] names = {"Alice", "Bob", "Charlie"};
String middle = Utils.getMiddle(names);  // 自动推断 T = String
System.out.println(middle);  // Bob
```

## 30.4 类型擦除

这是泛型最反直觉的一点，也是面试高频考点。

### 30.4.1 什么是类型擦除

Java 的泛型是**编译时**的：编译后，所有泛型类型信息都会被擦除，替换为原始类型（通常是 `Object` 或上界类型）。

```java
List<String> list1 = new ArrayList<>();
List<Integer> list2 = new ArrayList<>();

// 编译后，两者都变成了 List（原始类型）
// 运行时 JVM 看不到 <String> 或 <Integer>
System.out.println(list1.getClass() == list2.getClass()); // true！
```

> 🏷️ 此术语需进附录：**类型擦除**（Type Erasure）—— Java 泛型在编译后移除泛型类型信息，将类型参数替换为边界类型或 Object 的机制。这意味着泛型在运行时不可见。

### 30.4.2 类型擦除带来的限制

**限制1：不能用基本类型作为泛型参数**

```java
// List<int> list = new ArrayList<>();  // ❌ 编译错误
List<Integer> list = new ArrayList<>(); // ✅ 用包装类
```

原因：擦除后 `T` 必须能替换为 `Object`，而 `int` 不是对象。

**限制2：不能实例化泛型类型**

```java
public class Container<T> {
    // T item = new T();  // ❌ 编译错误
    // T[] array = new T[10]; // ❌ 编译错误
}
```

原因：运行时 `T` 已经不存在了，JVM 不知道 `new T()` 该调用哪个构造器。

**限制3：不能使用 `instanceof` 检查泛型类型**

```java
// if (obj instanceof List<String>) {}  // ❌ 编译错误
if (obj instanceof List) {}            // ✅ 只能检查原始类型
```

## 30.5 泛型通配符

### 30.5.1 无界通配符 `?`

`?` 表示"未知类型"，当你不需要关心具体类型时使用：

```java
// 打印任意类型的List
public static void printList(List<?> list) {
    for (Object obj : list) {
        System.out.println(obj);
    }
}

List<String> strings = List.of("a", "b");
List<Integer> ints = List.of(1, 2);
printList(strings);  // ✅
printList(ints);     // ✅
```

> ⚠️ 注意：`List<?>` 和 `List<Object>` 不同！
> - `List<Object>` 可以添加任何对象
> - `List<?>` 不能添加任何元素（除了 `null`），因为你不知道它到底是什么类型

### 30.5.2 上界通配符 `? extends T`

表示"T 或 T 的子类"，也叫**协变**（covariance）。

```java
// 计算数字列表的总和——可以接收 List<Integer>, List<Double>, List<Float> 等
public static double sum(List<? extends Number> list) {
    double total = 0;
    for (Number n : list) {
        total += n.doubleValue();
    }
    return total;
}

List<Integer> ints = List.of(1, 2, 3);
List<Double> doubles = List.of(1.5, 2.5);
System.out.println(sum(ints));    // 6.0
System.out.println(sum(doubles)); // 4.0
```

关键规则：`? extends T` 的集合**只能读，不能写**（除了 `null`）。

```java
List<? extends Number> list = new ArrayList<Integer>();
// list.add(10);   // ❌ 编译错误！编译器不知道具体是哪种Number子类
Number n = list.get(0); // ✅ 读取时可以保证至少是Number
```

### 30.5.3 下界通配符 `? super T`

表示"T 或 T 的父类"，也叫**逆变**（contravariance）。

```java
// 向列表中放入整数——可以接收 List<Integer>, List<Number>, List<Object>
public static void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
    list.add(3);
}

List<Number> numbers = new ArrayList<>();
addNumbers(numbers); // ✅
```

关键规则：`? super T` 的集合**只能写 T 类型，读出来只能是 Object**。

```java
List<? super Integer> list = new ArrayList<Number>();
list.add(10);            // ✅ 可以写入Integer
Object obj = list.get(0); // ✅ 读取只能当Object
```

## 30.6 PECS 原则

> 🏷️ 此术语需进附录：**PECS**（Producer Extends, Consumer Super）—— 通配符使用的黄金法则：如果要从泛型结构中**读取**数据（它是生产者），用 `? extends`；如果要向泛型结构中**写入**数据（它是消费者），用 `? super`。

快速记忆口诀：**"读用 Extends（E），写用 Super（S）"**。

### 图解 PECS

| 场景 | 你的角色 | 用哪个通配符 | 原因 |
|------|---------|------------|------|
| 只遍历不修改 | 消费者（读数据） | `? extends T` | 保证能取出 T 类型 |
| 只添加不读取 | 生产者（写数据） | `? super T` | 保证能放入 T 类型 |
| 又要读又要写 | — | 不用通配符，直接用 `<T>` | 明确具体类型 |

### 经典案例：Collections.copy()

JDK 源码中的 `Collections.copy()` 方法签名完美展示了 PECS：

```java
public static <T> void copy(List<? super T> dest, List<? extends T> src) {
    // dest 是消费者（写入），用 super
    // src  是生产者（读取），用 extends
    for (int i = 0; i < src.size(); i++) {
        dest.set(i, src.get(i));
    }
}
```

用起来的效果：

```java
List<Number> dest = new ArrayList<>(List.of(0, 0, 0));
List<Integer> src = List.of(1, 2, 3);

Collections.copy(dest, src);  // 可以把 List<Integer> 复制到 List<Number>
System.out.println(dest);     // [1, 2, 3]
```

## 30.7 泛型与继承

一个常见的误解：

```java
// Integer 是 Number 的子类
// 但 List<Integer> **不是** List<Number> 的子类！
List<Integer> ints = new ArrayList<>();
// List<Number> nums = ints;  // ❌ 编译错误
```

这就是"泛型的不变性"。如果允许这样做，就会出现类型安全问题：

```java
// 假设下面代码合法（实际不合法）
// List<Number> nums = ints;     // 假设可以
// nums.add(3.14);               // 加一个Double进List<Integer>
// Integer i = ints.get(0);      // 运行时爆炸！取出来的是Double
```

## 30.8 ❌ 常见错误

### 错误1：用基本类型当泛型参数

```java
// ❌ 错误
List<int> list = new ArrayList<>();
Map<String, double> map = new HashMap<>();

// ✅ 正确 —— 使用包装类
List<Integer> list = new ArrayList<>();
Map<String, Double> map = new HashMap<>();
```

### 错误2：在静态上下文中使用类的泛型参数

```java
public class Box<T> {
    // ❌ 错误：静态方法不能使用类的泛型参数
    // public static T create() { return null; }

    // ✅ 正确：静态方法声明自己的泛型参数
    public static <U> Box<U> create() {
        return new Box<>();
    }
}
```

原因是：类的泛型参数在实例化时确定，而静态方法属于类本身（先于实例存在），所以无法引用。

### 错误3：把 `List<?>` 当 `List<Object>` 用

```java
List<?> list = new ArrayList<String>();
// list.add("hello"); // ❌ 编译错误：? 是未知类型
// 只能读
Object obj = list.get(0); // ✅
```

## 30.9 小结

| 知识点 | 一句话总结 |
|--------|-----------|
| 泛型类/方法 | 用 `<T>` 占位，编译时确定类型，消除强制转型 |
| 类型擦除 | 编译后泛型信息消失，运行时不可见 |
| `? extends T` | 上界通配符，只能读不能写 |
| `? super T` | 下界通配符，只能写 T 类型 |
| PECS | Producer Extends, Consumer Super |
| 基本类型 | 不能作泛型参数，必须用包装类 |
| 不变性 | `List<Child>` 不是 `List<Parent>` 的子类 |

**1.** 下面代码哪些行会编译失败？为什么？

```java
List<? extends Number> list = new ArrayList<Integer>();
list.add(10);
list.add(null);
Number n = list.get(0);
```

**2.** 用 PECS 原则判断：如果方法参数是一个集合，你需要"遍历集合中的元素并打印"，应该用哪种通配符？如果你需要"把元素放入集合"，应该用哪种通配符？

**3.** 为什么 Java 的泛型不能在运行时保留类型信息（即为什么采用类型擦除）？这带来了哪些实际限制？

> 答案提示：1. `list.add(10)` 编译失败，因为 `? extends Number` 集合不能写入（除了 null）；2. 读用 `? extends`，写用 `? super`；3. 为兼容 Java 5 之前的字节码，限制包括不能用基本类型、不能 new 泛型实例等。
