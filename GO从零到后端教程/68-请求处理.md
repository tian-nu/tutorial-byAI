## 68.1 Query参数

Query参数是URL中 `?` 后面的部分：`/users?page=1&size=20&keyword=张三`。每个键值对用 `&` 分隔。

### c.Query：获取参数

```go
r.GET("/search", func(c *gin.Context) {
    keyword := c.Query("keyword")
    page := c.Query("page")

    c.JSON(200, gin.H{
        "keyword": keyword,
        "page":    page,
    })
})
```

访问 `/search?keyword=Go语言&page=1`，响应：

```json
{"keyword":"Go语言","page":"1"}
```

**如果参数不存在**，`c.Query` 返回空字符串 `""`，不会报错。

### c.DefaultQuery：带默认值

```go
page := c.DefaultQuery("page", "1")
size := c.DefaultQuery("size", "20")
```

如果URL中没有 `page` 参数，`page` 的值就是 `"1"`。这是最常用的写法——给分页、排序等参数提供默认值。

### c.GetQuery：判断是否存在

```go
keyword, exists := c.GetQuery("keyword")
if !exists {
    c.JSON(400, gin.H{"msg": "必须提供keyword参数"})
    return
}
```

当你需要区分"参数没传"和"参数传了空字符串"时，用 `GetQuery`。返回值是 `(string, bool)`。

### c.QueryArray：获取多值参数

```go
GET /items?tag=Go&tag=后端&tag=教程
```

```go
tags := c.QueryArray("tag")
fmt.Println(tags)
```

输出：`[Go 后端 教程]`

### c.QueryMap：获取Map参数

```go
GET /filter?price[min]=10&price[max]=100&color=red
```

```go
priceMap := c.QueryMap("price")
fmt.Println(priceMap)
```

输出：`map[min:10 max:100]`

## 68.2 路径参数

路径参数是嵌在URL路径中的动态值，在上一章已经介绍过。这里补充一些细节。

### c.Param获取

```go
r.GET("/users/:id/posts/:pid", func(c *gin.Context) {
    uid := c.Param("id")
    pid := c.Param("pid")
    c.JSON(200, gin.H{
        "user_id": uid,
        "post_id": pid,
    })
})
```

访问 `/users/5/posts/42` → `{"user_id":"5","post_id":"42"}`

### 路径参数vs查询参数：什么时候用哪个？

| 维度 | 路径参数 `/users/:id` | 查询参数 `?id=5` |
|------|----------------------|-------------------|
| 用于 | 标识"哪个资源" | 描述"怎么查资源" |
| 必选性 | 通常是必选的 | 通常是可选的 |
| 示例 | 用户ID、订单ID | 页码、排序、筛选 |
| RESTful | 资源标识 | 过滤条件 |

**经验法则**：URL如果不带这个参数就无法定位到具体资源→用路径参数；URL不带这个参数只是缺少一个筛选条件→用查询参数。

## 68.3 表单数据

当客户端用HTML `<form>` 或Postman的 `x-www-form-urlencoded` 提交数据时，数据在请求体中（不是JSON，不是Query）。

### c.PostForm

```go
r.POST("/login", func(c *gin.Context) {
    username := c.PostForm("username")
    password := c.PostForm("password")

    c.JSON(200, gin.H{
        "username": username,
        "password": password,
    })
})
```

对应的HTTP请求：

```http
POST /login
Content-Type: application/x-www-form-urlencoded

username=admin&password=123456
```

响应：

```json
{"username":"admin","password":"123456"}
```

### c.DefaultPostForm

```go
remember := c.DefaultPostForm("remember", "false")
```

用法和 `DefaultQuery` 完全一样——参数不存在时用默认值。

### c.GetPostForm

```go
username, exists := c.GetPostForm("username")
```

用法和 `GetQuery` 一样——需要区分"没传"和"传了空字符串"时使用。

### c.PostFormArray / c.PostFormMap

```go
tags := c.PostFormArray("tags")
attrs := c.PostFormMap("attrs")
```

## 68.4 JSON绑定

现代前后端分离项目，99%的请求体是JSON格式。Gin提供了非常简洁的JSON绑定功能。

### ShouldBindJSON

（`binding` 校验规则的完整讲解见第71章参数校验，这里先学会基本用法即可）

```go
type CreateUserRequest struct {
    Name  string `json:"name" binding:"required"`
    Email string `json:"email" binding:"required,email"`
    Age   int    `json:"age" binding:"required,min=1,max=120"`
}

r.POST("/users", func(c *gin.Context) {
    var req CreateUserRequest

    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(400, gin.H{
            "code": 400,
            "msg":  err.Error(),
        })
        return
    }

    c.JSON(201, gin.H{
        "code": 0,
        "msg":  "创建成功",
        "data": req,
    })
})
```

客户端发送：

```http
POST /users
Content-Type: application/json

{
  "name": "张三",
  "email": "zhangsan@example.com",
  "age": 25
}
```

`ShouldBindJSON` 做了三件事：
1. 读取请求体的JSON
2. 反序列化到结构体 `req`
3. 执行结构体tag中定义的 `binding` 校验规则

如果JSON格式错误，或缺少必填字段，或年龄不在1~120范围内，`err` 不为nil，返回400。

### BindJSON vs ShouldBindJSON

| 函数 | 错误处理 |
|------|---------|
| `c.BindJSON(&req)` | 出错时自动写响应头 400，并调用 `c.Abort()` |
| `c.ShouldBindJSON(&req)` | 只返回error，由你自己决定怎么响应 |

**推荐始终用 `ShouldBindJSON`**，自己控制错误响应格式。`BindJSON` 的自动行为不够灵活。

### 结构体Tag详解

结构体tag决定了JSON字段怎么映射到Go结构体：

```go
type Request struct {
    Name  string `json:"name"`                       // JSON中的 "name" → Go的 Name
    Email string `json:"email,omitempty"`             // 空字符串时不序列化
    Age   int    `json:"-"`                           // 不参与JSON序列化/反序列化
    Phone string `json:"phone" binding:"required"`    // 必填校验
}
```

`binding` tag的常见规则（第71章会深入）：

```go
`binding:"required"`           // 必填，零值报错
`binding:"min=1,max=120"`      // 数值范围
`binding:"email"`              // 邮箱格式
`binding:"url"`                // URL格式
`binding:"len=11"`             // 字符串长度
`binding:"oneof=male female"`  // 枚举值
```

### 绑定到map

有时你不知道JSON的结构是什么（比如接收webhook回调），可以绑定到map：

```go
r.POST("/webhook", func(c *gin.Context) {
    var data map[string]interface{}

    if err := c.ShouldBindJSON(&data); err != nil {
        c.JSON(400, gin.H{"msg": "JSON格式错误"})
        return
    }

    eventType, _ := data["event_type"].(string)
    c.JSON(200, gin.H{"received": eventType})
})
```

## 68.5 文件上传

### 单文件上传

```go
r.POST("/upload", func(c *gin.Context) {
    file, err := c.FormFile("avatar")
    if err != nil {
        c.JSON(400, gin.H{"msg": "请选择文件"})
        return
    }

    dst := "./uploads/" + file.Filename
    if err := c.SaveUploadedFile(file, dst); err != nil {
        c.JSON(500, gin.H{"msg": "保存文件失败"})
        return
    }

    c.JSON(200, gin.H{
        "msg":      "上传成功",
        "filename": file.Filename,
        "size":     file.Size,
    })
})
```

`c.FormFile("avatar")` 返回 `*multipart.FileHeader` 对象，包含文件名、大小、MIME类型等信息。`c.SaveUploadedFile` 把文件保存到指定路径。

### 多文件上传

```go
r.POST("/upload-multi", func(c *gin.Context) {
    form, err := c.MultipartForm()
    if err != nil {
        c.JSON(400, gin.H{"msg": "表单解析失败"})
        return
    }

    files := form.File["photos"]
    var filenames []string

    for _, file := range files {
        dst := "./uploads/" + file.Filename
        if err := c.SaveUploadedFile(file, dst); err != nil {
            c.JSON(500, gin.H{"msg": "保存文件失败: " + file.Filename})
            return
        }
        filenames = append(filenames, file.Filename)
    }

    c.JSON(200, gin.H{
        "msg":       "上传成功",
        "filenames": filenames,
    })
})
```

### 文件大小限制

Gin默认限制请求体大小为 **32MB**。你可以调整：

```go
r.MaxMultipartMemory = 8 << 20
```

`8 << 20` = 8 * 1024 * 1024 = 8388608 字节 ≈ **8MB**。

### 给上传的文件重命名

直接用客户端发送的文件名有安全风险——攻击者可能用 `../../../etc/passwd` 这样的路径穿越攻击。务必重命名：

```go
import (
    "fmt"
    "path/filepath"
    "time"

    "github.com/google/uuid"
)

func saveFile(c *gin.Context, file *multipart.FileHeader) (string, error) {
    ext := filepath.Ext(file.Filename)
    newName := fmt.Sprintf("%s_%d%s", uuid.New().String(), time.Now().Unix(), ext)
    dst := filepath.Join("./uploads", newName)

    if err := c.SaveUploadedFile(file, dst); err != nil {
        return "", err
    }
    return newName, nil
}
```

## 68.6 获取原始请求体

有些场景下需要读取原始的请求体字节（比如验签、日志记录、代理转发）。Gin提供了 `c.GetRawData()`：

```go
r.POST("/callback", func(c *gin.Context) {
    rawData, err := c.GetRawData()
    if err != nil {
        c.JSON(400, gin.H{"msg": "读取请求体失败"})
        return
    }

    fmt.Println("收到原始请求体:", string(rawData))

    c.JSON(200, gin.H{"received": true})
})
```

### 重要警告：GetRawData只能用一次

Gin的请求体是一个 `io.ReadCloser` 流——**读一次就消耗了**。如果你先调 `GetRawData()`，再调 `ShouldBindJSON()`，第二个调用会读到空内容并报错。

**解决方案**：如果你需要多次读取请求体，在读取后重新设置：

```go
rawData, _ := c.GetRawData()

c.Request.Body = io.NopCloser(bytes.NewBuffer(rawData))

var req SomeStruct
c.ShouldBindJSON(&req)
```

`io.NopCloser(bytes.NewBuffer(rawData))` 重新构造了一个包含同样数据的 `ReadCloser`，放回 `c.Request.Body`，后续的 `ShouldBindJSON` 就能正常读取了。

## 🤔 想多一点：Content-Type有什么用？

每次HTTP请求都有一个 `Content-Type` 头，告诉服务器"我发的数据是什么格式"。但Gin的大多数读取函数**不检查Content-Type**——你发JSON但声明 `Content-Type: text/plain`，`ShouldBindJSON` 照样能解析。这是Gin的灵活性设计，但也意味着：

1. 你不能依赖Content-Type来判断数据格式是否匹配
2. 客户端发错了Content-Type你不会知道
3. 安全的做法是自己加校验：

```go
if c.GetHeader("Content-Type") != "application/json" {
    c.JSON(415, gin.H{"msg": "只接受JSON格式"})
    return
}
```

HTTP状态码 **415 Unsupported Media Type** 是专门用于"服务器不支持请求中的媒体格式"的场景。

## 本章小结

| 知识点 | 要点 |
|--------|------|
| Query | `c.Query(key)` / `c.DefaultQuery(key, default)` / `c.GetQuery(key)` |
| Query多值 | `c.QueryArray(key)` 获取数组，`c.QueryMap(key)` 获取map |
| 路径参数 | `c.Param(":name")` 用于资源标识 |
| 路径vs查询 | 路径参数=定位资源（必选），查询参数=过滤条件（可选） |
| 表单 | `c.PostForm(key)` / `c.DefaultPostForm` / `c.GetPostForm` |
| JSON绑定 | `c.ShouldBindJSON(&struct)` 推荐用Should系列 |
| BindJSON vs ShouldBindJSON | BindJSON自动写400+Abort，ShouldBindJSON让你自己处理 |
| 结构体Tag | `json:"name"` 映射JSON字段，`binding:"required"` 校验 |
| 文件上传 | `c.FormFile(name)` + `c.SaveUploadedFile(file, dst)` |
| 多文件 | `c.MultipartForm()` 获取所有文件 |
| 大小限制 | `r.MaxMultipartMemory = 8 << 20` 默认32MB |
| 安全重命名 | 不要用客户端原始文件名，用UUID + 时间戳 |
| GetRawData | 读原始请求体，只能读一次，需要重新设置 `c.Request.Body` |
