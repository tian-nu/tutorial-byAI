- 对应文档版本：N/A（教程系列第20篇）
- 适用环境：Vue 3 + Vite / Node.js 18+
- 读者角色：前端开发者

## 一、目标与完成效果

**一句话目标**：在文章详情页下方实现评论功能，包括评论列表展示、发表评论、删除自己的评论，用嵌套组件 `CommentList → CommentItem` 组织代码。

**完成后的可观测效果**：
- 文章详情页下方有评论区
- 评论列表显示每条评论的：用户名、时间、内容
- 已登录用户可以看到评论输入框并发表评论
- 评论发表后立即出现在列表中
- 每条评论右侧有「删除」按钮（仅评论作者可见）
- 未登录用户只能看评论，不能发表

## 二、前置条件

| 前置项 | 版本/要求 | 验证命令 |
|--------|-----------|----------|
| 文章详情页可用 | `/post/:id` 正常显示文章 | 浏览器访问任意文章详情页 |

| 后端评论 API 可用 | 或使用 json-server 模拟 | 后端 `GET /api/posts/:id/comments` 有返回 |

**API 接口约定**（假设后端提供以下接口）：

| 方法 | 路径 | 说明 | 需要认证 |
|------|------|------|:---:|
| GET | `/api/posts/:id/comments` | 获取某篇文章的评论 | 否 |
| POST | `/api/posts/:id/comments` | 发表评论 | 是 |
| DELETE | `/api/comments/:id` | 删除评论 | 是（只能删自己的） |

### 步骤1：扩展 API 封装——添加评论相关方法

#### 我在做什么？

在 `api/index.js` 中增加评论相关的 API 方法。

**在 `src/api/index.js` 末尾添加**：

```js
/** 获取文章评论 */
export function getComments(postId) {
  return request(`/api/posts/${postId}/comments`)
}

/** 发表评论 */
export function addComment(postId, data, token) {
  return request(`/api/posts/${postId}/comments`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
    },
    body: JSON.stringify(data),
  })
}

/** 删除评论 */
export function deleteComment(commentId, token) {
  return request(`/api/comments/${commentId}`, {
    method: 'DELETE',
    headers: {
      'Authorization': `Bearer ${token}`,
    },
  })
}
```

### 步骤2：创建 CommentItem 组件——单条评论

#### 我在做什么？

每条评论是一个独立的组件，显示用户名、时间、内容，如果是自己的评论还可以删除。

**比喻**：CommentItem 是一张「便利贴」，贴在文章详情页的评论区墙上。CommentList 是整面墙，CommentItem 是墙上的一张张便利贴。

**创建 `src/components/CommentItem.vue`**：

```vue
<template>
  <div class="comment-item">
    <div class="comment-header">
      <strong>{{ comment.username }}</strong>
      <span class="comment-time">{{ formatTime(comment.createdAt) }}</span>
    </div>
    <p class="comment-content">{{ comment.content }}</p>

    <!-- 删除按钮：仅评论作者可见 -->
    <button
      v-if="isOwner"
      @click="$emit('delete', comment.id)"
      class="btn-delete-comment"
      :disabled="deleting"
    >
      {{ deleting ? '...' : '🗑️' }}
    </button>
  </div>
</template>

<script setup>
import { computed } from 'vue'

const props = defineProps({
  comment: {
    type: Object,
    required: true,
  },
  deleting: {
    type: Boolean,
    default: false,
  },
})

// 定义事件：通知父组件「我要删除这条评论」
defineEmits(['delete'])

// 判断当前用户是否是评论作者
const isOwner = computed(() => {
  const userStr = localStorage.getItem('user')
  if (!userStr) return false
  const user = JSON.parse(userStr)
  return user.username === props.comment.username
})

// 格式化时间（后端可能返回 ISO 格式，如 "2026-06-05T10:30:00Z"）
function formatTime(dateStr) {
  if (!dateStr) return ''
  const date = new Date(dateStr)
  // 如果转换失败，直接返回原始字符串
  if (isNaN(date.getTime())) return dateStr
  return date.toLocaleString('zh-CN', {
    year: 'numeric',
    month: '2-digit',
    day: '2-digit',
    hour: '2-digit',
    minute: '2-digit',
  })
}
</script>
```

#### 代码逐行解释

| 代码 | 含义 |
|------|------|
| `defineEmits(['delete'])` | 声明这个组件可以发出 `delete` 事件 |
| `$emit('delete', comment.id)` | 点击删除时，把评论ID传给父组件 |
| `isOwner` | 判断当前用户是否是评论作者 |
| `v-if="isOwner"` | 只有自己的评论才显示删除按钮 |
| `formatTime()` | 把 ISO 时间格式化为「2026/06/05 10:30」 |

### 步骤3：创建 CommentList 组件——评论列表

#### 我在做什么？

CommentList 负责：获取评论数据、用 `v-for` 渲染 `CommentItem`、提供发表评论的表单。

**创建 `src/components/CommentList.vue`**：

```vue
<template>
  <div class="comment-list">
    <h3>💬 评论 ({{ comments.length }})</h3>

    <!-- 评论列表 -->
    <div v-if="loading">⏳ 加载评论...</div>
    <div v-else-if="loadError" class="error">加载评论失败：{{ loadError }}</div>
    <div v-else-if="comments.length === 0" class="empty">暂无评论，来抢沙发吧！</div>

    <CommentItem
      v-for="comment in comments"
      :key="comment.id"
      :comment="comment"
      :deleting="deletingId === comment.id"
      @delete="handleDeleteComment"
    />

    <!-- 发表评论（仅登录用户可见） -->
    <div v-if="isLoggedIn" class="comment-form">
      <textarea
        v-model="newComment"
        rows="3"
        placeholder="写下你的评论..."
        maxlength="500"
      ></textarea>
      <div class="comment-form-footer">
        <span class="char-count">{{ newComment.length }}/500</span>
        <button
          @click="handleAddComment"
          :disabled="!newComment.trim() || submitting"
        >
          {{ submitting ? '发表中...' : '📮 发表评论' }}
        </button>
      </div>
      <p v-if="submitError" class="error">{{ submitError }}</p>
    </div>

    <p v-else class="login-hint">
      <router-link to="/login">登录</router-link> 后可以发表评论
    </p>
  </div>
</template>

<script setup>
import { ref, onMounted } from 'vue'
import { getComments, addComment, deleteComment } from '../api/index.js'
import CommentItem from './CommentItem.vue'

const props = defineProps({
  postId: {
    type: Number,
    required: true,
  },
})

const comments = ref([])
const loading = ref(true)
const loadError = ref('')
const newComment = ref('')
const submitting = ref(false)
const submitError = ref('')
const deletingId = ref(null)    // 正在删除的评论ID

const isLoggedIn = !!localStorage.getItem('token')

// 获取评论列表
async function fetchComments() {
  loading.value = true
  loadError.value = ''
  try {
    const data = await getComments(props.postId)
    comments.value = data
  } catch (err) {
    loadError.value = err.message
  } finally {
    loading.value = false
  }
}

// 发表评论
async function handleAddComment() {
  if (!newComment.value.trim()) return

  const token = localStorage.getItem('token')
  if (!token) return

  submitError.value = ''
  submitting.value = true
  try {
    await addComment(props.postId, { content: newComment.value.trim() }, token)
    newComment.value = ''           // 清空输入框
    await fetchComments()           // 重新获取评论列表
  } catch (err) {
    submitError.value = err.message || '发表失败'
  } finally {
    submitting.value = false
  }
}

// 删除评论
async function handleDeleteComment(commentId) {
  const confirmed = confirm('确定要删除这条评论吗？')
  if (!confirmed) return

  const token = localStorage.getItem('token')
  if (!token) return

  deletingId.value = commentId
  try {
    await deleteComment(commentId, token)
    // 从本地列表中移除该评论（无需重新请求）
    comments.value = comments.value.filter(c => c.id !== commentId)
  } catch (err) {
    alert('删除失败：' + (err.message || '请稍后重试'))
  } finally {
    deletingId.value = null
  }
}

onMounted(() => {
  fetchComments()
})
</script>
```

#### 代码逐行解释

| 代码 | 含义 |
|------|------|
| `defineProps({ postId })` | 接收当前文章ID，用于获取该文章的评论 |
| `@delete="handleDeleteComment"` | 监听 `CommentItem` 的 `delete` 事件 |
| `:deleting="deletingId === comment.id"` | 只有正在删除的那条评论显示 `...` |
| `comments.value = comments.value.filter(...)` | 删除成功后从本地列表移除，无需重新请求 |
| `v-if="isLoggedIn"` | 登录后才显示评论输入框 |
| `:disabled="!newComment.trim() \|\| submitting"` | 空内容或提交中时按钮不可用 |

```
PostDetail.vue
```

## 六、小结

| 我学了什么 | 关键点 |
|------------|--------|
| 嵌套组件三层结构 | `PostDetail` → `CommentList` → `CommentItem` |
| 子传父事件 | `$emit('delete', commentId)` + `@delete="handleDeleteComment"` |
| 发表评论流程 | 表单 → 验证 → POST → 清空输入框 → 刷新列表 |
| 删除评论流程 | 确认 → DELETE → 本地移除（`filter`） |
| 登录状态控制 | `v-if="isLoggedIn"` 显示输入框，否则显示登录提示 |
| 权限控制 | `isOwner` computed 判断，`v-if` 控制删除按钮 |
| 防重复提交 | `:disabled="!newComment.trim() \|\| submitting"` |

## 八、已知坑点与禁止事项

- ⚠️ **`$emit` 的拼写**：是 `$emit`，不是 `$emmit` 或 `$emit`。少一个 `t` 都不会报错，但事件传不出去。
- ⚠️ **子组件不能直接修改父组件数据**：`CommentItem` 不能直接 `comments.value = ...`，只能通过 `$emit` 通知父组件去修改。
- ⚠️ **删除后建议用 `filter` 本地移除**：比重新 fetch 整个列表更高效，用户体验更好（无闪烁）。
- ⚠️ **评论输入框在 `v-if` 外不能用 `v-model` 双向绑定**：因为 `v-if` 会在 DOM 中移除/重建元素，如有需要改用 `v-show`。
