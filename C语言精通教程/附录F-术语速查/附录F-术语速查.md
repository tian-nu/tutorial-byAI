## 我在做什么？

本教程中出现了大量专业术语。本附录将它们按拼音排序，提供英中对照和通俗解释，方便你随时查阅。反直觉的术语会特别标注"字面陷阱"。

## F.1 术语速查表（按拼音排序）

### A

| 术语 | 英文 | 解释 |
|------|------|------|
| ASCII | American Standard Code for Information Interchange | 美国信息交换标准代码。用 0-127 的整数表示英文字母、数字、标点和控制字符。每个字符占 1 字节。*参见附录E* |
| 安全路径 | safe path | 构造文件路径时防止 `../` 等跳转到上级目录的防护措施，防止目录遍历攻击。 |

### B

| 术语 | 英文 | 解释 |
|------|------|------|
| 贝尔实验室 | Bell Labs | 美国 AT&T 公司的研究机构，C 语言和 UNIX 操作系统的诞生地。 |
| 编译型语言 | compiled language | 需要编译器将源代码翻译为机器码后才能执行的语言，如 C、C++、Rust。与解释型语言相对。 |
| 编译器 | compiler | 将源代码翻译为机器码的程序。C 语言常用编译器：gcc、clang、MSVC。 |
| 编码 | encoding | 将字符映射为数字的规则。ASCII 是最基础的编码，Unicode 是现代的通用编码（UTF-8 是其实现之一）。 |
| 变量提升 | variable hoisting | *字面陷阱*：C 语言没有变量提升（JavaScript 有）。C 中变量必须先声明后使用。 |
| 标准库 | standard library | 随编译器一起提供的一组函数和类型，完成常见任务（输入输出、字符串操作、内存管理等）。C 标准库包含 `<stdio.h>`、`<stdlib.h>`、`<string.h>` 等。 |
| 标准输入 | stdin | 程序默认的输入来源，通常是键盘。对应 `FILE *stdin`。 |
| 标准输出 | stdout | 程序默认的输出目标，通常是屏幕。对应 `FILE *stdout`。 |
| 标准错误 | stderr | 程序默认的错误输出目标，通常也是屏幕，但与 stdout 独立。用于输出错误信息。 |
| 并发 | concurrency | 多个任务在同一时间段内交替执行，宏观上看起来同时进行。与并行（parallelism，真正同时执行）不同。 |

### C

| 术语 | 英文 | 解释 |
|------|------|------|
| CLI | Command Line Interface | 命令行界面。用户通过键盘输入命令与程序交互，如终端。 |
| crt0 | C Runtime 0 | 程序启动时最先执行的代码，负责初始化堆栈、调用 `main` 函数、处理 `main` 的返回值。通常由编译器自动链接。 |
| 参数传递 | parameter passing | 函数调用时，将实参的值复制给形参的过程。C 语言默认使用传值调用。 |
| 抽象数据类型 | ADT, Abstract Data Type | 将数据结构和操作封装在一起，对外只暴露接口不暴露实现细节。如栈、队列、哈希表。 |
| 传值调用 | call by value | C 语言的默认参数传递方式：将实参的值复制一份给形参，函数内修改形参不影响实参。 |
| 错误码 | error code | 函数返回的整数，用于表示操作是否成功。0 通常表示成功，非零表示错误类型。`errno` 是全局错误码变量。 |

### D

| 术语 | 英文 | 解释 |
|------|------|------|
| Dennis Ritchie | — | C 语言和 UNIX 操作系统的共同创造者，被誉为"C 语言之父"。 |
| 大顶堆 / 小顶堆 | max heap / min heap | 大顶堆：每个节点值大于等于子节点值；小顶堆：每个节点值小于等于子节点值。用于实现优先队列。 |
| 地址 | address | 内存中每个字节的唯一编号，从 0 开始递增。指针存储的就是地址值。 |
| 递归 | recursion | 函数直接或间接调用自身。必须设置基线条件防止无限递归。 |
| 段错误 | segmentation fault (segfault) | 程序访问了不属于自己的内存区域时，操作系统发送的终止信号。常见原因：空指针解引用、数组越界、访问已释放内存。 |
| 堆（Heap） | heap | 程序运行时动态分配的内存区域，由 `malloc`/`free` 管理。与数据结构中的"堆"不同。 |
| 堆（数据结构） | heap | 一种特殊的完全二叉树，满足堆性质（大顶堆或小顶堆）。与内存中的堆区域不同。 |
| 堆栈 | stack | 见"栈"。 |
| 队列 | queue | 先进先出（FIFO）的数据结构，像排队。基本操作：enqueue（入队）、dequeue（出队）。 |
| 短路求值 | short-circuit evaluation | 逻辑表达式求值时，如果能从左侧确定结果，右侧不再计算。如 `a && b` 中若 `a` 为假则不计算 `b`。 |

## F.2 三问

1. **"栈"有两个含义，怎么区分？** 内存中的栈（call stack）是程序运行时自动管理函数调用的区域；数据结构中的栈是一种 LIFO 数据结构。两者原理相同（后进先出），但内存栈是硬件/OS 实现的，数据结构栈是你在代码中实现的。

2. **"堆"也有两个含义，怎么区分？** 内存堆（heap）是 `malloc` 分配的内存区域；数据结构中的堆（heap）是一种特殊的完全二叉树。英文同为"heap"但含义完全不同，这是历史命名造成的巧合。

3. **"未定义行为"与"实现定义行为"有什么区别？** 未定义行为（UB）是 C 标准完全不规定结果，编译器可以做任何事；实现定义行为是 C 标准规定了可选方案，编译器必须选择一种并保持一致，如 `int` 的大小。
