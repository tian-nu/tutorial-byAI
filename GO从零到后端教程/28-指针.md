# 第28章 · 指针

> "你寄快递给朋友，快递员不需要把整栋房子搬过去——他只需要知道地址。指针就是地址——它不是房子本身，但拿着它就能找到房子。"

---

## 28.1 什么是指针

### 比喻：房子和门牌号

每个变量都是一栋房子。每栋房子有两个属性：
- **房子里住着谁**（变量的值）
- **房子在哪个地址**（变量的内存地址）

```go
var x int = 42
```

`x` 是一栋房子，房号是某个内存地址（比如 `0xc00001a0b0`），房子里住的是数字 `42`。

**指针（存储另一个变量内存地址的变量，通过地址间接访问数据，详见附录I）就是"写着地址的纸条"**。纸条本身不是房子——它很小，就写着地址。但拿着纸条，你就能找到房子，进门看看里面有什么，甚至换一个人住进去。

```go
var p *int = &x
```

`p` 是一张纸条，上面写着 `x` 的地址。`p` 本身也是一个变量，它有自己的地址（纸条也存在于某个位置），但它存的内容是另一个地址。

### 指针类型的声明

```go
var p *int
```

`*int` 的意思是：这是一个指向 `int` 类型变量的指针。读法是"指向int的指针"。

不同类型有不同的指针类型：

```go
var pi *int
var ps *string
var pb *bool
var pf *float64
```

`*int` 只能指向 `int`，不能指向 `string`——就像门牌号是某个城市某条街的格式，你不能拿中国门牌号去套美国地址。

## 28.2 & 和 *：取地址和解引用

两个操作符，一个管"拿地址"，一个管"通过地址找到东西"。

### &：取地址

```go
x := 42
p := &x

fmt.Println("x的值:", x)
fmt.Println("x的地址:", p)
fmt.Println("p的类型:", fmt.Sprintf("%T", p))
```

输出类似：
```
x的值: 42
x的地址: 0xc00001a0b0
p的类型: *int
```

`&x` 就是"把x的门牌号写下来"。

### *：解引用（通过地址找到值）

```go
x := 42
p := &x

fmt.Println(*p)

*p = 100
fmt.Println(x)
```

输出：
```
42
100
```

`*p` 是"根据纸条上的地址找到房子，看看里面住着什么"。因此 `*p` 就是 `x` 的值。

`*p = 100` 的意思是：根据纸条上的地址找到房子，把里面住的人换成 `100`。注意，我们改的不是纸条（`p` 还是指向同一个地址），而是 **p 指向的那个变量**。

### 比喻总结

```
x = 42          → 房子A里住着42
p = &x          → 纸条上写着"房子A的地址"
*p              → 根据纸条找到房子A，看看里面有什么 → 42
*p = 100        → 根据纸条找到房子A，把里面换成100 → x 现在是100
```

> 🤔 `*` 有两种用法：在类型声明里 `*int` 表示"指向int的指针类型"；在表达式里 `*p` 表示"解引用"。虽然都叫星号，含义不同——就像"行"字在"银行"和"行走"里读音不同。

### 指针的零值：nil

```go
var p *int
fmt.Println(p)
```

输出：`<nil>`

`nil` 的意思是"这张纸条是空白的——没写任何地址"。如果你尝试对 `nil` 指针解引用：

```go
var p *int
fmt.Println(*p)
```

程序会 `panic`——就像你拿着一张空白纸条去找房子，根本不知道该往哪走。

## 28.3 为什么要用指针

### 理由一：修改函数外的变量

第26章学过，Go是值传递。基本类型传进函数，改的是副本：

```go
func tryToChange(n int) {
    n = 999
}

x := 10
tryToChange(x)
fmt.Println(x)
```

输出是 `10`——没变。因为你传的是"复印件"。

那如果真的要改呢？传指针：

```go
func reallyChange(n *int) {
    *n = 999
}

x := 10
reallyChange(&x)
fmt.Println(x)
```

输出是 `999`。传的不是x本身，而是x的地址。函数通过地址找到x，把它改了。

**比喻**：你想让朋友帮你把家里的灯换成红色的。你不能把整个房子传过去——太重了搬不动。你只需要把地址（钥匙）给他——他去你家换了灯泡。灯确实在原来的房子里变了。

### 理由二：避免复制大对象

如果你有一个巨大的结构体（几百个字段），每次传参都复制一遍，内存和CPU都受不了：

```go
type BigData struct {
    Items [1000000]int
}

func processByValue(data BigData) {

}

func processByPointer(data *BigData) {

}
```

`processByPointer` 只需要传一个8字节的指针（64位系统上一个地址占8字节），而不是复制一整个巨大的结构体。

### 理由三：表示"不存在"

有时候，"0"和""和"false"是合法值，不能用来表示"没有"：

```go
func findUser(id int) *User {
    if id <= 0 {
        return nil
    }
    return &User{ID: id, Name: "张三"}
}
```

返回 `nil` 明确表示"没找到"。如果返回的是 `User` 值类型，空结构体也是合法的，无法区分"找到了一个空用户"和"没找到"。

---

> ❌ **常见错误 1：nil 指针解引用 → panic**
> 
> ```go
> var p *int         // p 是 nil
> fmt.Println(*p)    // ❌ panic: runtime error: invalid memory address or nil pointer dereference
> ```
> 
> nil 指针没有指向任何有效内存地址。对它解引用（`*p`）相当于拿着一本空白的地址簿去找地址——运行时直接崩溃。
> 
> ✅ **正确做法**：
> ```go
> var p *int
> if p != nil {
>     fmt.Println(*p)   // ✅ 先判空再使用
> } else {
>     fmt.Println("p 是 nil")
> }
> ```

> ❌ **常见错误 2：取函数返回值的地址**
> 
> ```go
> func getValue() int {
>     return 42
> }
> 
> p := &getValue()   // ❌ 编译错误！cannot take the address of getValue()
> ```
> 
> `&` 只能作用于可以寻址的表达式（变量、结构体字段、数组元素等）。函数返回值是临时的、没有固定地址，不能直接取地址。
> 
> ✅ **正确做法**：
> ```go
> func getValue() int {
>     return 42
> }
> 
> v := getValue()    // 先用变量接住返回值
> p := &v            // ✅ 再取变量的地址
> fmt.Println(*p)    // 42
> ```

> ❌ **常见错误 3：指针作为 map 的 key**
> 
> ```go
> type User struct {
>     Name string
> }
> 
> m := make(map[*User]string)
> u1 := &User{Name: "张三"}
> u2 := &User{Name: "张三"}
> 
> m[u1] = "值1"
> fmt.Println(m[u2])   // 输出空字符串——找不到！
> ```
> 
> 两个 `&User{Name: "张三"}` 虽然内容相同，但它们是不同的指针（地址不同）。map 用指针作为 key 时，比较的是**地址**而不是**值**。这意味着即使两个指针指向的内容完全一样，只要地址不同，就是不同的 key。
> 
> ✅ **正确做法**：
> ```go
> m := make(map[User]string)   // ✅ 用值类型做 key
> m[User{Name: "张三"}] = "值1"
> fmt.Println(m[User{Name: "张三"}])   // 输出: 值1
> ```
> 如果确实需要用指针做 key，必须始终用同一个指针实例。

## 28.4 new vs make

Go里有两个"创建"函数，它们的分工容易搞混。

### new：分配内存，返回指针

```go
p := new(int)
fmt.Println(*p)

*p = 42
fmt.Println(*p)
```

`new(int)` 做了两件事：
1. 在内存中分配一块能放下 `int` 的空间，初始化为零值 `0`
2. 返回指向这块空间的指针 `*int`

`new` 可以用于任何类型：

```go
p1 := new(int)
p2 := new(string)
p3 := new(bool)
p4 := new(MyStruct)
```

`new` 只是分配内存+返回指针，**不会做任何初始化**（除了赋予零值）。

### make：创建并初始化切片、Map、Channel

```go
s := make([]int, 0, 10)
m := make(map[string]int)
ch := make(chan int)
```

`make` 只能用于三种类型：`slice`、`map`、`channel`。它不只是分配内存——它会初始化内部数据结构。

为什么不能用 `new` 创建切片？

```go
s := new([]int)
```

`new([]int)` 返回的是 `*[]int`——一个指向切片描述符的指针，描述符本身是 nil。你得到的是一张指向空白纸条的纸条——没法直接用，还需要再分配底层数组。

而 `make([]int, 0, 10)` 一次性完成：分配底层数组 + 初始化切片描述符（指针、长度、容量）。

### 比喻：盖房子 vs 拿门牌号

- `new`：给你一块空地和一个门牌号。空地上什么都没有（只有零值）。
- `make`：给你盖好了一栋精装修的房子，水电气全通。只限三种房型（slice、map、chan）。

### new的实际使用频率

说实话，`new` 在Go代码中用得不多。大部分时候你直接用 `&Type{}` 就行了：

```go
user := &User{}
```

这和 `new(User)` 效果一样，但 `&User{}` 更常见——因为它可以在创建时直接给字段赋值：

```go
user := &User{Name: "张三", Age: 20}
```

## 28.5 什么时候用值，什么时候用指针

这是个经典问题。一张表讲清楚：

| 场景 | 用值 | 用指针 | 原因 |
|------|------|--------|------|
| 需要修改原数据 | ❌ | ✅ | 值传递改不了原数据 |
| 结构体很小（几个字段） | ✅ | ❌ | 复制成本可忽略 |
| 结构体很大 | ❌ | ✅ | 避免复制开销 |
| 需要表示nil | ❌ | ✅ | 值类型没有nil |
| 并发安全要求 | ✅ | ❌ | 值类型每个副本独立，天然安全 |
| Map/Slice/Chan类型 | ✅ | ❌ | 它们本身就是引用语义 |
| 需要实现某个接口 | 看情况 | 看情况 | 取决于方法集（第30章讲） |

### 具体判断方法

**先问自己一个问题：这个函数需要修改传进来的数据吗？**
- 需要 → 指针
- 不需要 → 再问下一个问题

**这个数据大吗（比如超过64字节）？**
- 大 → 指针
- 不大 → 值

**需要表示"不存在"吗？**
- 需要 → 指针
- 不需要 → 值

**是Map/Slice/Chan吗？**
- 是 → 值（它们内部已经是指针了）
