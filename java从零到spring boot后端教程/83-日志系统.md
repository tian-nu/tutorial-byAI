## 83.1 SLF4J + Logback ——门面 + 实现

```
你的代码 → SLF4J（门面接口）
                ↓ 绑定
          Logback（实现引擎）
                ↓ 输出
          控制台 / 文件 / 远程服务器
```

- **SLF4J**：统一的日志 API，你的代码只依赖它
- **Logback**：Spring Boot 默认日志实现

## 83.2 Lombok @Slf4j——一行注解搞定 Logger

```java
import lombok.extern.slf4j.Slf4j;

@Slf4j
@Service
public class UserService {
    public User create(User user) {
        log.info("创建用户: {}", user.getUsername());
        log.debug("用户详情: {}", user);
        return userRepository.save(user);
    }
}
```

`@Slf4j` 编译时自动生成 `private static final Logger log = LoggerFactory.getLogger(UserService.class);`。

## 83.3 日志级别——什么场合说什么话

| 级别 | 用途 | 示例 |
|------|------|------|
| TRACE | 最详细跟踪 | `log.trace("参数 a={}", a)` |
| DEBUG | 开发调试 | `log.debug("查询 SQL: {}", sql)` |
| INFO | 关键流程节点 | `log.info("用户 {} 登录成功", username)` |
| WARN | 潜在问题 | `log.warn("连接池使用率 90%")` |
| ERROR | 错误需处理 | `log.error("支付失败, 订单: {}", id, ex)` |

### 占位符必须使用

```java
// 错误：字符串拼接，即使日志不输出也会执行拼接
log.debug("用户: " + user.toString());

// 正确：占位符，日志不输出时不调用 toString()
log.debug("用户: {}", user);
```

## 83.4 日志配置——logback-spring.xml

在 `src/main/resources/` 下创建 `logback-spring.xml`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
    <springProperty scope="context" name="LOG_PATH" source="logging.file.path" defaultValue="logs"/>

    <!-- 控制台输出 -->
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>%d{HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>

    <!-- 文件输出（按天滚动） -->
    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>${LOG_PATH}/app.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <fileNamePattern>${LOG_PATH}/app.%d{yyyy-MM-dd}.log</fileNamePattern>
            <maxHistory>30</maxHistory>
        </rollingPolicy>
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>

    <!-- 错误日志单独文件 -->
    <appender name="ERROR_FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>${LOG_PATH}/error.log</file>
        <filter class="ch.qos.logback.classic.filter.ThresholdFilter">
            <level>ERROR</level>
        </filter>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <fileNamePattern>${LOG_PATH}/error.%d{yyyy-MM-dd}.log</fileNamePattern>
            <maxHistory>90</maxHistory>
        </rollingPolicy>
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>

    <!-- Profile 区分 -->
    <springProfile name="dev">
        <root level="INFO">
            <appender-ref ref="CONSOLE"/>
        </root>
        <logger name="com.example.demo" level="DEBUG"/>
    </springProfile>

    <springProfile name="prod">
        <root level="WARN">
            <appender-ref ref="CONSOLE"/>
            <appender-ref ref="FILE"/>
            <appender-ref ref="ERROR_FILE"/>
        </root>
        <logger name="com.example.demo" level="INFO"/>
    </springProfile>
</configuration>
```

## 83.5 MDC——全链路追踪 ID

MDC（Mapped Diagnostic Context）给每个请求打上唯一 ID：

### 创建 MDC Filter

```java
public class TraceIdFilter implements Filter {

    @Override
    public void doFilter(ServletRequest request, ServletResponse response,
                         FilterChain chain) throws IOException, ServletException {
        try {
            String traceId = UUID.randomUUID().toString()
                    .replace("-", "").substring(0, 16);
            MDC.put("traceId", traceId);
            chain.doFilter(request, response);
        } finally {
            MDC.clear();  // 必须清理，避免内存泄漏
        }
    }
}
```

### 修改日志格式

```xml
<pattern>%d{HH:mm:ss.SSS} [%thread] [%X{traceId}] %-5level %logger{36} - %msg%n</pattern>
```

输出效果：

```
14:30:15.123 [http-nio-8080-1] [a1b2c3d4e5f6g7h8] INFO  UserService - 创建用户: zhangsan
14:30:15.789 [http-nio-8080-2] [x9y8z7w6v5u4t3s2] INFO  UserService - 创建用户: lisi
```

用 `grep a1b2c3d4e5f6g7h8 logs/app.log` 即可提取一个请求的所有日志。

## 83.6 小结

| 知识点 | 核心内容 |
|--------|----------|
| SLF4J | 日志门面，解耦代码与实现 |
| Logback | Spring Boot 默认日志实现 |
| @Slf4j | Lombok 注解，自动生成 log 变量 |
| 日志级别 | TRACE < DEBUG < INFO < WARN < ERROR |
| logback-spring.xml | 控制台/文件/滚动策略/Profile 区分 |
| MDC | 全链路追踪 ID，Filter 注入 + finally 清理 |

## 83.7 自测题

**1. 哪种日志写法是正确的？**
```java
log.info("用户: " + username);       // A
log.info("用户: {}", username);      // B
```

**2. 生产环境日志级别应该设置为什么？DEBUG 打开有什么风险？**

**3. MDC 的 traceId 必须在哪里清理？为什么？**

---

**答案提示**：1→B。2→INFO 或 WARN；DEBUG 造成大量 I/O 和磁盘占用。3→在 Filter 的 finally 块中清理；不清理会导致内存泄漏和 traceId 串用。下一章——文件上传下载。
