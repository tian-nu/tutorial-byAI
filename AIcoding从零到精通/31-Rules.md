# 31-Rules

> **本篇属于**：第三篇·Cursor从入门到精通 | **预计阅读**：15分钟 | **难度**：★★★☆☆
>
> **前置条件**：Cursor已安装，会基本的Chat操作。

前几章你学会了用Cursor的Tab、Chat、Composer、Agent——但你可能发现一个问题：**AI有时候会写出不符合你预期的代码**。

比如你习惯用单引号，AI却写双引号。你习惯用小写下划线命名，AI却写驼峰式。你要求每个函数都必须有注释，AI有时候就忘了。

这不是AI的错——**AI不知道你的偏好，除非你告诉它**。

这就是这一章要讲的：**Cursor Rules**。

---

## 31.1 Cursor Rules是什么？

> **📖 术语：Cursor Rules**——一个项目级的配置文件，放在项目根目录的 `.cursorrules` 文件里。你在里面写项目规范、编码风格、偏好设置，AI在生成代码时会自动遵守。

**一个比喻**：你请了一个厨师到你家做饭。你告诉他"我家用淡盐，不放味精，辣椒要切细"。厨师记住了，以后做菜都按这个来。`.cursorrules` 就是你和AI之间的"厨房规矩"。

---

## 31.2 为什么需要Rules？

没有Rules的时候，AI按自己的理解写代码，可能和你习惯不一致：

❌ **没有Rules**：
```python
def GetUserData(userId):
    """获取用户数据"""
    # ...
```

✅ **有Rules（要求用下划线命名+中文注释）**：
```python
def get_user_data(user_id):
    # 获取用户数据
    # ...
```

差异很大，对吧？**Rules让AI的输出更符合你的预期。**

## 31.3 怎么创建和编写.cursorrules

### **1.** 在项目根目录创建文件

文件名必须是 `.cursorrules`（注意前面有个点，没有后缀名）。

在Cursor里：
- 在左侧文件列表空白处右键 → 新建文件
- 输入文件名 `.cursorrules`
- 回车

### **2.** 写规则内容

以下是一个**通用Python项目模板**，你可以直接复制使用：

```
# 编码规范
- 使用Python 3.10+语法
- 所有函数使用下划线命名（snake_case）
- 类名使用大驼峰命名（PascalCase）
- 常量使用全大写+下划线（UPPER_CASE）
- 缩进用4个空格，不用Tab

# 注释要求
- 每个函数必须有docstring（三引号注释）
- 复杂逻辑用行内注释解释
- 注释用中文

# 文件组织
- 每个类一个文件，文件名=类名小写+下划线
- 测试文件放在tests/目录下

# 禁止事项
- 不要使用全局变量
- 不要使用eval()或exec()
- 不要打印敏感信息（密码、密钥）
```

### 另外两个模板：

**Web项目模板**（HTML/CSS/JS）：
```
# 前端规范
- HTML使用语义化标签（header, main, footer等）
- CSS类名使用短横线命名（kebab-case）
- JavaScript函数使用小驼峰命名（camelCase）
- 页面必须支持移动端（响应式设计）
- 颜色使用CSS变量，不要直接写颜色值

# 禁止事项
- 不要使用内联样式（style属性）
- 不要使用alert()弹窗，用console.log替代
```

**脚本项目模板**（Python快速脚本）：
```
# 脚本规范
- 开头必须有 #!/usr/bin/env python3
- 所有外部依赖写在文件顶部注释中
- 错误处理用try/except，不要裸奔
- 命令行参数用argparse解析

# 禁止事项
- 不要写无用的打印，如"正在执行..."
- 脚本退出用sys.exit()返回状态码
```

## 31.4 验证Rules生效了

创建好 `.cursorrules` 后，怎么知道AI真的在遵守呢？

### 验证方法：

**1.** 打开Chat（Ctrl+L）

**2.** 输入：

```
我的.cursorrules里有要求吗？你看到了吗？
```

**3.** AI会列出它看到的规则，或者告诉你它注意到了哪些。

**4.** 再试一个实战验证：

在Chat里输入：
```
写一个Python函数，接收用户ID和用户名，返回用户信息字典
```

**5.** 如果你的Rules要求用下划线命名，AI应该生成 `get_user_info(user_id, user_name)`。如果它生成了 `getUserInfo(userId, userName)`，说明Rules没生效——检查文件是否保存、文件名是否正确。

## 31.5 最佳实践

### 实践1：每个项目都加一个.cursorrules

无论项目多小，加一个 `.cursorrules` 只要两分钟，但能省下后面无数次的"AI你没按我的要求写"。

### 实践2：规则要具体，不要模糊

❌ 模糊规则："代码要写得好"
✅ 具体规则："每个函数不超过50行，超过50行要拆分成子函数"

❌ 模糊规则："写注释"
✅ 具体规则："每个函数开头写docstring，说明参数和返回值"

### 实践3：规则不是越多越好

一个 `.cursorrules` 文件不建议超过50行。太多规则AI反而记不住。挑最重要的写。

### 实践4：团队项目统一.cursorrules

如果你们团队用Cursor，把 `.cursorrules` 提交到Git仓库里，所有人都用一样的规则，代码风格就统一了。

## 31.6 如果出错了怎么办

### 🔴 问题1：AI似乎没遵守我的Rules

**可能原因**：文件名不对，或者文件没保存。

**检查步骤**：
1. 确认文件名是 `.cursorrules`（不是 `.cursorrules.txt` 或 `cursorrules`）
2. 确认文件在项目根目录
3. 确认文件已保存（Ctrl+S）
4. 关掉Cursor再重新打开项目

### 🔴 问题2：Rules太长，AI记不住

**解决**：精简Rules，只保留最重要的10条以内。优先级排序：命名规范 > 注释要求 > 禁止事项 > 其他。

### 🔴 问题3：两个规则冲突了

**比如**：规则说"函数不超过50行"，但需求确实需要80行。

**解决**：规则是"建议"不是"死命令"。AI不会因为规则冲突就拒绝生成代码。如果确实需要80行，在Chat里补充说明"这个函数可以超过50行"，AI会优先听从你的最新指令。
