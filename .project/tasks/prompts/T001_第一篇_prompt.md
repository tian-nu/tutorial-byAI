#### 01 — 数据库是什么？为什么需要数据库？
- 学习目标：理解数据库存在的原因，能说出数据库 vs 文件系统的 4 个核心差异
- 核心内容拆解：
  1. 没有数据库的世界：用文件存数据会怎样？（并发写冲突、查询困难、无一致性保证）—— 用具体代码示例演示
  2. 数据库解决的 4 个问题：持久化、高效查询、并发控制、数据一致性
  3. 数据库分类全景图（每类一句话解释，各举一个产品名）：
     - 关系型（MySQL/PG）— 表格存数据，SQL 查询
     - 键值（Redis）— 像超大字典，key→value
     - 文档（MongoDB）— 存 JSON 文档，结构灵活
     - 列存（HBase）— 按列存储，适合分析
     - 图（Neo4j）— 存节点和关系，适合社交网络
     - 时序（InfluxDB）— 存带时间戳的数据，适合监控
  4. 为什么先学关系型？— 最成熟、市场需求最大、概念可迁移
- 比喻：数据库 = 图书馆（书架=表，书=行，管理员=DBMS）—— **注意：此处不提前引入"索引"概念**，只讲书架和书
- 验证方法：读者能用自己的话解释"为什么不用 Excel 存电商数据"
- 常见错误：认为数据库就是 MySQL；认为 NoSQL 比 SQL 高级
- 术语（需进附录）：关系型数据库、NoSQL、DBMS、持久化、并发、一致性

#### 02 — 关系型数据库核心概念（上）：表、行、列、主键
- 学习目标：掌握表、行、列、主键的基本概念，能看懂一张简单的关系模型图
- 核心内容拆解：
  1. 表（Table）= 二维表格，类比 Excel 的 Sheet
  2. 行（Row）= 一条记录，列（Column）= 一个字段
  3. 主键（Primary Key）= 每行的身份证号，不可重复、不可为空
  4. 用一张**两张表**的简单关系图：用户表（users）→ 订单表（orders），通过用户的 id 关联
  5. 为什么订单里不直接存用户全部信息？— 引出"引用"的概念（但不在此章深入讲范式，只给直觉）
- 注意：本章只讲 4 个概念，不引入外键、索引、约束、SQL。避免信息过载。
- 验证方法：读者能画出用户表和订单表的关系草图
- 常见错误：混淆主键和普通列；认为表名必须用英文复数
- 术语（需进附录）：表、行/记录、列/字段、主键

#### 02b — 关系型数据库核心概念（下）：外键、索引、约束、SQL
- 学习目标：掌握外键、索引、约束、SQL 四个概念，补全基础知识
- 核心内容拆解：
  1. 外键（Foreign Key）= 存另一张表的主键值，建立表间关联（**不用"指针"一词**，说"关联字段"）
  2. 索引（Index）= 加速查找的数据结构，先给直观感受：`SELECT * FROM users WHERE id = 1` 快 vs `WHERE name = '张三'` 慢 → 为什么？→ 简单提及"索引像书的目录，后面第17章详解"
  3. 约束（Constraint）= 数据规则（不能为空、不能重复等）
  4. SQL = 与数据库对话的语言，分为 5 类：
     - DDL（数据定义）：CREATE / ALTER / DROP
     - DML（数据操作）：INSERT / UPDATE / DELETE
     - DQL（数据查询）：SELECT
     - DCL（数据控制）：GRANT / REVOKE
     - TCL（事务控制）：BEGIN / COMMIT / ROLLBACK
  5. 这个 SQL 分类框架是后续所有 SQL 章节的顶层地图，建议画一张分类图
- 验证方法：读者能准确说出 SQL 的 5 个子语言及其用途
- 常见错误：外键和索引混淆；认为约束是多余的
- 术语（需进附录）：外键、索引、约束、SQL、DDL、DML、DQL、DCL、TCL

#### 03 — 安装 MySQL 与环境配置
- 学习目标：在本地成功安装 MySQL 并连接
- 核心内容拆解：
  1. Windows：MySQL Installer 下载 → 选择 Server + Workbench → 设置 root 密码 → 验证
  2. macOS：`brew install mysql` → `brew services start mysql` → 安全配置
  3. Linux：`apt install mysql-server` / `yum install mysql-server` → 安全配置
  4. MySQL Workbench 界面介绍：连接管理、SQL 编辑器、结果面板
  5. 第一个连接：`mysql -u root -p` → `SELECT VERSION();` → `SHOW DATABASES;`
  6. 安全配置：`mysql_secure_installation`（删除匿名用户、禁止远程 root、删除 test 库）
- 排错预案（必须包含）：
  - 服务未启动：`systemctl status mysql` / `services.msc` 查看
  - root 密码忘记：`mysql_secure_installation` 重新设置 / 安全模式启动
  - 端口 3306 被占用：`netstat -an | findstr 3306`（Win）/ `lsof -i :3306`（Mac/Linux）
  - 权限不足：`sudo` 前缀
  - my.ini / my.cnf 找不到：各平台默认路径列表
- 验证方法：终端执行 `mysql -u root -p -e "SELECT VERSION();"` 成功返回版本号
- 常见错误：MySQL 服务未启动；root 密码忘记；端口 3306 被占用
- 术语（需进附录）：MySQL Server、MySQL Workbench、root、端口 3306
- 01 章末尾标注：`> **[可暂停点 1/7]**：第一篇结束。重启命令：`mysql -u root -p -e "SELECT VERSION();"` 确认连接正常。`
