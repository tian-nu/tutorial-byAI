## 我在做什么？

到目前为止，你写的所有代码都挤在一个 `main.c` 里。但真正的项目可能包含几十个甚至几百个源文件。当你把"计算圆面积"的代码写在一个文件里，把"计算球体积"的代码写在另一个文件里，`main.c` 只需要"知道"这些函数存在就行——这就是多文件项目的核心思想：**声明与实现分离**。这一章你将学会如何拆分代码到多个文件，如何使用头文件共享声明，以及排查多文件编译中最常见的 `undefined reference` 链接错误。

读完本章你能：将代码拆分为多个 `.c` 文件，创建 `.h` 头文件存放声明，区分 `#include "..."` 和 `#include <...>` 的差异，编译多文件项目，定位并修复 `undefined reference` 链接错误。

## 一、为什么需要多文件？

### 1.1 单文件的问题

假设你写了一个计算器程序，所有代码都在 `main.c` 里：

```text
main.c (800 行)
├── 加法函数
├── 减法函数
├── 乘法函数
├── 除法函数
├── 平方根函数
├── 数学工具函数
├── 输入处理函数
├── 输出格式化函数
└── main 函数
```

问题：
- **难以维护**：改一个函数要在 800 行里找到它。
- **难以协作**：两个人同时修改 `main.c`，Git 冲突一堆。
- **编译慢**：改一行就要重新编译整个 800 行的文件。
- **难以复用**：想把"数学工具函数"用到另一个项目，需要从 800 行里手动复制。

### 1.2 多文件的解决方案

```text
项目/
├── main.c          (100 行，程序入口和主流程)
├── math_ops.c      (200 行，数学运算的实现)
├── math_ops.h      (30 行， 数学运算的声明)
├── io_utils.c      (150 行，输入输出的实现)
├── io_utils.h      (20 行， 输入输出的声明)
└── Makefile        (编译规则)
```

每个 `.c` 文件各司其职，`.h` 文件充当"说明书"——告诉其他文件"这里有哪些函数可以用"。

```text
                  math_ops.h                   io_utils.h
                 (声明：有哪些函数)            (声明：有哪些函数)
                      |                            |
                      v                            v
main.c  -----------> #include "math_ops.h" -----> 编译时知道函数签名
    |                 #include "io_utils.h" -----> 编译时知道函数签名
    |
    |  调用 add()、print_result() 等函数
    |
    v
链接时，链接器在 math_ops.o 和 io_utils.o 中找到函数的实际代码
```

## 二、头文件（.h）放声明，源文件（.c）放实现

### 2.1 声明 vs 定义

这是本章最重要的概念区分：

| 概念 | 含义 | 例子 |
|------|------|------|
| **声明** | 告诉编译器"这个东西存在，它长这样" | `int add(int a, int b);` |
| **定义** | 实际创建这个东西，分配内存/生成代码 | `int add(int a, int b) { return a + b; }` |

比喻：**声明**是名片——"我叫张三，电话号码是 138xxxx"。**定义**是真人——张三真的站在你面前。你可以有很多张名片（多次声明），但张三本人只有一个（只能定义一次）。

### 2.2 头文件里放什么

头文件（`.h`）只放**声明**，不放**定义**（除少数例外）：

```c
// math_ops.h — 头文件：只放声明
#ifndef MATH_OPS_H
#define MATH_OPS_H

// ✓ 函数声明（只有签名，没有函数体）
int add(int a, int b);
int subtract(int a, int b);
int multiply(int a, int b);
double divide(int a, int b);

// ✓ 宏定义
#define PI 3.14159

// ✓ 类型定义（typedef）
typedef struct {
    double x;
    double y;
} Point;

// ✓ 全局变量声明（注意 extern 关键字）
extern int operation_count;

// ✗ 绝对不要在头文件中放函数定义（有函数体的函数）！
// 除非是 static inline 函数（后面会讲）

#endif
```

### 2.3 源文件里放什么

源文件（`.c`）放**实现**：

```c
// math_ops.c — 源文件：放实现
#include "math_ops.h"  // 包含自己的头文件，检查声明和定义是否一致
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int subtract(int a, int b) {
    return a - b;
}

int multiply(int a, int b) {
    return a * b;
}

double divide(int a, int b) {
    if (b == 0) {
        fprintf(stderr, "错误：除数为零\n");
        return 0;
    }
    return (double)a / b;
}
```

### 2.4 main.c 使用头文件

```c
// main.c
#include <stdio.h>
#include "math_ops.h"  // 引入我们自己写的头文件

int main() {
    int a = 20, b = 5;

    printf("%d + %d = %d\n", a, b, add(a, b));
    printf("%d - %d = %d\n", a, b, subtract(a, b));
    printf("%d * %d = %d\n", a, b, multiply(a, b));
    printf("%d / %d = %.2f\n", a, b, divide(a, b));

    return 0;
}
```

### 2.5 编译多文件项目

```bash
# 方法一：一次性编译所有 .c 文件
gcc main.c math_ops.c -o calculator

# 方法二：先分别编译成 .o 文件，再链接
gcc -c main.c      # 生成 main.o
gcc -c math_ops.c  # 生成 math_ops.o
gcc main.o math_ops.o -o calculator   # 链接所有 .o 文件

# 方法三：使用通配符
gcc *.c -o calculator
```

方法二的优点：如果你只改了 `math_ops.c`，只需重新编译 `math_ops.o`，再链接一次，不需要重新编译 `main.c`。在大型项目中，这能节省大量编译时间。

## 三、#include 双引号 vs 尖括号

### 3.1 两种写法

```c
#include <stdio.h>       // 尖括号：系统头文件
#include "math_ops.h"    // 双引号：项目自己的头文件
```

### 3.2 搜索路径差异

| 写法 | 搜索顺序 | 用途 |
|------|----------|------|
| `#include <...>` | 先在系统头文件目录搜索，再在 `-I` 指定的目录搜索 | 标准库头文件（`stdio.h`、`stdlib.h` 等） |
| `#include "..."` | 先在当前目录搜索，再在 `-I` 指定的目录搜索，最后在系统头文件目录搜索 | 项目自己的头文件 |

### 3.3 使用 -I 指定头文件搜索路径

如果你的头文件放在子目录中：

```bash
# 项目结构
project/
├── src/
│   ├── main.c
│   └── math_ops.c
└── include/
    └── math_ops.h

# 编译时指定 include 路径
gcc src/main.c src/math_ops.c -Iinclude -o calculator
```

`-Iinclude` 告诉编译器在 `include/` 目录下搜索 `#include "..."` 或 `#include <...>` 引用的头文件。

### 3.4 规则总结

- 系统/标准库头文件 → 用 `<...>`
- 项目自己的头文件 → 用 `"..."`
- 第三方库的头文件 → 通常用 `<...>`（配合 `-I` 指定路径）

## 四、extern：跨文件共享全局变量

### 4.1 问题：全局变量不能定义两次

```c
// math_ops.c
int operation_count = 0;  // 定义全局变量

// main.c
int operation_count = 0;  // 再次定义 → 链接错误：重复定义！
```

### 4.2 解决方案：extern 声明

```c
// math_ops.h
#ifndef MATH_OPS_H
#define MATH_OPS_H

extern int operation_count;  // extern 声明：这个变量在别处定义

int add(int a, int b);
// ...

#endif
```

```c
// math_ops.c
#include "math_ops.h"

int operation_count = 0;  // 定义：只在这里定义一次

int add(int a, int b) {
    operation_count++;     // 使用全局变量
    return a + b;
}
```

```c
// main.c
#include <stdio.h>
#include "math_ops.h"

int main() {
    printf("初始操作计数: %d\n", operation_count);  // 0
    printf("1 + 2 = %d\n", add(1, 2));
    printf("操作计数: %d\n", operation_count);       // 1
    return 0;
}
```

`extern` 关键字告诉编译器："这个变量存在，但定义在别的 `.c` 文件中，链接时再找它。"

## 五、static 函数：限制函数的作用域

### 5.1 默认情况下函数是全局的

```c
// math_ops.c
int add(int a, int b) {
    return a + b;
}

// 其他 .c 文件可以直接声明并调用 add()
// 只要在链接时能找到它的定义
```

C 语言中，函数默认是"全局可见"的——只要你在任何 `.c` 文件中声明了它，链接时就能找到。

### 5.2 static 函数：只在当前文件可见

```c
// math_ops.c

// 辅助函数，不打算给外部使用
static int validate_input(int n) {
    return n >= 0 && n <= 1000;
}

// 公开函数
int add(int a, int b) {
    if (!validate_input(a) || !validate_input(b)) {
        return -1;  // 输入无效
    }
    return a + b;
}
```

- `static int validate_input(...)` 只能在 `math_ops.c` 内部使用。
- 其他 `.c` 文件无法调用 `validate_input`，即使声明了也不行（链接时会报 `undefined reference`）。
- 这就像给函数贴上了"私有"标签。

### 5.3 为什么用 static 函数

- **封装**：隐藏实现细节，只暴露需要的接口。
- **避免命名冲突**：两个 `.c` 文件可以有同名的 `static` 函数，互不干扰。
- **优化**：编译器知道 `static` 函数只在当前文件使用，可以做更激进的优化（如内联）。

## 六、链接错误 undefined reference 排查

### 6.1 经典错误

这是多文件项目中最常见的错误：

```bash
$ gcc main.c -o calculator
/tmp/ccxYzabc.o: In function `main':
main.c:(.text+0x15): undefined reference to `add'
collect2: error: ld returned 1 exit status
```

### 6.2 错误解读

```
undefined reference to `add'
  └─────┬─────┘          └─┬─┘
        │                  └── 函数名或变量名
        └── 链接器找不到这个符号的定义
```

- **编译**阶段（`gcc -c`）通过了——说明声明没问题。
- **链接**阶段失败了——说明找不到定义（函数体或变量）。

### 6.3 常见原因和解决

**原因一：忘记编译某个 `.c` 文件**

```bash
# ❌ 只编译了 main.c，没有 math_ops.c
gcc main.c -o calculator

# ✅ 包含所有 .c 文件
gcc main.c math_ops.c -o calculator
```

**原因二：函数名拼写不一致**

```c
// math_ops.h
int add(int a, int b);   // 声明

// math_ops.c
int Add(int a, int b) {  // ❌ 大小写不同！编译器认为这是两个不同的函数
    return a + b;
}
```

C 语言区分大小写，`add` 和 `Add` 是不同的符号。

**原因三：函数加了 static**

```c
// math_ops.c
static int add(int a, int b) {  // ❌ static 限制了作用域
    return a + b;
}

// main.c 尝试调用 add() → undefined reference
```

**原因四：头文件声明了但源文件没实现**

```c
// math_ops.h
int add(int a, int b);     // 声明了
int multiply(int a, int b); // 也声明了

// math_ops.c
int add(int a, int b) {    // 实现了
    return a + b;
}
// multiply 没有实现 → 如果 main.c 调用了 multiply，链接时报 undefined reference
```

**原因五：C 和 C++ 混编**

```bash
# 如果 math_ops.c 被 C++ 编译器编译，而 main.c 被 C 编译器编译
# 函数名会被 C++ 编译器改写（name mangling），导致链接失败
```

### 6.4 排查工具

```bash
# 查看 .o 文件中定义了哪些符号
nm math_ops.o | grep " T "

# 查看 .o 文件中需要哪些外部符号
nm main.o | grep " U "
```

`T` 表示该符号在文件中**定义**了，`U` 表示该符号是**未定义**的（需要从其他文件链接）。

```bash
$ nm math_ops.o
0000000000000000 T add          # T = 代码段中定义的符号
0000000000000024 T subtract
0000000000000000 D operation_count  # D = 数据段中定义的符号

$ nm main.o
                 U add          # U = 未定义，需要链接
                 U printf       # 来自 libc
0000000000000000 T main         # main 函数在这里定义
```

## 七、完整多文件项目示例

### 7.1 项目结构

```
calculator/
├── main.c
├── math_ops.h
├── math_ops.c
├── io_utils.h
└── io_utils.c
```

### 7.2 math_ops.h

```c
#ifndef MATH_OPS_H
#define MATH_OPS_H

// 函数声明
int add(int a, int b);
int subtract(int a, int b);
int multiply(int a, int b);
double divide(int a, int b);

// 常量
#define PI 3.14159

// 全局变量声明
extern int operation_count;

#endif
```

### 7.3 math_ops.c

```c
#include "math_ops.h"
#include <stdio.h>

int operation_count = 0;

int add(int a, int b) {
    operation_count++;
    return a + b;
}

int subtract(int a, int b) {
    operation_count++;
    return a - b;
}

int multiply(int a, int b) {
    operation_count++;
    return a * b;
}

double divide(int a, int b) {
    operation_count++;
    if (b == 0) {
        fprintf(stderr, "错误：除数为零\n");
        return 0;
    }
    return (double)a / b;
}
```

### 7.4 io_utils.h

```c
#ifndef IO_UTILS_H
#define IO_UTILS_H

void print_result(const char *op, int a, int b, double result);
int get_input(const char *prompt);

#endif
```

### 7.5 io_utils.c

```c
#include "io_utils.h"
#include <stdio.h>

void print_result(const char *op, int a, int b, double result) {
    printf("%d %s %d = %.2f\n", a, op, b, result);
}

int get_input(const char *prompt) {
    int value;
    printf("%s", prompt);
    scanf("%d", &value);
    return value;
}
```

### 7.6 main.c

```c
#include <stdio.h>
#include "math_ops.h"
#include "io_utils.h"

int main() {
    int a = get_input("请输入第一个数: ");
    int b = get_input("请输入第二个数: ");

    print_result("+", a, b, add(a, b));
    print_result("-", a, b, subtract(a, b));
    print_result("*", a, b, multiply(a, b));
    print_result("/", a, b, divide(a, b));

    printf("\n共执行了 %d 次运算\n", operation_count);

    return 0;
}
```

### 7.7 编译与运行

```bash
gcc main.c math_ops.c io_utils.c -o calculator
./calculator
```

预期交互：

```
请输入第一个数: 10
请输入第二个数: 3
10 + 3 = 13.00
10 - 3 = 7.00
10 * 3 = 30.00
10 / 3 = 3.33

共执行了 4 次运算
```

## 八、头文件中能放函数定义吗？

### 8.1 一般规则：不能

```c
// math_ops.h — ❌ 错误写法
#ifndef MATH_OPS_H
#define MATH_OPS_H

int add(int a, int b) {
    return a + b;   // 函数定义放在了头文件中
}

#endif
```

如果两个 `.c` 文件都 `#include "math_ops.h"`，编译时每个 `.o` 文件都包含一份 `add` 的代码，链接时就会报**重复定义**错误。

### 8.2 例外：static inline 函数

```c
// math_ops.h — ✓ 可以放 static inline 函数
#ifndef MATH_OPS_H
#define MATH_OPS_H

// static inline：每个包含此头文件的 .c 文件都有自己的副本
// 链接时不会冲突
static inline int max_int(int a, int b) {
    return a > b ? a : b;
}

#endif
```

`static inline` 函数的每个副本都限制在各自的 `.o` 文件中，链接器不会把它们视为重复定义。这是 C 语言中替代宏函数的一种安全方式。

### 8.3 例外：极为简短的辅助函数

在一些小型项目中，你可能看到头文件直接包含函数定义。这**不推荐**，但你需要知道它们存在：

```c
// 某些小型库头文件中的写法（不推荐）
static inline int clamp(int x, int lo, int hi) {
    if (x < lo) return lo;
    if (x > hi) return hi;
    return x;
}
```

## 九、常见错误

### 9.1 头文件中放函数定义

```c
// ❌ math_ops.h
int add(int a, int b) {
    return a + b;
}

// 后果：每个包含 math_ops.h 的 .c 文件都生成一份 add 的代码
// 链接时报 "multiple definition of `add'"
```

### 9.2 忘记包含自己的头文件

```c
// math_ops.c
// ❌ 忘记 #include "math_ops.h"

int add(int a, int b) {     // 声明和定义可能不一致也无法发现
    return a + b;
}
```

如果头文件中声明是 `double add(double a, double b)`，而源文件中实现是 `int add(int a, int b)`，编译器不会报错（因为两个文件各自独立编译），但链接后行为异常。

**正确做法**：每个 `.c` 文件始终 `#include` 自己的 `.h` 文件，让编译器检查声明和定义是否一致。

### 9.3 缺少头文件保护

```c
// ❌ math_ops.h — 没有保护
typedef struct { int x, y; } Point;

// main.c
#include "math_ops.h"
#include "math_ops.h"  // 间接重复包含 → 编译错误：Point 重复定义
```

### 9.4 全局变量只声明不定义

```c
// math_ops.h
extern int operation_count;  // 只有声明

// math_ops.c
// ❌ 忘记定义 operation_count！

// main.c 使用 operation_count → undefined reference
```

## 十、我做得对不对？

### 10.1 创建多文件项目

按第七节的项目结构创建所有文件，编译运行。

### 10.2 验证头文件保护

在 `main.c` 中故意重复包含头文件：

```c
#include "math_ops.h"
#include "math_ops.h"  // 故意重复
#include "math_ops.h"  // 故意重复
```

如果头文件保护正确，编译应该通过。如果去掉 `#ifndef`/`#define`/`#endif`，编译会失败。

### 10.3 故意制造链接错误

1. 编译时只编译 `main.c`（不包含 `math_ops.c`），观察 `undefined reference` 错误。
2. 把 `math_ops.c` 中的 `add` 函数名改为 `Add`，观察链接错误。
3. 修复后重新编译，验证成功。

### 10.4 验证 static 函数

在 `math_ops.c` 中添加一个 `static` 辅助函数，然后在 `main.c` 中尝试声明并调用它，观察链接错误。

## 十一、不对怎么办？

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| `undefined reference to 'xxx'` | 缺少 `.c` 文件或函数名拼写错误 | 检查编译命令是否包含所有 `.c` 文件；用 `nm` 检查符号 |
| `multiple definition of 'xxx'` | 头文件中有函数定义，被多个 `.c` 包含 | 函数定义移到 `.c` 文件；或在头文件中加 `static inline` |
| `redefinition of 'xxx'` | 头文件缺少保护，结构体/宏被重复定义 | 加上 `#ifndef`/`#define`/`#endif` 保护 |
| 编译通过但运行结果异常 | 声明和定义不一致（参数类型不同） | 确保 `.c` 文件 `#include` 了自己的 `.h` 文件 |
| 全局变量在别的文件中值为 0 | 只声明了 `extern` 但忘记定义 | 在某个 `.c` 文件中定义变量（不带 `extern`） |
| 头文件改了但重编译不生效 | 编译时用了旧的 `.o` 文件 | 执行 `make clean` 或手动删除 `.o` 后重新编译 |
| 找不到头文件 | 头文件不在编译器搜索路径中 | 用 `-I` 指定路径，或确认 `#include` 路径正确 |

## 本章小结

| 知识点 | 要点 |
|--------|------|
| 多文件好处 | 便于维护、协作、复用，编译速度更快 |
| 头文件（.h） | 放声明（函数签名、宏、typedef、extern 变量） |
| 源文件（.c） | 放实现（函数体、变量定义） |
| 声明 vs 定义 | 声明是"名片"（可以有多个），定义是"真人"（只能有一个） |
| #include "..." | 先在当前目录搜索，用于项目自己的头文件 |
| #include <...> | 在系统目录搜索，用于标准库头文件 |
| -I 选项 | 指定额外的头文件搜索路径 |
| extern | 声明变量在别处定义，用于跨文件共享全局变量 |
| static 函数 | 限制函数只在当前文件可见，用于封装 |
| undefined reference | 链接错误：找不到函数定义，检查编译命令和函数名拼写 |
| multiple definition | 链接错误：重复定义，检查头文件中是否有函数定义 |
| 头文件保护 | `#ifndef`/`#define`/`#endif` 防止重复包含 |
