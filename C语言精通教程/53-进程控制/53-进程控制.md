## 我在做什么？

到目前为止，我们写的所有程序都是"一个人干活"——一个进程从头执行到尾。但在真实操作系统中，一个程序可以"分身"出多个进程同时工作。网页服务器同时处理多个请求、Shell 执行命令时创建子进程，都是基于这个机制。

本章学习 C 语言中最重要的系统调用之一：`fork()`。它会**复制当前进程**，创建一个完全一样的子进程。父进程和子进程从 `fork()` 的下一行代码同时开始执行，但通过返回值区分彼此。

想象你是一个画家，正在画一幅画。`fork()` 就像你突然变出一个分身，他拿着和你一模一样的画笔、颜料和画布，从你变出他的那一刻继续画下去。你们俩画的是同一幅画的不同部分——最终通过 `wait()` 汇合。

`*此术语见附录F*`：fork、父进程、子进程、PID、僵尸进程、孤儿进程。

## 53.1 进程是什么

```
┌──────────────────────────────────────────────────────────┐
│                      操作系统                             │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐               │
│  │ 进程 A   │  │ 进程 B   │  │ 进程 C   │  ...          │
│  │ PID=1001 │  │ PID=1002 │  │ PID=1003 │               │
│  │          │  │          │  │          │               │
│  │ 代码段   │  │ 代码段   │  │ 代码段   │               │
│  │ 数据段   │  │ 数据段   │  │ 数据段   │               │
│  │ 堆       │  │ 堆       │  │ 堆       │               │
│  │ 栈       │  │ 栈       │  │ 栈       │               │
│  │ 文件描述符│ │ 文件描述符│ │ 文件描述符│               │
│  └──────────┘  └──────────┘  └──────────┘               │
└──────────────────────────────────────────────────────────┘
```

每个进程有：
- **PID**：进程ID，唯一标识
- **代码段**：程序的机器指令
- **数据段**：全局变量和静态变量
- **堆**：动态分配的内存
- **栈**：函数调用栈、局部变量
- **文件描述符表**：打开的文件

获取当前进程 PID：

```c
#include <stdio.h>
#include <unistd.h>  /* getpid(), getppid() */

int main(void) {
    printf("当前进程 PID: %d\n", getpid());
    printf("父进程 PID:   %d\n", getppid());
    return 0;
}
```

## 53.2 fork()：创建子进程

`fork()` 是 Unix/Linux 中创建新进程的唯一方式（底层都是它）。它做一件神奇的事：**调用一次，返回两次**。

```
fork() 调用前:
  ┌──────────┐
  │ 父进程    │
  │ PID=100  │
  │          │
  │ fork()   │  ← 即将调用
  └──────────┘

fork() 调用后:
  ┌──────────┐     ┌──────────┐
  │ 父进程    │     │ 子进程    │
  │ PID=100  │     │ PID=101  │
  │          │     │          │
  │ fork()   │     │ fork()   │
  │ 返回 101 │     │ 返回 0   │
  └──────────┘     └──────────┘
  (子进程PID)       (标识是子进程)
```

```c
#include <stdio.h>
#include <unistd.h>

int main(void) {
    printf("fork() 调用前...\n");

    pid_t pid = fork();

    if (pid < 0) {
        /* fork 失败（极少发生，如系统进程数达到上限） */
        perror("fork 失败");
        return 1;
    } else if (pid == 0) {
        /* 子进程：fork() 返回 0 */
        printf("  [子进程] PID=%d, 父PID=%d, fork()返回=%d\n",
               getpid(), getppid(), pid);
    } else {
        /* 父进程：fork() 返回子进程的 PID */
        printf("  [父进程] PID=%d, fork()返回子进程PID=%d\n",
               getpid(), pid);
    }

    /* 这行代码父子进程都会执行 */
    printf("  进程 %d 执行到这里\n", getpid());

    return 0;
}
```

运行输出（顺序可能不同，取决于调度）：
```
fork() 调用前...
  [父进程] PID=12345, fork()返回子进程PID=12346
  进程 12345 执行到这里
  [子进程] PID=12346, 父PID=12345, fork()返回=0
  进程 12346 执行到这里
```

### fork() 复制了什么？

子进程是父进程的**完整拷贝**（写时复制优化）：
- 代码段、数据段、堆、栈都被复制
- 文件描述符表被复制（共享文件偏移量）
- 但子进程拥有独立的地址空间——修改子进程的变量不影响父进程

```c
#include <stdio.h>
#include <unistd.h>

int main(void) {
    int x = 100;

    pid_t pid = fork();

    if (pid == 0) {
        /* 子进程：修改自己的 x */
        x = 200;
        printf("[子进程] x = %d (地址: %p)\n", x, (void*)&x);
    } else if (pid > 0) {
        /* 父进程：x 保持不变 */
        printf("[父进程] x = %d (地址: %p)\n", x, (void*)&x);
    }

    return 0;
}
```

输出：
```
[父进程] x = 100 (地址: 0x7ffc1234)
[子进程] x = 200 (地址: 0x7ffc1234)
```

注意：地址看起来相同，但它们是**不同的虚拟地址空间**中的相同虚拟地址。子进程修改 `x` 不会影响父进程。

## 53.3 wait()：父进程等待子进程

父进程通过 `wait()` 或 `waitpid()` 等待子进程结束，获取子进程的退出状态：

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>  /* wait() */

int main(void) {
    pid_t pid = fork();

    if (pid == 0) {
        /* 子进程 */
        printf("[子进程 %d] 开始工作...\n", getpid());
        sleep(2);  /* 模拟耗时操作 */
        printf("[子进程 %d] 工作完成，退出\n", getpid());
        exit(42);  /* 退出码 42 */
    } else if (pid > 0) {
        /* 父进程 */
        printf("[父进程] 等待子进程 %d 结束...\n", pid);

        int status;
        pid_t finished = wait(&status);

        printf("[父进程] 子进程 %d 已结束\n", finished);

        if (WIFEXITED(status)) {
            printf("[父进程] 子进程正常退出，退出码: %d\n",
                   WEXITSTATUS(status));
        } else if (WIFSIGNALED(status)) {
            printf("[父进程] 子进程被信号 %d 终止\n",
                   WTERMSIG(status));
        }
    }

    return 0;
}
```

### 检查退出状态的宏

| 宏 | 说明 |
|----|------|
| `WIFEXITED(status)` | 子进程是否正常退出（调用 `exit` 或 `return`） |
| `WEXITSTATUS(status)` | 获取退出码（`exit(n)` 中的 n，仅低 8 位） |
| `WIFSIGNALED(status)` | 子进程是否被信号杀死 |
| `WTERMSIG(status)` | 获取终止子进程的信号编号 |
| `WIFSTOPPED(status)` | 子进程是否被暂停 |

### waitpid：等待指定子进程

`wait()` 等待任意一个子进程，`waitpid()` 可以指定等待哪个子进程：

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>

int main(void) {
    pid_t pid1 = fork();

    if (pid1 == 0) {
        printf("[子进程1 %d] 开始\n", getpid());
        sleep(3);
        printf("[子进程1 %d] 结束\n", getpid());
        exit(10);
    }

    pid_t pid2 = fork();

    if (pid2 == 0) {
        printf("[子进程2 %d] 开始\n", getpid());
        sleep(1);
        printf("[子进程2 %d] 结束\n", getpid());
        exit(20);
    }

    /* 父进程：等待特定的子进程 */
    int status;

    printf("[父进程] 等待子进程2 (%d)...\n", pid2);
    waitpid(pid2, &status, 0);
    printf("[父进程] 子进程2 退出码: %d\n", WEXITSTATUS(status));

    printf("[父进程] 等待子进程1 (%d)...\n", pid1);
    waitpid(pid1, &status, 0);
    printf("[父进程] 子进程1 退出码: %d\n", WEXITSTATUS(status));

    return 0;
}
```

## 53.4 exec 系列：替换进程

`fork()` 创建子进程后，大多数情况下子进程不是继续执行父进程的代码，而是**替换成另一个程序**。这就是 `exec` 系列函数的作用：

```
fork()  +  exec()  =  运行一个新程序

┌──────────┐                    ┌──────────┐
│ 父进程    │  fork()            │ 子进程    │
│ (shell)  │ ───────────────>   │ (shell)  │
│          │                    │          │
│          │                    │ exec()   │
│          │                    │ ls -la   │
│          │  wait()            │          │
│          │ <───────────────   │ 输出结果  │
└──────────┘                    └──────────┘
```

这就是 Shell 执行命令的底层原理：`fork` 一个子进程，然后在子进程中 `exec` 你要执行的命令。

### exec 系列函数

```c
#include <unistd.h>

/* 六种 exec 变体，核心区别在于参数传递方式 */

/* l: 参数以可变参数列表传递（list），以 NULL 结尾 */
int execl (const char *path, const char *arg0, ..., NULL);
int execlp(const char *file, const char *arg0, ..., NULL);  /* p = 搜索 PATH */
int execle(const char *path, const char *arg0, ..., NULL, char *envp[]);  /* e = 环境变量 */

/* v: 参数以数组传递（vector） */
int execv (const char *path, char *const argv[]);
int execvp(const char *file, char *const argv[]);
int execvpe(const char *file, char *const argv[], char *envp[]);
```

**如果 exec 成功，它不会返回**（因为当前进程已经被替换了）。如果失败，返回 -1。

### 示例：执行 ls 命令

```c
#include <stdio.h>
#include <unistd.h>
#include <sys/wait.h>
#include <stdlib.h>

int main(void) {
    pid_t pid = fork();

    if (pid == 0) {
        /* 子进程：用 execlp 执行 ls */
        printf("[子进程] 即将执行 ls -la\n");
        execlp("ls", "ls", "-la", NULL);

        /* 如果 exec 失败，才会执行到这里 */
        perror("execlp 失败");
        exit(1);
    } else if (pid > 0) {
        /* 父进程：等待子进程 */
        printf("[父进程] 等待 ls 命令完成...\n");
        wait(NULL);
        printf("[父进程] ls 命令已完成\n");
    }

    return 0;
}
```

### 使用 execv 传递参数数组

```c
#include <stdio.h>
#include <unistd.h>
#include <sys/wait.h>
#include <stdlib.h>

int main(void) {
    pid_t pid = fork();

    if (pid == 0) {
        char *args[] = {"ls", "-la", "/home", NULL};
        execvp("ls", args);

        perror("execvp 失败");
        exit(1);
    } else if (pid > 0) {
        wait(NULL);
        printf("命令执行完毕\n");
    }

    return 0;
}
```

## 53.5 僵尸进程与孤儿进程

### 僵尸进程（Zombie）

当子进程先于父进程结束，但父进程还没有调用 `wait()` 来"收尸"时，子进程变成**僵尸进程**。它已不占用内存和 CPU，但在进程表中保留一条记录（PID 和退出状态）。

```
父进程                          子进程
  │                               │
  │ fork() ───────────────────>  │
  │                               │ 工作...
  │ 做其他事...                   │ exit()
  │                               X (进程结束，但变成僵尸)
  │                               │
  │ 做其他事...                   │ 等待父进程 wait()
  │                               │
  │ wait() <───────────────────  │ (父进程读取退出状态)
  │                               X (僵尸被清除)
```

```c
#include <stdio.h>
#include <unistd.h>
#include <stdlib.h>

int main(void) {
    pid_t pid = fork();

    if (pid == 0) {
        printf("[子进程 %d] 即将退出\n", getpid());
        exit(0);
    } else {
        printf("[父进程] 子进程 %d 已创建\n", pid);
        printf("[父进程] 按回车查看僵尸进程（用 ps aux | grep Z）\n");
        getchar();  /* 暂停，此时子进程是僵尸 */

        /* 现在收尸 */
        wait(NULL);
        printf("[父进程] 子进程已被回收\n");
        getchar();
    }

    return 0;
}
```

在另一个终端运行 `ps aux | grep Z` 可以看到僵尸进程（状态为 `Z`）。

### 孤儿进程（Orphan）

当父进程先于子进程结束时，子进程变成**孤儿进程**。孤儿进程会被 init 进程（PID=1）收养，init 会自动调用 `wait()` 清理它们。

```
父进程                          子进程
  │                               │
  │ fork() ───────────────────>  │
  │                               │ 工作...
  │ exit()                        │
  X (父进程结束)                  │ 变成孤儿进程
                                  │
                                  │ 被 init(PID=1) 收养
                                  │
                                  │ exit()
                                  X (init 自动回收)
```

```c
#include <stdio.h>
#include <unistd.h>
#include <stdlib.h>

int main(void) {
    pid_t pid = fork();

    if (pid == 0) {
        /* 子进程多活一会儿 */
        printf("[子进程 %d] 父进程 PID=%d\n", getpid(), getppid());
        sleep(5);
        printf("[子进程 %d] 现在的父进程 PID=%d\n", getpid(), getppid());
        exit(0);
    } else {
        printf("[父进程 %d] 即将退出，子进程变成孤儿\n", getpid());
        exit(0);  /* 父进程先退出 */
    }

    return 0;
}
```

输出：
```
[父进程 1000] 即将退出，子进程变成孤儿
[子进程 1001] 父进程 PID=1000
[子进程 1001] 现在的父进程 PID=1    <-- 被 init 收养
```

## 53.6 完整代码清单

```c
/*
 * process_demo.c —— 进程控制完整演示
 * 编译: gcc -std=c99 -Wall -o proc_demo process_demo.c
 * 注意: 仅在 Linux/macOS 上运行，需要 POSIX 支持
 */

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/wait.h>
#include <errno.h>
#include <string.h>

/* 演示 fork 基本用法 */
void demo_fork(void) {
    printf("\n===== 1. fork() 基本用法 =====\n");

    pid_t pid = fork();

    if (pid < 0) {
        perror("fork 失败");
        return;
    } else if (pid == 0) {
        printf("  [子进程 %d] 我是子进程，fork() 返回 %d\n",
               getpid(), pid);
    } else {
        printf("  [父进程 %d] 我是父进程，fork() 返回子进程 PID=%d\n",
               getpid(), pid);
        wait(NULL);  /* 等待子进程结束 */
    }
}

/* 演示父子进程变量独立 */
void demo_independent_memory(void) {
    printf("\n===== 2. 父子进程变量独立 =====\n");

    int counter = 0;
    pid_t pid = fork();

    if (pid == 0) {
        counter = 100;
        printf("  [子进程] counter = %d\n", counter);
        exit(0);
    } else {
        /* 父进程等待子进程结束后再打印 */
        wait(NULL);
        printf("  [父进程] counter = %d (不受子进程修改影响)\n", counter);
    }
}

/* 演示多个子进程 */
void demo_multiple_children(void) {
    printf("\n===== 3. 多个子进程 =====\n");

    for (int i = 0; i < 3; i++) {
        pid_t pid = fork();

        if (pid == 0) {
            printf("  [子进程 %d] 第 %d 个子进程\n", getpid(), i + 1);
            sleep(1 + i);  /* 不同子进程休眠不同时长 */
            exit(i + 10);
        }
    }

    /* 父进程等待所有子进程 */
    for (int i = 0; i < 3; i++) {
        int status;
        pid_t finished = wait(&status);
        printf("  [父进程] 子进程 %d 结束，退出码: %d\n",
               finished, WEXITSTATUS(status));
    }
}

/* 演示 exec：执行外部命令 */
void demo_exec(void) {
    printf("\n===== 4. exec 执行外部命令 =====\n");

    pid_t pid = fork();

    if (pid == 0) {
        /* 执行 echo 命令 */
        printf("  [子进程] 即将执行 'echo Hello from exec!'\n");
        execlp("echo", "echo", "Hello", "from", "exec!", NULL);

        /* 如果到这里，说明 exec 失败了 */
        perror("execlp 失败");
        exit(1);
    } else {
        wait(NULL);
        printf("  [父进程] 外部命令执行完毕\n");
    }
}

/* 演示 waitpid 等待特定子进程 */
void demo_waitpid(void) {
    printf("\n===== 5. waitpid 等待特定子进程 =====\n");

    pid_t child1 = fork();
    if (child1 == 0) {
        sleep(2);
        printf("  [子进程1 %d] 结束\n", getpid());
        exit(100);
    }

    pid_t child2 = fork();
    if (child2 == 0) {
        sleep(1);
        printf("  [子进程2 %d] 结束\n", getpid());
        exit(200);
    }

    /* 先等待 child2（它先结束） */
    int status;
    printf("  [父进程] 等待子进程2 (%d)...\n", child2);
    waitpid(child2, &status, 0);
    printf("  [父进程] 子进程2 退出码: %d\n", WEXITSTATUS(status));

    printf("  [父进程] 等待子进程1 (%d)...\n", child1);
    waitpid(child1, &status, 0);
    printf("  [父进程] 子进程1 退出码: %d\n", WEXITSTATUS(status));
}

int main(void) {
    printf("主进程 PID: %d, 父进程 PID: %d\n", getpid(), getppid());

    demo_fork();
    demo_independent_memory();
    demo_multiple_children();
    demo_exec();
    demo_waitpid();

    printf("\n===== 全部演示完成 =====\n");
    return 0;
}
```

## 我做得对不对？

1. `demo_fork()`：父子进程各自打印一行，`fork()` 返回值不同（子进程为 0，父进程为子进程 PID）。
2. `demo_independent_memory()`：子进程修改 `counter` 为 100，父进程的 `counter` 仍为 0。
3. `demo_multiple_children()`：3 个子进程以不同顺序结束，父进程正确回收所有子进程。
4. `demo_exec()`：`execlp` 成功执行 `echo` 命令，输出 "Hello from exec!"。
5. `demo_waitpid()`：先收回子进程2（退出码 200），后收回子进程1（退出码 100）。

---

## 不对怎么办？

### 常见Bug 1：fork 后不区分父子进程

```c
/* ❌ 没有区分 pid == 0 和 pid > 0，代码被父子进程各执行一遍 */
pid_t pid = fork();
printf("这段代码被执行了两次！\n");  /* 父子进程各执行一次 */
```

✅ 始终用 `if (pid == 0) { ... } else if (pid > 0) { ... }` 区分。

### 常见Bug 2：父进程不 wait 导致僵尸进程

```c
/* ❌ 父进程不 wait，子进程变成僵尸，累积多了会导致系统无法创建新进程 */
pid_t pid = fork();
if (pid == 0) {
    exit(0);
}
/* 父进程不做 wait，直接继续运行 */
```

✅ 每个子进程都应该被 `wait()` 或 `waitpid()` 回收。

### 常见Bug 3：多次 fork 导致进程爆炸

```c
/* ❌ 循环中 fork 但没有正确隔离，子进程也会继续 fork */
for (int i = 0; i < 3; i++) {
    fork();  /* 如果子进程也继续循环，会创建 2^3 = 8 个进程！ */
}
```

✅ 在子进程分支中 `exit(0)` 或 `break`，避免子进程继续循环。

### 常见Bug 4：exec 后还有代码

```c
/* ❌ exec 成功后不会返回，但万一失败会执行后续代码 */
execlp("nonexistent", "nonexistent", NULL);
printf("这行代码在 exec 失败时执行，但逻辑混乱\n");
```

✅ exec 后紧跟 `perror()` 和 `exit(1)`。

### 常见Bug 5：wait 和 waitpid 混用时忽略 SIGCHLD

```c
/* ❌ 如果设置了 SIGCHLD 信号处理，wait 可能被信号中断 */
pid_t finished = wait(&status);
if (finished == -1 && errno == EINTR) {
    /* 被信号中断，需要重新 wait */
}
```

✅ 检查 `wait` 返回 -1 时 `errno` 是否为 `EINTR`，如果是则重试。

## 本章小结

| 要点 | 说明 |
|------|------|
| 进程 | 运行中的程序实例，有独立的地址空间和 PID |
| PID | 进程ID，唯一标识一个进程 |
| fork() | 创建子进程，调用一次返回两次 |
| fork() 返回值 | 父进程返回子进程 PID，子进程返回 0，失败返回 -1 |
| 父子进程关系 | 子进程是父进程的拷贝，地址空间独立 |
| wait() | 等待任意子进程结束，回收退出状态 |
| waitpid() | 等待指定 PID 的子进程 |
| exec 系列 | 替换当前进程的代码为另一个程序 |
| fork + exec | 创建新进程并运行新程序的标准模式 |
| 僵尸进程 | 子进程已结束但父进程未 wait，残留在进程表中 |
| 孤儿进程 | 父进程先结束，子进程被 init(PID=1) 收养 |
