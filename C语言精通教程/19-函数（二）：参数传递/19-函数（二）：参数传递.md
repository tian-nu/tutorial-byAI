## 我在做什么？

上一章你学会了"造函数"——定义函数、调用函数、写递归。但还有一个关键问题没解决：当你把一个变量传给函数，函数里修改了它，外面会变吗？

答案是：**C语言中，不会变。** 这就是本章的核心——*传值调用* *此术语见附录F*。读完这一章，你会理解为什么C语言函数不能直接修改外部变量，以及怎么用数组作为函数参数。

---

## 一、C语言的核心规则：只有传值调用

### 1.1 一个让你惊讶的实验

先看代码，猜猜输出是什么：

```c
#include <stdio.h>

void change(int x) {
    x = 999;                // 修改 x
    printf("函数内部：x = %d\n", x);
}

int main() {
    int a = 10;
    change(a);              // 把 a 传进去
    printf("函数外部：a = %d\n", a);
    return 0;
}
```

运行结果：
```text
函数内部：x = 999
函数外部：a = 10       ← 没变！
```

**a 还是 10，没有变成 999。** 为什么？

### 1.2 传值调用的比喻：复印机

想象你有一份纸质文档，上面写着数字 `10`。你把它复印一份，在复印件上改成 `999`。原件上的数字变了吗？没有——原件还是 `10`。

```text
main 中的 a                    change 中的 x
  ┌──────┐                     ┌──────┐
  │  10  │  ── 复印 ──→        │  10  │  → 改成 999 →  ┌──────┐
  └──────┘                     └──────┘                 │ 999  │
     ↑                                                 └──────┘
  原件不变                                            复印件改了
```

**C语言的参数传递，就是把变量的值拷贝一份，传给函数。** 函数内修改的是这份拷贝，原件不受影响。这就是 *传值调用* *此术语见附录F*（Call by Value）。

### 1.3 传值调用的严格定义

```text
调用 change(a) 时：

1. 计算 a 的值 → 10
2. 创建新变量 x（形参），把 10 拷贝进去
3. 函数内对 x 的任何操作，与 a 无关
4. 函数返回后，x 被销毁（它的生命周期结束）
```

**C语言没有其他语言的"传引用"或"传地址"——只有传值。** 后面你会学到，指针也只是"传了一个地址值"而已。

## 二、形参 vs 实参

### 2.1 两个概念

```c
int add(int a, int b)    // a 和 b 是形参（形式参数）
{
    return a + b;
}

int main() {
    int x = 3, y = 5;
    int result = add(x, y);  // x 和 y 是实参（实际参数）
    return 0;
}
```

| | *形参* *此术语见附录F* | *实参* *此术语见附录F* |
|---|---|---|
| 全称 | 形式参数（Parameter） | 实际参数（Argument） |
| 位置 | 函数定义的小括号里 | 函数调用的小括号里 |
| 是什么 | 变量（在函数内部使用） | 值（传给函数） |
| 例子 | `int a, int b` | `x, y` |

### 2.2 形参是局部变量

形参和函数内部定义的变量没有本质区别——它们都是局部变量，函数返回后就消失了。

```c
void func(int param) {        // param 是形参，也是局部变量
    int local = 100;          // local 是局部变量
    param = 200;              // 修改形参，不影响实参
    local = 300;              // 修改局部变量，和外面无关
}
```

## 三、为什么需要指针？（预告）

### 3.1 问题：我确实想修改外部变量

上面的例子中，你只是想改一个数，但函数帮不了你。如果确实需要在函数里修改外部变量，怎么办？

```c
// 需求：写一个函数，把传入的变量变成它的两倍
void double_it(int x) {
    x = x * 2;     // 这样没用！x 是拷贝
}
```

答案：你需要**把变量的地址传给函数**，让函数通过地址找到原件。这就是**指针**的用武之地。

```c
void double_it(int *p) {     // 参数是指针（接收地址）
    *p = *p * 2;             // 通过地址修改原件
}

int main() {
    int a = 10;
    double_it(&a);           // 传 a 的地址
    printf("%d\n", a);       // 输出 20，真的改了！
    return 0;
}
```

这是第25-26章的内容，这里先留个印象。现在的重点是：**理解"传值"这个规则本身。**

## 四、数组作为函数参数

### 4.1 数组传参的奇怪行为

数组作为函数参数时，表现得和普通变量不一样：

```c
#include <stdio.h>

void modify_array(int arr[], int size) {
    arr[0] = 999;            // 修改数组的第一个元素
}

int main() {
    int nums[] = {1, 2, 3, 4, 5};
    modify_array(nums, 5);
    printf("nums[0] = %d\n", nums[0]);  // 输出 999！改了！
    return 0;
}
```

等等，不是说C语言只有传值调用吗？为什么数组被改了？

### 4.2 真相：数组传的是地址

C语言确实只有传值调用。但**数组作为参数时，传的不是数组的全部内容，而是数组首元素的地址**。

```text
普通变量传参：                    数组传参：
  a = 10                            nums = {1, 2, 3, 4, 5}
     |                                    |
  拷贝值 10 给形参                   拷贝首元素地址 给形参
     |                                    |
  形参有独立的 10                    形参指向同一个数组
```

所以，函数内的 `arr` 和外面的 `nums` **指向同一块内存**。修改 `arr[0]` 就是修改 `nums[0]`。

### 4.3 三种写法，同一个意思

数组作为函数参数，以下三种写法完全等价：

```c
void func1(int arr[]);         // 写法一：方括号，最常见
void func2(int arr[10]);       // 写法二：指定大小（编译器忽略这个数字）
void func3(int *arr);          // 写法三：指针（本质）
```

三种写法中，`arr` 都是一个指针，指向数组的首元素。推荐用第一种（`int arr[]`），因为它清晰表达"这里期望一个数组"。

### 4.4 数组传参的陷阱：不知道大小

数组传进函数后，**函数内部无法知道数组有多大**。`sizeof(arr)` 返回的是指针的大小（通常是 8 字节），而不是整个数组的大小。

```c
#include <stdio.h>

void print_array(int arr[]) {
    printf("sizeof(arr) = %zu\n", sizeof(arr));  // 总是 8（64位系统）
    // 不能这样遍历！
    // for (int i = 0; i < sizeof(arr)/sizeof(arr[0]); i++)  // 错误！
}

int main() {
    int nums[] = {1, 2, 3, 4, 5};
    print_array(nums);
    return 0;
}
```

所以，**数组作为参数时，必须同时传递数组的长度**：

```c
void print_array(int arr[], int size) {   // 正确：size 是额外参数
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}
```

### 4.5 完整示例：数组求和

```c
#include <stdio.h>

// 计算数组元素之和
int sum_array(int arr[], int size) {
    int total = 0;
    for (int i = 0; i < size; i++) {
        total += arr[i];
    }
    return total;
}

// 把数组每个元素翻倍
void double_array(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        arr[i] *= 2;
    }
}

int main() {
    int nums[] = {1, 2, 3, 4, 5};
    int size = sizeof(nums) / sizeof(nums[0]);  // 在 main 里能算大小

    printf("原始数组：");
    for (int i = 0; i < size; i++) {
        printf("%d ", nums[i]);
    }
    printf("\n");

    printf("数组之和：%d\n", sum_array(nums, size));

    double_array(nums, size);
    printf("翻倍之后：");
    for (int i = 0; i < size; i++) {
        printf("%d ", nums[i]);
    }
    printf("\n");

    return 0;
}
```

## 五、常见错误与排错

### 5.1 以为函数能修改外部变量

❌ 错误示例：
```c
void swap(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
    // a 和 b 是拷贝，不影响外面的变量
}

int main() {
    int x = 3, y = 5;
    swap(x, y);
    printf("x=%d, y=%d\n", x, y);  // 还是 3 和 5！
    return 0;
}
```

✅ 正确示例（需要指针，第26章会详细讲）：
```c
void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main() {
    int x = 3, y = 5;
    swap(&x, &y);
    printf("x=%d, y=%d\n", x, y);  // x=5, y=3
    return 0;
}
```

### 5.2 在函数内用 sizeof 求数组大小

❌ 错误示例：
```c
void print_array(int arr[]) {
    int size = sizeof(arr) / sizeof(arr[0]);  // 错误！arr 在此是指针
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);
    }
}
```

✅ 正确示例：
```c
void print_array(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);
    }
}
```

### 5.3 形参和实参类型不匹配

❌ 错误示例：
```c
void func(int arr[]) { ... }

int main() {
    int x = 10;
    func(x);    // 错误：传了一个 int，但函数期望 int[]
    return 0;
}
```

✅ 正确示例：
```c
void func(int arr[]) { ... }

int main() {
    int nums[] = {1, 2, 3};
    func(nums);  // 正确：传数组
    return 0;
}
```

## 我做得对不对？

用以下程序验证你对参数传递的理解：

```c
#include <stdio.h>

// 测试1：普通变量传参
void try_change(int x) {
    x = 999;
}

// 测试2：数组传参
void modify_first(int arr[], int size) {
    (void)size;  // 不用 size，只是演示
    arr[0] = 999;
}

// 测试3：数组求和
int sum(int arr[], int size) {
    int total = 0;
    for (int i = 0; i < size; i++) {
        total += arr[i];
    }
    return total;
}

int main() {
    // 测试1
    int a = 10;
    try_change(a);
    printf("测试1：a = %d (应为 10)\n", a);

    // 测试2
    int nums[] = {1, 2, 3, 4, 5};
    modify_first(nums, 5);
    printf("测试2：nums[0] = %d (应为 999)\n", nums[0]);

    // 测试3
    int nums2[] = {1, 2, 3, 4, 5};
    printf("测试3：sum = %d (应为 15)\n", sum(nums2, 5));

    return 0;
}
```

编译运行：
```bash
$ gcc -Wall -o test test.c
$ ./test
测试1：a = 10 (应为 10)
测试2：nums[0] = 999 (应为 999)
测试3：sum = 15 (应为 15)
```

全部输出与预期一致，说明你理解了。

## 不对怎么办？

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| 函数改了变量，外面没变 | 忘记了传值调用规则 | 需要修改外部变量时，传指针（第26章） |
| 数组在函数内遍历结果不对 | 在函数内用 sizeof 求数组大小 | 额外传一个 size 参数 |
| 编译警告 `incompatible pointer type` | 传了 int 但函数期望 int[] | 检查实参和形参类型是否匹配 |
| 数组内容被意外修改 | 忘了数组传参是传地址 | 如果不想修改数组，函数内不要修改 arr 元素 |

## 本章小结

| 知识点 | 一句话总结 |
|--------|-----------|
| 传值调用 | C语言唯一的参数传递方式，函数收到的是实参的拷贝 |
| 形参 vs 实参 | 形参是定义时的变量名，实参是调用时传的值 |
| 普通变量传参 | 传的是值的拷贝，函数内修改不影响外部 |
| 数组传参 | 传的是首元素地址，函数内修改会影响外部 |
| 数组传参的陷阱 | 函数内无法用 sizeof 求数组大小，必须额外传 size |
| 为什么需要指针 | 如果想让函数修改外部变量，需要传地址 |
