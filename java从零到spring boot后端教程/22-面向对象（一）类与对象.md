# 第22章 · 面向对象（一）类与对象

> "之前写代码，你把数据（变量）和操作（方法）分开写，就像一个公司只有一堆散乱的零件和独立的工人——能干活，但效率低。面向对象编程（OOP）告诉你：把相关的数据和操作打包成一个整体。类是设计图，对象是按图造出来的实体。从现在开始，你看待代码的方式将彻底改变。"

---

## 22.1 为什么需要面向对象？

### 用生活来比喻

你去银行办理业务：
- 有一个"账户"实体：它有自己的余额、账号、户主姓名
- 账户能做"存钱"和"取钱"的操作
- 存钱和取钱时，钱不能凭空出现或消失——余额必须正确变化

如果用之前学的"过程式"写法：

```java
// 过程式：数据和操作分离
double balance1 = 1000.0;
double balance2 = 500.0;

void deposit(double amount) { balance1 += amount; }
void withdraw(double amount) { balance1 -= amount; }
```

这有几个严重问题：
1. **数据不安全**：任何人都能直接改 `balance1 = -99999`，绕开存取款逻辑
2. **数据和操作没有绑定**：你不知道 `deposit()` 操作的是哪个账户的余额
3. **难以扩展**：来了第三个账户怎么办？再声明 `balance3`，再写一套方法？

**面向对象**的解法：

```java
class Account {
    private double balance;  // 余额藏起来，外部不能直接碰

    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }

    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
        }
    }

    public double getBalance() {
        return balance;
    }
}
```

类（此术语需进附录）`Account` 把"余额"这个数据和"存钱/取钱"这些操作**封装**（此术语需进附录）在一起。外部代码不能直接改余额，必须通过 `deposit()` 和 `withdraw()` 方法——这就保证了安全性。

**面向对象的精髓**：把现实世界的事物**映射**为代码里的类。车是类，你的那辆具体的车是对象。人类是类，你是对象，我是对象。

## 22.2 类：设计图

类是对象的"设计图"或"模板"。它定义了这种对象拥有什么数据（字段）和能做什么事（方法）。

```java
public class Student {
    // ========== 字段（Field）：类拥有的数据 ==========
    private String name;       // 姓名
    private String studentId;  // 学号
    private double score;      // 成绩

    // ========== 构造器（Constructor）：如何创建一个对象 ==========
    public Student(String name, String studentId, double score) {
        this.name = name;
        this.studentId = studentId;
        this.score = score;
    }

    // ========== 方法（Method）：类能做什么 ==========
    public void displayInfo() {
        System.out.println("姓名：" + name);
        System.out.println("学号：" + studentId);
        System.out.println("成绩：" + score);
    }

    // ========== Getter和Setter：安全访问字段 ==========
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public String getStudentId() { return studentId; }
    public void setStudentId(String studentId) { this.studentId = studentId; }

    public double getScore() { return score; }
    public void setScore(double score) { this.score = score; }
}
```

### 逐元素拆解

| 元素 | 作用 | 比喻 |
|------|------|------|
| 字段（Field） | 存数据 | 设计图上标记的尺寸、材质 |
| 构造器（Constructor）此术语需进附录 | 创建对象时自动执行的方法，用来初始化字段 | 工厂里的流水线——给这个产品装零件 |
| 方法（Method） | 对象的行为 | 这个产品能做什么 |
| Getter/Setter | 安全读写私有字段 | 银行窗口——你不直接进金库，通过窗口存取 |

## 22.3 对象：按照设计图造出来的实体

类只是设计图，没有实体。**用 `new` 关键字按照设计图创建出来的具体实例，叫对象**（此术语需进附录）。

```java
// 创建对象：new 类名(参数)
Student s1 = new Student("张三", "2024001", 85.5);
Student s2 = new Student("李四", "2024002", 92.0);
Student s3 = new Student("王五", "2024003", 58.0);

// 调用方法
s1.displayInfo();
// 输出：
// 姓名：张三
// 学号：2024001
// 成绩：85.5

// 通过Getter获取数据
System.out.println(s2.getName());  // 李四

// 通过Setter修改数据
s3.setScore(62.0);
System.out.println(s3.getScore());  // 62.0
```

**类的声明和文件的关系**：
- 一个 `.java` 文件里通常只有一个public类，类名等于文件名
- 上面的代码应该放在 `Student.java` 文件里
- 使用它的代码放在另一个文件（如 `Main.java`）里

## 22.4 构造器：对象的"出生仪式"

构造器（Constructor。此术语需进附录）是一个**特殊的方法**，名字必须和类名一样，**没有返回类型**（连 `void` 都没有）。它在 `new` 对象时自动调用。

```java
public class Student {
    private String name;
    private String studentId;
    private double score;

    // 带参数的构造器
    public Student(String name, String studentId, double score) {
        this.name = name;
        this.studentId = studentId;
        this.score = score;
    }
}
```

### 默认构造器

如果你**一个构造器都不写**，Java会自动提供一个**无参默认构造器**（什么都不做）：

```java
public class Student {
    private String name;

    // Java自动生成：public Student() { }
}

// 你可以这样创建对象
Student s = new Student();  // 可以！用了默认构造器
```

但是，**一旦你写了任何一个构造器，默认构造器就消失了**：

```java
public class Student {
    private String name;

    public Student(String name) {
        this.name = name;
    }
}

Student s = new Student();          // ❌ 编译错误！不存在无参构造器
Student s = new Student("张三");    // ✅
```

> ❌ **常见错误**：写了有参构造器后忘记了这一点，仍然用 `new Student()` 创建对象，编译器报错"找不到无参构造器"。如果你希望两种方式都能创建对象，就**手动补齐无参构造器**。

### 构造器重载

一个类可以有多个构造器（参数列表不同）：

```java
public class Student {
    private String name;
    private String studentId;
    private double score;

    // 无参构造器（默认值）
    public Student() {
        this.name = "未知";
        this.studentId = "0000000";
        this.score = 0.0;
    }

    // 只传姓名
    public Student(String name) {
        this.name = name;
        this.studentId = "0000000";
        this.score = 0.0;
    }

    // 全部参数
    public Student(String name, String studentId, double score) {
        this.name = name;
        this.studentId = studentId;
        this.score = score;
    }
}
```

## 22.5 this关键字：指代"当前对象"

`this` 是Java的一个关键字，**永远指向当前对象自身**。最常见的三个用途：

### 用途一：区分字段和局部变量

```java
public Student(String name, String studentId, double score) {
    this.name = name;            // this.name是字段，右边name是参数
    this.studentId = studentId;
    this.score = score;
}
```

如果不用this：
```java
public Student(String name, String studentId, double score) {
    name = name;          // 都是参数！字段根本没赋到值！
}
```

### 用途二：在构造器中调用另一个构造器

```java
public Student() {
    this("未知", "0000000", 0.0);  // 调用三参数构造器
}

public Student(String name) {
    this(name, "0000000", 0.0);    // 调用三参数构造器
}

public Student(String name, String studentId, double score) {
    this.name = name;
    this.studentId = studentId;
    this.score = score;
}
```

`this(...)` 调用另一个构造器必须是构造器的**第一条语句**。

### 用途三：返回当前对象本身（链式调用）

```java
public Student setName(String name) {
    this.name = name;
    return this;
}

// 链式调用
s.setName("张三").setScore(95.0);
```

## 22.6 封装：把数据藏起来

封装（Encapsulation。此术语需进附录）是面向对象的核心原则之一。它的意思是：**字段用private隐藏，对外提供public的方法来访问**。

### 访问修饰符

| 修饰符 | 类内部 | 同包 | 子类 | 任何地方 |
|--------|--------|------|------|----------|
| `private` | ✅ | ❌ | ❌ | ❌ |
| 默认（不写） | ✅ | ✅ | ❌ | ❌ |
| `protected` | ✅ | ✅ | ✅ | ❌ |
| `public` | ✅ | ✅ | ✅ | ✅ |

最常用的模式：**字段用 `private`，方法用 `public`**。

```java
public class BankAccount {
    private double balance;  // 余额：private，外部不能直接碰

    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }

    public boolean withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            return true;
        }
        return false;
    }

    // 只读——不提供setter
    public double getBalance() {
        return balance;
    }
}
```

封装的好处：
1. **数据安全**：外部不能直接把 `balance = -100000`
2. **逻辑内聚**：所有关于余额的规则都在 `BankAccount` 类里
3. **修改不影响外部**：哪天你想在取款时加个短信通知，只改 `withdraw()` 方法就够了

## 22.7 JavaBean规范

在Java企业开发中，有一种特殊的类叫**JavaBean**。它遵循以下规范：

```java
public class User {
    // 1. 所有字段用 private
    private String username;
    private String password;
    private int age;

    // 2. 必须有一个无参构造器
    public User() {
    }

    // 3. 可以有带参构造器
    public User(String username, String password, int age) {
        this.username = username;
        this.password = password;
        this.age = age;
    }

    // 4. 每个字段提供Getter和Setter（命名严格遵循：getXxx / setXxx）
    public String getUsername() {
        return username;
    }

    public void setUsername(String username) {
        this.username = username;
    }

    public String getPassword() {
        return password;
    }

    public void setPassword(String password) {
        this.password = password;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }
}
```

**为什么JavaBean这么重要？** 因为几乎所有Java框架（Spring Boot、MyBatis、Jackson等）都依赖JavaBean规范来操作对象——框架通过 `getXxx()` 读取你的数据，通过 `setXxx()` 写入数据。如果你的方法命名不符合规范（比如 `int age()` 获取年龄而不是 `getAge()`），框架就找不到它。

> ❌ **常见错误**：boolean类型的Getter特殊：`private boolean active;` 对应的Getter应该叫 `isActive()` 而不是 `getActive()`。框架在反射时会按这个规则去匹配。

## 22.8 静态成员：属于类本身，不属于任何对象

用 `static` 修饰的字段和方法，属于**类本身**，不属于任何一个具体的对象。所有对象共享同一个静态变量。

```java
public class Student {
    private String name;
    private static int totalCount = 0;  // 静态字段：存一共有多少个学生

    public Student(String name) {
        this.name = name;
        totalCount++;  // 每创建一个学生，总数+1
    }

    public static int getTotalCount() {  // 静态方法
        return totalCount;
    }
}

Student s1 = new Student("张三");
Student s2 = new Student("李四");
Student s3 = new Student("王五");

// 静态方法通过类名直接调用，不需要对象
System.out.println(Student.getTotalCount());  // 3
```

**内存理解**：静态字段存在方法区（Method Area），所有对象共享同一份。实例字段每个对象都有一份独立的。

## 学以致用：学生成绩管理系统（命令行版）

> "前面21章学了那么多语法——变量、运算符、if-else、循环、数组——现在终于可以把它们串联起来了。这是你的第一个完整可运行小项目，写完它，你会第一次真正感觉到'我能用Java做点东西了'。"

### 项目需求

写一个命令行程序，功能：
1. 添加学生（姓名、学号、成绩）
2. 显示所有学生信息
3. 统计平均成绩
4. 显示最高/最低分学生
5. 显示及格和不及格学生名单

### 完整代码

将以下代码放在 `StudentManager.java` 文件里：

```java
import java.util.Scanner;

public class StudentManager {

    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        Student[] students = new Student[100];  // 最多存100个学生
        int count = 0;  // 当前学生数量

        while (true) {
            System.out.println("\n========== 学生成绩管理系统 ==========");
            System.out.println("1. 添加学生");
            System.out.println("2. 显示所有学生");
            System.out.println("3. 统计平均成绩");
            System.out.println("4. 显示最高分与最低分学生");
            System.out.println("5. 显示及格与不及格学生");
            System.out.println("0. 退出");
            System.out.print("请选择操作：");

            int choice = scanner.nextInt();
            scanner.nextLine();  // 吃掉换行符

            if (choice == 0) {
                System.out.println("谢谢使用，再见！");
                break;
            }

            switch (choice) {
                case 1 -> addStudent(scanner, students, count);
                case 2 -> showAllStudents(students, count);
                case 3 -> showAverageScore(students, count);
                case 4 -> showMaxMinStudent(students, count);
                case 5 -> showPassFailStudents(students, count);
                default -> System.out.println("无效选项，请重新选择。");
            }

            // 重新计算count（因为addStudent可能改了）
            count = getActualCount(students);
        }

        scanner.close();
    }

    private static void addStudent(Scanner scanner, Student[] students, int count) {
        if (count >= students.length) {
            System.out.println("学生数量已达上限（100人），无法继续添加。");
            return;
        }

        System.out.print("请输入姓名：");
        String name = scanner.nextLine();

        System.out.print("请输入学号：");
        String studentId = scanner.nextLine();

        System.out.print("请输入成绩（0-100）：");
        double score = scanner.nextDouble();
        scanner.nextLine();  // 吃掉换行符

        if (score < 0 || score > 100) {
            System.out.println("成绩必须在0到100之间，添加失败。");
            return;
        }

        students[count] = new Student(name, studentId, score);
        System.out.println("添加成功！");
    }

    private static void showAllStudents(Student[] students, int count) {
        if (count == 0) {
            System.out.println("暂无学生数据。");
            return;
        }

        System.out.println("\n========== 所有学生信息 ==========");
        System.out.printf("%-6s %-10s %-10s %-6s %-6s\n", "序号", "姓名", "学号", "成绩", "状态");
        System.out.println("-----------------------------------");
        for (int i = 0; i < count; i++) {
            Student s = students[i];
            String status = (s.getScore() >= 60) ? "及格" : "不及格";
            System.out.printf("%-6d %-10s %-10s %-6.1f %-6s\n",
                    (i + 1), s.getName(), s.getStudentId(), s.getScore(), status);
        }
    }

    private static void showAverageScore(Student[] students, int count) {
        if (count == 0) {
            System.out.println("暂无学生数据。");
            return;
        }

        double total = 0;
        for (int i = 0; i < count; i++) {
            total += students[i].getScore();
        }
        double average = total / count;
        System.out.printf("共 %d 名学生，平均成绩：%.1f\n", count, average);
    }

    private static void showMaxMinStudent(Student[] students, int count) {
        if (count == 0) {
            System.out.println("暂无学生数据。");
            return;
        }

        Student maxStudent = students[0];
        Student minStudent = students[0];

        for (int i = 1; i < count; i++) {
            if (students[i].getScore() > maxStudent.getScore()) {
                maxStudent = students[i];
            }
            if (students[i].getScore() < minStudent.getScore()) {
                minStudent = students[i];
            }
        }

        System.out.println("最高分学生：");
        System.out.printf("  姓名：%s  学号：%s  成绩：%.1f\n",
                maxStudent.getName(), maxStudent.getStudentId(), maxStudent.getScore());
        System.out.println("最低分学生：");
        System.out.printf("  姓名：%s  学号：%s  成绩：%.1f\n",
                minStudent.getName(), minStudent.getStudentId(), minStudent.getScore());
    }

    private static void showPassFailStudents(Student[] students, int count) {
        if (count == 0) {
            System.out.println("暂无学生数据。");
            return;
        }

        System.out.println("\n========== 及格学生 ==========");
        boolean hasPass = false;
        for (int i = 0; i < count; i++) {
            Student s = students[i];
            if (s.getScore() >= 60) {
                System.out.printf("  %s（%s）：%.1f分\n",
                        s.getName(), s.getStudentId(), s.getScore());
                hasPass = true;
            }
        }
        if (!hasPass) {
            System.out.println("  无");
        }

        System.out.println("\n========== 不及格学生 ==========");
        boolean hasFail = false;
        for (int i = 0; i < count; i++) {
            Student s = students[i];
            if (s.getScore() < 60) {
                System.out.printf("  %s（%s）：%.1f分\n",
                        s.getName(), s.getStudentId(), s.getScore());
                hasFail = true;
            }
        }
        if (!hasFail) {
            System.out.println("  无");
        }
    }

    private static int getActualCount(Student[] students) {
        int c = 0;
        for (Student s : students) {
            if (s != null) {
                c++;
            } else {
                break;
            }
        }
        return c;
    }
}

class Student {
    private String name;
    private String studentId;
    private double score;

    public Student() {
        this("未知", "0000000", 0.0);
    }

    public Student(String name, String studentId, double score) {
        this.name = name;
        this.studentId = studentId;
        this.score = score;
    }

    public String getName() { return name; }

    public void setName(String name) { this.name = name; }

    public String getStudentId() { return studentId; }

    public void setStudentId(String studentId) { this.studentId = studentId; }

    public double getScore() { return score; }

    public void setScore(double score) { this.score = score; }
}
```

### 代码逐块解释

**Student类**（文件后半部分）：这是数据模型，3个字段（name/studentId/score）+ 构造器 + Getter/Setter。你之前学了变量（17章）和类（22章），这正是它们的结合。

**主菜单**：一个 `while(true)` 死循环（20章），用户选0时 `break` 退出。选择用 `switch`（19章）分发到不同功能。

**添加学生**：从 `Scanner` 读用户输入，创建 `new Student(...)` 对象存入数组（21章）。调用前校验成绩在0-100之间。

**显示所有学生**：用 `for` 循环遍历数组，`printf` 格式化输出。用三元运算符（18章）判断及格/不及格。

**统计平均**：用循环累加所有成绩，除以总人数。

**最高最低分**：遍历比较，用if语句（19章）找出最高/最低的学生对象。

**及格/不及格名单**：遍历数组，用if判断每个学生是否合格。

| 类 | 对象的模板/设计图。包含字段（数据）+ 构造器（初始化）+ 方法（行为） |
| 对象 | 用 `new` 创建的类的实例。此术语需进附录 |
| 构造器 | 与类同名、无返回值。`new` 时自动调用。可重载。一旦写了有参构造器，默认无参消失 |
| this | 指代当前对象。区分字段与参数、调用其他构造器、链式调用 |
| 封装 | private字段 + public getter/setter。此术语需进附录 |
| JavaBean | 私有字段 + 无参构造器 + getXxx/setXxx 命名规范的类。框架的基础 |
| static | 静态成员属于类本身，所有对象共享。用 `类名.成员` 访问 |
| 学生成绩管理系统 | 综合运用变量、数组、循环、if判断、类与对象、Scanner输入 |

## 自测题

**1.** 以下代码中，哪个是构造器？

```java
public class Person {
    private String name;
    public void Person() { this.name = "未知"; }
    public Person(String name) { this.name = name; }
    public void setName(String name) { this.name = name; }
}
```

**2.** 写了有参构造器后，如果没有手动写无参构造器，`new 类名()` 还能用吗？为什么？

**3.** 以下代码能否编译通过？如果不能，错在哪里？

```java
public class Demo {
    private int value;
    Demo d = new Demo();
    d.value = 10;
}
```
