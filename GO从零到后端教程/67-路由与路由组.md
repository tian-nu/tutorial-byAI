# 第67章 · 路由与路由组

> "一条高速公路如果没有路牌和分岔，所有车都会撞在一起。Web服务也一样——几百个API端点需要准确地把每个请求导向正确的处理函数。路由就是Gin的'交通指挥中心'，而路由组则是把相关路口归类的'城市分区规划'。"

---

## 67.1 路由注册

在Gin中，"注册路由"就是告诉引擎：当收到"某种方法"请求"某个路径"时，执行"某个处理函数"。

### 八种HTTP方法

Gin为每种HTTP方法提供了对应的注册函数：

```go
func main() {
    r := gin.Default()

    r.GET("/users", func(c *gin.Context) {
        c.String(200, "获取用户列表")
    })

    r.POST("/users", func(c *gin.Context) {
        c.String(201, "创建用户")
    })

    r.PUT("/users/:id", func(c *gin.Context) {
        c.String(200, "全量更新用户")
    })

    r.PATCH("/users/:id", func(c *gin.Context) {
        c.String(200, "部分更新用户")
    })

    r.DELETE("/users/:id", func(c *gin.Context) {
        c.String(204, "")
    })

    r.OPTIONS("/users", func(c *gin.Context) {
        c.String(200, "预检请求")
    })

    r.HEAD("/users", func(c *gin.Context) {
        c.String(200, "")
    })

    r.Run()
}
```

不同的方法 + 相同的路径 = 完全不同的处理逻辑。`GET /users` 和 `POST /users` 是两个独立的路由。

### Any：匹配所有方法

```go
r.Any("/ping", func(c *gin.Context) {
    c.String(200, "pong")
})
```

`Any` 会让 `/ping` 响应 GET、POST、PUT、DELETE……所有HTTP方法。适合做健康检查端点。

### NoRoute：404兜底

```go
r.NoRoute(func(c *gin.Context) {
    c.JSON(404, gin.H{
        "code": 404,
        "msg":  "你要找的页面不存在",
    })
})
```

当请求的路径没有任何路由匹配时，执行这个函数。

### NoMethod：405兜底

```go
r.NoMethod(func(c *gin.Context) {
    c.JSON(405, gin.H{
        "code": 405,
        "msg":  "该方法不被允许",
    })
})
```

当路径存在但没有匹配到对应HTTP方法时执行。比如只注册了 `GET /users`，但客户端发起 `DELETE /users`，就会触发405。

### Handle：手动指定方法

```go
r.Handle("GET", "/hello", func(c *gin.Context) {
    c.String(200, "hello")
})
```

`r.GET("/hello", ...)` 本质上就是 `r.Handle("GET", "/hello", ...)` 的简写。

## 67.2 路径参数

静态路径 `/users/list` 很好匹配，但现实中你需要这样：`/users/1`、`/users/2`、`/users/999`——用**路径参数**来捕获动态部分。

### 基本用法：`:参数名`

```go
r.GET("/users/:id", func(c *gin.Context) {
    id := c.Param("id")
    c.JSON(200, gin.H{
        "user_id": id,
    })
})
```

访问 `http://localhost:8080/users/42`，响应：

```json
{"user_id":"42"}
```

注意：`c.Param("id")` 返回的是**字符串**，即使URL中写的是数字。如果需要整数，要自己转换：

```go
idStr := c.Param("id")
id, err := strconv.Atoi(idStr)
if err != nil {
    c.JSON(400, gin.H{"msg": "ID必须是整数"})
    return
}
```

### 多个路径参数

```go
r.GET("/users/:uid/orders/:oid", func(c *gin.Context) {
    uid := c.Param("uid")
    oid := c.Param("oid")
    c.JSON(200, gin.H{
        "user_id":  uid,
        "order_id": oid,
    })
})
```

访问 `/users/10/orders/88` → `{"user_id":"10","order_id":"88"}`

### 通配符：`*`

```go
r.GET("/files/*filepath", func(c *gin.Context) {
    filepath := c.Param("filepath")
    c.String(200, "请求的文件路径: %s", filepath)
})
```

访问 `/files/images/avatar/photo.jpg`，`filepath` = `/images/avatar/photo.jpg`

**关键区别**：

| 模式 | 示例URL | 匹配 |
|------|---------|------|
| `:id` | `/users/42` | 只匹配一个路径段（`42`），不跨 `/` |
| `*path` | `/files/a/b/c` | 匹配从该位置到URL末尾的**所有内容**（`/a/b/c`） |

`:id` 是"单段通配符"，`*path` 是"多段通配符"。

## 67.3 路由组

当你的API有20个用户相关接口、15个订单相关接口、10个商品相关接口时，所有路由写在一个文件的 `main` 函数里会变得无法维护。路由组让你按**功能模块**把相关路由归类。

### 基本路由组

```go
func main() {
    r := gin.Default()

    userGroup := r.Group("/users")
    {
        userGroup.GET("", listUsers)
        userGroup.GET("/:id", getUserByID)
        userGroup.POST("", createUser)
        userGroup.PUT("/:id", updateUser)
        userGroup.DELETE("/:id", deleteUser)
    }

    orderGroup := r.Group("/orders")
    {
        orderGroup.GET("", listOrders)
        orderGroup.POST("", createOrder)
    }

    r.Run()
}
```

注意：`router.GET("", handler)` 中的空字符串表示路由组的前缀本身（即 `/users`）。

### API版本管理用路由组

```go
v1 := r.Group("/api/v1")
{
    v1.GET("/users", handleV1Users)
    v1.GET("/products", handleV1Products)
}

v2 := r.Group("/api/v2")
{
    v2.GET("/users", handleV2Users)
    v2.GET("/products", handleV2Products)
}
```

访问：
- `/api/v1/users` → `handleV1Users`
- `/api/v2/users` → `handleV2Users`

### 嵌套路由组

路由组可以无限嵌套：

```go
api := r.Group("/api")
{
    v1 := api.Group("/v1")
    {
        admin := v1.Group("/admin")
        {
            admin.GET("/dashboard", adminDashboard)
            admin.GET("/users", adminListUsers)
        }

        public := v1.Group("/public")
        {
            public.GET("/articles", publicArticles)
        }
    }
}
```

最终注册的路由：
- `/api/v1/admin/dashboard`
- `/api/v1/admin/users`
- `/api/v1/public/articles`

### 路由组的中间件

路由组最大的威力在于——**可以给整组路由统一加中间件**：

```go
auth := r.Group("/api")
auth.Use(AuthMiddleware())
{
    auth.GET("/profile", getProfile)
    auth.GET("/settings", getSettings)
    auth.POST("/logout", logout)
}

publicGroup := r.Group("/api")
{
    publicGroup.GET("/articles", listArticles)
    publicGroup.POST("/login", login)
    publicGroup.POST("/register", register)
}
```

同一个 `/api` 前缀下，`/profile`、`/settings`、`/logout` 都需要登录（走了Auth中间件），而 `/articles`、`/login`、`/register` 是公开的（不走Auth中间件）。

## 67.4 路由优先级

当多个路由模式可能匹配同一个URL时，Gin按照**注册顺序**来匹配。但Gin内部又分为两种冲突情况：

### 静态路径 vs 参数路径

```go
r.GET("/users/search", searchUsers)
r.GET("/users/:id", getUserByID)
```

访问 `/users/search` → **走 `searchUsers`**，不走 `getUserByID`。

Gin优先匹配精确的静态路径，然后才尝试匹配参数路径。这是Radix Tree的天然属性——静态节点优先于动态节点。

### 参数路径 vs 通配路径

```go
r.GET("/files/:type", getByType)
r.GET("/files/*filepath", getByFilepath)
```

访问 `/files/image` → 走 `getByType`（`type` = `"image"`）
访问 `/files/a/b/c` → 走 `getByFilepath`（`filepath` = `"/a/b/c"`）

参数路径 `:type` 优先于通配路径 `*filepath`，因为 `:type` 只匹配一个路径段，更具体。

### 冲突警告

```go
r.GET("/users/:name", handler1)
r.GET("/users/:id", handler2)
```

Gin在启动时检测到这两个路由会产生冲突——`/users/xxx` 到底匹配 `:name` 还是 `:id`？Gin会**直接panic**，不会让你带着这种歧义启动服务。解决方法是合并为一个路由或改变路径结构。

## 67.5 路由拆分：模块化组织

当项目有几十上百个路由时，不能把所有路由注册都写在 `main.go` 里。你需要按业务模块拆分路由注册。

### 方案：每个模块一个路由文件

项目结构：

```
main.go
routes/
  ├── user.go
  ├── order.go
  └── product.go
```

`routes/user.go`：

```go
package routes

import (
	"net/http"

	"github.com/gin-gonic/gin"
)

func RegisterUserRoutes(r *gin.RouterGroup) {
	userGroup := r.Group("/users")
	{
		userGroup.GET("", func(c *gin.Context) {
			c.JSON(http.StatusOK, gin.H{"msg": "用户列表"})
		})
		userGroup.GET("/:id", func(c *gin.Context) {
			c.JSON(http.StatusOK, gin.H{"msg": "用户详情"})
		})
		userGroup.POST("", func(c *gin.Context) {
			c.JSON(http.StatusCreated, gin.H{"msg": "创建用户"})
		})
		userGroup.PUT("/:id", func(c *gin.Context) {
			c.JSON(http.StatusOK, gin.H{"msg": "更新用户"})
		})
		userGroup.DELETE("/:id", func(c *gin.Context) {
			c.JSON(http.StatusNoContent, nil)
		})
	}
}
```

`routes/order.go`：

```go
package routes

import (
	"net/http"

	"github.com/gin-gonic/gin"
)

func RegisterOrderRoutes(r *gin.RouterGroup) {
	orderGroup := r.Group("/orders")
	{
		orderGroup.GET("", func(c *gin.Context) {
			c.JSON(http.StatusOK, gin.H{"msg": "订单列表"})
		})
		orderGroup.POST("", func(c *gin.Context) {
			c.JSON(http.StatusCreated, gin.H{"msg": "创建订单"})
		})
	}
}
```

`main.go`：

```go
package main

import (
	"github.com/gin-gonic/gin"
	"myweb/routes"
)

func main() {
	r := gin.Default()

	api := r.Group("/api/v1")
	{
		routes.RegisterUserRoutes(api)
		routes.RegisterOrderRoutes(api)
	}

	r.Run()
}
```

每次新增业务模块，只需新建一个 `routes/xxx.go` 文件并在 `main.go` 中调用注册函数。核心路由文件保持清爽。

### 进阶：给路由组传依赖

随着项目变大，handler需要访问数据库、缓存、配置。这时传入依赖：

```go
func RegisterUserRoutes(r *gin.RouterGroup, db *sql.DB, cache *redis.Client) { // redis.Client详见第60章
    userGroup := r.Group("/users")
    {
        userGroup.GET("", func(c *gin.Context) {
            rows, _ := db.Query("SELECT id, name FROM users")
            ...
        })
    }
}
```

更优雅的做法是定义一个结构体来承载依赖（详见第72章"项目结构设计"）。

## 🤔 想多一点：为什么不用 `http.HandleFunc` 的模式？

标准库的路由写法：

```go
http.HandleFunc("/users/", usersHandler)
```

有两个致命问题：

### 1. 最长前缀匹配

标准库的 `DefaultServeMux` 使用最长前缀匹配。如果你注册了 `/users/`，那么 `/users/123` 也会匹配到 `/users/` 的handler——但它不会告诉你URL里还有 `123` 这个信息。你得自己在handler里手动切割URL字符串来提取ID。

Gin的 `:id` 语法直接解决了这个问题——声明式地表达"这个位置是一个参数"。

### 2. 不支持方法区分

标准库的 `http.HandleFunc` 不区分GET和POST。你需要在handler内部判断 `r.Method == "GET"` 来分流：

```go
func handler(w http.ResponseWriter, r *http.Request) {
    if r.Method == "GET" {
        // ...
    } else if r.Method == "POST" {
        // ...
    }
}
```

Gin直接在注册时就按方法分流，不需要在handler里写 `if`。

## 本章小结

| 知识点 | 要点 |
|--------|------|
| 路由注册 | `r.GET/POST/PUT/PATCH/DELETE/OPTIONS/HEAD(path, handler)` |
| Any | `r.Any(path, handler)` 匹配所有HTTP方法 |
| NoRoute | `r.NoRoute(handler)` 处理404 |
| NoMethod | `r.NoMethod(handler)` 处理405 |
| 路径参数 | `:name` 单段通配，`*name` 多段通配 |
| c.Param | 获取路径参数的值（返回string） |
| 路由组 | `r.Group(prefix)` 给一组路由统一前缀和中间件 |
| 嵌套路由组 | 路由组无限嵌套，逐级拼接路径 |
| 中间件分层 | 公开路由和鉴权路由用不同的路由组区分 |
| 路由优先级 | 静态路径 > `:param` > `*wildcard` |
| 模块拆分 | 每业务模块一个文件，函数签名 `RegisterXxxRoutes(r *gin.RouterGroup)` |
